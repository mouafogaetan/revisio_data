Implémenter les endpoints CRUD
avec Entity Framework Core

Créez des opérations Create, Read, Update, Delete sur votre base de données avec une API ASP.NET Core moderne.

🎯 Niveau : Intermédiaire / Avancé ⏱️ Durée : 50 min 📦 EF Core + SQL

📌 Objectifs d'apprentissage

À la fin de cette leçon, vous serez capable de :

  • ✓ Créer un modèle de données (entités) et un DbContext pour EF Core.
  • ✓ Utiliser les migrations pour synchroniser le schéma de base de données.
  • ✓ Implémenter les opérations GET (list, single), POST, PUT, DELETE dans un contrôleur.
  • ✓ Tester les endpoints avec Swagger et comprendre les codes HTTP.

💡 Pourquoi CRUD ? Ce sont les opérations de base de toute API métier. Bien les maîtriser permet de construire n'importe quel backend.

🏗️ Création du modèle et du DbContext

Commençons par une entité Product et son DbContext.

// Models/Product.cs
public class Product
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public decimal Price { get; set; }
    public int Stock { get; set; }
}

// Data/AppDbContext.cs
using Microsoft.EntityFrameworkCore;

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }
    public DbSet<Product> Products { get; set; }
}

Ensuite, enregistrez le contexte dans Program.cs (par exemple avec SQL Server) :

builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));
📦 Packages nécessaires : Microsoft.EntityFrameworkCore.SqlServer, Microsoft.EntityFrameworkCore.Tools (ou InMemory pour tests).

🔄 Migrations Entity Framework Core

Les migrations permettent de créer et mettre à jour la base de données à partir des modèles C#.

# Installer les outils (une fois)
dotnet tool install --global dotnet-ef

# Créer une migration
dotnet ef migrations add InitialCreate

# Appliquer la migration à la base
dotnet ef database update

✅ Bonne pratique :

Versionnez vos dossiers Migrations dans Git. En production, utilisez dotnet ef database update ou des migrations au démarrage.

💡 Alternative Code First : après modification du modèle, créez une nouvelle migration (AddMigration UpdateSchema) puis appliquez-la.

📋 GET : Récupérer les ressources

Deux endpoints classiques : récupérer tous les produits, et un produit par identifiant.

[ApiController]
[Route("api/[controller]")]
public class ProductsController : ControllerBase
{
    private readonly AppDbContext _context;
    public ProductsController(AppDbContext context) => _context = context;

    // GET: api/products
    [HttpGet]
    public async Task<IActionResult> GetAll()
    {
        var products = await _context.Products.ToListAsync();
        return Ok(products);
    }

    // GET: api/products/5
    [HttpGet("{id}")]
    public async Task<IActionResult> GetById(int id)
    {
        var product = await _context.Products.FindAsync(id);
        if (product == null) return NotFound();
        return Ok(product);
    }
}

📌 FindAsync(id) recherche par clé primaire. ToListAsync() exécute la requête de manière asynchrone.

➕ POST : Créer une nouvelle ressource

Ajoutons un endpoint pour créer un produit. On utilise un DTO (Data Transfer Object) pour éviter de lier directement l’entité.

// DTOs/CreateProductDto.cs
public class CreateProductDto
{
    [Required]
    public string Name { get; set; }
    [Range(0.01, 10000)]
    public decimal Price { get; set; }
    public int Stock { get; set; }
}

// Dans le contrôleur
[HttpPost]
public async Task<IActionResult> Create([FromBody] CreateProductDto dto)
{
    if (!ModelState.IsValid) return BadRequest(ModelState);
    
    var product = new Product
    {
        Name = dto.Name,
        Price = dto.Price,
        Stock = dto.Stock
    };
    
    _context.Products.Add(product);
    await _context.SaveChangesAsync();
    
    return CreatedAtAction(nameof(GetById), new { id = product.Id }, product);
}

🎯 Détail important :

CreatedAtAction retourne le code 201 (Created) avec l’URL de localisation et l’objet créé.

🖊️ PUT : Mettre à jour une ressource existante

Le remplacement complet (PUT) nécessite l’envoi de toutes les propriétés. On utilise un DTO dédié.

// DTOs/UpdateProductDto.cs
public class UpdateProductDto
{
    [Required]
    public string Name { get; set; }
    public decimal Price { get; set; }
    public int Stock { get; set; }
}

[HttpPut("{id}")]
public async Task<IActionResult> Update(int id, [FromBody] UpdateProductDto dto)
{
    var product = await _context.Products.FindAsync(id);
    if (product == null) return NotFound();
    
    product.Name = dto.Name;
    product.Price = dto.Price;
    product.Stock = dto.Stock;
    
    _context.Products.Update(product);
    await _context.SaveChangesAsync();
    
    return NoContent(); // 204
}

⚠️ PUT vs PATCH :

PUT remplace TOUTE la ressource ; PATCH modifie partiellement. Dans cet exemple nous utilisons PUT complet.

🗑️ DELETE : Supprimer une ressource

Opération simple : rechercher l’entité, la retirer du contexte, puis sauvegarder.

[HttpDelete("{id}")]
public async Task<IActionResult> Delete(int id)
{
    var product = await _context.Products.FindAsync(id);
    if (product == null) return NotFound();
    
    _context.Products.Remove(product);
    await _context.SaveChangesAsync();
    
    return NoContent();
}

🔒 Bonne pratique :

Pour les ressources sensibles, envisagez un "soft delete" (ajouter une colonne IsDeleted) au lieu d’une suppression physique définitive.

🧪 Tester son API CRUD (Swagger) & Conseils

Une fois l’API lancée, ouvrez Swagger UI. Vous pouvez :

  • Effectuer un POST avec un corps JSON, vérifier la création.
  • Appeler GET /{id} pour récupérer la ressource.
  • Lancer PUT pour modifier les données, puis DELETE pour supprimer.
  • Tester les codes d’erreur (404 si ID inexistant).

✨ Liste des bonnes pratiques CRUD :

  • Utiliser des DTO pour éviter l’exposition directe des entités.
  • Retourner les codes HTTP appropriés (200, 201, 204, 400, 404).
  • Implémenter la validation (Data Annotations + [ApiController]).
  • Utiliser les méthodes asynchrones (async/await) pour ne pas bloquer les threads.
  • Ajouter des logs (ILogger) pour tracer les modifications critiques.

2️⃣ Add est synchrone (ne pas utiliser en async classique), AddAsync est asynchrone pour les cas où la génération de clé nécessite un appel DB.
