Sécurisation JWT & Authentification

API .NET (ASP.NET Core) – Durée estimée : 1h30  |  Barème /20

📌 Exercice 1 – QCM (4 points)

4 pts

1.1 Quel package NuGet est indispensable pour activer l'authentification JWT dans une API .NET Core ?

Réponse : Microsoft.AspNetCore.Authentication.JwtBearer. Ce package fournit les middlewares et extensions nécessaires (AddJwtBearer, UseAuthentication).

1.2 Quelle est la durée de vie recommandée pour un Access Token JWT ?

Réponse : Courte (15 min à quelques heures). Les Access Tokens sont éphémères pour limiter les risques. On utilise un Refresh Token pour renouveler.

1.3 Pour hacher un mot de passe en C# avant de le stocker, quelle bibliothèque est largement recommandée ?

BCrypt.Net-Next (ou BCrypt) est robuste, résistant aux attaques par force brute (salage + coût adaptatif).

1.4 Quel attribut place un contrôleur ou une méthode hors d'accès sans token valide ?

[Authorize] impose une authentification valide. [AllowAnonymous] laisse l'accès libre.

📝 Éléments de correction

  • QCM : 1.1 JwtBearer / 1.2 Courte durée / 1.3 BCrypt.Net-Next / 1.4 [Authorize]
  • Refresh token : Permet d'obtenir un nouvel Access Token sans reconnexion, stocké côté serveur/httponly cookie.
  • Extension : app.UseAuthentication(); avant UseAuthorization.
  • LocalStorage : vulnérable aux XSS, un attaquant peut voler le token.
  • Claims : new Claim(ClaimTypes.NameIdentifier, user.Id.ToString()) ...
  • Politique personnalisée : options.AddPolicy("MinimumAge", policy => policy.RequireClaim("Age", "18","19"...)) ou RequireAssertion.
