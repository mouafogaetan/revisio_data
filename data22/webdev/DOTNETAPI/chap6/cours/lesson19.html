Générer un token JWT

Leçon 19 : Créer un endpoint de connexion et générer des tokens sécurisés

Cours pour développeurs Cameroun | Formation API .NET

Objectifs de la leçon

  • ✓ Créer un endpoint /api/auth/login pour l'authentification.
  • ✓ Générer un token JWT avec SecurityTokenDescriptor et les classes JwtSecurityTokenHandler.
  • ✓ Inclure des claims personnalisés : rôles, userId, email.
  • ✓ Comprendre la structure du token (header, payload, signature).

🔐 Endpoint /api/auth/login

L'utilisateur envoie ses identifiants (email/mot de passe). Après vérification, on génère un token JWT et on le retourne au client.

POST /api/auth/login
Content-Type: application/json

{
  "email": "jean@exemple.cm",
  "password": "MonPass123"
}

➡️ Réponse attendue : { "token": "eyJhbGciOiJIUzI1NiIs...", "expiresIn": 3600 }

💡 Au Cameroun comme ailleurs, le JWT permet une authentification sans état (stateless) idéale pour les APIs mobiles ou web modernes.

⚙️ Générer un JWT en C#

Utilisation de System.IdentityModel.Tokens.Jwt et des classes JwtSecurityTokenHandler, SecurityTokenDescriptor.

// Extrait typique dans un service d'authentification
var tokenHandler = new JwtSecurityTokenHandler();
var key = Encoding.ASCII.GetBytes(secretKey);

var tokenDescriptor = new SecurityTokenDescriptor
{
    Subject = new ClaimsIdentity(new[]
    {
        new Claim(ClaimTypes.NameIdentifier, user.Id),
        new Claim(ClaimTypes.Email, user.Email),
        new Claim(ClaimTypes.Role, user.Role)
    }),
    Expires = DateTime.UtcNow.AddHours(2),
    Issuer = "monApiDotNet",
    Audience = "clientsWeb",
    SigningCredentials = new SigningCredentials(
        new SymmetricSecurityKey(key),
        SecurityAlgorithms.HmacSha256Signature)
};

var token = tokenHandler.CreateToken(tokenDescriptor);
string jwtString = tokenHandler.WriteToken(token);
return jwtString;

📌 Explication : On définit les claims (identité), la durée de vie, l'émetteur et la clé de signature.

📋 Les Claims : le cœur du token

Les claims sont des affirmations sur l'utilisateur. Ils sont encodés dans le payload du token.

ClaimTypes.NameIdentifier

Identifiant unique de l'utilisateur (userId)

ClaimTypes.Email

Adresse email pour identification

ClaimTypes.Role

Rôle (Admin, Utilisateur, etc.) pour autorisations

Custom "entreprise"

Vous pouvez ajouter vos propres claims métier

✨ Exemple de payload après décodage :
{"nameid":"42","email":"admin@exemple.cm","role":"Superviseur","exp":1712345678}

🔧 Implémentation pratique du endpoint

Dans votre contrôleur AuthController :

[HttpPost("login")]
public async Task<IActionResult> Login([FromBody] LoginRequest request)
{
    // 1. Vérifier utilisateur en base (email + mot de passe hashé)
    var user = await _userService.AuthenticateAsync(request.Email, request.Password);
    if (user == null) return Unauthorized(new { message = "Email ou mot de passe incorrect" });
    
    // 2. Générer token avec les claims
    var token = _jwtUtils.GenerateToken(user);
    
    return Ok(new { 
        token = token,
        username = user.Nom,
        expiresIn = 7200 // secondes
    });
}

La méthode GenerateToken utilise la logique vue précédemment (SecurityTokenDescriptor).

🧠 Notez bien: le mot de passe n'est jamais inclus dans le token JWT, seulement des informations non sensibles.

🛡️ Bonnes pratiques - Tokens JWT

  • ✅ Utiliser une clé secrète longue et complexe (stockée dans appsettings.json + variables d'environnement).
  • ✅ Définir une expiration raisonnable (1 à 24 heures). Pour plus de sécurité, implémenter un refresh token.
  • ✅ Toujours utiliser HTTPS en production pour éviter l'interception du token.
  • ✅ Ne jamais exposer les informations sensibles dans le payload (pas de mots de passe, ni numéro CB).
  • ✅ Vérifier l'émetteur (issuer) et l'audience (audience) lors de la validation.

🧪 Exercice pratique

Objectif : Créer une mini API avec endpoint /login retournant un token contenant le nom et le rôle "apprenant".

📌 Consignes :

  1. Ajouter le package Microsoft.AspNetCore.Authentication.JwtBearer.
  2. Configurer la clé secrète dans Program.cs.
  3. Simuler une liste d'utilisateurs en mémoire et vérifier le mot de passe.
  4. Générer le token avec une expiration de 1 heure et les claims voulus.
  5. Tester avec Postman ou Swagger.
💬 Défi supplémentaire : Ajouter un claim "école" avec la valeur "Université de Douala / Yaoundé" et lire le token sur jwt.io.

✨ Récapitulatif

Un endpoint /login valide les identifiants et génère un token JWT à l'aide de SecurityTokenDescriptor. Les claims (userId, rôles, email) sont intégrés dans le token pour autorisations ultérieures.
