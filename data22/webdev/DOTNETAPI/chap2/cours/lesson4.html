Créer un contrôleur API REST
avec ASP.NET Core

Décorateurs, routage, héritage et conventions REST : les bases d'un endpoint professionnel.

🎯 Niveau : Débutant / Intermédiaire ⏱️ Durée : 40 min 🌐 ASP.NET Core 8

📌 Objectifs d'apprentissage

À la fin de cette leçon, vous serez capable de :

  • ✓ Décorer une classe avec [ApiController] et [Route].
  • ✓ Hériter correctement de ControllerBase (et non Controller).
  • ✓ Appliquer les conventions de nommage REST (ex: ProductsController).
  • ✓ Comprendre pourquoi ControllerBase est préféré pour les API.

💡 Pourquoi c'est essentiel ? Un contrôleur bien structuré est la pierre angulaire d'une API REST maintenable, testable et conforme aux standards du web.

🏷️ Le décorateur [ApiController]

L'attribut [ApiController] active des comportements spécifiques aux API web :

  • Validation automatique des modèles : renvoie une erreur 400 si les attributs de validation échouent.
  • Inférence des paramètres : lie automatiquement les paramètres simples depuis la route, le corps ou la requête.
  • Réponse détaillée des erreurs (Problem Details).
  • Attribut [FromBody] implicite pour les types complexes.
using Microsoft.AspNetCore.Mvc;

[ApiController]   // ✅ Active les fonctionnalités API
public class ProductsController : ControllerBase
{
    // ... endpoints
}
📌 À retenir : Sans [ApiController], la validation et la liaison de modèles seront moins intelligentes. Toujours l'ajouter pour une API REST.

🗺️ Définir la route de base avec [Route]

L'attribut [Route] définit le préfixe d'URL pour tous les endpoints du contrôleur. La convention REST utilise le nom de la ressource au pluriel.

[ApiController]
[Route("api/[controller]")]   // 🧩 api/products
public class ProductsController : ControllerBase
{
}

[controller] est un token remplacé par le nom du contrôleur sans le suffixe "Controller".
Exemple : ProductsController → products.

✅ Bonnes pratiques REST :

  • Utiliser des noms de ressources en minuscules et au pluriel (/api/products, /api/users).
  • Éviter les verbes dans l'URL (utiliser les méthodes HTTP).
  • Structurer les routes hiérarchiques (/api/products/5/reviews).

🎯 Pourquoi hériter de ControllerBase ?

ControllerBase

Optimisé pour les API web. Fournit des méthodes utilitaires comme Ok(), BadRequest(), NotFound(), CreatedAtAction().

✅ Pas de support des vues (Razor) → plus léger.

Controller (hérité)

Inclut le support des vues (MVC) et des méthodes comme View(). Lourd pour une API pure.

⚠️ À éviter dans une API Web.

// ✅ Bonne pratique pour une API REST
[ApiController]
[Route("api/[controller]")]
public class ProductsController : ControllerBase
{
    [HttpGet]
    public IActionResult GetProducts() => Ok(new[] { "Produit A", "Produit B" });
}

💡 Rappel : ControllerBase expose Ok(), NotFound(), BadRequest(), Created()… tout ce dont une API a besoin.

📄 Exemple concret : contrôleur Product minimaliste

Voici un contrôleur simple avec deux actions : récupérer tous les produits et en créer un nouveau.

using Microsoft.AspNetCore.Mvc;

namespace MyApi.Controllers;

[ApiController]
[Route("api/[controller]")]
public class ProductsController : ControllerBase
{
    private static List<string> _products = new() { "Livre", "Stylo" };

    // GET: api/products
    [HttpGet]
    public IActionResult GetAll()
    {
        return Ok(_products);
    }

    // POST: api/products
    [HttpPost]
    public IActionResult Create([FromBody] string productName)
    {
        _products.Add(productName);
        return CreatedAtAction(nameof(GetAll), new { }, productName);
    }
}

🎯 Analyse : [HttpGet] sans paramètre répond sur la route de base. [HttpPost] ajoute un produit. CreatedAtAction retourne le code 201 + l'emplacement de la ressource.

✨ Bonnes pratiques & erreurs fréquentes

  • ✔️ Utilisez toujours [ApiController] : vous bénéficiez de la validation et de la liaison améliorées.
  • ✔️ Nommez votre contrôleur avec le suffixe "Controller" : convention découverte automatique.
  • ✔️ Préférez les routes explicites : [Route("api/products")] plutôt que [controller] si vous souhaitez contrôler précisément l'URL.
  • ❌ Évitez d'hériter de Controller pour une API, vous ajoutez des dépendances inutiles.
  • ❌ Ne pas oublier l'injection de dépendances : le contrôleur doit rester mince, déléguer la logique à des services.

⚠️ Piège classique :

Oublier l'attribut [ApiController] et se demander pourquoi les erreurs de validation ne sont pas automatiquement renvoyées.
