Pourquoi Docker ?

Docker permet d'encapsuler votre API avec son environnement (runtime, dépendances, configuration). Fini les “ça marche sur ma machine” !

  • Isolation : chaque service tourne dans son propre conteneur.
  • Portabilité : déployez sur Linux, Windows, cloud (Azure, AWS, GCP).
  • Reproductibilité : l'image contient exactement ce qu'il faut.
  • Scalabilité : on peut multiplier les instances facilement.

Astuce : Un conteneur est comme une "boîte légère" qui contient votre application et ses librairies, partageant le noyau de l'hôte.

Le Dockerfile pour API .NET

Un Dockerfile contient les instructions pour construire l'image. Voici un exemple multi-stage (build + runtime).

# Étape 1 : build
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY ["MonApi.csproj", "."]
RUN dotnet restore
COPY . .
RUN dotnet publish -c Release -o /app/publish

# Étape 2 : runtime
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS runtime
WORKDIR /app
COPY --from=build /app/publish .
ENTRYPOINT ["dotnet", "MonApi.dll"]
📌 FROM – Image de base (SDK pour compiler, aspnet pour exécuter)
⚙️ WORKDIR – Répertoire de travail
📦 COPY / RUN – Copier les fichiers et restaurer les packages
🚀 ENTRYPOINT – Commande de lancement de l’API

Docker Compose : API + Base de données

Avec Compose, on définit plusieurs services interconnectés dans un fichier docker-compose.yml.

version: '3.8'
services:
  sqlserver:
    image: mcr.microsoft.com/mssql/server:2022-latest
    environment:
      SA_PASSWORD: "Your_password123!"
      ACCEPT_EULA: "Y"
    ports:
      - "1433:1433"
  api:
    build: .
    ports:
      - "5000:8080"
    environment:
      - ConnectionStrings__Default=Server=sqlserver;Database=AppDb;User=sa;Password=Your_password123!;TrustServerCertificate=true
    depends_on:
      - sqlserver
Cela lance SQL Server et l'API, et la chaîne de connexion utilise le nom du service "sqlserver".

Déployer en production (Cloud)

Azure Container Instances (ACI)

Service serverless : poussez votre image sur Azure Container Registry (ou Docker Hub).

az container create --resource-group monGroupe --name monapi --image monregistry.azurecr.io/monapi:v1 --ports 80

AWS ECS (Elastic Container Service)

Utilisez ECS Fargate (sans gestion de serveurs). Définissez une tâche avec votre image et un équilibreur de charge.

Bon à savoir : Avant le déploiement cloud, construisez l'image et poussez-la sur un registre (Docker Hub, ACR, ECR).

Commandes Docker essentielles

docker build -t monapi .
Construire l'image
docker run -d -p 5000:8080 --name api monapi
Lancer un conteneur
docker-compose up -d
Démarrer les services (API + SQL)
docker logs api

Récapitulons et mettons en pratique !

À retenir :

  • Un Dockerfile contient les étapes pour fabriquer une image immuable de votre API.
  • Docker Compose facilite le développement multi-conteneurs (API + base de données).
  • Les clouds modernes (ACI, ECS) acceptent les images Docker et les orchestrent.
  • Toujours utiliser des variables d’environnement pour les secrets (connexion DB, clés).
