Sécurité des transmissions

🔐 HTTPS : non négociable

Pourquoi HTTPS est obligatoire pour les paiements ?

  • Chiffrement des données entre le navigateur et le serveur
  • Protection contre les attaques Man-in-the-Middle (MITM)
  • Authentification du serveur (certificat SSL/TLS)
  • Exigé par les normes PCI DSS
  • Requis par Stripe, PayPal et tous les prestataires de paiement
// Configuration HTTPS dans NestJS avec Express
import * as fs from 'fs';
import * as https from 'https';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  
  // Option 1: Utiliser un reverse proxy (Nginx) - Recommandé
  app.enable('trust proxy');
  
  // Option 2: HTTPS direct (pour test seulement)
  if (process.env.NODE_ENV === 'production') {
    const httpsOptions = {
      key: fs.readFileSync('./secrets/private-key.pem'),
      cert: fs.readFileSync('./secrets/public-certificate.pem'),
    };
    await app.listen(443, '0.0.0.0', { httpsOptions });
  } else {
    await app.listen(3000);
  }
}
⚠️ Important : En production, HTTPS doit être géré par un reverse proxy (Nginx, Traefik) plutôt que par l'application Node.js.
Normes de sécurité

📋 PCI DSS - Les 6 objectifs

1. Construire un réseau sécurisé
2. Protéger les données des titulaires
3. Gérer les vulnérabilités
4. Contrôler l'accès aux données
5. Surveiller et tester les réseaux
6. Maintenir une politique de sécurité

🚨 Règle d'or : Ne jamais stocker les données sensibles

Interdiction de stocker : numéro de carte complet, code CVV, piste magnétique. Utilisez la tokenisation !

Protection des données

🎫 Tokenisation des cartes bancaires

La tokenisation remplace les données sensibles par un token réutilisable mais non réversible.

// Exemple avec Stripe (côté client - frontend)
const stripe = Stripe('pk_test_xxx');

// Créer un token à partir des données de carte
const { token } = await stripe.createToken({
  card: {
    number: '4242424242424242',
    exp_month: 12,
    exp_year: 2025,
    cvc: '123'
  }
});

// Envoyer SEULEMENT le token au backend
fetch('/api/payments/charge', {
  method: 'POST',
  body: JSON.stringify({ token: token.id, amount: 5000 })
});

// Côté backend NestJS
@Post('charge')
async charge(@Body() body: { token: string; amount: number }) {
  // Stripe ne voit jamais les données de carte brutes
  const paymentIntent = await this.stripe.paymentIntents.create({
    amount: body.amount,
    currency: 'xaf',
    payment_method_data: {
      type: 'card',
      card: { token: body.token }
    }
  });
  
  // Stocker uniquement l'identifiant de paiement, pas la carte
  await this.paymentRepository.save({
    stripePaymentIntentId: paymentIntent.id,
    amount: body.amount,
    status: 'pending'
  });
}
Authentification forte

🛡️ 3D Secure (3DS v2)

Avantages :

  • Authentification à deux facteurs
  • Transfert de responsabilité (fraude → banque)
  • Réduction des rétrofacturations
  • Expérience utilisateur fluide (biométrie)

Flux 3DS :

  • 1. Client initie le paiement
  • 2. Stripe/PayPal demande authentification
  • 3. Client s'authentifie (SMS, app bancaire)
  • 4. Paiement confirmé
// NestJS - Stripe avec 3D Secure
async createPaymentIntent(amount: number) {
  const paymentIntent = await this.stripe.paymentIntents.create({
    amount,
    currency: 'xaf',
    payment_method_types: ['card'],
    // Forcer 3D Secure
    payment_method_options: {
      card: {
        request_three_d_secure: 'any' // 'any' ou 'automatic'
      }
    }
  });
  
  return {
    clientSecret: paymentIntent.client_secret,
    requiresAction: paymentIntent.status === 'requires_action'
  };
}

// Webhook pour confirmer après 3DS
@Post('webhook')
async handleWebhook(@Req() req: Request) {
  const event = this.stripe.webhooks.constructEvent(
    req.body,
    req.headers['stripe-signature'],
    process.env.STRIPE_WEBHOOK_SECRET
  );
  
  if (event.type === 'payment_intent.succeeded') {
    await this.orderService.confirmOrder(event.data.object.metadata.orderId);
  }
}
