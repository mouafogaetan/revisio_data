📘 Qu'est-ce que Swagger / OpenAPI ?

OpenAPI (anciennement Swagger) est une spécification pour décrire une API REST. NestJS fournit le module @nestjs/swagger qui génère automatiquement cette documentation à partir des décorateurs TypeScript.

✅ Avantages :
  • Documentation interactive (Swagger UI)
  • Tests d'API directement depuis le navigateur
  • Génération de clients SDK
  • Respect des standards
🔧 Installation :
npm install --save @nestjs/swagger swagger-ui-express

⚙️ Configuration initiale (main.ts)

import { NestFactory } from '@nestjs/core';
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  const config = new DocumentBuilder()
    .setTitle('API E-commerce')
    .setDescription('Documentation de notre plateforme e-commerce')
    .setVersion('1.0')
    .addTag('products')
    .addTag('auth')
    .addBearerAuth() // pour JWT
    .build();

  const document = SwaggerModule.createDocument(app, config);
  SwaggerModule.setup('api/docs', app, document);

  await app.listen(3000);
}
bootstrap();

🔗 Rendez-vous sur http://localhost:3000/api/docs pour voir l'interface.

🏷️ @ApiTags & @ApiOperation

Organiser les contrôleurs et décrire chaque endpoint.

import { Controller, Get } from '@nestjs/common';
import { ApiTags, ApiOperation } from '@nestjs/swagger';

@ApiTags('products')  // groupe "products" dans Swagger
@Controller('products')
export class ProductsController {
  
  @Get()
  @ApiOperation({ summary: 'Récupérer tous les produits', description: 'Retourne la liste paginée des produits' })
  findAll() {
    return [];
  }

  @Get(':id')
  @ApiOperation({ summary: 'Obtenir un produit par ID' })
  findOne() {
    return {};
  }
}
@ApiTags() = regroupement @ApiOperation() = description

📦 Documenter les DTO (CreateProductDto)

import { ApiProperty } from '@nestjs/swagger';

export class CreateProductDto {
  @ApiProperty({ example: 'Smartphone X', description: 'Nom du produit' })
  name: string;

  @ApiProperty({ example: 599.99, description: 'Prix en euros' })
  price: number;

  @ApiProperty({ required: false, example: 'Smartphone dernière génération' })
  description?: string;

  @ApiProperty({ enum: ['electronics', 'clothing', 'books'], example: 'electronics' })
  category: string;
}

✨ Swagger affichera automatiquement les types, exemples, et champs obligatoires.

💡 Astuce : Utilisez @ApiPropertyOptional() pour les champs facultatifs.

🔐 Ajouter l'authentification JWT

Swagger permet de tester les endpoints protégés en ajoutant un bouton "Authorize".

// Dans main.ts, lors de la construction du document
const config = new DocumentBuilder()
  ...
  .addBearerAuth(
    {
      type: 'http',
      scheme: 'bearer',
      bearerFormat: 'JWT',
    },
    'access-token', // nom du security scheme
  )
  .build();

// Dans un contrôleur protégé :
@ApiBearerAuth('access-token')
@UseGuards(JwtAuthGuard)
@Get('profile')
getProfile() { ... }
✅ Après cette configuration, un cadenas apparaît sur Swagger UI. L'utilisateur peut coller son token JWT et tester les routes sécurisées.

Exemple : AuthController documenté

import { Controller, Post, Body } from '@nestjs/common';
import { ApiTags, ApiOperation, ApiResponse } from '@nestjs/swagger';
import { LoginDto } from './dto/login.dto';
import { AuthService } from './auth.service';

@ApiTags('auth')
@Controller('auth')
export class AuthController {
  constructor(private authService: AuthService) {}

  @Post('login')
  @ApiOperation({ summary: 'Connexion utilisateur' })
  @ApiResponse({ status: 200, description: 'Token JWT retourné' })
  @ApiResponse({ status: 401, description: 'Identifiants invalides' })
  async login(@Body() loginDto: LoginDto) {
    return this.authService.login(loginDto);
  }
}

📌 @ApiResponse permet de décrire les codes HTTP possibles.

✅ Bonnes pratiques de documentation

  • Utiliser @ApiProperty({ example: ... }) pour guider les développeurs
  • Grouper les endpoints par domaine avec @ApiTags()
  • Documenter les codes d'erreur avec @ApiResponse
  • Ajouter une description globale dans DocumentBuilder
  • Ne pas oublier les query params : @ApiQuery({ name: 'page', required: false })
  • Pour les fichiers : @ApiConsumes('multipart/form-data')

"Une documentation claire réduit les allers-retours entre équipes backend et frontend"
