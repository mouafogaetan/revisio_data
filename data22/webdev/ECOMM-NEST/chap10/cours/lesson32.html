📦 Entité Coupon (basique)

// coupon.entity.ts
export enum CouponType { PERCENT = 'percent', FIXED = 'fixed' }

@Entity()
export class Coupon {
  @PrimaryGeneratedColumn() id: number;
  @Column({ unique: true }) code: string; // ex: "PROMO20"
  @Column({ type: 'enum', enum: CouponType }) type: CouponType;
  @Column('decimal') value: number; // 20 pour 20% ou 5000 FCFA
  @Column({ nullable: true }) validUntil: Date;
  @Column({ default: true }) isActive: boolean;
  @Column({ default: 0 }) usedCount: number; // utilisations totales
  @Column({ nullable: true }) maxUses: number; // limitation globale
}

💡 Type percent : valeur = pourcentage de réduction (ex: 20 → -20%). Type fixed : montant déduit (ex: 5000 FCFA).

🎯 Ajouter des conditions

On enrichit l’entité Coupon avec des relations optionnelles :

// Ajout dans l'entité Coupon
  @Column({ nullable: true, type: 'decimal' }) minOrderAmount: number;
  @ManyToMany(() => Product) // coupons applicables à certains produits
  @JoinTable() applicableProducts: Product[];
  @ManyToMany(() => User) // coupons réservés à certains utilisateurs
  @JoinTable() applicableUsers: User[];
  @Column({ default: 0 }) perUserLimit: number; // max utilisations par user

✨ Exemple concret :

  • Code "NOEL2024" : réduction 15%, valable jusqu’au 31/12, montant minimum 20000 FCFA.
  • Code "FIDELITE" : réduction fixe de 3000 FCFA, uniquement pour les utilisateurs ayant déjà commandé (liste blanche).

🛡️ Limiter les utilisations

Deux niveaux de limitation :

maxUses : nombre total d’utilisations du coupon (ex: 100).
perUserLimit : combien de fois un même utilisateur peut l’utiliser (ex: 1).

Pour tracer les utilisations par utilisateur, on crée une table pivot :

@Entity()
export class CouponUsage {
  @ManyToOne(() => Coupon) coupon: Coupon;
  @ManyToOne(() => User) user: User;
  @ManyToOne(() => Order) order: Order;
  @CreateDateColumn() usedAt: Date;
}

📌 Avant d’appliquer un coupon, on vérifie : usedCount < maxUses et countUsageByUser(userId) < perUserLimit.

🔍 Vérifier et appliquer un coupon

Service CouponService avec méthode validateAndApply :

async validateCoupon(code: string, cartTotal: number, userId: number, productIds?: number[]) {
  const coupon = await this.couponRepo.findOne({
    where: { code, isActive: true },
    relations: ['applicableProducts', 'applicableUsers']
  });
  if (!coupon) throw new NotFoundException('Code invalide');
  if (coupon.validUntil && coupon.validUntil < new Date())
    throw new BadRequestException('Coupon expiré');
  if (coupon.maxUses && coupon.usedCount >= coupon.maxUses)
    throw new BadRequestException('Coupon déjà utilisé trop de fois');
  // Vérifier perUserLimit via table CouponUsage
  // Vérifier minOrderAmount, produits applicables, etc.
  // Calculer la réduction
  let reduction = coupon.type === 'percent' ? cartTotal * coupon.value / 100 : coupon.value;
  return { coupon, reduction };
}

📌 La réduction ne peut pas dépasser le total du panier (éviter les montants négatifs).

🔧 Endpoints d’administration

  • POST /admin/coupons : création d’un coupon (authentification admin requise).
  • GET /admin/coupons : liste paginée avec filtres (actif/expiré).
  • PATCH /admin/coupons/:id : modification (changer valeur, prolonger validUntil, désactiver).
  • DELETE /admin/coupons/:id : suppression (ou soft delete).

✨ Bonnes pratiques :

  • Générer des codes uniques (ex: uuidv4() ou crypto.randomBytes pour les campagnes email).
  • Ne jamais exposer les coupons non valides ou expirés dans la réponse API.
  • Journaliser les tentatives d’utilisation frauduleuses (rate limiting, tentative avec codes erronés).
