Diagramme Entité-Association

Créez le modèle relationnel d'une plateforme e-commerce complète

📊 Entités 🔗 Relations 🎯 3NF
👆 Glissez ou utilisez les boutons 👆

🎯 Objectifs

  • ✔️ Créer le diagramme entité-association d’un e-commerce
  • ✔️ Identifier les entités : Utilisateur, Produit, Catégorie, Commande
  • ✔️ Définir les relations : OneToMany, ManyToMany
  • ✔️ Normaliser jusqu’à la Troisième Forme Normale (3NF)
💡 Pourquoi ? Un bon modèle évite la redondance, assure l'intégrité et facilite le développement avec TypeORM.
Entités de base

🏷️ Les 4 entités principales

👤 Utilisateur (User)
id, email, password, firstName, lastName, phone, role, createdAt, lastLogin
📦 Produit (Product)
id, name, description, price, stock, sku, images, createdAt
📁 Catégorie (Category)
id, name, slug, parentId (auto-référencement)
🧾 Commande (Order)
id, total, status, shippingAddress, paymentMethod, trackingNumber, createdAt

Cardinalités

🔢 Comprendre les cardinalités

👤 User → 🧾 Order : 1 utilisateur peut avoir 0...N commandes → OneToMany
📦 Product → 📁 Category : 1 produit appartient à 1 catégorie, 1 catégorie contient 0...N produits → ManyToOne
👤 User ⇄ 🔐 Role : Un utilisateur peut avoir plusieurs rôles, un rôle peut être attribué à plusieurs utilisateurs → ManyToMany (table pivot)
📘 Astuce NestJS/TypeORM : On utilise @OneToMany(() => Order, order => order.user) et @ManyToOne(() => User, user => user.orders)
Qualité de base

🧹 Normalisation jusqu'à la 3NF

1NF (Première forme normale)

Attributs atomiques, pas de groupes répétitifs.
✅ Exemple : orders séparée des products.

2NF (Deuxième forme normale)

Supprimer les dépendances partielles.
✅ Exemple : OrderItem contient productName ? Non, on garde productId.

3NF (Troisième forme normale)

Supprimer les dépendances transitives.
✅ Exemple : User ne doit pas stocker cityName si cityId existe.

Implémentation technique

💻 Traduction en entités NestJS/TypeORM

// user.entity.ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ unique: true })
  email: string;

  @OneToMany(() => Order, order => order.user)
  orders: Order[];

  @ManyToMany(() => Role)
  @JoinTable()
  roles: Role[];
}

// order.entity.ts
@Entity()
export class Order {
  @ManyToOne(() => User, user => user.orders)
  user: User;

  @OneToMany(() => OrderItem, item => item.order)
  items: OrderItem[];
}
