📈 Calculer la note moyenne par produit

Pour chaque produit, la note moyenne est la somme des ratings divisée par le nombre d'avis. Dans une API e-commerce, on peut la calculer :

// Exemple d'agrégation SQL (TypeORM)
async computeAverageRating(productId: string) {
  const result = await this.reviewRepository
    .createQueryBuilder('review')
    .select('AVG(review.rating)', 'avg')
    .addSelect('COUNT(review.id)', 'count')
    .where('review.productId = :productId', { productId })
    .getRawOne();
  return { average: parseFloat(result.avg) || 0, totalReviews: parseInt(result.count) };
}

⚙️ Méthode 1 : Cron job (NestJS)

Mettre à jour un champ averageRating chaque nuit ou après chaque nouvel avis.

⚡ Méthode 2 : Trigger SQL / View

Créer une vue matérialisée ou utiliser un événement PostgreSQL.

📌 Formule mathématique : \( \text{Moyenne} = \frac{\sum_{i=1}^{n} \text{rating}_i}{n} \) . Plus n est grand, plus la note est fiable.

🌟 Distribution des étoiles (star breakdown)

Afficher combien d'utilisateurs ont attribué 1, 2, 3, 4 ou 5 étoiles à un produit. Cela aide les clients à mieux comprendre l’avis général.

📊 Exemple pour "Smartphone X" (total 124 avis)

🎨 Visualisation maison : largeur proportionnelle au nombre d'avis

Endpoint recommandé : GET /products/:id/reviews/stats → retourne { average, total, distribution: {1: x, 2: y, 3: z, 4: a, 5: b} }

🏆 Trier les produits par note moyenne

Permettre aux visiteurs de trier le catalogue selon la popularité (meilleures notes) ou les plus mal notés. Améliore l'expérience utilisateur.

// Avec QueryBuilder & sous-requête pour trier par rating moyen
async findAllOrderByRating(sort: 'DESC' | 'ASC' = 'DESC') {
  return this.productRepository
    .createQueryBuilder('product')
    .leftJoin('product.reviews', 'review')
    .select('product.*')
    .addSelect('COALESCE(AVG(review.rating), 0)', 'avgRating')
    .groupBy('product.id')
    .orderBy('avgRating', sort)
    .getRawMany();
}
📬 Query param examples
GET /products?sortBy=rating&order=desc GET /products?minRating=4.2
💡 Optimisation : indexer le champ "averageRating" après calcul périodique pour tri rapide.
✅ Implémentez également un cache Redis pour les classements les plus demandés.
