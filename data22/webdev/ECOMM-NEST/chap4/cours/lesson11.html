Catalogue de produits

Entités Product, Category, Tags et relations ManyToOne/ManyToMany

📘 Lesson 11 · Gérer le catalogue de produits

🎯 Objectifs de la leçon

1️⃣ Créer l'entité Product : name, description, price, stock, images, sku.
2️⃣ Créer l'entité Category : name, slug, parent category (auto-référencement).
3️⃣ Ajouter les relations : ManyToOne (Product → Category) et ManyToMany (Product ↔ Tags).
Entité

Product

Un produit représente un article vendu sur la plateforme e-commerce.

// src/products/entities/product.entity.ts
import { Entity, Column, PrimaryGeneratedColumn, ManyToOne, ManyToMany, JoinTable } from 'typeorm';
import { Category } from '../../categories/entities/category.entity';
import { Tag } from '../../tags/entities/tag.entity';

@Entity()
export class Product {
    @PrimaryGeneratedColumn('uuid')
    id: string;

    @Column({ length: 255 })
    name: string;

    @Column({ type: 'text', nullable: true })
    description: string;

    @Column({ type: 'decimal', precision: 10, scale: 2 })
    price: number;

    @Column({ type: 'int', default: 0 })
    stock: number;

    @Column({ type: 'simple-array', nullable: true })
    images: string[];

    @Column({ unique: true, length: 100 })
    sku: string;  // Stock Keeping Unit

    @Column({ type: 'timestamp', default: () => 'CURRENT_TIMESTAMP' })
    createdAt: Date;

    @Column({ type: 'timestamp', onUpdate: 'CURRENT_TIMESTAMP', nullable: true })
    updatedAt: Date;

    @ManyToOne(() => Category, (category) => category.products)
    category: Category;

    @ManyToMany(() => Tag)
    @JoinTable()
    tags: Tag[];
}
💡 Bonnes pratiques : sku doit être unique, price utilise decimal pour éviter les erreurs de flottants.
Entité

Category

Les catégories permettent d'organiser les produits. Le champ parentCategory permet d'avoir une hiérarchie (ex: Électronique → Smartphones → iPhone).

// src/categories/entities/category.entity.ts
import { Entity, Column, PrimaryGeneratedColumn, OneToMany, ManyToOne, Tree, TreeChildren, TreeParent } from 'typeorm';
import { Product } from '../../products/entities/product.entity';

@Entity()
@Tree("closure-table")  // Pour les catégories imbriquées
export class Category {
    @PrimaryGeneratedColumn('uuid')
    id: string;

    @Column({ unique: true })
    name: string;

    @Column({ unique: true })
    slug: string;  // URL friendly : "electronique-smartphones"

    @Column({ nullable: true })
    description: string;

    // Relation avec les produits
    @OneToMany(() => Product, (product) => product.category)
    products: Product[];

    // Auto-référencement pour la hiérarchie
    @TreeParent()
    parentCategory: Category;

    @TreeChildren()
    subCategories: Category[];
}
🌳 Arbre de catégories : @Tree("closure-table") permet de gérer facilement les requêtes hiérarchiques.
Entité

Tag

Les tags sont des mots-clés flexibles associés aux produits (ex: "promo", "nouveau", "best-seller"). Relation ManyToMany.

// src/tags/entities/tag.entity.ts
import { Entity, Column, PrimaryGeneratedColumn, ManyToMany } from 'typeorm';
import { Product } from '../../products/entities/product.entity';

@Entity()
export class Tag {
    @PrimaryGeneratedColumn('uuid')
    id: string;

    @Column({ unique: true })
    name: string;

    @Column({ default: '#3b82f6' })
    color: string;  // Couleur pour l'affichage front

    @ManyToMany(() => Product, (product) => product.tags)
    products: Product[];
}
🏷️ Astuce : Les tags sont utiles pour le filtrage et les recommandations. Évitez d'en avoir trop (max ~50 par produit).

🔗 Schéma des relations

Category
id, name, slug
↓ OneToMany
Product
id, name, price, stock, sku
ManyToOne → Category ManyToMany ↔ Tag
Tag
id, name, color
↔ ManyToMany (Product)
Un produit appartient à UNE catégorie. Un produit peut avoir plusieurs tags.
📌 Note : Le ManyToMany entre Product et Tag utilise @JoinTable() du côté de Product (côté propriétaire).

📦 Génération des modules NestJS

Commandes NestJS CLI pour créer les ressources :

# Module Product avec CRUD complet
nest g resource products --no-spec

# Module Category
nest g resource categories --no-spec

# Module Tag
nest g resource tags --no-spec

Importation des entités dans TypeOrmModule :

// app.module.ts ou products.module.ts
import { TypeOrmModule } from '@nestjs/typeorm';
import { Product } from './entities/product.entity';
import { Category } from '../categories/entities/category.entity';
import { Tag } from '../tags/entities/tag.entity';

@Module({
  imports: [
    TypeOrmModule.forFeature([Product, Category, Tag])
  ],
  // ...
})

🛠️ Exemple d'utilisation

// Créer une catégorie
const category = new Category();
category.name = "Smartphones";
category.slug = "smartphones";
await categoryRepository.save(category);

// Créer des tags
const promoTag = new Tag();
promoTag.name = "promo";
await tagRepository.save(promoTag);

// Créer un produit
const product = new Product();
product.name = "iPhone 15 Pro";
product.price = 1099.99;
product.stock = 50;
product.sku = "IP15P-256-SP";
product.category = category;
product.tags = [promoTag];
await productRepository.save(product);

// Rechercher un produit avec ses relations
const productWithRelations = await productRepository.findOne({
    where: { id: productId },
    relations: ['category', 'tags']
});
