🍪 Gestion du consentement aux cookies

Le RGPD impose de recueillir le consentement explicite de l'utilisateur avant de déposer des cookies non essentiels.

// src/consent/entities/consent.entity.ts
import { Entity, Column, PrimaryGeneratedColumn, ManyToOne, CreateDateColumn } from 'typeorm';
import { User } from '../../users/entities/user.entity';

@Entity()
export class CookieConsent {
    @PrimaryGeneratedColumn('uuid')
    id: string;

    @ManyToOne(() => User, { nullable: true })
    user: User | null;  // null pour utilisateur anonyme

    @Column({ nullable: true })
    visitorId: string;  // Pour les utilisateurs non connectés

    // Préférences par catégorie
    @Column({ default: false })
    necessary: boolean;  // Toujours true (obligatoire)

    @Column({ default: false })
    functional: boolean;

    @Column({ default: false })
    analytics: boolean;

    @Column({ default: false })
    marketing: boolean;

    @CreateDateColumn()
    consentedAt: Date;

    @Column({ nullable: true })
    ipAddress: string;
}
🔒 Règle RGPD : Les cookies nécessaires (authentification, panier) n'ont pas besoin de consentement. Les cookies analytics (Google Analytics) et marketing nécessitent un consentement explicite.

📤 Export des données (Droit à la portabilité)

L'utilisateur a le droit de récupérer toutes ses données personnelles dans un format structuré (JSON).

// src/users/users.service.ts
async exportUserData(userId: string): Promise<object> {
    const user = await this.userRepository.findOne({
        where: { id: userId },
        relations: ['orders', 'addresses', 'reviews', 'wishlist', 'consents']
    });
    
    return {
        profile: {
            email: user.email,
            firstName: user.firstName,
            lastName: user.lastName,
            phone: user.phone,
            createdAt: user.createdAt
        },
        addresses: user.addresses.map(a => ({
            street: a.street,
            city: a.city,
            postalCode: a.postalCode,
            country: a.country
        })),
        orders: user.orders.map(o => ({
            id: o.id,
            date: o.createdAt,
            total: o.total,
            status: o.status,
            items: o.items.map(i => ({
                productName: i.productName,
                quantity: i.quantity,
                price: i.price
            }))
        })),
        reviews: user.reviews.map(r => ({
            productId: r.product.id,
            rating: r.rating,
            comment: r.comment,
            createdAt: r.createdAt
        })),
        consentHistory: user.consents
    };
}

🗑️ Suppression des données (Droit à l'oubli)

L'utilisateur peut demander la suppression de toutes ses données personnelles.

// src/users/users.service.ts
async deleteUserData(userId: string): Promise<void> {
    const user = await this.userRepository.findOne({
        where: { id: userId },
        relations: ['orders', 'addresses', 'reviews', 'wishlist']
    });
    
    // Anonymisation ou suppression totale
    // Option 1 : Suppression physique (cascade)
    await this.userRepository.remove(user);
    
    // Option 2 : Anonymisation (pour conservation des commandes pour la comptabilité)
    // user.email = `deleted_${user.id}@anonymized.com`;
    // user.firstName = '[Supprimé]';
    // user.lastName = '[Supprimé]';
    // user.phone = null;
    // user.password = null;
    // await this.userRepository.save(user);
    
    // Journaliser la demande de suppression
    await this.gdprLogService.log({
        userId,

        timestamp: new Date(),
        ipAddress: this.request.ip
    });
}
⚠️ Attention : La suppression physique peut violer les obligations légales de conservation (7 ans pour les factures). Préférez l'anonymisation pour les données à conserver.

📋 Journalisation des accès aux données personnelles

Le RGPD exige de tracer qui a consulté les données personnelles et pourquoi.

// src/gdpr/entities/gdpr-log.entity.ts
@Entity()
export class GdprAccessLog {
    @PrimaryGeneratedColumn('uuid')
    id: string;

    @Column()
    userId: string;  // L'utilisateur dont les données ont été consultées

    @Column()
    accessedByUserId: string;  // Qui a fait la consultation (admin, support)

    @Column()
    accessReason: string;  // Ex: 'order_management', 'support_ticket_#123'

    @Column()
    dataTypesAccessed: string;  // Ex: 'email,phone,address'

    @Column({ nullable: true })
    ipAddress: string;

    @CreateDateColumn()
    accessedAt: Date;
}

// Middleware/Interceptor pour logger automatiquement
@Injectable()
export class GdprLoggingInterceptor implements NestInterceptor {
    intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
        const request = context.switchToHttp().getRequest();
        const userId = request.params.userId; // ID de l'utilisateur consulté
        
        if (userId && !this.isOwnData(request.user.id, userId)) {
            // Log uniquement si on consulte les données d'un autre utilisateur (admin)
            this.gdprLogService.logAccess({
                userId,
                accessedByUserId: request.user.id,
                accessReason: request.headers['x-access-reason'] || 'admin_panel',
                dataTypesAccessed: this.extractDataTypes(request.route.path),
                ipAddress: request.ip
            });
        }
        
        return next.handle();
    }
}

✅ Bonnes pratiques RGPD pour l'API

  • Durée de conservation limitée : Supprimez les logs après 30 jours, les paniers abandonnés après 7 jours.
  • Chiffrement des données sensibles : Utilisez AES-256 pour les données comme les numéros de téléphone.
  • Audit régulier : Vérifiez mensuellement que les données non nécessaires sont supprimées.
  • Formulaire de demande en ligne : Permettez aux utilisateurs d'exercer leurs droits (export/suppression) via l'interface.
  • Délai de réponse de 30 jours : Le RGPD impose de répondre aux demandes dans ce délai.
  • Politique de confidentialité claire : Expliquez quelles données sont collectées et pourquoi.
🎓 Récapitulatif : Le respect du RGPD n'est pas une option pour une plateforme e-commerce moderne. Les amendes peuvent aller jusqu'à 20 millions d'euros ou 4% du chiffre d'affaires mondial.
