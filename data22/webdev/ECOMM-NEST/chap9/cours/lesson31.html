Notifier une promotion

Lorsque le prix d’un produit diminue ou qu’une campagne marketing s’applique (ex: -20%), le système doit envoyer une notification aux utilisateurs qui ont ce produit dans leur liste de souhaits.

Cas technique : Webhook / événement déclenché lors de la mise à jour du prix ou activation d’un coupon global. Utilisation de queues (Bull) pour envoyer des emails / notifications push.

✅ Implémentation type : ProductPriceService → vérifie les produits en promo → récupère les utilisateurs via la wishlist → envoie notification.

Déclenchement : baisse de prix → alerte email/in-app

📢

Alerte "De retour en stock"

Quand un produit en rupture (stock = 0) est réapprovisionné, tous les clients qui l'ont mis en favoris doivent être prévenus immédiatement. Cela permet de récupérer des ventes perdues.

🧠 Logique : Un trigger après mise à jour de stock > 0 si avant il était à 0.
✉️ Canal : Email transactionnel + notification push (Firebase) ou SMS.
💡 Bonne pratique : Éviter de spammer; ne notifier qu'une seule fois par produit et par utilisateur jusqu'à ce qu'il réagisse (achat ou retrait de la wishlist). 
// Exemple de méthode dans WishlistService (NestJS)
async notifyBackInStock(productId: string) {
  const product = await this.productRepo.findOne(productId);
  if(product.stock > 0 && product.wasOutOfStock) {
    const users = await this.wishlistRepo.findUsersByProduct(productId);
    for(const user of users) {
      await this.notificationService.send({
        userId: user.id,
        message: `${product.name} est de nouveau disponible !`,
        type: 'back_in_stock'
      });
    }
    product.wasOutOfStock = false;
    await this.productRepo.save(product);
  }
}

Partage de la Wishlist

Permettre aux utilisateurs de générer un lien unique, public ou privé, pour montrer leurs envies à leurs proches. Ce lien doit être accessible sans authentification.

Fonctionnement technique :

  • Génération d’un token unique (UUID) associé à la wishlist de l’utilisateur.
  • Endpoint GET /wishlist/public/:shareToken
  • La page publique affiche les produits (sans boutons de modification).
  • Option de désactiver le lien, régénérer un nouveau token.
🔗 https://api.example.com/share/abc123
✅ Partageable sur WhatsApp, Twitter, email
⚡ Sécurité : le token doit être complexe. Ne jamais exposer l'ID utilisateur en clair.
⚙️

Architecture des notifications

Pour gérer les alertes de promotion et de retour en stock, on utilise souvent un système asynchrone avec une file d'attente (Bull + Redis).

Avantages : Décharge l'API principale, gestion des erreurs, réessais automatiques.

🔐 Génération lien partage (NestJS / TypeORM)

// Entité Wishlist étendue
@Entity()
export class Wishlist {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ nullable: true, unique: true })
  shareToken: string;

  @Column({ default: false })
  isPublicShareEnabled: boolean;

  @ManyToOne(() => User)
  user: User;

  @ManyToMany(() => Product)
  products: Product[];
}

// Service : activation du partage
async enablePublicSharing(wishlistId: string) {
  const wishlist = await this.wishlistRepo.findOne(wishlistId);
  if(!wishlist.shareToken) {
    wishlist.shareToken = uuidv4();
  }
  wishlist.isPublicShareEnabled = true;
  await this.wishlistRepo.save(wishlist);
  return `https://ecom-api.com/share/${wishlist.shareToken}`;
}
🔧 Pensez à ajouter un endpoint GET /share/:token qui retourne la liste des produits (readonly).
