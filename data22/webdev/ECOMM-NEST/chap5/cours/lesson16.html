📦 Modélisation : Cart & CartItem

// cart.entity.ts
@Entity()
export class Cart {
  @PrimaryGeneratedColumn() id: number;
  @OneToOne(() => User, user => user.cart)
  user: User; // null si panier anonyme
  @Column({ nullable: true }) visitorId: string;
  @OneToMany(() => CartItem, item => item.cart, { cascade: true })
  items: CartItem[];
  @CreateDateColumn() createdAt: Date;
  @UpdateDateColumn() updatedAt: Date;
}

// cart-item.entity.ts
@Entity()
export class CartItem {
  @PrimaryGeneratedColumn() id: number;
  @ManyToOne(() => Product) product: Product;
  @Column() quantity: number;
  @Column({ nullable: true }) variation: string; // ex: "XL-rouge"
  @ManyToOne(() => Cart, cart => cart.items)
  cart: Cart;
}

💡 Un panier anonyme utilise visitorId (généré par le frontend). Un panier connecté a un user.

➕ Ajouter un produit au panier

Endpoint POST /cart/add avec DTO contenant productId, quantity, variation.

// add-to-cart.dto.ts
export class AddToCartDto {
  @IsNumber() productId: number;
  @IsNumber() @Min(1) quantity: number;
  @IsOptional() @IsString() variation?: string;
}

// cart.service.ts (extrait)
async addItem(userId: number | null, visitorId: string | null, dto: AddToCartDto) {
  let cart = await this.findOrCreateCart(userId, visitorId);
  let existingItem = cart.items.find(i => i.product.id === dto.productId && i.variation === dto.variation);
  if (existingItem) existingItem.quantity += dto.quantity;
  else cart.items.push(this.cartItemRepo.create({ ...dto, cart }));
  return this.cartRepo.save(cart);
}

📌 Si l’article existe déjà (même produit + variation), on incrémente la quantité.

✏️ Opérations courantes

🔄 Modifier quantité

PATCH /cart/item/:itemId
{ "quantity": 3 }
→ trouver l’item, mettre à jour, sauvegarder.

❌ Supprimer un item

DELETE /cart/item/:itemId
→ item.remove() + cartRepo.save()

🗑️ Vider tout le panier

DELETE /cart/clear
→ cart.items = [] ; await cartRepo.save(cart);

📌 Chaque modification doit recalculer le total côté serveur (mais on peut renvoyer le panier mis à jour).

🔄 Panier anonyme → fusion après connexion

Problème : un utilisateur ajoute des produits sans être connecté, puis se connecte. Il ne faut pas perdre ses articles.

async function mergeCart(visitorId: string, userId: number) {
  const anonymousCart = await getCartByVisitorId(visitorId);
  const userCart = await getOrCreateCartByUserId(userId);
  for (const item of anonymousCart.items) {
    addOrMergeItem(userCart, item);
  }
  await save(userCart);
  await remove(anonymousCart); // ou le marquer comme fusionné
}

✨ Stratégie :

  • Chaque client non connecté reçoit un visitorId (généré frontend, stocké dans localStorage).
  • Lors de la connexion, l’API fusionne les items (addition des quantités si mêmes produits/variations).
  • Après fusion, le panier anonyme est supprimé ou désactivé.

🔒 Bonnes pratiques & sécurité

  • ✔️ Toujours vérifier la disponibilité du stock avant d’ajouter/modifier la quantité.
  • ✔️ Ne pas exposer les IDs de panier dans le frontend (utiliser l’utilisateur ou le visitorId).
  • ✔️ Expiration : supprimer les paniers anonymes vieux de plus de 7 jours (cron job).
  • ✔️ Utiliser des transactions pour les opérations critiques (fusion, passage commande).
  • ✔️ Pour les variations, un champ variationKey (ex: "size:M-color:red") évite les collisions.
💡 Astuce : on peut stocker le visitorId dans un cookie sécurisé (HttpOnly) ou le générer côté serveur.

🧪 Exercice :

Un client non connecté ajoute 2 produits A et 1 produit B (variation "M"). Ensuite, il se connecte. Son panier existant déjà contient 1 produit A (variation "M") et 1 produit C.

Décrivez les étapes de fusion et le contenu final du panier connecté.

🔍 Voir solution

Étape 1 : récupérer panier anonyme et panier utilisateur.
Étape 2 : pour chaque item anonyme, chercher dans panier utilisateur un item identique (productId + variation).
- Produit A (var M) : on additionne les quantités : 2 + 1 = 3.
- Produit B (var M) : n’existe pas encore, on l’ajoute.
Étape 3 : supprimer le panier anonyme.
Résultat : panier final = A:3, B:1, C:1.
