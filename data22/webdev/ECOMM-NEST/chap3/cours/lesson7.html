👤 Entité User – champs et décorateurs

Chaque utilisateur aura un email unique, un mot de passe hashé, et des informations personnelles.

// src/users/user.entity.ts
import { Entity, Column, PrimaryGeneratedColumn, CreateDateColumn, UpdateDateColumn, ManyToMany, JoinTable } from 'typeorm';
import { Role } from '../roles/role.entity';

@Entity('users')
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ unique: true })
  email: string;

  @Column()
  password: string;

  @Column()
  firstName: string;

  @Column()
  lastName: string;

  @Column({ nullable: true })
  phone: string;

  @Column({ nullable: true, type: 'timestamp' })
  lastLogin: Date;

  @CreateDateColumn()
  createdAt: Date;

  @UpdateDateColumn()
  updatedAt: Date;

  @ManyToMany(() => Role, role => role.users)
  @JoinTable({ name: 'user_roles' })
  roles: Role[];
}

✨ @CreateDateColumn et @UpdateDateColumn sont gérés automatiquement par TypeORM. @JoinTable est obligatoire du côté propriétaire de la relation ManyToMany.

🔐 Entité Role – définition et relation inverse

// src/roles/role.entity.ts
import { Entity, Column, PrimaryGeneratedColumn, ManyToMany } from 'typeorm';
import { User } from '../users/user.entity';

@Entity('roles')
export class Role {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ unique: true })
  name: string;  // 'admin', 'customer', 'seller'

  @Column({ nullable: true })
  description: string;

  @ManyToMany(() => User, user => user.roles)
  users: User[];
}

📌 Important :

Un utilisateur peut avoir plusieurs rôles (ex: admin + seller). La table de liaison user_roles est créée automatiquement grâce à @JoinTable.

🔗 Comprendre la relation ManyToMany

Un utilisateur peut avoir plusieurs rôles. Un rôle peut être attribué à plusieurs utilisateurs. TypeORM crée une table de liaison user_roles avec deux clés étrangères.

Table users : id (PK), email, password ...

Table roles : id (PK), name, description

Table user_roles : userId (FK) | roleId (FK)

// Exemple d'assignation dans le service
const user = await userRepo.findOne({ where: { id: 1 }, relations: ['roles'] });
const adminRole = await roleRepo.findOne({ where: { name: 'admin' } });
user.roles.push(adminRole);
await userRepo.save(user);

✅ Pour récupérer un utilisateur avec ses rôles, il faut toujours préciser relations: ['roles'] dans le find.

🌱 Initialisation des rôles (Seed)

Au démarrage de l'application, nous devons garantir l'existence des rôles admin, customer, seller.

// src/seeds/role.seed.ts (exécuté via une commande ou dans app.module)
export const seedRoles = async (dataSource: DataSource) => {
  const roleRepo = dataSource.getRepository(Role);
  const roles = ['admin', 'customer', 'seller'];
  for (const name of roles) {
    const exists = await roleRepo.findOne({ where: { name } });
    if (!exists) {
      await roleRepo.save(roleRepo.create({ name, description: `${name} role` }));
      console.log(`✅ Rôle ${name} créé`);
    }
  }
};

On peut appeler cette fonction dans onModuleInit d’un module ou via une migration dédiée.

📌 Bonnes pratiques pour User & Role

  • ✅ Toujours hacher le mot de passe avant de sauvegarder (bcrypt).
  • ✅ Utiliser @BeforeInsert / @BeforeUpdate pour hasher automatiquement.
  • ✅ Ajouter un index sur email pour accélérer les recherches.
  • ✅ Préférer les DTO pour exposer les données (ne jamais renvoyer le mot de passe).
  • ✅ Pour les champs sensibles, utiliser @Exclude() (class-transformer).
// Exemple d'exclusion du mot de passe
import { Exclude } from 'class-transformer';

@Entity('users')
export class User {
  @Column()
  @Exclude()
  password: string;
}

✏️ Exercice : Créer un utilisateur avec rôle

Écrivez un service NestJS qui :

  1. Crée un utilisateur avec email, password (hashé), firstName, lastName.
  2. Lui assigne automatiquement le rôle customer.
  3. Retourne l'utilisateur sans le mot de passe.
// Exemple de solution
async createUser(createUserDto: CreateUserDto): Promise<Partial<User>> {
  const { email, password, firstName, lastName } = createUserDto;
  const hashedPassword = await bcrypt.hash(password, 10);
  const customerRole = await this.roleRepo.findOne({ where: { name: 'customer' } });
  const user = this.userRepo.create({
    email,
    password: hashedPassword,
    firstName,
    lastName,
    roles: [customerRole],
  });
  const savedUser = await this.userRepo.save(user);
  const { password: _, ...result } = savedUser;
  return result;
}
