💡 Pourquoi Kubernetes ? Automatisation du déploiement, mise à l’échelle et gestion des conteneurs en production.

🏛️ Architecture Kubernetes

Control Plane (cerveau du cluster) :

  • API Server : point d'entrée (kubectl, UI)
  • etcd : stockage clé-valeur (configuration, état)
  • Scheduler : place les pods sur les nodes
  • Controller Manager : régule l'état souhaité

🧠 À retenir :

Le Control Plane gère le cluster, les Nodes (workers) exécutent les applications conteneurisées.

📌 Chaque Node contient des Pods (le plus petit objet déployable), et un kubelet qui communique avec le control plane.

🔑 Concepts fondamentaux

📦 Pod

Groupe de conteneurs (souvent 1) partageant le réseau & stockage. Unité de base.

🚀 Deployment

Gère les mises à jour, rollbacks et réplicas des Pods. Déclare l'état souhaité.

🌐 Service

Expose des Pods en réseau (stable IP/DNS). Load balancing intégré.

🚪 Ingress

Routage HTTP/HTTPS externe vers les Services. Règles de chemin.

✨ À comprendre : ConfigMap & Secret

ConfigMap : variables de configuration non sensibles. Secret : données sensibles (mots de passe, tokens).

🧩 Pods & Noeuds (Nodes)

🎯 Pod : conteneur(s) avec IP partagée. Éphémère par défaut.

🖥️ Node : machine physique/virtuelle (VM) du cluster. Contient:

  • Kubelet (agent)
  • Kube-proxy (règles réseau)
  • Runtime container (Docker, containerd)

⚙️ Installer un cluster local

🟢 Minikube

Cluster Kubernetes mono-node, parfait pour l’apprentissage. Crée une VM ou conteneur local.

minikube start --driver=docker
kubectl get nodes

🟠 Kind (Kubernetes IN Docker)

Cluster multi-nœuds utilisant des conteneurs Docker. Idéal pour CI/CD ou tests.

kind create cluster --name moncluster
kubectl cluster-info
✅ Après installation, utilisez kubectl pour interagir avec le cluster. Vérifiez avec kubectl version.

🧠 Mise en pratique – schéma mental

Questions pour réfléchir :

  • Que se passe-t-il si un Node tombe en panne ? (Les Pods sont recréés sur un autre nœud par le contrôleur)
  • À quoi sert le Scheduler ? (Placer les pods sur les nodes selon ressources/disponibilité)
  • Quelle est la différence entre un Service de type ClusterIP et NodePort ?
