🧠 Qu'est-ce qu'un processus ?

Un processus est un programme en cours d’exécution. Chaque processus possède un identifiant unique appelé PID (Process ID). Linux gère les processus de façon hiérarchique : le premier processus est init ou systemd (PID 1).

🔍 Principales commandes de visualisation :

ps aux          # Affiche tous les processus avec détails
top             # Affichage dynamique interactif
htop            # Version améliorée (couleur, souris)
systemctl       # Gère les services systemd

⚙️ Contrôler les processus

🔫 kill & signaux

kill -9 PID     # Force l'arrêt (SIGKILL)
kill -15 PID    # Demande polie (SIGTERM)
pkill nom_proc  # Tue par nom
killall firefox # Tue tous les processus nommés

📦 systemctl (services)

systemctl start nginx
systemctl stop nginx
systemctl restart nginx
systemctl status nginx
systemctl enable nginx  # au démarrage

⚠️ Attention : Le signal SIGKILL (kill -9) ne permet pas au processus de sauvegarder son état. Préférez SIGTERM quand c’est possible.

$ ps aux | grep apache2
$ kill -15 1245
✔ Processus arrêté proprement

📦 Gestion des paquets Linux

Les gestionnaires de paquets permettent d'installer, mettre à jour et supprimer des logiciels facilement.

DistributionGestionnaireCommandes typiques
Debian/Ubuntuaptapt update && apt install nodejs
RHEL/CentOS 7yumyum install nginx -y
Fedora / RHEL 8+dnfdnf groupinstall "Development Tools"
 
# Exemple concret avec APT (Ubuntu/Debian)
sudo apt update                 # Rafraîchit la liste des paquets
sudo apt upgrade                # Met à jour tous les logiciels
sudo apt install git docker.io  # Installation
sudo apt remove vim             # Supprime mais garde config
sudo apt purge vim              # Supprime + fichiers config
🔧 Bon à savoir : dnf est le successeur moderne de yum, avec meilleure gestion des dépendances.

🛠️ Installation & bonnes pratiques

✅
Rechercher un paquet : apt search mysql ou dnf search postgres
📌
Informations sur un paquet : apt show curl / dnf info nginx
🧹
Nettoyer le cache : sudo apt autoremove , sudo apt autoclean / dnf clean all

💻 Exercice pratique pour les élèves :

1. Lancez htop (installez-le si nécessaire) et identifiez le processus utilisant le plus de CPU.
2. Utilisez systemctl pour redémarrer le service SSH.
3. Installez tree avec le gestionnaire de paquets, puis supprimez-le.

📊 Surveiller en temps réel

Outils essentiels pour le monitoring système :

top – vue classique, taux CPU/MEM, processus interactif.
htop – interface améliorée, gestion avec souris, arborescence.
ps aux --sort=-%cpu – liste triée par consommation CPU.
systemd-cgtop – affiche l'usage ressources par groupe systemd.
