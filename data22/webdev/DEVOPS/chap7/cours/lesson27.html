Ansible : Automatiser le provisionnement d’infrastructure

Leçon 27 · Cours pratique pour débutants

⚙️

Apprenez à écrire des playbooks YAML, découvrir l’architecture maître-nœud et automatiser vos serveurs comme un pro.

🎯 Objectifs de la leçon

  • 1 Installer Ansible et comprendre l’architecture (nœud de contrôle / nœuds managés).
  • 2 Écrire des playbooks YAML (tasks, handlers, variables) pour automatiser des configurations.
  • 3 Utiliser les rôles Ansible pour organiser et réutiliser le code proprement.

🏗️ Architecture Ansible : contrôleur vs nœuds managés

Ansible est un outil agentless (sans agent) qui utilise SSH pour se connecter aux machines. Pas besoin d'installer un agent sur les serveurs cibles !

Le nœud de contrôle exécute des playbooks qui décrivent l’état souhaité des serveurs. Ansible utilise des modules (apt, copy, service, etc.) pour appliquer les changements de manière idempotente.

💡 Astuce: Idempotence = on peut exécuter le playbook plusieurs fois sans casser la configuration. Ansible vérifie l’état actuel avant d’agir.

🖥️ Installation d’Ansible (Ubuntu / Debian / Mac)

Sur votre machine (contrôleur), installez Ansible via le gestionnaire de paquets :

# Ubuntu / Debian
sudo apt update
sudo apt install ansible -y

# Vérifier l’installation
ansible --version

Pour tester la connexion aux nœuds, créez un fichier inventory.ini :

[webservers]
192.168.1.10 ansible_user=ubuntu

[all:vars]
ansible_python_interpreter=/usr/bin/python3

Puis exécutez un ping Ansible : ansible all -i inventory.ini -m ping

📝 Playbook YAML : structure essentielle

Un playbook est une liste de plays. Chaque play cible un groupe d’hôtes et exécute des tâches.

---
- name: Premier playbook - Installer nginx
  hosts: webservers
  become: yes   # exécute avec privilèges sudo
  tasks:
    - name: Mettre à jour le cache apt
      apt:
        update_cache: yes

    - name: Installer Nginx
      apt:
        name: nginx
        state: present

    - name: Démarrer Nginx
      service:
        name: nginx
        state: started
        enabled: yes

🎯 Explication : hosts définit la cible, tasks liste des actions, apt et service sont des modules.

🔁 Variables, Handlers et Templates

Variables : pour rendre vos playbooks réutilisables

vars:
  package_name: nginx
  service_port: 80

Handlers : exécutés uniquement si une tâche notifie un changement (ex: redémarrer un service après modif conf)

tasks:
  - name: Modifier index.html
    copy:
      content: "Bienvenue"
      dest: /var/www/html/index.html
    notify: restart_nginx

handlers:
  - name: restart_nginx
    service:
      name: nginx
      state: restarted

Les templates Jinja2 permettent de générer des fichiers de configuration dynamiques (.j2).

📂 Rôles Ansible : modularité et réutilisabilité

Un rôle est une structure de dossiers standardisée :

mon_role/
├── tasks/
│   └── main.yml
├── handlers/
│   └── main.yml
├── templates/
├── files/
├── vars/
│   └── main.yml
└── defaults/
    └── main.yml

Utilisation dans un playbook :

- hosts: webservers
  roles:
    - nginx
    - firewall

Les rôles permettent de réutiliser du code entre différents projets et de collaborer via Ansible Galaxy.

🚀 Exemple réel : Déploiement d’une app Node.js

Objectif : configurer un serveur avec Node.js, PM2, et déployer une application depuis Git.

- name: Déployer app Node.js
  hosts: app_servers
  become: yes
  vars:
    app_repo: "https://github.com/exemple/mon-app.git"
    app_dir: "/opt/monapp"
  tasks:
    - name: Installer Node.js et npm
      apt:
        name: "{{ item }}"
        state: present
      loop:
        - nodejs
        - npm

    - name: Cloner le dépôt
      git:
        repo: "{{ app_repo }}"
        dest: "{{ app_dir }}"
        version: main

    - name: Installer les dépendances npm
      npm:
        path: "{{ app_dir }}"
        state: present

    - name: Installer PM2 globalement
      npm:
        name: pm2
        global: yes

    - name: Démarrer l'app avec PM2
      command: pm2 start {{ app_dir }}/app.js --name mon-app
      args:
        chdir: "{{ app_dir }}"
✅ Résultat : votre application tourne avec gestion automatique des redémarrages.

🎓 Récapitulatif et bonnes pratiques

  • ✔️ Toujours versionner vos playbooks (Git).
  • ✔️ Utiliser ansible-lint pour vérifier la syntaxe.
  • ✔️ Séparer les variables sensibles avec ansible-vault.
  • ✔️ Tester d’abord avec --check (mode dry-run).
  • ✔️ Organiser en rôles quand le projet grandit.

✨ Ansible transforme l’administration système en code clair et maintenable. Pratiquez en écrivant vos propres playbooks !
