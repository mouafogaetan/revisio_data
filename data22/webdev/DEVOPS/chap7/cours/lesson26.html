☁️ 1. Provisionner AWS avec Terraform

Terraform utilise des providers comme aws pour communiquer avec l'API Amazon. Exemple de ressources classiques :

  • aws_instance → créer une machine virtuelle EC2
  • aws_s3_bucket → stockage objet
  • aws_vpc → réseau virtuel isolé

📄 Exemple de fichier main.tf (AWS)

# configurer le provider AWS
provider "aws" {
  region = "eu-west-3"  # Paris
}

# créer un bucket S3
resource "aws_s3_bucket" "mon_bucket" {
  bucket = "mon-site-devops-2026"
  tags = {
    Environnement = "Cours"
  }
}

# lancer une instance EC2 (Ubuntu)
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  tags = {
    Name = "MonServeur-DevOps"
  }
}

✨ Après terraform apply, AWS crée ces ressources automatiquement !

🟦 2. Terraform sur Azure

Le provider azurerm permet de gérer l'infrastructure Microsoft Azure. Voici des ressources typiques :

  • azurerm_resource_group → groupe de ressources
  • azurerm_virtual_machine → VM Linux/Windows
  • azurerm_storage_account → Blob Storage / fichiers

📄 Exemple de configuration Azure

provider "azurerm" {
  features {}
}

resource "azurerm_resource_group" "rg" {
  name     = "rg-devops-cameroun"
  location = "France Central"
}

resource "azurerm_virtual_network" "vnet" {
  name                = "vnet-cours"
  location            = azurerm_resource_group.rg.location
  resource_group_name = azurerm_resource_group.rg.name
  address_space       = ["10.0.0.0/16"]
}

resource "azurerm_storage_account" "storage" {
  name                     = "devopsstoragecameroun"
  resource_group_name      = azurerm_resource_group.rg.name
  location                 = azurerm_resource_group.rg.location
  account_tier             = "Standard"
  account_replication_type = "LRS"
}

💡 Bon à savoir : Azure CLI doit être authentifié. Terraform exécute les appels API de façon sécurisée.

🧩 3. Modules Terraform : la puissance de la réutilisation

Les modules permettent d'encapsuler des ressources réutilisables. C'est comme des bibliothèques pour votre infrastructure.

🔧 Structure d'un module

modules/nginx-server/
  ├── main.tf      (définit instance, security group)
  ├── variables.tf (inputs: nom, instance_type)
  └── outputs.tf   (IP publique par exemple)

📦 Appeler un module dans votre configuration

module "mon_serveur_web" {
  source = "./modules/nginx-server"
  server_name = "web-prod"
  instance_type = "t3.small"
}

# Afficher l'IP du module
output "ip_du_serveur" {
  value = module.mon_serveur_web.public_ip
}

✅ Les modules officiels (Terraform Registry) existent pour AWS, Azure, GCP : gagnez un temps fou !

📌 Pratique recommandée : Utilisez des modules pour le réseau, les bases de données, les clusters Kubernetes.

⚙️ Workflow Terraform & Bonnes Pratiques

📌 Étapes essentielles

  • terraform init → initialise providers/modules
  • terraform plan → montre les changements
  • terraform apply → applique l'infra
  • terraform destroy → supprime tout proprement

🔐 Sécurité et état

  • Ne pas versionner terraform.tfstate (contient des secrets)
  • Utiliser un backend distant (S3, Azure Storage)
  • Isoler les environnements (dev/staging/prod) avec des workspaces

Créez un bucket S3 + une petite instance EC2 avec Terraform. Modifiez la taille de l'instance et observez le plan. Ajoutez une tag "Cours=DevOps".
