DevOps & Shift Left — protéger l'application de l’image jusqu’au pipeline

🐳 Docker 📦 Trivy 🔍 OWASP
🎯

Objectifs de la leçon

  • ✔️ Scanner les images Docker avec Trivy ou Clair (vulnérabilités système & langages)
  • ✔️ Analyser les dépendances (bibliothèques) avec OWASP Dependency Check
  • ✔️ Intégrer automatiquement les scans dans le pipeline CI/CD (GitHub Actions, GitLab CI)
  • ✔️ Comprendre comment "Shift Left" réduit les risques de sécurité en production

💡 Pourquoi c'est critique ? Une image Docker vulnérable ou une librairie obsolète = porte d'entrée pour les pirates.

🐳🔎

Scanner les images Docker

Des outils comme Trivy (rapide, simple) ou Clair détectent les failles dans les couches OS, les langages (Python, Node, Java...).

# Installation Trivy (Linux / Mac)
$ brew install aquasecurity/trivy/trivy   # ou apt-get install trivy

# Scanner une image Docker locale
$ trivy image python:3.9-slim

# Générer un rapport JSON pour CI/CD
$ trivy image --format json --output report.json monapp:latest
✅ Trivy

Analyse multicouches, base de données constamment mise à jour, CI/CD friendly.

⚙️ Clair

Solution modulaire, s'intègre avec registries (Quay, Harbor).

📦🧪

OWASP Dependency Check

Outil open source qui identifie les vulnérabilités connues (base NVD) dans les dépendances de votre projet (npm, pip, maven, composer, etc.)

# Exécution via CLI (Linux)
$ dependency-check --scan ./mon-projet --format HTML --out rapport

# Avec Maven (plugin)
mvn org.owasp:dependency-check-maven:check

# Dans un pipeline GitHub Actions
- name: OWASP Dependency Check
  uses: dependency-check/Dependency-Check_Action@main
  with:
    project: 'monapp'
    path: '.'
    format: 'HTML'

⚠️ Attention : Des librairies populaires comme Log4Shell ont montré l’importance de scanner en continu. OWASP donne un rapport détaillé avec CVSS score.

⚙️ Intégration CI/CD

Automatiser les scans à chaque commit / pull request avant le déploiement.

📝 Code Push
→
🔨 Build
→
🔒 Scan Docker + Dépendances
→
✅ Tests
→
🚀 Déploiement (si safe)

📌 Exemple workflow GitHub Actions intégré :

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Scan image Docker avec Trivy
        run: |
          docker build -t monapp .
          trivy image --exit-code 1 --severity CRITICAL monapp
      - name: OWASP Dependency Check
        uses: dependency-check/Dependency-Check_Action@main
        with:
          path: '.'
          failOnCVSS: 7

💡 L'option --exit-code 1 bloque le pipeline si une faille critique est trouvée.

🧠

Résumé & bonnes pratiques

📋 #1

Scanner tôt, scanner souvent

Intégrer Trivy & OWASP dès l’étape de build. "Shift left" : avant que l’image ne parte en production.

📋 #2

Utiliser des images de base minimales

Préférer alpine, slim, distroless pour réduire les vulnérabilités.

📋 #3

Mettre à jour régulièrement

Utiliser `renovate` ou `dependabot` pour les dépendances et images de base.

📋 #4

Dashboard de suivi

Générer des rapports HTML pour les équipes (OWASP, Trivy server).

🎯 À retenir : Une chaîne DevOps sécurisée évite les failles en production et protège les données des utilisateurs.
