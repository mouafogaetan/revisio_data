💡 À la fin de ce cours, vous saurez pourquoi les équipes modernes automatisent leurs tests et déploiements.

🔄 L’intégration continue (CI)

Pratique qui consiste à fusionner fréquemment les modifications du code dans un dépôt partagé (plusieurs fois par jour). Chaque fusion déclenche automatiquement une vérification.

🔁 Exemple de workflow CI :

  • 1. Un développeur pousse son code sur GitHub (pull request).
  • 2. Un serveur CI (Jenkins, GitLab CI, GitHub Actions) détecte le push.
  • 3. Il télécharge le code, installe les dépendances.
  • 4. Il exécute les tests unitaires et d'intégration.
  • 5. Si tout est vert → la branche peut être fusionnée.
✔️ Objectif : détecter rapidement les erreurs d’intégration et éviter le "merge hell".

🚀 Le déploiement continu (CD)

Extension de la CI : chaque modification validée (qui passe les tests) est automatiquement déployée dans un environnement de production ou de préproduction.

📦 Livraison continue

Le logiciel est prêt à être déployé manuellement à tout moment. Le déploiement final reste une décision humaine.

⚡ Déploiement continu

Déploiement automatique en production après validation des tests. Plus risqué mais plus rapide.

⚠️ Le déploiement continu nécessite des tests très robustes (unitaires, intégration, end-to-end) et une bonne couverture.

⚙️ Les étapes d’un pipeline CI/CD classique

📝 CodeCommit
→
🛠️ BuildCompilation
→
🧪 TestUnités / Intégration
→
📦 PackageImage Docker, artefact
→
🌍 DeployProd / Staging

💡 Chaque étape peut être automatisée dans un fichier de configuration (ex: .gitlab-ci.yml, Jenkinsfile, GitHub Actions).

Le pipeline permet d’avoir un retour immédiat sur la santé de l’application après chaque modification.

📈 Bénéfices de la CI/CD

⚡
Réduction des délais
Livraison plus fréquente, petites fonctionnalités.
🐞
Détection précoce des bugs
Tests exécutés à chaque commit.
🤝
Collaboration renforcée
Moins de conflits, intégration continue des changements.
📉
Risques maîtrisés
Déploiements plus petits et réversibles.
✅ Exemple concret : une entreprise camerounaise de e-commerce peut déployer une correction de bug en 30 minutes au lieu de 2 semaines grâce à un pipeline CI/CD bien conçu.
