⚙️ Jenkins : Le chef d'orchestre CI/CD

Jenkins est un automate open-source (Java) qui permet d'exécuter des tâches répétitives : compilation, tests, analyse de code, déploiement. Il est extensible via plus de 1800 plugins !

📥 Installer Jenkins (Linux / Docker / Windows)

🐧 Ubuntu/Debian

curl -fsSL https://pkg.jenkins.io/.../jenkins.io-2023.key | tee ...
sudo apt update && sudo apt install jenkins
sudo systemctl start jenkins

🐳 Docker (recommandé)

docker run -p 8080:8080 -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts

🔐 Première connexion : récupérez le mot de passe initial avec sudo cat /var/lib/jenkins/secrets/initialAdminPassword ou via les logs Docker. Ensuite, installez les plugins suggérés (Git, Pipeline, Docker).

📜 Jenkinsfile : Pipeline en code

Le Jenkinsfile (déposé à la racine du projet) permet un versionnement et une reproductibilité du pipeline.

pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps { git 'https://github.com/exemple/mon-app.git' }
        }
        stage('Build') {
            steps { sh 'npm install' }
        }
        stage('Test') {
            steps { sh 'npm test' }
        }
        stage('Package') {
            steps { sh 'docker build -t mon-app .' }
        }
    }
    post {
        success { echo '✅ Pipeline réussi !' }
        failure { echo '❌ Échec du pipeline' }
    }
}

⭐ Les stages représentent les étapes (build, test, déploiement). Très lisible.

🔗 Intégration puissante : Git + Docker + Tests

  • 📌 Git : Jenkins clone automatiquement le dépôt (via webhook ou polling).
  • 🐳 Docker : Builder des images et les pousser vers un registre (DockerHub).
  • 🧪 Outils de test : JUnit, Pytest, SonarQube, etc. Jenkins publie les résultats.

Exemple d'étape Docker dans Jenkinsfile :

stage('Docker Build') {
    steps {
        script {
            docker.build("monapp:${env.BUILD_ID}")
        }
    }
}
⚡ Pipeline complet déclenché à chaque push = Feedback immédiat pour l'équipe

🛠️ Bonnes pratiques & astuces

✔️ Jenkinsfile dans le code source

Versionner le pipeline comme le reste du code.

✔️ Utiliser les credentials

Jamais de mots de passe en clair : plugin "Credentials Binding".

✔️ Agents & parallélisme

Diviser les tests pour accélérer le pipeline.

✔️ Notifications

Envoyer un statut sur Slack / Email en cas d'échec.
