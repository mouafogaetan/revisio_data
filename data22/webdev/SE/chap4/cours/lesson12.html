UML en bref

Unified Modeling Language est un langage de modélisation graphique standardisé. Il permet de représenter les différentes vues d’un système logiciel : besoins, structure, comportement, déploiement.

Créé dans les années 1990 par Grady Booch, James Rumbaugh et Ivar Jacobson (les « trois amigos »), UML est aujourd’hui maintenu par l’OMG (Object Management Group).

✔️ Indépendant des langages de programmation, il facilite la communication entre développeurs, analystes, chefs de projet et clients.

📊 Les deux grandes familles UML

🏗️ Structurels (statiques)

  • Diagramme de classes (attributs, méthodes, relations)
  • Diagramme de composants (modules logiciels)
  • Diagramme de déploiement (matériel, serveurs, nœuds)
  • Diagramme d'objets (instances à un instant T)
  • Diagramme de paquetages

⚡ Comportementaux (dynamiques)

  • Diagramme de séquence (interactions temporelles)
  • Diagramme d'activité (workflow, algorithmes)
  • Diagramme d'états-transitions (cycle de vie d’un objet)
  • Diagramme de cas d'utilisation (besoins fonctionnels)
  • Diagramme de communication
🔍 Un projet complet utilise plusieurs diagrammes complémentaires.

📌 Exemple : Diagramme de classes UML

Représentation d'une boutique en ligne simplifiée.

classDiagram class Utilisateur { +String nom +String email +seConnecter() +seDeconnecter() } class Client { +adresseLivraison +passerCommande() } class Admin { +gererProduits() } class Produit { +String nom +double prix +int stock +afficherDetails() } class Commande { +Date date +double total +valider() } Utilisateur <|-- Client Utilisateur <|-- Admin Client "1" -- "0..*" Commande Commande "1" -- "1..*" Produit

💡 Les flèches représentent l’héritage (triangle vide) et les associations (multiplicités).

🔄 Exemple : Diagramme de séquence

Interaction entre un client et le système pour se connecter.

sequenceDiagram participant Client participant Application participant BaseDeDonnees Client->>Application: Saisir identifiants Application->>BaseDeDonnees: Vérifier credentials BaseDeDonnees-->>Application: Succès / Échec alt authentification réussie Application-->>Client: Redirection vers tableau de bord else échec Application-->>Client: Message d'erreur end

🛠️ Outils UML populaires :

  • Lucidchart / Draw.io : en ligne, collaboratif, interface glisser-déposer
  • PlantUML : text-based, intégrable dans du code (AsciiDoc, Markdown)
  • Enterprise Architect / StarUML : outils professionnels (reverse engineering, génération de code)

✅ Quand utiliser quel diagramme ?

🔹 Analyse des besoins → Diagramme de cas d’utilisation
🔹 Structure statique → Diagramme de classes
🔹 Description d’un workflow → Diagramme d’activité
🔹 Interaction entre objets → Diagramme de séquence
🔹 Comportement d’un objet → Diagramme d’états-transitions
🔹 Architecture logicielle → Diagramme de composants
⚠️ Évitez la sur-modélisation : ne créez que les diagrammes utiles pour vos parties prenantes.
