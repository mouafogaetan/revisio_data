🎯 Objectifs de la leçon

  • ✓ Distinguer les 4 types de maintenance : corrective, adaptive, perfective, préventive.
  • ✓ Comprendre le concept de dette technique et son impact sur le projet.
  • ✓ Mesurer la dette technique avec des outils comme SonarQube et l’analyse statique.
  • ✓ Appliquer les bonnes pratiques pour réduire la dette technique.
💡 La maintenance représente souvent 60 à 80% du coût total d’un logiciel sur son cycle de vie.

🛠️ Les 4 types de maintenance logicielle

🔧 Corrective

Corriger les bugs et défauts détectés après la livraison. Exemple : une calculatrice qui donne un résultat faux.

🔄 Adaptive

Adapter le logiciel aux changements d’environnement (nouveau système d’exploitation, nouvelle version de base de données).

✨ Perfective

Améliorer les performances, la maintenabilité ou ajouter de nouvelles fonctionnalités. Exemple : rendre une page web plus rapide.

🛡️ Préventive

Prévenir les futurs problèmes : refactoring, documentation, tests supplémentaires, sécurisation du code.

✏️ À retenir : La maintenance préventive réduit la dette technique.

💸 Qu’est-ce que la dette technique ?

Métaphore introduite par Ward Cunningham (co-créateur du manifeste agile) :

“Écrire du code rapidement mais de façon peu maintenable génère une dette technique qu’il faudra rembourser plus tard (refactoring, corrections).”

Causes fréquentes :
✔ Délais trop serrés
✔ Manque de tests
✔ Code dupliqué (DRY violé)
✔ Architecture non documentée
Conséquences :
✔ Ralentissement des développements
✔ Augmentation des bugs
✔ Difficulté à recruter
✔ Coût de maintenance élevé

📊 Mesurer la dette technique

On utilise des outils d’analyse statique pour détecter les anomalies, la complexité et la non-conformité.

// Exemple de métrique : complexité cyclomatique (Thomas McCabe) // Nombre de chemins indépendants dans une fonction. // Une complexité > 10 est souvent considérée comme trop élevée. function checkAccess(user, resource) { if (user.isAdmin) return true; // +1 if (user.isOwner && resource.owner === user.id) return true; // +1 if (resource.isPublic) return true; // +1 return false; // +1 } // Complexité cyclomatique = 4

📌 SonarQube : outil populaire qui agrège les métriques (bugs, vulnérabilités, code smells, couverture de tests, dette technique en jours-homme).

🔍 Autres outils : ESLint (JavaScript), Pylint (Python), Checkstyle (Java), PMD.

🧹 Réduire la dette technique

  • Refactoring régulier : améliorer la structure sans changer le comportement.
  • Tests automatisés : unitaires, intégration, E2E pour détecter les régressions.
  • Revues de code (code reviews) : détection précoce de code smells.
  • Documentation minimale mais à jour (README, commentaires utiles).
  • Intégration continue (CI) : exécuter les analyses statiques à chaque commit.
✅ Astuce : Allouer 20% du temps de chaque sprint à la réduction de la dette technique (Scrum).

🏢 Cas Cameroun : Application de gestion bancaire

Une startup fintech a développé rapidement une application de transfert d’argent. Après 2 ans :

  • Le code est devenu difficile à maintenir (dette technique élevée).
  • Chaque nouvelle fonctionnalité prend 3 fois plus de temps.
  • SonarQube détecte 450 code smells et une couverture de tests à 12%.

Solution adoptée :

Refactoring des modules critiques, augmentation progressive des tests (objectif 70%), formation des développeurs aux bonnes pratiques. Réduction de la dette technique de 40% en 6 mois.

📈 La maintenance préventive a permis de retrouver de la vélocité.
