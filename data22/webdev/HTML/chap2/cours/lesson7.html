🎯 Objectifs de la leçon

1️⃣

Intégrer des images avec la balise <img> et ses attributs (src, alt, width, height).

2️⃣

Connaître les formats d'images (JPEG, PNG, GIF, SVG, WebP) et savoir quand les utiliser.

💡 Pourquoi c'est important ?
Les images, vidéos et sons rendent vos pages web plus attractives et interactives. Un site sans médias est terne et peu engageant !

🖼️ La balise <img>

La balise <img> est une balise orpheline (pas de balise fermante). Elle sert à afficher une image.

<img src="chemin/vers/image.jpg" alt="Description de l'image" width="300" height="200">

📌 Attribut src

Chemin (URL) vers l'image. Obligatoire.

📌 Attribut alt

Texte alternatif (accessibilité, si image non chargée). Obligatoire.

📌 Attributs width et height

Dimensions en pixels. Optionnel (bonne pratique pour éviter les sauts de layout).

📌 Attribut title

Info-bulle au survol. Optionnel.

✅ Bonne pratique : Toujours ajouter l'attribut alt (même vide si l'image est décorative : alt=""). C'est essentiel pour l'accessibilité et le SEO.

📁 Les formats d'images pour le web

JPEG / JPG

Photos, images complexes. Bonne compression, pas de transparence.

PNG

Images avec transparence, logos, icônes. Sans perte mais plus lourd.

GIF

Animations simples. Limité à 256 couleurs.

SVG

Images vectorielles (logo, icônes). Redimensionnable sans perte.

WebP

Format moderne (Google). Compression supérieure à JPEG/PNG. Supporté par tous les navigateurs récents.

💡 Astuce : Utilisez des outils comme TinyPNG ou Squoosh pour compresser vos images sans perdre en qualité.

📂 Comment indiquer le chemin de l'image ?

🔗 Chemin absolu

<img src="https://www.monsite.com/images/photo.jpg" alt="...">

Utilise une URL complète. Utile pour les images externes.

📁 Chemin relatif (recommandé)

<img src="images/photo.jpg" alt="...">

Le navigateur cherche l'image dans le dossier "images" à côté de la page HTML.

⬆️ Remonter d'un dossier

<img src="../images/photo.jpg" alt="...">

../ signifie "dossier parent".

📱 Rendre les images responsives

Pour qu'une image s'adapte à la largeur de l'écran :

<img src="photo.jpg" alt="Paysage" style="max-width: 100%; height: auto;">

Ou avec CSS Tailwind :

<img src="photo.jpg" alt="..." class="w-full h-auto">

📌 Explication : max-width: 100% empêche l'image de dépasser la largeur de son conteneur. height: auto conserve les proportions.

🎯 Pour le responsive avancé : utilisez l'attribut srcset pour charger des images différentes selon la taille d'écran.

🎬 Intégrer une vidéo avec <video>

<video controls width="600" poster="vignette.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Votre navigateur ne supporte pas la vidéo.
</video>
controls : affiche les boutons play/pause/volume
autoplay : démarre automatiquement (⚠️ évitez sur mobile)
loop : rejoue en boucle
poster : image d'aperçu avant lecture

🎥 Formats supportés : MP4 (H.264) - universel, WebM (open source) - bonne compression, OGG.

🎵 Intégrer un fichier audio avec <audio>

<audio controls>
  <source src="musique.mp3" type="audio/mpeg">
  <source src="musique.ogg" type="audio/ogg">
  Votre navigateur ne supporte pas l'audio.
</audio>

Les attributs sont similaires à <video> : controls, autoplay, loop.

MP3
universel, bonne compression
OGG
open source, bonne qualité
WAV
qualité CD, fichier très lourd

💡 Astuce : Utilisez preload="metadata" pour charger seulement les infos du fichier sans télécharger tout l'audio.

📝 Récapitulatif des bonnes pratiques

✅ À faire

  • Toujours ajouter alt pour les images
  • Compresser les images avant de les mettre en ligne
  • Utiliser max-width: 100% pour les images responsives
  • Fournir plusieurs formats pour vidéo/audio
  • Ajouter l'attribut controls pour l'utilisateur

❌ À éviter

  • Images trop lourdes (ralentit le chargement)
  • Utiliser autoplay sur vidéo/audio (utilisateur agacé)
  • Oublier le texte alternatif (accessibilité)
  • Images dimensionnées avec width/height fixes non responsives

🎯 Règle d'or : Un site avec des médias optimisés charge plus vite et offre une meilleure expérience utilisateur !
