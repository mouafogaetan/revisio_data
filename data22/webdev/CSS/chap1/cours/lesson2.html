CSS en ligne (Inline)

On écrit le style directement dans l'attribut style d'une balise HTML.

<p style="color: blue; font-size: 20px;">Texte bleu avec CSS inline</p>

<div style="background: #f0f0f0; padding: 10px;">Boîte avec style direct</div>

⚠️ Inconvénients

  • Mélange fond & forme (contre la séparation)
  • Code difficile à maintenir et dupliqué
  • Poids de la page augmente
  • Spécificité trop élevée

✅ Quand l'utiliser ?

Cas très rares : tests rapides, emails HTML, ou dynamique ponctuelle via JS. En pratique, à éviter en production.

CSS interne (balise <style>)

On place les règles CSS dans l'en-tête <head> à l'intérieur d'une balise <style>.

<!DOCTYPE html>
<html>
<head>
  <style>
    body { font-family: Arial; }
    .ma-classe { color: green; }
  </style>
</head>
<body>
  <p class="ma-classe">Texte stylisé avec CSS interne</p>
</body>
</html>

👍 Avantages

  • Pas de requête HTTP supplémentaire
  • Utile pour des pages uniques ou prototypes

👎 Inconvénients

  • Non réutilisable sur plusieurs pages
  • Alourdit le HTML
💡 Idéal pour les cours ou petits projets temporaires, mais pas pour un site à plusieurs pages.
🔗

CSS externe (recommandé)

On place toutes les règles dans un fichier .css séparé, puis on le lie avec la balise <link>.

<!-- Dans le head du fichier HTML -->
<link rel="stylesheet" href="styles.css">
/* Fichier styles.css */
body {
  background-color: #f9f9f9;
  margin: 0;
}
.btn {
  background: #4f46e5;
  padding: 0.5rem 1rem;
}

⭐ Avantages majeurs

  • Réutilisable sur plusieurs pages
  • Code plus propre (séparation totale)
  • Mise en cache par le navigateur → performance
  • Maintenance facilitée

🌍 En entreprise

C'est LA méthode standard pour tous les sites modernes. Utilisée par Bootstrap, Tailwind (via CDN/fichier), et frameworks.

✅ Bonne pratique : un seul fichier CSS pour tout le site

⚖️ Comparaison & recommandations

Méthode Réutilisable Performance Usage pro
Inline❌ NonMoyen⚠️ Très rare
Interne❌ Non (une seule page)Bon📄 Pages spécifiques
Externe✅ Oui (tout le site)⭐ Excellent (cache)🏆 Standard

📌 Conseils pratiques :

  • Pour tout projet sérieux ➜ CSS externe avec un fichier dédié.
  • Pour une maquette rapide ou un exercice local : CSS interne peut convenir.
  • Évitez le CSS inline, sauf contrainte technique extrême.
  • En mode développement, séparez toujours style et structure.

📝 Synthèse de la leçon

Inline : Style dans l'attribut style. Spécifique, non maintenable. À éviter
Interne : Balise <style> dans <head>. Pour une page unique. Cas limité
Externe : Fichier .css lié avec <link>. Meilleure pratique! Recommandé

✨ Rappel: la séduction du CSS externe permet de styliser tout un site web avec un seul fichier, gain de temps et d'organisation. ✨
