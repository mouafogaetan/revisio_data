Unions, Intersections & Narrowing

TypeScript permet de combiner des types et de sécuriser le code grâce au rétrécissement de type.

🧩 Union ( | ) 🔗 Intersection ( & ) 🎯 Narrowing (typeof, in)
🎯

Objectifs de la leçon

  • ✓ Définir des types union (string | number) pour gérer plusieurs types possibles.
  • ✓ Utiliser les types intersection (A & B) pour combiner des types.
  • ✓ Maîtriser le narrowing (rétrécissement) avec typeof, instanceof et in.
  • ✓ Comprendre l’importance du narrowing pour la sécurité du code TypeScript.
💡 Saviez-vous ? TypeScript utilise le narrowing pour inférer automatiquement le type dans une branche conditionnelle.
🤝

Types Union |

Un type union permet à une variable d’être de plusieurs types. Exemple : string | number.

// Une variable peut être soit string soit number
let identifiant: string | number;
identifiant = "USER_123";   // ✅ valide
identifiant = 456;           // ✅ valide
// identifiant = true;      // ❌ Erreur : booléen non autorisé

// Dans une fonction
function afficherId(id: string | number) {
    console.log("ID :", id);
}
afficherId("abc");
afficherId(789);
⚠️ Attention : on ne peut utiliser des méthodes spécifiques à un type (comme .toUpperCase()) sans avoir fait de narrowing au préalable !

La puissance des unions réside dans la flexibilité tout en gardant le typage strict.

🧬

Intersection &

Un type intersection combine plusieurs types en un seul : l’objet doit satisfaire TOUS les types constituants.

interface Personne {
  nom: string;
  age: number;
}

interface Employe {
  salaire: number;
  poste: string;
}

type EmployeDetail = Personne & Employe;

let chef: EmployeDetail = {
  nom: "Alice",
  age: 32,
  salaire: 55000,
  poste: "Lead Dev"
};  // ✅ Toutes les propriétés sont requises
🌟 Cas pratique : L'intersection est parfaite pour fusionner des contrats (API, configuration étendue).
🔍

Narrowing : rétrécissement de type

Le narrowing est le processus par lequel TypeScript affine un type union à un type plus spécifique à l'intérieur d'une branche conditionnelle.

function traiterValeur(valeur: string | number) {
  if (typeof valeur === "string") {
    // TypeScript sait que ici 'valeur' est string
    console.log(valeur.toUpperCase());
  } else {
    // Ici 'valeur' est forcément number
    console.log(valeur.toFixed(2));
  }
}
typeof instanceof in égalité stricte

🛠️ Techniques avancées

🧩 typeof

if (typeof data === "string") { ... }

Utile pour les primitifs (string, number, boolean).

📦 in

if ("vol" in oiseau) { ... }

Vérifie l'existence d'une propriété (utile pour objets).

🎭 instanceof

if (err instanceof Error) { ... }

Teste la chaîne de prototypes (classes).

⚖️ truthiness narrowing

if (message) { ... }

Élimine null/undefined/0/"" automatiquement.

🏗️

Application réelle : Formulaire typé

type Chat = { miauler: () => void; nom: string };
type Chien = { aboyer: () => void; race: string };

type Animal = Chat | Chien;

function interagir(animal: Animal) {
  // Narrowing via la propriété spécifique
  if ("miauler" in animal) {
    console.log(`🐱 ${animal.nom} miaule !`);
    animal.miauler();
  } else {
    console.log(`🐶 ${animal.race} aboie !`);
    animal.aboyer();
  }
}

// Simulation
const monChat: Chat = { miauler: () => alert("Miaou ❤️"), nom: "Felix" };
interagir(monChat);

✔️ Le narrowing par `in` permet de différencier deux objets sans avoir de champ discriminant commun.

📌 En résumé

  • ✅ Union (|) : un OU plusieurs types — maximum de flexibilité.
  • ✅ Intersection (&) : fusionne les exigences — toutes les propriétés requises.
  • ✅ Le narrowing est indispensable avant d’utiliser des méthodes spécifiques à un type.
  • ✅ Privilégiez typeof / in / instanceof pour assurer la sécurité.
🧪 À retenir : TypeScript n’autorise pas l’accès à une propriété non garantie. Le narrowing est votre allié pour écrire du code robuste !
