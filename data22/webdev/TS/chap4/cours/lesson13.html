Interface vs Type Alias

Comparer, comprendre et choisir la bonne structure pour vos données.

📘 2 concepts clés ⚡ Déclaration merging 🎯 Choix contextuel

🎯 Objectifs de la leçon

✓ Créer des alias pour types primitifs, unions et tuples.
✓ Comparer interface vs type : ressemblances et différences.
✓ Comprendre le déclaration merging (fusion automatique des interfaces).
✓ Savoir choisir entre interface et type selon le contexte.
💡 Pourquoi c'est important ? En TypeScript, vous allez constamment décrire la forme des objets. Bien maîtriser ces outils rend le code plus propre et maintenable.

📌 Type Alias : donner un nom à n'importe quel type

Le mot-clé type permet de créer un alias (surnom) pour n'importe quelle combinaison de types : primitifs, unions, tuples, objets, etc.

// Types primitifs
type Identifiant = string | number;
type EstMajeur = boolean;

// Unions & intersections
type Statut = "actif" | "inactif" | "suspendu";
type Coordonnees = { x: number; y: number };

// Tuples nommés
type CoordGPS = [number, number];   // latitude, longitude

// Objets complexes
type Utilisateur = {
  nom: string;
  age: number;
  adresse?: string;   // optionnel
  readonly id: number;
};

✅ Avantage : on évite la répétition et on améliore la lisibilité.

🧩 Interface : décrire des objets / classes

interface est spécialement conçue pour décrire la forme d'un objet et peut être étendue ou implémentée par des classes.

interface Personne {
  nom: string;
  age: number;
  saluer(): void;
}

// Extension possible
interface Etudiant extends Personne {
  matricule: string;
  filiere: string;
}

const etu: Etudiant = {
  nom: "Tchio Fabrice",
  age: 22,
  matricule: "CAM0245",
  filiere: "Informatique",
  saluer() { console.log(`Bonjour, je suis ${this.nom}`); }
};

⭐ Les interfaces sont ouvrables (déclaration merging) : on peut les redéclarer pour ajouter des champs.

🧠 À retenir :

Utilisez interface pour les objets/classes que d'autres pourront étendre (API, librairies). Privilégiez type pour les unions, tuples, types primitifs, ou quand vous avez besoin de mappers conditionnels.

🔁 Déclaration merging : la force des interfaces

TypeScript fusionne automatiquement les interfaces portant le même nom. Pratique pour étendre des types existants (ex: ajouter des propriétés à une interface d’une bibliothèque).

interface Personne {
  nom: string;
}

// Redéclaration : fusion
interface Personne {
  age: number;
}

// Résultat final : Personne a { nom: string; age: number }
const moi: Personne = { nom: "Minka", age: 28 };
// Avec type, erreur
type Animal = { espece: string };
type Animal = { poids: number }; // ❌ Duplicate identifier 'Animal'

✨ Les types ne supportent pas la fusion. C’est un critère déterminant pour choisir interface quand on veut une extensibilité publique.

🌍 Cas pratique : Gestion d’une école

Nous devons typer des données d’étudiants, professeurs et matières. Utilisons à la fois interface et type pour montrer le bon choix.

📘 Interface (objet métier extensible) 
interface PersonneEcole {
  nom: string;
  email: string;
}

interface Eleve extends PersonneEcole {
  classe: string;
  notes: number[];
}

interface Professeur extends PersonneEcole {
  matieres: string[];
}
🔖 Type alias (union et utilité) 
type Niveau = "6ème" | "5ème" | "4ème" | "3ème";
type StatutInscription = "active" | "suspendue" | "expulsée";

type IdentifiantUnique = string | number;  // utile pour ID carte scolaire

type ResultatBilan = {
  eleve: Eleve;
  moyenne: number;
  statut: StatutInscription;
};

✅ Stratégie gagnante : interface pour les formes d’objets principaux, type pour les unions, utilitaires ou réutilisation de combinaisons.

✏️ Exercice : Choisir la bonne approche

Dans un projet de livraison de repas à Douala, vous devez typer les structures suivantes. Dites pour chaque cas si vous utilisez interface ou type et justifiez brièvement.

1️⃣ Un objet CoordonneesGPS avec lat et lng (nombres). Ce type doit pouvoir être étendu plus tard par d'autres modules pour ajouter altitude.

✅ Interface : car on souhaite bénéficier du déclaration merging pour étendre facilement.

2️⃣ Un alias représentant soit une chaîne (code promo), soit un nombre (montant), soit un tableau de produits.

✅ Type alias : parce que c’est une union (string | number | Produit[]). Les interfaces ne peuvent pas représenter des unions.

3️⃣ Une Classe représentant un cours avec des propriétés nom, credit et une méthode afficher(). Cette structure sera implémentée par plusieurs types d’activités pédagogiques.

✅ Interface (ou type objet) mais interface est plus idiomatique pour l'implémentation par des classes (implements) et l’extension.

📖 Bonnes pratiques : Interface vs Type

  • 📌 Par défaut, préférez interface pour décrire des objets – c’est plus conventionnel, meilleur pour l'auto-complétion et les messages d'erreur.
  • ⚡ Utilisez type quand vous avez besoin d’unions, tuples, types primitifs ou types conditionnels avancés.
  • 🔁 Déclaration merging : si vous construisez une bibliothèque ou un système modulaire où des utilisateurs pourront ajouter des champs, utilisez interface.
  • 🧩 Les deux peuvent décrire des objets, mais l'interface est plus « orientée objet » et extensible naturellement.

🎯 Règle d'or de la communauté TypeScript : “Utilisez interface jusqu'à ce que vous ayez besoin d’une fonctionnalité propre à type.”
