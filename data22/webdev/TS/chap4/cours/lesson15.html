🤔 Le problème des objets dynamiques

On a souvent besoin de stocker des données dont les clés ne sont pas connues à l'avance (ex: traduction, configuration, stockage clé-valeur).

// Sans TypeScript : n'importe quelle clé est acceptée
const translations: any = {};
translations.fr = "Bonjour";
translations.en = "Hello";
translations.es = "Hola";

// Avec TypeScript classique : impossible de typer dynamiquement
// interface Translations {
//   fr: string;
//   en: string;
//   es: string;
// } // ❌ pas flexible

Solution : les index signatures !

📝 Index Signatures

interface Translations {
  [key: string]: string;  // Toute clé de type string a une valeur string
}

const t: Translations = {
  fr: "Bonjour",
  en: "Hello",
  es: "Hola"
};

// ✅ Accès dynamique OK
t.de = "Guten Tag";
console.log(t.fr);  // "Bonjour"

// ❌ Erreur : la valeur doit être une string
t.jp = 42;  // Type 'number' non assignable à 'string'
✔️ Utile pour les dictionnaires, les configurations, les caches.

🔑 Types autorisés pour les index

Les index signatures peuvent utiliser string ou number comme type de clé.

// Clés de type string
interface StringMap {
  [key: string]: number;
}

// Clés de type number (tableau associatif numérique)
interface NumberMap {
  [index: number]: string;
}

const arr: NumberMap = ["a", "b", "c"];
console.log(arr[0]);  // "a" ✅

// ⚠️ JavaScript convertit automatiquement les clés number en string
// TypeScript les traite séparément

🔧 Mixer propriétés fixes et dynamiques

interface UserPreferences {
  // Propriétés fixes
  theme: 'light' | 'dark';
  notifications: boolean;
  
  // Propriétés dynamiques supplémentaires
  [key: string]: string | boolean | number;
}

const prefs: UserPreferences = {
  theme: 'dark',
  notifications: true,
  language: 'fr',      // ✅ OK (valeur string)
  fontSize: 14         // ✅ OK (valeur number)
};

// ❌ prefs.unknown = new Date(); // Erreur : Date n'est pas dans l'union
⚠️ Le type de l'index doit être compatible avec tous les types des propriétés fixes.

📦 Le type Record<K, V>

Record<K, V> est un type utilitaire qui construit un type objet dont les clés sont de type K et les valeurs de type V.

// Record avec des clés littérales
type UserRoles = Record<'admin' | 'user' | 'guest', string>;
// Équivalent à :
// type UserRoles = {
//   admin: string;
//   user: string;
//   guest: string;
// }

const roles: UserRoles = {
  admin: "Administrateur",
  user: "Utilisateur",
  guest: "Invité"
};

// Record avec des clés string quelconques
type StringNumberMap = Record<string, number>;
const map: StringNumberMap = { a: 1, b: 2, c: 3 };

⚖️ Record<K,V> vs { [key: string]: V }

\
CritèreIndex SignatureRecord<K,V>
Clésstring ou numberN'importe quel type (union, enum...)
Clés connuesNon (toutes les clés possibles)Oui (on peut limiter aux clés de l'union)
Syntaxe{ [key: string]: T }Record<K, T>
Exemple{ [key: string]: number }Record<string, number>

Préférez Record quand les clés sont connues (union), sinon l'index signature.

🔄 Mapped Types : introduction

Un mapped type permet de transformer un type existant en un nouveau type, en appliquant une transformation à chaque propriété.

// Syntaxe de base : [P in keyof T]: T[P]
type ReadOnly<T> = {
  readonly [P in keyof T]: T[P];
};

interface User {
  name: string;
  age: number;
}

type ReadonlyUser = ReadOnly<User>;
// Équivalent à :
// interface ReadonlyUser {
//   readonly name: string;
//   readonly age: number;
// }

📚 Mapped Types standards (built-in)

Partial<T> : rend toutes les propriétés optionnelles
Required<T> : rend toutes les propriétés obligatoires
Readonly<T> : rend toutes les propriétés en lecture seule
Pick<T, K> : sélectionne un sous-ensemble de propriétés
Omit<T, K> : exclut certaines propriétés
interface User {
  id: number;
  name: string;
  email: string;
  age: number;
}

type UserPreview = Pick<User, 'id' | 'name'>;  // { id: number; name: string }
type UserWithoutEmail = Omit<User, 'email'>; // { id: number; name: string; age: number }
type PartialUser = Partial<User>; // Tout est optionnel

🛠️ Créer des Mapped Types personnalisés

// Rendre toutes les propriétés optionnelles ET readonly
type ReadonlyPartial<T> = {
  readonly [P in keyof T]?: T[P];
};

// Transformer les types des propriétés (ex: toutes en string)
type Stringify<T> = {
  [P in keyof T]: string;
};

interface Product {
  id: number;
  name: string;
  price: number;
}

type StringProduct = Stringify<Product>;
// { id: string; name: string; price: string; }
🔧 Les mapped types sont très utilisés dans les bibliothèques comme Redux, Prisma, etc.

✏️ Modificateurs +? , -? , +readonly

// Supprimer le readonly (si présent)
type Mutable<T> = {
  -readonly [P in keyof T]: T[P];
};

// Rendre toutes les propriétés obligatoires (enlever le ?)
type Concrete<T> = {
  [P in keyof T]-?: T[P];
};

interface Config {
  url?: string;
  timeout?: number;
}

type RequiredConfig = Concrete<Config>;
// { url: string; timeout: number; }

+? ajoute l'optionnalité, -? l'enlève. Idem pour +readonly / -readonly.

🌐 Exemple concret : API REST

// Définition des endpoints
type ApiEndpoints = {
  '/users': { id: number; name: string }[];
  '/user/:id': { id: number; name: string; email: string };
  '/posts': { id: number; title: string }[];
};

// Type générique pour typer les réponses
type ApiResponse<T extends keyof ApiEndpoints> = {
  data: ApiEndpoints[T];
  status: number;
};

// Utilisation
async function fetchApi<T extends keyof ApiEndpoints>(
  url: T
): Promise<ApiResponse<T>> {
  // implémentation
  return { data: {} as any, status: 200 };
}

const users = await fetchApi('/users');
// users.data est typé comme { id: number; name: string }[] ✅

✅ Bonnes pratiques

  • Utilisez Record<K,V> quand les clés sont un ensemble connu (union).
  • Utilisez les index signatures pour les dictionnaires génériques.
  • Préférez les mapped types intégrés (Partial, Pick, Omit) à les recoder.
  • Évitez les index signatures trop permissives si vous pouvez définir un type plus précis.
  • Combinez keyof et les mapped types pour des transformations puissantes.

📌 Ce qu'il faut retenir

✓ [key: string]: Type pour propriétés dynamiques
✓ Record<K, V> : objet avec clés de type K, valeurs V
✓ Mapped types : { [P in keyof T]: T[P] }
✓ Types utilitaires : Partial, Pick, Omit, Readonly
✓ Modificateurs +?, -?, +readonly
✓ Puissant pour typer des API ou des transformations
🌟 Maîtrisez ces outils pour des types très flexibles !
