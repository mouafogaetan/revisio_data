Analyse et correction de code (6 points)

🐛 Exercice 1 : Identifier et corriger les erreurs (3 pts)

Le code TypeScript suivant contient plusieurs erreurs. Identifiez-les et proposez une version corrigée.

class Animal {
  private name: string;
  
  constructor(name: string) {
    this.name = name;
  }
  
  private makeSound(): void {
    console.log(`${this.name} fait un bruit`);
  }
}

class Dog extends Animal {
  constructor(name: string, private breed: string) {
    this.name = name;  // ERREUR 1
    this.breed = breed;
  }
  
  public bark(): void {
    this.makeSound();  // ERREUR 2
    console.log("Woof!");
  }
}

const dog = new Dog("Rex", "Labrador");
dog.makeSound();      // ERREUR 3
dog.name = "Max";     // ERREUR 4
  1. Expliquez chaque erreur (4 erreurs). (2 pts)
  2. Réécrivez le code corrigé. (1 pt)

📊 Exercice 2 : Implémenter une classe avec getters/setters (3 pts)

Écrivez une classe BankAccount qui respecte les spécifications suivantes :

  • Propriété privée _balance (number)
  • Propriété readonly owner (string)
  • Constructeur prenant owner et balance initiale optionnelle (par défaut 0)
  • Méthode deposit(amount: number) : ajoute à la balance (ne peut pas être négatif)
  • Méthode withdraw(amount: number) : retire si solde suffisant, sinon erreur
  • Getter balance qui retourne le solde (lecture seule)
  • Getter summary qui retourne une chaîne du type "Compte de [owner] : [balance] FCFA"
III

Étude de cas : Gestion d'une bibliothèque (6 points)

Vous devez modéliser en TypeScript le système de gestion d'une bibliothèque avec les contraintes suivantes :

  • Un Document a un titre, un auteur, un numéro d'identification (id) en lecture seule.
  • Un Livre est un Document avec un nombre de pages (number) et un ISBN.
  • Un DVD est un Document avec une durée en minutes (number).
  • Une classe abstraite LibraryItem définit une méthode abstraite getLoanDuration(): number (livre = 21 jours, DVD = 7 jours).
  • Une classe Library contient un tableau d'items et des méthodes pour ajouter un item, lister tous les titres et trouver les documents d'un auteur donné.
  1. Écrivez l'interface ou la classe de base Document avec les propriétés communes et l'id en lecture seule. (1 pt)
  2. Écrivez les classes Book et DVD qui héritent de Document et implémentent LibraryItem. (1.5 pts)
  3. Écrivez la classe Library avec ses méthodes. (2 pts)
  4. Ajoutez un exemple d'utilisation créant une bibliothèque, ajoutant deux livres et un DVD, puis affichant tous les titres. (1.5 pts)
