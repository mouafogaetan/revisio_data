Objectifs de la leçon

💡 Les namespaces sont utiles pour organiser du code côté navigateur sans bundler. Les modules ES6 sont recommandés pour les projets modernes.

📦 1. Les namespaces

Un namespace (anciennement "modules internes") permet de regrouper du code (variables, fonctions, classes) sous un nom commun pour éviter les pollutions de portée globale.

// Déclaration d'un namespace
namespace Geometrie {
    export const PI = 3.14159;
    
    export function aireCercle(rayon: number): number {
        return PI * rayon * rayon;
    }
    
    export class Carre {
        constructor(public cote: number) {}
        aire(): number {
            return this.cote * this.cote;
        }
    }
}

// Utilisation
console.log(Geometrie.PI);
console.log(Geometrie.aireCercle(5));
const petitCarre = new Geometrie.Carre(4);
console.log(petitCarre.aire());
⚠️ Le mot-clé export est obligatoire pour ce qui doit être accessible en dehors du namespace.

🗂️ 2. Namespaces imbriqués & répartis

On peut imbriquer des namespaces et même les répartir sur plusieurs fichiers (fusion automatique).

// Imbrication
namespace Univers {
    export namespace Terre {
        export const nom = "Terre";
        export function habitants(): number {
            return 8_000_000_000;
        }
    }
}

console.log(Univers.Terre.nom);

// Même namespace réparti (dans plusieurs fichiers)
// Fichier A.ts
namespace Utils {
    export function formaterPrix(prix: number): string {
        return `${prix} FCFA`;
    }
}
// Fichier B.ts (même namespace)
namespace Utils {
    export function majuscule(texte: string): string {
        return texte.toUpperCase();
    }
}
// Les deux fonctions sont disponibles dans Utils
✨ Grâce à la fusion de déclaration, plusieurs fichiers peuvent contribuer au même namespace.

🔗 3. Directives triple-slash (/// <reference />)

Les directives triple-slash indiquent au compilateur des dépendances entre fichiers. Elles sont utiles avec les namespaces.

// fichier maths.ts
namespace Maths {
    export function addition(a: number, b: number): number {
        return a + b;
    }
}

// fichier principal.ts
/// <reference path="maths.ts" />

namespace Maths {
    export function multiplication(a: number, b: number): number {
        return a * b;
    }
}

console.log(Maths.addition(5, 3));      // 8
console.log(Maths.multiplication(4, 2)); // 8
⚠️ Les directives triple-slash sont une fonctionnalité legacy. Préférez les modules ES6 (import/export) pour les nouveaux projets.

⚖️ 4. Namespaces vs Modules ES6

🏷️ Namespaces

  • Code global, pas besoin de bundler
  • Compilation avec --outFile
  • Fusion automatique sur plusieurs fichiers
  • Utilisation avec directives ///
  • Ancienne méthode (avant ES6)

📦 Modules ES6

  • Portée par fichier, chargement asynchrone
  • Nécessite un bundler (webpack, etc.)
  • Syntaxe import / export
  • Standard JavaScript moderne
  • Recommandé pour tous les nouveaux projets
💡 Conseil : Pour une application web moderne, utilisez les modules ES6. Les namespaces restent utiles pour des scripts simples ou des bibliothèques globales.

🏗️ 5. Exemple concret : utilitaire global

// helpers.ts
namespace Helpers {
    export function formatDate(date: Date): string {
        return `${date.getDate()}/${date.getMonth()+1}/${date.getFullYear()}`;
    }
    export const VERSION = "1.0.0";
}

// app.ts
/// <reference path="helpers.ts" />

namespace App {
    export function afficherVersion() {
        console.log(`Version ${Helpers.VERSION}`);
    }
    export function afficherDate() {
        console.log(Helpers.formatDate(new Date()));
    }
}

App.afficherVersion();
App.afficherDate();
🎯 Compilation : tsc --outFile bundle.js app.ts fusionne tous les fichiers en un seul.

🧪 Vérifions vos acquis

Quelle est la principale raison de préférer les modules ES6 aux namespaces aujourd'hui ?

  • A) Les namespaces ne permettent pas l'export de fonctions.
  • B) Les modules ES6 sont supportés nativement par les navigateurs et permettent le chargement asynchrone.
  • C) Les namespaces sont plus lents à l'exécution.
  • D) Il n'y a aucune différence.
✅ Réponse B : Les modules ES6 sont la norme moderne, supportés nativement (import/export), permettent le tree-shaking et le chargement asynchrone. Les namespaces produisent du code global et sont moins adaptés aux applications complexes.
📌 À retenir : utilisez export et import pour les projets modernes. Les namespaces restent utiles pour le code legacy ou les scripts sans bundler.

📌 En résumé

✓ Namespace : regroupement logique de code, évite la pollution globale.
✓ Directive /// <reference path="..." /> pour lier les fichiers.
✓ Modules ES6 : standard actuel, import/export, chargement asynchrone.
✓ Namespaces → scripts simples / bibliothèques globales. Modules → applications modernes.
