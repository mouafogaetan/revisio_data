⚙️ allowJs & checkJs

Deux options clés dans tsconfig.json pour accepter et vérifier les fichiers JavaScript.

{
  "compilerOptions": {
    "allowJs": true,      // autorise les fichiers .js dans le projet
    "checkJs": true,      // vérifie les types même dans les .js (suggestion)
    "outDir": "./dist",
    "strict": false       // on peut démarrer en mode moins strict
  },
  "include": ["src/**/*"]
}
✅ allowJs : on peut importer des fichiers JS depuis du TS.
🔍 checkJs : TypeScript analyse vos .js et remonte des erreurs potentielles.

📝 Directives commentées

On peut activer/désactiver la vérification par fichier avec des commentaires spéciaux.

// @ts-nocheck
// Ce fichier ne sera PAS vérifié par TypeScript
function addition(a, b) { return a + b; }

// @ts-check
// À partir d'ici, TypeScript analyse le code
const x = "10";
const y = x * 2;   // ❌ avertissement : nombre vs chaîne
🧠 Bon à savoir : // @ts-nocheck est parfait pour les vieux fichiers complexes qu'on migrera plus tard.

📌 Ajouter des types dans du JS pur

Même sans TypeScript, on peut documenter les types avec JSDoc – checkJs les utilise !

/**
 * Calcule le prix TTC au Cameroun (taux TVA 19.25%)
 * @param {number} prixHT - Prix hors taxes
 * @returns {number} Prix TTC
 */
function calculerTTC(prixHT) {
  const TVA = 0.1925;
  return prixHT * (1 + TVA);
}

// ✅ TypeScript infère que prixHT doit être un nombre
calculerTTC(1000);   // OK
calculerTTC("1000"); // ⚠️ avertissement
📘 JSDoc + checkJs = introduction douce au typage, sans changer l'extension.

📄 Les fichiers .d.ts

Fichiers décrivant la forme (signatures) de code JavaScript existant. Ils ne contiennent pas d'implémentation.

// types.d.ts
declare module "mon-utilitaire" {
  export function formaterDate(date: Date, format?: string): string;
  export const VERSION: string;
}

// Dans un fichier TS
import { formaterDate } from "mon-utilitaire";
formaterDate(new Date(), "DD/MM/YYYY"); // typé !
⚠️ Quand une librairie n'a pas de types (@types/), on peut créer notre propre .d.ts à la main.

🌍 Déclaration pour variables globales

Pour une librairie attachée à window (ex: jQuery, moment, ...) non typée :

// globals.d.ts
interface Window {
  monAnalytics: {
    envoyerEvent(nom: string, valeur?: number): void;
  };
}

// Maintenant TypeScript connaît window.monAnalytics
window.monAnalytics.envoyerEvent("clic_bouton", 1);

🚀 Plan de migration pratique

  1. Ajouter tsconfig.json avec allowJs: true, checkJs: false
  2. Changer progressivement checkJs: true pour détecter les problèmes
  3. Ajouter // @ts-check dans les fichiers critiques
  4. Convertir un fichier .js → .ts et corriger les erreurs de typage
  5. Créer des .d.ts pour les dépendances sans types
🔁 "On peut avoir 50% de fichiers TS et 50% JS – et compiler le tout sans problème."

🧩 Typer un module externe sans @types

Si une librairie n'existe pas sur DefinitelyTyped, on peut la typer rapidement.

// libs.d.ts
declare module "bibliotheque-ancienne" {
  export function init(config: { apiKey: string }): void;
  export function fetchData(endpoint: string): Promise<any>;
}

// usage dans le code
import { init, fetchData } from "bibliotheque-ancienne";
init({ apiKey: "abc123" });  // TypeScript valide
📦 @types/ est la première source, sinon on écrit notre surcharge.
