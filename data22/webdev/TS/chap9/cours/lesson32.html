TypeScript côté serveur

Installer @types/node, utiliser ts-node et configurer nodemon pour un développement efficace

📦 @types/node 🚀 ts-node 🔄 nodemon
👆 Glissez ou utilisez les boutons 👆

🎯 Objectifs

  • ✔️ Installer @types/node pour bénéficier des types Node.js
  • ✔️ Utiliser ts-node pour exécuter directement du TypeScript
  • ✔️ Configurer nodemon avec ts-node pour le rechargement automatique

💡 Pourquoi ? TypeScript apporte la sécurité des types même côté serveur, réduisant les bugs et améliorant la productivité.
Types pour Node.js

📦 @types/node

Node.js étant écrit en JavaScript, TypeScript ne connaît pas ses API natives (fs, path, http, etc.). Les définitions de types comblent ce manque.

# Installation
npm install --save-dev @types/node

# Exemple d'utilisation
import * as fs from 'fs';
import * as path from 'path';

const filePath = path.join(__dirname, 'data.txt');
// TypeScript connaît les signatures des méthodes !
fs.readFile(filePath, 'utf-8', (err, data) => {
  if (err) {
    console.error('Erreur:', err);
    return;
  }
  console.log(data);
});
📘 Remarque : @types/node fournit aussi les types pour process, Buffer, setTimeout, etc.
Exécution sans compilation

🚀 ts-node

ts-node exécute les fichiers TypeScript directement, sans passer par l'étape tsc. Idéal pour le développement !

# Installation globale
npm install -g ts-node

# Ou locale dans le projet
npm install --save-dev ts-node

# Exécuter un fichier TypeScript
ts-node mon-script.ts

# Exemple avec un serveur simple
ts-node server.ts
✅ Avantage : Pas besoin de compiler avant chaque test
⚠️ Inconvénient : Plus lent qu'exécuter du JS compilé (production)
💡 Astuce : En production, on compile avec tsc et on exécute le JS généré.
Rechargement automatique

🔄 nodemon + ts-node

nodemon surveille les fichiers et redémarre automatiquement le serveur à chaque modification.

# Installation
npm install --save-dev nodemon ts-node

# Ajouter un script dans package.json
{
  "scripts": {
    "dev": "nodemon --exec ts-node src/server.ts",
    "dev:watch": "nodemon --watch src --ext ts --exec ts-node src/server.ts"
  }
}
$ npm run dev
[nodemon] starting `ts-node src/server.ts`
🚀 Serveur démarré sur http://localhost:3000
[nodemon] restarting due to changes...
[nodemon] starting `ts-node src/server.ts`
Projet concret

🏗️ Serveur Express typé

# Installation des dépendances
npm init -y
npm install express
npm install --save-dev typescript @types/node @types/express ts-node nodemon

# tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true
  }
}
// src/server.ts
import express, { Request, Response } from 'express';

const app = express();
const port = 3000;

// Interface pour typage des données
interface Utilisateur {
  id: number;
  nom: string;
  email: string;
}

app.get('/utilisateurs', (req: Request, res: Response<Utilisateur[]>) => {
  const users: Utilisateur[] = [
    { id: 1, nom: "Alice", email: "alice@example.com" },
    { id: 2, nom: "Bob", email: "bob@example.com" }
  ];
  res.json(users);
});

app.listen(port, () => {
  console.log(`Serveur démarré sur http://localhost:${port}`);
});
Personnalisation

⚙️ Fichier nodemon.json

// nodemon.json
{
  "watch": ["src"],
  "ext": "ts,json",
  "ignore": ["src/**/*.spec.ts"],
  "exec": "ts-node src/server.ts",
  "env": {
    "NODE_ENV": "development",
    "PORT": "3000"
  },
  "delay": 500
}

// package.json
{
  "scripts": {
    "dev": "nodemon",
    "build": "tsc",
    "start": "node dist/server.js"
  }
}
📁 watch : dossiers surveillés
🔌 ext : extensions à surveiller
🚫 ignore : fichiers ignorés
⏱️ delay : délai avant redémarrage
Débogage

🐛 Déboguer avec ts-node

# Lancer avec inspection (Chrome DevTools)
node --inspect -r ts-node/register src/server.ts

# Ou avec nodemon pour le rechargement + debug
nodemon --inspect --exec ts-node src/server.ts

# Dans VS Code - launch.json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Démarrer serveur TS",
      "runtimeArgs": ["-r", "ts-node/register"],
      "args": ["${workspaceFolder}/src/server.ts"]
    }
  ]
}
💡 Conseil : Utilisez console.log ou les breakpoints directement dans les fichiers .ts !
