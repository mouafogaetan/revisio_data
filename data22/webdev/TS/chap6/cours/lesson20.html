Les génériques en TypeScript

Créer des composants réutilisables et typés

📘 Lesson 20 · Initiation au TypeScript

🎯 Objectifs de la leçon

1️⃣ Comprendre le problème résolu par les génériques : éviter le recours au type any tout en conservant la réutilisabilité.
2️⃣ Écrire une fonction générique avec la syntaxe <T> et exploiter l’inférence de type.
3️⃣ Utiliser plusieurs paramètres génériques <T, U> pour des fonctions manipulant des types hétérogènes.

🧩 Pourquoi avons-nous besoin des génériques ?

Imaginons une fonction "identité" qui retourne ce qu’on lui passe. Sans génériques, on aurait deux solutions imparfaites :

❌ Utiliser any

function identity(value: any): any {
  return value;
}
const result = identity("coucou");
result.toUpperCase(); // pas d'erreur, mais perte d'info type

⚠️ On perd le typage précis (pas de vérification à la compilation).

❌ Surcharger manuellement

function identity(val: string): string;
function identity(val: number): number;
// ...impossible pour tous les types

⚠️ Pas évolutif, code dupliqué.

💡 Le besoin : une fonction qui préserve le type d'entrée SANS sacrifier la sécurité. → Les génériques !

⚙️ Écrire une fonction générique

Syntaxe : <TypeParam> avant la liste des paramètres.

// Version générique : le type T est capturé
function identity<T>(value: T): T {
  return value;
}

// Utilisation
let str = identity<string>("Hello");   // T = string
let num = identity(42);                // Inférence : T = number
let bool = identity(true);             // inféré boolean
🔍 Explication : Le paramètre T est un marqueur qui sera remplacé par le type réel lors de l’appel. La fonction retourne exactement le même type que celui reçu.

🔁 Plusieurs types génériques : <T, U>

Lorsqu’une fonction manipule deux valeurs de types distincts, on peut utiliser deux paramètres génériques indépendants.

function fusion<T, U>(objet1: T, objet2: U): T & U {
  return { ...objet1, ...objet2 };
}

const personne = { nom: "Alice" };
const adresse = { ville: "Douala" };
const profil = fusion(personne, adresse);
// Type de profil : { nom: string } & { ville: string }
✅ Avantage : TypeScript calcule le type de retour comme l’intersection des types. Les génériques T et U sont indépendants.

🧠 Inférence automatique des types

TypeScript est assez intelligent pour deviner les types génériques. On peut donc omettre les <> dans la plupart des cas.

// Pas besoin de préciser <string> explicitement
const message = identity("TypeScript génial !"); 
// message est de type string

function fusionSimple<T,U>(a: T, b: U): [T, U] {
  return [a, b];
}
const tuple = fusionSimple(100, "cours");  // inféré [number, string]

🎓 Conseil : Laissez l’inférence faire son travail ! Utilisez les paramètres explicites seulement quand l’inférence échoue (cas complexes).

🌍 Exemple : fonction réutilisable pour les appels API

async function fetchData<T>(url: string): Promise<T> {
  const response = await fetch(url);
  const data: T = await response.json();
  return data;
}

// Typage précis de la réponse
interface Utilisateur {
  id: number;
  name: string;
  email: string;
}

// Appel avec le type attendu
const user = await fetchData<Utilisateur>('/api/user/1');
// user.name est typé string, auto-complétion assurée.
📦 Générique utile : fetchData<T> s’adapte à toutes les formes de données retournées (produits, utilisateurs, commandes…)

📌 Résumé & Vérification

✅ Les génériques permettent de créer des composants réutilisables tout en conservant le typage.

✅ Syntaxe : function nom<T>(arg: T): T {} ou const nom = <T>(arg:T):T => arg.

✅ Plusieurs génériques : <K, V>, <T, U, V> selon les besoins.

✅ TypeScript infère souvent les types, rendant le code plus concis.

🧪 Petit test mental :

Quelle est l’utilité de <T> dans function wrap<T>(val: T): { value: T } ?

Réponse : T capture le type de l’argument passé, et la fonction retourne un objet dont la propriété value est du même type T. Sans générique, on perdrait la précision du type.
