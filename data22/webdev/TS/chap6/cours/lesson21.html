💡 Rappel : Un générique permet de créer des composants réutilisables tout en gardant la sécurité des types.
Contrainte de forme

📏 T extends Interface

On peut exiger que le type générique ait certaines propriétés ou méthodes.

interface AvecLongueur {
  length: number;
}

// T doit avoir une propriété 'length' (comme string, Array, etc.)
function logLength<T extends AvecLongueur>(element: T): T {
  console.log(`Longueur : ${element.length}`);
  return element;
}

logLength("Bonjour");      // ✅ string a .length
logLength([1, 2, 3]);      // ✅ Array a .length
// logLength(123);         // ❌ number n'a pas .length → erreur
✅ Avantage : on peut utiliser .length en toute sécurité à l'intérieur de la fonction.
Sécurité des clés

🔑 Le mot-clé keyof

keyof T représente l'union de toutes les clés (noms de propriétés) du type T.

function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { name: "Alice", age: 30, city: "Douala" };
const userName = getProperty(user, "name"); // ✅ type: string
const userAge = getProperty(user, "age");   // ✅ type: number
// getProperty(user, "invalid"); ❌ Erreur : "invalid" n'est pas une clé de user
keyof User → "name" | "age" | "city"
✅ Utile pour : accès dynamiques sécurisés, transformations d'objets.
Flexibilité

🎛️ Paramètres par défaut <T = type>

Si l'utilisateur ne précise pas le type, TypeScript utilise le type par défaut.

class Boite<T = string> {
  contenu: T;
  constructor(valeur: T) {
    this.contenu = valeur;
  }
}

let boite1 = new Boite("cadeau");   // T par défaut → string
let boite2 = new Boite<number>(100); // T est number explicitement
let boite3 = new Boite(true);       // T devient boolean (inféré)

// Exemple avec fonction
function createArray<T = string>(length: number, value: T): T[] {
  return Array(length).fill(value);
}
const mots = createArray(3, "mot");     // string[]
const chiffres = createArray(2, 42);    // number[] (inféré)
📌 Les paramètres par défaut rendent les génériques plus pratiques sans perdre la sécurité.
Pratique avancée

⚡ keyof + extends : mise à jour sécurisée

function updateValue<T, K extends keyof T>(
  obj: T,
  key: K,
  newValue: T[K]
): void {
  obj[key] = newValue;
}

const config = {
  host: "localhost",
  port: 8080,
  ssl: true
};

updateValue(config, "port", 3000);   // ✅ port est number
updateValue(config, "host", "127.0.0.1"); // ✅
// updateValue(config, "port", "http");  ❌ string non assignable à number
// updateValue(config, "unknown", 42);   ❌ clé invalide
🚀 Résultat : la fonction est totalement typée — la clé et la valeur doivent correspondre exactement.

📋 Récapitulatif

1. T extends U

Assure que T a au moins les propriétés/méthodes de U.

2. keyof T

Union des noms de propriétés de T. Utilisé avec extends pour restreindre un paramètre aux clés valides.

3. <T = Default>

Fournit un type par défaut quand l'utilisateur n'en spécifie pas (ou que l'inférence ne suffit pas).
