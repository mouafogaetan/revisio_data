🌐 Qu'est-ce que HTTP ?

HTTP est le protocole de la couche application qui permet le transfert de documents hypermédias (pages web).

Modèle client-serveur : Le client (navigateur) envoie une requête, le serveur répond.

Sans état (stateless) : Chaque requête est indépendante (pas de mémoire entre deux requêtes).

Port par défaut : HTTP = 80, HTTPS = 443

Transport : HTTP utilise TCP (fiabilité). HTTPS = HTTP + TLS/SSL (chiffrement).

📤 Structure d'une requête HTTP

GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
Connection: keep-alive

[corps de la requête - optionnel]
Ligne de requête
Méthode + URL + Version HTTP
En-têtes (headers)
Métadonnées (Host, User-Agent, Accept...)
Corps (body)
Données envoyées (POST, PUT)

📥 Structure d'une réponse HTTP

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
Cache-Control: max-age=3600

<!DOCTYPE html>
<html>
  <body><h1>Bienvenue !</h1></body>
</html>
Ligne de statut
Version + Code + Phrase
En-têtes
Content-Type, Length, Cache...
Corps (body)
HTML, JSON, image, etc.

🛠️ Les méthodes HTTP

GET Récupérer des données (lecture seule, paramètres dans l’URL)
POST Envoyer des données pour créer une ressource (formulaire, API)
PUT Remplacer complètement une ressource existante
DELETE Supprimer une ressource
HEAD Comme GET mais sans le corps (pour les métadonnées)

Idempotence : GET, PUT, DELETE sont idempotents (plusieurs appels identiques ont le même effet). POST ne l’est pas.

📊 Codes d'état HTTP (familles)

1xx
Informatif
2xx
Succès
3xx
Redirection
4xx
Erreur client
5xx
Erreur serveur

Les plus courants :

200 OK - Succès
301 Moved Permanently - Redirection permanente
400 Bad Request - Requête mal formée
401 Unauthorized - Non authentifié
403 Forbidden - Accès interdit
404 Not Found - Ressource inexistante
500 Internal Server Error - Erreur serveur

📋 En-têtes HTTP importants

Host – Nom du serveur (obligatoire HTTP/1.1)
User-Agent – Infos sur le client (navigateur)
Accept – Types MIME acceptés (text/html, application/json)
Content-Type – Type du corps (ex: application/json)
Authorization – Token d’authentification (Bearer)
Cookie / Set-Cookie – Sessions utilisateur
Cache-Control – Politique de cache
Location – URL de redirection (3xx)

🔒 HTTP vs HTTPS

HTTP (non sécurisé)

  • Données en clair
  • Vulnérable à l’écoute (sniffing)
  • Pas d’intégrité des données
  • Port 80

HTTPS (sécurisé)

  • Chiffrement TLS/SSL
  • Authentification du serveur
  • Intégrité des données
  • Port 443

💻 Exemple complet : GET vers une API

Requête GET :
GET /api/users/42 HTTP/1.1
Host: jsonplaceholder.typicode.com
Accept: application/json

Réponse :
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 125

{
  "id": 42,
  "name": "Alice",
  "email": "alice@example.com"
}

🎯 Récapitulatif

  • ✓ HTTP est un protocole requête/réponse sans état
  • ✓ Méthodes : GET (lire), POST (créer), PUT (remplacer), DELETE (supprimer), HEAD
  • ✓ Codes : 2xx succès, 3xx redirection, 4xx erreur client, 5xx erreur serveur
  • ✓ HTTPS = HTTP + chiffrement TLS (sécurisé)
  • ✓ Les en-têtes transportent des métadonnées essentielles
