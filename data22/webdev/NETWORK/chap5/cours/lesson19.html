📁 FTP & SSH

Transfert de fichiers et accès sécurisé

Du FTP classique (ports 20/21) au SSH sécurisé (port 22) et ses dérivés SCP/SFTP

Cliquez sur les boutons ci-dessous pour naviguer

🎯 Objectifs pédagogiques

  • ✓ Comprendre FTP pour le transfert de fichiers (ports 20, 21)
  • ✓ Découvrir SSH pour l'accès sécurisé (port 22)
  • ✓ Utiliser SCP et SFTP pour les transferts sécurisés

💡 Le protocole FTP historique est non sécurisé. Aujourd'hui, on lui préfère SFTP ou FTPS pour protéger les données.

📂 FTP : File Transfer Protocol

FTP est un protocole de la couche application (TCP) conçu pour transférer des fichiers entre un client et un serveur.

🔹 Caractéristiques :

  • Utilise deux connexions TCP distinctes : canal de contrôle (port 21) et canal de données (port 20 en mode actif)
  • Authentification : nom d'utilisateur + mot de passe (en clair) ou anonyme
  • Commandes : LIST, RETR, STOR, DELE, MKD, RMD, etc.
  • Modes : actif (serveur initie la connexion données) / passif (client initie)
Exemple de session FTP simple :
> ftp serveur.exemple.cm
Connected to serveur.exemple.cm.
Name: user
Password: ***
230 User logged in.
ftp> ls
ftp> get fichier.txt
ftp> put photo.jpg
ftp> quit

🔄 Modes de connexion FTP

Mode actif (default)

  • Client se connecte sur port 21 (contrôle)
  • Client envoie PORT avec son IP+port (ex: 5000)
  • Serveur se connecte depuis port 20 vers port 5000 client
  • ⚠️ Bloqué par les pare-feu côté client

Mode passif (PASV)

  • Client demande mode passif (commande PASV)
  • Serveur répond avec IP + port (ex: 45000)
  • Client initie la connexion données sur ce port
  • ✅ Recommandé quand client est derrière un NAT/pare-feu
⚠️ FTP en mode actif pose souvent problème sur Internet. Le mode passif est préféré pour les clients derrière des box.

⚠️ Les failles de sécurité de FTP

  • Identification en clair : login/mot de passe transmissibles sans chiffrement → sniffing possible
  • Données non chiffrées : les fichiers transférés sont lisibles par un attaquant (man-in-the-middle)
  • Pas d'intégrité : les données peuvent être modifiées en transit
  • Authentification faible : supporte l'anonyme, facilement exploitable
🔴 Conséquence : FTP ne doit plus être utilisé sur Internet. Utilisez SFTP ou FTPS à la place.

🔒 SSH : Secure Shell

SSH est un protocole cryptographique permettant une connexion sécurisée à un serveur distant, principalement sur le port 22.

🔹 Fonctionnalités :

  • Connexion shell : exécuter des commandes sur le serveur distant
  • Tunnel sécurisé : redirection de ports (port forwarding)
  • Authentification par mot de passe ou par clé publique/privée
  • Chiffrement fort (AES, ChaCha20) et intégrité (HMAC)
Exemple de connexion SSH :
ssh utilisateur@serveur.cm -p 22
The authenticity of host 'serveur.cm' can't be established...
Are you sure you want to continue connecting (yes/no)? yes
utilisateur@serveur.cm's password: 
Last login: Mon Mar 10 08:30:22 2025
utilisateur@serveur:~$ ls -la
utilisateur@serveur:~$ sudo systemctl restart apache2

📋 SCP : Secure Copy

SCP (Secure Copy) est un utilitaire basé sur SSH pour copier des fichiers de manière sécurisée entre machines.

Syntaxe :

scp [options] source destination
Copier un fichier local → serveur :
scp document.pdf user@serveur.cm:/home/user/

Copier du serveur → local :
scp user@serveur.cm:/home/user/backup.tar.gz ./backup.tar.gz

Copier récursivement un dossier (-r) :
scp -r dossier/ user@serveur.cm:/home/user/

✅ SCP chiffre à la fois l'authentification et les données. Remplace avantageusement rcp non sécurisé.

📡 SFTP : SSH File Transfer Protocol

SFTP est une extension de SSH, mais ne doit pas être confondu avec FTPS. Il fournit des fonctionnalités avancées de gestion de fichiers.

✅ Avantages de SFTP

  • Une seule connexion (port 22)
  • Chiffrement complet
  • Commandes riches : ls, rm, mkdir, chmod, rename
  • Reprise de transfert partiel

⚠️ Inconvénients

  • Plus lent que FTP brut (chiffrement)
  • Nécessite SSH côté serveur
Exemple de session SFTP interactive :
sftp user@serveur.cm
sftp> ls
sftp> cd documents
sftp> get rapport.pdf
sftp> put photo.png
sftp> mkdir archives
sftp> exit

⚖️ Comparaison : FTP vs FTPS vs SFTP

ProtocolePort(s)ChiffrementAuthentificationFacilité de traversée NAT
FTP21 + 20 (ou dynamique)❌ NonEn clairMoyenne (mode passif requis)
FTPS (FTP over TLS)21 + dynamique✅ TLS/SSLChiffréeMoyenne
SFTP22 (SSH)✅ SSHChiffrée (mot de passe ou clé)✅ Excellente (une seule connexion)
💡 Recommandation : Pour un usage moderne et sécurisé, utilisez SFTP. Il est supporté par la plupart des clients (FileZilla, WinSCP, Cyberduck).

🛠️ Commandes essentielles

ssh-keygen -t rsa -b 4096

→ Générer une paire de clés SSH (publique/privée) pour l'authentification sans mot de passe.

ssh-copy-id user@serveur.cm

→ Copier la clé publique vers le serveur (plus besoin de mot de passe).

scp -i ~/.ssh/ma_cle user@serveur.cm:/remote/file ./local/

→ Utiliser une clé privée spécifique pour SCP.

📝 À retenir

  • FTP : historique, transfert de fichiers, 2 connexions, non sécurisé.
  • SSH : accès sécurisé à distance, port 22, remplace Telnet et rsh.
  • SCP : copie simple de fichiers sur SSH.
  • SFTP : protocole complet de transfert sécurisé, utilisant SSH (port 22).
  • Préférez SFTP pour toute nouvelle infrastructure.

🎓 Au Cameroun : de nombreux hébergeurs et FAI proposent l'accès SSH/SFTP pour l'administration des serveurs. C'est la norme de sécurité.
