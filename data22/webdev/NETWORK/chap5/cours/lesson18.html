📤 SMTP : L'expéditeur

SMTP (Simple Mail Transfer Protocol) est utilisé pour envoyer des emails d’un client vers un serveur, ou entre serveurs de messagerie.

Ports SMTP
  • Port 25 : classique, souvent bloqué par les FAI (spam)
  • Port 587 : SMTP avec authentification (recommandé)
  • Port 465 : SMTP sur SSL (déprécié mais encore utilisé)
Fonctionnement simple

Client → (SMTP) → Serveur de l’expéditeur → (SMTP) → Serveur du destinataire

Le client utilise la commande MAIL FROM: et RCPT TO:.
🔁 SMTP utilise une philosophie "store and forward" : le message est relayé de serveur en serveur jusqu’à destination.

📬 Réception : POP3 ou IMAP ?

📥 POP3

Post Office Protocol version 3

  • Télécharge les emails localement sur l’appareil.
  • Supprime généralement les messages du serveur.
  • Idéal si un seul ordinateur consulte la boîte.
  • Port : 110 (non chiffré), 995 (SSL).

🔄 IMAP

Internet Message Access Protocol

  • Laisse les messages sur le serveur (synchronisation).
  • Accès depuis plusieurs appareils (mobile, PC, webmail).
  • Permet de gérer des dossiers distants.
  • Port : 143 (non chiffré), 993 (SSL).
🧠 Règle simple : Si vous utilisez un seul ordinateur → POP3. Si vous consultez vos emails sur mobile + PC + tablette → IMAP.

📎 MIME : Pièces jointes & enrichissement

MIME (Multipurpose Internet Mail Extensions) étend le format des emails pour transporter autre chose que du texte brut (ASCII).

🎯 Rôle de MIME :
  • Images, audio, vidéo
  • Fichiers joints (PDF, Word, ZIP)
  • Textes formatés (HTML, gras, couleurs)
  • Jeux de caractères (UTF-8, ISO)
🧩 Structure MIME :

Content-Type: multipart/mixed;
boundary="---bound"

Chaque partie a son propre type (text/html, image/jpeg, application/pdf).

📌 Exemple : un email avec pièce jointe PDF contient du texte en HTML + le fichier encodé en base64 (MIME).

🔄 Cycle de vie d’un email

Alice envoie via SMTP → serveurs relayent → Bob récupère via POP3 ou IMAP.
