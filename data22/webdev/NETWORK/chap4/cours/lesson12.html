User Datagram Protocol

Simple, rapide, sans connexion et sans fiabilité

💡 UDP est le protocole de la vitesse : on envoie sans attendre de confirmation.

📌 Qu'est-ce qu'UDP ?

UDP (User Datagram Protocol) est un protocole de la couche transport (niveau 4 du modèle TCP/IP).

🔹 Ses 3 grandes caractéristiques :

  • Sans connexion : pas d'établissement de connexion avant d'envoyer
  • Non fiable : pas d'accusé de réception, pas de retransmission
  • Sans contrôle de flux ni congestion : envoi à la volée
⚠️ UDP est souvent appelé "fire-and-forget" (envoi et oubli). Les paquets peuvent être perdus, dupliqués ou arriver dans le désordre.

📐 En-tête UDP (8 octets)

 0      7 8     15 16    23 24    31
+--------+--------+--------+--------+
|     Port source      |   Port destination |
+--------+--------+--------+--------+
|      Longueur        |     Checksum       |
+--------+--------+--------+--------+
|                    Données               |
+------------------------------------------+
  • Port source (16 bits) : port émetteur (optionnel, 0 si non utilisé)
  • Port destination (16 bits) : port du service destinataire
  • Longueur (16 bits) : taille totale (en-tête + données) en octets, minimum 8
  • Checksum (16 bits) : détection d'erreurs (optionnel sur IPv4, obligatoire sur IPv6)

📺 Pourquoi utiliser UDP ?

🎮 Jeux vidéo en ligne

Importe d'avoir la position du joueur rapidement, même si quelques paquets sont perdus.

📡 Streaming vidéo/live

Une image qui tremble vaut mieux qu'une pause.

📞 VoIP (téléphonie)

Les petits décrochages sont acceptables, pas le délai.

🌐 DNS

Une seule question, une réponse rapide. Perte = on recommence.

💡 On utilise UDP quand la rapidité prime sur la fiabilité absolue.

🧮 Checksum UDP

Le checksum UDP permet de détecter les erreurs dans l'en-tête et les données. Il utilise un pseudo-en-tête IP.

Pseudo-en-tête (12 octets) : IP source, IP destination, protocole (17 pour UDP), longueur UDP.

Calcul : complément à 1 de la somme des mots de 16 bits (complément à 1). Si le résultat est 0xFFFF, le paquet est valide.

Processus simplifié :
1. Addition des mots de 16 bits (en-tête UDP + données + pseudo-en-tête)
2. Ajout des retenues (end-around carry)
3. Complément à 1 du résultat

⚠️ Sur IPv4, le checksum est optionnel (mis à 0 pour signifier non utilisé). Sur IPv6, il est obligatoire.

🔍 Exemple : requête DNS en UDP

Quand vous tapez www.google.cm, votre ordinateur envoie une requête DNS en UDP au serveur DNS (port 53).

Datagramme UDP typique (simple requête) :
- Port source : 54321 (aléatoire)
- Port destination : 53 (DNS)
- Longueur : 48 octets
- Checksum : 0x2a3f

[Données] : "www.google.cm" + type de requête (A)

📌 Si la réponse est trop longue (> 512 octets), DNS passe automatiquement en TCP.

📝 Synthèse : forces et faiblesses d'UDP

✅ Avantages

  • Très faible latence
  • Pas d'overhead de connexion
  • Idéal pour temps réel / broadcast
  • En-tête léger (8 octets)

❌ Inconvénients

  • Aucune garantie de livraison
  • Pas de contrôle de congestion
  • Peut encombrer le réseau
  • Les données peuvent arriver dans le désordre
🎓 À retenir : UDP = simple, rapide, sans fioritures. On le choisit quand la perte de quelques paquets est moins gênante que le délai.
