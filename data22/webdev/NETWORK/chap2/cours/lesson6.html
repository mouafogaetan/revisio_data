Comprendre le rôle de la couche liaison (détection/correction d'erreurs)

💡 La couche liaison permet aux appareils d'un même réseau local de s'identifier et de s'échanger des trames fiables.

📡 Rôle de la couche liaison (Data Link)

La couche 2 du modèle OSI a pour missions :

  • Découpage en trames (framing) : encapsule les paquets IP dans des trames
  • Adressage physique : utilise l'adresse MAC (unique par carte réseau)
  • Détection et correction d'erreurs : CRC (Cyclic Redundancy Check), checksum
  • Contrôle d'accès au support : CSMA/CD (Ethernet), CSMA/CA (WiFi)
  • Contrôle de flux (optionnel) : empêche la saturation du récepteur
Les switches sont des équipements de couche 2 (ils lisent les adresses MAC).

🏷️ L'adresse MAC (Media Access Control)

Caractéristiques :

  • 48 bits (6 octets)
  • Notation hexadécimale : AA:BB:CC:DD:EE:FF
  • Premiers 24 bits = OUI (constructeur)
  • Derniers 24 bits = numéro série unique
  • Gravée dans la carte réseau (théoriquement unique)

Types d'adresses MAC :

  • Unicast : vers une seule interface
  • Multicast : vers un groupe (bit de poids faible = 1)
  • Broadcast : FF:FF:FF:FF:FF:FF (vers tout le réseau local)
Commande pour afficher sa MAC : ipconfig /all (Windows) ou ifconfig (Linux/Mac)

🔍 Détection et correction d'erreurs

La couche liaison ajoute un checksum ou CRC à la fin de chaque trame.

+-------------------+-------------------+---------------------+ | En-tête MAC (14B) | Données (paquet IP) | CRC (4B) | +-------------------+-------------------+---------------------+

CRC (Cyclic Redundancy Check) : calcul mathématique qui permet de détecter les altérations de bits. Si le récepteur recalcule un CRC différent → trame erronée → demande de retransmission.

Ethernet utilise CRC-32 (taux de détection > 99,9999%)

🔄 ARP (Address Resolution Protocol)

ARP permet de trouver l'adresse MAC à partir d'une adresse IP sur un même réseau local.

Fonctionnement :

  1. Machine A veut envoyer un paquet à IP X
  2. Elle consulte sa table ARP (cache)
  3. Si pas trouvé, elle émet une requête ARP broadcast : "Qui a l'IP X ?"
  4. La machine X répond avec son adresse MAC
  5. La table ARP est mise à jour (durée de vie : 15-30 min)
arp -a (Windows/Linux) pour afficher la table ARP.

📊 Exemple ARP pas à pas

Scénario : PC1 (192.168.1.10) veut parler à PC2 (192.168.1.20)

  1. PC1 vérifie sa table ARP → pas d'entrée
  2. PC1 envoie un broadcast MAC FF:FF:FF:FF:FF:FF avec requête ARP
  3. PC2 reçoit la requête, reconnaît son IP
  4. PC2 répond (unicast) avec sa MAC (par ex. AA:BB:CC:11:22:33)
  5. PC1 stocke (192.168.1.20 → AA:BB:CC:11:22:33) dans sa table
  6. Les deux machines peuvent maintenant communiquer directement au niveau Ethernet
ARP est un protocole de niveau 2.5 (entre liaison et réseau).

⚠️ Sécurité : ARP spoofing

ARP n'a pas d'authentification → vulnérabilité : une machine malveillante peut répondre à toutes les requêtes ARP en prétendant être une autre machine.

Conséquences :

  • Interception du trafic (MITM)
  • Détournement de flux
  • Déni de service

Protections : ARP statique (sur réseaux sécurisés), Dynamic ARP Inspection (sur switches gérés).

✏️ Exercice : Analyser une trame

Une trame Ethernet contient :

  • MAC destination : FF:FF:FF:FF:FF:FF
  • MAC source : 00:14:22:01:23:45
  • Type : 0x0806 (ARP)

Questions :

  1. Quel type de communication est-ce (unicast/multicast/broadcast) ?
  2. Quel est l'OUI du constructeur de la MAC source ?
  3. À quel protocole correspond le type 0x0806 ?
🧩 Voir les réponses
1. Broadcast (MAC destination = FF:FF:FF:FF:FF:FF).
2. OUI = 00:14:22 (constructeur Dell, par exemple).
3. 0x0806 = protocole ARP (requête/réponse).
