🔧 Configurer un pare-feu (ACL)

Une ACL (Access Control List) est une liste de règles appliquée sur un routeur ou un pare-feu. Exemple sur un routeur Cisco :

! Bloquer tout le trafic entrant depuis 203.0.113.0/24
access-list 100 deny ip 203.0.113.0 0.0.0.255 any
! Autoriser le trafic HTTP et HTTPS entrant
access-list 100 permit tcp any any eq 80
access-list 100 permit tcp any any eq 443
! Appliquer l'ACL sur l'interface entrante
interface gigabitethernet0/0
 ip access-group 100 in

💡 Sous Linux, on utilise iptables ou nftables. Sous Windows, le pare-feu intégré (Windows Defender Firewall).

⚔️ Simuler des attaques (environnement isolé)

📡 Ping flood (ICMP echo)

Sous Linux/Mac : ping -f 192.168.1.10
Sous Windows : ping -t -l 65500 192.168.1.10
⚠️ Testez uniquement sur votre propre réseau ou labo.

🔍 Scan de ports (nmap)

nmap -sS 192.168.1.10 (SYN scan)
nmap -p 1-1000 192.168.1.10
Détecte les services ouverts.

⚠️ ATTENTION : N'effectuez ces tests que sur des équipements dont vous êtes propriétaire (réseau de test, Packet Tracer). Les attaques non autorisées sont illégales.

📡 Analyser l'attaque avec Wireshark

Étapes :

  1. Lancer Wireshark et démarrer la capture sur l'interface réseau.
  2. Lancer l'attaque (ping flood ou scan de ports).
  3. Appliquer des filtres :
    - Pour ICMP : icmp
    - Pour un scan SYN : tcp.flags.syn == 1 and tcp.flags.ack == 0
  4. Observer le volume de trafic, les adresses sources, les réponses (ou absence).
💡 Un ping flood se reconnaît par une avalanche de paquets ICMP Echo Request sans délai. Un scan SYN apparaît comme une série de paquets SYN vers différents ports.

🩺 Démarche de diagnostic

1. Problème : "Pas d'accès Internet"
✔️ ping 8.8.8.8 (test IP)
✔️ ping google.com (test DNS)
✔️ ipconfig /all (vérifier IP, passerelle, DNS)
✔️ Redémarrer routeur / modem.
2. Problème : "Site web lent ou inaccessible"
✔️ tracert google.com (trouver le saut problématique)
✔️ Vérifier pare-feu : autorise-t-il le port 80/443 ?
✔️ Consulter les logs du serveur web.
3. Problème : "IP conflict" (conflit d'adresses)
✔️ Libérer/renouveler IP : ipconfig /release puis /renew
✔️ Passer en DHCP si possible.
