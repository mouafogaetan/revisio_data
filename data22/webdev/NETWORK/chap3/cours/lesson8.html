Masques, sous-réseaux & VLSM

Comprendre le masque CIDR, calculer adresse réseau, broadcast, hôtes valides et découpage en sous-réseaux

IPv4CIDRSous-réseau

Lesson 8 · Chapitre 3 · Adressage IP

🎯 Objectifs de la leçon

1. Comprendre le masque de sous‑réseau et la notation CIDR (/8, /16, /24).
2. Calculer l’adresse réseau, l’adresse de broadcast et la plage d’hôtes valides.
3. Découper un réseau en sous‑réseaux (VLSM) selon des besoins précis.
4. Optimiser l’espace d’adressage avec le masque variable.
💡 À retenir : le masque indique la partie réseau de l’adresse IP. Plus le masque est grand (ex: /30), moins il y a d’hôtes.

🧠 Le masque de sous-réseau

Le masque est un nombre de 32 bits : bits à 1 = partie réseau ; bits à 0 = partie hôte.
Notation CIDR : /n où n = nombre de bits à 1.

/8 (255.0.0.0)
16 777 214 hôtes
/16 (255.255.0.0)
65 534 hôtes
/24 (255.255.255.0)
254 hôtes

🔢 Calcul du masque :

Exemple : /26 → 24 bits + 2 bits = 255.255.255.192 (car 128+64 = 192). Nombre d’hôtes : $2^{(32-26)} - 2 = 2^6 - 2 = 62$ hôtes.

📐 Calculs essentiels

Étant donnés une IP et un masque, comment trouver :

  • Adresse réseau = IP & masque (ET binaire)
  • Adresse broadcast = réseau + (masque inversé) ou tous les bits hôtes à 1
  • Premier hôte = réseau + 1
  • Dernier hôte = broadcast − 1

Exemple : 192.168.1.45 /26 (masque 255.255.255.192)

Réseau = 192.168.1.0 (car 45 & 192 = 0)
Broadcast = 192.168.1.63
Plage hôtes = 192.168.1.1 → 192.168.1.62

📌 Formule : nombre d’hôtes = $2^{(32 - \text{mask})} - 2$

✂️ VLSM : découpage sur mesure

On attribue à chaque sous-réseau le masque qui correspond exactement au nombre d’hôtes requis.

Problème classique :

Réseau 192.168.10.0/24. Besoins :
✔ Sous-réseau A : 50 hôtes → masque /26 (62 hôtes)
✔ Sous-réseau B : 28 hôtes → masque /27 (30 hôtes)
✔ Sous-réseau C : 12 hôtes → masque /28 (14 hôtes)
✔ Liaison point à point : 2 hôtes → masque /30 (2 hôtes)

Réseau initial : 192.168.10.0/24
Sous-réseau A : 192.168.10.0/26  (1-62)
Sous-réseau B : 192.168.10.64/27 (65-94)
Sous-réseau C : 192.168.10.96/28 (97-110)
Liaison P2P : 192.168.10.112/30 (113-114)

On économise les adresses en évitant le gaspillage.
