💡 Les modèles en couches simplifient la compréhension des réseaux : chaque couche a une fonction spécifique.

🏛️ Modèle OSI (Open Systems Interconnection)

Standard ISO 7498, 7 couches de la physique à l'application :

7. Application – Interface avec l'utilisateur (HTTP, SMTP, FTP)
6. Présentation – Traduction des données (chiffrement, compression)
5. Session – Gestion des sessions (début/fin de dialogue)
4. Transport – Fiabilité, contrôle de flux (TCP, UDP)
3. Réseau – Routage, adressage IP
2. Liaison – Accès au support, adresses MAC
1. Physique – Signaux électriques, câbles, fréquences

🎓 Mnémotechnique : “Pas si simple, toutes les personnes ne savent pas lire ces histoires”

🌐 Modèle TCP/IP (Internet Protocol Suite)

Le modèle réellement utilisé sur Internet, plus concis :

4. Application – HTTP, DNS, SMTP, FTP, SSH (regroupe OSI 5-6-7)
3. Transport – TCP (fiable) et UDP (rapide)
2. Internet – IP, routage, ARP, ICMP (équivaut OSI couche 3)
1. Accès réseau – Ethernet, WiFi (physique + liaison OSI)
TCP/IP est le modèle opérationnel ; OSI reste un modèle théorique/pédagogique.

📦 Encapsulation & PDU

Chaque couche ajoute son propre en-tête (header) aux données reçues de la couche supérieure.

Données utilisateur (couche 7)
↓ Ajout en-tête Transport → SEGMENT (TCP) ou DATAGRAMME (UDP)
↓ Ajout en-tête Réseau → PAQUET (IP)
↓ Ajout en-tête Liaison → TRAME (Frame)
↓ Bits sur le support (PHY)
PDU = Protocol Data Unit (unité de données à chaque couche)
SDU = Service Data Unit (données servies à la couche supérieure)

🔍 Zoom sur les couches

Application (OSI7/TCP4) : protocoles utilisateurs (HTTP pour le Web, SMTP pour emails).

Transport (OSI4/TCP3) : TCP (connexion, fiabilité) / UDP (léger, temps réel). Ports (80, 443, 53...).

Réseau (OSI3/TCP2) : adressage IP, routage, paquets.

Liaison (OSI2/TCP1) : adresses MAC, détection d'erreurs (CRC), trames.

Physique (OSI1/TCP1) : bits, tensions, fréquences, câbles.

Un routeur travaille jusqu'à la couche 3 (réseau), un switch jusqu'à la couche 2, un hub couche 1.

📨 Exemple : navigation web

Navigateur (Application) : "GET /index.html"
↓ Transport (TCP) : ajoute port 80, numéro de séquence
↓ Réseau (IP) : ajoute IP source et destination
↓ Liaison (Ethernet) : ajoute adresses MAC source et destination
↓ Physique : bits sur le câble / onde radio

À la réception, chaque couche dépile son en-tête jusqu'à retrouver la donnée brute.

⬇️ Encapsulation à l'émission, décapsulation à la réception.

✏️ Exercice : Associez les équipements

À quelle(s) couche(s) du modèle OSI travaillent ces équipements ?

  • Hub
  • Switch
  • Routeur
  • Pare-feu (firewall)
  • Carte réseau
🧩 Voir les réponses
Hub : couche 1 (Physique).
Switch : couche 2 (Liaison).
Routeur : couche 3 (Réseau).
Pare-feu : couche 3 à 7 (filtrage paquets, inspection applicative).
Carte réseau : couches 1 et 2.

📌 Récapitulatif

OSI : 7 couches, modèle théorique de référence
TCP/IP : 4 couches, modèle opérationnel d'Internet
Encapsulation : ajout d'en-têtes successifs (Données → Segment → Paquet → Trame → Bits)
PDU : unité de données à chaque couche
