Les Pare-feux (Firewalls)

Filtrage de paquets · ACL · Matériel vs Logiciel

Classe : Développement Web – Initiation aux Réseaux informatiques
Objectifs

Ce que vous allez apprendre

  • ✓ Comprendre le rôle d’un pare-feu (filtrage de paquets).
  • ✓ Distinguer pare-feu matériel et pare-feu logiciel.
  • ✓ Découvrir les règles de filtrage (ACL, état de connexion).
🔒 Un pare-feu est comme un gardien qui vérifie chaque personne (paquet) avant de l’autoriser à entrer ou sortir.

🛡️ Définition et rôle

Un pare-feu (firewall) est un équipement (matériel ou logiciel) qui contrôle le trafic réseau entrant et sortant selon des règles de sécurité.

✔️ Ce qu'il fait :
  • Filtre les paquets (IP, ports, protocoles)
  • Empêche les intrusions
  • Protège le réseau interne
❌ Ce qu'il ne fait pas :
  • Il ne chiffre pas les données (contrairement à un VPN)
  • Il n'arrête pas tous les virus (d'où l'antivirus)

Le pare-feu se place entre le réseau interne et l'extérieur.

📜 Règles ACL (Access Control List)

Le filtrage se base sur des listes de contrôle d'accès (ACL). Chaque règle peut autoriser (permit) ou refuser (deny) un trafic selon plusieurs critères :

IP source → 192.168.1.10
IP destination → 8.8.8.8
Port source → 12345
Port destination → 80 (HTTP)
Protocole → TCP / UDP / ICMP
Interface → entrante / sortante
Exemple de règle :
deny tcp from 192.168.1.100 to any port 22 → bloque SSH depuis un poste précis.

🖥️ Matériel vs Logiciel

📡 Pare-feu matériel
  • Appareil dédié (box internet, routeur pro)
  • Protège tout le réseau
  • Performance élevée
  • Ex: routeur domestique, Fortinet, Cisco ASA
💻 Pare-feu logiciel
  • Programme installé sur un OS (Windows Defender, iptables)
  • Protège une seule machine
  • Plus flexible mais consomme des ressources
🏠 À la maison : votre box internet (pare-feu matériel) + Windows Defender (logiciel) travaillent ensemble.

🔄 Filtrage stateful (avec état)

Un pare-feu stateful garde en mémoire les connexions établies. Il autorise automatiquement les réponses aux requêtes sortantes.

Exemple : Vous demandez une page web (port 80). Le pare-feu mémorise votre demande et laisse passer la réponse du serveur, même si aucune règle entrante n'autorise tout le trafic HTTP.

💡 Stateless (sans état) examine chaque paquet individuellement, sans contexte → moins intelligent mais plus rapide.
