📦 1. JavaScript externe (fichier .js)

Pour alléger votre code HTML et réutiliser vos scripts, placez le JavaScript dans un fichier séparé (extension .js).

<script src="monScript.js"></script>

✅ Avantages : réutilisable, plus lisible, mis en cache par le navigateur.

⚠️ Important : une balise <script> qui a un attribut src ne doit pas contenir de code JavaScript à l'intérieur (il sera ignoré).

✍️ 2. JavaScript interne (dans le HTML)

Vous pouvez écrire du code directement entre les balises <script> ... </script>.

<script>
  console.log("Bonjour depuis le JavaScript interne !");
  alert("Bienvenue sur mon site");
</script>

➡️ Idéal pour des petits bouts de code ou des tests rapides. Mais pour des projets sérieux, privilégiez l’externe.

📌 Placement : souvent juste avant la fermeture de <body> pour éviter de bloquer l’affichage.

⏳ 3. Ordre d'exécution & placement

Le navigateur lit le HTML de haut en bas. Quand il rencontre une balise <script> (sans attributs particuliers) il exécute immédiatement le code, ce qui peut ralentir l’affichage de la page.

❌ Mauvais placement :

<head>
  <script src="lourd.js"></script>
</head>

→ bloque le chargement du contenu HTML.

✅ Bon placement :

<body>
  ... contenu ...
  <script src="script.js"></script>
</body>

→ HTML affiché avant l’exécution JS.

Mais il existe deux attributs magiques : defer et async !

⚙️ 4. Attributs defer & async

Ces attributs permettent de mieux contrôler le chargement des scripts externes sans bloquer l’affichage.

defer exécution différée

Le script est téléchargé en parallèle du HTML, mais exécuté seulement après que le document est complètement analysé. Respecte l’ordre des scripts.

<script src="app.js" defer></script>
async asynchrone

Téléchargement parallèle, mais le script est exécuté dès qu'il est prêt (sans attendre le HTML). Peut impacter l’ordre.

<script src="analytics.js" async></script> 
Sans attribut :  HTML [ blocage script ] → suite HTML
defer :         HTML ────────────┐   Exécution après HTML
               (téléchargement parallele)    
async :         HTML ────┐ Exécution dès que dispo (pas d'ordre)

📌 Récapitulatif & bonnes pratiques

  • Pour les scripts externes : <script src="fichier.js"></script>
  • Pour les petits snippets : du code interne, mais à éviter si réutilisable.
  • Placez les scripts juste avant </body> ou utilisez defer si dans <head>.
  • defer : idéal pour la plupart des scripts qui dépendent du DOM complet.
  • async : pour des scripts indépendants (analytics, widgets externes).

🧠 Rappel : Un même fichier peut contenir plusieurs fonctions, mais l’ordre des balises compte !

🤔 Question rapide : Si je veux charger un script sans bloquer l'affichage et qu'il accède au DOM, quel attribut choisir ?

🖥️ Exemple concret

Observez le comportement selon l’emplacement et l’attribut (dans l’onglet console de votre navigateur).

<!-- Exemple externe avec defer -->
<script src="log.js" defer></script>
<!-- interne -->
<script>
console.log("Message depuis l'interne");
</script>
💡 Pour tester l’ordre, utilisez la console navigateur (F12). Vous verrez que defer garde l’ordre et s’exécute après le parsing HTML.
