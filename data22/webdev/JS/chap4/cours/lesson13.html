💡 Pourquoi c'est important ?
Une fonction sans paramètres est comme une machine qui fait toujours la même chose. Les paramètres permettent de personnaliser son comportement. Le return permet de récupérer le résultat pour l'utiliser ailleurs.

📥 Les paramètres de fonction

Les paramètres sont des variables locales qui reçoivent les valeurs passées lors de l'appel.

// Déclaration d'une fonction avec deux paramètres
function direBonjour(prenom, nom) {
    console.log("Bonjour " + prenom + " " + nom);
}

// Appel de la fonction avec des arguments
direBonjour("Jean", "Dupont");  // Affiche : Bonjour Jean Dupont
direBonjour("Marie", "Lambert"); // Affiche : Bonjour Marie Lambert

📌 Vocabulaire :

  • Paramètre : variable définie dans la fonction (prenom, nom)
  • Argument : valeur réelle passée lors de l'appel ("Jean", "Dupont")

⚠️ Paramètres optionnels

Si on appelle une fonction avec moins d'arguments que de paramètres, les paramètres manquants valent undefined.

function saluer(prenom, titre) {
    console.log("Bonjour " + prenom + " " + titre);
}

saluer("Jean");           // Affiche : Bonjour Jean undefined
saluer("Marie", "Docteur"); // Affiche : Bonjour Marie Docteur

⚠️ Attention : undefined peut causer des bugs. Utilisez les paramètres par défaut pour éviter cela !

✨ Paramètres par défaut (ES6)

On peut assigner une valeur par défaut directement dans la déclaration du paramètre.

// Avec valeur par défaut
function saluer(prenom, titre = "Cher client") {
    console.log(`Bonjour ${prenom} ${titre}`);
}

saluer("Jean");                    // Bonjour Jean Cher client
saluer("Marie", "Docteur");        // Bonjour Marie Docteur
saluer("Paul", undefined);         // Bonjour Paul Cher client

💡 Astuce : Les valeurs par défaut sont utilisées uniquement si l'argument est undefined. null n'est pas remplacé.

🛠️ Exemples de paramètres par défaut

// Calcul de TVA (taux par défaut 19.25% pour le Cameroun)
function calculerTva(prixHT, tauxTva = 19.25) {
    const tva = prixHT * tauxTva / 100;
    return prixHT + tva;
}

console.log(calculerTva(1000));      // 1192.5 (TVA par défaut)
console.log(calculerTva(1000, 10));  // 1100 (TVA à 10%)

// Message personnalisé
function afficherMessage(nom, message = "Bienvenue sur notre site !") {
    console.log(`${nom} : ${message}`);
}

afficherMessage("Claude");           // Claude : Bienvenue sur notre site !
afficherMessage("Claude", "Bonjour !"); // Claude : Bonjour !

📤 Le mot-clé return

return permet de renvoyer une valeur depuis une fonction. La fonction s'arrête immédiatement après return.

// Fonction qui retourne la somme
function addition(a, b) {
    return a + b;
    console.log("Ce code ne sera jamais exécuté !");
}

const resultat = addition(5, 3);
console.log(resultat); // 8

// Fonction qui retourne une chaîne formatée
function formaterPrix(prix, devise = "FCFA") {
    return `${prix} ${devise}`;
}

const prixFormate = formaterPrix(2500);
console.log(prixFormate); // 2500 FCFA

❓ Une fonction sans return

Si une fonction n'a pas d'instruction return, elle retourne undefined.

function afficherMessage(message) {
    console.log(message);
    // Pas de return
}

const valeur = afficherMessage("Coucou !");
console.log(valeur); // undefined

// Exemple avec calcul mais sans return
function double(x) {
    x * 2;  // Calcule mais ne retourne rien !
}

const result = double(5);
console.log(result); // undefined (pas de return)

⚠️ Piège fréquent : Oublier le return alors qu'on veut récupérer une valeur. La variable récupérera undefined !

📝 Bonnes pratiques

✅ À faire

  • Toujours définir des valeurs par défaut pour les paramètres optionnels
  • Utiliser return pour renvoyer le résultat d'un calcul
  • Nommer les paramètres de manière explicite (prixHT plutôt que p)
  • Une fonction doit idéalement faire une seule chose

❌ À éviter

  • Trop de paramètres (plus de 3-4 → utiliser un objet)
  • Modifier les paramètres d'entrée (effet de bord)
  • Oublier le return dans une fonction qui doit retourner une valeur
  • Mettre du code après un return (code mort)

🎯 Règle d'or : Une fonction doit être prédictible : pour les mêmes arguments, elle doit retourner la même valeur (sans effets de bord).
