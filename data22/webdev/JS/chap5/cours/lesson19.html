📌 Méthodes d'objet

Une méthode = une fonction qui appartient à un objet

En JavaScript, les objets peuvent contenir non seulement des données (propriétés) mais aussi des actions : ce sont des méthodes. Une méthode est simplement une propriété dont la valeur est une fonction.

const chien = {
  nom: "Rex",
  aboyer: function() {
    console.log("Ouaf ! Ouaf !");
  }
};

chien.aboyer(); // "Ouaf ! Ouaf !"

On peut aussi utiliser la syntaxe raccourcie (ES6) :

const chat = {
  nom: "Mina",
  miauler() {        // raccourci moderne
    console.log(`${this.nom} dit Miaou !`);
  }
};

💡 À retenir :

Les méthodes permettent de créer des objets plus autonomes, capables d’effectuer des actions en utilisant leurs propres propriétés.

🔑 Mot-clé magique

Le mystérieux this

this représente l’objet qui appelle la méthode. Dans une méthode, this fait référence à l’objet propriétaire.

const personne = {
  prenom: "Aisha",
  nom: "Mbarga",
  presentation() {
    return "Je m'appelle " + this.prenom + " " + this.nom;
  }
};

console.log(personne.presentation()); // Je m'appelle Aisha Mbarga

✅ Sans this, on ne pourrait pas accéder aux propriétés de l'objet depuis l'intérieur. C’est ce qui rend les méthodes dynamiques.

⚠️ Attention !

Le this dépend de comment la fonction est appelée. Dans une méthode classique, c’est l’objet. Mais si on prend la méthode toute seule, this peut devenir undefined (mode strict) ou l’objet global.

⚡️ Différence cruciale

Fonctions fléchées : this lexical

Les fonctions fléchées ne possèdent pas leur propre this. Elles capturent le this du contexte parent (lexical).

❌ Avec méthode classique perdue :

const user = {
  name: "Claude",
  greet: function() {
    setTimeout(function() {
      console.log("Bonjour " + this.name); // undefined ou window
    }, 100);
  }
};

✅ Avec fléchée :

const user = {
  name: "Claude",
  greet: function() {
    setTimeout(() => {
      console.log("Bonjour " + this.name); // "Claude" ✅
    }, 100);
  }
};

➡️ Règle d’or : dans les méthodes d’objet, utilisez des fonctions classiques (ou la syntaxe courte) pour que this désigne bien l’objet. Utilisez les fléchées pour conserver le this environnant.

🛠️ Utilitaires objet

Parcourir un objet avec Object.keys, values, entries

JavaScript moderne offre des méthodes statiques pour interagir facilement avec les propriétés d’un objet :

  • Object.keys(obj) → retourne un tableau des clés (noms des propriétés).
  • Object.values(obj) → retourne un tableau des valeurs.
  • Object.entries(obj) → retourne un tableau de paires [clé, valeur].
const voiture = {
  marque: "Toyota",
  modele: "Corolla",
  annee: 2022,
  klaxonner() { console.log("Pouet"); }
};

console.log(Object.keys(voiture));   // ["marque","modele","annee","klaxonner"]
console.log(Object.values(voiture)); // ["Toyota","Corolla",2022, ƒ]
Object.entries(voiture).forEach(([cle, valeur]) => {
  console.log(`${cle} : ${valeur}`);
});

C’est très utile pour itérer sur des objets sans boucle for...in et pour convertir des données.

🧪 Mise en pratique

Exemple complet : carnet d'adresses

Créons un objet contact avec une méthode d’affichage utilisant this et utilisons Object.values pour lister les infos.

const contact = {
  nom: "Ngo Mbarga",
  telephone: "698 12 34 56",
  ville: "Douala",
  afficherInfos() {
    return `📞 ${this.nom} - ${this.telephone} (${this.ville})`;
  },
  listerProprietes() {
    console.log("Valeurs : ", Object.values(this));
  }
};

console.log(contact.afficherInfos()); // 📞 Ngo Mbarga - 698 12 34 56 (Douala)
contact.listerProprietes(); // ["Ngo Mbarga", "698 12 34 56", "Douala", ƒ, ƒ]

✨ Petit défi pour vous :

Créez un objet eleve avec nom, notes (tableau) et une méthode moyenne() qui calcule la moyenne des notes en utilisant this.notes. Essayez ensuite d’afficher les clés avec Object.keys.

const eleve = {
  nom: "Yannick",
  notes: [14, 16, 12, 18],
  moyenne() {
    const somme = this.notes.reduce((acc, note) => acc + note, 0);
    return somme / this.notes.length;
  }
};
console.log(eleve.moyenne());  // 15
console.log(Object.keys(eleve)); // ["nom", "notes", "moyenne"]
📌 Mémorise

Ce qu’il faut retenir

✨ Méthodes

Une fonction stockée dans une propriété d’objet → elle agit sur l’objet.

✨ this dynamique

Dans une méthode classique, this = l’objet qui appelle la méthode. Les flèches ne redéfinissent pas this.

🔍 Object.keys/values/entries

Pratiques pour parcourir, transformer ou déboguer des objets.

🧠 Bonne pratique

Pour accéder aux propriétés d’un objet dans une méthode, toujours utiliser this.propriete, jamais le nom de la variable objet (sauf si référence externe).
