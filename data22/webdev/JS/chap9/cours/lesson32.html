🗄️

Web Storage : localStorage vs sessionStorage

Le navigateur offre deux objets de stockage clé-valeur pour sauvegarder des données sous forme de chaînes de caractères.

💾 localStorage
  • Persistant : les données restent même après fermeture du navigateur.
  • Partagé entre tous les onglets / fenêtres de la même origine.
  • Capacité ~5-10 Mo.
  • Idéal pour les préférences à long terme.
🕒 sessionStorage
  • Éphémère : les données sont effacées à la fermeture de l'onglet / fenêtre.
  • Unique par onglet : deux onglets n’ont pas accès aux mêmes données.
  • Utile pour des données temporaires (ex: formulaire en cours).
⚠️ Remarque : Les deux stockent uniquement des chaînes. Pour les objets, il faut les convertir en JSON.

🛠️ Les méthodes essentielles

MéthodeDescriptionExemple
setItem(key, value)Stocker une paire clé/valeurlocalStorage.setItem('nom', 'Dupont')
getItem(key)Récupérer la valeur associéelocalStorage.getItem('nom')
removeItem(key)Supprimer une clé spécifiquelocalStorage.removeItem('nom')
clear()Vider tout le stockagelocalStorage.clear()

// Exemple d'utilisation simple avec jQuery 🧪


// Stocker un thème
$('#saveTheme').click(() => {
  localStorage.setItem('theme', 'dark');
  alert('Thème sauvegardé !');
});

// Lire au chargement
let savedTheme = localStorage.getItem('theme');
if(savedTheme) console.log('Thème:', savedTheme);

🧩 Stocker des objets : JSON.stringify & parse

Comme localStorage ne gère que les chaînes, on convertit les objets JavaScript en JSON.

📦 Exemple avec un profil utilisateur :


const user = { name: "Claire", age: 28, skills: ["HTML", "CSS", "JS"] };
// Sauvegarde
localStorage.setItem("userProfile", JSON.stringify(user));

// Récupération
const storedUser = JSON.parse(localStorage.getItem("userProfile"));
console.log(storedUser.name); // "Claire"
✅ JSON.stringify() transforme objet → chaîne JSON.
🔄 JSON.parse() transforme chaîne JSON → objet JavaScript.

⚠️ Si la clé n'existe pas, getItem retourne null ; penser à vérifier avant parse.

🔄 sessionStorage : un espace temporaire par onglet

Contrairement à localStorage, sessionStorage est nettoyé à la fermeture de l'onglet. Très pratique pour des données sensibles ou temporaires.


// Démo : stocker un compteur de navigation
sessionStorage.setItem('compteurVisite', '1');
let count = sessionStorage.getItem('compteurVisite');
console.log("Visites dans cet onglet :", count);

// Si vous dupliquez l'onglet, chaque onglet a sa propre copie !

localStorage

Durée : illimitée (jusqu'à effacement manuel). Portée : tous onglets et redémarrages.

sessionStorage

Durée : vie de l'onglet. Portée : uniquement l'onglet courant.

⚡ Rappel : ils partagent les mêmes méthodes (setItem, getItem, etc.)

✅ Bonnes pratiques & limites

  • Ne stockez pas d'informations sensibles (mots de passe, données bancaires) car accessibles via la console.
  • Vérifiez la disponibilité : feature detection if(typeof(Storage) !== "undefined").
  • Limite de taille ~5Mo (peut varier selon navigateur). Pensez à gérer les exceptions QuotaExceededError.
  • Utilisez des noms de clés explicites pour éviter collisions.
  • Pour des données volumineuses ou structurées, préférez IndexedDB.
💡 Cas d'usage typiques : sauvegarde de brouillon, thème sombre/clair, panier d'achat, historique de navigation partiel.
