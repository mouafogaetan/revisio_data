⏸️ 1. Points d'arrêt (Breakpoints)

Dans l'onglet Sources du navigateur, cliquez sur le numéro d'une ligne pour ajouter un point d'arrêt. L'exécution s'arrête à cette ligne.

function calculer(a, b) {
  let resultat = a + b;   // 🔴 Cliquez sur le numéro de ligne
  console.log(resultat);
  return resultat * 2;
}
calculer(5, 3);
  • Une fois arrêté, vous pouvez passer d'une instruction à l'autre avec F10 (pas à pas) ou F11 (entrer dans une fonction).
  • Observez la colonne Scope : les variables locales, globales et leurs valeurs actuelles.
🖍️ [Schéma] Breakpoint: le code se fige → inspecteur affiche les valeurs.

🛑 2. L'instruction debugger;

Écrivez debugger; dans votre code. Lorsque JavaScript l'exécute, les outils de développement s'ouvrent automatiquement (si ouverts) et l'exécution se met en pause, exactement comme un point d'arrêt.

let age = 21;
debugger; // 🛑 Pause ici, inspectez la variable age
age = age + 1;
console.log("Nouvel âge:", age);
✅ Avantage : Très pratique quand on ne veut pas chercher la ligne exacte, on insère debugger; directement.

🔎 Inspecter variables & Scope

Lorsque le code est en pause, la section Scope dans l'onglet Sources affiche :

  • Variables locales (dans la fonction courante).
  • Closures (variables des fonctions parentes).
  • Portée globale (window).

Exemple : 

function saluer(nom) {
  let message = "Bonjour " + nom;
  debugger;    // ✅ dans Scope: nom = "Cameroun", message = "Bonjour Cameroun"
  return message;
}
saluer("Cameroun");

Vous pouvez aussi survoler une variable directement dans le code (outil moderne).

👁️ Surveiller des expressions (Watch)

Dans l'onglet Sources, à côté de "Scope" se trouve "Watch". Cliquez sur + pour ajouter une expression à évaluer en temps réel pendant le débogage.

Exemple d'expression :

counter * 2

Suivez comment elle change à chaque étape.

Condition complexe :

items.filter(i => i.active).length

Permet de surveiller une variable ou calcul sans modifier le code source.

📟 Console : votre meilleur allié

La console (Ctrl+Shift+J ou F12 puis onglet Console) permet d’exécuter du code à la volée.

  • console.log(), console.table() pour afficher des tableaux.
  • console.trace() pour voir la pile d'appels.
  • Tapez des variables globales ou testez des fonctions directement.
> console.log("Débogage au Cameroun !");
> let x = 42; x * 2;
← 84

🧪 Mise en pratique

Code buggé : corrige-le avec les outils !

let compteur = 0;
function incrementer() {
  compteur++;
  if(compteur === 5) {
    console.log("Cinq atteint !");
    // Pourquoi cela ne s'affiche pas? mets un breakpoint.
  }
  return compteur;
}
for(let i=0; i<=5; i++) incrementer();

Utilisez debugger; ou un breakpoint sur compteur++; pour inspecter la valeur. Vérifiez le scope.

💡 Petit défi : Ajouter une expression watch "compteur === 5" et voir quand elle devient true.
🏁
