Chapitre 3 : Structures de contrôle

Instruction switch

Comparer une valeur à plusieurs cas avec élégance

🎯 Condition avancée • 🔁 Alternative à if/else

🎯 Objectifs de la leçon

  • ✓ Comprendre l'utilité de switch pour comparer une valeur à plusieurs cas.
  • ✓ Utiliser break pour éviter l'exécution en cascade (fallthrough).
  • ✓ Maîtriser le cas default pour gérer les valeurs inattendues.
  • ✓ Savoir quand utiliser switch plutôt qu'une série de if/else if.
💡 À la fin de cette leçon, vous serez capable d'écrire des conditions multiples claires et performantes.

🔍 Pourquoi utiliser switch ?

Lorsqu’on doit comparer une même expression ou variable à plusieurs valeurs différentes, l’instruction switch rend le code plus lisible et organisé qu’une longue série de else if.

📌 Syntaxe de base :

switch(expression) {
  case valeur1:
    // code si expression === valeur1
    break;
  case valeur2:
    // code si expression === valeur2
    break;
  default:
    // code si aucun cas ne correspond
}
📘 Évaluation stricte (===) ⚡ Plus rapide qu’un long if/else

⚠️ Le mot-clé break et la cascade

Sans break, l’exécution continue vers le case suivant (même si la condition n’est pas vraie). C’est ce qu’on appelle le fallthrough.

let jour = "Lundi";
switch(jour) {
  case "Lundi":
    console.log("Début de semaine");
    // PAS de break → exécute aussi le mardi !
  case "Mardi":
    console.log("On avance");
    break;
  default:
    console.log("Autre jour");
}

➡️ Résultat : "Début de semaine" puis "On avance". On utilise rarement volontairement le fallthrough, sauf cas précis.

🧠 Rappel : mettez toujours break sauf si vous voulez délibérément un enchaînement.

🎲 Le mot-clé default

Le bloc default s’exécute si aucun case ne correspond à la valeur testée. Il est l’équivalent du else final.

let fruit = "Ananas";
switch(fruit) {
  case "Pomme":
    console.log("Une pomme par jour...");
    break;
  case "Banane":
    console.log("Jaune et énergétique");
    break;
  default:
    console.log("Fruit inconnu mais délicieux !");
}

💡 Conseil : placez default à la fin par convention, mais ce n’est pas obligatoire.

🧪 Exemple pratique : Groupe de cas

On peut regrouper plusieurs case pour exécuter le même code.

let note = "B";
switch(note) {
  case "A":
  case "B":
  case "C":
    console.log("Félicitations, vous avez réussi !");
    break;
  case "D":
  case "E":
    console.log("Reposez-vous et progressez encore");
    break;
  default:
    console.log("Note invalide");
}
✅ Si note vaut "B" → message "Félicitations, vous avez réussi !". Les cases A, B, C partagent le même bloc.

⚖️ switch vs if/else

✅ Switch

  • Idéal pour comparer une même variable avec plusieurs constantes
  • Code plus clair pour + de 3 cas
  • Exécution plus rapide dans certains moteurs JS

✅ If/else

  • Idéal pour des conditions complexes (intervalles, combinaisons logiques)
  • Plus flexible avec les expressions booléennes
"Choisissez switch quand vous testez l'égalité stricte d'une valeur unique."

✏️ Mini exercice interactif

Testez votre compréhension : quel message affiche ce switch ?

let mois = 3;
switch(mois) {
  case 1: console.log("Janvier"); break;
  case 2: console.log("Février"); break;
  case 3: console.log("Mars"); break;
  default: console.log("Autre mois");
}

💡 Indice : mois = 3, case 3 correspond à Mars.

📌 Récapitulatif & bonnes pratiques

  • Toujours utiliser break pour chaque case sauf regroupement volontaire.
  • Inclure un bloc default pour gérer les valeurs imprévues.
  • Les comparaisons sont strictes (type et valeur). "3" === 3 → false.
  • Switch peut être imbriqué mais attention à la lisibilité.
  • Privilégiez switch quand vous avez plus de 3 cas distincts pour une même variable.
✨ Résumé : switch structure clairement vos choix multiples. Associez-le à break et default pour un code robuste.
