🖥️ Machine Virtuelle (VM) : le cœur du calcul

Une VM est un ordinateur logiciel qui fonctionne sur un serveur physique. Dans le cloud, on peut créer une VM en quelques clics.

⚙️

General Purpose

Équilibre entre calcul, mémoire et réseau. Idéal pour serveurs web, petites bases de données.

Ex: AWS t3, Azure B series
🚀

Compute Optimized

Processeurs puissants pour calculs intensifs (jeux, analyse vidéo, HPC).

Ex: AWS c6g, Azure F-series
🧠

Memory Optimized

Beaucoup de RAM pour bases de données en mémoire, caches.

Ex: AWS r6i, Azure M-series
📌 Astuce: sur AWS vous choisissez un "type d'instance" comme t2.micro (gratuit), sur Azure c'est "taille de VM".

📸 Images de machines virtuelles

Une image contient le système d'exploitation et des logiciels préinstallés. On distingue :

  • Images officielles : Ubuntu, Windows Server, Debian, CentOS fournies par le cloud.
  • Marketplace : solutions préconfigurées (LAMP, WordPress, Docker).
  • Images personnalisées : Vous créez votre propre image après avoir configuré une VM (économie de temps).
# Exemple avec AWS CLI pour lancer une VM Ubuntu 22.04
aws ec2 run-instances --image-id ami-0c55b159cbfafe1f0 --instance-type t2.micro --key-name MyKeyPair

🔧 Azure: utilisez "az vm create --image UbuntuLTS" ou depuis le portail.

⚖️ Auto-scaling & Load Balancer

Pour gérer les pics de trafic, le cloud propose des mécanismes intelligents :

📈 Auto-scaling

Ajoute ou supprime automatiquement des VMs selon la charge (CPU, mémoire, requêtes réseau).

Condition: CPU > 70% → ajouter 2 VMs
🔄 Load Balancer

Distribue le trafic entre plusieurs VMs pour éviter qu'une seule ne soit surchargée.

Round-robin, vérification de santé (health check)
[Client] → 🌐 Load Balancer → 🖥️ VM1 / 🖥️ VM2 / 🖥️ VM3
Schéma: Répartition des requêtes

💡 Exemples & Bonnes pratiques

🔹 Site e-commerce pendant Black Friday

Utiliser un groupe d’auto-scaling avec des règles de scaling basées sur le nombre de requêtes. Combiner avec un load balancer applicatif (ALB).

🔹 Application de traitement vidéo

Choisir des instances Compute Optimized (CPU puissant) et utiliser des instances spot pour réduire les coûts.

🔹 Bonne pratique : étiquettes (tags)

Nommez vos VMs avec des tags "environnement=prod", "projet=web" pour mieux gérer la facturation.

🎓 À retenir : Lancez toujours vos VMs dans des groupes de disponibilité (Availability Sets) pour une haute disponibilité.
