Bases de données dans le cloud

SQL managé, NoSQL et Data Warehouses : choisir la bonne solution pour vos applications.

🎯 Objectifs de la leçon 14

  • ✓ Distinguer les bases de données relationnelles managées (RDS, Cloud SQL, Azure SQL).

  • ✓ Savoir choisir la base de données adaptée selon vos besoins (volume, latence, structure).

🗄️ 1. Bases de données relationnelles managées (SQL)

Dans le cloud, plus besoin d'installer ni d'administrer soi-même MySQL ou PostgreSQL. Les services managés s'occupent des sauvegardes, mises à jour et haute disponibilité.

☁️ AWS RDS

MySQL, PostgreSQL, MariaDB, Oracle, SQL Server. Multi-AZ, Read Replicas.

☁️ GCP Cloud SQL

MySQL, PostgreSQL, SQL Server. Intégration avec App Engine et fonctions serverless.

☁️ Azure SQL Database

Basé sur Microsoft SQL Server, performance intelligente et scaling automatique.

📌 Exemple concret : Une boutique en ligne utilise une base relationnelle pour stocker commandes, clients et produits. Le cloud permet d'ajouter un réplica en lecture pour les rapports sans impacter le site.

📦 2. Les bases NoSQL dans le cloud

Pour des applications à très forte échelle, faible latence ou données semi-structurées, les bases NoSQL sont idéales. Chaque fournisseur propose des solutions clés-en-main.

FournisseurService NoSQLType / particularité
AWSDynamoDBClé-valeur + documents, performance < 10 ms, scale automatique
AzureCosmos DBMulti-modèle (document, graphe, colonne), distribution mondiale
GCPFirestore / BigtableFirestore : temps réel, mobile ; Bigtable : analytique gros volume
⚡ Cas typique: applications IoT, jeux en ligne, catalogues produits massifs

📊 3. Data Warehouses cloud : BigQuery, Redshift, Synapse

Les entrepôts de données sont optimisés pour l’analyse de très gros volumes de données, avec des requêtes SQL complexes et rapides.

SELECT année, SUM(ventes)
FROM `projet.clients.transactions`
GROUP BY année -- BigQuery analyse des téraoctets en secondes
BigQuery (GCP) – serverless, analyse massive, paiement à la requête.
Amazon Redshift – colonnaire, performances élevées avec clustering.
Azure Synapse – intégration avec Power BI et pipelines de données.
🧠

Comment choisir sa base de données cloud ?

📐 Relationnelle (SQL)

Schéma strict, transactions ACID, jointures complexes. Pour applications critiques (ERP, facturation, CRM).

⚡ NoSQL

Élastique, grande volumétrie, faible latence. Pour jeux, flux temps réel, données hétérogènes.

📈 Data Warehouse

Analyse historique, BI, machine learning sur données agrégées. Idéal pour rapports décisionnels.

💡 Conseil enseignant : « Souvent un projet cloud utilise plusieurs bases: une base SQL pour le back-office, une NoSQL pour le cache/panier, et un Data Warehouse pour les analytics ! »

📝

Petit exercice & à retenir

✏️ Question rapide : Vous développez une application de livraison en temps réel avec des millions de géolocalisations par seconde et vous devez faire évoluer sans limites. Quelle famille de base de données cloud privilégier ?

Voir la réponse

✅ NoSQL (DynamoDB, Cosmos DB) car elles supportent des écritures massives avec faible latence et scaling horizontal. Les bases SQL relationnelles auraient du mal à suivre ce rythme sans complexité.

🔁 Résumé de la lesson14 :

  • RDS / Cloud SQL / Azure SQL = SQL managé (simplicité + haute dispo).
  • DynamoDB / Cosmos DB / Firestore = NoSQL pour l'échelle horizontale.
  • BigQuery / Redshift / Synapse = Data Warehouses pour l'analytique Big Data.
  • Le choix dépend des besoins : transactions, volumes, latence, analyse.
