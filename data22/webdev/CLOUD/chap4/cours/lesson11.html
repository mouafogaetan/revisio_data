Stockage Objet dans le Cloud

Buckets, objets, classes de stockage, versioning & cycle de vie

Leçon 11 · Cours Cloud Computing

🎯 Objectifs de la leçon

1. Comprendre le stockage objet : bucket, clé, objet.
2. Utiliser les classes de stockage (Standard, Infrequent Access, Glacier).
3. Gérer le versioning, la réplication et le cycle de vie des objets.
🔑 notions clés : scalabilité illimitée, accès HTTP, durabilité

📦 Stockage objet : Définition

Bucket → conteneur global
Clé (Key) → nom unique de l'objet
Objet → donnée + métadonnées

Chaque fichier (image, vidéo, sauvegarde) est un objet, identifié par une clé unique dans un bucket (comme un seau).

Bucket = espace de stockage global

✨ Avantages clés :

  • Scalabilité illimitée (pas de limite de taille)
  • Accès via API HTTP / HTTPS (REST)
  • Durabilité 99.999999999% (11 neufs)

📊 Classes de stockage

Standard

Accès fréquent, latence faible. Idéal pour données actives (sites web, vidéos populaires).

Infrequent Access (IA)

Données consultées rarement mais besoin d'accès rapide. Coût de stockage réduit, frais de récupération.

Glacier / Archive

Archivage longue durée (minutes à heures de dégel). Très faible coût, parfait pour sauvegardes légales.

💡 Exemple: Une photo de profil → Standard. Une facture de 2018 → IA. Une sauvegarde annuelle → Glacier.

🔄 Versioning, Réplication & Cycle de vie

📌Versioning

Conserver plusieurs versions d’un objet. Protège contre les suppressions accidentelles ou modifications.

Ex: si vous modifiez "doc.pdf", l'ancienne version reste disponible.

🌍Réplication

Copie automatique des objets vers un autre bucket (même région ou différente). Haute disponibilité, conformité.

⏳Règles de cycle de vie

Automatiser la transition ou la suppression des objets. Exemple : déplacer vers "Infrequent Access" après 30 jours, puis vers Glacier après 90 jours, et supprimer après 365 jours.

Règle type: {"transition": {"days": 30, "storageClass": "IA"}}

🔁 Illustration : cycle de vie automatique

📘 Bonne pratique :

Activez le versioning + règles de cycle de vie pour réduire les coûts tout en conservant les données critiques.
Exemple : les journaux d'application deviennent rarement consultés après une semaine → transition vers IA puis nettoyage.
