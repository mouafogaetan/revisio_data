Modèles de service cloud

IaaS · PaaS · SaaS

Comprendre les trois piliers du cloud computing et leur usage concret

🎯 Objectifs de la leçon

✓ Comprendre l'IaaS (Infrastructure as a Service) : VM, stockage, réseau.
✓ Maîtriser le PaaS (Platform as a Service) : hébergement d'applications sans gestion OS.
✓ Comprendre le SaaS (Software as a Service) : logiciels prêts à l'emploi (Gmail, Office 365).
✓ Savoir différencier les trois modèles et choisir selon le besoin.
💡 À la fin de cette leçon, vous saurez quel type de service cloud utiliser pour votre projet.

☁️ Les trois couches du cloud

Imaginez un restaurant :

🥩 IaaS
Vous louez la cuisine, vous apportez vos ingrédients, préparez vous-même.
🍕 PaaS
La plateforme fournit les ustensiles & le four, vous faites la recette.
🥗 SaaS
Vous commandez le plat, tout est prêt, vous consommez.

IaaS (Infrastructure as a Service) → Donne accès à des ressources informatiques virtualisées : serveurs, stockage, réseaux. Vous gérez l'OS, les applis, mais pas l'infrastructure physique.

PaaS (Platform as a Service) → Plateforme prête à l'emploi pour développer et déployer des applications sans vous soucier des serveurs ou OS.

SaaS (Software as a Service) → Logiciel complet accessible via navigateur, aucune installation ni maintenance.

🏗️

IaaS (Infrastructure as a Service)

Exemples populaires : Amazon EC2, Google Compute Engine, Azure Virtual Machines, DigitalOcean.

  • ⚙️ Ce que vous gérez : Systèmes d'exploitation, middleware, données, applications.
  • 📦 Ce que le fournisseur gère : Virtualisation, serveurs, stockage physique, réseau, centres de données.
  • ✅ Avantages : Contrôle total, scalabilité à la demande, facturation à l'usage.
  • 📌 Cas d'usage : Héberger un site avec configuration spécifique, bases de données, DRP, calculs HPC.
🖥️ [ VM1 ]   [ VM2 ]   [ Stockage ] ← Tout est sous votre contrôle (OS, logiciels)

🚀 PaaS (Platform as a Service)

"Déployez votre code, on s'occupe du reste !"

  • ✔️ Environnement de développement intégré (langages, bases de données, outils middleware).
  • ✔️ Pas de gestion OS, mises à jour automatiques, scalabilité horizontale facile.
  • ✔️ Exemples : Heroku, Google App Engine, Azure App Service, AWS Elastic Beanstalk.
  • ✔️ Idéal pour développeurs qui veulent se concentrer sur le code plutôt que l'infra.
📱 Déploiement d'API REST sans administrer de serveur web
🐍 Applications Django / Node.js / Spring boot prêtes en quelques clics
Le PaaS vous offre la plateforme, vous livre votre application.

📱 SaaS (Software as a Service)

Gmail Microsoft 365 Google Drive Salesforce Zoom

Principe : Utilisation d'un logiciel hébergé dans le cloud, accessible via navigateur ou API. L'utilisateur n'a aucune charge technique.

🔹 Avantages :

  • Pas d'installation, mises à jour automatiques, accessible partout.
  • Tarification par abonnement ou utilisateur, faible coût initial.

🔸 Inconvénients :

  • Moins de personnalisation, dépendance au fournisseur (verrouillage).
✨ SaaS = "prêt à consommer" sans gestion technique.

📊 Comparaison visuelle

ModèleVous gérezFournisseur gèreExemple
IaaSOS, middleware, données, applicationsVirtualisation, serveurs, stockage, réseauAWS EC2, VM Azure
PaaSDonnées, applications, codeOS, middleware, runtime, infrastructureGoogle App Engine, Heroku
SaaSConfiguration utilisateur uniquementTout (applications, données, OS, infrastructure)Gmail, Office 365
🔍 Niveau de contrôle décroissant : IaaS (max) → PaaS → SaaS (min). En contrepartie, la simplicité d'utilisation est croissante.

🤔 Comment bien choisir ?

✅ IaaS si :

  • Vous avez besoin d'un contrôle total (OS personnalisé).
  • Vous migrez des applications existantes sans réécriture.
  • Workloads flexibles avec scaling manuel ou auto.

✅ PaaS si :

  • Vous développez des applications web/mobile rapidement.
  • Vous voulez éviter la gestion des serveurs.
  • Scalabilité automatique intégrée.

✅ SaaS si :

  • Vous utilisez un logiciel standard (email, CRM, collaboration).
  • Vous voulez un déploiement instantané sans équipe technique.
Dans le monde professionnel, on utilise souvent une combinaison des trois !

📌 Points clés à retenir

  • ✔️ IaaS : contrôle total, VM, stockage, réseau.
  • ✔️ PaaS : focus sur le développement, plateforme managée.
  • ✔️ SaaS : logiciel clé en main, accessibilité universelle.
  • ✔️ Chaque modèle répond à des besoins différents (coût, expertise, flexibilité).
