Provisioning vs Configuration

Intégrer Terraform et Ansible

🔧 Infrastructure as Code ⚙️ Configuration Management ☁️ Approche complémentaire

Classe : Développement Web | Matière : Initiation au Cloud Computing

🎯 Objectifs de la leçon

  • • Comprendre la différence entre provisioning (création d’infrastructure) et configuration (installation/logiciels)
  • • Écrire des playbooks Ansible pour configurer des machines virtuelles (déployer nginx, docker, etc.)
  • • Intégrer Terraform et Ansible : provisionner l’infrastructure avec Terraform, puis la configurer avec Ansible
  • • Maîtriser les bonnes pratiques d'automatisation hybride (local-exec, provisioners, inventaire dynamique)
💡 À la fin de ce cours, vous saurez automatiser le déploiement complet d’une application web cloud de A à Z.

📌 Provisioning ≠ Configuration

🏗️ Terraform

Provisioning (création d'infrastructure)

  • Crée des VMs, réseaux, bases de données, load balancers
  • Gère l'état (terraform.tfstate)
  • Langage déclaratif HCL
  • Idempotent : "créer ce qui n'existe pas"
🛠️ Ansible

Configuration (installation / paramétrage)

  • Installe des logiciels (nginx, docker, python)
  • Configure les services, fichiers, utilisateurs
  • Agentless (SSH uniquement)
  • Playbooks YAML, modules puissants

🔁 Complémentarité :

Terraform prépare le terrain (serveurs, réseau). Ansible transforme ces serveurs bruts en machines prêtes à exécuter l'application.

📜 Écrire un playbook Ansible

Fichier YAML qui définit les tâches à exécuter sur les serveurs distants.

---
- name: Configurer un serveur web
  hosts: webservers
  become: yes
  tasks:
    - name: Installer Nginx
      apt:
        name: nginx
        state: present
      when: ansible_os_family == "Debian"
    
    - name: Démarrer Nginx
      service:
        name: nginx
        state: started
        enabled: yes
    
    - name: Copier une page personnalisée
      copy:
        content: "<h1>Bienvenue sur mon cloud !</h1>"
        dest: /var/www/html/index.html

✨ Commande : ansible-playbook -i inventaire.ini playbook.yml

🔗 Intégrer Terraform & Ansible

Trois méthodes populaires :

1️⃣ Provisioner local-exec dans Terraform

Après création des VMs, Terraform lance une commande Ansible sur la machine locale.

2️⃣ Utiliser l’inventaire dynamique

Ansible récupère les IPs depuis l’état Terraform (terraform output) ou depuis un script python.

3️⃣ Module Ansible dans Terraform (community)

Il existe un provider Ansible (expérimental) pour exécuter des playbooks directement depuis le plan Terraform.

[Terraform] → crée VM (AWS/GCP/Azure) → exporte IP publique
[Ansible] → utilise ces IPs → configure l'application → ✅ serveur opérationnel

⚡ Exemple concret - Workflow hybride

1. main.tf (Terraform) : provisionne une instance EC2 (ou un droplet) et génère un fichier inventory.ini via template.

2. playbook.yml (Ansible) : installe Docker, déploie un conteneur nginx.

3. Commande combinée :

terraform apply -auto-approve && ansible-playbook -i inventory.ini playbook.yml
💡 On peut aussi utiliser le provisioner remote-exec mais Ansible est plus robuste pour la configuration logicielle.

🧩 Terraform + local-exec

Extraire les IPs après création et lancer Ansible automatiquement.

resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  tags = { Name = "ansible-target" }

  provisioner "local-exec" {
    command = "echo ${self.public_ip} > inventory.ini && ansible-playbook -i inventory.ini playbook.yml"
  }
}

output "instance_ip" {
  value = aws_instance.web.public_ip
}

⚠️ Note : Les provisioners Terraform sont à utiliser en dernier recours. On préfère générer un inventaire dynamique pour plus de flexibilité.

🔄 Inventaire dynamique Ansible

On utilise terraform output -json pour générer un inventaire automatique.

#!/bin/bash
# inventory.sh (exécutable)
terraform output -json | jq '{ "all": { "hosts": [ .instance_ip.value ] } }'

Puis on lance :

ansible-playbook -i inventory.sh playbook.yml
✅ Avantage : Évite de générer des fichiers statiques, toujours synchronisé avec l'état réel du cloud.

🌟 Bonnes pratiques d'intégration

✔ Séparer les responsabilités : Terraform pour l’infra, Ansible pour les softs.
✔ Utiliser des modules Terraform réutilisables.
✔ Gérer les dépendances (attendre que les VMs soient SSH-ready).
✔ Stocker l’état Terraform à distance (S3, Azure Storage).

📚 À retenir :

Terraform prépare l'infrastructure (le "matériel"), Ansible la transforme en environnement fonctionnel. Leur intégration permet l'automatisation totale de l'infrastructure cloud : du provisionning jusqu'au déploiement d'applications.

💻

Petit atelier pratique

En utilisant Terraform (mock ou local) et Ansible, essayez de :

  • Provisionner 2 VMs (localhost via docker ou VirtualBox)
  • Écrire un playbook pour installer Python3 et un serveur HTTP simple
  • Lancer la config après le "terraform apply"
💡 À faire en lab : utilisation de terraform localstack ou libvirt ou alors s’entraîner sur du vrai cloud avec crédits gratuits.

Cette compétence est clé pour les métiers DevOps / Cloud Engineer.
