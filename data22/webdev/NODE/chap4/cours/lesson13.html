🔍 Qu'est-ce qu'un middleware ?

Un middleware est une fonction qui a accès à l'objet req (requête), res (réponse) et à la fonction suivante next dans le cycle requête-réponse.

📥 Requête ➡️ Middleware 1 ➡️ Middleware 2 ➡️ Route finale ➡️ 📤 Réponse
Chaque middleware peut modifier req/res ou arrêter le cycle

📌 Exemple simple : app.use((req, res, next) => { console.log('Time:', Date.now()); next(); });

✍️ Créer un middleware personnalisé

Une fonction qui reçoit (req, res, next). N'oubliez pas d'appeler next() pour passer au suivant.

📝 Middleware de logging

function logger(req, res, next) {
  console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
  next();
}
app.use(logger);

🔐 Middleware d'authentification (simulé)

function authMiddleware(req, res, next) {
  const token = req.headers['authorization'];
  if (token === 'secret123') return next();
  res.status(401).json({ message: 'Accès non autorisé' });
}

⚙️ Middleware intégrés dans Express

Express fournit plusieurs middlewares prêts à l'emploi pour les besoins courants.

express.json()

Analyse les requêtes avec Content-Type application/json et remplit req.body.

express.urlencoded({extended: true})

Analyse les données de formulaire HTML (x-www-form-urlencoded).

express.static('public')

Sert des fichiers statiques (CSS, images, JS front).

express.Router()

Crée des routeurs modulaires (souvent utilisé comme mini-application).

💡 Bon à savoir : express.static est idéal pour les images, CSS, etc. Il délivre les fichiers sans traitement dynamique.

🔄 next() et l'ordre d'exécution

L’ordre dans lequel vous déclarez les middlewares est crucial : Express les exécute dans l’ordre d’enregistrement.

app.use(middleware1); // s'exécute en premier
app.use(middleware2); // second
app.get('/profile', handler); // troisième si next() appelé
⚠️ Important :

Si un middleware n'appelle pas next() et n'envoie pas de réponse, la requête reste bloquée !

✅ Bonne pratique :

Toujours terminer par next() ou envoyer une réponse (res.send()).

💻 Exemple concret : API avec plusieurs middlewares

const express = require('express');
const app = express();

// Middleware intégré pour parser le JSON
app.use(express.json());

// Middleware perso de log
app.use((req, res, next) => {
  console.log(`${req.method} ${req.path} - ${new Date().toISOString()}`);
  next();
});

// Middleware d'authentification simple
app.use('/admin', (req, res, next) => {
  if (req.headers['x-admin'] === 'yes') next();
  else res.status(403).send('Accès interdit');
});

// Route protégée
app.get('/admin/dashboard', (req, res) => {
  res.json({ message: 'Bienvenue admin !' });
});

app.listen(3000);

🧪 Essayez avec Postman : envoyez un header x-admin: yes pour accéder.

✅ Bonnes pratiques des middlewares

  • Organiser les middlewares par catégorie (sécurité, logs, validation).
  • Utiliser des middlewares spécifiques à une route : app.get('/route', mid1, mid2, handler)
  • Ne pas surcharger avec des traitements lourds synchro (privilégier asynchrone).
  • Créer un dossier middlewares/ pour mieux structurer le projet.
  • Utiliser des bibliothèques comme morgan pour les logs ou cors pour les CORS.
🧩 next('route') permet de passer au middleware/route suivant dans certaines conditions avancées.
