1. Installation & premier serveur

Express est un framework web minimaliste et flexible pour Node.js. Il ajoute une couche d'abstraction au module HTTP natif.

# Créer un projet et installer Express
mkdir mon-api && cd mon-api
npm init -y
npm install express
Serveur basique : 
const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.send('Bienvenue sur mon API Express !');
});

app.listen(PORT, () => console.log(`🚀 Serveur lancé sur http://localhost:${PORT}`));
Lancez avec node server.js puis visitez http://localhost:3000

2. Le middleware: cœur d'Express

Un middleware est une fonction qui a accès à l'objet requête (req), réponse (res) et à la fonction suivante (next()).

Cycle de vie :

Requête entrante Middleware 1 Middleware 2 Route handler Réponse
// Exemple de middleware logger
app.use((req, res, next) => {
  console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
  next(); // passe au middleware suivant
});
app.get('/api', (req,res) => res.json({message: "ok"}));
À retenir: l'ordre d'enregistrement des middlewares est crucial !

3. Routes : GET, POST, PUT, DELETE

GET

Récupérer des données

app.get('/users', (req,res) => res.send(listeUtilisateurs));
POST

Créer une ressource

app.post('/users', (req,res) => {
// créer user });
PUT

Mettre à jour (remplacement)

app.put('/users/:id', (req,res) => {})
DELETE

Supprimer une ressource

app.delete('/users/:id', (req,res) => {})
Bon à savoir : Express rend automatiquement les objets JSON avec res.json().

4. Middlewares intégrés & personnalisation

express.json() – Analyse les requêtes avec Content-Type application/json.
express.static('public') – Sert des fichiers statiques (CSS, images).
Créer un middleware perso : authentification simulée 
const authMiddleware = (req, res, next) => {
  const token = req.headers['authorization'];
  if (token === 'secret123') next();
  else res.status(401).json({error: 'Non autorisé'});
};
app.use('/admin', authMiddleware);
⚡ next() est vital pour passer la main au middleware suivant ; sans appel, la requête reste bloquée.

Exemple complet : mini API produits

const express = require('express');
const app = express();
app.use(express.json()); // middleware pour lire JSON
let produits = [{id:1, nom:"Livre Node.js", prix:25}];
app.get('/produits', (req,res) => res.json(produits));
app.post('/produits', (req,res) => {
  const newProd = {id: Date.now(), ...req.body};
  produits.push(newProd); res.status(201).json(newProd);
});
app.put('/produits/:id', (req,res) => { /* mise à jour */ });
app.delete('/produits/:id', (req,res) => { /* suppression */ });
app.listen(3000, () => console.log('API prête'));
Testez avec Postman ou curl
Middleware de logging personnalisé possible

Récapitulons & à vous de jouer !

  • Express simplifie la création de serveurs Node.js.
  • Les middlewares sont des fonctions intermédiaires qui interceptent req/res.
  • Les méthodes HTTP (GET, POST, PUT, DELETE) définissent des routes pour l'API.
  • L'ordre des middlewares détermine le flux d'exécution.
