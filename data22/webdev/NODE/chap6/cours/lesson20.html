🔓 Qu'est-ce que JWT ?

JWT (JSON Web Token) est un standard ouvert (RFC 7519) qui définit un moyen compact et auto-suffisant de transmettre des informations entre parties sous forme d'objet JSON.

🔐 Pourquoi l'utiliser ?

Les tokens JWT sont stateless : le serveur n'a pas besoin de stocker de session. Tout est contenu dans le token !

🧩 Structure détaillée

Header (en-tête)

{
  "alg": "HS256",
  "typ": "JWT"
}

Définit l'algorithme de signature (HMAC SHA256) et le type de token.

Payload (charge utile)

{
  "userId": 123,
  "email": "user@example.com",
  "role": "admin",
  "iat": 1516239022,
  "exp": 1516242622
}

Contient les claims (données). iat = issued at, exp = expiration.

Signature

HMACSHA256(
  base64UrlEncode(header) + "." +
  base64UrlEncode(payload),
  secret
)

Permet de vérifier que le token n'a pas été modifié.

⚙️ Installation & premier token

1. Installer le package

npm install jsonwebtoken

2. Générer un token (exemple simple)

const jwt = require('jsonwebtoken');

const payload = { userId: 42, role: 'user' };
const secret = 'maSuperCleSecrete';

const token = jwt.sign(payload, secret, { expiresIn: '1h' });
console.log(token);

🔍 Explication :

jwt.sign() prend le payload, la clé secrète et des options (expiration). Le token généré est une chaîne sécurisée.

🕵️ Vérifier et décoder un token 

// Vérifier la validité et décoder
const token = "eyJhbGciOiJIUzI1NiIs...";

try {
  const decoded = jwt.verify(token, secret);
  console.log(decoded); // { userId: 42, role: 'user', iat: ..., exp: ... }
} catch (err) {
  console.error('Token invalide ou expiré');
}

// Décoder sans vérification (attention !)
const decodedInfo = jwt.decode(token);
console.log(decodedInfo); // lit le payload sans vérifier signature

✅ verify()

Vérifie la signature ET l'expiration. Lance une erreur si invalide.

⚠️ decode()

Décode le payload sans vérifier la signature. À utiliser uniquement pour inspection.

🛡️ Middleware JWT avec Express 

const jwt = require('jsonwebtoken');

function authenticateToken(req, res, next) {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1]; // Bearer TOKEN

  if (!token) {
    return res.sendStatus(401); // Unauthorized
  }

  jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
    if (err) return res.sendStatus(403); // Forbidden
    req.user = user;
    next();
  });
}

// Protéger une route
app.get('/api/profil', authenticateToken, (req, res) => {
  res.json({ message: 'Profil accessible', user: req.user });
});
💡 Le token est généralement envoyé dans l'en-tête Authorization: Bearer <token>

🔒 Bonnes pratiques

🔑
Clé secrète robuste – utilisez une chaîne longue et aléatoire (via crypto.randomBytes(64).toString('hex')). Stockez-la dans .env.
⏰
Expiration courte – privilégiez expiresIn: '15m' pour les tokens d'accès. Utilisez un refresh token pour les longues sessions.
🌐
Transmission sécurisée – toujours via HTTPS pour éviter l'interception.
⚠️
Ne stockez pas de données sensibles dans le payload (ex: mot de passe). Le payload est simplement encodé en base64 (non chiffré).
📘 La formule mathématique de la signature HMAC : \( \text{signature} = HMAC\text{-}SHA256( base64(header) + '.' + base64(payload), secret ) \)
