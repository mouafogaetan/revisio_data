Socket.IO

Communication temps réel avec Node.js

📡 Événements en direct • 🔄 Bidirectionnel • 💬 Chat & Notifications

Niveau: Intermédiaire | Durée: 45 min

🎯 Objectifs de la leçon

  • Installer et configurer socket.io côté serveur avec Express.
  • Installer et configurer socket.io-client côté navigateur.
  • Comprendre l'émission et la réception d'événements avec emit() et on().
  • Utiliser les namespaces pour séparer la logique.
  • Créer des rooms pour diffuser à des groupes spécifiques.
À la fin, vous serez capable de construire une application de chat ou des notifications en direct !

🔌 Qu'est-ce que Socket.IO ?

Socket.IO est une bibliothèque JavaScript qui permet une communication bidirectionnelle et en temps réel entre un client (navigateur) et un serveur Node.js.

Client ↔️ Serveur (WebSocket + fallbacks)

Il utilise WebSocket quand c'est possible, sinon des techniques alternatives (long polling, etc.). Idéal pour les chats, jeux multijoueurs, notifications push, collaboration en direct.

📦 Installation côté serveur

npm init -y
npm install express socket.io

Création d'un serveur Express de base avec Socket.IO :

const express = require('express');
const http = require('http');
const socketIo = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = socketIo(server);

io.on('connection', (socket) => {
  console.log('Nouvel utilisateur connecté');
});

server.listen(3000, () => console.log('Serveur sur port 3000'));
⚡ Important : Le client doit inclure /socket.io/socket.io.js automatiquement servie par le module.

🌐 Côté client (navigateur)

Dans votre fichier HTML, incluez la bibliothèque client et connectez-vous :

<script src="https://cdn.socket.io/4.5.0/socket.io.min.js"></script>
<script>
  const socket = io('http://localhost:3000');
  // Écouter un événement
  socket.on('connect', () => {
    console.log('Connecté au serveur');
  });
</script>

La fonction io() retourne un objet socket qui permet d'envoyer/recevoir des événements.

La connexion est automatique et se maintient en vie.

📡 Émettre et recevoir des événements

Depuis le serveur :

// Envoyer un message à TOUS les clients connectés
io.emit('nouveauMessage', { texte: 'Bonjour tout le monde' });

// Envoyer à un client spécifique
socket.emit('messagePrive', 'Ceci est privé');

Depuis le client :

// Émettre un événement
socket.emit('chatMessage', 'Salut !');

// Écouter un événement
socket.on('nouveauMessage', (data) => {
  console.log('Message reçu:', data);
});
💡 Les événements peuvent transporter n'importe quel objet JSON (chaînes, nombres, tableaux, objets).

🏷️ Namespaces

Les namespaces permettent de séparer la logique de communication (ex: un namespace pour le chat, un autre pour les notifications).

// Côté serveur
const chatNamespace = io.of('/chat');
chatNamespace.on('connection', (socket) => {
  socket.on('message', (msg) => {
    chatNamespace.emit('message', msg);
  });
});

// Côté client
const chatSocket = io('/chat');
chatSocket.emit('message', 'Hello depuis namespace');

Namespace par défaut : / . Utile pour organiser de grosses applications.

🚪 Rooms (salons)

Une room est un canal à l'intérieur d'un namespace. Les clients peuvent rejoindre/quitter une room pour recevoir des messages groupés.

// Serveur : rejoindre une room
socket.join('salleJeu');
// Envoyer un message à tous dans la room
io.to('salleJeu').emit('evenementRoom', 'Données importantes');

// Client : rien de spécial, l'événement est reçu normalement
Room "générale" Room "jeu" Room "admin"

Les rooms sont gérées automatiquement par Socket.IO, idéal pour des discussions privées ou des groupes.

💬 Exemple: Chat en direct

Structure basique d'un chat avec Socket.IO :

// Serveur
io.on('connection', (socket) => {
  socket.on('chatMessage', (msg) => {
    io.emit('chatMessage', msg);
  });
});

// Client
socket.emit('chatMessage', 'Bonjour !');
socket.on('chatMessage', (msg) => {
  $('#messages').append(`<div>${msg}</div>`);
});

Résultat : chaque message est diffusé instantanément à tous les utilisateurs connectés.

📌 Synthèse & Astuces

  • Socket.IO utilise WebSocket par défaut, mais avec fallback automatique.
  • Toujours gérer la déconnexion : socket.on('disconnect', ...).
  • Pour les applications à grande échelle, utilisez Redis adapter.
  • Les événements personnalisés doivent avoir des noms explicites.
  • Pensez à la sécurité : valider les données entrantes côté serveur.
Documentation officielle : socket.io/docs

  1. Installer Express + Socket.IO.
  2. Afficher une page HTML avec un compteur.
