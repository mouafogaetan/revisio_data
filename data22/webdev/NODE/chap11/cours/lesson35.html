🐳 Docker en 2 minutes

Docker est une plateforme qui permet d'empaqueter une application et ses dépendances dans un conteneur isolé.

Conteneur = environnement portable et léger (pas de machine virtuelle lourde)

✅ Avantages : portable, reproductible, léger, idéal pour le déploiement moderne.

📄 Le Dockerfile

Un fichier texte contenant les instructions pour construire l'image Docker.

# Utiliser l'image officielle Node.js
FROM node:18-alpine

# Définir le répertoire de travail
WORKDIR /app

# Copier les fichiers de dépendances
COPY package*.json ./

# Installer les dépendances
RUN npm install

# Copier le reste du code
COPY . .

# Exposer le port de l'application
EXPOSE 3000

# Commande de démarrage
CMD ["node", "server.js"]
FROM Image de base
WORKDIR Dossier de travail
COPY Copie de fichiers
RUN Exécute commande (build)
EXPOSE Port exposé
CMD Commande par défaut

🔧 Construire et exécuter

Commandes essentielles :

# Construire l'image (docker build -t nom_image chemin)
docker build -t mon-api-node .

# Lancer un conteneur (docker run -p hôte:conteneur nom_image)
docker run -p 3000:3000 mon-api-node

# Mode détaché (background)
docker run -d -p 3000:3000 --name mon-app mon-api-node

# Lister les conteneurs actifs
docker ps

# Arrêter un conteneur
docker stop mon-app

# Supprimer un conteneur
docker rm mon-app
L'option -p 3000:3000 mappe le port 3000 de l'hôte au port 3000 du conteneur.

🐙 Docker Compose

Pour orchestrer plusieurs conteneurs (Node.js + MongoDB / PostgreSQL).

version: '3.8'
services:
  api:
    build: .
    ports:
      - "3000:3000"
    environment:
      - MONGO_URI=mongodb://db:27017/mydb
    depends_on:
      - db
  db:
    image: mongo:6
    ports:
      - "27017:27017"
    volumes:
      - mongo-data:/data/db

volumes:
  mongo-data:

Commandes :

docker-compose up -d     # Lance tous les services en arrière-plan
docker-compose down      # Arrête et supprime les conteneurs
docker-compose logs -f   # Voir les logs en direct

💾 Les volumes Docker

Les conteneurs sont éphémères → les données disparaissent à l'arrêt. Les volumes persistent les données.

Volume nommé
docker volume create

Bind mount
./dossier:/app/data

# Exemple de bind mount pour le développement
docker run -v $(pwd):/app -p 3000:3000 mon-api-node

📌 Avec docker-compose, on déclare les volumes dans la section `volumes` et on les attache aux services.

⚡ Optimiser l'image

Utiliser le multi-stage build pour réduire la taille de l'image finale.

# Étape 1 : build
FROM node:18 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

# Étape 2 : image finale légère
FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
Réduction typique : de 900MB → ~150MB !

☁️ Déployer sur un serveur

  1. Installer Docker sur le serveur (Ubuntu/CentOS).
  2. Copier le code source (ou utiliser Git).
  3. Construire l'image : docker build -t mon-api .
  4. Lancer avec docker run -d -p 80:3000 --restart always mon-api.
  5. Option avancée : utiliser Docker Compose en production.
Exemple sur un VPS (DigitalOcean, OVH, etc.) : 
scp -r ./mon-projet user@server:~
ssh user@server
cd mon-projet
docker build -t api .
docker run -d -p 80:3000 api

🎯 Bonnes pratiques Docker Node.js

  • Utiliser des images officielles node:*-alpine (légères).
  • Ordonner les couches du Dockerfile du plus stable au plus changeant (cache).
  • Ne pas exécuter en tant que root à l'intérieur du conteneur : USER node.
  • Utiliser .dockerignore pour exclure node_modules, logs, etc.
  • Ne pas stocker les secrets dans l'image → variables d'environnement ou secrets Docker.
# .dockerignore
node_modules
npm-debug.log
.env
.git

🧪 Exercice : Dockeriser une API simple

Objectif : Créer un fichier Dockerfile et docker-compose pour une API Express qui se connecte à MongoDB.

Structure attendue :

mon-api/
├── Dockerfile
├── docker-compose.yml
├── package.json
├── server.js
└── .dockerignore
Utilisez docker-compose up -d et connectez-vous à la base via l'URI mongodb://db:27017 (hostname = nom du service).
# docker-compose.yml simplifié
version:'3'
services:
  app:
    build: .
    ports: ["3000:3000"]
    depends_on: [mongo]
  mongo:
    image: mongo:latest
    volumes: ["mongo-data:/data/db"]
volumes:
  mongo-data:
