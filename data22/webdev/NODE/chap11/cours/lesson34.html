Préparer l'application

Avant tout déploiement, on doit configurer :

  • NODE_ENV=production (optimisations, moins de logs)
  • Utiliser dotenv pour charger .env mais ne jamais versionner ce fichier
  • Définir le port via process.env.PORT (Requis pour Heroku/Railway)
  • Script "start": "node app.js" dans package.json
// app.js
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;
app.get('/', (req, res) => res.send('API en ligne !'));
app.listen(PORT, () => console.log(`prêt sur ${PORT}`));

Nginx : Reverse Proxy

Nginx écoute sur le port 80 (HTTP) ou 443 (HTTPS) et redirige le trafic vers votre app Node.js (ex: localhost:3000).

# /etc/nginx/sites-available/monapp
server {
    listen 80;
    server_name mon-domaine.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

🔒 Avantages : gestion SSL, équilibrage de charge, cache, protection contre les attaques.

🔐 Sécurité & configuration

🔑 Ne jamais stocker de clés dans le code. Utiliser .env et les variables d'environnement de la plateforme.
🛡️ Activer Helmet.js et CORS restreint en production.
📁 Les fichiers uploadés : utiliser un service cloud (S3) ou un volume persistant.
# Exemple de gestion des variables
DB_PASSWORD=XXXX
JWT_SECRET=super_long_secret
NODE_ENV=production

🔄 Garder l'application en vie

Sur VPS (sans PaaS), on utilise PM2 ou systemd pour redémarrer l'app en cas de crash.

npm install -g pm2
pm2 start app.js --name "mon-api"
pm2 save
pm2 startup   # redémarrage au boot du serveur
📌 Avec Nginx + PM2 : votre API résiste aux pannes et redémarre automatiquement.

🏁 Déploiement express (VPS Ubuntu)

1. git clone votre projet → npm install

2. Configurer Nginx (proxy vers localhost:3000)

3. Démarrer l'app avec PM2

4. Configurer le firewall (UFW) : autoriser 22, 80, 443

5. Pointer votre domaine (DNS) vers l'IP du serveur

6. Optionnel : Certbot pour HTTPS (Let's Encrypt)
