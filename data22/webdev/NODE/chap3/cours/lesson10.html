📜 Créer des scripts personnalisés

Dans Node.js, le fichier package.json possède une section "scripts". Ces commandes simplifient l'exécution de tâches répétitives.

"scripts": {
  "start": "node server.js",
  "dev": "nodemon server.js",
  "test": "jest",
  "lint": "eslint ."
}
  • npm start : commande standard pour lancer la production.
  • npm run dev : pour le développement (avec rechargement auto).
  • npm test : exécute la suite de tests.
  • Tout script personnalisé s'exécute via npm run <nom>.
💡 Astuce : Les scripts permettent aussi d'enchaîner des commandes avec && ou npm run script1 && npm run script2.

🔄 Nodemon : redémarrage automatique

À chaque modification d’un fichier, nodemon redémarre le serveur Node automatiquement, ce qui accélère le développement.

# Installation globale ou locale
npm install -g nodemon # globale
npm install --save-dev nodemon # recommandé en dev
# Utilisation via script
"dev": "nodemon app.js"

Après installation, exécutez npm run dev et nodemon surveille tous les fichiers .js. Plus besoin de relancer manuellement à chaque changement !

✅ Gain de temps ✅ Garde la console propre ✅ Ignore node_modules par défaut

🔢 Versioning sémantique (Semver)

Chaque paquet npm suit un numéro de version : MAJEUR.MINEUR.PATCH (ex: 4.18.2).

  • MAJEUR : changements incompatibles (breaking changes).
  • MINEUR : ajouts rétrocompatibles.
  • PATCH : corrections de bugs.
^4.18.2

Autorise les versions MINEUR et PATCH supérieures, pas de changement MAJEUR → jusqu'à 4.x.x

^ = compatible avec le MAJEUR
~4.18.2

Autorise uniquement les versions PATCH supérieures (fix de bugs) → 4.18.3, 4.18.5 mais pas 4.19.0

~ = correctifs seulement
🧠 Exemple concret :
"express": "^4.18.0" lors du npm install, vous obtiendrez la dernière version 4.x.x (ex 4.19.2) mais jamais Express 5.

⭐ En résumé & Bonnes pratiques

✓

Toujours utiliser --save-dev pour nodemon (dépendance de développement).

✓

Dans le package.json, préférez des scripts explicites comme "dev": "nodemon src/index.js".

✓

Le symbole ^ est par défaut avec npm install --save. Cela permet de recevoir automatiquement les correctifs mineurs.

✓

Pour éviter les surprises, verrouillez les versions avec package-lock.json – toujours présent dans votre projet.

🧑‍🏫 À retenir :

Les scripts npm transforment votre workflow en une série de commandes simples. Nodemon accélère le développement et semver garantit la stabilité des dépendances.

🧪 Petit exercice pratique

Mettez en application ce que vous avez appris !

1️⃣ Créez un nouveau dossier et initialisez npm init -y.

2️⃣ Ajoutez un script "hello": "node -e \"console.log('Bienvenue Node.js')\"".

3️⃣ Installez nodemon en dev-dépendance, puis créez le script "dev" qui lance un fichier server.js fictif.

4️⃣ Observez la différence entre "^1.2.3" et "~1.2.3" dans le package.json.

💡 Indice : utilisez npm run hello pour tester votre premier script.
