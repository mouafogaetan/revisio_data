Partie 1 : QCM – Concepts fondamentaux (25 points)

Partie 2 : Mise en pratique – API RESTful (30 points)

Contexte :

Vous développez une API de gestion d’articles de blog pour une plateforme éducative. Vous devez implémenter les endpoints suivants en respectant l’architecture MVC.

📌 1. Modèle Mongoose (5 pts)

Écrivez le schéma Mongoose pour un article contenant : titre (String requis), contenu (String), auteur (String), dateCreation (Date, défaut: maintenant). Ajoutez une validation : titre au moins 3 caractères.

📌 2. Route & Contrôleur (10 pts)

Implémentez la route POST /api/articles qui accepte les données JSON, crée un nouvel article en base et retourne l’objet créé avec le statut 201. Gestion des erreurs incluse (try/catch).

📌 3. Récupération paginée (10 pts)

Créez la route GET /api/articles qui accepte les paramètres page et limit (par défaut page=1, limit=10) et renvoie la liste paginée avec le nombre total d’articles. Utilisez skip et limit de Mongoose.

Conseil : Pensez à utiliser async/await, structurez bien votre réponse JSON. Un code propre et commenté rapporte des points.

Partie 3 : Sécurité & JWT – Protection des routes (25 points)

Middleware d'authentification

Implémentez un middleware verifyToken qui extrait le token JWT depuis l’en-tête Authorization: Bearer <token>. Si le token est valide, ajoutez req.userId à la requête, sinon renvoyez une erreur 401.

Hachage de mot de passe

Décrivez avec un extrait de code comment hacher un mot de passe utilisateur avant sauvegarde (modèle User) en utilisant bcrypt, ainsi que la comparaison lors de la connexion.

Question réflexe (5 pts) :

Quelle est la différence entre authentification stateful (sessions) et stateless (JWT) ? Donnez un avantage et un inconvénient de JWT dans le contexte d'une API REST.

Partie 5 : Architecture MVC – Schéma séquentiel (10 points)

Dessinez l’organigramme (ou décrivez clairement) du flux d’une requête POST /api/articles dans une architecture Express + MongoDB : Middleware → Route → Contrôleur → Modèle → Réponse. Vous pouvez utiliser du texte structuré ou un diagramme Mermaid interprété.

flowchart TD
    A[Client HTTP POST] --> B[Middleware express.json]
    B --> C[Route /api/articles]
    C --> D[Contrôleur: validation & extraction]
    D --> E[Modèle Article.create()]
    E --> F[Base MongoDB]
    F --> G[Retour JSON 201]
    G --> H[Client reçoit la réponse]

Vous pouvez soit valider ce schéma, soit proposer des améliorations ou justifier l’ordre des middlewares.
