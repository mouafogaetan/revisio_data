🐳 1. Sécuriser les images conteneur

📌 Pourquoi scanner les images ?

Les images Docker peuvent contenir des vulnérabilités (bibliothèques obsolètes, backdoors). Un scanner de vulnérabilités analyse les couches et les dépendances.

🔧 Outils populaires
  • Trivy (Aqua Security)
  • Clair (CoreOS)
  • Grype (Anchore)
  • Docker Scout
✅ Bonnes pratiques
  • Utiliser des images officielles et minimales (alpine, distroless)
  • Scanner dans le pipeline CI/CD
  • Ne pas exécuter en root
💡 Exemple Trivy : trivy image monapp:latest --severity HIGH,CRITICAL

☸️ 2. Politiques de sécurité Kubernetes

🛡️ Pod Security Standards (PSS) / PodSecurity Admission

Kubernetes intègre trois niveaux : Privileged, Baseline, Restricted. Ils restreignent les comportements des pods (ex: interdire les privilèges root, hostNetwork, etc.)

⚙️ Open Policy Agent (OPA) + Gatekeeper

Permet des politiques avancées (ex: "toutes les images doivent provenir d'un registre privé", "interdire les labels spécifiques"). OPA s'intègre via admission controller.

kind: ConstraintTemplate | kind: K8sRequiredLabels
📌 Évitez les pods en mode privileged 📌 Limitez les capacités Linux 📌 Appliquez SecurityContext

🔒 3. Isoler les conteneurs

🌐 Namespaces Linux

Isolation des processus (PID, réseau, montage, UTS, IPC, User). Un conteneur voit son propre espace réseau et arbre de processus.

⚖️ Cgroups

Limitent les ressources (CPU, mémoire, I/O). Empêchent un conteneur de monopoliser l'hôte. Évite les attaques par déni de service.

🔐 Runtime Security : Falco (détection des comportements anormaux), AppArmor/SELinux, seccomp.

# Exemple d'annotation Kubernetes pour seccomp : container.seccomp.security.alpha.kubernetes.io/pod: localhost/profiles/audit.json

Bonus : Utilisez gVisor ou Kata Containers pour une isolation renforcée.

📌 Synthèse et mise en pratique

  • ➜ Avant déploiement : Scanner les images (Trivy, Clair).
  • ➜ Au niveau cluster : Activer Pod Security Admission (Restricted par défaut).
  • ➜ Politiques avancées : OPA/Gatekeeper pour règles custom.
  • ➜ Runtime : Activer Falco pour détection des menaces, AppArmor/SELinux.
  • ➜ Limiter les ressources : requests/limits CPU & mémoire évitent les dénis de service.
🧪 Atelier suggéré : Déployer un pod Nginx avec `securityContext.privileged: false` et lui appliquer un profil seccomp personnalisé. Scanner l'image avec Trivy.

🧩 Isolation des conteneurs : Vue simplifiée

Chaque conteneur possède ses propres namespaces et limites de ressources, isolés les uns des autres. Des couches de sécurité (seccomp, Falco) surveillent les appels système.
