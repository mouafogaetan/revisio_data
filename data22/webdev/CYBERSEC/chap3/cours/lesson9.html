🔍 IDS (Intrusion Detection System)

Système passif qui analyse le trafic réseau ou les logs pour détecter des activités suspectes. Il alerte l’administrateur mais n’empêche pas l’attaque.

📡 Exemple : Un IDS voit une tentative d’attaque SQL injection et envoie une alerte, mais la requête malveillante arrive jusqu’au serveur.

🛑 IPS (Intrusion Prevention System)

Système actif placé en ligne (inline). Non seulement il détecte, mais il bloque automatiquement les trafics malveillants avant qu'ils n'atteignent la cible.

⚡ Exemple : Un IPS analyse un paquet contenant une signature d’exploit et le rejette immédiatement. L’attaque échoue.

IDS → surveille & alerte 🚨
IPS → surveille & bloque 🚫

📊 Les 3 grandes méthodes de détection

📄Signature-based

Compare le trafic à une base de signatures d'attaques connues (comme un antivirus). Rapide, efficace sur les menaces connues, mais inefficace face aux zero-day.

📈Anomaly-based

Établit un profil de "comportement normal" et détecte les écarts statistiques (ex: trafic soudain anormal). Peut générer des faux positifs mais détecte des attaques inconnues.

🧠Comportementale (behavioral)

Analyse des séquences d’événements, corrèle des activités sur le réseau pour repérer des comportements malicieux complexes (C2, exfiltration lente). Plus sophistiqué.

🛠️ Outils célèbres IDS/IPS open source

SNORT

IDS/IPS le plus connu. Utilise des règles signatures, supporte le mode inline. Très performant pour la détection.

⭐ Référence historique
SURICATA

Moderne, multi-thread, accélération GPU, compatible règles Snort. Excellent pour le trafic à haut débit.

⚡ Performance & TLS
ZEEK (ex-Bro)

Orienté analyse réseau, génère des logs riches, langage scriptable. Plutôt IDS mais très efficace pour la métrologie.

🔍 Analyse profonde
💬 Pour les débutants : Snort est excellent pour apprendre, Suricata est son grand frère plus rapide, Zeek est parfait pour fouiller finement les connexions.

🌐 Où placer IDS/IPS ?

En entreprise, on installe souvent un IPS en périphérie (derrière le pare-feu) et des IDS dans les segments sensibles (DMZ, LAN interne).

📌 Bon à savoir : L'IPS étant en ligne, il peut introduire une latence. L'IDS, hors bande, ne perturbe pas le trafic mais n’arrête pas les attaques. On les utilise souvent en complément.

📌 Exemples d’utilisation quotidienne

  • Signature IDS : Snort détecte un scan de ports Nmap typique → alerte sur console.
  • IPS en action : Suricata voit une tentative d’exploitation de vulnérabilité SMB et rejette le paquet, protégeant le poste.
  • Détection d’anomalie : Zeek signale qu’un serveur transfère 10 Go vers l’extérieur à minuit (exfiltration possible).
  • Sécurité Cloud : Les appliances IDS/IPS virtuelles sont déployées sur AWS/Azure pour inspecter le trafic VPC.
🧪 Activité pratique possible : installer Snort sur une VM Linux et générer des alertes sur des ping anormaux.
