🎯 Objectifs de la leçon

1. Typer les props d’un composant avec interface ou type.
2. Utiliser les types génériques (React.FC<Props>, useState<Type>, useRef<HTMLInputElement>).
3. Typer les événements : React.MouseEvent, React.ChangeEvent, etc.
4. Écrire des composants robustes avec TypeScript pour éviter les bugs à l’exécution.
✨ TypeScript + React = productivité et confiance dans le typage des props et des callbacks.

📦 Typer les props : interface

On utilise interface ou type pour décrire la forme des props.

// Composant Carte.tsx
interface CarteProps {
  titre: string;
  contenu: string;
  actif?: boolean;        // optionnel
  onClique?: () => void;
}

const Carte: React.FC<CarteProps> = ({ titre, contenu, actif = true, onClique }) => {
  return (
    <div onClick={onClique} className={actif ? 'bg-white' : 'bg-gray-200'}>
      <h3>{titre}</h3>
      <p>{contenu}</p>
    </div>
  );
};

✔️ TypeScript vérifie que les props obligatoires sont fournies et que les types correspondent.

⚛️ Hooks génériques : useState et useRef

// useState avec type explicite
const [compteur, setCompteur] = useState<number>(0);
const [utilisateur, setUtilisateur] = useState<{nom: string; age: number} | null>(null);

// useRef pour référencer un élément DOM
const inputRef = useRef<HTMLInputElement>(null);
const focusInput = () => inputRef.current?.focus();

// useRef pour stocker une valeur mutable (sans re-rendu)
const intervalId = useRef<number | undefined>(undefined);

💡 Les génériques aident TypeScript à inférer correctement les types retournés et à éviter les erreurs.

🖱️ Événements typés

React fournit des types pour chaque événement : React.MouseEvent, React.ChangeEvent, React.FormEvent...

const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
  console.log('click sur', e.currentTarget);
};

const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  setTexte(e.target.value);
};

const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();
  // ...
};

<button onClick={handleClick}>Clique</button>
<input onChange={handleChange} />
✅ Le typage évite les erreurs comme l’accès à e.target.value sur un bouton.

📝 Exemple complet : formulaire typé

interface UserForm {
  email: string;
  password: string;
  remember: boolean;
}

const LoginForm = () => {
  const [form, setForm] = useState<UserForm>({ email: '', password: '', remember: false });

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const { name, value, type, checked } = e.target;
    setForm(prev => ({ ...prev, [name]: type === 'checkbox' ? checked : value }));
  };

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log(form);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name=\"email\" value={form.email} onChange={handleChange} />
      <input name=\"password\" type=\"password\" onChange={handleChange} />
      <input name=\"remember\" type=\"checkbox\" checked={form.remember} onChange={handleChange} />
      <button type=\"submit\">Se connecter</button>
    </form>
  );
};

🧠 Bonnes pratiques

  • Préférer interface pour les props d’un composant (extensible).
  • Utiliser React.FC<Props> ou typer directement (props: Props) => JSX.Element.
  • Ne pas oublier children si nécessaire : interface PropsWithChildren.
  • Pour les refs : useRef<HTMLDivElement>(null) et vérifier current avant appel.
  • Typer les gestionnaires d’événements pour éviter les any implicites.

🚀 Conclusion

Typer son code React avec TypeScript permet :

  • une meilleure autocomplétion dans l’IDE,
  • une réduction des bugs liés aux types (props manquantes, mauvais type d’événement),
  • une documentation vivante grâce aux interfaces.
