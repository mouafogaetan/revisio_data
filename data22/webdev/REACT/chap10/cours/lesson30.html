🏗️ Composants génériques et réutilisables

Un composant générique peut être utilisé dans différents contextes avec des données différentes.

// ❌ Mauvais - composant trop spécifique
function CarteProduitChaussure({ nom, prix }) {
  return <div>Chaussure {nom} - {prix}€</div>
}

// ✅ Bon - composant générique
function Carte({ titre, description, imageUrl, onClique }) {
  return (
    <div className="border rounded p-4 shadow">
      <img src={imageUrl} alt={titre} />
      <h3>{titre}</h3>
      <p>{description}</p>
      <button onClick={onClique}>En savoir plus</button>
    </div>
  );
}

✅ Conseil : Un composant générique ne connaît pas son contexte d'utilisation. Il se contente d'afficher ce qu'on lui donne via les props.

👶 Pattern children (composition)

children permet de passer du JSX arbitraire à l'intérieur d'un composant.

// Composant conteneur générique
function Card({ title, children, className }) {
  return (
    <div className={`border rounded-lg p-4 ${className}`}>
      <h3 className="font-bold text-lg mb-2">{title}</h3>
      <div className="content">{children}</div>
    </div>
  );
}

// Utilisation - on peut y mettre N'IMPORTE QUOI !
<Card title="Bienvenue">
  <p>Ceci est un paragraphe</p>
  <button>Cliquez ici</button>
</Card>

<Card title="Liste d'élèves">
  <ul>
    <li>Amina</li>
    <li>Samuel</li>
  </ul>
</Card>

🎨 Pattern Render Props

Une prop qui est une fonction retournant du JSX. Très puissant pour partager de la logique.

// Composant qui partage l'état du compteur
function Counter({ render, initialCount = 0 }) {
  const [count, setCount] = useState(initialCount);
  
  const increment = () => setCount(c => c + 1);
  const decrement = () => setCount(c => c - 1);
  
  return render({ count, increment, decrement });
}

// Utilisation : chaque appel peut avoir une UI différente !
<Counter
  initialCount={5}
  render={({ count, increment, decrement }) => (
    <div>
      <button onClick={decrement}>-</button>
      <span>{count}</span>
      <button onClick={increment}>+</button>
    </div>
  )}
/>

✏️ Composants contrôlés

La valeur de l'input est contrôlée par React via value et onChange.

function FormulaireControle() {
  const [nom, setNom] = useState('');
  const [email, setEmail] = useState('');
  
  const handleSubmit = (e) => {
    e.preventDefault();
    console.log({ nom, email });
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input 
        type="text"
        value={nom}
        onChange={(e) => setNom(e.target.value)}
        placeholder="Nom"
      />
      <input 
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Email"
      />
      <button type="submit">Envoyer</button>
    </form>
  );
}

🎯 Avantage : Vous avez un contrôle total sur la valeur et pouvez la valider, la transformer ou la synchroniser avec d'autres champs.

📝 Composants non contrôlés

L'input gère sa propre valeur. React n'intervient pas. On utilise useRef pour lire la valeur au moment voulu.

function FormulaireNonControle() {
  const nomRef = useRef(null);
  const emailRef = useRef(null);
  
  const handleSubmit = (e) => {
    e.preventDefault();
    console.log({
      nom: nomRef.current.value,
      email: emailRef.current.value
    });
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input type="text" ref={nomRef} placeholder="Nom" />
      <input type="email" ref={emailRef} placeholder="Email" />
      <button type="submit">Envoyer</button>
    </form>
  );
}

⚡ Avantage : Plus simple, moins de re-rendus. Idéal pour les formulaires simples ou l'intégration avec du code non-React.

⚖️ Contrôlé vs Non contrôlé : le guide

SituationRecommandation
Validation en temps réel✅ Contrôlé
Formatage automatique (téléphone, carte bancaire)✅ Contrôlé
Formulaire simple avec peu d'interactions✅ Non contrôlé
Intégration avec bibliothèque tierce (jQuery, etc.)✅ Non contrôlé
Champ avec valeur par défaut statique✅ Non contrôlé
Performance critique (très grand nombre d'inputs)✅ Non contrôlé

📌 Bonnes pratiques composants réutilisables

  • • Props bien nommées : claires, explicites et typées (si TypeScript)
  • • Valeurs par défaut : utilisez des valeurs par défaut pour les props optionnelles
  • • Composition plutôt que configuration : préférez children \u00e0 une multiplication de props booléennes
  • • Un composant = une responsabilité : si un composant fait trop de choses, découpez-le
  • • Documentez les composants réutilisables (Storybook, commentaires)

⭐ Le secret : Pensez toujours "Est-ce que ce composant pourrait être utile ailleurs ?"

📌 Points clés à retenir :

  • Composants génériques = props + indifférence au contexte
  • Pattern children pour conteneurs flexibles
  • Render props pour partager de la logique
  • Contrôlé = React gère la valeur / Non contrôlé = DOM gère la valeur
  • Choisissez selon vos besoins (validation, performance, simplicité)
