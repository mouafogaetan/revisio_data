La méthode map()

`map()` parcourt un tableau et retourne un nouveau tableau. En React, on l'utilise pour transformer des données en JSX.

function ListeFruits() {
  const fruits = ["Pomme", "Banane", "Orange", "Mangue"];

  return (
    <ul>
      {fruits.map((fruit) => (
        <li key={fruit}>🍎 {fruit}</li>
      ))}
    </ul>
  );
}

✨ Rendu dans le navigateur :

  • 🍎 Pomme
  • 🍎 Banane
  • 🍎 Orange
  • 🍎 Mangue

🔑 La prop `key` est OBLIGATOIRE

React utilise la `key` pour identifier chaque élément d'une liste. Cela lui permet de savoir quels éléments ont changé, été ajoutés ou supprimés.

⚠️ Sans key, une erreur apparaît dans la console :

Warning: Each child in a list should have a unique "key" prop.

// ❌ Mauvais (pas de key)
{fruits.map(fruit => <li>{fruit}</li>)}

// ✅ Bon (avec key)
{fruits.map(fruit => <li key={fruit}>{fruit}</li>)}

✅ Choisir une bonne key

La clé doit être unique et stable parmi les éléments de la liste.

✔️ Bonnes clés :

  • ID d'une base de données : key={user.id}
  • Identifiant unique généré : key={uuid()}
  • Combinaison unique : key={`${name}-${index}`}

❌ Mauvaises clés :

  • L'index du tableau (sauf liste statique)
  • Une valeur qui peut changer (ex: date)
  • Une valeur non unique (ex: nom répété)
💡 L'index n'est pas recommandé car si vous ajoutez/supprimez des éléments au début, tous les indices changent et React va re-rendre toute la liste.

⚡ key et performances

Une bonne `key` permet à React d'optimiser le "reconciliation" (mise à jour du DOM virtuel).

✓
React compare les anciennes et nouvelles clés pour savoir quels éléments réutiliser.
✗
Avec l'index comme clé, si vous insérez un élément en tête, tous les éléments sont re-rendus !

Illustration : insertion au début d'une liste

Avec index (❌)
[0,1,2] → [0,1,2,3]
Tous les index changent → tout re-rend
Avec ID unique (✅)
[id1,id2] → [id0,id1,id2]
Seul le nouvel élément est ajouté

📌 Bonnes pratiques

1️⃣ Utiliser un ID existant

Si vos données viennent d'une API, utilisez l'ID fourni : key={user.id}

2️⃣ Générer un ID temporaire

Pour une liste locale sans ID, utilisez crypto.randomUUID() ou Date.now() au moment de la création.

3️⃣ L'index en dernier recours

Utilisez l'index SEULEMENT si la liste est statique (ne change jamais : tri, ajout, suppression).
