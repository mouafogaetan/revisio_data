Les Props dans React

Transmettez des données entre composants comme un chef !

Props Composants React Communication

Cliquez sur "Suivant" pour découvrir les props

🎯 Objectifs de la leçon

  • ✓ Passer des données d'un composant parent à un composant enfant via les props
  • ✓ Utiliser la destructuration des props pour un code plus propre
  • ✓ Comprendre l'immutabilité : les props sont en lecture seule
  • ✓ Créer des composants réutilisables grâce à la puissance des props

💡 Pourquoi c'est important ? Les props sont le cœur de la communication entre composants React. Sans elles, vos composants seraient isolés et incapables d'afficher des données dynamiques !

📦 Qu'est-ce qu'une prop ?

Une prop (propriété) est un objet JavaScript qui contient toutes les valeurs passées à un composant. C'est comme un attribut HTML mais pour vos composants React !

// App.jsx - Parent

<Salutation nom="Jean" age={25} />

// Salutation.jsx - Enfant

function Salutation(props) {

return <h1>Bonjour {props.nom}, tu as {props.age} ans</h1>;

}

✨ Analogies : Une prop est comme un cadeau qu'on offre à un composant enfant. Le parent prépare le cadeau, l'enfant l'ouvre et l'utilise !

🔧 Utilisation des props (sans et avec destructuration)

Deux manières d'accéder aux props dans votre composant enfant :

Sans destructuration

function Carte(props) {
  return (
    <div>
      <h2>{props.titre}</h2>
      <p>{props.description}</p>
    </div>
  );
}

Avec destructuration (✅ recommandé)

function Carte({ titre, description }) {
  return (
    <div>
      <h2>{titre}</h2>
      <p>{description}</p>
    </div>
  );
}

✅ Bonnes pratiques : Toujours utiliser la destructuration ! Cela rend le code plus lisible, évite de répéter props. et facilite le débogage.

🔒 Les props sont immutables (lecture seule)

Une règle fondamentale : on ne peut PAS modifier une prop directement à l'intérieur d'un composant enfant.

❌ CE CODE EST INTERDIT :

function MaTodo({ texte }) {
  texte = texte.toUpperCase(); // ❌ ERREUR !!!
  return <li>{texte}</li>;
}

✅ Solution : Si vous avez besoin de modifier une donnée, utilisez useState (état local). Les props viennent du parent et appartiennent au parent.

💡 Pourquoi ? Pour garder un flux de données à sens unique (parent → enfant). Cela rend l'application plus prévisible et facile à déboguer.

📊 Tous types de données dans les props

Vous pouvez passer n'importe quelle valeur JavaScript :

nom="Marie"
Chaîne de caractères
age={25}
Nombre
estActif={true}
Booléen
tags={['react', 'js']}
Tableau
utilisateur={{ nom: 'Ali', ville: 'Douala' }}
Objet (avec doubles accolades !)

⚠️ Attention : Pour passer un nombre, un booléen, un tableau ou un objet, on utilise { } (accolades). Pour une chaîne de caractères, on peut utiliser des guillemets.

🎨 Valeurs par défaut avec defaultProps

Vous pouvez définir des valeurs par défaut pour éviter les erreurs :

function Bienvenue({ nom, role }) {
  return <p>{nom || 'Visiteur'} est {role || 'invité'}</p>
}

// OU avec defaultProps (ancienne syntaxe)
Bienvenue.defaultProps = {
  nom: 'Visiteur',
  role: 'invité'
};

✨ Astuce : Dans les projets modernes, on utilise souvent l'opérateur || ou l'opérateur ?? (nullish coalescing) directement dans le composant.

👶 La prop spéciale children

children est une prop particulière qui contient tout ce qui est placé entre les balises d'ouverture et de fermeture d'un composant.

// Composant Card
function Card({ children, titre }) {
  return (
    <div className="border rounded p-4">
      <h3>{titre}</h3>
      <div>{children}</div>  {/* Contenu imbriqué */}
    </div>
  );
}

// Utilisation
<Card titre="Mon article">
  <p>Ce texte est la prop children !</p>
  <button>Lire plus</button>
</Card>

💡 Utilité : children permet de créer des composants conteneurs génériques (Cartes, Modales, Layouts) très flexibles.

🏫 Exemple concret : Liste d'étudiants

// Composant Enfant (EtudiantCard)
function EtudiantCard({ nom, note, estAdmis }) {
  return (
    <div className="p-2 border-b">
      <strong>{nom}</strong> - Note: {note}/20
      {estAdmis ? ' ✅ Admis' : ' ❌ Non admis'}
    </div>
  );
}

// Composant Parent (ListeEtudiants)
function ListeEtudiants() {
  const etudiants = [
    { id: 1, nom: 'Amina', note: 16, admis: true },
    { id: 2, nom: 'Samuel', note: 9, admis: false }
  ];
  
  return (
    <div>
      {etudiants.map(etu => (
        <EtudiantCard 
          key={etu.id}
          nom={etu.nom}
          note={etu.note}
          estAdmis={etu.admis}
        />
      ))}
    </div>
  );
}

Félicitations ! 🎉

Vous maîtrisez maintenant les props : le langage universel de communication entre composants React.

📌 Points clés à retenir :

  • Les props transmettent des données du parent vers l'enfant
  • Utilisez la destructuration : ({ nom, age })
  • Les props sont immutables (ne pas les modifier)
  • La prop children contient le contenu imbriqué
