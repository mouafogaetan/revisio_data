📦 La propriété `children`

`children` est une prop spéciale qui représente le contenu placé entre les balises d'ouverture et de fermeture d'un composant.

// Composant Wrapper
function Wrapper({ children }) {
  return <div className="border p-4 rounded-lg">{children}</div>;
}

// Utilisation : ce qui est à l'intérieur devient children
<Wrapper>
  <h3>Bonjour Cameroun !</h3>
  <p>React est puissant</p>
</Wrapper>

✔️ Pas besoin de passer manuellement des props : tout élément imbriqué est accessible via `children`.

✨ Avantage : flexibilité totale pour personnaliser l'intérieur d'un composant générique.

🃏 Composant Card réutilisable

Une `Card` peut avoir un titre, une image et des actions variables via `children`.

function Card({ title, children, footer }) {
  return (
    <div className="bg-white shadow-md rounded-xl overflow-hidden">
      {title && <div className="bg-indigo-600 text-white p-3 font-bold">{title}</div>}
      <div className="p-4">{children}</div>
      {footer && <div className="border-t p-3 text-gray-500 text-sm">{footer}</div>}
    </div>
  );
}

// Exemple d'utilisation
<Card title="Développeur React" footer="Mise à jour 2026">
  <p>Apprenez à composer des interfaces modernes.</p>
  <button className="bg-blue-500 text-white px-3 py-1 rounded">Action</button>
</Card>

🪟 Composant Modal générique

Une modale réutilisable qui contrôle son état et affiche un contenu personnalisé.

function Modal({ isOpen, onClose, title, children }) {
  if (!isOpen) return null;
  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center z-50">
      <div className="bg-white rounded-xl max-w-md w-full p-5">
        <div className="flex justify-between">
          <h3 className="text-xl font-bold">{title}</h3>
          <button onClick={onClose} className="text-gray-500">✖</button>
        </div>
        <div className="mt-3">{children}</div>
      </div>
    </div>
  );
}
💡 `children` accepte du texte, des composants complexes ou même d'autres modales.

🔁 Exercice mental : Une barre d'outils peut recevoir des boutons via `children` plutôt que d'être codée en dur.

Mathématiquement, on peut voir la composition comme une fonction : $C(f,g) = f \circ g$. React applique ce concept aux interfaces.

⚛️ Composition avancée : React.cloneElement

Parfois, on veut ajouter automatiquement des props aux enfants. Exemple : un composant `List` qui injecte une classe à chaque élément.

function List({ items, renderItem }) {
  return <ul>{items.map(renderItem)}</ul>;
}
// Ou bien en utilisant children avec map
function ButtonGroup({ children }) {
  return React.Children.map(children, child =>
    React.cloneElement(child, { className: 'group-btn' })
  );
}

⚠️ Astuce : `React.Children.map` gère les enfants uniques ou multiples et permet d'ajouter des comportements sans casser la réutilisabilité.

📌 Récapitulatif & bonnes pratiques

  • Utilisez `children` pour créer des composants génériques comme des cartes, alertes, modales.
  • Préférez la composition à l'héritage (pas de classes extends Component inutiles).
  • Nommez les props de façon explicite : `renderHeader`, `footer` permettent une meilleure lisibilité.
  • Utilisez `React.Children` pour manipuler dynamiquement les enfants.
  • Pensez à la réutilisabilité : un composant bien composé peut être utilisé dans plusieurs contextes (mobile, desktop, dark mode).

🔗 Exemple concret : Créez une Alert personnalisable avec type success/error

<Alert type="success">✓ Opération réussie</Alert>
<Alert type="error">⚠️ Une erreur est survenue</Alert>
