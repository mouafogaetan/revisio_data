Choisir le bon projet

Pour un premier projet React, voici trois niveaux de difficulté :

🌱 Débutant

  • Todo-list
  • Calculateur
  • Galerie photo

📘 Intermédiaire

  • Blog avec API
  • Météo
  • Gestionnaire de tâches

🚀 Avancé

  • E-commerce
  • Réseau social
  • Dashboard analytics
✨ Conseil : Commencez par une todo-list, puis ajoutez une API, puis l'authentification.

✂️ Découper en composants

Appliquez le principe de responsabilité unique : chaque composant fait une chose.

❌ Mauvais découpage

function App() {
  // Affichage header
  // Formulaire d'ajout
  // Liste des tâches
  // Pied de page
  // TOUT DANS UN SEUL COMPOSANT !
}

✅ Bon découpage

<App>
  <Header />
  <TodoForm />
  <TodoList />
  <Footer />
</App>
💡 Règle simple : si un composant dépasse 150-200 lignes, c'est probablement qu'il fait trop de choses.

📁 Structure professionnelle

src/
├── components/        # Composants réutilisables (Button, Card, Modal)
│   ├── Button.jsx
│   ├── Card.jsx
│   └── Navbar.jsx
├── pages/             # Pages complètes (Home, About, Dashboard)
│   ├── Home.jsx
│   ├── About.jsx
│   └── Dashboard.jsx
├── hooks/             # Hooks personnalisés (useFetch, useLocalStorage)
│   ├── useFetch.js
│   └── useLocalStorage.js
├── context/           # Contextes React (ThemeContext, AuthContext)
│   └── AuthContext.jsx
├── services/          # Appels API (api.js)
│   └── api.js
├── utils/             # Fonctions utilitaires (formatDate, helpers)
│   └── helpers.js
├── styles/            # Fichiers CSS globaux
│   └── global.css
├── App.jsx
└── main.jsx

Code minimal de TodoItem

function TodoItem({ todo, onToggle, onDelete }) {
  return (
    <li>
      <input 
        type="checkbox" 
        checked={todo.completed}
        onChange={() => onToggle(todo.id)}
      />
      <span className={todo.completed ? 'line-through' : ''}>
        {todo.text}
      </span>
      <button onClick={() => onDelete(todo.id)}>✖</button>
    </li>
  );
}

🎨 Créer des composants génériques

Des composants réutilisables dans toute l'application :

// components/Button.jsx - Version générique
function Button({ children, variant = 'primary', onClick, disabled = false }) {
  const variants = {
    primary: 'bg-blue-600 hover:bg-blue-700',
    secondary: 'bg-gray-500 hover:bg-gray-600',
    danger: 'bg-red-600 hover:bg-red-700'
  };
  
  return (
    <button 
      className={`px-4 py-2 rounded-lg text-white transition ${variants[variant]} ${disabled ? 'opacity-50' : ''}`}
      onClick={onClick}
      disabled={disabled}
    >
      {children}
    </button>
  );
}

// Utilisation
<Button variant="primary" onClick={handleSave}>Enregistrer</Button>
<Button variant="danger" onClick={handleDelete}>Supprimer</Button>
✨ Avantage : Un seul endroit pour modifier le style de TOUS les boutons.

📌 Bonnes pratiques de structuration

1️⃣
Un dossier par fonctionnalité (feature-based) pour les gros projets : features/auth/, features/products/
2️⃣
Nommer les fichiers avec .jsx pour les composants React, .js pour les utilitaires
3️⃣
Exporter nommé vs défaut : privilégier l'export nommé pour éviter les confusions
4️⃣
Fichier index.js pour réexporter facilement (export { default as Button } from './Button')

🏋️ Exercice pratique

Imaginez une application de galerie photos avec les fonctionnalités :

  • Affichage des photos (grille)
  • Barre de recherche
  • Filtre par catégorie
  • Upload de nouvelle photo
  • Like sur une photo

À faire :

  1. Identifier les composants nécessaires (minimum 5)
  2. Dessiner l'arborescence des dossiers
  3. Proposer la structure des props pour chaque composant

💡 Piste de correction :

Composants : PhotoGrid, PhotoCard, SearchBar, FilterButtons, UploadModal
Dossiers : components/, pages/Gallery.jsx, hooks/usePhotos.js, services/photoApi.js
