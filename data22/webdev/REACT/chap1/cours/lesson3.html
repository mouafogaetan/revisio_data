🔍 Qu'est-ce que JSX ?

JSX (JavaScript XML) est une extension syntaxique de JavaScript. Il permet d'écrire du code qui ressemble à du HTML directement dans vos fichiers JavaScript.

❌ Sans JSX (React.createElement)

const element = React.createElement( 'h1', { className: 'titre' }, 'Bonjour le monde !' );

✅ Avec JSX (plus lisible)

const element = <h1 className="titre"> Bonjour le monde ! </h1>;

💡 À retenir : JSX est transformé en React.createElement() par Babel. C'est du sucre syntaxique !

📐 Les accolades {} : passer du JavaScript en JSX

Les accolades permettent d'insérer n'importe quelle expression JavaScript dans votre JSX.

function MonComposant() { const nom = "Marie"; const age = 25; const estMajeur = age >= 18; return ( <div> <h1>Bonjour {nom} !</h1> <p>Vous avez {age} ans.</p> <p>Vous êtes {estMajeur ? "majeur" : "mineur"}.</p> <p>2 + 2 = {2 + 2}</p> </div> ); }
✅ Variables : {nom}
✅ Opérateurs : {age + 5}
✅ Ternaires : {condition ? "Vrai" : "Faux"}
✅ Appels de fonction : {formaterDate(now)}

🔄 Différences entre JSX et HTML

1. className au lieu de class

En JSX, on utilise className car class est un mot réservé en JavaScript.

<div className="ma-classe">Contenu</div>

2. CamelCase pour les styles inline

<div style={{ backgroundColor: 'blue', fontSize: '16px' }}>Texte stylé</div>

📌 Note : les styles prennent un objet, et les propriétés sont en camelCase.

3. Auto-fermeture des balises orphelines

<img src="/photo.jpg" alt="description" /><br /><input type="text" />

Toutes les balises sans enfant doivent être auto-fermées avec />.

🎯 Rendu conditionnel et listes

Grâce aux accolades, on peut afficher du JSX de manière conditionnelle ou répéter des éléments.

function ListeTaches({ taches, estConnecte }) { return ( <div> {/* Condition avec && */} {estConnecte && <p>Bienvenue utilisateur !</p>} {/* Condition if/else avec ternaire */} {taches.length === 0 ? ( <p>Aucune tâche pour le moment</p> ) : ( <ul> {taches.map((tache, index) => ( <li key={index}>{tache.nom}</li> ))} </ul> )} </div> ); }

⚠️ Attention : Les commentaires en JSX s'écrivent {/* ... */} et non pas // ou /* */.

✏️ Exercice : Transformez du HTML en JSX

Consigne : Convertissez le code HTML suivant en JSX valide :

<div class="card"> <h2>Bonjour à tous</h2> <img src="photo.jpg" alt="photo"> <p style="color: red; font-size: 18px">Ceci est un texte</p> <input type="text" placeholder="Votre nom"> <button onclick="alert('Cliqué')">Cliquez</button> </div>
<div className="card"> <h2>Bonjour à tous</h2> <img src="photo.jpg" alt="photo" /> <p style={{ color: 'red', fontSize: '18px' }}>Ceci est un texte</p> <input type="text" placeholder="Votre nom" /> <button onClick={() => alert('Cliqué')}>Cliquez</button> </div>
  • class → className
  • style reçoit un objet avec camelCase
  • img, input doivent être auto-fermés ( /> )
  • onclick devient onClick et reçoit une fonction
