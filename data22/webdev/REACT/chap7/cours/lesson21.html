🎯 Objectifs de la leçon

  • ✓ Créer un thème global (Dark mode / Light mode) avec Context

  • ✓ Optimiser avec la séparation des contextes pour éviter les re-rendus inutiles
  • ✓ Combiner plusieurs contexts dans une application réelle

💡 Pourquoi c'est important ? La Context API est la solution officielle de React pour partager des données globales (thème, utilisateur, langue) sans passer par des props intermédiaires. Bien l'utiliser améliore les performances et la maintenabilité !

📦 Rappel : Créer et utiliser un Context

// 1. Créer le contexte
const ThemeContext = createContext();

// 2. Fournisseur (Provider)
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <ComposantEnfant />
    </ThemeContext.Provider>
  );
}

// 3. Consommateur (useContext)
function ComposantEnfant() {
  const theme = useContext(ThemeContext);
  return <div>Thème : {theme}</div>
}

✨ Rappel : createContext + Provider + useContext = la base de la Context API !

🌓 Thème global : Dark / Light mode

// ThemeContext.jsx
import { createContext, useState, useContext } from 'react';

const ThemeContext = createContext();

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  
  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };
  
  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      <div className={theme}>{children}</div>
    </ThemeContext.Provider>
  );
}

export const useTheme = () => useContext(ThemeContext);

💡 Astuce : Exportez un hook personnalisé useTheme pour simplifier l'utilisation !

🔐 Gestion de l'authentification

// AuthContext.jsx
const AuthContext = createContext();

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  
  const login = (email, password) => {
    // Appel API ici
    setUser({ email, name: 'Jean' });
  };
  
  const logout = () => setUser(null);
  
  return (
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => useContext(AuthContext);

// Utilisation dans un composant
function Profile() {
  const { user, logout } = useAuth();
  if (!user) return <p>Non connecté</p>;
  return <button onClick={logout}>Déconnexion ({user.email})</button>
}

⚠️ Problème : re-rendus inutiles

Par défaut, tous les composants consommateurs d'un contexte sont re-rendus lorsque la valeur du contexte change, même s'ils n'utilisent qu'une petite partie des données.

❌ Problème typique :

// Mauvaise pratique : Tout dans un seul contexte
const AppContext = createContext();
// user, theme, language, notifications... TOUT ensemble !
// Une seule modification → TOUS les composants re-rendent

Solution : Séparer les contextes par domaine !

✨ Optimisation : Séparer les contextes

// ✅ Bonne pratique : Contextes séparés
function App() {
  return (
    <AuthProvider>      {/* Gère user et auth */}
      <ThemeProvider>   {/* Gère le thème */}
        <LanguageProvider>{/* Gère la langue */}
          <MainContent />
        </LanguageProvider>
      </ThemeProvider>
    </AuthProvider>
  );
}

AuthContext

user, login, logout

ThemeContext

theme, toggleTheme

NotifContext

notifications, addNotif

✅ Résultat : Si le thème change, seuls les composants qui dépendent du thème sont re-rendus. Les composants qui n'utilisent que l'auth ou les notifications ne sont PAS affectés !

🏗️ Exemple : Application Camerounaise

// App.jsx - Structure complète
import { AuthProvider, useAuth } from './contexts/AuthContext';
import { ThemeProvider, useTheme } from './contexts/ThemeContext';

function Header() {
  const { theme, toggleTheme } = useTheme();  // Re-rendu uniquement si theme change
  const { user } = useAuth();                 // Re-rendu uniquement si user change
  
  return (
    <header>
      <span>Bienvenue {user?.name || 'Visiteur'}</span>
      <button onClick={toggleTheme}>Mode {theme === 'light' ? '🌙' : '☀️'}</button>
    </header>
  );
}

function App() {
  return (
    <AuthProvider>
      <ThemeProvider>
        <Header />
        <ContenuPrincipal />
      </ThemeProvider>
    </AuthProvider>
  );
}

📌 Bonnes pratiques Context API

  • • Séparez par domaine fonctionnel : Auth, Theme, Notifications, Langue...
  • • Créez des hooks personnalisés (useAuth, useTheme) pour chaque contexte
  • • Évitez de mettre trop de données dans un seul contexte
  • • Utilisez React.memo sur les composants lourds qui ne dépendent pas du contexte
  • • Pour les états très complexes et fréquents → envisagez Redux ou Zustand

⭐ Règle d'or : Un contexte = une responsabilité unique !
