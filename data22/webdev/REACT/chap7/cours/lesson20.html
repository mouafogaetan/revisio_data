💡 Context = téléportation de données à travers l'arborescence !

🕳️ Le problème : Props Drilling

Quand un composant profond a besoin d'une donnée, on doit la passer manuellement à travers tous les composants parents intermédiaires.

<App user={user}>
  <Layout user={user}>
    <Header user={user}>
      <Avatar user={user} />  ← besoin de user
    </Header>
  </Layout>
</App>

Inconvénients :

  • Code redondant et difficile à maintenir
  • Les composants intermédiaires reçoivent des props qui ne les concernent pas
  • Risque d'erreurs lors du renommage ou du déplacement

✨ Context API : la solution

Le contexte permet de rendre une donnée accessible à tous les composants descendants sans la passer explicitement à chaque niveau.

<UserProvider value={user}>
  <App />                 ← tout sous-arbre a accès à user
</UserProvider>
1. createContext()
2. Provider + value
3. useContext()

📦 createContext()

On crée un objet contexte dans un fichier dédié (ex: UserContext.jsx).

// UserContext.jsx
import { createContext } from 'react';

// Valeur par défaut (utilisée si pas de Provider)
export const UserContext = createContext(null);

La valeur par défaut sert de "fallback" quand aucun Provider n'est trouvé au-dessus.

🎁 Fournir la valeur : Provider

Tout composant qui a besoin de partager une donnée doit envelopper ses enfants avec <Context.Provider>.

import { UserContext } from './UserContext';

function App() {
  const user = { name: 'Alice', email: 'alice@example.com' };
  return (
    <UserContext.Provider value={user}>
      <Profile />
      <Dashboard />
    </UserContext.Provider>
  );
}

Tous les composants dans Profile et Dashboard pourront accéder à user.

🥤 Consommer : useContext

N'importe quel composant descendant peut lire la valeur avec le hook useContext.

import { useContext } from 'react';
import { UserContext } from './UserContext';

function Avatar() {
  const user = useContext(UserContext);
  return <img src={user.avatar} alt={user.name} />;
}

✅ Plus besoin de passer user via les props de composants intermédiaires !

🎨 Exemple pratique : Thème (Dark/Light)

// ThemeContext.jsx
import { createContext, useState } from 'react';
export const ThemeContext = createContext();

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// Header.jsx
import { useContext } from 'react';
import { ThemeContext } from './ThemeContext';

function Header() {
  const { theme, setTheme } = useContext(ThemeContext);
  return (
    <header className={theme}>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        Changer thème
      </button>
    </header>
  );
}

🔁 Combiner plusieurs contextes

On peut avoir autant de contextes que nécessaire : auth, thème, langue, panier…

function Root() {
  return (
    <AuthProvider>
      <ThemeProvider>
        <CartProvider>
          <App />
        </CartProvider>
      </ThemeProvider>
    </AuthProvider>
  );
}

Dans un composant :

const { user } = useContext(AuthContext);
const { theme } = useContext(ThemeContext);

⚠️ Pièges fréquents

  • Oublier le Provider : useContext renvoie la valeur par défaut (souvent null ou undefined).
  • Provider sans valeur : ne pas mettre d'attribut value.
  • Remplacer l'objet à chaque rendu : si value est un objet créé directement, tous les consommateurs se re-renderont à chaque rendu du Provider. Solution : utiliser useMemo ou useState.
❌ Mauvais : <Provider value={{ user, setUser }}> (créé à chaque rendu)
✅ Bon : const value = useMemo(() => ({ user, setUser }), [user]);

⚖️ useContext vs Redux

Context API

✅ Natif, simple
✅ Léger
❌ Pas de devtools avancés
❌ Peut causer des re-rendus excessifs

Redux

✅ DevTools puissants
✅ Middleware (thunk, saga)
❌ Plus lourd
❌ Courbe d'apprentissage

Astuce : Context + useReducer remplace Redux pour des applications petites/moyennes.

🧪 Mini exercice

Créez un contexte de langue (fr/en). L'application doit :

  • Avoir un bouton pour basculer entre français et anglais
  • Afficher un message de bienvenue dans la langue choisie (ex: "Bonjour" / "Hello")
  • Le bouton et le message se trouvent dans des composants différents
🔎 Voir solution 
// LanguageContext.jsx
import { createContext, useState } from 'react';
export const LanguageContext = createContext();

export function LanguageProvider({ children }) {
  const [lang, setLang] = useState('fr');
  return (
    <LanguageContext.Provider value={{ lang, setLang }}>
      {children}
    </LanguageContext.Provider>
  );
}

// ToggleButton.jsx
function ToggleButton() {
  const { lang, setLang } = useContext(LanguageContext);
  return <button onClick={() => setLang(lang === 'fr' ? 'en' : 'fr')}>
    {lang === 'fr' ? '🇫🇷 Français' : '🇬🇧 English'}
  </button>;
}

// Greeting.jsx
function Greeting() {
  const { lang } = useContext(LanguageContext);
  return <h1>{lang === 'fr' ? 'Bonjour !' : 'Hello !'}</h1>;
}
