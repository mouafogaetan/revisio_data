Objectifs de la leçon

🎣 Hooks personnalisés
Créer des fonctions qui utilisent des hooks React pour réutiliser la logique.
📡 useFetch
Construire un hook générique pour récupérer des données d'API.
📜 Règles des hooks
Les appeler uniquement au niveau racine et uniquement dans des composants/hooks.
♻️ Réutilisabilité
Partager la logique d'état et d'effets entre plusieurs composants.
💡 Un hook personnalisé commence toujours par use (ex: useFetch, useLocalStorage, useToggle).
🤔

Pourquoi créer des hooks personnalisés ?

Ils permettent d'extraire et de réutiliser la logique avec état entre plusieurs composants.

❌ Sans hook personnalisé

Logique fetch répétée dans chaque composant. Code dupliqué, difficile à maintenir.

✅ Avec useFetch

Une seule définition, utilisable partout. Facile à tester et à modifier.

// Sans hook : dupliqué dans Users.js et Products.js
function Users() {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(true);
  useEffect(() => {
    fetch('/api/users')
      .then(res => res.json())
      .then(data => { setData(data); setLoading(false); });
  }, []);
  // ...
}

// Avec useFetch : une seule fois !
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  useEffect(() => {
    fetch(url)
      .then(res => res.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);
  return { data, loading, error };
}

📡 Création du hook useFetch

// hooks/useFetch.js
import { useState, useEffect } from 'react';

function useFetch(url, options = {}) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const abortController = new AbortController();
    
    fetch(url, { ...options, signal: abortController.signal })
      .then(res => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then(setData)
      .catch(err => {
        if (err.name !== 'AbortError') setError(err);
      })
      .finally(() => setLoading(false));

    return () => abortController.abort(); // nettoyage
  }, [url]);

  return { data, loading, error };
}

export default useFetch;
✨ Bonus : Ajout d'un AbortController pour annuler la requête si le composant est démonté.

🎮 Utilisation dans un composant

function UsersList() {
  const { data: users, loading, error } = useFetch('https://api.example.com/users');

  if (loading) return <div>Chargement...</div>;
  if (error) return <div>Erreur : {error.message}</div>;
  
  return (
    <ul>
      {users?.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

function ProductsList() {
  const { data: products, loading, error } = useFetch('https://api.example.com/products');
  // Même logique, URL différente !
}

📜 Les règles des hooks (indispensables !)

Règle n°1 : Appeler les hooks uniquement au niveau racine

Pas dans des conditions, des boucles ou des fonctions imbriquées. React a besoin que l'ordre des hooks soit identique à chaque rendu.

// ❌ MAUVAIS
if (condition) { useState(false); }

// ✅ BON
const [state, setState] = useState(false);
if (condition) { /* utiliser state */ }

Règle n°2 : Appeler les hooks uniquement depuis React

Depuis un composant fonctionnel OU depuis un autre hook personnalisé. Jamais depuis une fonction JavaScript classique.

// ❌ MAUVAIS
function normalFunction() { useState(); }

// ✅ BON
function useMonHook() { useState(); } // hook personnalisé
⚠️ Conséquence : Si vous violez ces règles, React ne peut pas garantir le bon fonctionnement des hooks (état mélangé, bugs subtils).

🔍 Pourquoi l'ordre des hooks est important ?

React identifie chaque hook par sa position dans le composant.

// Premier rendu
useState('name')    → Hook n°1 : name
useEffect(...)      → Hook n°2 : effet
useState(false)     → Hook n°3 : isActive

// Deuxième rendu (même ordre)
useState('name')    → Hook n°1 : récupère 'name'
useEffect(...)      → Hook n°2 : récupère l'effet
useState(false)     → Hook n°3 : récupère isActive

// SI CONDITION : l'ordre change → 💥 ERREUR !
💡 Bon réflexe : Mettez toujours vos hooks en haut du composant, avant les retours conditionnels.

✨ Autres exemples de hooks personnalisés

useLocalStorage

function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  return [value, setValue];
}

useToggle

function useToggle(initial = false) {
  const [state, setState] = useState(initial);
  const toggle = () => setState(s => !s);
  return [state, toggle];
}
📦 Les hooks personnalisés sont le meilleur moyen de partager la logique d'état dans React.
