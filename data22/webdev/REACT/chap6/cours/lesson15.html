Cycle de vie & useEffect

Les hooks qui donnent vie à vos composants

Montage • Mise à jour • Dé montage • Effets secondaires

Niveau: Intermédiaire Durée: 55 min

🎯 Objectifs de la leçon

  • Comprendre les 3 phases du cycle de vie d'un composant React (montage, mise à jour, démontage)
  • Utiliser useEffect pour exécuter du code après le rendu
  • Maîtriser le tableau de dépendances pour contrôler quand l'effet s'exécute
  • Nettoyer les effets secondaires avec la fonction de nettoyage (cleanup)
À la fin, vous saurez gérer les appels API, les timers, les abonnements et bien plus !

🌱 Le cycle de vie en 3 phases

Un composant React passe par trois grandes phases :

Montage : Création, insertion dans le DOM
Mise à jour : Changement de props ou d'état
Démontage : Retrait du DOM

⚡ useEffect : Le couteau suisse des effets

useEffect est un Hook qui permet d'exécuter du code après le rendu du composant.

import { useEffect } from 'react';

function MonComposant() {
  useEffect(() => {
    // Code exécuté APRÈS le rendu
    console.log('Effet exécuté !');
  });

  return <div>Mon composant</div>;
}

Les effets secondaires sont des opérations qui interagissent avec le monde extérieur : appel API, manipulation directe du DOM, timers, abonnements...

🎛️ Le tableau de dépendances

Le second argument de useEffect contrôle quand l'effet se ré-exécute.

Cas 1 : Pas de tableau → Exécution à chaque rendu

useEffect(() => { /* exécuté à chaque rendu */ });

Cas 2 : Tableau vide [] → Exécution UNE SEULE fois (au montage)

useEffect(() => { /* exécuté au montage uniquement */ }, []);

Cas 3 : Dépendances → Exécution quand les valeurs changent

useEffect(() => { /* exécuté quand count change */ }, [count]);

🎬 Exécuter du code au montage

Le tableau de dépendances vide [] signifie que l'effet s'exécute une seule fois après le premier rendu.

import { useEffect, useState } from 'react';

function UserProfile() {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Charger les données utilisateur au montage du composant
    fetch('https://api.example.com/user/1')
      .then(res => res.json())
      .then(data => {
        setUser(data);
        setLoading(false);
      });
  }, []); // ← Tableau vide : une seule fois !

  if (loading) return <div>Chargement...</div>;
  return <div>Bonjour {user.name}</div>;
}
Idéal pour les appels API, l'initialisation de bibliothèques, etc.

🔄 Réagir aux changements de props ou d'état

Ajoutez des dépendances pour exécuter l'effet quand elles changent.

function ProductList({ category }) {
  const [products, setProducts] = useState([]);

  useEffect(() => {
    // Recharger les produits quand la catégorie change
    fetch(`/api/products?category=${category}`)
      .then(res => res.json())
      .then(setProducts);
  }, [category]); // ← S'exécute à chaque changement de category

  return <div>{products.map(p => <Product key={p.id} {...p} />)}</div>;
}
Attention : assurez-vous d'inclure TOUTES les valeurs utilisées dans l'effet !

🧹 Nettoyer avec la fonction de cleanup

La fonction retournée dans useEffect s'exécute au démontage OU avant le prochain effet.

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);

    // Fonction de nettoyage (cleanup)
    return () => {
      clearInterval(interval); // Supprime l'intervalle au démontage
      console.log('Timer nettoyé !');
    };
  }, []); // Tableau vide = cleanup au démontage uniquement

  return <div>Temps : {seconds}s</div>;
}
Nettoyage obligatoire pour les timers, abonnements, event listeners, requêtes en cours...

📡 Exemple : Abonnement WebSocket

function LiveChat() {
  const [messages, setMessages] = useState([]);

  useEffect(() => {
    const socket = new WebSocket('wss://chat.example.com');

    socket.onmessage = (event) => {
      setMessages(prev => [...prev, event.data]);
    };

    // Nettoyer : fermer la connexion au démontage
    return () => {
      socket.close();
      console.log('WebSocket fermé');
    };
  }, []);

  return <div>{messages.length} messages reçus</div>;
}

Sans cleanup, la connexion resterait ouverte même si le composant n'est plus affiché, causant des fuites mémoire !

Tous combinésUn seul useEffect peut gérer montage, mise à jour ET cleanup !

Les hooks simplifient le code en regroupant la logique liée au lieu de la disperser dans plusieurs méthodes.

📜 Règles importantes des hooks

  • Appeler les hooks au niveau racine : pas dans des conditions, boucles ou fonctions imbriquées
  • Appeler les hooks uniquement dans des composants fonctionnels ou des hooks personnalisés
  • Toujours inclure toutes les dépendances utilisées dans l'effet (ou utiliser ESLint avec eslint-plugin-react-hooks)
  • Ne pas appeler hooks après un return conditionnel

❌ Mauvais : if (condition) { useEffect(...) }

✅ Bon : Toujours appeler useEffect au même endroit, utiliser condition à l'intérieur si besoin

📚 Séparer les préoccupations

On peut avoir plusieurs useEffect dans un même composant. Chacun gère une responsabilité distincte.

function Dashboard({ userId }) {
  // Effet 1 : Charger le profil utilisateur
  useEffect(() => {
    fetchUser(userId).then(setUser);
  }, [userId]);

  // Effet 2 : S'abonner aux notifications
  useEffect(() => {
    const subscription = subscribeToNotifications(userId);
    return () => subscription.unsubscribe();
  }, [userId]);

  // Effet 3 : Mettre à jour le titre de la page
  useEffect(() => {
    document.title = `Tableau de bord - ${userId}`;
  }, [userId]);

  return <div>...</div>;
}

✅ Meilleure lisibilité et maintenance : chaque effet a un objectif unique !

📌 Récapitulatif

Bonnes pratiques

  • Toujours nettoyer les effets (timers, abonnements)
  • Utiliser le tableau de dépendances correctement
  • Séparer les effets par responsabilité
  • Déplacer la logique complexe dans des hooks personnalisés

Pièges à éviter

  • Oublier les dépendances → comportement imprévisible
  • Mettre [] pour des effets qui dépendent de props → stale data
  • Modifier l'état dans un effet sans condition → boucle infinie

Règle d'or : useEffect = synchroniser votre composant avec le monde extérieur !

🧪 Exercice : Horloge en temps réel

Créez un composant Clock qui affiche l'heure actuelle et se met à jour chaque seconde.

Consignes :

  1. Utiliser useState pour stocker l'heure
  2. Utiliser useEffect pour créer un intervalle setInterval au montage
  3. Nettoyer l'intervalle au démontage
  4. Optionnel : ajouter un bouton pour cacher/montrer l'horloge et observer le nettoyage
function Clock() {
  const [time, setTime] = useState(new Date());

  useEffect(() => {
    const interval = setInterval(() => {
      setTime(new Date());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return <h2>Il est {time.toLocaleTimeString()}</h2>;
}
