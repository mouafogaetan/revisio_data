Propriétés et encapsulation

Créez des classes robustes, protégez vos données et maîtrisez l’art des propriétés automatiques et personnalisées en C#.

⬅️ Utilisez les boutons ci-dessous ➡️

🎯 Objectifs de la leçon

  • ✓ Créer des propriétés automatiques (auto-implemented) pour simplifier le code.
  • ✓ Utiliser les accesseurs get et set avec logique personnalisée (validation, calcul).
  • ✓ Comprendre les modificateurs d’accès : public, private, protected et le principe d’encapsulation.
  • ✓ Différencier champ (field) et propriété, et savoir quand utiliser l’un ou l’autre.
💡 Une propriété bien conçue protège l’état interne de l’objet tout en offrant une interface claire.

🔒 Pourquoi l'encapsulation ?

En programmation orientée objet, on cache les détails internes d’une classe. Les champs (private par convention) ne sont pas accessibles directement depuis l’extérieur. On utilise des propriétés pour contrôler l’accès.

// Mauvaise pratique : champ public
public class Personne {
    public int age;  // danger : n'importe qui peut mettre -5
}

// Bonne pratique : champ privé + méthodes/propriétés
public class Personne {
    private int age;
    public int Age {
        get { return age; }
        set { if(value >= 0) age = value; }
    }
}
⬇️ Encapsulation = contrôle + sécurité ⬇️

⚡ Propriétés automatiques

Depuis C# 3.0, on peut déclarer une propriété sans champ explicite. Le compilateur crée un champ privé anonyme.

public class Produit
{
    // Propriété automatique
    public string Nom { get; set; }
    public decimal Prix { get; set; }
    
    // Propriété avec init only (C# 9+)
    public string Reference { get; init; }
}

// Utilisation
Produit p = new Produit();
p.Nom = "Ordinateur";
p.Prix = 1200.99m;
// p.Reference = "REF123"; // possible uniquement lors de l'initialisation si init

✅ Gain de temps, code plus lisible, encapsulation respectée.

🧠 Propriétés personnalisées (get / set logique)

Vous pouvez ajouter des validations, déclencher des événements ou transformer la valeur.

public class CompteBancaire
{
    private decimal solde;
    
    public decimal Solde
    {
        get { return solde; }
        private set { // accesseur privé
            if(value < 0) 
                throw new ArgumentException("Solde négatif interdit");
            solde = value;
        }
    }
    
    public string Titulaire { get; set; } // auto
    
    public void Crediter(decimal montant) {
        if(montant > 0) Solde += montant;
    }
}

📌 Bonnes pratiques : évitez les effets de bord lourds dans un get ; préférez des méthodes pour des opérations complexes.

🔐 Modificateurs d'accès & encapsulation

public — accessible partout
private — accessible uniquement dans la classe
protected — accessible dans la classe et les classes dérivées
internal — accessible dans le même assembly

✨ Principe fondamental :

Exposez le minimum nécessaire (privilégiez private pour les champs, public pour les propriétés / méthodes utiles). L'encapsulation protège l’intégrité.

📌 Champs readonly & bonnes pratiques

Un champ readonly ne peut être modifié qu’au moment de la déclaration ou dans le constructeur. Idéal pour les valeurs immutables.

public class Etudiant
{
    private readonly string matricule; // ne change jamais
    public string Nom { get; set; }
    
    public Etudiant(string matricule, string nom)
    {
        this.matricule = matricule;
        Nom = nom;
    }
    
    public string Matricule => matricule; // propriété lecture seule
}
  • ✔️ Préférez les propriétés publiques aux champs publics.
  • ✔️ Utilisez private set pour les propriétés modifiables uniquement par la classe.
  • ✔️ Pour l’immutabilité, utilisez init (C# 9+) ou readonly + getter sans set.

🧑‍🏫 Exemple complet : classe Personne

public class Personne
{
    private string nom;
    private int age;
    
    // Propriété nom avec validation
    public string Nom
    {
        get => nom;
        set => nom = string.IsNullOrWhiteSpace(value) ? "Anonyme" : value.Trim();
    }
    
    public int Age
    {
        get => age;
        set
        {
            if (value < 0 || value > 130)
                throw new ArgumentOutOfRangeException(nameof(value), "Âge invalide");
            age = value;
        }
    }
    
    // Propriété calculée (aucun champ de stockage)
    public string Presentation => $"Je m'appelle {Nom}, j'ai {Age} ans.";
    
    public Personne(string nom, int age)
    {
        Nom = nom;
        Age = age;
    }
}
✅ Résultat : les données sont toujours cohérentes, et l'utilisateur interagit simplement via les propriétés.

🎯 Exercice : Créez une classe Thermostat avec une propriété Temperature (double). La température ne doit jamais descendre en dessous de -10°C ni dépasser 45°C. Si la valeur est invalide, conservez l’ancienne valeur ou levez une exception. Écrivez la structure.

🛠️ Voir une solution possible 
public class Thermostat {
    private double temperature = 20;
    public double Temperature {
        get => temperature;
        set {
            if(value >= -10 && value <= 45)
                temperature = value;
            else
                throw new ArgumentOutOfRangeException("Température hors plage");
        }
    }
}
