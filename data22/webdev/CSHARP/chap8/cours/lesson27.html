📊 Filtrage ➜ Projection ➜ Agrégation : le triptyque LINQ

🔍 Filtrer : Where & OfType

LINQ to Objects

Where permet de sélectionner les éléments d'une collection qui satisfont une condition (expression lambda booléenne). OfType filtre selon un type précis.

Exemple simple : nombres pairs

int[] nombres = { 5, 12, 7, 18, 3, 20 };
var pairs = nombres.Where(n => n % 2 == 0);
// Résultat : 12, 18, 20

Avec des objets : étudiants majeurs

var majeurs = etudiants.Where(e => e.Age >= 18);
// OfType : récupère uniquement les strings d’une liste mixte
object[] data = { "Bonjour", 42, "C#", 3.14 };
var chaines = data.OfType<string>(); // "Bonjour", "C#"

✅ Bonnes pratiques : Where ne modifie pas la source, il retourne une nouvelle requête. L'exécution est différée (Deferred Execution).

📐 Projection : Select & SelectMany

Select transforme chaque élément en un nouveau formulaire (projection). SelectMany aplatit des collections imbriquées en une seule séquence.

✨ Select : calculer le carré d'une liste

int[] nombres = { 1, 2, 3, 4 };
var carres = nombres.Select(n => n * n);
// carres = [1, 4, 9, 16]

✨ Projeter sur un type anonyme

var nomsMaj = personnes.Select(p => new { 
    NomComplet = p.Prenom + " " + p.Nom,
    AnneeNaissance = DateTime.Now.Year - p.Age 
});

🌀 SelectMany : aplatir des listes de listes

List<string[]> phrases = new() {
    new[] { "Bonjour", "le", "monde" },
    new[] { "C#", "LINQ" }
};
var mots = phrases.SelectMany(p => p);
// Résultat : "Bonjour","le","monde","C#","LINQ"
📚 SelectMany transforme [ [a,b], [c] ] → [a,b,c]

📊 Agrégations standards

Ces méthodes exécutent un calcul sur une séquence et retournent une valeur unique. Très pratiques pour les statistiques.

Count() → nombre d'éléments 
int[] ages = { 18, 25, 30 };
int total = ages.Count(); // 3
Sum() → somme 
double prix = { 9.99, 4.5 };
double total = prix.Sum(); // 14.49
Average() → moyenne 
var notes = new List<int>{ 12, 15, 18 };
double moy = notes.Average(); // 15.0
Min() / Max() → min/max 
var vals = new[] { 3, 9, 1, 7 };
int min = vals.Min(); // 1
int max = vals.Max(); // 9

💡 Ces méthodes acceptent aussi une lambda pour projeter avant calcul : eleves.Average(e => e.Note).

⚙️ Aggrégation sur mesure : Aggregate

Aggregate applique cumulativement une fonction d'accumulation. Très flexible pour des calculs complexes (concaténation, produit, recherche personnalisée).

🔁 Exemple : concaténer des chaînes avec séparateur

string[] mots = { "C#", "LINQ", "est", "puissant" };
string phrase = mots.Aggregate((acc, mot) => acc + " " + mot);
// Résultat : "C# LINQ est puissant"

🧮 Calcul factoriel avec Aggregate

int[] facteurs = { 1, 2, 3, 4, 5 };
int factoriel5 = facteurs.Aggregate(1, (prod, val) => prod * val);
// 120 (1*2*3*4*5)
⚠️ Remarque : Aggregate nécessite que la collection ne soit pas vide (sinon exception). Une graine (seed) peut être fournie.

🚀 Exemple complet : combiner Where, Select, Sum

Imaginons une liste de commandes : nous voulons la somme des montants des commandes validées pour un client spécifique.

class Commande {
    public string Client { get; set; }
    public decimal Montant { get; set; }
    public bool EstValidee { get; set; }
}

List<Commande> commandes = new() {
    new Commande { Client="Alice", Montant=150, EstValidee=true },
    new Commande { Client="Bob", Montant=80, EstValidee=false },
    new Commande { Client="Alice", Montant=200, EstValidee=true },
};

decimal totalAlice = commandes
    .Where(c => c.Client == "Alice" && c.EstValidee)
    .Select(c => c.Montant)
    .Sum();

// totalAlice = 350
✔ Résultat : 350€ – seulement les commandes validées d'Alice.

On peut aussi utiliser Average sur le montant, Count sur les commandes validées, etc.

Filtre ➜ Projection ➜ Agrégation = Requête déclarative efficace

📌 Récapitulatif & astuces

• Where → pour filtrer selon condition booléenne.
• Select → transformer / projeter des données.
• SelectMany → aplatir des collections imbriquées.
• Agrégations (Sum, Average, Count, Min, Max, Aggregate) → résumer des données.

💡 Exécution différée :

La plupart des opérateurs LINQ ne sont exécutés que lorsqu'on itère sur le résultat (ou appel à ToList(), Count(), etc.). Cela permet d'optimiser les performances.
