1. Commenter son code

Les commentaires sont ignorés par le compilateur, mais très utiles pour expliquer votre logique ou pour temporairement désactiver du code.

// Commentaire sur une ligne
// Ceci est un commentaire
int age = 25; // après du code
/* Commentaire multiligne */
/* Ceci est
   un commentaire
   sur plusieurs lignes */
/// Commentaire XML
/// <summary>
/// Additionne deux nombres
/// </summary>

Utilisé pour la documentation automatique.

🧑‍🏫 Bon à savoir : les commentaires aident votre futur vous-même et vos camarades. Un code bien commenté est plus facile à maintenir !

🖥️

2. Afficher et lire des données

La console est une fenêtre texte. Avec C#, nous utilisons la classe Console pour dialoguer avec l'utilisateur.

🔹 Console.WriteLine()

Affiche une ligne de texte et passe à la ligne suivante.

Console.WriteLine("Bonjour le Cameroun !");
🔹 Console.Write()

Affiche sans retour à la ligne.

Console.Write("Entrez votre nom : ");
🔹 Console.Read() / Console.ReadLine()

Lit une touche ou une ligne saisie par l'utilisateur.

string reponse = Console.ReadLine();
Console.WriteLine("Vous avez dit : " + reponse);
🔤

3. Casse et points-virgules

⚠️ C# est sensible à la casse (case-sensitive)

Console.WriteLine() est différent de console.writeline(). Les noms de variables maVariable et mavariable sont deux entités distinctes.

🔸 Le point-virgule ; termine chaque instruction.

int a = 10;   // ✅ Correct
string nom = "Joël"   // ❌ Erreur : point-virgule manquant
Console.WriteLine(nom); // ✅

Sans point-virgule, le compilateur ne comprend pas où s'arrête l'instruction. Pensez-y comme un point dans une phrase !

📌 Exemple complet :

// Programme simple
string prenom = "Alain";
Console.Write("Bonjour ");
Console.WriteLine(prenom);   // Affiche "Bonjour Alain"
Console.Read();              // Attend une touche
