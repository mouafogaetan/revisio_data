🔄 La boucle for

Idéale quand on connaît le nombre d'itérations à l'avance.

// Syntaxe générale
for (initialisation; condition; incrément)
{
    // instructions à répéter
}

// Exemple : afficher les nombres de 1 à 5
for (int i = 1; i <= 5; i++)
{
    Console.WriteLine("Valeur : " + i);
}

📌 Décomposition :

  • initialisation : int i = 1 (déclare et initialise un compteur)
  • condition : i <= 5 (tant que vrai, la boucle continue)
  • incrément : i++ (augmente la variable à chaque tour)
// Boucle décroissante
for (int i = 10; i > 0; i--)
{
    Console.Write(i + " "); // 10 9 8 ... 1
}

📦 La boucle foreach

Parcourt automatiquement chaque élément d'une collection (tableau, liste, etc.). Simple et lisible.

int[] notes = { 12, 15, 18, 10, 14 };
foreach (int note in notes)
{
    Console.WriteLine($"Note obtenue : {note}");
}

// Avec une List<string>
List<string> noms = new List<string> { "Alice", "Bob", "Clara" };
foreach (var nom in noms)
{
    Console.WriteLine("Bonjour " + nom);
}
✅ Avantages
Pas de gestion d'index, code épuré, impossible de dépasser la taille.
⚠️ Limitation
On ne peut pas modifier la collection pendant l'itération.

Tableau : [12, 15, 18, 10, 14]

12→ 15→ 18→ 10→ 14
foreach parcourt chaque case automatiquement

⚙️ while et do-while

while

Vérifie la condition avant d'exécuter. Peut ne s'exécuter jamais.

int compteur = 1;
while (compteur <= 3)
{
    Console.WriteLine("Tour " + compteur);
    compteur++;
}
// Output : Tour 1, Tour 2, Tour 3
⚠️ Attention à la condition infinie !

do-while

Exécute au moins une fois, puis teste la condition.

int saisie;
do {
    Console.Write("Entrez un nombre >0 : ");
    saisie = int.Parse(Console.ReadLine());
} while (saisie <= 0);
Console.WriteLine("Merci !");
✅ Parfait pour les menus ou validation de saisie.

📌 Quand utiliser quelle boucle ?

BoucleMeilleur usage
forNombre d'itérations connu (tableau classique, compteur)
foreachParcourir intégralité d'une collection sans modifier l'index
whileBoucler tant qu'une condition est vraie (lecture fichier, attente utilisateur)
do-whileAu moins une exécution nécessaire (saisie contrôlée)

🧪 Petit test interactif (jQuery) :

Laquelle de ces boucles exécute le code au moins une fois avant de tester la condition ?

💪 À retenir : Les boucles évitent la répétition de code. Choisissez la plus adaptée pour un code lisible et efficace.
