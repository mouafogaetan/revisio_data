🧬 Qu'est-ce que l'héritage ?

L'héritage est un principe fondamental de la programmation orientée objet. Il permet de créer une nouvelle classe (classe dérivée / enfant) à partir d'une classe existante (classe de base / parent).

✨ Exemple concret :

Une classe Véhicule peut être parent de Voiture et Moto. La voiture hérite des propriétés (couleur, vitesse) et des méthodes (démarrer, freiner).

// Syntaxe C#
class Parent { }
class Enfant : Parent { }
👉 En C#, une classe ne peut hériter que d'une seule classe de base (héritage simple).

🔗 Hériter avec : et appeler base

Pour hériter, on utilise le symbole deux-points :. Le constructeur de la classe enfant peut appeler explicitement le constructeur du parent avec le mot-clé base.

public class Animal
{
    public string Nom { get; set; }
    public Animal(string nom)  // constructeur parent
    {
        Nom = nom;
    }
}

public class Chien : Animal
{
    public string Race { get; set; }
    // appel du constructeur parent avec base(nom)
    public Chien(string nom, string race) : base(nom)
    {
        Race = race;
    }
}

✔️ base(nom) assure que le constructeur du parent est exécuté avant celui de l'enfant.

⛓️ Chaînage des constructeurs

Le "chaînage" signifie que lorsqu'on instancie un objet enfant, les constructeurs sont appelés dans l'ordre : d'abord le constructeur de la classe de base, puis celui de la classe dérivée.

⚠️ Si le parent n'a pas de constructeur sans paramètre, l'enfant doit utiliser base(...) explicitement.

Schéma d'exécution :

💻 Exemple concret : Personne / Étudiant

// Classe parent
public class Personne
{
    public string Nom { get; set; }
    public int Age { get; set; }
    
    public Personne(string nom, int age)
    {
        Nom = nom;
        Age = age;
        Console.WriteLine($"Constructeur Personne: {nom}");
    }
}

// Classe enfant
public class Etudiant : Personne
{
    public string Matricule { get; set; }
    
    public Etudiant(string nom, int age, string matricule) 
        : base(nom, age)   // appel au constructeur parent
    {
        Matricule = matricule;
        Console.WriteLine($"Constructeur Etudiant: {matricule}");
    }
}

// Utilisation
Etudiant eleve = new Etudiant("Mbiada", 20, "E001");
// Affichera :
// Constructeur Personne: Mbiada
// Constructeur Etudiant: E001
🔍 Remarquez l'ordre d'affichage : Parent d'abord, Enfant ensuite.

⚠️ Pièges et bonnes pratiques

  • Si la classe parent n’a pas de constructeur par défaut (sans paramètre), l’enfant doit appeler base(...) explicitement.
  • Le mot-clé base peut aussi servir à appeler une méthode du parent (ex: base.Afficher()).
  • On ne peut pas hériter d'une classe sealed.
  • L'héritage représente une relation "est un" : Un Étudiant est une Personne. Évitez l'héritage abusif.
🌟 À retenir : L’héritage favorise la réutilisation du code et la hiérarchisation des concepts.

📝 Vérifions votre compréhension

Que fait le mot-clé base dans un constructeur enfant ?

💡 Réponse: Le mot-clé base permet d’appeler le constructeur du parent, essentiel pour initialiser les membres hérités.
