🏗️ Concevoir l’architecture d’une app e-commerce mobile

Définir les écrans, maquettes mobiles et spécificités tactiles

📲 Écrans mobiles · Maquettes Figma · Gestes & navigation

🎯 Objectifs de la leçon

  • Définir les écrans essentiels d’une application mobile e-commerce (home, catalogue, produit, panier, compte).
  • Concevoir des maquettes adaptées au mobile avec Figma (taille d’écran, composants tactiles).
  • Prendre en compte les spécificités mobiles : gestes (swipe, tap long), navigation par onglets (bottom tabs).
  • Identifier les bonnes pratiques d’architecture pour React Native (navigation, gestion d’état, adaptabilité).
💡 Résultat : savoir modéliser et structurer une app mobile performante, évolutive et centrée utilisateur.

📱 Écrans clés & expérience mobile

User flow prioritaire

Pour une app e-commerce multiplateforme, on retrouve ces écrans incontournables :

🏠 Accueil
bannière, catégories
📋 Catalogue
grille produits, filtres
🔍 Produit
détail, variations, avis
🛒 Panier
récap, modification quantité
📦 Checkout
adresse, paiement
👤 Compte
commandes, adresses, favoris

✨ La navigation se fait souvent via bottom tabs (onglets en bas) adaptés aux pouces.

🎨 Design mobile-first

Créer des maquettes avec Figma

Pour adapter l’interface au mobile, on travaille avec des frames de taille 375x667 (iPhone SE) ou 390x844. Les points clés :

  • Espacer les zones cliquables (minimum 44x44 pixels).
  • Prévoir des composants tactiles : swipe pour supprimer, tiroir de filtres (BottomSheet).
  • Utiliser des variants pour les cartes produits, boutons, etc.

📐 Frame Figma recommandée : "iPhone 13/15" ou "Pixel 6" pour tester le responsive.

🎯 Avantage : le passage au code est fluide grâce à des design tokens (couleurs, typo).

🖐️ Gestes & interactions mobiles

React Native permet d’intégrer des gestes tactiles grâce à la bibliothèque react-native-gesture-handler. Exemples utiles :

✔️ Swipe horizontal : carrousel de produits vedettes.
✔️ Swipe to delete : suppression d’un article dans le panier.
✔️ Long press : ajouter aux favoris ou partager.
✔️ Pinch to zoom : agrandir les photos produit.
// Exemple simple de détection de swipe (GestureDetector) import { GestureDetector, Gesture } from 'react-native-gesture-handler'; const pan = Gesture.Pan().onEnd(() => console.log('swipe détecté'));

🧭 Navigation adaptée mobile

React Navigation

Une structure classique combine Bottom Tab Navigator (Accueil, Catalogue, Panier, Profil) et des Stack Navigators imbriqués.

Chaque onglet possède sa propre pile de navigation (ex: dans "Accueil", on peut cliquer sur un produit pour ouvrir "Détail produit" puis "Avis").

📏 Responsive & accessibilité mobile

  • Utiliser Dimensions ou useWindowDimensions pour adapter la mise en page.
  • Privilégier les unités relatives (%, flex) plutôt que les pixels fixes.
  • Prévoir un espacement pour les zones d’affichage sécurisées (SafeAreaView).
  • Adapter les polices selon la taille d’écran (Dynamic Type sur iOS).
📱 Tester systématiquement sur simulateur iOS + Android et sur vrai appareil pour vérifier les gestes.
