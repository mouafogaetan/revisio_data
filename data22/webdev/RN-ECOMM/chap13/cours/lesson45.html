💡 Pourquoi tester ?
Une application e-commerce complexe a besoin de tests pour éviter les régressions lors des mises à jour. Les tests unitaires permettent de valider chaque brique indépendamment.

📦 Installation de Jest pour React Native

Pour un projet Expo, installez les dépendances :

npm install --save-dev jest @testing-library/react-native \
  @types/jest jest-expo babel-jest

# Pour un projet React Native CLI
npm install --save-dev jest @testing-library/react-native \
  @types/jest @babel/preset-env babel-jest metro-react-native-babel-preset

Configuration jest.config.js :

module.exports = {
  preset: 'jest-expo',
  transformIgnorePatterns: [
    'node_modules/(?!((jest-)?react-native|@react-native(-community)?)|expo(nent)?|@expo(nent)?/.*|@expo-google-fonts/.*|react-navigation|@react-navigation/.*|@unimodules/.*|unimodules|sentry-expo|native-base|react-native-svg)'
  ],
  setupFilesAfterEnv: ['@testing-library/jest-native/extend-expect']
};

🧩 Tester un composant ProductCard

Composant à tester :

// ProductCard.tsx
import { View, Text, Image, Pressable } from 'react-native';

export const ProductCard = ({ name, price, image, onPress }) => (
  <Pressable onPress={onPress} testID="product-card">
    <Image source={{ uri: image }} testID="product-image" />
    <Text testID="product-name">{name}</Text>
    <Text testID="product-price">{price} FCFA</Text>
  </Pressable>
);

Test avec React Native Testing Library :

import { render, fireEvent } from '@testing-library/react-native';
import { ProductCard } from './ProductCard';

test('affiche les informations produit', () => {
  const { getByTestId } = render(
    <ProductCard name="T-shirt" price="5000" image="url" onPress={() => {}} />
  );
  expect(getByTestId('product-name')).toHaveTextContent('T-shirt');
  expect(getByTestId('product-price')).toHaveTextContent('5000 FCFA');
});

🏪 Tester le panier (cartStore) avec Zustand

// cartStore.ts
import { create } from 'zustand';

interface CartItem { id: number; name: string; quantity: number; price: number; }

interface CartStore {
  items: CartItem[];
  addItem: (item: CartItem) => void;
  removeItem: (id: number) => void;
  getTotal: () => number;
}

export const useCartStore = create<CartStore>((set, get) => ({
  items: [],
  addItem: (item) => set((state) => ({ items: [...state.items, item] })),
  removeItem: (id) => set((state) => ({ items: state.items.filter(i => i.id !== id) })),
  getTotal: () => get().items.reduce((sum, i) => sum + i.price * i.quantity, 0),
}));
// cartStore.test.ts
import { useCartStore } from './cartStore';

describe('cartStore', () => {
  beforeEach(() => {
    useCartStore.setState({ items: [] }); // Reset avant chaque test
  });

  test('ajoute un produit au panier', () => {
    const { addItem, items } = useCartStore.getState();
    addItem({ id: 1, name: 'Produit A', quantity: 1, price: 1000 });
    
    expect(useCartStore.getState().items).toHaveLength(1);
    expect(useCartStore.getState().items[0].name).toBe('Produit A');
  });

  test('calcule le total correctement', () => {
    const { addItem, getTotal } = useCartStore.getState();
    addItem({ id: 1, name: 'A', quantity: 2, price: 500 });
    addItem({ id: 2, name: 'B', quantity: 1, price: 1000 });
    
    expect(getTotal()).toBe(2000); // 2*500 + 1*1000 = 2000
  });
});

🎭 Mocker les dépendances (API, navigation)

Simuler un appel API avec Jest :

// api/products.ts
export const fetchProducts = () => fetch('/api/products').then(res => res.json());

// Test
jest.mock('../api/products');
import { fetchProducts } from '../api/products';

test('récupère la liste des produits', async () => {
  const mockProducts = [{ id: 1, name: 'Test' }];
  (fetchProducts as jest.Mock).mockResolvedValue(mockProducts);
  
  const result = await fetchProducts();
  expect(result).toEqual(mockProducts);
});

Mocker la navigation React Navigation :

const mockNavigate = jest.fn();
jest.mock('@react-navigation/native', () => ({
  useNavigation: () => ({ navigate: mockNavigate })
}));

📝 Tester un formulaire de connexion

import { render, fireEvent, waitFor } from '@testing-library/react-native';
import { LoginForm } from './LoginForm';

test('affiche une erreur si les champs sont vides', async () => {
  const { getByPlaceholderText, getByText } = render(<LoginForm />);
  
  fireEvent.press(getByText('Se connecter'));
  
  await waitFor(() => {
    expect(getByText('Email requis')).toBeTruthy();
    expect(getByText('Mot de passe requis')).toBeTruthy();
  });
});

test('appelle la fonction de connexion avec les bons identifiants', async () => {
  const mockLogin = jest.fn();
  const { getByPlaceholderText, getByText } = render(<LoginForm onLogin={mockLogin} />);
  
  fireEvent.changeText(getByPlaceholderText('Email'), 'test@email.com');
  fireEvent.changeText(getByPlaceholderText('Mot de passe'), 'password123');
  fireEvent.press(getByText('Se connecter'));
  
  await waitFor(() => {
    expect(mockLogin).toHaveBeenCalledWith({
      email: 'test@email.com',
      password: 'password123'
    });
  });
});

📝 Bonnes pratiques de test

✅ À faire

  • Tester le comportement, pas l'implémentation
  • Utiliser getBy* pour les éléments attendus
  • Nettoyer l'état entre les tests (beforeEach)
  • Tester les cas limites (valeurs vides, erreurs)
  • Isoler les tests (pas d'ordre d'exécution)

❌ À éviter

  • Tester des détails d'implémentation (props internes)
  • Utiliser des timeouts arbitraires
  • Mocker trop de dépendances
  • Tests qui dépendent de données réelles

🎯 Règle d'or : Un test unitaire doit être rapide (quelques millisecondes), isolé et reproductible.
