🔬 Analyser le bundle avec Expo

Avant d’optimiser, il faut mesurer. Expo propose plusieurs outils pour visualiser la taille du bundle JavaScript et des assets.


# Installer expo-optimize (optimisation images)
npm install -g sharp-cli
npx expo-optimize

# Afficher le rapport du bundle
npx expo export --dump-assetmap
# ou utiliser "expo customize" + "metro-bundler" 
# Extension recommandée : "expo-bundle-analyzer"

💡 Bon à savoir :

Le bundle principal contient tout votre code JS. Un gros bundle ralentit le Time To Interactive (TTI). Sur mobile, l'objectif est de rester sous 2-3 Mo.

⏳ Lazy loading : charger les écrans à la demande

Au lieu d’importer tous les écrans au démarrage, on les charge uniquement quand l’utilisateur navigue vers eux. React Native supporte React.lazy avec Suspense depuis React 18 (ou via @react-navigation).


// Avant (chargement synchrone)
import ProfilScreen from './screens/ProfilScreen';
import CartScreen from './screens/CartScreen';

// Après (lazy loading)
const ProfilScreen = React.lazy(() => import('./screens/ProfilScreen'));
const CartScreen = React.lazy(() => import('./screens/CartScreen'));

// Dans le composant de navigation
<Suspense fallback={<LoadingSpinner />}>
  <Stack.Screen name="Profil" component={ProfilScreen} />
  <Stack.Screen name="Panier" component={CartScreen} />
</Suspense>

⚠️ Attention : Le lazy loading fonctionne parfaitement avec Metro et Webpack. Sur React Navigation, on préfère aussi getComponent pour le chargement différé.

🌳 Tree shaking : élaguer le code inutilisé

Le "tree shaking" est une technique de bundler (Metro, Webpack) qui supprime les fonctions/exportations non utilisées. Il fonctionne uniquement avec la syntaxe ES modules (import/export).

❌ Mauvaise pratique (CommonJS)

const lodash = require('lodash');

→ Importe toute la bibliothèque

✅ Tree shaking possible

import { debounce } from 'lodash-es';

→ Seulement la fonction utilisée

📦 Conseils pour React Native :

  • Utiliser des bibliothèques compatibles ES modules (ex: lodash-es, date-fns).
  • Vérifier la configuration Metro : transformer.minifierConfig.compress.drop_dead_code.
  • Éviter les imports sauvages de gros paquets (comme `moment` entier → préférer `dayjs`).

📥 Import dynamique : code splitting à la volée

L'import dynamique permet de ne charger un module lourd que lorsqu’une action se produit (clic, affichage d’un modal).


// Exemple : ouvrir une carte lourde (react-native-maps) uniquement quand l'utilisateur clique
const handleOpenMap = async () => {
  const { default: MapView } = await import('react-native-maps');
  // Utiliser le composant chargé dynamiquement
  setMapComponent(<MapView region={region} />);
};

// Également utilisable avec React.lazy pour des composants
const LottieView = React.lazy(() => import('lottie-react-native'));

✨ Cas typique : Animation Lottie sur l'écran de succès de commande → importée uniquement quand la commande est validée.

🛠️ Mise en pratique dans un projet e-commerce

1️⃣
Analyser : Lancer npx expo-optimize puis npx expo export --dump-assetmap pour identifier les plus gros modules.
2️⃣
Découper : Mettre en place le lazy loading sur les écrans non critiques (Profil, Commandes, Favoris).
3️⃣
Transformer : Remplacer les require() par des import ES6 et activer le tree shaking.
4️⃣
Importer au besoin : Isoler la librairie de graphiques (Victory Native) ou de paiement Stripe derrière un import dynamique.

📉 Résultat attendu : Réduction de 30 à 50% du temps de chargement initial !

📌 Exercice type ressource (niveau medium)

Expliquez pourquoi le code suivant empêche le tree shaking et corrigez-le :

const moment = require('moment'); const formatDate = moment().format('DD/MM');
Voir la réponse

Le `require` importe tout le module. Correction : import moment from 'moment'; mais mieux : import { default as moment } from 'moment'; ou utiliser un sous-ensemble comme `dayjs`. Tree shaking ne fonctionne pas avec CommonJS.

💼 Exercice type compétence (niveau hard)

Dans l'application e-commerce, le module "Carte de suivi (react-native-maps)" pèse 800ko. Implémentez le lazy loading afin que la carte ne soit chargée que lorsque l'utilisateur clique sur le bouton "Voir le magasin". Écrivez le code du composant parent.

Proposition de code 
const TrackingMap = React.lazy(() => import('./TrackingMap'));
function OrderDetails() {
  const [showMap, setShowMap] = useState(false);
  return (
    <View>
      <Button title="Voir le magasin" onPress={() => setShowMap(true)} />
      {showMap && (
        <Suspense fallback={<Text>Chargement carte...</Text>}>
          <TrackingMap />
        </Suspense>
      )}
    </View>
  );
}
