🖼️ Optimisation des images avec expo-image

Chapitre 11 : Optimiser les performances

Apprenez à charger, mettre en cache et optimiser les images pour une expérience utilisateur fluide.

expo-image Caching Lazy loading Blurhash

🎯 Objectifs de la leçon

1

Utiliser expo-image pour remplacer Image de React Native

2

Mettre en place le caching automatique pour réduire la bande passante

3

Implémenter le lazy loading et le blurhash pour les placeholders

4

Redimensionner les images selon la taille de l'écran pour économiser la mémoire

💡 Pourquoi c'est important ? Dans une application e-commerce avec des centaines de produits, des images mal optimisées ralentissent l'application, consomment trop de mémoire et frustrent l'utilisateur (surtout sur connexion 3G/4G au Cameroun).

📦 Installation de expo-image

expo-image est une bibliothèque officielle d'Expo qui offre des performances bien supérieures au composant Image par défaut.

Installation
npx expo install expo-image

Comparaison des fonctionnalités :

❌ Image de React Native

  • Pas de caching automatique
  • Pas de lazy loading
  • Gestion mémoire limitée
  • Pas de blurhash

✅ expo-image

  • Caching disque/mémoire
  • Lazy loading natif
  • Blurhash et placeholders
  • Redimensionnement automatique

🚀 Utilisation de base et caching

Le composant <Image> de expo-image accepte les mêmes props que celui de React Native, avec des options supplémentaires.

import { Image } from 'expo-image';

// Utilisation simple
<Image
  source={{ uri: 'https://example.com/produit.jpg' }}
  style={{ width: 200, height: 200 }}
  cachePolicy="disk"  // Cache sur disque
/>

// Avec placeholder flou
<Image
  source="https://example.com/produit.jpg"
  placeholder={{ blurhash: 'L5H2EC=ON]RBV?t7t7R*00xZt7R*' }}
  contentFit="cover"
  transition={500}  // Animation de fondu
/>
📌 cachePolicy : 'memory' (RAM), 'disk' (stockage permanent), 'memory-disk' (les deux). Recommandé : 'disk' pour les catalogues produits.

🎨 Blurhash : l'effet "flou au chargement"

Blurhash est un algorithme qui génère une image floue compacte (30-40 caractères) à afficher pendant le chargement de l'image réelle. L'utilisateur voit immédiatement une zone colorée, puis l'image nette apparaît.

// 1. Générer un blurhash sur le serveur (ou avec un outil en ligne)
// Exemple de blurhash pour une image de produit : 'L5H2EC=ON]RBV?t7t7R*00xZt7R*'

// 2. Dans l'application, on l'utilise comme placeholder
<Image
  source={{ uri: product.imageUrl }}
  placeholder={{ blurhash: product.blurhash }}
  contentFit="cover"
  transition={300}
  style={{ width: '100%', height: 200 }}
/>

// Pour une galerie avec lazy loading (FlatList)
const renderItem = ({ item }) => (
  <Image
    source={{ uri: item.url }}
    placeholder={{ blurhash: item.blurhash }}
    recyclingKey={item.id}
    style={styles.image}
  />
);
⚡ Astuce : Générez les blurhashes côté backend lors de l'upload des images pour éviter de les calculer côté mobile.

📏 Redimensionnement des images

Pour économiser la mémoire et la bande passante, il est crucial d'afficher des images à la taille exacte dont l'écran a besoin.

import { Dimensions, PixelRatio } from 'react-native';

const { width: screenWidth } = Dimensions.get('window');
const pixelRatio = PixelRatio.get();

// Fonction pour obtenir l'URL redimensionnée (exemple avec Cloudinary)
const getOptimizedImageUrl = (originalUrl, width, height) => {
  // Si vous utilisez un service comme Cloudinary, Imgix, ou votre propre API
  return `${originalUrl}?w=${width}&h=${height}&fit=crop`;
};

// Dans votre composant :
const imageWidth = screenWidth - 32;  // Largeur écran - marges
const imageHeight = imageWidth * 0.75; // Ratio 4:3

<Image
  source={{ uri: getOptimizedImageUrl(product.imageUrl, imageWidth, imageHeight) }}
  style={{ width: imageWidth, height: imageHeight }}
  cachePolicy="disk"
/>
❌ Sans redimensionnement : image 4000×3000 (12 Mo) chargée en mémoire
✅ Avec redimensionnement : image 400×300 (120 Ko) seulement

🛡️ Gestion des erreurs et fallback

Une image peut ne pas se charger (serveur down, URL invalide). Il faut prévoir une image de remplacement.

import { Image } from 'expo-image';
import { useState } from 'react';

const ProductImage = ({ source, fallbackSource, ...props }) => {
  const [error, setError] = useState(false);

  return (
    <Image
      source={error ? fallbackSource : source}
      onError={() => setError(true)}
      placeholder={{ blurhash: 'L5H2EC=ON]RBV?t7t7R*00xZt7R*' }}
      contentFit="cover"
      {...props}
    />
  );
};

// Utilisation avec une image par défaut locale
<ProductImage
  source={{ uri: product.imageUrl }}
  fallbackSource={require('../assets/placeholder.png')}
  style={{ width: 150, height: 150 }}
/>
💾 Bonnes pratiques : Utilisez des placeholders locaux légers (images PNG de moins de 10 Ko) et vérifiez les URLs avant affichage.

📌 Récapitulatif & bonnes pratiques

  • ✅ Utilisez expo-image au lieu de Image de React Native pour toutes les images distantes.
  • ✅ Activez le caching disque (cachePolicy="disk") pour les catalogues de produits.
  • ✅ Générez des blurhashes côté serveur pour un affichage placeholder élégant.
  • ✅ Redimensionnez les images à la taille d'affichage (économie de mémoire jusqu'à 95%).
  • ✅ Prévoyez toujours un fallback (image par défaut) en cas d'erreur de chargement.

🌍 Cas pratique pour le Cameroun :

Avec des connexions parfois limitées (Orange, MTN, Camtel), le caching et le redimensionnement sont essentiels. Un catalogue de 100 produits peut passer de 200 Mo à moins de 5 Mo grâce à ces optimisations.
