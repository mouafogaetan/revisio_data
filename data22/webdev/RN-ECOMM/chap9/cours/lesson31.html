Développement mobile e-commerce | Enseignant: M. Tchinda - Cameroun

Objectifs de la leçon

  • Afficher la liste des commandes passées par l'utilisateur avec FlatList.
  • Visualiser les détails d'une commande (produits, montant, adresse).
  • Suivre l'état d'une commande (pending, shipped, delivered) avec interface visuelle.
  • Implémenter la fonction "Re-commander" pour ajouter les articles au panier.
À la fin, vous saurez créer une interface complète d'historique de commandes et de suivi de livraison.

⚠️ Les données sont généralement récupérées via une API protégée (token JWT).

2. Affichage des commandes (FlatList)

Composant React Native pour lister les commandes :


import { FlatList, View, Text, TouchableOpacity } from 'react-native';

const OrdersList = ({ orders, onPressOrder }) => {
  const formatDate = (isoDate) => new Date(isoDate).toLocaleDateString('fr-FR');

  const renderOrder = ({ item }) => (
    <TouchableOpacity 
      onPress={() => onPressOrder(item.id)}
      className="bg-white p-4 mb-3 rounded-xl shadow-sm border"
    >
      <View className="flex-row justify-between">
        <Text className="font-bold">Commande #{item.id.slice(-6)}</Text>
        <Text className="text-gray-500">{formatDate(item.date)}</Text>
      </View>
      <View className="flex-row justify-between mt-2">
        <Text>{item.items.length} article(s)</Text>
        <Text className="font-bold text-emerald-600">{item.totalAmount} FCFA</Text>
      </View>
      <View className="mt-2">
        <StatusBadge status={item.status} />
      </View>
    </TouchableOpacity>
  );

  return <FlatList data={orders} renderItem={renderOrder} keyExtractor={item => item.id} />;
};
💡 Optimisation : Utilisez FlashList (shopify/flash-list) pour les très longues listes.

3. Suivi de commande (Tracking)

Affichage des étapes sous forme de timeline :


const OrderTracking = ({ steps }) => {
  return (
    <View className="p-4">
      {steps.map((step, idx) => (
        <View key={idx} className="flex-row items-center mb-4">
          <View className={`w-3 h-3 rounded-full ${step.completed ? 'bg-green-500' : 'bg-gray-300'}`} />
          <View className="ml-3 flex-1">
            <Text className={`font-medium ${step.completed ? 'text-gray-800' : 'text-gray-400'}`}>
              {step.status}
            </Text>
            {step.date && <Text className="text-xs text-gray-500">{step.date}</Text>}
          </View>
        </View>
      ))}
    </View>
  );
};
Exemple visuel d'une timeline :

✅ Expédiée - 16/01/2025
⏳ En livraison
⌛ Livrée

4. Re-commander un produit

Ajouter au panier tous les articles d'une commande précédente :


import { useCartStore } from '../store/cartStore';

const OrderDetails = ({ order }) => {
  const addToCart = useCartStore(state => state.addToCart);
  const reorder = () => {
    order.items.forEach(item => {
      addToCart({
        productId: item.productId,
        name: item.name,
        quantity: item.quantity,
        price: item.price
      });
    });
    // Afficher un toast de confirmation
    Toast.show('Articles ajoutés au panier');
    // Naviguer vers l'écran panier
    navigation.navigate('Cart');
  };

  return (
    <View>
      <Button title="🔄 Re-commander" onPress={reorder} />
    </View>
  );
};
Astuce : Vérifiez la disponibilité des produits avant d'ajouter au panier (stock, prix actualisé).

5. Bonnes pratiques

✅ Mise en cache – Utilisez React Query pour stocker les commandes et éviter les appels réseau inutiles.
✅ Pull-to-refresh – Ajoutez refreshing et onRefresh à la FlatList.
❌ Erreur fréquente – Oublier de gérer le cas où la liste est vide (afficher un message "Aucune commande").
📦 Rappel : Les statuts doivent être synchronisés en temps réel via WebSocket ou polling si nécessaire.
