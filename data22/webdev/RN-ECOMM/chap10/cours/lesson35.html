📦 Installation de react-native-toast-message

npm install react-native-toast-message

Ensuite, envelopper l'application avec ToastProvider (ou ajouter le composant racine).

// App.js
import Toast from 'react-native-toast-message';

export default function App() {
  return (
    <>
      <NavigationContainer>...</NavigationContainer>
      <Toast />
    </>
  );
}

Aperçu d'un toast basique

🎨 Types de toasts prédéfinis

import Toast from 'react-native-toast-message';

// Toast succès
Toast.show({
  type: 'success',
  text1: '✅ Ajouté !',
  text2: 'Le produit est dans votre panier',
  position: 'top', // ou 'bottom'
  visibilityTime: 2000, // 2 secondes
});

// Toast erreur
Toast.show({
  type: 'error',
  text1: '❌ Oups !',
  text2: 'Stock insuffisant',
});

// Toast info
Toast.show({
  type: 'info',
  text1: '🎉 Promotion',
  text2: '-20% sur les smartphones',
});
Personnalisation : couleurs, icônes, animations personnalisées via toastConfig.

⚡ Alertes promotion et flash sales

Déclencher un toast info lorsqu'une promotion mondiale est active (ex: Black Friday, soldes).

// À l'arrivée sur l'écran Accueil
useEffect(() => {
  const checkPromo = async () => {
    const flashSale = await api.get('/active-promo');
    if (flashSale.active) {
      Toast.show({
        type: 'info',
        text1: `🔥 FLASH SALE : ${flashSale.discount}%`,
        text2: `Jusqu'à ${flashSale.endTime}`,
        autoHide: false, // Ne se ferme pas automatiquement
        onPress: () => navigation.navigate('Promotions')
      });
    }
  };
  checkPromo();
}, []);
Bonnes pratiques : Ne pas spammer ; proposer un bouton "Ne plus afficher".

⏱️ File d'attente des toasts

react-native-toast-message gère automatiquement une file d'attente : les toasts s'affichent les uns après les autres.

// Plusieurs toasts simultanés → ils s'affichent en séquence
Toast.show({ text1: 'Toast 1' });
Toast.show({ text1: 'Toast 2' }); // S'affiche après la disparition du 1er

// Forcer le remplacement
Toast.show({ text1: 'Important', visibilityTime: 5000 });
Toast.hide(); // Cache le toast actuel
Toast.show({ text1: 'Nouveau message' });
Éviter les toasts trop longs (>5s) pour ne pas frustrer l'utilisateur.

✨ Toast totalement personnalisé

Créer son propre composant toast pour inclure une image produit, un bouton, etc.

// toastConfig.js
const toastConfig = {
  customSuccess: ({ text1, text2, props }) => (
    <View style={{ backgroundColor: '#2e7d32', borderRadius: 12, padding: 12, flexDirection: 'row' }}>
      <Image source={{ uri: props.image }} style={{ width: 40, height: 40 }} />
      <View>
        <Text style={{ color: 'white', fontWeight: 'bold' }}>{text1}</Text>
        <Text style={{ color: '#ddd' }}>{text2}</Text>
      </View>
    </View>
  )
};

// Dans App.js
<Toast config={toastConfig} />

// Utilisation
Toast.show({
  type: 'customSuccess',
  text1: 'Nouveau produit',
  text2: 'Découvrez la dernière collection',
  props: { image: 'https://.../product.jpg' }
});

🎯 Synthèse & bonnes pratiques

  • ✔️ Utiliser des messages courts et précis (max 50 caractères).
  • ✔️ Durée adaptée : succès (1.5s), erreur (3s), promo longue (5s).
  • ✔️ Ne pas surcharger : maximum 3 toasts par minute.
  • ✔️ Feedback sonore optionnel (vibration pour erreur).
  • ✔️ A/B tester les positions (haut vs bas).

Indicateurs clés :

Taux de clic sur le toast + impact sur les ventes flash
