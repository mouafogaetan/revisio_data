🔔 Pourquoi un badge dynamique ?

Sur une application mobile, l’icône du panier doit donner un feedback visuel immédiat à l’utilisateur. Le badge (petite pastille rouge avec un chiffre) permet de savoir en un coup d’œil combien d’articles attendent dans le panier, sans ouvrir l’écran.

⚙️ Implémentation du compteur temps réel

On utilise un store Zustand contenant le panier (cartItems) et une fonction getTotalQuantity().


// store/cartStore.js
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

export const useCartStore = create(
  persist(
    (set, get) => ({
      items: [],
      addItem: (product) => set((state) => ({
        items: [...state.items, product]
      })),
      removeItem: (id) => set((state) => ({
        items: state.items.filter(i => i.id !== id)
      })),
      getTotalQuantity: () => {
        return get().items.reduce((total, item) => total + (item.quantity || 1), 0);
      }
    }),
    { name: 'cart-storage' }
  )
);

Dans l’icône de navigation (BottomTab), on écoute les changements et on affiche le badge.


// Composant CartIcon.jsx
import { useCartStore } from '../store/cartStore';
import { View, Text } from 'react-native';

export default function CartIcon() {
  const totalItems = useCartStore((state) => state.getTotalQuantity());
  return (
    <View style={{ position: 'relative' }}>
      <Icon name="cart-outline" size={26} />
      {totalItems > 0 && (
        <View style={{ position: 'absolute', right: -8, top: -4, backgroundColor: 'red', borderRadius: 12, paddingHorizontal: 5, paddingVertical: 1 }}>
          <Text style={{ color: 'white', fontSize: 12, fontWeight: 'bold' }}>{totalItems}</Text>
        </View>
      )}
    </View>
  );
}

📲 Badge sur l’icône de l’application (native)

En plus du badge dans l’UI, on peut modifier le badge de l’icône sur l’écran d’accueil du téléphone. Utile pour rappeler les articles en attente.

// Avec Expo (expo-notifications) ou react-native-push-notification


import * as Notifications from 'expo-notifications';

// Mettre à jour le badge natif
async function updateAppBadge(count) {
  await Notifications.setBadgeCountAsync(count);
}
// Appeler cette fonction à chaque modification du panier
🍎 iOS

Badge sur l’icône du bureau, support total.

🤖 Android

Nécessite une notification ou une bibliothèque (shortcutbadger).

💡 Important : Sur Android, le badge apparaît souvent avec les notifications push ou certaines launchers. `expo-notifications` gère cela.

🚀 Optimisation et pièges à éviter

  • Réactivité : Utilisez useMemo ou des sélecteurs Zustand pour ne pas re-render toute l’application.
  • Persistance : Le badge doit refléter le panier même après fermeture/ouverture de l’app (AsyncStorage).
  • Accessibilité : Annoncer le nombre d’articles aux lecteurs d’écran (aria-label).
  • Badge natif : Éviter de l’utiliser trop fréquemment (fatigue utilisateur) → à réserver pour un rappel significatif.

⚠️ Erreur fréquente :

Ne pas recalculer le total après suppression ou modification de quantité ⇒ toujours dériver la quantité à partir du tableau d’items.
