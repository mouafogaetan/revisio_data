💳 Intégration Stripe PaymentSheet

React Native (Expo / CLI) · Paiement sécurisé

Stripe SDK PaymentSheet Webhook
Développement mobile e-commerce | Enseignant: M. Tchinda - Cameroun

Objectifs de la leçon

  • Installer et configurer `@stripe/stripe-react-native` dans un projet Expo.
  • Créer un PaymentIntent côté backend et le présenter sur le mobile.
  • Gérer la réponse du paiement (succès, échec) et le webhook de confirmation.
  • Sécuriser les transactions avec les bonnes pratiques Stripe.
À la fin, vous saurez intégrer un paiement par carte bancaire, Apple Pay ou Google Pay dans votre app.

1. Installation du SDK Stripe

Pour un projet Expo, on utilise :

npx expo install @stripe/stripe-react-native

⚠️ Si vous utilisez React Native CLI, suivez la documentation officielle pour le linking natif.

Sur iOS, ajoutez la clé NSFaceIDUsageDescription dans Info.plist pour Apple Pay. Sur Android, aucune configuration supplémentaire.

Ensuite, enveloppez votre application avec le StripeProvider :


import { StripeProvider } from '@stripe/stripe-react-native';

export default function App() {
  return (
    <StripeProvider publishableKey="pk_test_...">
      <NavigationContainer>
        <AppNavigator />
      </NavigationContainer>
    </StripeProvider>
  );
}

2. Créer un PaymentIntent côté serveur

Le client mobile ne doit jamais manipuler directement la clé secrète Stripe. On appelle un endpoint backend (Node.js/Express) qui crée un PaymentIntent.


// Endpoint /create-payment-intent
app.post('/create-payment-intent', async (req, res) => {
  const { amount, currency, customerId } = req.body;
  const paymentIntent = await stripe.paymentIntents.create({
    amount: Math.round(amount), // en cents
    currency: currency,
    customer: customerId,
    automatic_payment_methods: { enabled: true }
  });
  res.json({ clientSecret: paymentIntent.client_secret });
});
💰 Attention : le montant est en cents (ex: 5000 = 50,00 €).

3. Présenter PaymentSheet (frontend)

Dans votre écran de paiement, utilisez le hook useStripe :


import { useStripe } from '@stripe/stripe-react-native';
import { useEffect, useState } from 'react';

const PaymentScreen = ({ route }) => {
  const { initPaymentSheet, presentPaymentSheet } = useStripe();
  const [loading, setLoading] = useState(false);

  const fetchPaymentIntent = async () => {
    const response = await fetch('https://api.example.com/create-payment-intent', {
      method: 'POST',
      body: JSON.stringify({ amount: 2999, currency: 'eur' })
    });
    const { clientSecret } = await response.json();
    return clientSecret;
  };

  const initializePayment = async () => {
    const clientSecret = await fetchPaymentIntent();
    const { error } = await initPaymentSheet({
      paymentIntentClientSecret: clientSecret,
      merchantDisplayName: 'Ma Boutique',
    });
    if (!error) setLoading(true);
  };

  const openPaymentSheet = async () => {
    const { error } = await presentPaymentSheet();
    if (error) alert(`Échec: ${error.message}`);
    else alert('Paiement réussi ! Commande validée');
  };

  useEffect(() => { initializePayment(); }, []);

  return <Button title="Payer" onPress={openPaymentSheet} disabled={!loading} />;
};

PaymentSheet gère automatiquement les méthodes de paiement (carte, Apple Pay, Google Pay).

4. Webhook Stripe (confirmation backend)

Pour mettre à jour votre base de données (statut commande = "payée"), Stripe envoie un événement payment_intent.succeeded sur votre serveur.


// Webhook endpoint
app.post('/webhook', express.raw({type: 'application/json'}), (req, res) => {
  const sig = req.headers['stripe-signature'];
  let event;
  try {
    event = stripe.webhooks.constructEvent(req.body, sig, endpointSecret);
  } catch (err) { return res.status(400).send(`Webhook Error: ${err.message}`); }

  if (event.type === 'payment_intent.succeeded') {
    const paymentIntent = event.data.object;
    // Mettre à jour la commande dans MongoDB/PostgreSQL
    updateOrderStatus(paymentIntent.id, 'paid');
  }
  res.json({received: true});
});
🔐 Sécurité : Vérifiez toujours la signature du webhook pour éviter les faux appels.

5. Sécurité et erreurs courantes

❌ Ne jamais stocker le clientSecret dans AsyncStorage ! Il est éphémère et doit être re-créé à chaque session de paiement.
⚠️ Mode test vs live : Utilisez les clés "pk_test_" en développement, et "pk_live_" en production.
Bonnes pratiques : Idempotence des requêtes, timeout gestion, fallback si paiement échoue.

💡 Pour tester sans carte réelle : utilisez la carte 4242 4242 4242 4242 (date future, CVC 123).
