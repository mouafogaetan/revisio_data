➕➖ Modifier les quantités

Des boutons d'incrémentation/décrémentation permettent d'ajuster la quantité. On empêche de descendre en dessous de 1.

// Fonction de mise à jour locale (Zustand ou state)
const updateQuantity = (id, newQty) => {
  if (newQty < 1) return;
  setCartItems(prev => prev.map(item =>
    item.id === id ? { ...item, quantity: newQty } : item
  ));
};

// Dans le JSX
<View style={{ flexDirection: 'row', alignItems: 'center' }}>
  <TouchableOpacity onPress={() => updateQuantity(item.id, item.quantity - 1)}>
    <Text style={{ fontSize: 20 }}>-</Text>
  </TouchableOpacity>
  <Text style={{ marginHorizontal: 10 }}>{item.quantity}</Text>
  <TouchableOpacity onPress={() => updateQuantity(item.id, item.quantity + 1)}>
    <Text style={{ fontSize: 20 }}>+</Text>
  </TouchableOpacity>
</View>
✨ Chaque changement recalcule automatiquement le total (réactivité Zustand/useState).

💰 Calcul du total & animation

$$Total = \\sum_{i=1}^{n} (prix_i \\times quantit\\acute{e}_i)$$

Exemple : 2 × 15 000 + 1 × 9 900 = 39 900 FCFA

// Calcul du total avec useMemo pour optimisation
const total = useMemo(() => {
  return cartItems.reduce((sum, item) => sum + (item.price * item.quantity), 0);
}, [cartItems]);

// Animation au changement (exemple avec Reanimated)
useEffect(() => {
  // Déclencher une animation sur le texte du total
  totalAnimated.value = withSpring(total, { damping: 10 });
}, [total]);

⚡ Optimisations & bonnes pratiques

📌 useMemo / useCallback

Mémoriser le calcul du total et les fonctions pour éviter les re-rendus inutiles.

📌 Animer avec Reanimated

Utiliser withSpring ou withTiming pour des transitions fluides du prix total.

📌 Gestion des conflits

En mode offline, stocker les actions (ajout/suppr) et synchroniser au retour en ligne.

📌 Accessibilité

Ajouter des labels pour les boutons + / – et confirmer la suppression.

💡 Le total animé donne un feedback instantané : l'utilisateur perçoit immédiatement l'impact de ses actions.
