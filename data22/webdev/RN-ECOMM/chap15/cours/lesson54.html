🔊 VoiceOver (iOS) et TalkBack (Android)

Les lecteurs d'écran vocalisent les éléments de l'interface. React Native fournit des props d'accessibilité :

<TouchableOpacity
  accessible={true}
  accessibilityLabel="Ajouter le produit au panier"
  accessibilityHint="Double-tap pour ajouter l'article à votre panier"
  accessibilityRole="button"
  onPress={addToCart}
>
  <Text>Ajouter</Text>
</TouchableOpacity>

<Image
  source={{ uri: product.image }}
  accessibilityLabel={`Image de ${product.name}`}
  accessibilityRole="image"
/>

<Text accessibilityRole="header">Détails du produit</Text>
Bonnes pratiques : décrire clairement les actions, ne pas utiliser "cliquez ici" mais l'action réelle.

📱 Tailles de police adaptatives

Respecter les préférences de police de l'utilisateur (paramètres système).

import { Text, Appearance } from 'react-native';

// Utiliser des unités relatives (rem-like)
const styles = StyleSheet.create({
  title: {
    fontSize: 20, // base
    // Alternative : utiliser PixelRatio
  },
  body: {
    fontSize: 16,
  }
});

// Ou utiliser la police système avec scaling automatique
<Text allowFontScaling={true}>Texte adaptable</Text>

// Désactiver le scaling pour certains éléments (optionnel)
<Text allowFontScaling={false}>Texte fixe</Text>
Police normale (16px)
Police agrandie (20px)
Très grande (28px)

Les personnes malvoyantes augmentent souvent la taille des polices dans les paramètres.

🎨 Contrastes pour la lisibilité

Respecter le ratio de contraste minimum (WCAG 2.1) : 4.5:1 pour le texte normal, 3:1 pour le texte large.

Bon contraste (blanc sur bleu foncé)
Mauvais contraste (jaune clair sur jaune)
// Calcul du ratio de contraste
// Luminance relative (formule WCAG)
const luminance = (r, g, b) => {
  const [rs, gs, bs] = [r, g, b].map(c => {
    c = c / 255;
    return c <= 0.03928 ? c / 12.92 : Math.pow((c + 0.055) / 1.055, 2.4);
  });
  return 0.2126 * rs + 0.7152 * gs + 0.0722 * bs;
};

const contrastRatio = (l1, l2) => (Math.max(l1, l2) + 0.05) / (Math.min(l1, l2) + 0.05);

$$ \text{Ratio} = \frac{L_1 + 0.05}{L_2 + 0.05} \geq 4.5 $$

🌙 Support du Dark Mode et High Contrast

Utiliser useColorScheme pour détecter le thème système.

import { useColorScheme } from 'react-native';

const App = () => {
  const colorScheme = useColorScheme(); // 'light' ou 'dark'

  const theme = {
    background: colorScheme === 'dark' ? '#121212' : '#FFFFFF',
    text: colorScheme === 'dark' ? '#FFFFFF' : '#000000',
    primary: colorScheme === 'dark' ? '#90caf9' : '#1976d2'
  };

  return (
    <View style={{ backgroundColor: theme.background }}>
      <Text style={{ color: theme.text }}>Contenu adapté</Text>
    </View>
  );
};

// High Contrast (accessibilité)
import { AccessibilityInfo } from 'react-native';
const isHighContrast = await AccessibilityInfo.isHighContrastEnabled();

✅ Tester l'accessibilité

  • iOS : Settings → Accessibility → VoiceOver → Activer (triple-clic home/power)
  • Android : Settings → Accessibility → TalkBack → Activer
  • Outils :
    • axe-react-native (linting)
    • react-native-accessibility-engine
    • Tests manuels avec lecteurs d'écran

Checklist minimale :

✓ Toutes les images ont un label ✓ Boutons descriptifs ✓ Tailles tactiles ≥ 44x44pt ✓ Contrastes suffisants

🎯 Synthèse & mini-projet

Points clés à retenir :

  • ✅ Ajouter accessibilityLabel à TOUS les éléments interactifs
  • ✅ Utiliser allowFontScaling pour respecter les préférences utilisateur
  • ✅ Maintenir un contraste ≥ 4.5:1
  • ✅ Tester avec VoiceOver/TalkBack au moins une fois par sprint

Mini-projet :

Rendez votre écran produit accessible : ajoutez des labels aux images, boutons et zones cliquables. Testez avec TalkBack sur Android. Calculez le ratio de contraste de vos couleurs et corrigez-le si nécessaire.
