📦 Installation d'Axios

Axios est un client HTTP qui fonctionne nativement avec React Native (supporte les promesses, intercepteurs, annulation de requêtes).

npm install axios

Créer un fichier services/api.ts pour configurer l'instance Axios :

import axios from 'axios';

const api = axios.create({
  baseURL: process.env.EXPO_PUBLIC_API_URL || 'https://api.maboutique.com',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

export default api;

📌 Variables d'environnement Expo : utiliser EXPO_PUBLIC_ comme préfixe pour exposer au code client.

🔐 Intercepteur de requête - Ajout automatique du token

L'intercepteur de requête s'exécute avant l'envoi de chaque requête. On y ajoute le token JWT stocké.

import * as SecureStore from 'expo-secure-store';

api.interceptors.request.use(
  async (config) => {
    // Récupérer le token depuis le stockage sécurisé
    const token = await SecureStore.getItemAsync('access_token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

💡 Expo SecureStore est recommandé pour stocker les tokens sensibles (plus sûr que AsyncStorage).

🔄 Intercepteur de réponse - Refresh token automatique

Lorsqu'un token expire (code 401), on tente de le rafraîchir silencieusement avant de réessayer la requête originale.

let isRefreshing = false;
let failedQueue: any[] = [];

const processQueue = (error: any, token: string | null = null) => {
  failedQueue.forEach(prom => {
    if (error) prom.reject(error);
    else prom.resolve(token);
  });
  failedQueue = [];
};

api.interceptors.response.use(
  (response) => response,
  async (error) => {
    const originalRequest = error.config;
    
    if (error.response?.status === 401 && !originalRequest._retry) {
      if (isRefreshing) {
        return new Promise((resolve, reject) => {
          failedQueue.push({ resolve, reject });
        }).then(token => {
          originalRequest.headers.Authorization = `Bearer ${token}`;
          return api(originalRequest);
        }).catch(err => Promise.reject(err));
      }
      
      originalRequest._retry = true;
      isRefreshing = true;
      
      try {
        const refreshToken = await SecureStore.getItemAsync('refresh_token');
        const response = await axios.post(`${api.defaults.baseURL}/auth/refresh`, {
          refresh_token: refreshToken,
        });
        
        const { access_token, refresh_token } = response.data;
        await SecureStore.setItemAsync('access_token', access_token);
        await SecureStore.setItemAsync('refresh_token', refresh_token);
        
        processQueue(null, access_token);
        originalRequest.headers.Authorization = `Bearer ${access_token}`;
        return api(originalRequest);
      } catch (refreshError) {
        processQueue(refreshError, null);
        // Déconnexion : rediriger vers login
        return Promise.reject(refreshError);
      } finally {
        isRefreshing = false;
      }
    }
    
    return Promise.reject(error);
  }
);

⏳ File d'attente des requêtes pendant le refresh

Pendant qu'on rafraîchit le token, d'autres requêtes peuvent arriver. Il faut les mettre en file d'attente et les exécuter une fois le nouveau token obtenu.

📌 Principe :

  1. Première requête 401 → déclenche le refresh token
  2. Les requêtes suivantes (pendant le refresh) sont mises dans une queue
  3. Après obtention du nouveau token, on les exécute toutes avec le nouveau token
  4. Si le refresh échoue, on rejette toutes les requêtes en queue
// Structure de la queue
let failedQueue: Array<{
  resolve: (token: string) => void;
  reject: (error: any) => void;
}> = [];

// Ajouter une requête à la queue
const queueRequest = () => {
  return new Promise((resolve, reject) => {
    failedQueue.push({ resolve, reject });
  });
};

// Traiter la queue après refresh
const processQueue = (error: any, token: string | null = null) => {
  failedQueue.forEach(prom => {
    if (error) prom.reject(error);
    else prom.resolve(token!);
  });
  failedQueue = [];
};

📱 Utilisation dans les composants React Native

Une fois Axios configuré, l'utiliser dans les composants ou services :

// services/productService.ts
import api from './api';

export const getProducts = async (page = 1) => {
  const response = await api.get('/products', { params: { page } });
  return response.data;
};

export const getProductById = async (id: string) => {
  const response = await api.get(`/products/${id}`);
  return response.data;
};

export const createOrder = async (orderData: any) => {
  const response = await api.post('/orders', orderData);
  return response.data;
};
// Composant React Native
import { getProducts } from '../services/productService';

function ProductList() {
  const [products, setProducts] = useState([]);
  
  useEffect(() => {
    const fetchProducts = async () => {
      try {
        const data = await getProducts();
        setProducts(data);
      } catch (error) {
        console.error('Erreur chargement produits', error);
      }
    };
    fetchProducts();
  }, []);
  
  // ... render
}

✅ Plus besoin de gérer manuellement le token dans chaque appel !

⚠️ Gestion centralisée des erreurs

Ajouter un intercepteur de réponse pour les erreurs globales (toast, logs Sentry, déconnexion).

api.interceptors.response.use(
  (response) => response,
  async (error) => {
    // Erreur réseau
    if (!error.response) {
      Toast.show('Problème de connexion internet');
      return Promise.reject(error);
    }
    
    const { status, data } = error.response;
    
    // Erreur 500
    if (status >= 500) {
      Toast.show('Erreur serveur, réessayez plus tard');
    }
    
    // Erreur 403 (accès interdit)
    if (status === 403) {
      Toast.show('Vous n\'avez pas les droits nécessaires');
    }
    
    // Erreur 404
    if (status === 404) {
      Toast.show('Ressource non trouvée');
    }
    
    // Envoyer l'erreur à Sentry
    Sentry.captureException(error);
    
    return Promise.reject(error);
  }
);

💡 Bonnes pratiques : Ne pas afficher les erreurs techniques à l'utilisateur, mais des messages clairs et utiles.
