1. Détection de connexion avec NetInfo

@react-native-community/netinfo permet de connaître l'état du réseau : connecté/déconnecté, type (wifi, cellular, aucun).

import NetInfo from '@react-native-community/netinfo';

// Écouter les changements
const unsubscribe = NetInfo.addEventListener(state => {
  console.log('Type:', state.type);
  console.log('Connecté:', state.isConnected);
  console.log('Internet:', state.isInternetReachable);
});

// Récupérer l'état une fois
NetInfo.fetch().then(state => {
  setOnline(state.isConnected && state.isInternetReachable);
});

// Nettoyage
useEffect(() => unsubscribe, []);
isConnected : connecté à un réseau (même sans internet)
isInternetReachable : véritable accès internet
Astuce : Combinez les deux pour une fiabilité maximale.

2. File d'attente offline (Queue)

On stocke les requêtes échouées dans un tableau persisté. Exemple avec Zustand :

// offlineQueueStore.js
import create from 'zustand';
import { persist } from 'zustand/middleware';

const useOfflineQueueStore = create(
  persist(
    (set, get) => ({
      queue: [],
      addToQueue: (request) => set({ queue: [...get().queue, request] }),
      removeFromQueue: (index) => set({ queue: get().queue.filter((_, i) => i !== index) }),
      clearQueue: () => set({ queue: [] })
    }),
    { name: 'offline-queue-storage' }
  )
);

Chaque élément de la queue peut contenir : méthode, url, body, timestamp.

Stockée dans AsyncStorage, la queue survit au redémarrage de l'app.

3. Intercepter les requêtes échouées

On étend Axios pour mettre automatiquement en queue les requêtes quand le réseau est absent.

// axiosInstance.js
import axios from 'axios';
import NetInfo from '@react-native-community/netinfo';
import useOfflineQueueStore from './offlineQueueStore';

const axiosInstance = axios.create({ baseURL: 'https://api.example.com' });

axiosInstance.interceptors.response.use(
  response => response,
  async error => {
    const netState = await NetInfo.fetch();
    if (!netState.isConnected) {
      // Sauvegarder la requête originale
      const { config } = error;
      useOfflineQueueStore.getState().addToQueue(config);
      return Promise.reject({ offline: true, message: 'Pas de connexion' });
    }
    return Promise.reject(error);
  }
);
export default axiosInstance;

✔️ Si hors ligne, la requête est pushée dans la queue. Sinon, l'erreur remonte normalement.

4. Synchronisation automatique

On écoute l'événement de reconnexion et on vide la queue.

// hooks/useSyncOfflineQueue.js
import { useEffect } from 'react';
import NetInfo from '@react-native-community/netinfo';
import useOfflineQueueStore from '../stores/offlineQueueStore';
import axiosInstance from '../services/axiosInstance';

const useSyncOfflineQueue = () => {
  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener(state => {
      if (state.isConnected && state.isInternetReachable) {
        syncQueue();
      }
    });
    return () => unsubscribe();
  }, []);

  const syncQueue = async () => {
    const { queue, clearQueue } = useOfflineQueueStore.getState();
    for (const request of queue) {
      try {
        await axiosInstance(request);
      } catch (e) {
        console.log('Échec synchro', e);
      }
    }
    clearQueue();
  };
};
export default useSyncOfflineQueue;
Appeler ce hook dans App.js pour activer la synchro automatique.

6. Cas d'usage : ajout au panier offline

L'utilisateur ajoute un produit. Si offline, on stocke l'action. Au retour en ligne, on l'envoie.

const addToCart = async (product) => {
  try {
    await api.post('/cart', product);
    showToast('Produit ajouté');
  } catch (err) {
    if (err.offline) {
      // Stocker dans une autre queue spécifique ou la queue générale
      offlineQueueStore.addToQueue({ method: 'POST', url: '/cart', data: product });
      showToast('Ajouté localement - sera synchronisé plus tard');
    }
  }
};

📊 Résultat attendu :

→ 100% des actions sont sauvegardées

→ Pas de perte de données lors de micro-coupures

→ Conformité avec les usages au Cameroun (réseaux mobiles parfois instables)

Bonnes pratiques & Récapitulatif

  • Toujours vérifier isInternetReachable et non seulement isConnected.
  • Limiter la taille de la queue (max 100 requêtes) pour éviter la lenteur.
  • Ajouter un timestamp pour gérer l'expiration des requêtes trop anciennes.
  • Offrir un bouton "Réessayer" manuel pour les utilisateurs impatients.
  • Utiliser React Query + NetInfo pour une synchro encore plus poussée.

Formule mathématique de la résilience : $$ P(\text{perte}) = \prod_{t=1}^{n} (1 - \text{synchro}_{t}) \approx 0 $$ quand la queue est bien implémentée.
