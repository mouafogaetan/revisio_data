1. Header avec notification et recherche

Le header de l'application mobile contient généralement :

  • Le logo ou le nom de l'application
  • Une barre de recherche (ou une icône de recherche)
  • Une icône de notification avec un badge indiquant le nombre d'alertes non lues

🧩 Exemple de code React Native avec React Native Paper :


import { Appbar, Searchbar, Badge } from 'react-native-paper';
import { View } from 'react-native';

const HomeHeader = ({ notificationsCount, onSearch }) => {
  const [searchQuery, setSearchQuery] = useState('');
  
  return (
    <Appbar.Header>
      <Appbar.Content title="ShopEase" />
      <View style={{ flexDirection: 'row', alignItems: 'center' }}>
        <Searchbar
          placeholder="Rechercher..."
          value={searchQuery}
          onChangeText={setSearchQuery}
          onSubmitEditing={() => onSearch(searchQuery)}
          style={{ width: 180, height: 40 }}
        />
        <View>
          <Appbar.Action icon="bell" onPress={() => {}} />
          {notificationsCount > 0 && <Badge size={18}>{notificationsCount}</Badge>}
        </View>
      </View>
    </Appbar.Header>
  );
};

Le badge se met à jour en temps réel grâce à un state (Zustand/Context) contenant le nombre de notifications.

2. Carrousel de produits vedettes

Pour un rendu horizontal optimisé, on utilise FlatList avec horizontal={true}. Chaque élément est une carte produit.

✅ Avantages :
  • Défilement fluide
  • Recyclage des composants
  • Performance mobile
🎨 Extrait code : 

<FlatList
  data={featuredProducts}
  horizontal
  showsHorizontalScrollIndicator={false}
  renderItem={({item}) => (
    <ProductCard product={item} onPress={() => navigateToProduct(item.id)} />
  )}
  keyExtractor={item => item.id}
/>
Exemple visuel : [Image d’un carrousel de smartphones défilant horizontalement]

3. Affichage des catégories & promotions

Organiser les catégories en grille (2 ou 3 colonnes) à l'aide de FlatList avec numColumns={2} ou FlashList pour les performances.

Exemple de données catégories :

4. Structure complète de l'écran Home

Généralement, l'écran d'accueil utilise un ScrollView ou une FlatList avec en-tête personnalisé pour regrouper :

  • Header (recherche + notifications)
  • Carrousel des produits vedettes
  • Grille des catégories
  • Bannières promotionnelles
  • Liste des produits recommandés

<SafeAreaView style={{flex:1}}>
  <HomeHeader />
  <FlatList
    ListHeaderComponent={
      <>
        <FeaturedCarousel />
        <CategoriesGrid />
        <PromoBanner />
      </>
    }
    data={recommendedProducts}
    renderItem={ProductItem}
  />
</SafeAreaView>

Cette architecture est modulaire et réutilisable.

5. Bonnes pratiques & optimisations

✨ Utiliser FlashList pour les longues listes (performance ++).
🖼️ Images optimisées : expo-image avec caching et placeholder blurhash.
⚠️ Erreur fréquente : Ne pas oublier keyExtractor dans les FlatList pour éviter les bugs de réutilisation.
📱 À retenir : La fluidité du scroll sur mobile dépend du bon usage de la virtualisation et des images légères.
