⚙️ Installation d'expo-location et permissions

Pour accéder à la position de l'utilisateur, on utilise le module expo-location.


# Installation
npx expo install expo-location

# Pour iOS : ajouter dans app.json
"ios": {
  "infoPlist": {
    "NSLocationWhenInUseUsageDescription": "Cette application a besoin de votre position pour trouver des magasins proches"
  }
}

# Pour Android (automatique, mais vérifier AndroidManifest.xml)

💡 Bonnes pratiques : Demander la permission uniquement quand l'utilisateur en a besoin (ex: clic sur "Trouver des magasins"). Toujours gérer le refus de permission.

📍 Récupérer la position de l'utilisateur

Exemple complet avec demande de permission et gestion des erreurs.


import * as Location from 'expo-location';
import { useState } from 'react';
import { View, Text, Button, Alert } from 'react-native';

export default function GetLocation() {
  const [location, setLocation] = useState(null);

  const getUserLocation = async () => {
    let { status } = await Location.requestForegroundPermissionsAsync();
    if (status !== 'granted') {
      Alert.alert('Permission refusée', 'Activez la localisation pour voir les magasins proches');
      return;
    }
    let currentLocation = await Location.getCurrentPositionAsync({});
    setLocation(currentLocation.coords);
    // currentLocation.coords = { latitude, longitude, accuracy ... }
  };

  return (
    <View>
      <Button title="Obtenir ma position" onPress={getUserLocation} />
      {location && <Text>Lat : {location.latitude}, Lon : {location.longitude}</Text>}
    </View>
  );
}

🏪 Afficher les magasins proches

Combinaison d'expo-location et de react-native-maps : on récupère la position, puis on affiche les magasins autour.


import MapView, { Marker } from 'react-native-maps';
import * as Location from 'expo-location';
import { useState, useEffect } from 'react';

const stores = [
  { id: 1, name: "Market Douala", lat: 4.0511, lon: 9.7679 },
  { id: 2, name: "Shop Yaoundé", lat: 3.8480, lon: 11.5021 },
  { id: 3, name: "E-store Garoua", lat: 9.3012, lon: 13.3997 }
];

export default function NearbyStores() {
  const [userLoc, setUserLoc] = useState(null);

  useEffect(() => {
    (async () => {
      let { status } = await Location.requestForegroundPermissionsAsync();
      if (status === 'granted') {
        let loc = await Location.getCurrentPositionAsync({});
        setUserLoc(loc.coords);
      }
    })();
  }, []);

  if (!userLoc) return <Text>Chargement carte...</Text>;

  return (
    <MapView
      style={{ flex: 1 }}
      initialRegion={{
        latitude: userLoc.latitude,
        longitude: userLoc.longitude,
        latitudeDelta: 0.5,
        longitudeDelta: 0.5,
      }}
    >
      <Marker coordinate={userLoc} title="Vous êtes ici" pinColor="blue" />
      {stores.map(store => (
        <Marker
          key={store.id}
          coordinate={{ latitude: store.lat, longitude: store.lon }}
          title={store.name}
          description="Magasin partenaire"
        />
      ))}
    </MapView>
  );
}

📦 Cas pratique : Sélectionner un point de retrait

L'utilisateur choisit un magasin proche pour le retrait de sa commande. On peut combiner distance, carte et validation.

Étapes clés :

  1. Obtenir la localisation de l'utilisateur.
  2. Calculer la distance entre l'utilisateur et chaque magasin (formule de Haversine).
  3. Afficher la carte avec les marqueurs et un bouton "Choisir ce magasin".
  4. Stocker le magasin sélectionné dans le store (Zustand) pour l'utiliser lors du checkout.

📐 Formule de distance (Haversine) :

$d = 2R \\cdot \\arcsin\\left(\\sqrt{\\sin^2\\left(\\frac{\\Delta\\phi}{2}\\right) + \\cos\\phi_1 \\cos\\phi_2 \\sin^2\\left(\\frac{\\Delta\\lambda}{2}\\right)}\\right)$

où $R = 6371$ km (rayon terrestre).

Implémentation simple avec `geolib` ou `haversine-distance`.

🔍 Exercice ressource (niveau medium)

Expliquez la différence entre requestForegroundPermissionsAsync et requestBackgroundPermissionsAsync. Dans quel cas utiliser l'une ou l'autre pour une app e-commerce ?

Voir la réponse

La permission foreground permet d'accéder à la localisation uniquement lorsque l'app est ouverte. Background autorise un accès en arrière-plan. Pour une app e-commerce, la permission "when in use" (foreground) est suffisante pour afficher les magasins proches. La permission background serait utile pour envoyer des notifications de livraison basées sur la position.

Proposition de code 
const getDistance = (lat1, lon1, lat2, lon2) => {
  const R = 6371;
  const dLat = (lat2-lat1)*Math.PI/180;
  const dLon = (lon2-lon1)*Math.PI/180;
  const a = Math.sin(dLat/2)**2 + Math.cos(lat1*Math.PI/180)*Math.cos(lat2*Math.PI/180)*Math.sin(dLon/2)**2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
};

// Dans le composant : on utilise useEffect pour la position et on stocke les distances
const handleMarkerPress = (store) => {
  const dist = getDistance(userLoc.latitude, userLoc.longitude, store.lat, store.lon);
  Alert.alert(store.name, `Distance : ${dist.toFixed(1)} km`);
};
