💡 Les widgets et raccourcis augmentent l'engagement utilisateur en donnant un accès rapide aux fonctionnalités clés.

🍎 Widgets iOS (WidgetKit)

Les widgets iOS nécessitent une configuration native. Avec Expo, on utilise les config plugins.

// app.json
{
  "expo": {
    "plugins": [
      [
        "expo-widgets",
        {
          "widgets": [
            {
              "name": "CartWidget",
              "description": "Voir mon panier",
              "family": ["systemSmall", "systemMedium"],
              "kind": "com.monapp.cart"
            }
          ]
        }
      ]
    ]
  }
}

📌 Families iOS : `systemSmall` (carré), `systemMedium` (2x plus large), `systemLarge` (grand).

📌 Le widget est un fichier Swift (hors Expo) ; Expo le génère via le plugin.

🛒 Widget panier - Concept et données

Le widget doit afficher des données dynamiques (nombre d'articles, total). Les données sont partagées via App Group (iOS) ou SharedPreferences.

// Partager les données depuis l'app
import { AppGroup } from 'expo-app-group';

const saveCartToWidget = async (items) => {
  const totalItems = items.reduce((sum, i) => sum + i.quantity, 0);
  const totalPrice = items.reduce((sum, i) => sum + i.price * i.quantity, 0);
  
  await AppGroup.setDataAsync(
    'group.com.monapp.cart',
    JSON.stringify({ totalItems, totalPrice })
  );
};

🤖 Android Quick Actions (raccourcis)

Quick Actions permettent d'ajouter des raccourcis dynamiques ou statiques via une longue pression sur l'icône de l'application.

npm install expo-quick-actions
import * as QuickActions from 'expo-quick-actions';

// Définir les actions statiques (dans app.json ou en runtime)
QuickActions.setShortcutItems([
  {
    id: 'view-cart',
    title: 'Voir mon panier',
    icon: { type: 'symbol', name: 'cart.fill' },
    params: { screen: 'Cart' }
  },
  {
    id: 'search',
    title: 'Rechercher',
    icon: { type: 'system', name: 'search' },
    params: { screen: 'Search' }
  }
]);

// Écouter le clic sur un raccourci
QuickActions.addListener(({ actionId, params }) => {
  if (actionId === 'view-cart') {
    navigation.navigate('Cart');
  }
});

💡 Sur Android, les icônes doivent être des drawable resources (PNG).

🔄 Actions dynamiques (ex: dernier produit consulté)

Les actions peuvent être mises à jour dynamiquement selon le comportement de l'utilisateur.

// Mettre à jour les actions après consultation d'un produit
const updateRecentProductQuickAction = (product) => {
  QuickActions.setShortcutItems([
    {
      id: `product-${product.id}`,
      title: product.name,
      subtitle: `${product.price} XAF`,
      icon: { type: 'uri', uri: product.iconUrl },
      params: { productId: product.id }
    },
    ...previousActions // garder les actions par défaut
  ]);
};

⚠️ Limite : 5 actions maximum (Android). iOS supporte 4 actions statiques uniquement.

🔄 Synchronisation widget panier en temps réel

À chaque modification du panier, mettre à jour le widget iOS et Android.

// Dans cartStore
import { AppGroup } from 'expo-app-group';
import * as QuickActions from 'expo-quick-actions';

const updateWidgets = async (items) => {
  const totalItems = items.reduce((sum, i) => sum + i.quantity, 0);
  const totalPrice = items.reduce((sum, i) => sum + i.price * i.quantity, 0);
  
  // iOS Widget
  await AppGroup.setDataAsync(
    'group.com.monapp.cart',
    JSON.stringify({ totalItems, totalPrice })
  );
  
  // Android Quick Action (mettre à jour le badge dynamique)
  if (totalItems > 0) {
    QuickActions.setShortcutItems([
      {
        id: 'cart-badge',
        title: `Panier (${totalItems})`,
        icon: { type: 'symbol', name: 'cart' },
        params: { screen: 'Cart' }
      }
    ]);
  }
};

📌 Appeler `updateWidgets()` dans `addItem`, `removeItem`, `clearCart`.

🔗 Ouvrir l'application depuis un widget

Lorsqu'on clique sur un widget, l'application s'ouvre. Il faut capturer l'intention pour naviguer vers le bon écran.

// Dans App.js ou racine
import * as Linking from 'expo-linking';
import { useEffect } from 'react';

useEffect(() => {
  const handleDeepLink = (event) => {
    const url = event.url;
    if (url?.includes('openCart')) {
      navigation.navigate('Cart');
    }
  };
  
  const subscription = Linking.addEventListener('url', handleDeepLink);
  
  // Vérifier l'URL au démarrage
  Linking.getInitialURL().then(url => {
    if (url?.includes('openCart')) {
      navigation.navigate('Cart');
    }
  });
  
  return () => subscription.remove();
}, []);

💡 Sur iOS, le widget déclenche un deep link configuré dans `app.json` (scheme).

📋 Bonnes pratiques

  • Mettre à jour les widgets en arrière-plan : utiliser `expo-background-fetch` pour rafraîchir le widget panier.
  • Limiter les actions dynamiques : ne pas dépasser 5 actions pour ne pas surcharger.
  • Widget panier : afficher un message "Panier vide" si aucun article.
  • Tester sur plusieurs versions iOS (iOS 14+ pour les widgets).
  • Fallback : proposer une alternative si les widgets ne sont pas supportés.

📘 Widget vs Quick Action : Les widgets affichent du contenu dynamique (iOS uniquement). Les Quick Actions Android sont des raccourcis d'actions.
