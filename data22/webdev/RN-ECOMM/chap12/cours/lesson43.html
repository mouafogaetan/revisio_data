📤 Partager un produit avec expo-sharing

La bibliothèque expo-sharing permet d'ouvrir la boîte de dialogue native de partage sur iOS et Android. Idéal pour envoyer un lien produit, une image ou un texte personnalisé.

# Installation

npx expo install expo-sharing

Exemple : Partager un produit avec titre, description et URL

import { Share } from 'react-native'; import * as Sharing from 'expo-sharing'; async function shareProduct(product) { const url = `https://monapp.com/produit/${product.id}`; const message = `Découvre ${product.name} à ${product.price}€ !\n${url}`; // Vérifier si le partage est disponible if (await Sharing.isAvailableAsync()) { await Sharing.shareAsync(url, { dialogTitle: 'Partager ce produit', mimeType: 'text/plain', UTI: 'public.url' }); } else { Alert.alert('Partage non disponible'); } }
💡 Astuce : Utilisez react-native-share si vous voulez personnaliser les plateformes (WhatsApp, Instagram, etc.)

🔗 Liens de parrainage (référencement)

Un lien de parrainage est unique pour chaque utilisateur. Il contient généralement un code promo ou un identifiant de parrain. On le génère côté serveur après création du compte.

🔐 Génération coté client (exemple)
const generateReferralLink = (userId) => { const baseUrl = "https://monapp.com/ref"; const code = btoa(userId + Date.now()).slice(0, 10); return `${baseUrl}?ref=${code}`; };

Attention : le code doit être validé par le backend pour éviter la fraude.

📊 Suivi des conversions

Lorsqu'un nouveau clique sur le lien, on enregistre le référant dans AsyncStorage et on l'envoie à l'API lors de l'inscription.

// stocker le paramètre 'ref' au chargement const urlParams = new URLSearchParams(window.location.search); const refCode = urlParams.get('ref'); if(refCode) AsyncStorage.setItem('referral', refCode);

✨ Exemple de lien final :

https://monapp.com/register?ref=XYZ789&productId=123

L'utilisateur qui s'inscrit avec ce lien offre une remise au parrain.

🎯 Deep Linking : ouvrir un produit directement

Le deep linking permet d'ouvrir votre application mobile à partir d'un lien web (ex: https://monapp.com/produit/42). Si l'app n'est pas installée, on redirige vers le store.

# Installation pour Expo

npx expo install expo-linking

# Configuration dans app.json

{
  "expo": {
    "scheme": "monapp"
  }
}

Ainsi un lien monapp://produit/123 ouvrira l'application.

📱 Gestion du lien entrant (dans App.js)

import * as Linking from 'expo-linking'; import { useEffect } from 'react'; function App() { useEffect(() => { const handleDeepLink = (event) => { const url = event.url; const { path, queryParams } = Linking.parse(url); if (path === 'produit') { navigation.navigate('Produit', { id: queryParams.id }); } }; Linking.addEventListener('url', handleDeepLink); // Vérifier le lien au démarrage Linking.getInitialURL().then(url => { if (url) handleDeepLink({ url }); }); }, []); }

🌐 Universal Links / App Links

Pour qu'un lien https://monapp.com/produit/1 ouvre directement l'application (et non le navigateur), on utilise les Universal Links sur iOS et App Links sur Android.

📱 iOS (Universal Links)
  • Héberger un fichier apple-app-site-association sur votre serveur (sans extension JSON).
  • Configurer le domaine dans Xcode / app.json.
  • Fichier exemple :
    { "applinks": { "apps": [], "details": [ { "appID": "TEAMID.com.monapp", "paths": ["*"] } ] } }
🤖 Android (App Links)
  • Fichier assetlinks.json sur votre site à la racine /.well-known/.
  • Vérification avec Digital Asset Links.
  • Exemple :
    [{"relation": ["delegate_permission/common.handle_all_urls"],"target": {"namespace": "android_app","package_name": "com.monapp","sha256_cert_fingerprints": ["XX:XX:..."]}}]
🧠 À retenir : Les liens universels permettent une expérience fluide : l'utilisateur clique sur un lien web, l'application s'ouvre (si installée), sinon le site web s'affiche.

⚙️ Cas pratique : partage avec lien profond

Imaginons un écran "Détail produit". L'utilisateur peut partager un lien qui, lorsqu'il est cliqué, ouvre directement ce produit dans l'application.

// Dans ProductScreen.js const handleShare = async () => { const deepLink = `monapp://produit/${product.id}`; const message = `✨ ${product.name} - ${product.price}€\nTélécharge l'app : ${deepLink}`; await Sharing.shareAsync(deepLink, { dialogTitle: 'Envoyer ce produit' }); }; // Dans App.js (gestion du lien) useEffect(() => { const handleUrl = ({ url }) => { if(url.includes('monapp://produit')) { const id = url.split('/').pop(); navigation.navigate('ProductDetail', { productId: id }); } }; Linking.addEventListener('url', handleUrl); }, []);

📱 Schéma de fonctionnement :

1. Utilisateur A partage le lien → 2. Utilisateur B clique sur le lien → 3. Appareil détecte le scheme "monapp://" → 4. Lance l'application → 5. Extrait l'ID produit et affiche la page correspondante.

📝 Bonnes pratiques & résumé

  • Toujours vérifier la disponibilité du partage avec isAvailableAsync().
  • Encoder les paramètres dans l'URL pour éviter les caractères spéciaux (encodeURIComponent).
  • Pour les liens de parrainage, générez un code unique et stockez-le dans un backend.
  • Deep linking : testez à la fois sur simulateur et vrai appareil (les schemes personnalisés marchent mieux en physique).
  • Fallback Web : si l'application n'est pas installée, redirigez vers une page web ou le store via Linking.openURL.

🏋️ Exercice pratique

Mise en situation :

Vous développez une application e-commerce "AfriMarket". Un utilisateur veut inviter un ami à télécharger l'application et bénéficier d'une réduction de 5%.

  1. Générez un lien de parrainage unique pour l'utilisateur courant (avec son ID).
  2. Proposez deux façons de partager : message texte et copie du lien.
  3. Lorsque l'ami clique sur le lien et installe l'application, le lien profond doit ouvrir la page d'accueil et appliquer la réduction lors de l'inscription.

💡 Indice : utilisez Linking.getInitialURL() pour récupérer le code de parrainage au premier lancement.
