📦 useState : le hook d’état

useState est un Hook React qui permet d’ajouter un état local à un composant fonctionnel.

import { useState } from 'react';

const [state, setState] = useState(valeurInitiale);
  • state : la valeur actuelle de l’état
  • setState : la fonction qui met à jour l’état (déclenche un re-rendu)
  • valeurInitiale : valeur au premier rendu
Important : Chaque appel à setState provoque un re-rendu du composant !

🔢 Exemple : Compteur simple

import { useState } from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <View style={styles.container}>
      <Text style={styles.countText}>Nombre de clics : {count}</Text>
      <Button title="Incrémenter" onPress={() => setCount(count + 1)} />
      <Button title="Réinitialiser" onPress={() => setCount(0)} />
    </View>
  );
}

À chaque clic, setCount met à jour l’état et le texte s’actualise automatiquement.

✏️ Input contrôlé

Un champ de saisie est contrôlé quand sa valeur est liée à un état React.

const [nom, setNom] = useState('');

<TextInput
  style={styles.input}
  value={nom}
  onChangeText={setNom}
  placeholder="Votre nom"
/>

<Text>Bonjour {nom || 'invité'} !</Text>
Avantage : La source de vérité est unique (l’état React), facile à valider ou transformer.

🛡️ L’immutabilité : ne jamais muter l’état directement

❌ À éviter :

// Ne faites PAS ceci !
const [user, setUser] = useState({ name: 'Jean', age: 25 });
user.age = 26;        // Mutation directe !
setUser(user);        // Ne déclenche PAS de re-rendu (même référence)

✅ Bonne pratique :

// Créez une nouvelle copie
setUser({ ...user, age: 26 });   // Nouvel objet → re-rendu

⚡ Mise à jour fonctionnelle (prevState)

Quand la nouvelle valeur dépend de l’ancienne, utilisez la forme fonctionnelle pour éviter les bugs asynchrones.

// ❌ Risqué si plusieurs mises à jour rapprochées
setCount(count + 1);
setCount(count + 1); // Ne donnera pas +2 car count n'a pas encore changé

// ✅ Sûr : utilise prevState
setCount(prev => prev + 1);
setCount(prev => prev + 1); // Donne bien +2
Règle : Si vous utilisez la valeur actuelle pour calculer la suivante, privilégiez la forme fonctionnelle.

📚 États complexes : objets et tableaux

Objet

const [form, setForm] = useState({ email: '', password: '' });

setForm({ ...form, email: 'test@mail.com' });

Tableau

const [items, setItems] = useState([]);

// Ajout
setItems([...items, newItem]);
// Suppression
setItems(items.filter(i => i.id !== id));
Toujours créer une nouvelle copie (spread operator, filter, map, etc.) pour que React détecte le changement.

📝 Exercice & Synthèse

À réaliser dans votre projet

  • Créez un compteur avec 3 boutons : +1, -1, Reset.
  • Créez un formulaire d’inscription avec nom, email et mot de passe (champs contrôlés).
  • Affichez en temps réel un résumé des informations saisies.
  • Utilisez la forme fonctionnelle de setState pour les incrémentations successives.

💡 Question : Pourquoi ne peut-on pas modifier directement user.age sans spread operator ?

Réponse : Parce que React compare les références des objets. Une mutation directe ne change pas la référence, donc React ne détecte pas le changement et ne re-rend pas.
