🧩 Composant réutilisable : définition

Un composant réutilisable est une brique indépendante que l'on peut utiliser plusieurs fois dans une application avec des données différentes.

Exemple concret : Un bouton personnalisé peut être utilisé 10 fois avec 10 couleurs et textes différents.
{/* Composant réutilisable */}
const CustomButton = ({ title, color }) => (
  <TouchableOpacity style={{ backgroundColor: color }}>
    <Text>{title}</Text>
  </TouchableOpacity>
);

// Utilisation 3 fois
<CustomButton title="Sauvegarder" color="blue" />
<CustomButton title="Annuler" color="gray" />

📦 Props : transmettre des données

Les props (properties) sont des arguments passés à un composant, comme des attributs HTML.

Props essentielles:
  • title : texte du bouton
  • onPress : fonction au clic
  • variant : style primaire/secondaire
  • disabled : désactiver
// Exemple d'utilisation
<Button
  title="Se connecter"
  onPress={() => alert('Bienvenue')}
  variant="primary" />
✅ Bon à savoir : Les props sont en lecture seule (immutables).

👶 children : imbriquer du contenu

children permet d'afficher ce qui se trouve entre les balises d'ouverture et fermeture d'un composant.

const Card = ({ children, style }) => (
  <View style={[styles.card, style]}>
    {children}
  </View>
);

// Utilisation
<Card>
  <Text>Mon contenu dynamique</Text>
  <Image source={...} />
</Card>

Très utile pour créer des Layouts (cart, modale, conteneur).

🏷️ Exemple pratique : Composant ProductCard

Créez une carte produit réutilisable qui affiche nom, prix et image.

const ProductCard = ({ name, price, imageUrl, onPress }) => (
  <TouchableOpacity onPress={onPress} style={styles.card}>
    <Image source={{ uri: imageUrl }} style={styles.image}/>
    <Text style={styles.name}>{name}</Text>
    <Text style={styles.price}>{price} FCFA</Text>
  </TouchableOpacity>
);
✅ Réutilisable ✅ Paramétrable ✅ Événement personnalisable

✨ Bonnes pratiques

  • Nommer clairement les props (title, onPress, isVisible).
  • Props par défaut avec defaultProps ou valeurs par défaut ES6.
  • Composition > Héritage : préférer children ou passer des composants en props.
  • Typage avec PropTypes ou TypeScript pour éviter les bugs.
Composer plusieurs petits composants réutilisables = application robuste et maintenable.

✏️ Exercice : Crée ton composant Input

À partir des connaissances acquises, décrivez ou codez un composant CustomInput réutilisable qui accepte :

  • placeholder (texte indicatif)
  • value et onChangeText (contrôlé)
  • secureTextEntry (optionnel)
  • icône à gauche ou à droite (facultatif)
Voir une solution possible 
const CustomInput = ({ placeholder, value, onChangeText, secure, icon }) => (
  <View style={{ flexDirection: 'row', alignItems: 'center', borderWidth:1, padding:8 }}>
    {icon && <Image source={icon} />}
    <TextInput 
      placeholder={placeholder}
      value={value}
      onChangeText={onChangeText}
      secureTextEntry={secure}
      style={{ flex:1 }}
    />
  </View>
);

📌 Récapitulatif

Les props rendent les composants dynamiques.
children permet des compositions flexibles.
Réutiliser des composants = moins de code, moins de bugs.
