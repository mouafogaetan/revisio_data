🛠️ Outils de débogage puissants

React Native Debugger

Outil standalone (Electron) qui combine : Redux DevTools, React DevTools, et inspecteur réseau.

# Installation (macOS)
brew install --cask react-native-debugger

# Lancement
"React Native Debugger" puis dans l'app : Cmd + D → "Debug JS"

Flipper (Meta)

Plateforme extensible pour logs, requêtes réseau, SQLite, layout inspector.

# macOS
brew install --cask flipper

# Dans React Native 0.62+ : déjà intégré
# Plugins utiles : React DevTools, Network, Hermes Debugger
🔍 Bridge errors : Quand JS ne communique pas avec le natif, Flipper affiche des logs détaillés.

✅ Tests : Jest + Testing Library

Assurez la fiabilité de vos composants et hooks.

1. Installation
npm install --save-dev @testing-library/react-native jest
2. Tester un composant simple
import { render, fireEvent } from '@testing-library/react-native';
import Counter from '../Counter';

test('incrémente le compteur', () => {
  const { getByText } = render();
  const button = getByText('Incrémenter');
  fireEvent.press(button);
  expect(getByText('Compteur : 1')).toBeTruthy();
});
3. Mocker une API (ex: fetch)
global.fetch = jest.fn(() => Promise.resolve({ json: () => Promise.resolve([]) }));
💡 Bonne pratique : tester le comportement utilisateur (ce qui s'affiche) et non les détails d'implémentation.

⚡ Problèmes courants : Bridge & Performance

🔌 Erreurs de bridge

  • Module natif non implémenté
  • Méthode appelée sur le mauvais thread
  • Callback non invoqué → fuite mémoire
// Debug: console.tron.log? 
// Utiliser Flipper + hermes debugger

🐌 Problèmes de perf

  • Re-rendus inutiles (React.memo)
  • Images non optimisées
  • Animations sur le thread JS
  • FlatList sans getItemLayout

🔧 Profiler avec Performance Monitor (React Native Debugger)

✅ Solution : useNativeDriver: true + React.memo()

Animated.timing(animValue, {
  toValue: 1,
  duration: 300,
  useNativeDriver: true  // ← crucial pour la fluidité
}).start();

🧠 Hermes & Chrome DevTools

Hermes est un moteur JavaScript optimisé pour React Native (par défaut sur Android).

Interface Chrome DevTools avec onglet "Performance"

🔧 Étapes :

  1. Lancer l'app avec npx react-native start --reset-cache
  2. Appuyer sur Cmd + D (Android : Cmd+M) → "Debug with Chrome"
  3. Ouvrir chrome://inspect → "Inspect" sur l'appareil
  4. Onglet "Performance" pour enregistrer les blocages
⚡ Hermes réduit le temps de démarrage et la consommation mémoire. Activez-le dans metro.config.js.

📋 Checklist quotidienne du développeur

🔲 Utiliser console.log avec des labels distincts (console.log('[Login]', data))
🔲 Explorer l'état Redux avec React Native Debugger
🔲 Surveiller les requêtes réseau (Flipper / onglet Network)
🔲 Mettre .only ou .skip dans Jest pour isoler des tests
🔲 Vérifier les avertissements (yellowbox) – ne pas les ignorer

🧪 Exercice rapide : Écrivez un test Jest pour un bouton "Afficher message" qui déclenche une Alert.alert() en utilisant jest.mock('react-native/Libraries/Alert/Alert').

Indice : jest.spyOn(Alert, 'alert')
import { Alert } from 'react-native';
jest.spyOn(Alert, 'alert');

test('affiche une alerte quand on presse le bouton', () => {
  const { getByText } = render();
  fireEvent.press(getByText('Alerte'));
  expect(Alert.alert).toHaveBeenCalledWith('Titre', 'Mon message');
});
