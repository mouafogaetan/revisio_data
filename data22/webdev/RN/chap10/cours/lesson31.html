Structure, composants réutilisables et variables d'environnement

🎯 Objectifs pédagogiques

  • ✓ Organiser son projet en dossiers (components, screens, navigation, hooks, services, utils)
  • ✓ Créer des composants réutilisables et typés (TypeScript)
  • ✓ Utiliser des variables d'environnement avec expo-constants

💡 À la fin de cette leçon, vous saurez organiser un projet React Native comme un développeur professionnel.

🤔 Pourquoi bien organiser son projet ?

Une bonne organisation vous permet de :

  • Retrouver facilement vos fichiers
  • Travailler en équipe sans conflits
  • Réutiliser vos composants
  • Maintenir et faire évoluer l'application

⚠️ Un projet mal organisé = perte de temps et bugs difficiles à corriger.

📂 Structure recommandée


src/
├── components/      # Composants UI réutilisables (Button, Card...)
├── screens/         # Écrans complets (HomeScreen, ProfileScreen...)
├── navigation/      # Configuration des navigateurs (Stack, Tabs...)
├── hooks/           # Hooks personnalisés (useAuth, useFetch...)
├── services/        # Appels API, logique métier
├── utils/           # Fonctions utilitaires (formatDate...)
├── constants/       # Constantes globales (couleurs, URLs...)
├── assets/          # Images, polices, icônes
└── App.js           # Point d'entrée principal

Cette organisation sépare clairement les responsabilités et facilite la maintenance.

🧩 Composants réutilisables

Un composant réutilisable est indépendant et configurable via ses props.

// components/PrimaryButton.js
import { TouchableOpacity, Text } from 'react-native';

export default function PrimaryButton({ title, onPress, disabled = false }) {
  return (
    <TouchableOpacity
      onPress={onPress}
      disabled={disabled}
      className={`py-3 px-6 rounded-xl ${disabled ? 'bg-gray-400' : 'bg-blue-600'}`}
    >
      <Text className="text-white font-bold text-center">{title}</Text>
    </TouchableOpacity>
  );
}

On peut ensuite l'importer partout : import PrimaryButton from '../components/PrimaryButton';

📝 TypeScript pour un typage sûr

TypeScript détecte les erreurs avant l'exécution et améliore l'autocomplétion.

// components/UserCard.tsx
interface UserCardProps {
  name: string;
  age: number;
  avatar?: string; // optionnel
}

const UserCard = ({ name, age, avatar }: UserCardProps) => {
  return (
    <View>
      <Text>{name} ({age} ans)</Text>
    </View>
  );
};

✅ Avantages : code auto-documenté, moins de bugs, meilleure collaboration.

🔐 Variables d'environnement

Stockez les clés API et URLs sensibles hors du code.

1. Installer expo-constants :

npx expo install expo-constants

2. Fichier .env à la racine :

API_URL=https://api.monsite.com
GOOGLE_MAPS_KEY=AZERTY123

3. Utilisation :

import Constants from 'expo-constants';

const apiUrl = Constants.expoConfig?.extra?.apiUrl;
// ou avec expo-constants dans app.config.js

✨ Bonnes pratiques

  • Nommer les fichiers en PascalCase pour les composants (ButtonPrimary.js)
  • Créer un fichier index.js dans chaque dossier pour exporter facilement
  • Utiliser des hooks personnalisés pour la logique métier
  • Séparer les styles (StyleSheet.create) ou utiliser Tailwind RN
  • Éviter les fichiers trop longs (max 200 lignes)

"Une bonne architecture aujourd'hui évite les maux de tête demain."

src/

├── screens/
│   ├── auth/
│   │   ├── LoginScreen.js
│   │   └── RegisterScreen.js
│   └── home/
│       └── HomeScreen.js
├── navigation/
│   ├── AppNavigator.js
│   └── AuthNavigator.js

│   └── useAuth.js

Cette arborescence est scalable pour des projets de toutes tailles.

🏋️ Exercice pratique

Créez un petit projet React Native avec l'organisation suivante :

  1. Un composant Card réutilisable (avec titre et contenu)
  2. Un écran HomeScreen qui affiche 3 cartes
  3. Une variable d'environnement pour l'URL d'une API fictive
  4. Un hook personnalisé useFetch pour récupérer des données

💡 Astuce : Commencez par créer la structure de dossiers, puis implémentez chaque partie une par une.
