🧠 Qu'est-ce que useReducer ?

useReducer est un hook React qui permet de gérer des états complexes avec une logique de transition basée sur des actions. Il est idéal quand l'état suivant dépend de l'état précédent ou quand plusieurs sous-valeurs sont impliquées.

const [state, dispatch] = useReducer(reducer, initialState);

🔁 Reducer

Une fonction pure qui reçoit l'état actuel et une action, et retourne le nouvel état.

📤 Dispatch

Fonction qui envoie une action au reducer pour déclencher une mise à jour.

Exemple typique : compteur, formulaire multi-champs, panier d'achat.

📱 Exemple : Compteur avec useReducer

// reducer.js
const initialState = { count: 0 };

function counterReducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return { count: 0 };
    default:
      return state;
  }
}

// Dans le composant
const [state, dispatch] = useReducer(counterReducer, initialState);

return (
  <View>
    <Text>Compteur: {state.count}</Text>
    <Button onPress={() => dispatch({type: 'increment'})} title="+" />
    <Button onPress={() => dispatch({type: 'decrement'})} title="-" />
  </View>
);

📌 Avantage : la logique de mise à jour est centralisée et facile à déboguer.

🌍 Combiner Context & useReducer

Pour partager l'état et le dispatch à toute l'application, on enrobe tout dans un Context.Provider.

// StoreContext.js
import React, { createContext, useReducer, useContext } from 'react';

const StoreContext = createContext();

const initialState = { user: null, todos: [] };

function mainReducer(state, action) {
  switch(action.type) {
    case 'LOGIN': return { ...state, user: action.payload };
    case 'ADD_TODO': return { ...state, todos: [...state.todos, action.payload] };
    default: return state;
  }
}

export function StoreProvider({ children }) {
  const [state, dispatch] = useReducer(mainReducer, initialState);
  return (
    <StoreContext.Provider value={{ state, dispatch }}>
      {children}
    </StoreContext.Provider>
  );
}

export const useStore = () => useContext(StoreContext);
Astuce : On utilise useStore partout pour lire ou modifier l'état global.

🏪 Store simple façon Redux

Avec useReducer + Context, on obtient une architecture proche de Redux sans les boilerplates. C'est parfait pour les apps de taille moyenne.

✅ Avantages

  • Pas de lib externe
  • Typage facile (TypeScript)
  • Logique prévisible

⚠️ Limites

  • Pas de middleware (sans effort)
  • Pas de devtools Redux
📦 Exemple de dispatch : dispatch({ type: 'ADD_TODO', payload: 'New task' })

🛒 Application pratique : Panier d'achat

const cartReducer = (state, action) => {
  switch(action.type) {
    case 'ADD_ITEM':
      return { ...state, items: [...state.items, action.payload], total: state.total + action.payload.price };
    case 'REMOVE_ITEM':
      const newItems = state.items.filter(item => item.id !== action.payload.id);
      const newTotal = newItems.reduce((sum, i) => sum + i.price, 0);
      return { ...state, items: newItems, total: newTotal };
    default:
      return state;
  }
};

// Utilisation dans un composant produit
dispatch({ type: 'ADD_ITEM', payload: { id: 1, name: 'Chemise', price: 5000 } });

Ce pattern est idéal pour les apps e-commerce ou de gestion.

⚖️ useState vs useReducer

CritèreuseStateuseReducer
Logique simple✔️ Très simple❌ Overkill
État avec sous-valeurs⚠️ Plus lourd✔️ Structuré
Transitions complexes❌ Risque d'erreur✔️ Prévisible
Partage globalNécessite Context+ Context = parfait
🎯 Règle : useState pour les states locaux simples, useReducer pour les états globaux ou avec beaucoup de transitions.
