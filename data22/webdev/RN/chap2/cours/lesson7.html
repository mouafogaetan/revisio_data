📝 Qu'est-ce que TextInput ?

TextInput est le composant React Native qui permet à l'utilisateur de saisir du texte. Il est l'équivalent de la balise <input> en HTML.

import { TextInput } from 'react-native';

const [texte, setTexte] = useState('');

<TextInput 
  value={texte}
  onChangeText={setTexte}
  placeholder="Écrivez quelque chose..."
  style={{ borderWidth: 1, borderColor: 'gray', padding: 10 }}
/>
💡 Note : Contrairement au web, onChangeText renvoie directement le texte saisi, pas un événement. C'est plus simple !

🎮 Composant contrôlé vs non contrôlé

✅ Contrôlé (recommandé)

const [valeur, setValeur] = useState('');
<TextInput 
  value={valeur}
  onChangeText={setValeur}
/>

React gère la valeur → re-rendu à chaque frappe

⚠️ Non contrôlé

const inputRef = useRef();
<TextInput 
  ref={inputRef}
  defaultValue="Texte initial"
/>

Le DOM gère la valeur. Moins de re-rendus mais moins de contrôle.

🌟 En pratique : Utilisez TOUJOURS un composant contrôlé dans React Native sauf cas très spécifiques.

⚙️ Props essentielles de TextInput

placeholder – Texte indicatif (grisé) quand le champ est vide
secureTextEntry – Cache la saisie (ex: mot de passe) → affiche •••
multiline – Permet plusieurs lignes (comme textarea)
keyboardType – Type de clavier : default, numeric, email-address, phone-pad
returnKeyType – Texte du bouton "Entrée" : done, go, next, search, send
onSubmitEditing – Fonction appelée quand l'utilisateur valide (appuie sur "Entrée")
maxLength – Limite le nombre de caractères
editable – Désactive la saisie (lecture seule)

📱 Exemples d'utilisation

🔐 Champ mot de passe

<TextInput 
  secureTextEntry={true}
  placeholder="Mot de passe"
  onChangeText={setMotDePasse}
/>

📄 Zone de texte multiligne

<TextInput 
  multiline
  numberOfLines={4}
  placeholder="Votre message..."
/>

📧 Clavier email + bouton "Suivant"

<TextInput 
  keyboardType="email-address"
  returnKeyType="next"
  onSubmitEditing={() => inputPasswordRef.current?.focus()}
/>

⌨️ Gérer le clavier

React Native propose l'API Keyboard pour contrôler le clavier programmatiquement.

import { Keyboard } from 'react-native';

// Cacher le clavier
Keyboard.dismiss();

// Écouter l'ouverture/fermeture
useEffect(() => {
  const showSubscription = Keyboard.addListener('keyboardDidShow', () => {
    console.log('Clavier ouvert');
  });
  const hideSubscription = Keyboard.addListener('keyboardDidHide', () => {
    console.log('Clavier fermé');
  });
  return () => {
    showSubscription.remove();
    hideSubscription.remove();
  };
}, []);
⚠️ Astuce : Utilisez ScrollView ou KeyboardAvoidingView pour que le clavier ne cache pas vos champs de saisie.

✅ Bonnes pratiques

  • Toujours utiliser un composant contrôlé avec value et onChangeText
  • Donner un placeholder clair pour guider l'utilisateur
  • Adapter le keyboardType au type de donnée attendue (email, téléphone, numérique)
  • Pour les formulaires longs, utiliser returnKeyType="next" pour naviguer entre les champs
  • Envelopper les formulaires dans KeyboardAvoidingView pour éviter que le clavier cache les inputs
  • Pour les mots de passe, toujours activer secureTextEntry
  • Limiter la taille avec maxLength si nécessaire

📱 Exemple de formulaire complet

<TextInput placeholder="Nom" returnKeyType="next" />
<TextInput placeholder="Email" keyboardType="email-address" returnKeyType="next" />
<TextInput placeholder="Mot de passe" secureTextEntry returnKeyType="done" />
