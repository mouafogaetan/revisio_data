Leçon 6 – TouchableOpacity, Pressable et retours visuels

Apprenez à rendre vos applications React Native interactives avec des gestes et des animations tactiles.

👆 TouchableOpacity 🤏 Pressable ✨ Feedbacks visuels

Objectifs de la leçon

  • ✓ Rendre des éléments interactifs avec TouchableOpacity.
  • ✓ Utiliser Pressable pour un contrôle plus fin des gestes.
  • ✓ Comprendre les retours visuels (highlight, opacity, ripple).
💡 Sur mobile, chaque interaction doit donner un retour visuel à l'utilisateur. C'est essentiel pour une bonne expérience utilisateur.

📱 Le défi du tactile

Sur mobile, il n'y a pas de curseur souris ni d'événement hover. Les utilisateurs tapent directement avec leurs doigts. React Native fournit des composants spécialisés pour gérer ces interactions.

🎯 Objectif :

Donner un retour immédiat à l'utilisateur : le bouton s'assombrit, une onde apparaît, un son (optionnel), etc. Cela indique que l'appareil a bien enregistré la pression.

🔘 TouchableOpacity : le bouton qui s'assombrit

C'est le composant interactif le plus utilisé. Quand on appuie dessus, son opacité diminue (par défaut à 0.2), donnant un retour visuel immédiat.

import { TouchableOpacity, Text, Alert } from 'react-native';

function MonBouton() {
  return (
    <TouchableOpacity 
      onPress={() => Alert.alert('Appuyé !')}
      activeOpacity={0.5}  // Opacité au toucher (0.2 par défaut)
    >
      <Text style={{ backgroundColor: 'blue', color: 'white', padding: 10 }}>
        Cliquez-moi
      </Text>
    </TouchableOpacity>
  );
}
✨ Props utiles : onPress, onLongPress, disabled, activeOpacity.
👆 Tester le clic (simulation)

🎮 Pressable : la flexibilité maximale

Introduit dans React Native 0.63, Pressable permet de réagir à différentes phases de la pression (début, fin, pression prolongée, etc.).

import { Pressable, Text } from 'react-native';

function BoutonAvance() {
  return (
    <Pressable
      onPress={() => console.log('Tap simple')}
      onLongPress={() => console.log('Appui long !')}
      onPressIn={() => console.log('Doigt posé')}
      onPressOut={() => console.log('Doigt levé')}
      style={({ pressed }) => ({
        backgroundColor: pressed ? '#ddd' : '#007AFF',
        padding: 10,
        borderRadius: 8,
      })}
    >
      <Text style={{ color: 'white' }}>Bouton dynamique</Text>
    </Pressable>
  );
}
🎯 Avantage : on peut styler dynamiquement selon l'état pressed, et gérer les retours haptiques (vibration).

✨ Différents types de retours visuels

🔘 Opacité (TouchableOpacity)

Le composant devient semi-transparent. Simple et efficace.

🎨 Highlight (TouchableHighlight)

Change la couleur de fond ou d'arrière-plan. Déprécié au profit de Pressable.

💧 Ripple (Android uniquement)

Effet d'onde qui se propage depuis le point de contact. Utiliser TouchableNativeFeedback (Android) ou Pressable avec android_ripple.

// Ripple sur Android avec Pressable
<Pressable 
  android_ripple={{ color: '#ccc', radius: 20 }}
  onPress={() => {}}
>
  <Text>Bouton avec ripple</Text>
</Pressable>

⚖️ Comparaison rapide

ComposantPoints fortsQuand l'utiliser ?

💡 Recommandation : utilisez TouchableOpacity pour la simplicité, Pressable pour les cas avancés.

📝 Exemple concret : Todo list interactive

function TodoItem({ task, onPress, onLongPress }) {
  return (
    <Pressable
      onPress={onPress}
      onLongPress={onLongPress}
      style={({ pressed }) => ({
        backgroundColor: pressed ? '#e2e8f0' : 'white',
        padding: 12,
        marginVertical: 4,
        borderRadius: 8,
      })}
    >
      <Text style={{ fontSize: 16 }}>{task.title}</Text>
    </Pressable>
  );
}

// Utilisation
<TodoItem 
  task={{ title: 'Acheter du lait' }}
  onPress={() => console.log('Tap : marquer comme fait')}
  onLongPress={() => console.log('Long press : supprimer')}
/>
Tap normal
Long press → supprimer

✅ Bonnes pratiques tactiles

  • Taille minimale des zones tactiles : 44x44 points (recommandation Apple/Google).
  • Donnez toujours un retour visuel (opacité, changement de couleur, ripple).
  • Évitez de mettre trop d'éléments cliquables trop proches (doigts peu précis).
  • Utilisez onLongPress pour des actions destructrices (supprimer, modifier).
  • Testez sur un vrai téléphone, pas seulement sur l'émulateur.
🧪 À tester : activez les indicateurs de touches dans les options développeur Android pour visualiser où l'utilisateur tape.

🧪 Exercice : Créez un bouton personnalisé

Utilisez Pressable pour créer un bouton qui change de couleur et de taille lorsqu'on appuie dessus, et qui fait vibrer l'appareil (indice : Vibration.vibrate()).

import { Vibration } from 'react-native';

Vibration.vibrate(50); // vibre 50ms

📌 Solution 
function SuperButton({ title, onPress }) {
  const handlePress = () => {
    Vibration.vibrate(50);
    onPress();
  };
  return (
    <Pressable
      onPress={handlePress}
      style={({ pressed }) => ({
        transform: [{ scale: pressed ? 0.96 : 1 }],
        backgroundColor: pressed ? '#4f46e5' : '#6366f1',
        padding: 12,
        borderRadius: 8,
      })}
    >
      <Text style={{ color: 'white', textAlign: 'center' }}>
        {title}
      </Text>
    </Pressable>
  );
}
