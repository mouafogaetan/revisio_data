📱 React Native, c’est quoi ?

React Native est un framework open-source créé par Meta (Facebook) en 2015. Il permet de développer des applications mobiles natives pour iOS et Android en utilisant JavaScript et React.

🔑 Un seul code, deux plateformes !

Avec React Native, vous écrivez une seule base de code qui fonctionne à la fois sur iOS et Android, sans perdre les performances d’une application native.

🖥️ React Native vs React (web)

React (web)

  • Tourne dans un navigateur
  • Utilise le DOM (div, span, h1...)
  • CSS classique / modules CSS / styled-components
  • Optimisé pour le web

React Native

  • Tourne sur mobile (iOS/Android)
  • Utilise des composants natifs (View, Text, Image...)
  • Style via StyleSheet (propriétés CSS-like)
  • Accès aux fonctionnalités natives (caméra, GPS, capteurs)
Astuce : Les connaissances React (props, state, hooks, composants) sont directement réutilisables en React Native !

⚡ React Native vs Solutions hybrides

// Applications hybrides (Cordova/Ionic)
// = WebView qui affiche du HTML/CSS/JS
// = Performances moins bonnes, rendu web
  • React Native : rendu avec des composants natifs (UI native, performances proches du natif)
  • Hybrides : affichent du contenu web dans une WebView (plus lent, moins d’intégration native)

Résultat : React Native est plus performant et offre une meilleure expérience utilisateur que les solutions hybrides classiques.

🌉 Le Bridge : cœur de React Native

React Native utilise un pont (bridge) pour faire communiquer le code JavaScript et le code natif (Java/Kotlin pour Android, Objective-C/Swift pour iOS).

📊 Avantages & Limites

Avantages

  • Un seul code pour iOS et Android
  • Performance native (UI native)
  • Réutilise les compétences React
  • Hot reloading (rechargement à chaud)
  • Large communauté et écosystème

Limites

  • Quelques différences entre iOS et Android
  • Performance du Bridge pour animations complexes (mais Reanimated 2 améliore)
  • Taille de l’APK plus élevée
  • Nécessite des connaissances natives pour modules personnalisés

📝 Exercice & Synthèse

Questions de compréhension

  • Expliquez la principale différence entre React Native et une application hybride (Cordova/Ionic).
  • À quoi sert le Bridge dans React Native ?
  • Citez trois composants React Native qui n’existent pas en React web.

Réponses :
1) React Native utilise des composants natifs, les hybrides affichent du web dans une WebView.
2) Le Bridge permet la communication asynchrone entre JavaScript et le code natif iOS/Android.
3) View, Text, ScrollView, FlatList, Image, TouchableOpacity, etc.

À retenir

React Native = natif avec JavaScript et React.
