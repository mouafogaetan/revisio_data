📏 Dimensions fixes : width et height

Comme en CSS, on peut définir des dimensions fixes en pixels (width: 100, height: 200).

import { View, Text } from 'react-native';

<View style={{ width: 200, height: 200, backgroundColor: 'blue' }}>
  <Text style={{ color: 'white' }}>Boîte carrée</Text>
</View>

💡 Note : En React Native, les dimensions sont en points indépendants de la densité (dp), pas en pixels physiques.

📊 Dimensionnement avec flex

La propriété flex permet de répartir l'espace restant entre les éléments.

<View style={{ flex: 1, backgroundColor: 'red' }} />   // 1 part
<View style={{ flex: 2, backgroundColor: 'blue' }} />  // 2 parts
<View style={{ flex: 1, backgroundColor: 'green' }} /> // 1 part

🔢 Règle de calcul :

Un conteneur avec flex: 1 prend tout l'espace disponible.
Si plusieurs enfants ont flex: 1, ils se partagent l'espace équitablement.
Avec flex: 2, un élément prend deux fois plus d'espace qu'un élément avec flex: 1.

📍 Position relative

Avec position: 'relative' (valeur par défaut), l'élément est dans le flux normal. Les propriétés top, right, bottom, left le décalent par rapport à sa position normale.

<View style={{ position: 'relative', top: 20, left: 10 }}>
  <Text>Décalé de 20px vers le bas et 10px vers la droite</Text>
</View>
⚠️ Important : Les autres éléments ne sont PAS réarrangés. L'espace d'origine reste réservé.

🎯 Position absolute

position: 'absolute' retire l'élément du flux normal. Il se positionne par rapport à son parent le plus proche en position relative/absolute, sinon par rapport à l'écran.

<View style={{ position: 'relative', height: 200, backgroundColor: '#ddd' }}>
  <View style={{ position: 'absolute', bottom: 10, right: 10, backgroundColor: 'blue' }}>
    <Text style={{ color: 'white' }}>Coin bas droit</Text>
  </View>
</View>
🎨 Cas d'usage : Badges, tooltips, overlay, boutons flottants, images de fond.

🔄 zIndex - L'ordre d'affichage

zIndex détermine quel élément apparaît par-dessus les autres. Plus le zIndex est élevé, plus l'élément est visible en avant-plan.

<View style={{ width: 100, height: 100, backgroundColor: 'red', zIndex: 1 }} />
<View style={{ width: 100, height: 100, backgroundColor: 'blue', marginTop: -50, zIndex: 2 }} />
// L'élément bleu apparaît PAR-DESSUS le rouge (zIndex plus élevé)
💡 Rappel : Sans zIndex, l'ordre d'affichage suit l'ordre dans le code (dernier élément = dessus).

✅ Bonnes pratiques

  • Privilégier flex et les dimensions relatives pour un design adaptatif
  • Utiliser width/height fixes pour des éléments comme des icônes ou boutons
  • Le position: absolute est idéal pour les éléments superposés (badges, flottants)
  • Toujours définir un parent relative si l'enfant absolute doit être positionné par rapport à ce parent
  • Éviter les valeurs négatives de zIndex (peuvent causer des bugs sur certaines versions)
  • Tester les positions sur plusieurs tailles d'écran (émulateurs ou appareils réels)

📱 Exemple : Badge de notification

<View style={{ position: 'relative' }}>
  <Image source={require('./icone.png')} style={{ width: 40, height: 40 }} />
  <View style={{ position: 'absolute', top: -5, right: -5, backgroundColor: 'red', borderRadius: 10, padding: 4 }}>
    <Text style={{ color: 'white', fontSize: 10 }}>3</Text>
  </View>
</View>
