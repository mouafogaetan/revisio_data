Partie 1 : Questions théoriques – Flexbox, Navigation & API (20 points)

Partie 2 : Mise en page avec Flexbox (React Native) – 25 points

🎨 Exercice : On souhaite réaliser une carte produit pour une application e-commerce mobile. Le design doit respecter les contraintes suivantes :

  • Une View principale avec fond blanc, coins arrondis (borderRadius: 16), ombre légère.
  • Une image en haut (largeur 100%, hauteur 150).
  • Le titre du produit en gras, suivi du prix en rouge.
  • Un conteneur flex en ligne (row) avec un bouton "Acheter" (TouchableOpacity) et un icône "❤️" alignés à droite.
  • Utiliser flexDirection, justifyContent, alignItems pour aligner les éléments correctement.

✍️ Écrire le code du composant fonctionnel ProductCard avec StyleSheet (React Native).

Rappel : React Native utilise flexbox avec flexDirection: 'column' par défaut. Pensez à bien gérer l'espacement.

Partie 3 : Navigation (Stack + Tabs) – 25 points

3.1 Configuration Stack (12 pts)

Créez une Stack Navigator avec deux écrans : HomeScreen et DetailScreen. Dans HomeScreen, ajoutez un bouton qui navigue vers DetailScreen en passant un paramètre produitId: 42. Dans DetailScreen, récupérez et affichez ce paramètre.

3.2 Bottom Tab Navigator (13 pts)

Ajoutez une navigation par onglets contenant deux écrans : Accueil (icône maison) et Profil (icône utilisateur). Chaque onglet doit être associé à un composant simple. Décrivez le code et la structure.

Organigramme de navigation (méthode Mermaid) :

graph TD A[Root Navigator] --> B(Stack Navigator) B --> C[HomeScreen] B --> D[DetailScreen] A --> E(Bottom Tabs) E --> F[AccueilTab] E --> G[ProfilTab]

Diagramme illustrant la combinaison Stack + Tabs. Justifiez pourquoi combiner ces navigateurs dans une app mobile.

Partie 4 : Appels API & AsyncStorage – 20 points

On souhaite afficher une liste de posts depuis https://jsonplaceholder.typicode.com/posts. Implémentez un composant PostsList utilisant useState, useEffect et FlatList pour :

  • Charger les données via fetch() au montage (loading state).
  • Afficher un indicateur d'activité (ActivityIndicator) pendant le chargement.
  • En cas d'erreur, afficher un message d'erreur convivial.
  • Chaque élément de liste doit afficher le titre du post.
