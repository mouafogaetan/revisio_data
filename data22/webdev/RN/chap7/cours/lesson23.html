🎯 Objectifs de la leçon

Installer expo-camera ou react-native-camera
Prendre des photos et les afficher dans l'application
Accéder à la galerie avec expo-image-picker
Gérer les permissions (Android & iOS)
💡 À la fin, vous saurez intégrer l'appareil photo et la galerie comme dans les applications professionnelles (WhatsApp, Instagram...)

📦 Installation des packages

Avec Expo (recommandé pour débuter) :

expo install expo-camera expo-image-picker

Pour React Native CLI :

npm install react-native-camera react-native-image-picker
Sur Android, pensez à ajouter les permissions dans android/app/src/main/AndroidManifest.xml : <uses-permission android:name="android.permission.CAMERA" />

📷 Prendre une photo (Camera)

Exemple de composant simple :

import { Camera } from 'expo-camera';
const [hasPermission, setHasPermission] = useState(null);
const [photo, setPhoto] = useState(null);
let cameraRef = useRef();

const takePicture = async () => {
  if (cameraRef.current) {
    let options = { quality: 0.5, base64: true };
    let data = await cameraRef.current.takePictureAsync(options);
    setPhoto(data.uri);
  }
};
// Rendu : {hasPermission ? <Camera ref={cameraRef} /> : <Text>Pas de permission</Text>}
Toujours demander la permission avant d'utiliser la caméra.

🖼️ Choisir une image depuis la galerie

Utilisation de launchImageLibraryAsync

import * as ImagePicker from 'expo-image-picker';

const pickImage = async () => {
  let result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    allowsEditing: true,
    aspect: [4,3],
    quality: 0.8
  });
  if (!result.canceled) {
    setSelectedImage(result.assets[0].uri);
  }
};
✨ Bonus : allowsEditing permet de recadrer l’image avant validation.

🔐 Permissions : étape cruciale

Sur mobile, l'utilisateur doit accorder l'accès à la caméra et au stockage.

useEffect(() => {
  (async () => {
    const { status } = await Camera.requestCameraPermissionsAsync();
    setHasPermission(status === 'granted');
  })();
}, []);
Android : permissions au moment de l'exécution
iOS : ajouter description dans Info.plist

⭐ Astuces et bonnes pratiques

  • 🔸 Toujours compresser les images (quality: 0.5) pour éviter de surcharger l’app.
  • 🔸 Gérer l’état de chargement (loading) pendant la capture ou la sélection.
  • 🔸 Penser à libérer la caméra quand le composant est démonté (cleanup).
  • 🔸 Utiliser MediaTypeOptions.All pour autoriser vidéos et photos dans la galerie.
  • 🔸 Tester sur un vrai mobile : l’émulateur Android ne dispose pas toujours d’une caméra fonctionnelle.
Pour un projet plus robuste, combinez ces fonctionnalités avec AsyncStorage pour sauvegarder les photos localement.
