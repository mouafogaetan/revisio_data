Stockage sécurisé, gestion de session et logout avec SecureStore & AsyncStorage

🎯 Objectifs de la leçon

1.

Différencier SecureStore (données sensibles) et AsyncStorage (données non critiques).

2.

Implémenter une authentification complète (login / logout) avec stockage du token.

3.

Sécuriser les informations utilisateur (mot de passe, JWT) via `expo-secure-store`.

4.

Gérer la persistance de session et la déconnexion propre.

💡 À retenir : Un token d’authentification est comme une clé de maison — à garder dans un coffre (SecureStore) et non sous le paillasson (AsyncStorage).

📦 Stockage local : deux mondes

AsyncStorage

Stockage simple, persistant, non chiffré. Idéal pour : thème sombre, préférences, données publiques.

await AsyncStorage.setItem('@theme', 'dark');
const theme = await AsyncStorage.getItem('@theme');

SecureStore (Expo)

Stockage chiffré (clé AES-256). Parfait pour tokens JWT, mots de passe, données biométriques.

import * as SecureStore from 'expo-secure-store';
await SecureStore.setItemAsync('userToken', 'jwt_secret_123');
const token = await SecureStore.getItemAsync('userToken');

🔐 Règle d'or : Jamais de données sensibles dans AsyncStorage. Sur Android, les bases AsyncStorage sont lisibles en root / debug.

🚪 Authentification pas-à-pas

Structure typique : écran de connexion → stockage token → accès aux écrans privés

1. Installation
npx expo install expo-secure-store @react-native-async-storage/async-storage
2. Fonction Login (simulée avec API)
const handleLogin = async (email, password) => {
  try {
    const response = await fetch('https://api.example.com/login', {
      method: 'POST',
      body: JSON.stringify({ email, password })
    });
    const { token } = await response.json();
    await SecureStore.setItemAsync('authToken', token);
    setIsLoggedIn(true);
  } catch (error) {
    alert('Échec connexion');
  }
};
3. Logout : suppression sécurisée
const handleLogout = async () => {
  await SecureStore.deleteItemAsync('authToken');
  setIsLoggedIn(false);
};

🔄 Persistance de session

Au lancement de l'app, on vérifie si un token existe dans SecureStore pour rediriger l'utilisateur.

// App.js ou contexte d'authentification
useEffect(() => {
  const checkToken = async () => {
    const token = await SecureStore.getItemAsync('authToken');
    if (token) {
      // Token valide → accès direct à l'écran principal
      setIsLoading(false);
      setUserToken(token);
    } else {
      setIsLoading(false);
    }
  };
  checkToken();
}, []);

✅ Bonne pratique :

Combiner SecureStore avec un état global (React Context ou Redux) pour que toute l'application sache si l'utilisateur est connecté.

📱 Sur iOS, SecureStore utilise le keychain ; sur Android, le Keystore system.

⚠️ Gérer les cas d'échec

SecureStore peut échouer (simulateur, permissions). Toujours entourer d'un try/catch.

const saveSecureData = async (key, value) => {
  try {
    await SecureStore.setItemAsync(key, value);
  } catch (error) {
    console.error('Erreur stockage sécurisé', error);
    // Fallback : avertir l'utilisateur ou utiliser AsyncStorage (déconseillé)
  }
};
🚫 À éviter
  • Stocker mot de passe en clair
  • Token dans AsyncStorage
  • Ignorer les erreurs de récupération
✅ Recommandé
  • Utiliser SecureStore pour JWT
  • Ajouter un écran de chargement
  • Définir un timeout d'expiration
💡 MathJax : La sécurité repose sur le chiffrement symétrique $E(K, P) = C$ où la clé $K$ est gérée par le système.

🧪 Mini-projet : Application sécurisée

Consigne :

Créer deux écrans : LoginScreen et ProfileScreen. À la validation du login (email: user@exemple.com, mdp: "cameroun2025"), stocker un token fictif dans SecureStore. ProfileScreen doit lire le token et afficher "Bienvenue 👋". Un bouton "Déconnexion" supprime le token et retourne au login.

📌 Indice : utiliser useState + useEffect pour vérifier la présence du token au chargement du composant principal.

// Dans ProfileScreen
const [user, setUser] = useState(null);
useEffect(() => {
  const loadToken = async () => {
    const token = await SecureStore.getItemAsync('authToken');
    if(token) setUser({ name: 'Étudiant Cameroun' });
  };
  loadToken();
}, []);
