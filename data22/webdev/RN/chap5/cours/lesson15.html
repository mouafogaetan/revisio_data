📱 Chapitre 5 - Navigation

Pourquoi React Navigation ?

Sur mobile, les applications sont composées de plusieurs écrans (accueil, profil, détails). React Navigation est LA bibliothèque officielle et la plus populaire pour gérer la navigation dans React Native.

✔️ Avantages

  • Navigation fluide et gestuelle (comme iOS/Android)
  • Gestion automatique de l'historique (pile d'écrans)
  • Personnalisation du header / transition
  • Compatible avec Expo et React Native CLI

Principe de stack navigation : chaque nouvel écran s'empile

📦 Installation des dépendances

Avant de naviguer, il faut installer les paquets nécessaires.

# Avec npm
npm install @react-navigation/native @react-navigation/stack

# Dépendances obligatoires pour React Navigation
npm install react-native-screens react-native-safe-area-context

# Pour Expo (recommandé)
expo install @react-navigation/native @react-navigation/stack
expo install react-native-screens react-native-safe-area-context
⚠️ Important : Enveloppez toute votre application avec NavigationContainer.

🧩 Créer un Stack Navigator

Exemple basique avec deux écrans : Accueil et À propos.

import * as React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createStackNavigator } from '@react-navigation/stack';
import AccueilScreen from './screens/AccueilScreen';
import AProposScreen from './screens/AProposScreen';

const Stack = createStackNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator initialRouteName="Accueil">
        <Stack.Screen name="Accueil" component={AccueilScreen} />
        <Stack.Screen name="APropos" component={AProposScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}
📌 initialRouteName : définit l'écran affiché au démarrage.
📌 Stack.Screen : chaque écran a un nom unique et un composant.

🚀 navigation.navigate()

Dans un composant écran, on reçoit la prop navigation. On appelle navigate('NomEcran') pour changer de page.

// Dans AccueilScreen.js
function AccueilScreen({ navigation }) {
  return (
    <View>
      <Text>Page d'accueil</Text>
      <Button 
        title="Aller à A propos" 
        onPress={() => navigation.navigate('APropos')}
      />
    </View>
  );
}
✅ Retour automatique avec ← header
✅ navigation.goBack() pour revenir

📨 Passer des paramètres entre écrans

On peut envoyer des données (ID utilisateur, titre, etc.) via le deuxième argument de navigate.

📤 Envoi (depuis écran A)

navigation.navigate('Details', {
  productId: 42,
  productName: 'React Native Guide'
});

📥 Réception (écran B)

function DetailsScreen({ route }) {
  const { productId, productName } = route.params;
  return <Text>{productName} - ID {productId}</Text>
}
💡 Astuce: Pensez à gérer le cas où les paramètres sont optionnels en utilisant route.params?.valeur.

🎨 Mise en pratique : mini-app

Scénario : Une liste d'articles, on clique sur un article pour voir les détails.

// Écran Liste
<FlatList 
  data={articles}
  renderItem={({item}) => (
    <TouchableOpacity onPress={() => navigation.navigate('Detail', { 
        id: item.id, 
        title: item.title 
      })}>
      <Text>{item.title}</Text>
    </TouchableOpacity>
  )}
/>

// Écran Detail
function Detail({ route }) {
  const { id, title } = route.params;
  return <Text>Affichage de {title} (id: {id})</Text>
}

✨ Récapitulatif & conseils

✔️
Toujours envelopper l'app avec NavigationContainer – c'est le fournisseur de contexte.
✔️
Nommez clairement vos screens (ex: "Home", "Profile", "Settings") pour éviter les confusions.
✔️
Pour aller en arrière : soit le bouton natif du header, soit navigation.goBack().
✔️
Vérifiez toujours les paramètres : if (route.params) {...} pour éviter les erreurs.

🧠 À retenir : navigation.navigate() + route.params = la clé d'une application mobile fluide !
