🧠 Mémoire virtuelle & Algorithmes de remplacement

Comment un système peut exécuter des programmes plus gros que la RAM physique

📘 Leçon 12 ⏱️ Durée : 25 min 🎯 Niveau : Intermédiaire

🎯 Objectifs de la leçon

  • ✓ Comprendre le concept de mémoire virtuelle : espace d'adressage plus grand que la RAM physique.
  • ✓ Expliquer le mécanisme de page fault et la gestion des swaps.
  • ✓ Maîtriser les algorithmes de remplacement : FIFO, LRU, Second Chance, Clock.
  • ✓ Calculer le nombre de défauts de page pour une séquence d'accès donnée.

💾 Qu'est-ce que la mémoire virtuelle ?

La mémoire virtuelle permet à un processus de disposer d'un espace d'adressage plus grand que la mémoire physique (RAM).

🔹 Adresse logique (virtuelle)

Celle que voit le processus (ex: 0x0000 à 0xFFFFFF...).

🔹 Adresse physique

Celle réelle dans les barrettes RAM.

💡 Idée clé : Seules les pages utilisées actuellement sont en RAM. Le reste est stocké sur le disque (swap).

⚠️ Page fault & Swapping

Lorsqu'un processus accède à une page qui n'est pas en RAM, le processeur déclenche une page fault (défaut de page). L'OS doit alors :

  1. Trouver un cadre de page libre en RAM.
  2. Si aucun cadre libre, appliquer un algorithme de remplacement pour en évincer une.
  3. Charger la page manquante depuis le disque (swap).
  4. Mettre à jour la table des pages.
  5. Reprendre l'exécution du processus.
// Exemple simplifié
if (page présente en RAM) {
    accès direct;
} else {
    page_fault_handler();
    charger_page_depuis_swap();
    reprendre_instruction();
}

📌 FIFO (First In, First Out)

Principe : La page chargée le plus tôt est évincée en premier (comme une file d'attente).

Exemple : 3 cadres, séquence d'accès : 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5

➜ Remplacement : 1 → 4, 2 → 1, 3 → 2, 4 → 5, ...

⚠️ Anomalie de Belady : Augmenter le nombre de cadres peut augmenter le nombre de défauts de page (contre-intuitif).

⏱️ LRU (Least Recently Used)

Principe : On évince la page qui n'a pas été utilisée depuis le plus longtemps (meilleure approximation de l'optimal).

Implémentation possible : liste doublement chaînée ou utilisation d'un compteur (bits d'utilisation).

✅ Avantage : Bonne performance en pratique, évite l'anomalie de Belady.

❌ Inconvénient : Coût matériel/logiciel élevé pour maintenir l'ordre (souvent approchée par l'algorithme de la seconde chance).

🕰️ Second Chance (Algorithme de l'horloge)

Chaque page possède un bit d'utilisation (reference bit). Au moment de chercher une page à évincer :

  • Si le bit = 1, on lui donne une "seconde chance" : on met le bit à 0 et on passe à la suivante.
  • Si le bit = 0, la page est évincée.

🧠 Variante améliorée : algorithme Clock (implémentation circulaire de Second Chance).

while (true) {
    if (current_page.reference_bit == 0) {
        page_a_remplacer = current_page;
        break;
    } else {
        current_page.reference_bit = 0;
        passer_a_la_page_suivante();
    }
}
