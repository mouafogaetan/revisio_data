🎯 Objectifs de la leçon

À la fin de ce cours, vous serez capable de :

  • ✓ Comprendre comment le CPU communique avec les périphériques.
  • ✓ Distinguer le polling, les interruptions et le DMA.
  • ✓ Expliquer le rôle des pilotes (drivers) et de la couche d'abstraction.
  • ✓ Connaître les composants matériels : contrôleurs, bus, registres.
  • ✓ Analyser un scénario d'entrée/sortie (lecture clavier → application).
💡 Pourquoi c'est important ? Les E/S sont un goulot d'étranglement. Bien les gérer, c'est garantir la réactivité du système.

🔌 Architecture matérielle des E/S

Un ordinateur n'est pas seulement un CPU et de la RAM. Les périphériques sont connectés via des bus et gérés par des contrôleurs.

Schéma simplifié :
   CPU  <--- Bus système --->  RAM
    |
    | (Bus d'E/S)
    v
┌──────────────┐
│ Contrôleur   │   ← Pilote
│ disque dur   │
└──────────────┘
┌──────────────┐
│ Contrôleur   │
│ USB          │ → Clavier, souris
└──────────────┘
🔹 Contrôleur (controlleur) : petit processeur spécialisé qui gère un périphérique.
🔹 Bus : ensemble de lignes de communication (données, adresses, commandes).

✨ L'OS ne dialogue pas directement avec le matériel, mais avec le contrôleur via ses registres.

🔄 Méthode 1 : Polling (scrutation active)

Le CPU interroge en boucle le périphérique pour savoir s'il a des données prêtes.

tant que (pas de donnée) {
   vérifier registre d'état du périphérique;
}
lire la donnée;

📌 Exemple : lecture d'un clavier en polling.

✔️ Avantages : simple à implémenter, aucun matériel spécial.
❌ Inconvénients : gaspille du temps CPU (attente active), inefficace pour périphériques lents.

Aujourd'hui, le polling est utilisé pour des périphériques très rapides (ex: lecture d'un registre matériel) ou lorsque la latence doit être ultra-faible.

⚡ Méthode 2 : Interruptions (IRQ)

Le périphérique signale au CPU qu'il a besoin d'attention → interruption matérielle. Le CPU suspend son travail, exécute un gestionnaire d'interruption (ISR), puis reprend.

📡 Déroulement typique :

  1. L'utilisateur tape une touche.
  2. Le contrôleur du clavier envoie une interruption au CPU.
  3. Le CPU sauvegarde son contexte (registres).
  4. Il exécute le gestionnaire d'interruption du clavier (lecture de la touche).
  5. Il restaure le contexte et continue le programme interrompu.
✔️ Efficace : pas de gaspillage CPU, réactif.
⚠️ Contrainte : nécessite une table de vecteurs d'interruption (IDT).

Les interruptions logicielles (trap, syscall) existent aussi, déclenchées par des instructions comme int 0x80 ou syscall.

💾 DMA : Libérer le CPU des transferts lourds

Pour les périphériques rapides (disque dur, carte réseau), copier chaque octet via le CPU est coûteux. Le DMA permet au contrôleur d'écrire/lire directement la RAM sans passer par le CPU.

🔁 Fonctionnement DMA :

  1. Le CPU programme le contrôleur DMA (adresse source, destination, taille).
  2. Le contrôleur DMA transfère les données pendant que le CPU travaille sur autre chose.
  3. À la fin du transfert, le contrôleur envoie une interruption au CPU.

✅ Bénéfice : énorme gain de performance pour les lectures/écritures disque ou les trames réseau. Exemple : copie d'un fichier ou streaming vidéo.

Sans DMA : CPU → registres → RAM → ... Avec DMA : périphérique → RAM (sans CPU).

🔧 Les pilotes (drivers)

Chaque périphérique a ses spécificités. Le pilote est un module logiciel qui traduit les appels génériques de l'OS en commandes spécifiques au matériel.

Application → API standard (ex: ReadFile) 
              → OS (gestionnaire E/S)
              → Pilote du disque dur
              → Contrôleur disque
              → Disque physique
  • Windows : modèles WDM, KMDF.
  • Linux : modules chargeables (.ko) dans /lib/modules.
  • Couche d'abstraction matérielle (HAL) : isole l'OS des différences matérielles.

⚡ Sans pilote, le système ne peut pas utiliser le périphérique.

📊 Tableau comparatif : Polling, Interruption, DMA

💡 A retenir : Les OS modernes combinent interruptions + DMA pour les E/S de masse, et utilisent le polling seulement pour des cas particuliers (ex: latence extrême).

🗄️ Buffering & Spooling

Les tampons (buffers) lissent les différences de débit entre le producteur et le consommateur de données.

🧴 Buffer (tampon)

Zone mémoire temporaire. Exemple : lecture disque par blocs avant d'être traitée.

🖨️ Spooling

Simultanéité périphérique lent : les données sont mises en file d'attente (ex: impression). Acronyme : Simultaneous Peripheral Operation On-Line.

Exemple spooling imprimante :
Application 1 → fichier temporaire → spooler (gestionnaire)
Application 2 → fichier temporaire → spooler
Imprimante ← spooler (un fichier après l'autre)

Le spooling est géré par un démon (ex: cups pour Linux).

⌨️ Étude de cas : Appui sur une touche

  1. L'utilisateur presse la touche 'A'.
  2. Le contrôleur du clavier encode le scan code.
  3. Il envoie une interruption (IRQ 1 typiquement).
  4. Le CPU exécute l'ISR du clavier (dans le noyau).
  5. Le scan code est converti en caractère (ex: 'a' ou 'A' selon Maj).
  6. Le caractère est placé dans un buffer du driver.
  7. L'application en premier plan récupère la saisie via l'API (ex: ReadConsole).
🧠 Pour aller plus loin : Sous Linux, vous pouvez voir les interruptions avec cat /proc/interrupts.

🎯 Cet exemple illustre la chaîne : matériel → interruption → driver → OS → application.

Résumé de la leçon

Les entrées/sorties sont gérées par une combinaison de polling, interruptions et DMA. Les pilotes offrent l'abstraction.

✔️ Polling : attente active, simple mais inefficace.
✔️ Interruption : notification asynchrone, réactive.
✔️ DMA : transfert direct mémoire ↔ périphérique.
✔️ Driver : logiciel spécifique au matériel.

📚 Pour approfondir : chapitre "Input/Output" du livre Operating System Concepts (Silberschatz), section "Interrupts" et "DMA".
