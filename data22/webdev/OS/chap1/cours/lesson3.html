Concept fondamental

Qu’est-ce qu’un processus ?

📌 Définition : Un processus est un programme en cours d’exécution. Il ne se limite pas au code source : il inclut son contexte d’exécution (compteur programme, pile, registres, données).

📄 Programme (passif)
  • Fichier exécutable stocké sur le disque
  • Statique : une suite d’instructions
  • Ne consomme pas de ressources CPU
⚙️ Processus (actif)
  • Programme chargé en mémoire (RAM)
  • Possède un état, un contexte, des ressources
  • Géré par l’ordonnanceur du système
💡 Exemple concret : Le fichier "calculatrice.exe" sur le disque est un programme. Quand vous double-cliquez, le système crée un processus avec son propre espace mémoire.

📋 Le PCB (Process Control Block)

La « carte d’identité » du processus

Le noyau maintient pour chaque processus une structure de données appelée PCB. Elle contient toutes les informations nécessaires pour suspendre puis reprendre l’exécution.

🔢 Identifiant (PID)

Numéro unique attribué à chaque processus

💾 État du processus

Prêt, exécution, bloqué …

📝 Compteur programme (PC)

Adresse de la prochaine instruction

🧮 Registres CPU

Sauvegarde du contexte lors d’une interruption

🗂️ Gestion mémoire

Tables de pages, segments, limites

📂 Fichiers ouverts

Descripteurs des ressources utilisées

🧩 Représentation simplifiée du PCB (structure noyau)
struct PCB {
    pid: 1245,
    état: "PRÊT",
    PC: 0x0040A3,
    registres: {...},
    limites mémoire: ...
};

🔄 Cycle de vie : les états d’un processus

Un processus change d’état selon les événements

Nouveau
Création
Prêt
Prêt à être élu
Élu
Sélectionné
En exécution
Utilise le CPU
Bloqué
Attente E/S
Terminé
Fin

🔁 Transitions clés :

  • Nouveau → Prêt : le processus est prêt à être exécuté (admis dans la file des prêts).
  • Prêt → Élu → Exécution : l’ordonnanceur choisit le processus.
  • Exécution → Bloqué : lorsqu’il demande une opération d’E/S.
  • Bloqué → Prêt : quand l’E/S se termine.
  • Exécution → Terminé : fin normale ou erreur.
  • Exécution → Prêt : fin de quantum (temps CPU écoulé) préemption.

📈 Ordonnancement et files d’attente

Le système d’exploitation gère plusieurs files de processus :

  • 📌 File des prêts : Tous les processus prêts à être exécutés.
  • ⏸️ Files d’attente (périphériques) : Processus bloqués en attente d’une ressource (disque, imprimante, réseau).
  • ⚙️ Ordonnanceur long terme / court terme : Le court terme (CPU scheduler) sélectionne le prochain processus à exécuter.
💡

Exemple de vie : Un processus en exécution demande une lecture disque → il passe dans l’état bloqué et rejoint la file d’attente du disque. Une fois la lecture terminée, il retourne dans la file des prêts.

✨ À retenir ✨

✓

Processus ≠ programme : un processus est une instance active avec son contexte.

✓

PCB : contient toutes les informations pour gérer un processus (PID, état, registres, etc.).

✓

Modèle à 5/6 états : nouveau, prêt, élu, exécution, bloqué, terminé.

📝 Question réflexe : Si un processus en exécution effectue un appel système pour lire un fichier sur le disque, quel sera son prochain état ?

→ Réponse : État Bloqué (attente d’E/S) jusqu’à la fin de l’opération disque.
