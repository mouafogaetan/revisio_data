📋 Visualiser l’activité : ps, top, htop

ps affiche un instantané des processus. Options courantes :

$ ps aux # tous les processus (format BSD)
$ ps -ef # format standard System V
$ ps -ejH # arborescence des processus

top : interface dynamique en temps réel (CPU, mémoire, processus).
htop : version améliorée avec couleurs et gestion souris.

🔍 Exemple concret :

$ ps aux | grep firefox   # chercher le processus firefox
$ top -u $USER            # voir seulement mes processus

║ PID %CPU CMD║

⚡ Signaux et commande kill

Les signaux sont des notifications asynchrones envoyées à un processus. Principaux :

SIGTERM (15)
Terminaison propre (par défaut). Le processus peut nettoyer.
SIGKILL (9)
Terminaison forcée, immédiate. Ne peut être ignoré.
SIGHUP (1)
Traditionnellement "raccrocher" ; recharger configuration.
$ kill -15 1234 # demande polie d'arrêt
$ kill -9 1234 # destruction immédiate (⚠️ risque de pertes)
$ kill -HUP 1234 # recharger la config (ex : nginx)

killall / pkill : par nom de processus.

$ killall firefox # envoie SIGTERM à tous les firefox
$ pkill -9 -f "mon_script" # tuer par motif
💡 Astuce : kill -l liste tous les signaux disponibles sur votre système.

🔄 Processus en arrière‑plan et contrôle de jobs

Lancer une commande en arrière‑plan avec & :

$ sleep 100 &
[1] 2456 # job numéro 1, PID 2456

Commandes associées :

  • jobs : lister les jobs du shell
  • fg %1 : ramener le job 1 au premier plan
  • bg %1 : relancer un job stoppé en arrière‑plan
  • Ctrl+Z : suspendre un processus (devient stopped)

🧪 Exemple d’atelier :

$ long_calcul &
$ jobs -l
$ fg %1
Ctrl+Z
$ bg %1
$ disown %1   # détacher du shell

🛡️ nohup : résister à la déconnexion

Quand on se déconnecte d’un terminal (SSH), les processus enfants reçoivent SIGHUP et s’arrêtent. nohup ignore ce signal.

$ nohup ./long_script.sh &
nohup: ignoring input and appending output to 'nohup.out'

La sortie est redirigée vers nohup.out. Pratique pour des traitements batch sur serveur !

📁 Fichier de log auto
═══════ Comparaison rapide ═══════
commande    → dépend du terminal, SIGHUP → mort
nohup cmd & → détaché, persiste après déconnexion

🏁 Synthèse & mise en pratique

🔹 Visualiser → ps aux | less , top, htop (à installer)
🔹 Contrôler → kill -9 PID, pkill -f nom
🔹 Arrière‑plan → &, jobs, fg/bg
🔹 Persistance → nohup pour les longs scripts

🧪 Exercice interactif (à tester sur votre machine Linux) :

  1. Lancez sleep 300 & , puis utilisez jobs et kill %1
  2. Affichez les processus système avec ps -ef | grep systemd
  3. Lancez top, triez par mémoire (touche M) et identifiez le processus le plus gourmand.
  4. Simulez un script long : nohup bash -c "while true; do date >> log.txt; sleep 5; done" & puis déconnectez vous (ou fermez terminal) → la tâche continue !
