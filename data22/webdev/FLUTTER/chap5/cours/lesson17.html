Objectifs de la leçon

  • ✓ Définir des routes nommées dans MaterialApp(routes: {}).
  • ✓ Naviguer entre écrans avec Navigator.pushNamed() et retour arrière.
  • ✓ Passer des arguments aux routes nommées.
  • ✓ Gérer les routes inconnues avec onGenerateRoute.

💡 Pourquoi c'est utile ? Les routes nommées permettent une navigation claire, centralisée et facile à maintenir, surtout dans les grandes applications.

📘 PARTIE 1

Définir des routes dans MaterialApp

Au lieu d'utiliser Navigator.push(context, MaterialPageRoute(...)), on pré-déclare toutes les routes de l'application dans le paramètre routes du MaterialApp.

// Dans main.dart
MaterialApp(
  initialRoute: '/',
  routes: {
    '/': (context) => HomeScreen(),
    '/details': (context) => DetailsScreen(),
    '/profile': (context) => ProfileScreen(),
  },
);

✔️ Chaque route est associée à un nom (ex: '/details') et un constructeur d’écran.
✔️ initialRoute définit l'écran de démarrage (souvent '/').

🧭 PARTIE 2

Naviguer avec pushNamed et retour

Pour aller vers un écran nommé :

// Depuis n'importe quel widget avec BuildContext
onPressed: () {
  Navigator.pushNamed(context, '/details');
}

Revenir en arrière :

Navigator.pop(context);  // ferme l'écran courant

💡 Remarque : Navigator.pop() fonctionne aussi bien avec les routes nommées qu'avec les routes classiques.

📦 PARTIE 3

Transmettre des données

Pour passer des arguments (ID, objet utilisateur, etc.) lors d'une navigation nommée, on utilise Navigator.pushNamed(context, routeName, arguments: data).

// Envoi
Navigator.pushNamed(
  context,
  '/details',
  arguments: {'id': 42, 'title': 'Flutter'},
);

// Réception dans l'écran cible
class DetailsScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final args = ModalRoute.of(context)!.settings.arguments as Map;
    return Scaffold(
      body: Text('ID: ${args['id']}'),
    );
  }
}

⚠️ Important : il faut typer et vérifier les arguments pour éviter les erreurs.

⚠️ PARTIE 4

Route 404: onGenerateRoute

Si l'utilisateur tente d'accéder à une route non déclarée, Flutter affiche une erreur. Pour personnaliser ce comportement, on utilise onGenerateRoute.

MaterialApp(
  onGenerateRoute: (settings) {
    if (settings.name == '/notfound') {
      return MaterialPageRoute(builder: (context) => NotFoundScreen());
    }
    // Route par défaut en cas d'absence
    return MaterialPageRoute(builder: (context) => HomeScreen());
  },
  // On peut aussi définir une route dédiée pour les erreurs :
  onUnknownRoute: (settings) => MaterialPageRoute(
    builder: (context) => ErrorScreen(),
  ),
);

✅ Bonne pratique : toujours prévoir une route "404" pour rediriger vers une page d'erreur agréable.

📱 Exemple concret

Application simple avec deux écrans:

// 1. Déclaration des routes
MaterialApp(
  initialRoute: '/',
  routes: {
    '/': (context) => Home(),
    '/contact': (context) => ContactPage(),
  },
);

// 2. Dans Home : bouton pour aller à '/contact' avec argument
Navigator.pushNamed(context, '/contact', arguments: "Salut depuis Home");

// 3. Dans ContactPage :
final message = ModalRoute.of(context)!.settings.arguments as String;
// Afficher message
✔ Avantages: Lisibilité, centralisation, maintenance facile
⚠️ Attention: Les arguments doivent être sécurisés et typés.
