Architecture du projet final

Nous adopterons une architecture propre inspirée du Clean Architecture / MVVM en utilisant Riverpod ou Provider pour la gestion d'état.

lib/
├── models/          # classes (User, Product, Post)
├── data/            # repositories & API (http client)
├── providers/       # state providers (Riverpod)
├── screens/         # pages (Home, Details, Profile, Cart)
├── widgets/         # composants réutilisables
├── utils/           # constantes, thèmes, helpers
└── main.dart        # entry point + MaterialApp.router

Exemple d'API gratuite: jsonplaceholder.typicode.com ou fakestoreapi.com pour un mock e-commerce.

🌐 Consommer une API avec Riverpod

Exemple concret : récupérer une liste d'articles via http et affichage dynamique.

final articleProvider = FutureProvider<List<Article>>((ref) async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts'));
  if (response.statusCode == 200) {
    List jsonList = jsonDecode(response.body);
    return jsonList.map((json) => Article.fromJson(json)).toList();
  }
  throw Exception('Erreur API');
});

Dans le widget UI : ref.watch(articleProvider) → gestion automatique du chargement, erreur et données.

✨ Astuce Cameroun : Utilisez des intercepteurs HTTP pour gérer les tokens d'authentification si vous implémentez login.

🧭 Navigation & routes profondes

Avec go_router nous gérons les URLs, paramètres et transitions personnalisées.

final router = GoRouter(routes: [
  GoRoute(path: '/', name: 'home', builder: (context, state) => HomeScreen()),
  GoRoute(path: '/details/:id', name: 'details',
    builder: (context, state) {
      final id = state.pathParameters['id']!;
      return DetailsScreen(postId: id);
    }),
]);

Utiliser context.push('/details/5') et gérer les animations PageTransition pour une expérience riche.

✨ Animations fluides — Hero & transitions

Dans une galerie d'articles, enveloppez l'image avec un widget Hero pour une transition magique entre les écrans.

Hero( tag: "hero_${post.id}", child: Image.network(post.imageUrl) )

Ajoutez des animations implicites avec AnimatedContainer pour les boutons, cards et feedback utilisateur. Cela rend l’application plus professionnelle.

🧪 Tester avant déploiement

  • Tests unitaires : models, utilitaires, providers.
  • Tests widgets : vérifier l'affichage des éléments clés.
  • Tests d'intégration : simuler un parcours complet (connexion, achat, ajout panier).
  • Optimisation : utiliser const constructeurs, ListView.builder, éviter les rebuilds inutiles.
  • DevTools : inspecter le widget tree, memory et performance.
