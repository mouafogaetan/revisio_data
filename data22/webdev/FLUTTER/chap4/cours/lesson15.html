🧠 Qu’est-ce que BLoC ?

BLoC = Business Logic Component . Un pattern créé par Google pour séparer la logique métier de l'interface.

  • Événements : actions de l'utilisateur (ex: LoginSubmitted)
  • États : réaction du bloc (ex: LoadingState, SuccessState, ErrorState)
  • Flux unidirectionnel : UI → événement → BLoC → nouvel état → UI

📦 Installation de flutter_bloc

Ajoutez dans le fichier pubspec.yaml :

dependencies:
  flutter:
    sdk: flutter
  flutter_bloc: ^8.1.3   # dernière version stable
  equatable: ^2.0.5      # utile pour comparer les états

dev_dependencies:
  bloc_test: ^9.1.5      # pour les tests

Puis lancez flutter pub get.

⚡ Bon à savoir : Le package equatable permet de comparer facilement les objets (états) et éviter les rebuilds inutiles.

📐 Exemple pratique : BLoC compteur

Étapes typiques : définir les événements, les états, puis le bloc.

// events/counter_event.dart
abstract class CounterEvent {}
class Increment extends CounterEvent {}
class Decrement extends CounterEvent {}

// states/counter_state.dart
class CounterState {
  final int value;
  CounterState(this.value);
}

// bloc/counter_bloc.dart
import 'package:flutter_bloc/flutter_bloc.dart';

class CounterBloc extends Bloc<CounterEvent, CounterState> {
  CounterBloc() : super(CounterState(0)) {
    on<Increment>((event, emit) {
      emit(CounterState(state.value + 1));
    });
    on<Decrement>((event, emit) {
      emit(CounterState(state.value - 1));
    });
  }
}

✅ on<Event> : écoute chaque événement et produit un nouvel état avec emit().

🔌 Intégration dans l'UI

Trois widgets clés pour connecter votre interface au Bloc :

BlocProvider

Fournit le Bloc à l'arbre de widgets.

BlocBuilder

Reconstruit l'UI selon l'état.

BlocListener

Réagit aux états (toasts, navigation).

BlocProvider(
  create: (context) => CounterBloc(),
  child: MyHomePage(),
);

// Dans un widget enfant :
BlocBuilder<CounterBloc, CounterState>(
  builder: (context, state) {
    return Text('${state.value}');
  },
);

// Pour les actions ponctuelles :
BlocListener<CounterBloc, CounterState>(
  listener: (context, state) {
    if (state is ErrorState) showSnackBar(...);
  },
  child: ...,
);

🌐 Cas concret : LoginBloc

Événements : LoginSubmitted(email, password)
États : LoginInitial, LoginLoading, LoginSuccess, LoginFailure

class LoginBloc extends Bloc<LoginEvent, LoginState> {
  final AuthRepository repo;
  LoginBloc(this.repo) : super(LoginInitial()) {
    on<LoginSubmitted>((event, emit) async {
      emit(LoginLoading());
      try {
        await repo.login(event.email, event.password);
        emit(LoginSuccess());
      } catch (e) {
        emit(LoginFailure(e.toString()));
      }
    });
  }
}

💡 Dans l'UI, on utilise BlocBuilder pour afficher un loader ou un message d'erreur.

📝 événement → ⚙️ BLoC → 🔄 état

✨ Bonnes pratiques BLoC

  • Un Bloc par fonctionnalité (ex: AuthBloc, CartBloc)
  • Utilisez Equatable pour que les états soient comparés correctement (évite les rebuilds intempestifs)
  • Ne placez pas de logique UI dans le Bloc : laissez la présentation aux widgets
  • Pour les gros projets, structurez : lib/blocs/auth/, lib/pages/ etc.
  • Testez facilement vos blocs avec blocTest 🧪

"BLoC rend votre application prévisible et facile à déboguer. Chaque état est un snapshot clair."
