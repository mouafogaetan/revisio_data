🌟 Qu'est-ce que Hero ?

Hero est un widget Flutter qui permet de créer une animation fluide lorsqu’un élément passe d’un écran à un autre. Par exemple, une image miniature dans une liste s’agrandit doucement vers une vue détaillée.

🔍 Comment ça marche ?

  • Deux widgets Hero avec le même tag (identifiant) sur deux écrans différents.
  • Flutter détecte automatiquement le mouvement et anime la transition (position, taille, forme).
  • Résultat : un effet "partagé" qui guide l'utilisateur.

Le widget Hero avec le même tag anime la transition.

✍️ Syntaxe de base

Enveloppez le widget à animer avec Hero et donnez lui un tag unique.

// Premier écran (liste)
Hero(
  tag: 'profil_avatar',
  child: CircleAvatar(
    radius: 40,
    backgroundImage: NetworkImage('url_photo'),
  ),
)

// Deuxième écran (détail)
Hero(
  tag: 'profil_avatar',   // MÊME TAG
  child: Image.network('url_photo', width: 200, height: 200),
)

⚠️ Important : Le tag doit être identique et non-null. Les deux widgets doivent être de type similaire (mais pas strictement identiques, Flutter fera la morphose).

📱 Exemple concret

Imaginons une galerie d'images: quand on clique sur une miniature, l'image s'agrandit en fondu vers l'écran détail.

// Dans la liste (miniature)
GestureDetector(
  onTap: () => Navigator.push(context, MaterialPageRoute(
    builder: (_) => DetailScreen(imageUrl: url, tag: 'img_$index')
  )),
  child: Hero(
    tag: 'img_$index',
    child: Image.network(url, width: 100, height: 100),
  ),
)

// Écran de détail
class DetailScreen extends StatelessWidget {
  final String imageUrl;
  final String tag;
  // ...
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Détail')),
      body: Center(
        child: Hero(
          tag: tag,
          child: Image.network(imageUrl, width: 300, height: 300),
        ),
      ),
    );
  }
}

✅ Résultat: L'image "vole" vers sa nouvelle position.

🎨 Personnaliser l’animation Hero

Vous pouvez modifier la durée, la courbe et même l’animation du vol.

Hero(
  tag: 'customHero',
  flightShuttleBuilder: (flightContext, animation, direction, fromContext, toContext) {
    // Customiser le widget pendant l'animation
    return ScaleTransition(scale: animation, child: ...);
  },
  placeholderBuilder: (context, size, child) => Container(color: Colors.grey),
  transitionOnUserGestures: true, // pour les gestes retour
  child: MonWidget(),
)
🔁 flightShuttleBuilder : personnalise l’élément volant.
⏱️ createRectTween : pour changer l'interpolation du rectangle.
🎯 transitionOnUserGestures : anime aussi lors du swipe retour.

📌 Bonnes pratiques & astuces

  • ✔️ Utilisez des tags uniques par session (ex: identifiant de l’objet).
  • ✔️ Les deux widgets Hero doivent être de taille différente? Flutter s’en charge !
  • ✔️ Évitez les Hero avec des widgets qui changent radicalement de type (ex: Text → Container), privilégiez des formes proches.
  • ✔️ Testez sur des appareils réels, l'animation est très fluide sur mobile.
  • ❌ Erreur fréquente : oublier d’encapsuler le widget de destination dans Hero avec le même tag → l’animation ne fonctionne pas.
💪 Challenge personnel : créez une application de profil avec une photo qui s’anime entre l’écran d’accueil et l’écran de détails.

🛠️ Atelier pratique

Reproduisez cette structure dans votre projet Flutter : 

// main.dart (extrait)
import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) => MaterialApp(home: HomePage());
}

class HomePage extends StatelessWidget {
  final List images = [
    'https://picsum.photos/id/101/200/200',
    'https://picsum.photos/id/102/200/200',
  ];
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Hero Demo')),
      body: GridView.builder(
        gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(crossAxisCount: 2),
        itemCount: images.length,
        itemBuilder: (ctx, i) => GestureDetector(
          onTap: () => Navigator.push(ctx, MaterialPageRoute(builder: (_) => DetailPage(imgUrl: images[i], tag: 'hero_$i'))),
          child: Hero(tag: 'hero_$i', child: Image.network(images[i], fit: BoxFit.cover)),
        ),
      ),
    );
  }
}

class DetailPage extends StatelessWidget {
  final String imgUrl, tag;
  DetailPage({required this.imgUrl, required this.tag});
  @override
  Widget build(BuildContext context) => Scaffold(
    appBar: AppBar(),
    body: Center(
      child: Hero(tag: tag, child: Image.network(imgUrl, width: 300, height: 300)),
    ),
  );
}

🎉 Exécutez, touchez une image et admirez l’animation magique !
