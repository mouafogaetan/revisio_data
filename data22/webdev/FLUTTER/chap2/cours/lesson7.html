Null Safety : les bases

En Dart, les variables ne peuvent pas contenir null sauf si vous le spécifiez explicitement avec ?.

// Non-nullable (ne peut jamais être null)
int age = 25;
String nom = "Alice";
// age = null; // ERREUR !!!

// Nullable (peut contenir null)
int? ageNullable = null;
String? adresse;  // null par défaut
✔️ Toute variable doit être initialisée avant utilisation. Le compilateur Dart garantit la sûreté.

⚡ Opérateurs essentiels : ?? et !

🔸 ?? (coalescence nulle) : fournit une valeur par défaut si l'expression est null.

String? nomUtilisateur;
String affichage = nomUtilisateur ?? "Invité"; // "Invité"

🔸 ??= (assignation conditionnelle) : assigne seulement si la variable est null.

int? score;
score ??= 100; // score devient 100
score ??= 200; // reste 100

🔸 ! (assertion non-null) : dites à Dart "je suis sûr que cette valeur n'est pas null".

String? possibleNull = "Bonjour";
int longueur = possibleNull!.length; // OK si non-null
⚠️ Utilisez ! avec précaution : une erreur d'exécution survient si la valeur est réellement null.

⏳ Le mot-clé late

late permet de déclarer une variable non-nullable sans l'initialiser immédiatement. L'initialisation aura lieu plus tard (avant la première utilisation).

class MonWidget extends StatefulWidget {
  late String _nom; // promesse d'initialisation avant utilisation

  void initialiser(String nom) {
    _nom = nom;
  }

  void afficher() {
    print(_nom); // ici, _nom DOIT être initialisé
  }
}
✨ Utilisation typique : late pour les contrôleurs (TextEditingController, AnimationController) dans initState().
⚠️ Si vous accédez à une variable late avant son assignation → LateInitializationError.

✅ Bonnes pratiques du null safety

  • Préférez les types non-nullables par défaut : évitez ? sauf nécessité.
  • Utilisez ?? plutôt que ! pour fournir une valeur par défaut.
  • Évitez late si possible ; préférez l'initialisation dans le constructeur ou via des paramètres requis.
  • Vérifiez les nulls explicitement avec if (variable != null) pour plus de sécurité.
  • En Flutter, de nombreux widgets acceptent des paramètres optionnels (nullable) : respectez la sémantique.
// Exemple dans un widget
Text(
  user?.name ?? "Anonyme",  // safe
  style: Theme.of(context).textTheme.bodyLarge,
)
