📦 Classes et constructeurs en Dart

Une classe est un plan qui définit des propriétés (données) et des méthodes (comportements).

class Utilisateur { String nom; int age; // Constructeur classique Utilisateur(this.nom, this.age); // Constructeur nommé Utilisateur.anonyme() : nom = "Invité", age = 0; // Méthode void afficher() { print("$nom a $age ans"); } } void main() { var user1 = Utilisateur("Marie", 25); var guest = Utilisateur.anonyme(); user1.afficher(); // Marie a 25 ans }

✏️ Les paramètres this.nom dans le constructeur assignent automatiquement les valeurs aux propriétés.

👨‍👧 Héritage avec extends

Une classe enfant hérite des propriétés et méthodes de la classe parente, et peut ajouter ou surcharger (override) des comportements.

class Vehicule { void demarrer() => print("Véhicule démarré"); } class Voiture extends Vehicule { @override void demarrer() { super.demarrer(); // appelle la méthode parent print("La voiture roule"); } } void main() { Voiture maVoiture = Voiture(); maVoiture.demarrer(); }

⚠️ Remarque : Dart supporte l’héritage simple (une seule classe parente).

🔌 Interfaces avec implements

Dart n’a pas de mot-clé dédié interface : toute classe peut être utilisée comme interface. Une classe qui implements doit fournir une implémentation de tous ses membres.

class Machine { void allumer() => print("Machine allumée"); } class Ordinateur implements Machine { @override void allumer() => print("Ordinateur boot en 5s"); } void main() { Machine pc = Ordinateur(); pc.allumer(); }

💡 On peut implémenter plusieurs interfaces : class A implements B, C { ... }

🧩 Mixins : réutiliser du code sans héritage

Un mixin est un morceau de code qu’on peut ajouter à une classe avec with. Très utilisé dans Flutter pour les SingleTickerProviderStateMixin.

mixin Volant { void voler() => print("Je vole !"); } mixin Nageur { void nager() => print("Je nage !"); } class Canard with Volant, Nageur {} void main() { var canard = Canard(); canard.voler(); // Je vole ! canard.nager(); // Je nage ! }

✅ Les mixins permettent d’éviter les problèmes du multiple héritage.

📐 Classes abstraites et membres statiques

Classe abstraite (abstract) :

abstract class Forme { double calculerAire(); // méthode abstraite (sans corps) } class Cercle extends Forme { double rayon; Cercle(this.rayon); @override double calculerAire() => 3.14 * rayon * rayon; }

Méthode statique (static) :

class MathUtils { static double carre(double x) => x * x; } // Appel : MathUtils.carre(5);
