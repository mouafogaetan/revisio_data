Flutter est un SDK créé par Google pour développer des applications iOS, Android, Web et Desktop à partir d'un seul codebase.

🔍 Qu'est-ce que Flutter ?

Flutter est un framework open-source créé par Google. Il permet de développer des applications natives pour iOS, Android, Web, Windows, macOS et Linux à partir d'un seul codebase écrit en Dart.

À retenir :

Flutter ne produit pas du code webview (comme Cordova) mais compile en code natif ARM (pour mobiles) ou en JavaScript (pour le web).

Quelques chiffres :

  • Créé en 2015, première version stable en 2018
  • Utilisé par Google (Google Pay, Stadia), Alibaba, BMW, eBay, etc.
  • Plus de 160 000 stars sur GitHub

🏗️ Architecture Flutter

Flutter est structuré en couches :

Le moteur de rendu Skia permet d'avoir le même rendu sur toutes les plateformes.

🧩 Les Widgets : tout est widget !

Dans Flutter, tout est widget : un bouton, un texte, une mise en page, une animation. On distingue deux types :

StatelessWidget

Widget immuable, son apparence ne change pas après sa création. Ex: Text, Icon.

StatefulWidget

Widget dynamique, peut se reconstruire avec setState(). Ex: Checkbox, Slider.

// Exemple simple de StatelessWidget
class MonTexte extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Bonjour Flutter !');
  }
}

⚡ Les avantages clés

Hot reload : visualisez les changements en moins d'une seconde sans perdre l'état.
Performances natives : compilation AOT vers ARM/C++.
UI expressive : widgets Material Design et Cupertino (iOS).
Code unique : une base de code pour 6 plateformes.
Contrôle total : chaque pixel est dessiné par Flutter.
Communauté active : nombreux packages, documentation riche.
Comparaison rapide : Flutter offre de meilleures performances que React Native pour les animations complexes, grâce à son moteur Skia.

📌 Récapitulatif

  • Flutter = SDK multiplateforme (mobile, web, desktop).
  • Langage Dart (performant, facile à apprendre).
  • Architecture en couches : Framework (Dart) → Engine (C++) → Embedder.
  • Widgets : StatelessWidget (statique) et StatefulWidget (dynamique).
  • Avantages : hot reload, performance native, UI cohérente, communauté active.
