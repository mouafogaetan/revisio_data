Gestion du panier d'achat

Créer un store Zustand performant pour gérer les articles, les quantités et synchroniser avec l'API

⚡ Zustand • 🛒 Panier temps réel • 💾 localStorage

🎯 Objectifs de la leçon

① Créer un store cart (Zustand) avec items, quantité, total.
② Implémenter les actions : addToCart, removeFromCart, updateQuantity.
③ Synchroniser le panier avec le backend après connexion.
④ Persister le panier (localStorage) et gérer le total TTC.
✨ À la fin, vous saurez créer un panier fluide et réactif pour votre e-commerce

📦 Pourquoi utiliser Zustand pour le panier ?

Zustand est une librairie d'état global minimaliste. Pour un panier e-commerce, on a besoin de :

  • Accéder au panier depuis n'importe quel composant (header, page produit, page panier).
  • Mettre à jour les quantités sans re-rendus inutiles.
  • Persister les données même après un rafraîchissement (localStorage).
  • Calcul automatique du total, taxes, etc.
💡 Avantage : Zustand est plus léger que Redux et plus simple que Context API pour ce genre de logique métier.

🛠️ Création du store cart (Zustand)

Voici la structure de base : items, quantités, total et les actions essentielles.

// stores/cartStore.js
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

export const useCartStore = create(
  persist(
    (set, get) => ({
      items: [],           // { id, name, price, quantity, image }
      total: 0,
      
      addToCart: (product) => set((state) => {
        const existing = state.items.find(item => item.id === product.id);
        let newItems;
        if (existing) {
          newItems = state.items.map(item =>
            item.id === product.id ? { ...item, quantity: item.quantity + 1 } : item
          );
        } else {
          newItems = [...state.items, { ...product, quantity: 1 }];
        }
        const newTotal = newItems.reduce((sum, i) => sum + (i.price * i.quantity), 0);
        return { items: newItems, total: newTotal };
      }),

      removeFromCart: (id) => set((state) => {
        const newItems = state.items.filter(item => item.id !== id);
        const newTotal = newItems.reduce((sum, i) => sum + (i.price * i.quantity), 0);
        return { items: newItems, total: newTotal };
      }),

      updateQuantity: (id, quantity) => set((state) => {
        if (quantity <= 0) return { items: state.items.filter(i => i.id !== id) };
        const newItems = state.items.map(item =>
          item.id === id ? { ...item, quantity } : item
        );
        const newTotal = newItems.reduce((sum, i) => sum + (i.price * i.quantity), 0);
        return { items: newItems, total: newTotal };
      }),

      clearCart: () => set({ items: [], total: 0 }),
    }),
    { name: 'cart-storage' } // sauvegarde localStorage
  )
);

💾 Persistance du panier & calcul automatique

Avec le middleware persist, le panier reste même après fermeture du navigateur. Idéal pour les utilisateurs non connectés.

💰 Formule du total HT : \( \text{total} = \sum_{i=1}^{n} (prix_i \times quantité_i) \)

🇨🇲 Avec TVA (19.25%) : \( \text{total TTC} = \text{total} \times 1.1925 \)

Zustand recalcule le total à chaque modification (add/remove/update). Aucune logique supplémentaire n'est nécessaire dans les composants.

⚠️ Important : synchroniser avec le backend après login – nous verrons cela avec Axios.

➕ Ajouter un produit au panier

Dans n’importe quel composant, on appelle addToCart du store.

// components/ProductCard.jsx
import { useCartStore } from '../stores/cartStore';

const ProductCard = ({ product }) => {
  const addToCart = useCartStore((state) => state.addToCart);
  
  return (
    <div>
      <h3>{product.name}</h3>
      <p>{product.price} €</p>
      <button onClick={() => addToCart(product)}>Ajouter au panier</button>
    </div>
  );
};
✅ Le store met à jour la quantité si le produit existe déjà (incrémentation).

🛒 Afficher le nombre d’articles (badge)

Dans le Header, on peut facilement afficher le compteur d’articles :

// components/Header.jsx
import { useCartStore } from '../stores/cartStore';

const Header = () => {
  const itemCount = useCartStore((state) => 
    state.items.reduce((acc, item) => acc + item.quantity, 0)
  );
  
  return (
    <header>
      <span>🛒 Panier ({itemCount})</span>
    </header>
  );
};
🔁 Grâce à l’abonnement sélectif de Zustand, seul le badge se re-rend quand le panier change → performances optimisées.

🔄 Synchronisation avec le backend après connexion

Quand l’utilisateur se connecte, il faut :

  1. Récupérer son panier depuis l’API (GET /cart).
  2. Fusionner avec le panier local actuel (Zustand).
  3. Envoyer le panier fusionné vers le backend (PUT /cart/merge).
// Dans AuthStore après login
const login = async (email, password) => {
  const res = await authApi.login({ email, password });
  const remoteCart = await cartApi.getCart();    // panier distant
  const localCart = useCartStore.getState().items;
  
  // Fusion simple : on concatène et on agrège les quantités
  const merged = mergeCarts(localCart, remoteCart);
  useCartStore.setState({ items: merged, total: computeTotal(merged) });
  
  // Mise à jour backend
  await cartApi.syncCart(merged);
};
