❤️ Gestion des favoris (Wishlist)

Un favori est un produit que l'utilisateur souhaite garder pour plus tard. On utilise Zustand avec persistance dans localStorage et synchronisation avec le backend après connexion.

// store/favoriteStore.js
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

const useFavoriteStore = create(
  persist(
    (set, get) => ({
      favorites: [],
      toggleFavorite: (product) => {
        const exists = get().favorites.some(p => p.id === product.id);
        let newFavorites;
        if (exists) {
          newFavorites = get().favorites.filter(p => p.id !== product.id);
        } else {
          newFavorites = [...get().favorites, product];
        }
        set({ favorites: newFavorites });
        // Option: Appel API pour sauvegarder côté serveur si connecté
      },
      isFavorite: (id) => get().favorites.some(p => p.id === id),
      syncFromBackend: (backendFavs) => set({ favorites: backendFavs })
    }),
    { name: 'wishlist-storage' }
  )
);

🔁 Synchro après login : récupérer les favoris depuis l'API et les fusionner avec ceux du localStorage (éviter les doublons).

🔍 Comparer des produits

Permettre à l'utilisateur de comparer les caractéristiques de plusieurs produits (max 4). Stockage en localStorage ou Zustand.

const useCompareStore = create(
  persist(
    (set, get) => ({
      compareList: [],
      addToCompare: (product) => {
        const { compareList } = get();
        if (compareList.length >= 4) {
          toast.error("Vous ne pouvez comparer que 4 produits maximum");
          return false;
        }
        if (compareList.some(p => p.id === product.id)) {
          toast.info("Produit déjà dans la comparaison");
          return false;
        }
        set({ compareList: [...compareList, product] });
        toast.success(`${product.name} ajouté à la comparaison`);
        return true;
      },
      removeFromCompare: (id) => {
        set({ compareList: get().compareList.filter(p => p.id !== id) });
        toast.success("Produit retiré de la comparaison");
      },
      clearCompare: () => set({ compareList: [] })
    }),
    { name: 'compare-storage' }
  )
);

🔔 Notifier les changements (Toasts)

Améliorez l'expérience utilisateur avec des notifications contextuelles : ajout au panier, favori, comparaison, erreur.

📦 Installation 
npm install sonner
⚙️ Configuration 
import { Toaster, toast } from 'sonner';
<Toaster position="top-right" />
toast.success('Ajouté aux favoris ❤️');
toast.error('Limite de 4 produits atteinte');

📌 Bonnes pratiques :

  • Utiliser des messages clairs et courts.
  • Ne pas surcharger l'utilisateur (éviter les toasts multiples).
  • Associer icône et couleur au type (succès/erreur/info).

🚀 Mise en pratique & Défi

✨ Intégration dans un composant produit :

function ProductCard({ product }) {
  const { toggleFavorite, isFavorite } = useFavoriteStore();
  const { addToCompare } = useCompareStore();

  return (
    <div className="border rounded-lg p-4">
      <h3>{product.name}</h3>
      <button onClick={() => toggleFavorite(product)}>
        {isFavorite(product.id) ? '❤️' : '🤍'}
      </button>
      <button onClick={() => addToCompare(product)}>
        🔍 Comparer
      </button>
    </div>
  );
}

🏆 Défi pour les élèves :

  • Créer une page "/favoris" listant tous les produits favoris.
  • Ajouter une page "/comparer" affichant un tableau comparatif (4 colonnes max).
  • Implémenter un toast personnalisé lors de l'ajout/suppression d'un favori.
  • Persister la liste de comparaison même après fermeture du navigateur.
