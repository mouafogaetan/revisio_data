🎯 Objectifs de la leçon

  • Créer une application React avec Vite (alternative rapide à Create React App).
  • Configurer ESLint & Prettier pour maintenir un code propre et uniforme.
  • Mettre en place Husky + lint-staged pour automatiser la vérification avant chaque commit.

À la fin, vous aurez une base moderne, robuste et prête pour développer une interface e-commerce performante.

🚀 Pourquoi Vite plutôt que CRA ?

Vite (mot français pour "rapide") est un build tool qui offre un démarrage instantané et un HMR (Hot Module Replacement) ultra-rapide.

Démarrage froid

Pas de bundling pendant le dev : ES modules natifs.

Optimisé pour la production

Rollup pour le build final, plus petit et plus rapide.

# Créer un nouveau projet React avec TypeScript
npm create vite@latest ecommerce-front -- --template react-ts

ESLint + Prettier : code parfait

ESLint analyse le code pour trouver des erreurs, Prettier formate automatiquement.

npm install -D eslint prettier eslint-config-prettier eslint-plugin-react @typescript-eslint/parser @typescript-eslint/eslint-plugin

Fichier .eslintrc.cjs :

module.exports = {
  root: true,
  env: { browser: true, es2020: true },
  extends: [ 'eslint:recommended', 'plugin:@typescript-eslint/recommended',
    'plugin:react-hooks/recommended', 'prettier' ],
  parser: '@typescript-eslint/parser',
  plugins: ['react-refresh']
};

✨ Ajoutez un script dans package.json : "lint": "eslint . --ext ts,tsx --report-unused-disable-directives"

🐶 Husky & lint-staged

Husky permet d'exécuter des scripts (hooks) avant chaque commit, garantissant que le code est toujours propre.

npx husky-init
npm install
npx husky add .husky/pre-commit "npx lint-staged"

Configuration dans package.json :

"lint-staged": {
  "*.{js,jsx,ts,tsx}": ["eslint --fix", "prettier --write"]
}

Ainsi, chaque commit déclenche la correction automatique du code.

Chemins d'alias (imports propres)

Au lieu de ../../../components/Button on utilise @/components/Button.

1️⃣ Modifier vite.config.ts :

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'
export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: { '@': path.resolve(__dirname, './src') }
  }
});

2️⃣ Synchroniser TypeScript (tsconfig.json) :

"compilerOptions": {
  "baseUrl": ".",
  "paths": { "@/*": ["./src/*"] }
}

Plus besoin de remonter les dossiers : import Navbar from '@/components/Navbar'

Architecture suggérée

src/
├── @types/
├── assets/
├── components/   (atoms, molecules, organisms)
├── pages/         (Home, Cart, Checkout, Account)
├── hooks/         (useAuth, useCart)
├── services/    (api, axios)
├── store/         (zustand stores)
├── utils/        (formatPrice, validateForm)
├── App.tsx
└── main.tsx

Cette organisation suit le principe Atomic Design et facilite la maintenance.

💡 Conseil : Après la configuration des alias, vous pouvez importer n'importe quel composant via `@/components/...`

Commandes essentielles

npm run dev – Lance le serveur de développement (port 5173)
npm run build – Génère le dossier dist/ pour la production
npm run preview – Prévisualise le build localement
npm run lint – Vérifie la qualité du code

🎬 Résumé du workflow :

  1. Créer projet avec Vite + React + TS
  2. Ajouter ESLint + Prettier + configurer les règles
  3. Mettre en place Husky + lint-staged pour les commits
  4. Configurer alias `@` pour des imports élégants
  5. Structurer les dossiers et commencer le développement du store
