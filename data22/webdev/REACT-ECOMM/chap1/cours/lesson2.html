Architecture & Atomic Design

Organiser son projet React comme une usine à composants : réutilisable, maintenable, scalable.

👨‍🏫 Classe: Développement Web | React · E-commerce

🎯 Objectifs de la leçon

  • ✓ Appliquer l’Atomic Design (atomes, molécules, organismes, templates, pages)
  • ✓ Planifier les routes et la navigation avec React Router
  • ✓ Choisir et implémenter une solution de gestion d'état (Context API / Zustand / Redux)
  • ✓ Structurer un dossier e-commerce maintenable pour l’équipe

💡 À la fin, vous saurez organiser un gros projet React sans spaghetti code !

🧱 Atomic Design : découpage malin

Inspiré de la chimie, ce pattern organise les composants du plus simple au plus complexe.

Atomes
Button, Input, Label
Molécules
SearchBar, ProductCard
Organismes
Header, ProductGrid
Templates
Layout, CheckoutTemplate
Pages
HomePage, CartPage
src/
├── components/
│   ├── atoms/
│   │    ├── Button.tsx, Input.tsx, Icon.tsx
│   ├── molecules/
│   │    ├── ProductCard.tsx, SearchBar.tsx
│   ├── organisms/
│   │    ├── Navbar.tsx, ProductList.tsx
│   ├── templates/
│   │    ├── MainLayout.tsx, AuthLayout.tsx
├── pages/
│   ├── HomePage.tsx, ProductPage.tsx, CartPage.tsx

✅ Avantage : réutilisabilité maximale, tests facilités, UI cohérente.

📁 Organisation pratique pour e-commerce

src/
├── assets/            (images, fonts)
├── components/        (atomic folders)
├── hooks/             (useAuth, useCart)
├── services/          (API calls axios)
├── stores/            (Zustand / Redux stores)
├── types/             (interfaces TS)
├── utils/             (helpers, formatters)
├── pages/             (pages de l'app)
├── routes/            (router config)
└── App.tsx, main.tsx

👉 Pour notre boutique, on aura : stores/cartStore.ts, services/productService.ts, components/organisms/ProductGrid.tsx

🧠 Règle d'or : un composant = une responsabilité. Les pages ne font qu’assembler des templates/organismes.

⚛️ Gestion d’état : le bon choix

🔁 Context API

Idéal pour thème, authentification simple, pas trop de re-renders.

const AuthContext = createContext()

📦 Zustand (recommandé)

Léger, performant, boilerplate minimal → parfait pour panier, favoris, filtres.

const useCartStore = create((set) => ({ items: [], addItem: ... }))

Exemple concret : store panier avec persist middleware (localStorage) et actions (add, remove).

import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface CartState {
  items: CartItem[];
  addItem: (product) => void;
  total: number;
}

export const useCartStore = create<CartState>()(
  persist((set) => ({
    items: [],
    addItem: (product) => set((state) => ({ items: [...state.items, product] })),
    total: 0,
  }), { name: 'cart-storage' })
);

🎛️ Gestion des filtres et état partagé

Pour le catalogue produits, on centralise les filtres (prix, catégorie, recherche) dans un store dédié.

// stores/filterStore.ts
export const useFilterStore = create((set) => ({
  category: '',
  minPrice: 0,
  maxPrice: 1000,
  sortBy: 'popular',
  setCategory: (cat) => set({ category: cat }),
  setPriceRange: (min, max) => set({ minPrice: min, maxPrice: max }),
  resetFilters: () => set({ category: '', minPrice: 0, maxPrice: 1000 }),
}));

// Dans ProductsPage
const { category, minPrice, maxPrice } = useFilterStore();

✨ Avantage : les filtres sont partagés entre la sidebar et la page, faciles à synchroniser avec l'API via React Query.

⚠️ Attention à ne pas surcharger le store global : état UI (modal ouverte, chargement) peut rester local.

📐 Mise en pratique : construire un composant "ProductCard"

Approche atomique :

  • Atome : Button, PriceTag
  • Molécule : ProductCard (assemble Image, Titre, Prix, Bouton)
  • Organisme : ProductGrid (liste de ProductCard + gestion grille)

🧪 Atome Button → 🌟 Molécule ProductCard → 🧩 Organisme ProductGrid → 🖥️ Template ShopTemplate → 📄 Page ProductsPage

⚛️ → 🧬 → 🧫 → 📐 → 📄
// ProductCard.tsx (molécule)
import { Button } from '@/components/atoms/Button';
import { formatPrice } from '@/utils/format';

export const ProductCard = ({ product, onAddToCart }) => {
  return (
    <div className="border rounded p-3">
      <img src={product.image} alt={product.name} loading="lazy"/>
      <h3 className="font-bold">{product.name}</h3>
      <span className="text-indigo-600">{formatPrice(product.price)}</span>
      <Button onClick={() => onAddToCart(product)}>Ajouter</Button>
    </div>
  );
};

📌 Récapitulatif – Bonnes pratiques

  • ✅ Organisation en atomic design pour réutiliser sans effort.
  • ✅ Planifier les routes avec react-router-dom et layouts protégés.
  • ✅ Choisir Zustand pour l’état global (panier, filtres, auth).
  • ✅ Séparer la logique métier (services/hooks) des composants UI.
