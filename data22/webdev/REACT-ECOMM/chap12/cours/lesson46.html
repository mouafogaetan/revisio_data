📦 Installation et configuration

On ajoute le plugin officiel à notre projet Vite :

npm install -D vite-plugin-pwa

Configuration dans vite.config.js :

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { VitePWA } from 'vite-plugin-pwa';

export default defineConfig({
  plugins: [
    react(),
    VitePWA({
      registerType: 'autoUpdate',
      includeAssets: ['favicon.ico', 'apple-touch-icon.png', 'masked-icon.svg'],
      manifest: {
        name: 'Mon E-commerce',
        short_name: 'E-shop',
        theme_color: '#4f46e5',
        background_color: '#ffffff',
        display: 'standalone',
        icons: [
          { src: 'pwa-192x192.png', sizes: '192x192', type: 'image/png' },
          { src: 'pwa-512x512.png', sizes: '512x512', type: 'image/png' }
        ]
      },
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
        runtimeCaching: [
          {
            urlPattern: /^https:\/\/api\.mon-site\.com\/products/,
            handler: 'NetworkFirst',
            options: { cacheName: 'api-cache' }
          }
        ]
      }
    })
  ]
});
✨ Le manifest génère l'icône d'installation ; Workbox gère la stratégie de cache.

🔄 Stratégies de cache (Workbox)

NetworkFirst : requête réseau d'abord, fallback cache → parfait pour l'API (produits, prix)
CacheFirst : d'abord le cache, très rapide → pour les assets statiques (CSS, JS, images de marque)
StaleWhileRevalidate : retourne le cache immédiatement, met à jour en arrière-plan → idéal pour les avis clients. 
// Exemple pour les images produits : CacheFirst
{
  urlPattern: /\.(png|jpg|jpeg|webp)$/,
  handler: 'CacheFirst',
  options: { cacheName: 'image-cache', expiration: { maxEntries: 50, maxAgeSeconds: 30*24*60*60 } }
}

💾 Cacher les assets critiques

Le manifest définit quels fichiers sont pré-cachés à l'installation :

workbox: {
  globPatterns: ['**/*.{js,css,html,png,svg,ico,woff2}'],
  // Inclusion manuelle
  additionalManifestEntries: [
    { url: '/offline.html', revision: '1' }
  ]
}

Le service worker installe ces fichiers dans le cache dès la première visite. Grâce à cela, l'application se lance même sans réseau (offline-first).

📌 Astuce : proposez une page hors-ligne personnalisée offline.html pour informer l'utilisateur.

📱 Consultation du catalogue hors-ligne

Pour que les produits restent accessibles sans connexion :

  • Utiliser NetworkFirst sur l'API GET /products
  • Stocker les résultats dans IndexedDB via Workbox
  • Fournir un fallback si aucune donnée en cache.
// runtimeCaching
{
  urlPattern: /\/api\/v1\/products/,
  handler: 'NetworkFirst',
  options: {
    cacheName: 'products-cache',
    networkTimeoutSeconds: 3,
    expiration: { maxEntries: 100, maxAgeSeconds: 86400 }
  }
}
✅ L'utilisateur peut ainsi naviguer dans les derniers produits consultés, même en avion ou zone blanche.

🆙 Mise à jour et installation

Quand on modifie l'application, le service worker se met à jour automatiquement (registerType: 'autoUpdate').

Pour afficher un bouton "Installer l'application" :

// Dans un composant React
useEffect(() => {
  window.addEventListener('beforeinstallprompt', (e) => {
    e.preventDefault();
    setDeferredPrompt(e);
    setShowInstall(true);
  });
}, []);

const installApp = () => {
  deferredPrompt.prompt();
  deferredPrompt.userChoice.then(() => setShowInstall(false));
};
💡 L'utilisateur pourra ajouter votre e-commerce à l'écran d'accueil comme une vraie app.

🧪 Exercice pratique

🎯 À réaliser :

  • Ajouter vite-plugin-pwa à un projet Vite + React existant
  • Configurer le manifest (icônes, nom, thème)
  • Mettre en cache l'API des produits avec stratégie NetworkFirst
  • Tester le mode hors-ligne (DevTools → Application → Service workers → Offline)
  • Ajouter un prompt d'installation personnalisé
💡 Indice : utilisez le panneau Application des DevTools pour inspecter le cache et simuler l'absence de réseau.
