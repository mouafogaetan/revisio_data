📦 Leçon 37

Gestion des commandes (Admin)

Lister, filtrer, modifier le statut et générer des factures PDF pour les commandes côté administration.

👨‍🏫 Classe: Développement Web | React · Interface e-commerce

🎯 Objectifs de la leçon

  • ✓ Lister toutes les commandes avec filtres (statut, date)
  • ✓ Changer le statut d’une commande (useMutation)
  • ✓ Générer des factures PDF côté frontend ou via API
  • ✓ Protéger l’accès avec un rôle admin (RoleBasedRoute)

💡 Un tableau de bord des commandes performant est essentiel pour la gestion de la boutique !

📋 Liste des commandes (admin)

Composant AdminOrders qui récupère toutes les commandes avec filtres.

// pages/admin/AdminOrders.tsx
import { useQuery } from '@tanstack/react-query';
import { api } from '@/services/api';

export const AdminOrders = () => {
  const [statusFilter, setStatusFilter] = useState('all');
  const { data: orders, isLoading } = useQuery({
    queryKey: ['admin-orders', statusFilter],
    queryFn: () => api.get(`/admin/orders?status=${statusFilter}`).then(res => res.data)
  });

  if (isLoading) return <div>Chargement...</div>;

  return (
    <div className="overflow-x-auto">
      <select value={statusFilter} onChange={(e) => setStatusFilter(e.target.value)} className="mb-4 p-2 border rounded">
        <option value="all">Tous</option>
        <option value="pending">En attente</option>
        <option value="paid">Payée</option>
        <option value="shipped">Expédiée</option>
        <option value="delivered">Livrée</option>
        <option value="cancelled">Annulée</option>
      </select>
      <table className="min-w-full bg-white border">
        <thead><tr><th>ID</th><th>Client</th><th>Total</th><th>Statut</th><th>Actions</th></tr></thead>
        <tbody>
          {orders?.map(order => (
            <tr key={order.id}>
              <td>{order.id}</td>
              <td>{order.user.email}</td>
              <td>{order.total} FCFA</td>
              <td><StatusBadge status={order.status} /></td>
              <td><button className="text-indigo-600">Voir</button></td>
            </tr>
          ))}
        </tbody>
      </table>
    </div>
  );
};

Le filtre est envoyé au backend via query param.

🔄 Mutation : changement de statut

On utilise useMutation pour mettre à jour le statut, puis on invalide le cache.

// hooks/useUpdateOrderStatus.ts
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { api } from '@/services/api';
import { toast } from 'sonner';

export const useUpdateOrderStatus = () => {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: ({ orderId, status }: { orderId: string; status: string }) =>
      api.patch(`/admin/orders/${orderId}/status`, { status }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['admin-orders'] });
      toast.success('Statut mis à jour');
    },
    onError: () => toast.error('Erreur lors de la mise à jour')
  });
};

// Dans le tableau
const { mutate: updateStatus } = useUpdateOrderStatus();
<select 
  value={order.status}
  onChange={(e) => updateStatus({ orderId: order.id, status: e.target.value })}
  className="border rounded p-1"
>
  <option value="pending">En attente</option>
  <option value="paid">Payée</option>
  <option value="shipped">Expédiée</option>
  <option value="delivered">Livrée</option>
  <option value="cancelled">Annulée</option>
</select>
⚠️ Certaines transitions peuvent être restreintes (ex: delivered → cancelled). Le backend doit valider.

📄 Génération de factures PDF

Deux approches :

  • Côté frontend : librairie jspdf + html2canvas ou @react-pdf/renderer
  • Côté backend : générer le PDF et renvoyer un lien de téléchargement (recommandé)
// Option 1: Télécharger depuis le backend
const downloadInvoice = async (orderId) => {
  const response = await api.get(`/admin/orders/${orderId}/invoice`, {
    responseType: 'blob'
  });
  const url = window.URL.createObjectURL(new Blob([response.data]));
  const link = document.createElement('a');
  link.href = url;
  link.setAttribute('download', `facture_${orderId}.pdf`);
  document.body.appendChild(link);
  link.click();
  link.remove();
};

// Option 2: Avec react-pdf (frontend)
import { PDFDownloadLink } from '@react-pdf/renderer';
import { InvoicePDF } from './InvoicePDF';

<PDFDownloadLink document={<InvoicePDF order={order} />} fileName={`facture_${order.id}.pdf`}>
  {({ loading }) => (loading ? 'Génération...' : 'Télécharger PDF')}
</PDFDownloadLink>
✅ Le backend garantit des données cohérentes (calculs, TVA) et évite la manipulation côté client.

🔍 Vue détaillée d’une commande

Composant AdminOrderDetail pour voir les produits, adresse, historique.

// pages/admin/AdminOrderDetail.tsx
import { useQuery } from '@tanstack/react-query';
import { useParams } from 'react-router-dom';

export const AdminOrderDetail = () => {
  const { id } = useParams();
  const { data: order } = useQuery({
    queryKey: ['admin-order', id],
    queryFn: () => api.get(`/admin/orders/${id}`).then(res => res.data)
  });

  return (
    <div className="space-y-6">
      <div className="grid md:grid-cols-2 gap-4">
        <div>
          <h3>Informations client</h3>
          <p>{order?.user.email}</p>
          <p>{order?.shippingAddress.addressLine}, {order?.shippingAddress.city}</p>
        </div>
        <div>
          <h3>Détails commande</h3>
          <table>...</table>
        </div>
      </div>
      <button onClick={() => downloadInvoice(order.id)} className="bg-indigo-600 text-white px-4 py-2 rounded">
        Télécharger facture PDF
      </button>
    </div>
  );
};

On peut aussi afficher une timeline des statuts avec date.

🛡️ Sécurisation des pages admin

On utilise RoleBasedRoute pour restreindre l’accès aux admins.

// router.tsx
import { RoleBasedRoute } from '@/components/RoleBasedRoute';
import { AdminLayout } from '@/layouts/AdminLayout';

const router = createBrowserRouter([
  {
    element: <RoleBasedRoute allowedRoles={['admin']} />,
    children: [
      {
        path: 'admin',
        element: <AdminLayout />,
        children: [
          { index: true, element: <AdminDashboard /> },
          { path: 'orders', element: <AdminOrders /> },
          { path: 'orders/:id', element: <AdminOrderDetail /> }
        ]
      }
    ]
  }
]);

Le store auth contient le rôle de l’utilisateur (ex: user.role === 'admin').

🔐 La sécurité frontale n’est qu’une première barrière : l’API doit également vérifier le rôle admin.

✨ Optimisations pour l’admin

📅 Filtrage par date

Ajouter des filtres startDate et endDate pour les commandes.

🔍 Pagination serveur

Pour des milliers de commandes, utiliser la pagination côté API.

📊 Tableau de bord

Afficher des graphiques (Recharts) pour visualiser CA et volume.

🔄 Webhook statut

Notifier le client par email lors du changement de statut (backend).

🧪 À retenir : l’interface admin doit être rapide, intuitive et sécurisée.

📌 Résumé – Ce qu'il faut retenir

  • ✅ Liste des commandes : useQuery avec filtres (statut, date)
  • ✅ Changement de statut : useMutation + invalidation du cache
  • ✅ Factures PDF : privilégier la génération côté backend
  • ✅ Sécurité : RoleBasedRoute + vérification API

🎯 Exercice pratique : Ajoutez un filtre par période (date de commande) et un export CSV des commandes filtrées.
