🧩 Shadcn/ui vs Material-UI

✨ Shadcn/ui

Composants accessibles, personnalisables, copier-coller. Utilise Tailwind et Radix UI. Pas de dépendance lourde, vous contrôlez le code source.

📘 Material-UI (MUI)

Très complet, riche en composants, thème Material Design. Idéal pour des apps d'entreprise. Bundle plus gros.

👉 Pour notre e-commerce, nous choisissons Shadcn/ui car léger, facilement adaptable à notre charte graphique.

# Installation dans un projet Vite React
npx shadcn-ui@latest init

Le CLI posera des questions : style Tailwind, couleur primaire (indigo par exemple), alias de composants.

⚙️ Étapes d'installation

  1. Démarrer un projet avec Vite : npm create vite@latest ecommerce-ui -- --template react-ts
  2. Installer TailwindCSS et le configurer (voir leçon 4).
  3. Lancer npx shadcn-ui@latest init
  4. Choisir les options : style "Default", couleur "Zinc" ou personnalisée, oui pour les alias.
  5. Ajouter un premier composant : npx shadcn-ui@latest add button
✅ Une fois installé, les composants se trouvent dans src/components/ui/button.tsx – vous pouvez les modifier directement ! 
// Usage simple dans une page
import { Button } from "@/components/ui/button"

function Home() {
  return <Button variant="outline">Acheter maintenant</Button>
}

⚡ Composants métier réutilisables

Au-delà des primitives, on crée des composants spécifiques à notre domaine :

  • ProductCard : affiche image, nom, prix, bouton ajouter au panier
  • QuantitySelector : incrémenter/décrémenter la quantité
  • PriceTag : format monnaie, affichage promo barré
  • RatingStars : affichage des étoiles basé sur une note
// src/components/products/ProductCard.tsx
import { Card, CardContent } from "@/components/ui/card";
import { Button } from "@/components/ui/button";

export const ProductCard = ({ product, onAdd }) => {
  return (
    <Card className="overflow-hidden">
      <img src={product.image} alt={product.name} className="h-48 w-full object-cover"/>
      <CardContent className="p-4">
        <h3 className="font-semibold">{product.name}</h3>
        <p className="text-indigo-600 font-bold">{product.price} FCFA</p>
        <Button onClick={() => onAdd(product)} className="w-full mt-2">Ajouter</Button>
      </CardContent>
    </Card>
  );
};

🎨 Personnaliser le thème

Shadcn/ui utilise des variables CSS. Modifiez src/index.css pour changer les couleurs globales.

:root {
  --primary: 99 102 241;   /* indigo */
  --primary-foreground: 255 255 255;
  --secondary: 241 245 249;
}
.dark {
  --primary: 129 140 248;
}

Ou bien, vous pouvez surcharger les classes Tailwind des composants via className.

🪟 Utilisation avancée : Dialog (modal)

Pour le panier ou la confirmation d'ajout, on utilise le composant Dialog :

import {
  Dialog,
  DialogContent,
  DialogHeader,
  DialogTitle,
  DialogTrigger,
} from "@/components/ui/dialog"

<Dialog>
  <DialogTrigger asChild>
    <Button>Voir panier</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Votre panier</DialogTitle>
    </DialogHeader>
    {/* Liste des articles */}
  </DialogContent>
</Dialog>

Le composant gère le focus, la fermeture au clic en dehors, et l’accessibilité.

🎯 Exercice : Créez un composant AddToCartModal qui s'ouvre après l'ajout d'un produit.

✅ Récapitulatif et recommandations

  • 📦 Shadcn/ui accélère le développement avec des composants prêts à l’emploi.
  • 🧩 On crée des composants métier (ProductCard, CartDrawer) en assemblant les briques UI.
  • 🎨 Le thème se personnalise via CSS ou les props variant.
  • 📱 Tous les composants sont responsives et accessibles (clavier, lecteurs d’écran).

🚀 À retenir : Ne réinventez pas la roue – Utilisez une librairie UI robuste pour vous concentrer sur la logique métier et l'expérience utilisateur !
