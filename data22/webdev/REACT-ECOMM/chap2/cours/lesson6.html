⚙️ Configuration Tailwind

Dans tailwind.config.js, activez le dark mode basé sur une classe CSS :

// tailwind.config.js
module.exports = {
  darkMode: 'class', // ou 'media' pour basé sur préférence système
  theme: {
    extend: {
      colors: {
        // vos couleurs personnalisées
      }
    }
  },
  plugins: []
}

✅ darkMode: 'class' permet de basculer manuellement en ajoutant la classe dark sur l'élément <html>.

🎨 Les classes dark:

Tailwind génère des versions dark pour chaque classe utilitaire. Il suffit de préfixer par dark: :

bg-white dark:bg-gray-900

text-gray-800 dark:text-gray-100

border-gray-300 dark:border-gray-600

Ce bloc change automatiquement de couleur en dark mode.
📌 Astuce : N'oubliez pas de définir la couleur de fond du body et du texte principal pour les deux thèmes.

🌗 Composant ThemeToggle (React)

// components/ThemeToggle.jsx
import { useEffect, useState } from 'react';

export default function ThemeToggle() {
  const [darkMode, setDarkMode] = useState(false);

  useEffect(() => {
    // Vérifier localStorage et préférence système
    const stored = localStorage.getItem('theme');
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
    
    if (stored === 'dark' || (!stored && prefersDark)) {
      document.documentElement.classList.add('dark');
      setDarkMode(true);
    } else {
      document.documentElement.classList.remove('dark');
      setDarkMode(false);
    }
  }, []);

  const toggleTheme = () => {
    if (darkMode) {
      document.documentElement.classList.remove('dark');
      localStorage.setItem('theme', 'light');
      setDarkMode(false);
    } else {
      document.documentElement.classList.add('dark');
      localStorage.setItem('theme', 'dark');
      setDarkMode(true);
    }
  };

  return (
    <button onClick={toggleTheme} className="p-2 rounded-full bg-gray-200 dark:bg-gray-700">
      {darkMode ? '☀️' : '🌙'}
    </button>
  );
}

🔁 Ce composant gère le stockage local et synchronise l'interface.

💾 Persistance avec localStorage

Le choix de l'utilisateur doit être mémorisé pour les prochaines visites.

🔒 Bonne pratique :

  • Clé localStorage : 'theme'
  • Valeurs possibles : 'light' ou 'dark'
  • Lire au chargement de l'application (dans un effet ou avant le rendu)
  • Ne jamais stocker d'informations sensibles dans localStorage
📌 Code pour lire localStorage au démarrage :
const theme = localStorage.getItem('theme');

🖥️ prefers-color-scheme

La media query CSS prefers-color-scheme détecte si l'utilisateur a choisi un thème clair ou sombre dans son système d'exploitation.

// Détection JavaScript
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;

// Si localStorage est vide, on utilise la préférence système
if (!localStorage.getItem('theme')) {
  if (prefersDark) {
    document.documentElement.classList.add('dark');
  } else {
    document.documentElement.classList.remove('dark');
  }
}
🎯 Objectif : offrir une expérience cohérente sans configuration manuelle.

🔄 Réagir aux changements système

Si l'utilisateur change le thème système pendant que l'application est ouverte, on peut écouter l'événement change.

useEffect(() => {
  const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
  
  const handleChange = (e) => {
    // Ne change que si l'utilisateur n'a pas forcé un choix manuel
    if (!localStorage.getItem('theme')) {
      if (e.matches) {
        document.documentElement.classList.add('dark');
      } else {
        document.documentElement.classList.remove('dark');
      }
    }
  };
  
  mediaQuery.addEventListener('change', handleChange);
  return () => mediaQuery.removeEventListener('change', handleChange);
}, []);

⚠️ Important : ne pas écraser le choix explicite de l'utilisateur stocké dans localStorage.

📦 Store Zustand pour le thème

// stores/themeStore.js
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

const useThemeStore = create(
  persist(
    (set) => ({
      theme: 'system', // 'light', 'dark', 'system'
      setTheme: (theme) => {
        set({ theme });
        // Appliquer la classe au HTML
        if (theme === 'dark' || (theme === 'system' && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
          document.documentElement.classList.add('dark');
        } else {
          document.documentElement.classList.remove('dark');
        }
      },
    }),
    { name: 'theme-storage' }
  )
);

✅ Le store persiste automatiquement dans localStorage et gère le mode système.

♿ Accessibilité et bonnes pratiques

Vérifiez tous les composants (cards, formulaires, modales).
