💳 Stripe Elements

Stripe fournit des composants React (@stripe/react-stripe-js) pour collecter les informations de carte sans toucher aux données sensibles.

import { Elements, CardElement, useStripe, useElements } from "@stripe/react-stripe-js";
import { loadStripe } from "@stripe/stripe-js";

const stripePromise = loadStripe("pk_test_...");

function CheckoutForm() {
  const stripe = useStripe();
  const elements = useElements();

  const handleSubmit = async (event) => {
    event.preventDefault();
    const {error, paymentMethod} = await stripe.createPaymentMethod({
      type: "card",
      card: elements.getElement(CardElement),
    });
    // Envoyer paymentMethod.id au backend
  };
  return (<form onSubmit={handleSubmit}><CardElement /></form>);
}

Le backend crée un PaymentIntent pour capturer le paiement.

📘 PayPal Smart Buttons

Intégration via @paypal/react-paypal-js. Les boutons s'adaptent automatiquement.

import { PayPalScriptProvider, PayPalButtons } from "@paypal/react-paypal-js";

function PayPalButton({ total }) {
  return (
    <PayPalScriptProvider options={{ "client-id": "YOUR_CLIENT_ID" }}>
      <PayPalButtons
        createOrder={(data, actions) => {
          return actions.order.create({
            purchase_units: [{ amount: { value: total } }],
          });
        }}
        onApprove={(data, actions) => {
          return actions.order.capture().then((details) => {
            alert("Paiement réussi !");
          });
        }}
      />
    </PayPalScriptProvider>
  );
}

✔️ PayPal gère la logique de paiement et renvoie une capture.

🏷️ Gestion des codes promo

Un code promo applique une réduction (pourcentage ou montant fixe) avant validation de la commande.

Exemple : PROMO10 → −10% sur le total HT

const applyCoupon = async (code) => {
  const res = await axios.get(`/api/coupons/validate?code=${code}`);
  if (res.data.valid) {
    const reduction = res.data.type === 'percent' 
      ? (total * res.data.value) / 100 
      : res.data.value;
    setDiscount(reduction);
    setFinalTotal(total - reduction);
  } else {
    alert("Code invalide ou expiré");
  }
};

💡 Le backend vérifie la validité, les conditions (montant mini, utilisateur, date) et le nombre d’utilisations.

🔒 Sécurité : PCI DSS & tokenisation

Stripe et PayPal gèrent la conformité PCI. Ne stockez jamais les numéros de carte ou CVV dans votre base de données.

  • Utilisez toujours des tokens (paymentMethod.id ou order.id).
  • Communiquez en HTTPS uniquement.
  • Les webhooks Stripe/PayPal notifient votre serveur des confirmations de paiement.
✅ La délégation à Stripe/PayPal élimine la manipulation des données sensibles.

🔄 Basculer entre Stripe et PayPal

On propose à l’utilisateur de choisir sa méthode de paiement préférée via un état paymentMethod.

const [paymentType, setPaymentType] = useState("stripe");

return (
  <div>
    <select onChange={(e) => setPaymentType(e.target.value)}>
      <option value="stripe">Carte bancaire (Stripe)</option>
      <option value="paypal">PayPal</option>
    </select>
    {paymentType === "stripe" && <StripeForm />}
    {paymentType === "paypal" && <PayPalButton total={finalTotal} />}
  </div>
);

On recalcule le total avec éventuel code promo avant d’afficher le formulaire.

🧪 Tester les paiements

Stripe fournit des cartes de test : 4242 4242 4242 4242 (succès), 4000 0000 0000 0002 (refus). PayPal propose un sandbox avec comptes factices.

À faire avant mise en production :

  • Passer en mode "live" (clés réelles).
  • Tester les webhooks localement avec stripe listen ou ngrok.
  • Vérifier que les codes promo sont bien appliqués.
