✅ Leçon 30

Validation de commande & Confirmation

Récapituler la commande, valider le paiement, créer la commande et afficher la page de confirmation avec numéro de commande.

👨‍🏫 Classe: Développement Web | React · Interface e-commerce

🎯 Objectifs de la leçon

  • ✓ Afficher un récapitulatif complet de la commande (articles, adresse, total)
  • ✓ Valider et créer la commande via l'API (useMutation)
  • ✓ Gérer les états de chargement et les erreurs
  • ✓ Afficher la page de confirmation avec le numéro de commande et vider le panier

💡 Une étape cruciale : rassurer le client et finaliser l'achat en toute sécurité !

📋 Récapitulatif de la commande

Avant validation, on affiche un résumé : produits, quantités, sous-total, frais de livraison, taxes, total général. On utilise les données du store panier et du store adresse.

// components/OrderSummary.tsx
import { useCartStore } from '@/stores/cartStore';
import { useDeliveryStore } from '@/stores/deliveryStore';

export const OrderSummary = () => {
  const { items, total, shippingCost, tax } = useCartStore();
  const { shippingAddress } = useDeliveryStore();

  return (
    <div className="bg-gray-50 p-5 rounded-xl space-y-3">
      <h3 className="font-bold text-lg">Récapitulatif</h3>
      {items.map(item => (
        <div key={item.id} className="flex justify-between">
          <span>{item.name} x {item.quantity}</span>
          <span>{item.price * item.quantity} FCFA</span>
        </div>
      ))}
      <div className="border-t pt-2">
        <div className="flex justify-between"><span>Sous-total</span><span>{total} FCFA</span></div>
        <div className="flex justify-between"><span>Livraison</span><span>{shippingCost} FCFA</span></div>
        <div className="flex justify-between font-bold"><span>Total TTC</span><span>{total + shippingCost + tax} FCFA</span></div>
      </div>
      <div className="text-sm">
        <p>Livraison à : {shippingAddress?.addressLine}, {shippingAddress?.city}</p>
      </div>
    </div>
  );
};

📦 Envoi au backend (useMutation)

On envoie les données du panier et de l'adresse à l'API pour créer la commande.

// hooks/useCreateOrder.ts
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { api } from '@/services/api';
import { useCartStore } from '@/stores/cartStore';
import { useDeliveryStore } from '@/stores/deliveryStore';

export const useCreateOrder = () => {
  const queryClient = useQueryClient();
  const { items, total, clearCart } = useCartStore();
  const { shippingAddress } = useDeliveryStore();

  return useMutation({
    mutationFn: async (paymentDetails: any) => {
      const orderData = {
        items: items.map(i => ({ productId: i.id, quantity: i.quantity, price: i.price })),
        total,
        shippingAddress,
        paymentMethod: paymentDetails.method,
        paymentIntentId: paymentDetails.intentId
      };
      const { data } = await api.post('/orders', orderData);
      return data;
    },
    onSuccess: (data) => {
      clearCart(); // vide le panier local
      queryClient.invalidateQueries({ queryKey: ['orders'] });
      // redirection vers page confirmation avec orderId
      window.location.href = `/confirmation/${data.id}`;
    },
    onError: (error) => {
      toast.error('Erreur lors de la création de la commande');
    }
  });
};

La mutation reçoit les détails de paiement (ex: stripePaymentIntent) et retourne l'objet commande.

🔁 Page Checkout finale

Le composant CheckoutPage affiche le résumé, le formulaire de paiement (Stripe/PayPal) et gère la soumission.

// pages/CheckoutPage.tsx
import { OrderSummary } from '@/components/OrderSummary';
import { PaymentForm } from '@/components/PaymentForm';
import { useCreateOrder } from '@/hooks/useCreateOrder';

export const CheckoutPage = () => {
  const { mutate, isPending } = useCreateOrder();

  const handlePaymentSuccess = (paymentResult) => {
    mutate({ method: 'stripe', intentId: paymentResult.id });
  };

  return (
    <div className="grid md:grid-cols-2 gap-8">
      <OrderSummary />
      <div>
        <h2 className="text-xl font-bold mb-4">Paiement</h2>
        <PaymentForm onSuccess={handlePaymentSuccess} loading={isPending} />
      </div>
    </div>
  );
};
✅ Le bouton de paiement est désactivé pendant l'envoi et affiche un spinner.

🎉 Page de confirmation

Après création réussie, on redirige vers /confirmation/:orderId et on affiche les détails de la commande.

// pages/ConfirmationPage.tsx
import { useParams } from 'react-router-dom';
import { useQuery } from '@tanstack/react-query';
import { api } from '@/services/api';

export const ConfirmationPage = () => {
  const { orderId } = useParams();
  const { data: order, isLoading } = useQuery({
    queryKey: ['order', orderId],
    queryFn: () => api.get(`/orders/${orderId}`).then(res => res.data)
  });

  if (isLoading) return <div>Chargement...</div>;

  return (
    <div className="text-center py-12">
      <div className="bg-green-100 w-20 h-20 rounded-full flex items-center justify-center mx-auto mb-4">✓</div>
      <h1 className="text-3xl font-bold">Merci pour votre commande !</h1>
      <p className="text-gray-600 mt-2">N° commande : <strong>{order.id}</strong></p>
      <p>Un email de confirmation a été envoyé à {order.email}</p>
      <Link to="/" className="mt-6 inline-block bg-indigo-600 text-white px-6 py-2 rounded-full">Continuer mes achats</Link>
    </div>
  );
};

⚠️ Gestion des erreurs et réessais

Si la création échoue (paiement refusé, panier expiré), on affiche un message clair et on laisse l'utilisateur réessayer.

// Dans useCreateOrder
onError: (error) => {
  if (error.response?.status === 409) {
    toast.error('Votre panier a expiré, veuillez recommencer la sélection');
    setTimeout(() => window.location.href = '/cart', 2000);
  } else {
    toast.error('Erreur technique, veuillez réessayer ou contacter le support');
  }
}

// Dans CheckoutPage on peut aussi retry manuellement avec mutate()
🔐 Sécurité : Ne jamais stocker les informations de carte bancaire côté frontend. Utiliser Stripe/PayPal en mode tokenisation.

📧 Email de confirmation

Empiler les sections résumé / paiement sur petits écrans.

🧪 À retenir : la page de confirmation est aussi une opportunité pour proposer des produits complémentaires (cross-selling).

📌 Résumé – Ce qu'il faut retenir

  • ✅ Récapitulatif : afficher clairement produits, adresse et total avant paiement.
  • ✅ Mutation : envoyer la commande au backend avec useMutation, vider le panier après succès.
  • ✅ Confirmation : page dédiée avec numéro de commande, email envoyé.
  • ✅ Erreurs : gérer les cas de panier vide, paiement refusé, expiration.

🎯 Exercice pratique : Ajoutez une étape "Récapitulatif" avant le paiement avec possibilité de modifier la quantité depuis cette page.
