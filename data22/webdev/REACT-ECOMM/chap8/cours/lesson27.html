Formulaire intelligent, sélection d’adresses et validation avec React Hook Form + Zod.

📝 Formulaire avec React Hook Form

React Hook Form est performant et réduit les re-renders. Exemple de base :

import { useForm } from "react-hook-form";

function AddressForm() {
  const { register, handleSubmit, formState: { errors } } = useForm();
  const onSubmit = (data) => console.log(data);

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register("fullName")} placeholder="Nom complet" />
      <input {...register("street")} placeholder="Rue" />
      <button type="submit">Enregistrer</button>
    </form>
  );
}

✏️ L’attribut register connecte chaque champ à React Hook Form.

✅ Validation avec Zod

Zod permet de définir un schéma de validation. On l’intègre avec zodResolver.

import { z } from "zod";
import { zodResolver } from "@hookform/resolvers/zod";

const addressSchema = z.object({
  fullName: z.string().min(3, "Nom trop court"),
  street: z.string().min(5, "Rue requise"),
  city: z.string().min(2, "Ville requise"),
  postalCode: z.string().regex(/^\d{5}$/, "Code postal à 5 chiffres"),
  country: z.string().min(2, "Pays requis")
});

const { register, handleSubmit, formState: { errors } } = useForm({
  resolver: zodResolver(addressSchema)
});

📌 Les messages d’erreur sont accessibles via errors.fullName?.message.

🏠 Gérer plusieurs adresses

L’utilisateur peut : choisir une adresse parmi celles enregistrées ou ajouter une nouvelle adresse.

Exemple d’interface :

✔️ Quand l’utilisateur sélectionne une adresse, les champs se remplissent automatiquement (setValue de RHF).
const handleSelectAddress = (address) => {
  setValue("fullName", address.fullName);
  setValue("street", address.street);
  setValue("city", address.city);
  setValue("postalCode", address.postalCode);
  setValue("country", address.country);
};

⚠️ Gestion des erreurs utilisateur

Afficher les messages d’erreur sous chaque champ améliore l’UX.

Exemple : errors.street && <p>{errors.street.message}</p>

<input {...register("postalCode")} className={errors.postalCode ? "border-red-500" : "border-gray-300"} />
{errors.postalCode && <p className="text-red-500 text-sm">{errors.postalCode.message}</p>}

On peut aussi désactiver le bouton de validation tant que le formulaire est invalide (!formState.isValid).

🛒 Intégration dans la page Checkout

Le composant d’adresse est une partie essentielle du tunnel d’achat.

// Checkout.jsx
import AddressForm from "../components/AddressForm";

export default function Checkout() {
  const [savedAddresses, setSavedAddresses] = useState([]);
  // ... récupération des adresses depuis le backend

  const onSubmitAddress = (data) => {
    // Sauvegarde l'adresse + passage à l'étape suivante
    saveAddressToOrder(data);
    goToPayment();
  };

  return (
    <div>
      <AddressForm addresses={savedAddresses} onSubmit={onSubmitAddress} />
    </div>
  );
}

Le formulaire peut être réutilisé aussi dans la page "Mon compte" pour gérer le carnet d’adresses.

♿ Accessibilité & UX

  • Utiliser des <label> associés aux champs (aria-labelledby).
  • Fournir des indications claires pour les champs obligatoires (*).
  • Gérer la validation au moment de la soumission (ou en temps réel selon préférence).
  • Proposer l’autocomplétion native du navigateur (autoComplete="address-line1").
📌 Un bon formulaire d’adresse réduit les erreurs de livraison de 40%.
