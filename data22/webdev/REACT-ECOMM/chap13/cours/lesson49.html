Playwright : Tests E2E

Automatiser les parcours utilisateur (login, achat) et exécuter les tests en CI

🎭 Playwright 🔄 CI/CD 🧪 Tests e2e

🎯 Objectifs de la leçon

✓Installer et configurer Playwright sur un projet Vite + React
✓Écrire des tests e2e pour le flux de connexion et d'achat
✓Utiliser les sélecteurs robustes (data-testid, getByRole)
✓Intégrer les tests dans un pipeline CI (GitHub Actions)
💡 Playwright est plus fiable et rapide que Puppeteer/Cypress pour les apps modernes.

📦 Installation de Playwright

npm init playwright@latest

Répondez aux questions : TypeScript (recommandé), dossier tests, ajout de GitHub Actions.

// playwright.config.ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './e2e',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  use: {
    baseURL: 'http://localhost:5173',
    trace: 'on-first-retry',
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'firefox', use: { ...devices['Desktop Firefox'] } },
    { name: 'webkit', use: { ...devices['Desktop Safari'] } }
  ],
});

Lancez les tests : npx playwright test

🔐 Tester le flux de connexion

// e2e/login.spec.ts
import { test, expect } from '@playwright/test';

test('connexion réussie avec un utilisateur valide', async ({ page }) => {
  await page.goto('/login');
  await page.getByLabel('Email').fill('client@example.com');
  await page.getByLabel('Mot de passe').fill('password123');
  await page.getByRole('button', { name: 'Se connecter' }).click();

  // Vérifier redirection vers tableau de bord
  await expect(page).toHaveURL('/account');
  await expect(page.getByText('Bienvenue')).toBeVisible();
});
✨ Utilisez getByRole ou data-testid pour des sélecteurs résistants.

🛒 Flux complet : ajout panier → commande → paiement

test('parcours d’achat', async ({ page }) => {
  await page.goto('/products');
  await page.locator('[data-testid="product-card"]').first().click();
  await page.click('button:has-text("Ajouter au panier")');

  await page.goto('/cart');
  await expect(page.locator('.cart-item')).toHaveCount(1);

  await page.click('a:has-text("Procéder au paiement")');
  await page.fill('#address', 'Yaoundé, Cameroun');
  await page.click('button:has-text("Valider la commande")');

  await expect(page.locator('.confirmation')).toContainText('Commande confirmée');
});

💡 Idéal pour détecter les régressions lors des mises à jour.

📌 Bonnes pratiques avec Playwright

🔍 Utiliser data-testid
Préférer data-testid="login-btn" plutôt que des classes CSS fragiles.
🧹 Nettoyage
Avant chaque test, réinitialiser la base de données ou les états.
🎭 Isolation
Chaque test doit être indépendant (éviter les dépendances).
🖼️ Captures
Utiliser await page.screenshot({ path: 'screenshot.png' }); pour débuguer.

⚙️ CI avec GitHub Actions

# .github/workflows/playwright.yml
name: Playwright Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with: { node-version: 18 }
      - run: npm ci
      - run: npx playwright install --with-deps
      - run: npm run build
      - run: npm run preview & npx wait-on http://localhost:4173
      - run: npx playwright test
🚀 Les tests s'exécutent sur chaque push, garantissant la non-régression.

🧪 Exercice : tester l'ajout au panier et la déconnexion

🎯 À réaliser :

  • Installer Playwright dans votre projet e-commerce React
  • Écrire un test de connexion (email: test@shop.com, mdp: test123)
  • Ajouter un test de déconnexion (vérifier la redirection vers /login)
  • Exécuter les tests localement, puis simuler un CI avec GitHub Actions
💡 Indice : utilisez page.getByRole('button', { name: /déconnexion/i }) pour cliquer.
test('déconnexion', async ({ page }) => {
  await page.goto('/account');
  await page.click('button:has-text("Se déconnecter")');
  await expect(page).toHaveURL('/login');
});
