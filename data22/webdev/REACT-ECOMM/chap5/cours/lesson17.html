💡 Pourquoi c’est crucial ? Sur un site e-commerce, un cache bien réglé accélère l’affichage des produits et réduit la charge serveur.
⏳

staleTime : durée de "fraîcheur" du cache

Dans React Query, chaque requête a un état "fresh" ou "stale". staleTime définit combien de temps les données sont considérées comme récentes. Pendant ce délai, aucune nouvelle requête réseau n’est déclenchée.

✅ Sans staleTime (défaut=0)

Dès qu’un composant se remonte → nouvel appel API. Inefficace.

⚡ Avec staleTime: 60000 (1min)

Pendant 60 secondes, React Query sert le cache → zéro requête. Performances maximales.

import { useQuery } from '@tanstack/react-query';

const { data } = useQuery({
  queryKey: ['products'],
  queryFn: fetchProducts,
  staleTime: 1000 * 60 * 5, // 5 minutes frais
});

💡 Pour un catalogue produit, on peut mettre staleTime: 5 min car les prix ne changent pas toutes les secondes.

🔄

Revalidation automatique des données

React Query offre deux moyens principaux pour garder les données à jour :

  • refetchInterval : revalider périodiquement (ex: données de stock en temps réel)
  • invalidateQueries : invalider manuellement après une mutation (ex: ajouter un produit)
// Revalidation toutes les 10 secondes (ex: statut commande)
useQuery({
  queryKey: ['order', orderId],
  queryFn: fetchOrderStatus,
  refetchInterval: 10000,    // 10s
  refetchIntervalInBackground: true,
});

// Invalider après une mutation
const mutation = useMutation({
  mutationFn: addToCart,
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['cart'] });
  },
});
📌 La revalidation ne vide pas le cache, elle va chercher les données à jour en arrière-plan.
🖱️

Prefetch : charger avant d’en avoir besoin

Le préchargement consiste à déclencher une requête en cache avant que l'utilisateur ne clique. Idéal pour les liens de navigation, pages produits, etc.

Exemple pratique : survoler un lien "Détails produit"

import { useQueryClient } from '@tanstack/react-query';

function ProductLink({ productId }) {
  const queryClient = useQueryClient();

  const prefetchProduct = () => {
    queryClient.prefetchQuery({
      queryKey: ['product', productId],
      queryFn: () => fetchProductById(productId),
      staleTime: 5 * 60 * 1000,
    });
  };

  return (
    <a href={`/produit/${productId}`}
       onMouseEnter={prefetchProduct}
       className="text-blue-600 hover:underline">
      Voir produit
    </a>
  );
}

✨ Au survol, les données sont déjà dans le cache. La navigation devient instantanée !

📌 Cas d’usage : pages catalogue, panier, dashboard admin. Le prefetch réduit le temps de chargement perçu.

🏆 Synthèse & bonnes pratiques

staleTime adapté au contexte Éviter le sur-rafraîchissement Prefetch uniquement sur interactions probables
  • Pour des données quasi-statiques (ex: liste catégories), staleTime: Infinity.
  • Utilisez refetchOnWindowFocus pour garder les données fraîches quand l'utilisateur revient sur l'onglet.
  • Le prefetch peut aussi être déclenché dans useEffect ou via router (Next.js, React Router).
  • Combinez cache et revalidation pour un équilibre performance / fraîcheur.

⚙️ Exemple config globale du QueryClient :

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 60_000,   // 1 min par défaut
      refetchOnWindowFocus: true,
      retry: 1,
    },
  },
});

✨ Avec ces techniques, votre application React e-commerce devient fluide et réactive.
