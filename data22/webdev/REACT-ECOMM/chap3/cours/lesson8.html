💡 Pourquoi ? Un layout cohérent améliore l'expérience utilisateur et facilite la maintenance du code.

🏗️ Structure du layout principal

Un layout typique pour un e-commerce contient :

// src/layouts/MainLayout.jsx
import { Outlet } from 'react-router-dom';
import Header from '../components/Header';
import Footer from '../components/Footer';
import Sidebar from '../components/Sidebar';

export default function MainLayout() {
  return (
    <div className="min-h-screen flex flex-col">
      <Header />
      <div className="flex flex-1">
        <Sidebar />
        <main className="flex-1 p-4">
          <Outlet />  {/* ← Les pages enfants s'affichent ici */}
        </main>
      </div>
      <Footer />
    </div>
  );
}

`` est le point d'ancrage où s'afficheront les routes enfants (Home, Products, Cart...).

📱 Header responsive (mobile-first)

// components/Header.jsx
import { useState } from 'react';
import { Link, NavLink } from 'react-router-dom';

export default function Header() {
  const [isMenuOpen, setIsMenuOpen] = useState(false);

  return (
    <header className="bg-white shadow-md sticky top-0 z-50">
      <div className="container mx-auto px-4 py-3 flex justify-between items-center">
        <Link to="/" className="text-2xl font-bold text-blue-600">ShopCam</Link>
        
        {/* Desktop Navigation */}
        <nav className="hidden md:flex space-x-6">
          <NavLink to="/" className={({isActive}) => isActive ? 'text-blue-600' : 'text-gray-700'}>Accueil</NavLink>
          <NavLink to="/products">Produits</NavLink>
          <NavLink to="/cart">Panier</NavLink>
        </nav>
        
        {/* Mobile menu button */}
        <button onClick={() => setIsMenuOpen(!isMenuOpen)} className="md:hidden">
          {isMenuOpen ? '✕' : '☰'}
        </button>
      </div>
      
      {/* Mobile Navigation */}
      {isMenuOpen && (
        <nav className="md:hidden bg-white border-t p-4 flex flex-col space-y-3">
          <Link to="/" onClick={() => setIsMenuOpen(false)}>Accueil</Link>
          <Link to="/products" onClick={() => setIsMenuOpen(false)}>Produits</Link>
          <Link to="/cart" onClick={() => setIsMenuOpen(false)}>Panier</Link>
        </nav>
      )}
    </header>
  );
}

📂 Sidebar avec Outlet imbriqué

// layouts/ProductsLayout.jsx
import { Outlet } from 'react-router-dom';
import FilterSidebar from '../components/FilterSidebar';

export default function ProductsLayout() {
  return (
    <div className="flex flex-col md:flex-row gap-6">
      {/* Sidebar - visible uniquement sur desktop ou toggle */}
      <aside className="md:w-64 bg-gray-50 p-4 rounded-lg">
        <FilterSidebar />
      </aside>
      
      {/* Zone des produits (liste, détail) */}
      <div className="flex-1">
        <Outlet />
      </div>
    </div>
  );
}
💡 Les routes enfants de `/products` (liste, détail) s'afficheront dans l'Outlet, tout en gardant la sidebar visible.

🔄 Configuration des routes imbriquées

// router/index.jsx
import { createBrowserRouter } from 'react-router-dom';
import MainLayout from '../layouts/MainLayout';
import ProductsLayout from '../layouts/ProductsLayout';
import HomePage from '../pages/HomePage';
import ProductsPage from '../pages/ProductsPage';
import ProductDetailPage from '../pages/ProductDetailPage';
import CartPage from '../pages/CartPage';

const router = createBrowserRouter([
  {
    path: '/',
    element: <MainLayout />,
    children: [
      { index: true, element: <HomePage /> },
      {
        path: 'products',
        element: <ProductsLayout />,  // Layout avec sidebar
        children: [
          { index: true, element: <ProductsPage /> },
          { path: ':id', element: <ProductDetailPage /> }
        ]
      },
      { path: 'cart', element: <CartPage /> }
    ]
  }
]);

✅ L'URL `/products/123` affichera : MainLayout → ProductsLayout → ProductDetailPage

✨ NavLink pour les liens actifs

NavLink de React Router permet de styliser automatiquement le lien correspondant à la route courante.

import { NavLink } from 'react-router-dom';

<NavLink 
  to="/products" 
  className={({ isActive }) => 
    isActive ? 'text-blue-600 font-bold' : 'text-gray-600 hover:text-blue-500'
  }
>
  Produits
</NavLink>

// Avec fonction de style
<NavLink 
  to="/cart"
  style={({ isActive }) => ({
    color: isActive ? '#2563eb' : '#4b5563',
    fontWeight: isActive ? 'bold' : 'normal'
  })}
>
  Panier
</NavLink>
💡 On peut aussi utiliser end pour éviter que /products ne matche aussi /products/123.

🔐 Layout Admin (dashboard)

// layouts/AdminLayout.jsx
import { Outlet, Navigate } from 'react-router-dom';
import { useAuthStore } from '../stores/authStore';
import AdminSidebar from '../components/AdminSidebar';

export default function AdminLayout() {
  const { user } = useAuthStore();
  
  if (!user || user.role !== 'admin') {
    return <Navigate to="/login" replace />;
  }
  
  return (
    <div className="flex min-h-screen">
      <AdminSidebar />
      <main className="flex-1 p-6 bg-gray-50">
        <Outlet />
      </main>
    </div>
  );
}

// Routes

  element: <AdminLayout />,
  children: [
    { index: true, element: <AdminDashboard /> },
    { path: 'products', element: <AdminProducts /> },
    { path: 'orders', element: <AdminOrders /> }
  ]
}

✅ Bonnes pratiques

  • Layouts imbriqués : séparer la logique d'affichage (header/footer) du contenu principal
  • Mobile-first : utiliser les classes Tailwind flex-col md:flex-row, menu hamburger
  • NavLink : pour les liens de navigation avec style actif
  • Outlet : chaque layout peut avoir son propre Outlet pour les sous-routes
  • Lazy loading des layouts : pour optimiser le bundle

📌 À retenir :

Un layout est un composant React standard qui utilise <Outlet /> comme placeholder pour les routes enfants. On peut avoir plusieurs niveaux d'imbrication.
