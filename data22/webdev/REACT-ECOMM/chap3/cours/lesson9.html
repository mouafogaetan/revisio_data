🔒 Leçon 9

Routes protégées & Gestion des rôles

Sécuriser l'accès aux pages (checkout, compte, admin) et restreindre selon le rôle utilisateur (client / admin / vendeur).

👨‍🏫 Classe: Développement Web | React · Interface e-commerce

🎯 Objectifs de la leçon

  • ✓ Créer un composant ProtectedRoute pour les pages privées
  • ✓ Rediriger vers login si utilisateur non authentifié
  • ✓ Implémenter la vérification des rôles (admin vs user) avec RoleBasedRoute
  • ✓ Sécuriser le tableau de bord admin et les routes sensibles

💡 À la fin, vous saurez protéger votre application comme un pro, avec une expérience utilisateur fluide !

🛡️ Composant ProtectedRoute

Il vérifie si l'utilisateur est connecté via le store d'authentification. Si oui, il affiche la page enfant, sinon il redirige vers /login.

// components/ProtectedRoute.tsx
import { Navigate, Outlet } from "react-router-dom";
import { useAuthStore } from "@/stores/authStore";

const ProtectedRoute = () => {
  const { isAuthenticated } = useAuthStore();
  return isAuthenticated ? <Outlet /> : <Navigate to="/login" replace />;
};

export default ProtectedRoute;

On utilise Outlet pour les routes enfants et replace pour éviter de revenir en arrière.

⚠️ Important : le store auth persiste (localStorage) pour ne pas perdre la session après un refresh.

🗺️ Déclaration des routes protégées

Avec createBrowserRouter, on imbrique les routes privées dans un layout sécurisé.

import { createBrowserRouter } from "react-router-dom";
import MainLayout from "@/layouts/MainLayout";
import ProtectedRoute from "@/components/ProtectedRoute";
import HomePage from "@/pages/HomePage";
import CartPage from "@/pages/CartPage";
import CheckoutPage from "@/pages/CheckoutPage";
import AccountPage from "@/pages/AccountPage";
import LoginPage from "@/pages/LoginPage";

export const router = createBrowserRouter([
  { path: "/login", element: <LoginPage /> },
  {
    element: <MainLayout />,
    children: [
      { index: true, element: <HomePage /> },
      { path: "cart", element: <CartPage /> },
      {
        element: <ProtectedRoute />,   // 🔒 Barrière d'authentification
        children: [
          { path: "checkout", element: <CheckoutPage /> },
          { path: "account", element: <AccountPage /> }
        ]
      }
    ]
  }
]);

✅ Le visiteur non connecté qui tente d'accéder à /checkout sera redirigé vers /login, puis de retour après connexion.

👑 Route basée sur les rôles

Pour l'administration, seuls les utilisateurs avec le rôle "admin" peuvent accéder.

// components/RoleBasedRoute.tsx
import { Navigate, Outlet } from "react-router-dom";
import { useAuthStore } from "@/stores/authStore";

interface Props {
  allowedRoles: string[];
}

const RoleBasedRoute = ({ allowedRoles }: Props) => {
  const { user, isAuthenticated } = useAuthStore();
  if (!isAuthenticated) return <Navigate to="/login" replace />;
  if (!allowedRoles.includes(user?.role || "")) {
    return <Navigate to="/unauthorized" replace />;
  }
  return <Outlet />;
};

export default RoleBasedRoute;

Utilisation : <RoleBasedRoute allowedRoles={['admin']} />

📌 On peut aussi créer une page /unauthorized pour les accès refusés.

📊 Mise en place de l'admin layout

// router.tsx ajout section admin
{
  element: <RoleBasedRoute allowedRoles={['admin']} />,
  children: [
    {
      path: "admin",
      element: <AdminLayout />,
      children: [
        { index: true, element: <AdminDashboard /> },
        { path: "products", element: <AdminProducts /> },
        { path: "orders", element: <AdminOrders /> },
        { path: "users", element: <AdminUsers /> }
      ]
    }
  ]
}
  • Un utilisateur classique ne verra jamais ces routes.
  • Le store auth contient user.role récupéré depuis le JWT.
  • Le middleware RoleBasedRoute bloque l'accès côté client.
🔐 Ne jamais oublier de sécuriser aussi les endpoints API ! Le front n'est qu'une couche de présentation.

⚛️ Store Zustand avec authentification et rôle

// stores/authStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface AuthState {
  user: { id: string; email: string; role: 'user' | 'admin' | 'seller' } | null;
  token: string | null;
  isAuthenticated: boolean;
  login: (email: string, token: string, role: string) => void;
  logout: () => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      user: null,
      token: null,
      isAuthenticated: false,
      login: (email, token, role) => 
        set({ user: { id: '1', email, role: role as any }, token, isAuthenticated: true }),
      logout: () => set({ user: null, token: null, isAuthenticated: false })
    }),
    { name: 'auth-storage' }
  )
);

Après login, on stocke l'utilisateur + token. Le composant ProtectedRoute lira isAuthenticated.

✨ Astuces supplémentaires

🔁 Redirection après login

Utiliser useLocation pour stocker la page demandée et rediriger l'utilisateur après connexion.

const location = useLocation(); const from = location.state?.from || '/';

⚠️ Page "Non autorisé"

Créez une route /unauthorized avec un message clair.

📦 Lazy loading des routes admin

Utilisez React.lazy pour alléger le bundle principal : les pages admin chargent seulement si l'utilisateur a le rôle.

✅ La sécurité front-end n'est qu'une première barrière : protégez toujours votre API !

📌 Récapitulatif – À retenir

  • 🔐 ProtectedRoute : vérifie l'authentification avant d'afficher les pages privées
  • 👑 RoleBasedRoute : restreint l'accès selon le rôle (admin, vendeur, user)
  • 🔄 Redirection intelligente : retour vers la page demandée après login
  • 📦 Stockage sécurisé du token (localStorage / httpOnly cookie à étudier)

🏁 Défi : Implémentez un tableau de bord "seller" où seuls les vendeurs peuvent gérer leurs produits. Utilisez RoleBasedRoute avec allowedRoles={['seller', 'admin']}.
