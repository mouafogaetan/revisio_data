Fonctions d'agrégation SQL

Compter, additionner, faire la moyenne, trouver min & max

Module: Initiation aux Bases de données | Leçon 14
🎯

Objectifs de la leçon

  • ✓ Utiliser COUNT, SUM, AVG, MIN et MAX.
  • ✓ Combiner DISTINCT avec COUNT pour dénombrer les valeurs uniques.
  • ✓ Comprendre comment les valeurs NULL sont gérées par les fonctions d'agrégation.
  • ✓ Appliquer ces fonctions dans des requêtes simples pour analyser des données.
💡 Ces fonctions sont essentielles pour faire des rapports et statistiques dans une base de données.

📊 COUNT() – Compter les enregistrements

COUNT(*) compte toutes les lignes d'une table, même celles avec des NULL.
COUNT(colonne) compte uniquement les valeurs non NULL.

-- Nombre total d'étudiants
SELECT COUNT(*) FROM etudiants;

-- Nombre d'étudiants ayant un email renseigné
SELECT COUNT(email) FROM etudiants;
⚠️ À retenir : COUNT(*) inclut les NULL, COUNT(colonne) les ignore.

➕ SUM() et AVG()

SUM(colonne) additionne les valeurs numériques.
AVG(colonne) calcule la moyenne (somme / nombre de valeurs non NULL).

-- Somme des notes obtenues
SELECT SUM(notes) FROM examens;

-- Moyenne des salaires des employés
SELECT AVG(salaire) FROM employes;
📌 Exemple concret

notes: 12, 15, 8, NULL, 20 → SUM = 55, AVG = 55/4 = 13.75 (NULL ignoré)

⚠️ Attention NULL

AVG ne tient pas compte des NULL, ce qui peut fausser l'interprétation.

⬇️⬆️ MIN() et MAX()

MIN() retourne la valeur minimale, MAX() la valeur maximale d'une colonne.

-- Le produit le moins cher
SELECT MIN(prix) FROM produits;

-- La date d'embauche la plus récente
SELECT MAX(date_embauche) FROM employes;

🔄 COUNT(DISTINCT)

Pour compter le nombre de valeurs différentes dans une colonne, on utilise COUNT(DISTINCT colonne).

-- Combien de villes différentes dans la table clients ?
SELECT COUNT(DISTINCT ville) FROM clients;

-- Nombre de catégories de produits distinctes
SELECT COUNT(DISTINCT categorie) FROM produits;

🎲 Exemple :

couleurs = ['rouge', 'bleu', 'rouge', 'vert', 'bleu']
→ COUNT(DISTINCT couleur) = 3

⚠️ NULL et fonctions d’agrégation

Les fonctions d'agrégation ignorent les valeurs NULL (sauf COUNT(*)). Cela peut surprendre !

FonctionComportement avec NULL
COUNT(col)Ignore les NULL
SUM(col)Ignore les NULL (si toutes NULL → NULL)
AVG(col)Ignore les NULL, moyenne sur non-NULL
MIN(col) / MAX(col)Ignore les NULL, sauf si toutes NULL → NULL
🧠 Piège classique : Si une colonne contient beaucoup de NULL, AVG peut être différent de la moyenne réelle attendue. Pensez à utiliser COALESCE si nécessaire.

📋 Exemples complets

Imaginons une table ventes(id, produit, quantite, prix_unitaire) :

-- Chiffre d'affaires total
SELECT SUM(quantite * prix_unitaire) AS CA_total FROM ventes;

-- Quantité moyenne vendue par produit
SELECT AVG(quantite) FROM ventes;

-- Nombre total de produits différents vendus
SELECT COUNT(DISTINCT produit) FROM ventes;

-- Le prix minimum, maximum et le prix moyen
SELECT MIN(prix_unitaire), MAX(prix_unitaire), AVG(prix_unitaire) FROM ventes;
🏆 Résultat facile à lire 📈 Analyse statistique rapide

📌 Récapitulatif & astuces

  • COUNT(*) → nombre total de lignes.
  • SUM(col) → additionne les valeurs numériques (NULL ignorés).
  • AVG(col) → moyenne arithmétique.
  • MIN / MAX → valeurs extrêmes.
  • COUNT(DISTINCT col) → compter les valeurs uniques.
  • Toujours tenir compte des NULL pour éviter des erreurs d'interprétation.

✨ Ces fonctions sont souvent utilisées avec GROUP BY (prochaine leçon) pour des analyses plus fines !
