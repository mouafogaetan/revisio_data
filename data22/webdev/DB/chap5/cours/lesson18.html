📖 Les Vues SQL

Simplifiez vos requêtes, renforcez la sécurité et structurez vos données avec les vues.

Leçon 18 · Cours Initiation aux Bases de données

🎯 Objectifs de la leçon

  • ✅ Créer une vue avec CREATE VIEW
  • ✅ Comprendre les avantages : sécurité, simplification des requêtes complexes
  • ✅ Distinguer vue simple et vue matérialisée
  • ✅ Savoir quand et pourquoi utiliser une vue dans vos projets
💡 Une vue est comme une « fenêtre » sur vos données : sans stockage physique, elle affiche un résultat prédéfini.

🔍 Qu'est-ce qu'une vue ?

Une vue (VIEW) est une requête SQL nommée stockée dans la base de données. Elle se comporte comme une table virtuelle : on peut l’interroger avec SELECT, mais elle ne contient pas physiquement les données.

CREATE VIEW vue_clients_actifs AS
SELECT id, nom, email FROM clients WHERE actif = 1;

✅ Exemple concret : au lieu d’écrire chaque fois la clause WHERE actif=1 et joindre plusieurs tables, créez une vue. L'utilisateur final interroge simplement SELECT * FROM vue_clients_actifs.

✨ Avantages des vues

🔐

Sécurité renforcée

Masquer certaines colonnes sensibles (ex: mot de passe, salaire) en exposant seulement une vue sans ces champs.

📦

Simplification des requêtes

Envelopper des jointures complexes, sous-requêtes ou agrégations. Les utilisateurs voient une « table propre ».

🔄

Indépendance des données

Modifier la structure des tables sous-jacentes sans casser les applications qui utilisent la vue.

⚡

Lisibilité du code

Nommer des vues métier (ex: ventes_du_mois) rend le SQL plus intuitif.

🛠️ Créer et utiliser une vue

Syntaxe
CREATE VIEW nom_vue AS
SELECT colonnes FROM table(s) WHERE condition;

📌 Exemple concret :

-- Création d'une vue pour les commandes récentes
CREATE VIEW commandes_recentes AS
SELECT id_client, montant_total, date_commande
FROM commandes
WHERE date_commande > '2025-01-01';

-- Interroger la vue comme une table
SELECT * FROM commandes_recentes WHERE montant_total > 1000;
💡 Remarque : certaines bases autorisent les mises à jour via une vue (si elle est simple et sans agrégats) mais l’usage principal reste la lecture.

⚖️ Vue simple vs Vue matérialisée

CritèreVue classique (VIEW)Vue matérialisée (MATERIALIZED VIEW)
StockageAucune donnée stockée (virtuelle)Données physiquement stockées
PerformanceRelance la requête à chaque accèsAccès ultra-rapide (pré-calculé)
ActualisationAutomatique (toujours à jour)Nécessite un REFRESH manuel ou programmé
Index possible ?NonOui, on peut indexer comme une table
💎 Bon à savoir : Les vues matérialisées sont idéales pour les entrepôts de données, rapports lourds et agrégations complexes. MySQL ne les supporte pas nativement, mais PostgreSQL, Oracle oui.

🗄️ Gérer les vues (DROP / ALTER)

DROP VIEW nom_vue;

Supprime complètement la vue de la base.

CREATE OR REPLACE VIEW nom_vue AS ...

Modifie la définition d’une vue existante (pratique pour itérer).

🔍 Afficher les vues dans une base (MySQL) :

SHOW FULL TABLES WHERE Table_type = 'VIEW';

Sous PostgreSQL : \dv ou requête sur information_schema.views.

🏢 Cas pratique : Vue avec jointures

Imaginez deux tables : employes et departements. Une vue peut combiner les informations.

CREATE VIEW vue_employes_departements AS
SELECT e.nom, e.salaire, d.nom_dept
FROM employes e
JOIN departements d ON e.dept_id = d.id
WHERE e.salaire > 2000;

-- Utilisation simple :
SELECT * FROM vue_employes_departements WHERE nom_dept = 'Informatique';
📢 Avantage métier : les analystes n'ont pas besoin de connaître la structure des clés étrangères, ils travaillent sur une vue intuitive.

📌 Récapitulatif & Bonnes pratiques

  • 🔹 Utilisez des vues pour masquer la complexité des bases aux utilisateurs finaux.
  • 🔹 Appliquez les vues pour restreindre l’accès à certaines colonnes (sécurité fines).
  • 🔹 Évitez les vues trop lentes qui enchaînent des sous-requêtes coûteuses (préférez les matérialisées si besoin).
  • 🔹 Nommez clairement les vues avec un préfixe vw_ ou vue_ pour les identifier.
  • 🔹 Certains SGBD permettent d’indexer les vues matérialisées pour des performances extrêmes.
✨ À retenir : Une vue simplifie la vie des développeurs et des analystes tout en protégeant l’intégrité des données.
