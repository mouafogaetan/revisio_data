Manipulation des données

INSERT · UPDATE · DELETE

Maîtrisez les commandes SQL pour insérer, modifier et supprimer des enregistrements dans une base de données.

💡 "Sans données, une base n'est qu'un squelette"

🎯 Objectifs de la leçon

📥 INSERT

Insérer des lignes avec INSERT INTO ... VALUES.

✏️ UPDATE

Modifier des données existantes avec UPDATE ... SET ... WHERE.

🗑️ DELETE

Supprimer des enregistrements avec DELETE FROM ... WHERE.

⚠️ Bonnes pratiques

Toujours utiliser WHERE pour UPDATE et DELETE (sauf cas spécifique).

📌 Résultat attendu :   Être capable d'écrire des requêtes SQL sécurisées pour insérer, mettre à jour et effacer des données dans une table.

📤 INSERT INTO : Ajouter des enregistrements

La commande INSERT INTO permet d'ajouter une ou plusieurs lignes dans une table.

Syntaxe de base

INSERT INTO nom_table (colonne1, colonne2, ...)
VALUES (valeur1, valeur2, ...);

🔹 Si vous insérez toutes les colonnes dans l'ordre, vous pouvez omettre les noms de colonnes.

📘 Exemple concret (table "étudiants")

INSERT INTO étudiants (id, nom, age, filière)
VALUES (1, 'Moussa Diallo', 20, 'Informatique');

✅ Insertion d'un nouvel étudiant.

-- Insertion multiple :
INSERT INTO étudiants (id, nom, age) VALUES 
(2, 'Aïssa Ndiaye', 22),
(3, 'Paul Biya', 21);
💡 Astuce: Toujours vérifier les types des colonnes (INT, VARCHAR, DATE) pour éviter les erreurs.

✏️ UPDATE ... SET : Modifier les données

La commande UPDATE modifie les valeurs existantes. Attention : sans WHERE, toutes les lignes seront modifiées !

UPDATE nom_table
SET colonne1 = nouvelle_valeur, colonne2 = ...
WHERE condition;

📖 Exemple :

-- Augmenter l'âge de l'étudiant dont l'id est 2
UPDATE étudiants
SET age = 23
WHERE id = 2;

-- Modifier plusieurs colonnes
UPDATE étudiants
SET nom = 'Moussa Diallo Junior', age = 21
WHERE id = 1;

⚠️ Piège fréquent : Oublier le WHERE entraîne des mises à jour massives et irréversibles. Toujours tester avec SELECT d'abord.

🔍Vous pouvez utiliser des expressions: SET prix = prix * 1.10 pour appliquer une augmentation.

🗑️ DELETE FROM : Supprimer des lignes

DELETE efface des enregistrements d'une table. Sans WHERE, toute la table est vidée !

DELETE FROM nom_table WHERE condition;
-- Supprimer l'étudiant ayant l'id 3
DELETE FROM étudiants WHERE id = 3;

-- Supprimer les étudiants de moins de 18 ans
DELETE FROM étudiants WHERE age < 18;

🧹 TRUNCATE vs DELETE

TRUNCATE TABLE supprime toutes les lignes plus rapidement mais ne peut pas utiliser WHERE. Idéal pour vider une table.

📌 Bonne pratique

Faire un BEGIN TRANSACTION (sur SGBD supportant) ou une sauvegarde avant de faire un DELETE massif.

🔧 Aller plus loin : conditions combinées

Les trois commandes marchent avec les mêmes conditions que SELECT (AND, OR, IN, LIKE...).

INSERT avec sous-requête

INSERT INTO archive_étudiants
SELECT * FROM étudiants WHERE age > 25;

UPDATE conditionnelle

UPDATE produits
SET prix = prix * 0.85
WHERE categorie = 'électronique' AND stock > 0;

DELETE avec jointure (dans certains SGBD comme MySQL)

DELETE étudiants FROM étudiants
LEFT JOIN inscriptions ON étudiants.id = inscriptions.etudiant_id
WHERE inscriptions.id IS NULL;

👉 Supprime les étudiants sans aucune inscription.

🧠 Rappel :

Avant d'exécuter UPDATE ou DELETE, écrivez d'abord un SELECT avec la même clause WHERE pour visualiser les enregistrements concernés.

📝 Synthèse & exercice rapide

📋 Récapitulatif

  • INSERT ➕ Ajoute des lignes.
  • UPDATE ✏️ Modifie les valeurs.
  • DELETE 🗑️ Supprime des enregistrements.
  • ⚠️ WHERE est crucial pour UPDATE/DELETE !

🧪 Mini quiz (mental)

1. Quelle commande utiliserait-on pour effacer tous les étudiants dont la note est inférieure à 10 ?

👉 DELETE FROM etudiants WHERE note < 10;

2. Que se passe-t-il si on exécute UPDATE employes SET salaire = 0 sans WHERE ?

👉 Tous les salaires des employés deviennent zéro !
