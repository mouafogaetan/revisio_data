Fonctions stockées
vs Procédures stockées

Comprendre la différence, créer et utiliser des fonctions SQL dans vos requêtes

Cours interactif · Approche pratique

🎯 Objectifs de la leçon

  • 1 Distinguer fonction stockée et procédure stockée : comprendre leurs différences fondamentales (retour de valeur, utilisation dans SELECT).
  • 2 Créer une fonction avec RETURN : écrire une fonction SQL qui retourne une valeur calculée.
  • 3 Utiliser les fonctions dans les requêtes SELECT : exploiter les fonctions comme outil d’enrichissement des données.
  • 4 Bonnes pratiques et exemples concrets : savoir quand utiliser une fonction plutôt qu'une procédure.
💡 À la fin de cette leçon, vous pourrez créer vos propres fonctions SQL et les intégrer naturellement dans vos projets web.

🧠 Fonction vs Procédure : les bases

📦 Procédure stockée
  • Ne retourne pas obligatoirement une valeur (peut avoir des paramètres OUT).
  • Peut modifier des données (INSERT, UPDATE, DELETE) et exécuter plusieurs instructions.
  • S’appelle avec CALL nom_procedure(...).
  • Ne peut pas être utilisée directement dans une requête SELECT.
⚡ Fonction stockée
  • Retourne TOUJOURS une valeur (scalaire ou table).
  • Idéale pour les calculs, transformations ou règles métier simples.
  • S’utilise dans SELECT, WHERE, ORDER BY.
  • En général, elle ne modifie pas les données (purement calcul).
🔍 CREATE FUNCTION vs CREATE PROCEDURE — la différence clé réside dans la clause RETURNS et l’appel.

✍️ Créer une fonction avec RETURN

Exemple sur un environnement MySQL / MariaDB (adaptable à PostgreSQL).

DELIMITER $$

CREATE FUNCTION calculer_tva(prix_ht DECIMAL(10,2), taux_tva DECIMAL(5,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    DECLARE tva DECIMAL(10,2);
    SET tva = prix_ht * (taux_tva / 100);
    RETURN tva;
END$$

DELIMITER ;
📌 RETURNS : définit le type de la valeur retournée.
📌 DETERMINISTIC : toujours le même résultat pour mêmes entrées.
📌 RETURN : mot-clé qui renvoie la valeur finale.
📌 Corps de la fonction : peut contenir variables locales, conditions, boucles.

📊 Utiliser les fonctions dans SELECT

Une fois la fonction créée, elle s’utilise comme n’importe quelle fonction native.

🛒 Exemple : liste de produits avec TVA calculée

SELECT 
    nom_produit, 
    prix_ht, 
    calculer_tva(prix_ht, 20) AS montant_tva,
    prix_ht + calculer_tva(prix_ht, 20) AS prix_ttc
FROM produits;

✅ La fonction calculer_tva est appelée pour chaque ligne, améliore la réutilisabilité et la lisibilité.

💡 Astuce : Vous pouvez aussi utiliser une fonction dans les clauses WHERE : SELECT * FROM commandes WHERE calculer_remise(total) > 100;

🔧 Exemple concret : fonction de formatage

Créons une fonction qui renvoie le nom complet d’un employé à partir du prénom et nom.

CREATE FUNCTION nom_complet(prenom VARCHAR(50), nom VARCHAR(50))
RETURNS VARCHAR(101)
DETERMINISTIC
BEGIN
    RETURN CONCAT(UPPER(LEFT(prenom,1)), LOWER(SUBSTRING(prenom,2)), 
                  ' ', UPPER(nom));
END
✅ Utilisation dans SELECT : 
SELECT id_employe, nom_complet(prenom, nom) AS employe_formate FROM employes;

Résultat : "Jean DUPONT" au lieu de "jean dupont".

🏆 Avantages : centralisation de la logique, maintenance simplifiée.

⚠️ Selon votre SGBD (MySQL, PostgreSQL, SQL Server), les détails syntaxiques peuvent varier, mais les concepts restent similaires.

🎯 Bonnes pratiques & cas d'usage

✅
Utilisez une fonction pour : calculs, conversions, règles de validation, formatage de texte, extraction de sous-chaînes complexes.
⚠️
Évitez les fonctions pour : lourdes manipulations de données, appels à des API externes (sauf exception) ou opérations non déterministes aléatoires.

👉 Toujours nommer vos fonctions avec un préfixe explicite (ex: `fn_calcul_total`, `get_solde_client`).

💡 Petite mise en garde : trop de fonctions dans une requête complexe peut ralentir les performances, mais bien utilisées, elles améliorent la clarté.

🧪 Mini exercice pratique

« Dans une base e-commerce, écrivez une fonction remise_fidelite(montant_total INT, anciennete_mois INT) qui retourne 5% de réduction si ancienneté > 12 mois, sinon 0%. Utilisez-la dans une requête pour afficher le montant final. »
🔎 Voir une solution possible (MySQL) 
CREATE FUNCTION remise_fidelite(montant DECIMAL(10,2), anciennete INT)
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    DECLARE reduction DECIMAL(10,2);
    IF anciennete > 12 THEN
        SET reduction = montant * 0.05;
    ELSE
        SET reduction = 0;
    END IF;
    RETURN reduction;
END;

-- Utilisation :
SELECT client, total_achats, remise_fidelite(total_achats, mois_inscription) AS remise FROM clients;
