🎯 Objectifs de la leçon

  • ✓ Transformer les entités en tables : Comprendre comment chaque entité devient une table et chaque attribut une colonne.
  • ✓ Transformer les associations en clés étrangères ou tables d’association : Modéliser les relations (1-1, 1-N, N-N) proprement en SQL.
  • ✓ Normaliser les données (1NF, 2NF, 3NF) : Éliminer les redondances et garantir l’intégrité des données.
💡 À la fin de cette leçon, vous saurez convertir un diagramme ER en un schéma relationnel fonctionnel, prêt à être implémenté en SQL.

📌 Étape 1 : Transformer les entités en tables

Chaque entité du diagramme Entité-Association devient une table dans la base de données. Chaque attribut devient une colonne avec son type de données.

📇 Exemple : Entité "Étudiant"

CREATE TABLE Etudiant (
    id_etudiant INT PRIMARY KEY,
    nom VARCHAR(50),
    prenom VARCHAR(50),
    date_naissance DATE,
    email VARCHAR(100) UNIQUE
);

📚 Entité "Cours"

CREATE TABLE Cours (
    id_cours INT PRIMARY KEY,
    titre VARCHAR(100),
    credits INT NOT NULL
);

✅ Règle d'or : Une clé primaire (Primary Key) unique est définie pour chaque table, souvent un identifiant technique.

🔗 Étape 2 : Associations → Clés étrangères ou tables d'association

Selon la cardinalité, le passage au relationnel change :

1 - N (un à plusieurs)

Ajouter la clé primaire de l'entité "un" comme clé étrangère dans la table du côté "plusieurs".

-- Ex: Un professeur donne plusieurs cours
ALTER TABLE Cours ADD COLUMN professeur_id INT;
ALTER TABLE Cours ADD FOREIGN KEY (professeur_id) REFERENCES Professeur(id);
N - N (plusieurs à plusieurs)

Créer une table d'association contenant les deux clés étrangères (clé primaire composite).

CREATE TABLE Inscription (
    id_etudiant INT,
    id_cours INT,
    date_inscription DATE,
    PRIMARY KEY (id_etudiant, id_cours),
    FOREIGN KEY (id_etudiant) REFERENCES Etudiant(id),
    FOREIGN KEY (id_cours) REFERENCES Cours(id)
);
1 - 1 (un à un)

Choisir une table qui référence l'autre avec une clé étrangère unique.

📐 Étape 3 : Normaliser les données (1NF, 2NF, 3NF)

La normalisation évite la redondance et les anomalies. En pratique, on vise au moins la 3e forme normale (3NF).

🔹 1NF – Première forme normale

Chaque cellule contient une valeur unique (pas de liste ni de groupe répété).

❌ "téléphones: 677889900, 655112233" → ✅ Créer une table séparée Téléphone.

🔹 2NF – Deuxième forme normale

Déjà en 1NF + tous les attributs non clé dépendent de la clé primaire entière (pas de dépendance partielle).

-- Table avec clé composite (id_commande, id_produit) -> prix_produit ne dépend que id_produit → 2NF violée

🔹 3NF – Troisième forme normale

2NF + aucun attribut non clé ne dépend d'un autre attribut non clé (transitivité).

Exemple: (code_postal → ville) : extraire ville dans une table séparée.

🧠 Astuce mémo : "Chaque donnée n’est stockée qu’une seule fois, et dépend directement de la clé."

🏗️ Exemple complet : Système universitaire

À partir du DER suivant (entités : Étudiant, Cours, Professeur). Appliquons transformation + normalisation.

-- 1. Tables d'entités
CREATE TABLE Etudiant (
  id_etudiant INT PRIMARY KEY,
  nom VARCHAR(40),
  email VARCHAR(100) UNIQUE
);

CREATE TABLE Professeur (
  id_prof INT PRIMARY KEY,
  nom_prof VARCHAR(40),
  specialite VARCHAR(80)
);

CREATE TABLE Cours (
  id_cours INT PRIMARY KEY,
  titre VARCHAR(100),
  professeur_id INT,
  FOREIGN KEY (professeur_id) REFERENCES Professeur(id_prof)
);

-- 2. Table d'association (N-N entre Etudiant et Cours)
CREATE TABLE Inscription (
  etudiant_id INT,
  cours_id INT,
  annee INT,
  note DECIMAL(4,2),
  PRIMARY KEY (etudiant_id, cours_id),
  FOREIGN KEY (etudiant_id) REFERENCES Etudiant(id_etudiant),
  FOREIGN KEY (cours_id) REFERENCES Cours(id_cours)
);

✨ Résultat : une base propre, sans redondance, prête à interroger avec des jointures.

✏️ Vérifions la compréhension

📌 Question 1

Une association "Plusieurs à plusieurs" se traduit toujours par :

  • A) Une clé étrangère dans l’une des deux tables
  • B) Une table d’association
  • C) Un trigger

✅ Réponse: B) Table d'association (ex: table Inscription).

📌 Question 2

Que fait la 3ème forme normale (3NF) ?

  • A) Supprimer les dépendances transitives
  • B) Créer des index
  • C) Supprimer les clés étrangères

✅ Réponse: A) Supprimer les dépendances transitives (éviter qu’un attribut non clé dépende d’un autre attribut non clé).

📌 Points clés de la leçon 6

1Entité → Table : Chaque entité devient une table, attributs = colonnes.
2Association 1-N : clé étrangère dans la table du côté "N".
3Association N-N : création d'une table de liaison avec clés étrangères.
4Normalisation : 1NF (valeurs atomiques), 2NF (dépendance totale), 3NF (pas de transitivité).
