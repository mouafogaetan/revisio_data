📖 Vous êtes stagiaire dans la startup "BookVerse", une librairie en ligne spécialisée.

Le responsable technique vous fournit le schéma relationnel simplifié ci-dessous. À l'aide de requêtes SQL (MySQL / PostgreSQL compatible), vous allez répondre aux besoins métier.

Schéma de la base

📚 Table `livres`

livres(
  id_livre INT PRIMARY KEY,
  titre VARCHAR(150),
  auteur VARCHAR(100),
  prix DECIMAL(6,2),
  stock INT,
  date_publication DATE,
  id_categorie INT
)

🏷️ Table `categories`

categories(
  id_categorie INT PRIMARY KEY,
  nom_categorie VARCHAR(50)
)

👥 Table `clients`

clients(
  id_client INT PRIMARY KEY,
  nom VARCHAR(60),
  email VARCHAR(100) UNIQUE,
  ville VARCHAR(80),
  date_inscription DATE
)

🛒 Table `commandes`

commandes(
  id_commande INT PRIMARY KEY,
  id_client INT,
  date_commande DATETIME,
  montant_total DECIMAL(8,2),
  FOREIGN KEY (id_client) REFERENCES clients(id_client)
)

(Une commande peut avoir plusieurs lignes détaillées dans `lignes_commandes`)

📝 Table `lignes_commandes`

lignes_commandes(
  id_commande INT,
  id_livre INT,
  quantite INT,
  prix_unitaire DECIMAL(6,2),
  PRIMARY KEY (id_commande, id_livre),
  FOREIGN KEY (id_commande) REFERENCES commandes(id_commande),
  FOREIGN KEY (id_livre) REFERENCES livres(id_livre)
)

  • B) DROP TABLE livres;
  • C) TRUNCATE TABLE livres;
  • D) REMOVE FROM livres;

2 Quelle clause SQL permet d’exclure les doublons d’une requête SELECT ?

  • A) UNIQUE
  • B) DISTINCT
  • C) DIFFERENT
  • D) GROUP BY

3 Une requête nécessite de récupérer les livres dont le prix est compris entre 12 et 25 euros (inclus). Quel WHERE est correct ?

  • A) WHERE prix BETWEEN 12 AND 25
  • B) WHERE prix IN (12,25)
  • C) WHERE prix >= 12 AND prix <= 25
  • D) A et C sont correctes
(Répondez sur votre copie : 1-C, 2-B, 3-D)
