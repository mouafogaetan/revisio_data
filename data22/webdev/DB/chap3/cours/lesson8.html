🏗️ CREATE TABLE : Créer une table

Pour créer une nouvelle table, on utilise la commande CREATE TABLE suivie du nom de la table et de la définition des colonnes.

CREATE TABLE nom_table (
    colonne1 type_données contraintes,
    colonne2 type_données contraintes,
    ...
);

Exemple concret : création d'une table 'etudiants'

CREATE TABLE etudiants (
    id INT PRIMARY KEY,
    nom VARCHAR(50) NOT NULL,
    prenom VARCHAR(50),
    age INT,
    inscription DATE
);

🔑 La colonne id est la clé primaire. NOT NULL signifie que 'nom' doit toujours être rempli.

✏️ ALTER TABLE : modifier la structure

Une fois la table créée, on peut ajouter, supprimer ou modifier des colonnes avec ALTER TABLE.

➕ ADD colonne

ALTER TABLE etudiants ADD email VARCHAR(100);

Ajoute une colonne 'email' à la table existante.

❌ DROP colonne

ALTER TABLE etudiants DROP COLUMN age;

Supprime définitivement la colonne 'age' (attention aux données).

🔄 MODIFY / ALTER COLUMN (type)

-- MySQL / PostgreSQL
ALTER TABLE etudiants MODIFY COLUMN nom VARCHAR(100);
-- ou
ALTER TABLE etudiants ALTER COLUMN prenom TYPE VARCHAR(80);

Permet de changer le type de données ou la taille d'une colonne.

⚠️ La modification peut échouer si les données existantes ne sont pas compatibles.

🗑️ Supprimer une table : DROP vs TRUNCATE

Il existe plusieurs manières de supprimer des données ou la structure entière.

CommandeEffetRestauration possible ?
DROP TABLE ma_table;Supprime la table + sa structure + toutes les données❌ Non (sauf backup)
TRUNCATE TABLE ma_table;Supprime TOUTES les lignes, mais garde la structure (vide)⚠️ Non (pas de rollback simple)
DELETE FROM ma_table;Supprime les lignes (avec WHERE possible) transactionnel✅ Oui (avec COMMIT/ROLLBACK)

DROP TABLE = enlève table et tout son contenu.
TRUNCATE = vide toutes les lignes plus rapidement que DELETE.

📌 Exemples complets & bonnes pratiques

🔹 Création table 'produits' avec contraintes :

CREATE TABLE produits (
    id_produit INT PRIMARY KEY AUTO_INCREMENT,
    nom_produit VARCHAR(100) NOT NULL,
    prix DECIMAL(10,2) CHECK (prix >= 0),
    stock INT DEFAULT 0
);

🔹 Ajout d'une colonne 'categorie' après création :

ALTER TABLE produits ADD categorie VARCHAR(50);

🔹 Supprimer la table entière (irréversible !) :

DROP TABLE produits;

🔹 Vider rapidement une table (supprime toutes les lignes) :

TRUNCATE TABLE commandes;

Contrairement à DELETE, TRUNCATE réinitialise aussi les compteurs d'auto-incrémentation.

💡 Conseil : Avant de faire DROP TABLE, assurez-vous d’avoir une sauvegarde ! En environnement scolaire, entraînez-vous sur des tables de test.

🧠 Résumé & Petit test de compréhension

📖 Ce qu’il faut retenir :

  • CREATE TABLE → créer une table de zéro.
  • ALTER TABLE (ADD/DROP/MODIFY) → ajuster le schéma sans perdre les données existantes.
  • DROP TABLE → supprimer table + structure.
  • TRUNCATE → vider rapidement toutes les lignes.
