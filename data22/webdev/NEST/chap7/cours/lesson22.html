🔄 Pourquoi asynchrone ?

Parfois la configuration d'un module nécessite :

  • De lire des variables d'environnement via ConfigService (qui peut être asynchrone).
  • De récupérer des secrets depuis un gestionnaire externe (Vault, AWS Secrets Manager).
  • D'attendre la connexion à une base de données avant d'exposer certaines routes.
Exemple : TypeORM a besoin de connaître l'URL de la base de données après chargement du fichier .env.
💡 La solution : registerAsync() / forRootAsync() avec useFactory et inject.

📦 ConfigModule asynchrone (exemple simple)

// app.module.ts import { Module } from '@nestjs/common'; import { ConfigModule, ConfigService } from '@nestjs/config'; @Module({ imports: [ ConfigModule.forRoot({ isGlobal: true, envFilePath: ['.env.development', '.env'], }), // Autre module asynchrone DatabaseModule.forRootAsync({ imports: [ConfigModule], useFactory: (configService: ConfigService) => ({ host: configService.get('DB_HOST'), port: configService.get('DB_PORT'), }), inject: [ConfigService], }), ], }) export class AppModule {}

useFactory est appelée après que ConfigService soit prêt. Les valeurs sont injectées dynamiquement.

✅ Bonnes pratiques : Toujours utiliser inject pour que NestJS résolve les dépendances automatiquement.

🗄️ TypeORM asynchrone – Exemple concret

// database.module.ts import { Module } from '@nestjs/common'; import { TypeOrmModule } from '@nestjs/typeorm'; import { ConfigModule, ConfigService } from '@nestjs/config'; @Module({ imports: [ TypeOrmModule.forRootAsync({ imports: [ConfigModule], useFactory: (configService: ConfigService) => ({ type: 'postgres', host: configService.get('DB_HOST'), port: configService.get('DB_PORT'), username: configService.get('DB_USER'), password: configService.get('DB_PASSWORD'), database: configService.get('DB_NAME'), entities: [__dirname + '/../**/*.entity{.ts,.js}'], synchronize: configService.get('NODE_ENV') !== 'production', }), inject: [ConfigService], }), ], }) export class DatabaseModule {}

Grâce à cette approche, on évite de dupliquer la configuration. La synchronisation automatique est désactivée en production.

⚠️ synchronize: true en développement uniquement — dangereux en production.

🔐 JwtModule asynchrone

// auth.module.ts import { Module } from '@nestjs/common'; import { JwtModule } from '@nestjs/jwt'; import { ConfigModule, ConfigService } from '@nestjs/config'; @Module({ imports: [ JwtModule.registerAsync({ imports: [ConfigModule], useFactory: (config: ConfigService) => ({ secret: config.get('JWT_SECRET'), signOptions: { expiresIn: config.get('JWT_EXPIRES_IN', '1h') }, }), inject: [ConfigService], }), ], }) export class AuthModule {}

Les variables JWT_SECRET et JWT_EXPIRES_IN sont lues depuis .env.

Avantage : le secret n'est pas codé en dur, modifiable sans rebuild.

🧩 Modules configurables : forRoot, register

NestJS permet de créer des modules dynamiques qui peuvent être configurés différemment selon le contexte.

// mon-module.config.ts export interface MonModuleOptions { apiKey: string; timeout: number; } // mon-module.module.ts @Module({}) export class MonModule { static register(options: MonModuleOptions): DynamicModule { return { module: MonModule, providers: [ { provide: 'CONFIG_OPTIONS', useValue: options, }, MonService, ], exports: [MonService], }; } }
Utilisation : MonModule.register({ apiKey: 'abc', timeout: 5000 })
✨ Courant dans les librairies tierces (ex: GraphQLModule.forRoot()).

🎛️ Configuration conditionnelle

On peut charger des fichiers différents selon l'environnement :

// .env.development, .env.production ConfigModule.forRoot({ envFilePath: `.env.${process.env.NODE_ENV}`, ignoreEnvFile: process.env.NODE_ENV === 'production', });

Ou encore activer un module seulement si une variable est définie :

imports: [ process.env.USE_CACHE === 'true' ? CacheModule.register() : [], ],
🔧 Utile pour désactiver la documentation Swagger en production, etc.

✏️ Exercice & bonnes pratiques

Exercice guidé :

  1. Créez un fichier .env avec DB_HOST, DB_PORT, JWT_SECRET.
  2. Configurez ConfigModule.forRoot() global.
  3. Modifiez TypeOrmModule pour utiliser forRootAsync et injecter ConfigService.
  4. Testez avec NODE_ENV=production pour désactiver synchronize.
💡 Rappel : Ne jamais logguer les secrets (mot de passe, clé API). Utilisez ConfigService.get() partout.
