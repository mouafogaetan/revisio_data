Lesson 24 — Modules de test et mocks avancés

💡 Saviez-vous ? NestJS utilise Jest par défaut, mais on peut aussi utiliser Jasmine ou Vitest.

🧪 Test.createTestingModule

Pour tester des composants isolément, on crée un module de test spécial :

const moduleRef = await Test.createTestingModule({
  imports: [],
  controllers: [CatController],
  providers: [CatService, ValidationPipe, RolesGuard],
}).compile();

On peut ensuite récupérer les instances avec moduleRef.get().

🔍 Avantage : on n'a pas besoin de démarrer tout le serveur HTTP. Tests plus rapides et ciblés.

📏 Tester un Pipe personnalisé

Prenons un pipe ParseIntPipe qui transforme une chaîne en entier.

// parse-int.pipe.ts
@Injectable()
export class ParseIntPipe implements PipeTransform {
  transform(value: string) {
    const val = parseInt(value, 10);
    if (isNaN(val)) throw new BadRequestException('Invalid number');
    return val;
  }
}

Test unitaire :

it('should transform "42" to 42', () => {
  const pipe = new ParseIntPipe();
  expect(pipe.transform('42')).toBe(42);
});

it('should throw when NaN', () => {
  const pipe = new ParseIntPipe();
  expect(() => pipe.transform('abc')).toThrow(BadRequestException);
});

🛡️ Tester un Guard (ex: RolesGuard)

Un guard vérifie les rôles avant d’accorder l’accès à une route.

@Injectable()
export class RolesGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const requiredRoles = this.reflector.get<string[]>('roles', context.getHandler());
    const { user } = context.switchToHttp().getRequest();
    return requiredRoles.some(role => user.roles?.includes(role));
  }
}

Test avec jest.spyOn pour mocker le reflector :

it('should allow admin user', () => {
  const mockReflector = { get: jest.fn().mockReturnValue(['admin']) };
  const guard = new RolesGuard(mockReflector as any);
  const context = { switchToHttp: () => ({ getRequest: () => ({ user: { roles: ['admin'] } }) }) } as ExecutionContext;
  expect(guard.canActivate(context)).toBe(true);
});

⚡ Tester un Interceptor

Un interceptor peut logger ou transformer la réponse. Exemple : LoggingInterceptor

intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
  console.log('Before...');
  return next.handle().pipe(tap(() => console.log('After...')));
}

Test avec un of mock :

it('should log before and after', (done) => {
  const interceptor = new LoggingInterceptor();
  const mockCallHandler = { handle: () => of('test') };
  interceptor.intercept({} as ExecutionContext, mockCallHandler).subscribe({
    next: (val) => expect(val).toBe('test'),
    complete: done
  });
});

On peut aussi utiliser jest.spyOn(console, 'log') pour vérifier les logs.

🎭 jest.spyOn – le super pouvoir

jest.spyOn(object, methodName) permet de surveiller et mocker une méthode sans perdre l'implémentation originale si besoin.

const userService = new UserService();
const spy = jest.spyOn(userService, 'findOne').mockResolvedValue({ id: 1, name: 'John' });
// dans le test
await userService.findOne(1);
expect(spy).toHaveBeenCalledWith(1);
spy.mockRestore(); // restaure la méthode originale
⚠️ Important : Toujours restaurer les mocks après chaque test (afterEach) pour éviter les interférences.

🔁 Intégration avec un contrôleur

Exemple complet : tester un contrôleur avec un guard mocké et un pipe.

const module = await Test.createTestingModule({
  controllers: [ProductController],
  providers: [ProductService, RolesGuard, ValidationPipe],
})
.overrideGuard(RolesGuard).useValue({ canActivate: () => true })
.compile();

const controller = module.get(ProductController);
const result = await controller.create({ name: 'Laptop' });
expect(result).toEqual({ id: expect.any(Number) });

overrideGuard / overridePipe / overrideInterceptor remplacent facilement les composants globaux.

📊 Schéma de la stratégie de test

Chaque élément (guard, pipe, interceptor) se teste individuellement avec le module de test.

🏁 Récapitulatif & bonnes pratiques

  • ✅ Toujours isoler les composants lors des tests unitaires.
  • ✅ Test.createTestingModule remplace le module principal.
  • ✅ Utiliser .overrideProvider, .overrideGuard pour mocker des dépendances complexes.
  • ✅ jest.spyOn est idéal pour vérifier qu’une méthode a été appelée.
  • ✅ Penser à nettoyer les mocks avec jest.clearAllMocks() dans afterEach.
🧪 À retenir : Test unitaire = rapidité, contrôle, confiance dans le déploiement.
