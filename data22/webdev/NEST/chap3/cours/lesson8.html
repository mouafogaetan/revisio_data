ValidationPipe et pipes personnalisés

Sécuriser vos endpoints avec des validations automatiques

ValidationPipe + class-validator = données propres et sûres

Objectifs pédagogiques

ValidationPipe global – Configuration et activation
Whitelist & forbidNonWhitelisted – Sécurité renforcée
Transform – Conversion automatique des types
Pipes personnalisés – Création et utilisation

Une validation robuste = moins de bugs, plus de sécurité.

📦 Installation des dépendances

npm install class-validator class-transformer

Configuration du ValidationPipe global dans main.ts :

import { NestFactory } from '@nestjs/core';
import { ValidationPipe } from '@nestjs/common';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  
  app.useGlobalPipes(new ValidationPipe({
    whitelist: true,           // Supprime les propriétés non définies dans le DTO
    forbidNonWhitelisted: true, // Rejette les propriétés non définies (erreur 400)
    transform: true,           // Convertit automatiquement les types (string → number)
    transformOptions: { enableImplicitConversion: true },
  }));
  
  await app.listen(3000);
}
bootstrap();
Explication : whitelist: true nettoie les champs superflus, forbidNonWhitelisted bloque la requête si un champ inattendu est présent.

📝 Création d'un DTO avec class-validator

// create-user.dto.ts
import { IsString, IsEmail, IsInt, Min, Max, IsOptional, IsPhoneNumber } from 'class-validator';

export class CreateUserDto {
  @IsString({ message: 'Le nom doit être une chaîne de caractères' })
  @IsOptional()
  name?: string;

  @IsEmail({}, { message: 'Email invalide' })
  email: string;

  @IsInt()
  @Min(18, { message: 'Âge minimum : 18 ans' })
  @Max(120, { message: 'Âge maximum : 120 ans' })
  age: number;

  @IsPhoneNumber('CM', { message: 'Numéro camerounais invalide' })
  phone: string;
}
Le pipe ValidationPipe valide automatiquement tout body annoté avec @Body()

⚙️ Options avancées

whitelist: true

Supprime automatiquement les propriétés qui n'ont pas de décorateur dans le DTO.

forbidNonWhitelisted: true

Retourne une erreur 400 si des propriétés non whitelistées sont présentes.

transform: true

Convertit les types : "123" string → 123 number, "true" → true boolean, etc.

// Exemple d'erreur avec forbidNonWhitelisted
// POST /users Body: { "name": "John", "email": "john@test.com", "admin": true }
// Réponse: 400 Bad Request - "Property admin should not exist"

🔧 Création d'un pipe personnalisé

Un pipe transforme ou valide une donnée avant qu'elle n'atteigne le contrôleur.

// parse-id.pipe.ts
import { PipeTransform, Injectable, BadRequestException } from '@nestjs/common';

@Injectable()
export class ParseIdPipe implements PipeTransform {
  transform(value: any) {
    const id = parseInt(value, 10);
    if (isNaN(id)) {
      throw new BadRequestException(`L'ID "${value}" n'est pas un nombre valide`);
    }
    if (id <= 0) {
      throw new BadRequestException(`L'ID doit être positif`);
    }
    return id;
  }
}

// Utilisation dans un contrôleur
@Get(':id')
async findOne(@Param('id', ParseIdPipe) id: number) {
  // id est déjà un nombre valide
  return this.userService.findOne(id);
}

🔍 Validation des query parameters

Les paramètres de requête peuvent aussi être validés avec un DTO et ValidationPipe.

// pagination.dto.ts
import { IsInt, Min, Max, IsOptional } from 'class-validator';
import { Type } from 'class-transformer';

export class PaginationDto {
  @IsOptional()
  @Type(() => Number)  // transforme la string en number
  @IsInt()
  @Min(1)
  page: number = 1;

  @IsOptional()
  @Type(() => Number)
  @IsInt()
  @Min(1)
  @Max(100)
  limit: number = 10;
}

// Contrôleur
@Get('users')
async findAll(@Query() pagination: PaginationDto) {
  return this.userService.findAll(pagination);
}
La validation fonctionne aussi avec @Param(), @Query(), @Body()

⚠️ Personnalisation des réponses d'erreur

Par défaut, NestJS renvoie un objet avec les erreurs. Vous pouvez personnaliser via un filtre d'exception.

// validation-exception.filter.ts
import { ExceptionFilter, Catch, ArgumentsHost, BadRequestException } from '@nestjs/common';

@Catch(BadRequestException)
export class ValidationExceptionFilter implements ExceptionFilter {
  catch(exception: BadRequestException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const status = exception.getStatus();
    const exceptionResponse = exception.getResponse() as any;

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      message: 'Erreur de validation',
      errors: exceptionResponse.message || exceptionResponse,
    });
  }
}
