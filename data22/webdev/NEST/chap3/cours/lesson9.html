Transformer & sérialiser avec class-transformer

NestJS – Lesson 9 | Chapitre 3 : Validation et transformation

class-transformer @Transform @Exclude @SerializeOptions

🎯 Objectifs de la leçon

  • ✓ Transformer les types avec class-transformer dans NestJS.
  • ✓ Utiliser @Transform() pour modifier la valeur d’une propriété.
  • ✓ Exclure des champs sensibles avec @Exclude().
  • ✓ Configurer la sérialisation des réponses avec @SerializeOptions().
Pourquoi c'est important ? Quand vous renvoyez un utilisateur, vous ne voulez pas exposer son mot de passe. class-transformer vous aide à contrôler les données sortantes.

📦 Qu'est-ce que class-transformer ?

C'est une bibliothèque qui permet de transformer des objets plats en instances de classes et vice‑versa. Avec NestJS, elle est souvent utilisée avec ValidationPipe et l'option transform: true.

// Exemple basique : transformer une chaîne en nombre

import { Transform } from 'class-transformer';

class ProductDTO {
  @Transform(({ value }) => parseInt(value, 10))
  price: number;
}

Installation : npm install class-transformer (souvent déjà présente avec @nestjs/mapped-types).

⚙️ Décorateurs clés : @Transform et @Exclude

🔁 @Transform

Modifie la valeur d’une propriété lors de la transformation.

// Exemple : formater une date
@Transform(({ value }) => new Date(value))
createdAt: Date;

🚫 @Exclude

Exclut une propriété lors de la sérialisation (ex: mot de passe).

// Cache le mot de passe
@Exclude()
password: string;
💡 Bon à savoir : @Expose() permet au contraire d'inclure un champ même s'il n'existe pas dans l'objet source.

📤 Sérialiser les réponses (controlleurs)

Dans NestJS, le décorateur @SerializeOptions() (ou l'interceptor ClassSerializerInterceptor) permet d’appliquer des règles de transformation globalement ou localement.

import { Controller, Get, UseInterceptors, ClassSerializerInterceptor } from '@nestjs/common';
import { UserEntity } from './user.entity';

@Controller('users')
@UseInterceptors(ClassSerializerInterceptor)  // Active la sérialisation
export class UserController {
  @Get()
  findAll() {
    // Les propriétés @Exclude() seront automatiquement retirées
    return new UserEntity({ id:1, name:"Alice", password:"secret" });
  }
}

Résultat renvoyé au client : { "id": 1, "name": "Alice" } → le mot de passe est exclu !

🔧 Cas pratique : API utilisateur sécurisée

// user.dto.ts
import { Exclude, Transform } from 'class-transformer';

export class UserResponseDto {
  id: number;

  @Transform(({ value }) => value.toUpperCase())
  name: string;

  @Exclude()
  password: string;

  @Transform(({ value }) => value?.toISOString().split('T')[0])
  createdAt: Date;
}
✔️ name en majuscules ✔️ password masqué ✔️ date formatée en YYYY-MM-DD

N'oubliez pas d'activer transform: true dans le ValidationPipe global pour que class-transformer s'applique automatiquement.
