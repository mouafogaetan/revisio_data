Partie 2

Exercice Pratique – Module & Injection

/7 points

Contexte : Vous développez une API de gestion de produits. Inspectez le code ci-dessous, puis répondez aux questions.

// product.service.ts
import { Injectable } from '@nestjs/common';

@Injectable()
export class ProductService {
  private products = [];

  getAll() {
    return this.products;
  }

  create(productDto) {
    this.products.push(productDto);
    return productDto;
  }
}

// product.controller.ts
import { Controller, Get, Post, Body } from '@nestjs/common';
import { ProductService } from './product.service';

@Controller('products')
export class ProductController {
  constructor(private readonly productService: ProductService) {}

  @Get()
  fetchAll() {
    return this.productService.getAll();
  }

  @Post()
  addProduct(@Body() newProduct) {
    return this.productService.create(newProduct);
  }
}

2.1 (2 pts) – Quel est le rôle du décorateur @Injectable() dans la classe ProductService ? Expliquez comment NestJS va instancier ce service.

2.2 (2 pts) – Pourquoi le constructeur du contrôleur reçoit-il productService en paramètre ? Quel principe de conception NestJS illustre-t-il ?
