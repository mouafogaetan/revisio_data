🏗️ Structure de base d'un contrôleur

Un contrôleur NestJS regroupe des routes (endpoints) liées à une ressource.


// products.controller.ts
import { Controller, Get, Post, Body, Param } from '@nestjs/common';

@Controller('products')  // Préfixe de toutes les routes
export class ProductsController {
  
  @Get()           // GET /products
  findAll() {
    return 'Liste des produits';
  }

  @Get(':id')      // GET /products/123
  findOne(@Param('id') id: string) {
    return `Produit n°${id}`;
  }

  @Post()          // POST /products
  create(@Body() createProductDto: any) {
    return 'Produit créé';
  }
}

💡 Le décorateur @Controller('products') définit le préfixe de toutes les routes du contrôleur.

🌐 Les décorateurs HTTP

NestJS fournit des décorateurs pour chaque méthode HTTP standard.

DécorateurMéthode HTTPExemple URL
@Get()GET/products
@Post()POST/products
@Put(':id')PUT (remplacement total)/products/123
@Patch(':id')PATCH (mise à jour partielle)/products/123
@Delete(':id')DELETE/products/123
 

@Controller('products')
export class ProductsController {
  @Get() findAll() { }
  @Get(':id') findOne(@Param('id') id: string) { }
  @Post() create(@Body() dto: any) { }
  @Put(':id') update(@Param('id') id: string, @Body() dto: any) { }
  @Patch(':id') partialUpdate(@Param('id') id: string, @Body() dto: any) { }
  @Delete(':id') remove(@Param('id') id: string) { }
}

🏷️ @Param : Récupérer les paramètres d'URL

Les paramètres dynamiques sont définis avec :nom dans le chemin.


// Route : GET /products/123
@Get(':id')
findOne(@Param('id') id: string) {
  return `Produit avec l'id ${id}`;
}

// Route : GET /categories/5/products/42
@Get('categories/:catId/products/:prodId')
findProductInCategory(
  @Param('catId') catId: string,
  @Param('prodId') prodId: string
) {
  return `Catégorie ${catId}, Produit ${prodId}`;
}

// Récupérer TOUS les paramètres
@Get(':id/:slug')
getAllParams(@Param() params: any) {
  return params;  // { id: '123', slug: 'mon-produit' }
}
💡 Astuce : Typez vos paramètres avec TypeScript ! @Param('id') id: number (mais attention, viennent toujours en string).

🔍 @Query : Paramètres de requête (query string)

Pour les paramètres après le ? : /products?page=2&limit=10&search=phone


// GET /products?page=2&limit=10
@Get()
findAll(@Query('page') page: string, @Query('limit') limit: string) {
  return `Page ${page}, Limite ${limit}`;
}

// Récupérer tous les paramètres
@Get()
findAll(@Query() query: any) {
  const { page = 1, limit = 10, search = '' } = query;
  return `Recherche: ${search}, Page: ${page}, Limite: ${limit}`;
}

// Exemple réel avec pagination
@Get()
async getProducts(@Query() paginationDto: PaginationDto) {
  return this.productService.findAll(paginationDto);
}

📦 @Body : Corps de la requête (données JSON)

Utilisé pour les méthodes POST, PUT, PATCH pour recevoir les données du client.


// Définition du DTO (Data Transfer Object)
export class CreateProductDto {
  name: string;
  price: number;
  category: string;
}

// Dans le contrôleur
@Post()
create(@Body() createProductDto: CreateProductDto) {
  return this.productService.create(createProductDto);
}

// Accéder à une propriété spécifique
@Post()
create(@Body('name') name: string, @Body('price') price: number) {
  return `Produit ${name} à ${price}€`;
}

// Validation (avec class-validator vue dans lesson7)
@Post()
create(@Body() createProductDto: CreateProductDto) {
  // Les validations sont automatiquement appliquées avec ValidationPipe
}
📌 Bonnes pratiques : Utilisez toujours des DTO typés plutôt que any pour le body.

📋 @Headers, @Req, @Res

Décorateurs pour accéder aux objets natifs de la requête/réponse Express/Fastify.


import { Headers, Req, Res } from '@nestjs/common';

// Récupérer un en-tête spécifique
@Get()
getUserAgent(@Headers('user-agent') userAgent: string) {
  return `Votre navigateur : ${userAgent}`;
}

// Récupérer tous les en-têtes
@Get()
getAllHeaders(@Headers() headers: any) {
  return headers;
}

// ⚠️ Utiliser @Req et @Res (à éviter)
@Get()
getRequest(@Req() request: Request, @Res() response: Response) {
  // Problème : vous perdez les interceptors NestJS
  response.status(200).send('Hello');
}
⚠️ Attention : Évitez d'utiliser @Res() ou @Response() sauf cas particulier (streaming, fichiers). Leur utilisation vous fait perdre les interceptors et les exceptions filters de NestJS. Préférez retourner directement les valeurs.

📝 Exercices pratiques

🎯 Exercice 1 - Écrire un contrôleur (niveau easy)

Créez un contrôleur UsersController avec les routes suivantes :

  • GET /users → retourne "Liste des utilisateurs"
  • GET /users/:id → retourne "Utilisateur n°X"
  • POST /users → retourne "Utilisateur créé"
  • DELETE /users/:id → retourne "Utilisateur X supprimé"
Voir la solution 
import { Controller, Get, Post, Delete, Param, Body } from '@nestjs/common';

@Controller('users')
export class UsersController {
  @Get()
  findAll() {
    return 'Liste des utilisateurs';
  }

  @Get(':id')
  findOne(@Param('id') id: string) {
    return `Utilisateur n°${id}`;
  }

  @Post()
  create(@Body() createUserDto: any) {
    return 'Utilisateur créé';
  }

  @Delete(':id')
  remove(@Param('id') id: string) {
    return `Utilisateur ${id} supprimé`;
  }
}

🎯 Exercice 2 - Pagination avec @Query (niveau medium)

Créez une route GET /products qui accepte les paramètres page (défaut 1) et limit (défaut 10) et retourne un message avec ces valeurs.

Voir la solution 
@Get('products')
getProducts(@Query('page') page = 1, @Query('limit') limit = 10) {
  return `Page ${page}, Limite ${limit}`;
}

// Ou avec un DTO
export class PaginationDto {
  page: number = 1;
  limit: number = 10;
}

@Get('products')
getProducts(@Query() pagination: PaginationDto) {
  return `Page ${pagination.page}, Limite ${pagination.limit}`;
}

🎯 Exercice 3 - API REST complète (niveau hard)

Implémentez un contrôleur OrdersController avec les méthodes HTTP appropriées pour gérer des commandes (GET all, GET by id, POST, PUT, PATCH, DELETE).

Voir la solution 
@Controller('orders')
export class OrdersController {
  @Get()
  findAll() { return 'Toutes les commandes'; }

  @Get(':id')
  findOne(@Param('id') id: string) { return `Commande ${id}`; }

  @Post()
  create(@Body() dto: any) { return 'Commande créée'; }

  @Put(':id')
  update(@Param('id') id: string, @Body() dto: any) { return `Commande ${id} remplacée`; }

  @Patch(':id')
  partialUpdate(@Param('id') id: string, @Body() dto: any) { return `Commande ${id} modifiée partiellement`; }

  @Delete(':id')
  remove(@Param('id') id: string) { return `Commande ${id} supprimée`; }
}
