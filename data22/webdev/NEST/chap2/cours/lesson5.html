🤔 Injection de dépendances (DI)

L'injection de dépendances est un patron de conception où une classe reçoit ses dépendances de l'extérieur plutôt que de les créer elle-même. NestJS possède un conteneur IoC (Inversion of Control) qui gère automatiquement la création et l'injection des instances.

🎯 Principe : "Ne cherchez pas vos dépendances, demandez-les !"

Contrôleur
→ demande → IoC Container
Service injecté

📦 @Injectable() : déclarer un provider

Le décorateur @Injectable() marque une classe comme pouvant être gérée par le conteneur IoC.

// users.service.ts
import { Injectable } from '@nestjs/common';

@Injectable()
export class UsersService {
  private users = [];

  findAll() {
    return this.users;
  }

  create(user) {
    this.users.push(user);
    return user;
  }
}
💡 Un service est un provider. Il contient la logique métier et peut être injecté partout.

💉 Injection dans le constructeur

Dans un contrôleur, on déclare la dépendance dans le constructeur. NestJS l'injecte automatiquement.

// users.controller.ts
import { Controller, Get, Post, Body } from '@nestjs/common';
import { UsersService } from './users.service';

@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

  @Post()
  create(@Body() user) {
    return this.usersService.create(user);
  }
}
✨ Le mot-clé private readonly permet de déclarer et d'initialiser la propriété en une ligne.

⚙️ Providers personnalisés

Dans un module, on peut définir des providers de différentes manières :

🔹 useValue 
providers: [{ provide: 'API_URL', useValue: 'https://api.example.com' }]

Injecter une valeur constante (chaîne, nombre, objet).

🔹 useClass 
providers: [{ provide: Logger, useClass: CustomLogger }]

Utiliser une classe différente de celle demandée (polymorphisme).

🔹 useFactory 
providers: [{
  provide: 'CONFIG',
  useFactory: (configService: ConfigService) => configService.get('app'),
  inject: [ConfigService]
}]

Création dynamique (asynchrone possible).

🏷️ Injection avec token personnalisé

Lorsqu'on utilise un provider défini avec une chaîne (ou un symbole), on doit utiliser @Inject().

// module
providers: [{ provide: 'API_URL', useValue: 'https://api.example.com' }]

// service
import { Inject, Injectable } from '@nestjs/common';

@Injectable()
export class AppService {
  constructor(@Inject('API_URL') private readonly apiUrl: string) {}

  getUrl() {
    return this.apiUrl;
  }
}
💡 Les tokens personnalisés sont utiles pour les librairies tierces ou les valeurs de configuration.

♻️ Portée des providers (Scope)

Par défaut, les providers en NestJS sont des singletons : une seule instance est partagée dans toute l'application.

DEFAULT
Singleton (une instance)
REQUEST
Une instance par requête HTTP
TRANSIENT
Une nouvelle instance à chaque injection
🧠 Changer la portée : @Injectable({ scope: Scope.REQUEST })
