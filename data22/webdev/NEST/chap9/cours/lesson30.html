📦 Installation

Commencez par installer le package nécessaire :

npm install --save @nestjs/schedule

Ensuite, importez ScheduleModule dans votre module principal :

import { Module } from '@nestjs/common';
import { ScheduleModule } from '@nestjs/schedule';

@Module({
  imports: [
    ScheduleModule.forRoot()
  ],
})
export class AppModule {}

✨ .forRoot() initialise le planificateur et exécute les tâches.

⏲️ Le décorateur @Cron()

Permet d'exécuter une méthode selon une expression cron.

import { Injectable } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';

@Injectable()
export class TasksService {
  @Cron('45 * * * * *')  // Toutes les minutes à 45 secondes
  handleCron() {
    console.log('Tâche exécutée chaque minute à 45 secondes');
  }

  @Cron(CronExpression.EVERY_10_SECONDS)
  handleEvery10s() {
    console.log('Toutes les 10 secondes');
  }
}

🔹 Syntaxe cron :

* * * * * *
│ │ │ │ │ │
│ │ │ │ │ └─── seconde (0-59)
│ │ │ │ └───── minute (0-59)
│ │ │ └─────── heure (0-23)
│ │ └───────── jour du mois (1-31)
│ └─────────── mois (1-12)
└───────────── jour de semaine (0-7, 0=dimanche)

🔸 Expressions prédéfinies :

CronExpression.EVERY_5_MINUTES
CronExpression.EVERY_HOUR
CronExpression.EVERY_DAY_AT_10AM
CronExpression.EVERY_WEEKEND

⏱️ @Interval() et @Timeout()

Des alternatives plus simples pour les répétitions et délais uniques.

@Injectable()
export class TasksService {
  @Interval(5000)  // toutes les 5 secondes
  handleInterval() {
    console.log('Répétition toutes les 5 secondes');
  }

  @Timeout(10000)  // une seule fois après 10 secondes
  handleTimeout() {
    console.log('Exécuté une fois 10s après démarrage');
  }
}

⚠️ Attention : @Timeout() s'exécute une fois, puis la tâche est supprimée.

⚙️ Jobs dynamiques

Vous pouvez créer, modifier ou supprimer des tâches à l'exécution via SchedulerRegistry.

import { Injectable } from '@nestjs/common';
import { SchedulerRegistry } from '@nestjs/schedule';
import { CronJob } from 'cron';

@Injectable()
export class DynamicService {
  constructor(private schedulerRegistry: SchedulerRegistry) {}

  addCronJob(name: string, seconds: string) {
    const job = new CronJob(`${seconds} * * * * *`, () => {
      console.log(`Job ${name} exécuté !`);
    });
    this.schedulerRegistry.addCronJob(name, job);
    job.start();
  }

  deleteCronJob(name: string) {
    this.schedulerRegistry.deleteCronJob(name);
    console.log(`Job ${name} supprimé`);
  }
}
💡 Utile pour des planifications paramétrables par l'utilisateur (ex: envoi de newsletters).

🌍 Exemple concret : Application camerounaise

Imaginons une application de e-learning qui doit envoyer un rappel de cours chaque lundi à 8h et nettoyer les logs toutes les nuits.

@Injectable()
export class NotificationService {
  @Cron('0 8 * * 1')  // chaque lundi à 8h
  sendWeeklyReminder() {
    console.log('📧 Rappel : vos cours de la semaine sont disponibles !');
  }

  @Cron('0 0 2 * * *')  // chaque nuit à 2h
  cleanLogs() {
    console.log('🧹 Nettoyage des anciens logs...');
  }

  @Interval(86400000)  // toutes les 24h (alternative)
  dailyBackup() {
    console.log('💾 Sauvegarde automatique des données');
  }
}

🎯 Résultat : votre application s'occupe toute seule des tâches répétitives !

✅ Bonnes pratiques

  • Utilisez des expressions cron lisibles ou les constantes CronExpression.
  • Évitez les tâches qui durent trop longtemps (préférez les déléguer à des workers).
  • Pour les environnements de production, assurez-vous que la planification est bien unique (ex: avec un verrou distribué).
  • Loggez les débuts/fins de tâches pour faciliter le débogage.
  • Testez vos tâches avec SchedulerRegistry en mockant le temps.

🚫 À éviter : planifier une tâche toutes les secondes qui fait des requêtes lourdes en base de données.
