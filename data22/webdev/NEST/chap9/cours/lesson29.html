Microservices avec NestJS

TCP, REDIS, RMQ : communication robuste entre services

🔌 @nestjs/microservices 📡 ClientProxy & MessagePattern 🔄 Transporters (TCP/Redis/RMQ)

🎯 Objectifs de la leçon

  • ✓ Comprendre l'architecture microservices dans NestJS et le rôle du transport.
  • ✓ Configurer un microservice avec les transporteurs TCP, Redis et RMQ (RabbitMQ).
  • ✓ Établir la communication client ↔ serveur via ClientProxy et @MessagePattern.

  • ✓ Appliquer les bonnes pratiques dans un système distribué.
💡 Pourquoi les microservices ? Découpage, scalabilité, résilience et indépendance technologique.
🔌 1. Module & transporteurs

Installation et création d'un microservice

Installer @nestjs/microservices. Ensuite on crée un microservice avec createMicroservice().

npm install @nestjs/microservices

// main.ts (microservice TCP)
import { NestFactory } from '@nestjs/core';
import { MicroserviceOptions, Transport } from '@nestjs/microservices';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.createMicroservice<MicroserviceOptions>(
    AppModule,
    {
      transport: Transport.TCP,
      options: { host: '0.0.0.0', port: 3001 },
    },
  );
  await app.listen();
}
bootstrap();
📌 Transporters disponibles : TCP, Redis, MQTT, NATS, RabbitMQ (RMQ), gRPC, Kafka.

📨 Gestion des messages : @MessagePattern

Dans le contrôleur ou le service, on utilise @MessagePattern('nom.pattern') pour écouter les requêtes.

// math.controller.ts
import { Controller } from '@nestjs/common';
import { MessagePattern, Payload } from '@nestjs/microservices';

@Controller()
export class MathController {
  @MessagePattern('sum')
  accumulate(@Payload() data: number[]): number {
    return (data || []).reduce((a, b) => a + b, 0);
  }
}

Le client envoie un message avec le pattern 'sum' et reçoit le résultat. On peut aussi utiliser @EventPattern pour les événements sans réponse.

✨ Astuce : @Payload() extrait la donnée utile. NestJS prend automatiquement en charge la sérialisation JSON.
📡 2. ClientProxy

Créer un client pour communiquer

Un microservice client envoie des requêtes via ClientProxyFactory ou @Client() (déprécié). On injecte ensuite le proxy.

// app.module.ts
import { ClientsModule, Transport } from '@nestjs/microservices';

@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'MATH_SERVICE',
        transport: Transport.TCP,
        options: { host: 'localhost', port: 3001 },
      },
    ]),
  ],
})
export class AppModule {}

// math-client.service.ts
@Injectable()
export class MathClientService {
  constructor(@Inject('MATH_SERVICE') private client: ClientProxy) {}

  async sum(numbers: number[]) {
    return this.client.send<number>('sum', numbers).toPromise();
    // ou .subscribe() pour flux asynchrone
  }
}

🔄 Transports : Redis et RabbitMQ (RMQ)

Modifiez simplement l'objet transport pour changer de protocole.

Redis 
transport: Transport.REDIS,
options: {
  host: 'localhost',
  port: 6379,
}
RabbitMQ (RMQ) 
transport: Transport.RMQ,
options: {
  urls: ['amqp://localhost:5672'],
  queue: 'tasks_queue',
  queueOptions: { durable: false },
}
// Enregistrement du client RMQ
ClientsModule.register([
  {
    name: 'TASK_SERVICE',
    transport: Transport.RMQ,
    options: {
      urls: ['amqp://user:pass@rabbitmq:5672'],
      queue: 'tasks_queue',
      noAck: false,
    },
  },
]);

✅ Redis : ultra rapide pour les communications légères. ✅ RMQ : files d'attente robustes, routage avancé.

📬 Patterns vs Events

@MessagePattern attend une réponse (requête-réponse). @EventPattern est utilisé pour les notifications sans réponse (fire-and-forget).

// notification microservice
@EventPattern('user_created')
async handleUserCreated(@Payload() data: any) {
  console.log(`Nouvel utilisateur: ${data.email}`);
  // envoi email de bienvenue...
}

// Client émet un événement
this.client.emit('user_created', { email: 'new@example.com' });
💡 Bonnes pratiques : utilisez des patterns versionnés (ex: "user.created.v1"). Gérez les erreurs avec des stratégies de repli.

🧠 Synthèse – Microservices NestJS

✅ Créer un microservice → NestFactory.createMicroservice()
✅ Transports supportés → TCP, Redis, RMQ, NATS, gRPC, Kafka
✅ @MessagePattern → Requête/Réponse
✅ @EventPattern → Événements unidirectionnels
✅ ClientProxy → communication via send() ou emit()
✅ Scalabilité horizontale avec plusieurs instances
