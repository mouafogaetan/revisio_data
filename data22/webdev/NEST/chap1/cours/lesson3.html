Objectifs de la leçon

  • Comprendre le rôle des modules pour organiser l'application.
  • Créer des contrôleurs pour définir les routes HTTP.

À la fin, vous saurez organiser une API REST modulaire et maintenable avec NestJS.

1. Modules (@Module)

Un module est un ensemble de fonctionnalités regroupées. Chaque application NestJS a au moins un module racine (AppModule).


import { Module } from '@nestjs/common';
import { UsersController } from './users.controller';
import { UsersService } from './users.service';

@Module({
  controllers: [UsersController],   // Les contrôleurs de ce module
  providers: [UsersService],        // Les services accessibles
  exports: [UsersService]           // Services à partager avec d'autres modules
})
export class UsersModule {}
Bonnes pratiques : Un module par fonctionnalité (ex: UsersModule, ProductsModule, AuthModule).

2. Contrôleurs (@Controller)

Les contrôleurs gèrent les requêtes HTTP et retournent les réponses. Ils sont décorés avec @Controller().


import { Controller, Get, Post, Body, Param } from '@nestjs/common';
import { UsersService } from './users.service';

@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

  @Get(':id')
  findOne(@Param('id') id: string) {
    return this.usersService.findOne(+id);
  }

  @Post()
  create(@Body() createUserDto: any) {
    return this.usersService.create(createUserDto);
  }
}
Le décorateur @Controller('users') définit le préfixe de route /users.

3. Services (@Injectable)

Les services contiennent la logique métier (ex: accès à la base de données, calculs, appels API). Ils sont décorés avec @Injectable().


import { Injectable } from '@nestjs/common';

export interface User {
  id: number;
  name: string;
  email: string;
}

@Injectable()
export class UsersService {
  private users: User[] = [
    { id: 1, name: 'Alice', email: 'alice@example.com' },
    { id: 2, name: 'Bob', email: 'bob@example.com' },
  ];

  findAll(): User[] {
    return this.users;
  }

  findOne(id: number): User | undefined {
    return this.users.find(user => user.id === id);
  }

  create(user: Omit): User {
    const newUser = { id: this.users.length + 1, ...user };
    this.users.push(newUser);
    return newUser;
  }
}

⚠️ Le service est injecté dans le contrôleur via le constructeur (injection de dépendances).

4. Injection de dépendances

NestJS utilise un conteneur IoC (Inversion of Control) pour gérer les dépendances entre les composants.

🔹 Principe :

Au lieu de créer manuellement une instance de UsersService dans le contrôleur, on la déclare dans le constructeur. NestJS injecte automatiquement l'instance.


// Dans UsersController
constructor(private readonly usersService: UsersService) {}
Avantages : testabilité, découplage, code plus propre.

5. Structure d'un projet NestJS


src/
├── main.ts                 # Point d'entrée
├── app.module.ts           # Module racine
├── users/
│   ├── users.module.ts     # Module Users
│   ├── users.controller.ts # Contrôleur Users
│   ├── users.service.ts    # Service Users
│   └── dto/
│       └── create-user.dto.ts
├── products/
│   ├── products.module.ts
│   ├── products.controller.ts
│   └── products.service.ts
└── auth/
    ├── auth.module.ts
    ├── auth.controller.ts
    └── auth.service.ts

Cette organisation par fonctionnalité (feature-based) est recommandée par l'équipe NestJS.

Consigne :

Créez un module Products avec :

  1. Un contrôleur ProductsController avec les routes : GET /products, GET /products/:id, POST /products
  2. Un service ProductsService qui gère une liste de produits en mémoire (id, nom, prix).
  3. Importer le module dans AppModule.
  4. Tester avec Postman ou curl.
Utilisez la commande nest g module products, nest g controller products, nest g service products pour générer automatiquement les fichiers.
