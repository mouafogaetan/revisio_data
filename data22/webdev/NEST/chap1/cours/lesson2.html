💡 Prérequis : Node.js (version 16 ou supérieure) et npm/yarn installés sur votre machine.

🖥️ Installation du NestJS CLI

Le NestJS CLI (Command Line Interface) est un outil indispensable pour générer des fichiers, créer des modules et gérer votre projet.

$ npm install -g @nestjs/cli

Après installation, vérifiez que tout fonctionne :

$ nest --version
⚠️ Note : Sur Linux/Mac, vous pourriez avoir besoin de sudo pour l'installation globale. Sur Windows, utilisez PowerShell en mode administrateur.

🆕 Création d'un nouveau projet

La commande nest new crée un projet NestJS complet avec toute la configuration nécessaire.

$ nest new mon-premier-projet

Le CLI vous demandera de choisir un gestionnaire de paquets :

  • npm (recommandé pour la simplicité)
  • yarn (plus rapide pour les gros projets)
  • pnpm (économise de l'espace disque)
📁 Arborescence générée :
mon-premier-projet/
├── src/
│ ├── app.controller.ts
│ ├── app.service.ts
│ ├── app.module.ts
│ └── main.ts
├── test/
├── package.json
└── nest-cli.json

📁 Exploration de la structure générée

Les fichiers principaux à connaître :

📄 main.ts - Point d'entrée

Crée l'application NestJS et démarre le serveur HTTP sur le port 3000.

import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();

📄 app.module.ts - Module racine

Définit le module principal qui regroupe contrôleurs et services.

📄 app.controller.ts - Contrôleur

Gère les routes HTTP et les requêtes entrantes.

📄 app.service.ts - Service

Contient la logique métier (récupération des données, calculs, etc.).

🚀 Démarrer le serveur de développement

Après la création du projet, entrez dans le dossier et lancez l'application :

$ cd mon-premier-projet
$ npm run start:dev

Le mode start:dev active le rechargement à chaud (hot reload) : à chaque modification d'un fichier, le serveur redémarre automatiquement.
