📚 Qu'est-ce que Passport ?

Passport est le middleware d'authentification le plus populaire pour Node.js. NestJS l'intègre nativement via @nestjs/passport.

✅ Avantages

  • Plus de 500 stratégies (Google, Facebook, JWT, Local...)
  • Modulaire et extensible
  • Intégration transparente avec NestJS

🔧 Stratégies courantes

  • passport-local : email/mot de passe
  • passport-jwt : token JWT
  • passport-google-oauth20
$ npm install @nestjs/passport passport passport-local passport-jwt @nestjs/jwt

🔑 LocalStrategy - Authentification locale

La LocalStrategy permet de valider un utilisateur à partir d'un email/username et d'un mot de passe.

// local.strategy.ts
import { Strategy } from 'passport-local';
import { PassportStrategy } from '@nestjs/passport';
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { AuthService } from './auth.service';

@Injectable()
export class LocalStrategy extends PassportStrategy(Strategy) {
  constructor(private authService: AuthService) {
    super({ usernameField: 'email' }); // par défaut 'username', on change pour 'email'
  }

  async validate(email: string, password: string): Promise {
    const user = await this.authService.validateUser(email, password);
    if (!user) {
      throw new UnauthorizedException('Identifiants invalides');
    }
    return user;
  }
}
📌 Note : La méthode validate() est appelée automatiquement par Passport. Elle doit retourner l'utilisateur ou lever une exception.

🔐 JwtStrategy - Validation des tokens

La JwtStrategy extrait et valide le token JWT présent dans l'en-tête Authorization: Bearer <token>.

// jwt.strategy.ts
import { ExtractJwt, Strategy } from 'passport-jwt';
import { PassportStrategy } from '@nestjs/passport';
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor(private configService: ConfigService) {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      ignoreExpiration: false,
      secretOrKey: configService.get('JWT_SECRET'),
    });
  }

  async validate(payload: any) {
    // payload contient les données du token (sub, email, roles...)
    return { userId: payload.sub, email: payload.email, roles: payload.roles };
  }
}
⚠️ Important : La méthode validate() ne doit pas faire de vérification supplémentaire du token (déjà validé). Elle sert à ajouter l'utilisateur à la requête (req.user).

⚙️ AuthService : validation et génération JWT

Le service d'authentification contient la logique de validation des utilisateurs et la génération des tokens.

// auth.service.ts
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';
import { UsersService } from '../users/users.service';
import * as bcrypt from 'bcrypt';

@Injectable()
export class AuthService {
  constructor(
    private usersService: UsersService,
    private jwtService: JwtService,
  ) {}

  async validateUser(email: string, password: string): Promise {
    const user = await this.usersService.findByEmail(email);
    if (user && await bcrypt.compare(password, user.password)) {
      const { password, ...result } = user;
      return result;
    }
    return null;
  }

  async login(user: any) {
    const payload = { sub: user.id, email: user.email, roles: user.roles };
    return {
      access_token: this.jwtService.sign(payload),
    };
  }
}

🛡️ Protection des routes avec Guards

Utilisez @UseGuards(AuthGuard('jwt')) pour protéger vos routes. Seules les requêtes avec un token valide y auront accès.

// products.controller.ts
import { Controller, Get, UseGuards } from '@nestjs/common';
import { AuthGuard } from '@nestjs/passport';
import { ProductsService } from './products.service';

@Controller('products')
export class ProductsController {
  constructor(private productsService: ProductsService) {}

  // Route publique
  @Get()
  findAll() {
    return this.productsService.findAll();
  }

  // Route protégée (nécessite authentification JWT)
  @Get('admin')
  @UseGuards(AuthGuard('jwt'))
  adminOnly() {
    return { message: 'Accès réservé aux administrateurs' };
  }
}
✅ Récupérer l'utilisateur : Utilisez @Req() request et request.user pour accéder aux données du payload.

LocalStrategy
