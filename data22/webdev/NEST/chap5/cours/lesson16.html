⚠️ Pourquoi ne jamais stocker en clair ?

Stocker un mot de passe en clair dans une base de données est extrêmement dangereux. En cas de fuite, l'attaquant peut utiliser ces identifiants sur d'autres services.

❌ Mauvais exemple

password: "monSuperMotDePasse123"

✅ Bonne pratique

password: "$2b$10$91mX4fKjYg5GZj... (hash bcrypt + salage)

Le hachage est irréversible : même l'administrateur ne peut pas connaître le mot de passe original.

🧠 bcrypt ajoute automatiquement un salt (valeur aléatoire) pour rendre les attaques par rainbow tables inefficaces.

📦 Installation de bcrypt

npm install bcrypt
npm install -D @types/bcrypt

Exemple d'utilisation basique :

import * as bcrypt from 'bcrypt'; const saltRounds = 10; const plainPassword = "secret123"; // Hachage const hash = await bcrypt.hash(plainPassword, saltRounds); console.log(hash); // $2b$10$91mX4fKjYg5GZj... // Comparaison const isMatch = await bcrypt.compare(plainPassword, hash); console.log(isMatch); // true
💡 saltRounds = 10 correspond à environ 2^10 itérations – bon compromis sécurité/performance.

🏭 Création d'un service Hashing (BCryptService)

// src/auth/bcrypt.service.ts import { Injectable } from '@nestjs/common'; import * as bcrypt from 'bcrypt'; @Injectable() export class BcryptService { private readonly saltRounds = 10; async hash(password: string): Promise<string> { return bcrypt.hash(password, this.saltRounds); } async compare(password: string, hash: string): Promise<boolean> { return bcrypt.compare(password, hash); } }

On injecte ensuite ce service dans AuthService pour hâcher les mots de passe avant enregistrement.

✅ Avantage : centralise la logique de hachage, facile à mocker dans les tests.

🔁 Intégration dans AuthService

Lors de la création d'un compte, on hache le mot de passe avant de le sauvegarder en base.

// auth.service.ts @Injectable() export class AuthService { constructor( private bcryptService: BcryptService, @InjectRepository(User) private userRepo: Repository<User>, ) {} async register(createUserDto: CreateUserDto) { const hashedPassword = await this.bcryptService.hash(createUserDto.password); const newUser = this.userRepo.create({ ...createUserDto, password: hashedPassword, }); return this.userRepo.save(newUser); } async validateUser(email: string, pass: string) { const user = await this.userRepo.findOneBy({ email }); if (!user) return null; const isMatch = await this.bcryptService.compare(pass, user.password); return isMatch ? user : null; } }
⚠️ Ne jamais logguer le mot de passe ou le hash, même en développement.

⚙️ Bonnes pratiques & paramètres

  • Salt rounds : utiliser entre 10 et 12 (2^10 = 1024 itérations). Augmenter progressivement avec la puissance matérielle.
  • Bcrypt vs scrypt/argon2 : bcrypt est mature, simple et largement supporté. Pour très haute sécurité, préférer argon2 (mais plus complexe).
  • Longueur du mot de passe : bcrypt limite à 72 bytes. Penser à tronquer ou valider avant hachage.
  • Ne pas désactiver le salage : bcrypt le gère automatiquement.
🔐 Exemple de hash complet : $2b$10$9Rr5xQbPwA1/3Zq8YcLQOuvJb9kL8nMpCxJfX3YlZ5wVq2RfUaW

🌐 Exemple : Contrôleur Auth complet

// auth.controller.ts @Post('register') async register(@Body() dto: CreateUserDto) { return this.authService.register(dto); } @Post('login') async login(@Body() loginDto: LoginDto) { const user = await this.authService.validateUser(loginDto.email, loginDto.password); if (!user) throw new UnauthorizedException(); const token = this.jwtService.sign({ sub: user.id, email: user.email }); return { access_token: token }; }

Le mot de passe est haché avant stockage, et jamais renvoyé dans la réponse.

📘 Résumé du flux : Inscription → hash password → enregistrement → Connexion → comparaison bcrypt → JWT.

✏️ Exercice pratique

1. Créez un projet NestJS et installez bcrypt.

2. Implémentez un BcryptService avec les méthodes hash et compare.

3. Dans un UsersService, hachez le mot de passe avant de le stocker (simulez une base avec un tableau).

4. Écrivez une méthode validateUser qui compare un mot de passe saisi avec le hash stocké.

5. Testez avec un petit script ou un contrôleur simple.

💡 Bonus : mesurez le temps d'exécution de bcrypt.hash avec saltRounds = 10, 12, 14. Que constatez-vous ?
