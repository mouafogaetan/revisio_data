🔍 Qu'est-ce qu'un Interceptor ?

Un Interceptor dans NestJS est une classe décorée avec @Injectable() qui implémente l'interface NestInterceptor. Il permet d'exécuter du code avant et après l'exécution d'une méthode de contrôleur.

📌 Analogie simple :

Imaginez un videur (interceptor) à l'entrée d'une boîte de nuit : il vérifie les invités (requête) avant qu'ils entrent, et après leur sortie, il peut faire un rapport (log).

🔧 Avant l'exécution

  • Logguer la requête entrante
  • Modifier les données du flux
  • Vérifier certaines conditions

✨ Après l'exécution

  • Transformer la réponse
  • Ajouter des métadonnées
  • Mesurer le temps de réponse

📐 Structure & création d’un interceptor

Pour créer un interceptor, on utilise la CLI ou manuellement :

import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable } from 'rxjs';
import { map } from 'rxjs/operators';

@Injectable()
export class TransformInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    // Avant l'exécution de la route
    console.log('🟢 Avant la méthode du contrôleur');
    
    return next.handle().pipe(
      map(data => {
        // Après l'exécution : transformation
        console.log('🔵 Après la méthode');
        return { status: 'success', data };
      })
    );
  }
}
ExecutionContext : donne accès à la requête, la réponse, etc.
CallHandler : permet d’appeler la méthode du contrôleur.
pipe(map()) : opérateur RxJS pour modifier la réponse.

🔄 Interceptor de transformation des réponses

Idéal pour uniformiser le format de toutes les réponses JSON de l’API. Exemple : envelopper le résultat avec { success: true, result: ... }.

// response.interceptor.ts
import { map } from 'rxjs/operators';

export class ResponseInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler) {
    return next.handle().pipe(
      map((originalResponse) => ({
        timestamp: new Date().toISOString(),
        path: context.switchToHttp().getRequest().url,
        data: originalResponse,
        statusCode: context.switchToHttp().getResponse().statusCode
      }))
    );
  }
}

✅ Résultat avant → "Bonjour le monde" devient :

{ "timestamp": "2025-...", "path": "/hello", "data": "Bonjour le monde", "statusCode": 200 }

Pour utiliser globalement : app.useGlobalInterceptors(new ResponseInterceptor())

⏱️ Mesurer le temps d’exécution (Performance Logger)

Un interceptor très utile pour analyser les performances de chaque endpoint :

import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const req = context.switchToHttp().getRequest();
    const method = req.method;
    const url = req.url;
    const start = Date.now();
    
    console.log(`📥 Requête entrante : ${method} ${url}`);
    
    return next.handle().pipe(
      tap(() => {
        const duration = Date.now() - start;
        console.log(`📤 Réponse envoyée pour ${method} ${url} - ⏱️ ${duration}ms`);
      })
    );
  }
}

💡 Explication : tap() permet d’exécuter une fonction sans modifier le flux. Idéal pour le logging ou le profiling.

🔗 Utiliser les intercepteurs (liens & portée)

1. Intercepteur lié à un contrôleur

@UseInterceptors(LoggingInterceptor)
@Controller('users')
export class UsersController {}

2. Intercepteur global (dans main.ts)

app.useGlobalInterceptors(new TransformInterceptor());

⭐ Convient pour une transformation standard sur toutes les routes.

3. Intercepteur spécifique à une méthode

@Get()
@UseInterceptors(MyInterceptor)
findAll() { ... }

⚠️ Les intercepteurs s'exécutent dans l'ordre : les intercepteurs globaux d'abord, puis niveau contrôleur, puis niveau méthode.

⚡ Cas pratique : Intercepteur de cache simple

Un interceptor peut également mettre en cache les réponses pour éviter les calculs redondants. Idée illustrée :

const cache = new Map();

@Injectable()
export class CacheInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();
    const key = request.url;
    if (cache.has(key)) {
      return of(cache.get(key)); // retourne la valeur en cache
    }
    return next.handle().pipe(
      tap(response => cache.set(key, response))
    );
  }
}

🎓 On utilise of() de RxJS pour retourner une réponse immédiate. Cela booste les temps de réponse pour les routes statiques !

📦 Client ➡️ Interceptor 🔍 cache ? OUI → réponse cache NON → Contrôleur

📌 Récapitulatif & Bonnes pratiques

  • Les intercepteurs permettent d'ajouter des pré-traitements et post-traitements sans polluer la logique métier.
  • Idéal pour : logging, transformation, monitoring, caching, gestion des timeouts.
  • Utilisez les opérateurs RxJS (map, tap, catchError) pour manipuler le flux.
  • Ne pas oublier d’injecter dans un module ou de l’utiliser avec @UseInterceptors().
  • Pour un interceptor global préférer app.useGlobalInterceptors() ou injection via APP_INTERCEPTOR.
