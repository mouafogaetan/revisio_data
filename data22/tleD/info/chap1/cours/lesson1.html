Définition et structure d'un algorithme

Informatique · Terminale C & D (Scientifiques) · Bac 2026

📌 Éléments clés à retenir

🔍 Qu'est-ce qu'un algorithme ?

Un algorithme est une suite finie et ordonnée d'instructions précises et non ambiguës qui, exécutées dans un ordre déterminé, permet de résoudre un problème donné ou d'effectuer un calcul.

  • Caractéristiques : finitude (nombre d'étapes limité), déterminisme (chaque étape est claire), entrées/sorties définies.
  • Un algorithme s'exprime en pseudocode (langage proche du français) avant d'être traduit dans un langage de programmation (C, Python...).
  • Exemple simple : algorithme de calcul de la moyenne de deux notes.

🏗️ Structure générale d'un algorithme

Tout algorithme comporte trois grandes parties :

  • 1. Partie déclarative : Déclaration des constantes et des variables utilisées.
  • 2. Partie instructions (corps) : Suite d'instructions (saisies, calculs, affectations, sorties, structures conditionnelles ou itératives).
  • 3. Partie retour (optionnelle) : Résultat ou valeur renvoyée.
💡 En pseudocode, on utilise souvent les mots-clés : Début, Fin, Variables, Constantes.

📝 Instructions fondamentales

Trois types d'instructions sont à maîtriser absolument :

  • Instruction de lecture (saisie) : Lire(nom_variable) ou Input nom_variable → permet d'entrer une donnée depuis l'utilisateur.
  • Instruction d'affectation : variable ← expression ou variable = expression → stocke une valeur dans une variable.
  • Instruction d'écriture (sortie) : Ecrire(expression) ou Print(expression) → affiche un résultat.

📦 Variables et constantes

Une variable est un espace mémoire nommé qui peut contenir une valeur modifiable au cours de l'exécution. Une constante est une valeur fixe qui ne change pas.

  • Nommage : commencer par une lettre, sans espaces, sans caractères spéciaux (ex: note1, moyenne).
  • Types de base : numérique (entier, réel), chaîne de caractères, booléen.
  • Exemple de déclaration : Variables : a, b, somme : entiers.

💡 Exemples de compréhension

Exemple 1 : Algorithme de calcul de la somme de deux nombres

Écrire un algorithme qui lit deux nombres entiers, calcule leur somme et affiche le résultat.

Pseudocode :
Début
    Variables : a, b, somme : entiers
    Ecrire("Entrez le premier nombre : ")
    Lire(a)
    Ecrire("Entrez le second nombre : ")
    Lire(b)
    somme ← a + b
    Ecrire("La somme est : ", somme)
Fin
💬 Astuce : L'affectation somme ← a + b signifie "on calcule a+b et on met le résultat dans la variable somme".

Exemple 2 : Algorithme avec constante et périmètre d'un cercle

Calculer le périmètre d'un cercle de rayon donné par l'utilisateur en utilisant la constante π = 3.14159.

Pseudocode :
Début
    Constante : PI ← 3.14159
    Variables : rayon, perimetre : réels
    Ecrire("Rayon du cercle : ")
    Lire(rayon)
    perimetre ← 2 * PI * rayon
    Ecrire("Périmètre = ", perimetre)
Fin
💡 La constante évite d'écrire plusieurs fois la même valeur et rend le code plus clair.

Exemple 3 : Échange de deux variables (avec variable temporaire)

Algorithme qui lit deux valeurs A et B, puis les échange (A prend la valeur de B, B prend la valeur de A).

Pseudocode :
Début
    Variables : A, B, temp : entiers
    Lire(A)
    Lire(B)
    temp ← A
    A ← B
    B ← temp
    Ecrire("Après échange : A = ", A, " B = ", B)
Fin
💬 Astuce : Sans variable temporaire, la valeur initiale de A serait perdue. C'est une manipulation classique à bien maîtriser.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Confondre l'affectation (←) avec l'égalité mathématique (=). En algorithmique, x ← x + 1 est valide (incrémentation), mais mathématiquement c'est faux.
  • Attention : Oublier de déclarer les variables avant de les utiliser dans la partie instructions → provoque une erreur logique.
  • Piège classique : Ne pas initialiser une variable (ex: pour une somme, partir de 0) avant de l'utiliser dans un calcul.
  • À ne pas confondre : Lire(variable) pour entrée clavier et Ecrire("texte") pour afficher. Inverser les deux peut rendre l'algorithme incompréhensible.

🎯 Lien avec l'épreuve du Bac (Session 2025)

D'après le programme officiel (Session 2025, Partie III, questions 1, 2 et 3), cette leçon est évaluée à travers :

  • La définition d'un algorithme et l'identification de ses différentes parties (déclarations, corps).
  • L'écriture d'instructions simples : lecture, affectation, écriture.
  • La traduction d'un algorithme simple en langage de programmation (souvent en C).
  • La capacité à analyser un pseudocode et à en donner le résultat (tracer l'exécution).
🔔 Conseil de révision : Entraînez-vous à écrire des algorithmes en pseudocode pour des problèmes simples (somme, produit, échange, calcul de moyenne). Cela vous aidera pour la partie III de l'épreuve.
