📡 Adressage IP et sous-réseaux

Informatique · Terminale C & D (Scientifiques) · Réseaux informatiques

📌 Éléments clés à retenir

🔢 1. Adresse IP : identifiant unique sur le réseau

Une adresse IP (Internet Protocol) est un numéro unique attribué à chaque périphérique connecté à un réseau informatique (ordinateur, imprimante, smartphone, routeur…). Elle permet d'identifier et de localiser les machines pour qu'elles puissent communiquer entre elles.

  • IPv4 : 32 bits → 4 nombres décimaux séparés par des points, chacun entre 0 et 255. Exemple : 192.168.1.10
  • IPv6 : 128 bits (format hexadécimal) – plus moderne, mais au bac on travaille surtout avec IPv4.
  • Une adresse IP possède deux parties : NetID (identifie le réseau) + HostID (identifie l’hôte/machine dans ce réseau).

🏷️ 2. Classes d'adresses IPv4 (historique)

Les classes permettent de déterminer la partie réseau et la partie hôte selon les premiers bits. À connaître pour l’examen (sessions 2023, Partie I).

📊 Tableau mémo :
• Classe A : 1.0.0.0 à 126.255.255.255 → Masque par défaut 255.0.0.0 (/8)
• Classe B : 128.0.0.0 à 191.255.255.255 → Masque 255.255.0.0 (/16)
• Classe C : 192.0.0.0 à 223.255.255.255 → Masque 255.255.255.0 (/24)
• Adresses spéciales : 127.0.0.1 (loopback), 0.0.0.0, 255.255.255.255 (broadcast).
  • Classe A : grands réseaux (peu de réseaux, beaucoup d’hôtes). Premier octet entre 1 et 126.
  • Classe B : réseaux de taille moyenne. Premier octet entre 128 et 191.
  • Classe C : petits réseaux (beaucoup de réseaux, peu d’hôtes). Premier octet entre 192 et 223.
  • Astuce bac : Le premier octet détermine la classe. Ex: 172.16.5.1 → 172 est entre 128 et 191 → classe B.

🎭 3. Masque de sous-réseau

Le masque de sous-réseau est un nombre de 32 bits qui permet de distinguer la partie réseau (NetID) de la partie hôte (HostID) dans une adresse IP. Il se note aussi en notation CIDR : /n (ex: /24 correspond à 255.255.255.0).

  • Les bits à 1 dans le masque indiquent la partie réseau.
  • Les bits à 0 indiquent la partie hôte.
  • Pour calculer l’adresse réseau : on applique un ET logique bit à bit entre l’adresse IP et le masque.
  • Exemple : IP 192.168.1.45, masque 255.255.255.0 → adresse réseau = 192.168.1.0

⚙️ 4. Modes d'attribution : statique vs dynamique

Deux méthodes pour attribuer une adresse IP à une machine sur un réseau.

  • Statique : l'administrateur configure manuellement l'adresse IP, le masque, la passerelle. Idéal pour les serveurs ou imprimantes réseau.
  • Dynamique (DHCP) : un serveur DHCP attribue automatiquement une adresse IP disponible à chaque périphérique qui se connecte au réseau. Plus simple et flexible.
  • À retenir : DHCP = Dynamic Host Configuration Protocol. Le client envoie une requête, le serveur répond avec une IP valide.

💡 Exemples de compréhension

📝 Examen type : Identifier la classe et l’adresse réseau

On donne l’adresse IP suivante : 172.25.120.8 avec le masque par défaut de sa classe. Déterminez la classe, le masque et l’adresse réseau.

✅ Résolution :
1️⃣ Premier octet = 172 → compris entre 128 et 191 → Classe B.
2️⃣ Masque par défaut d’une classe B : 255.255.0.0 (soit /16).
3️⃣ Adresse réseau : on conserve les deux premiers octets (partie réseau) et on met les bits hôte à 0 → 172.25.0.0.
💡 Les machines de ce réseau auront des IP allant de 172.25.0.1 à 172.25.255.254.
💬 Astuce bac : Pour une IP de classe C (ex: 192.168.1.45), l’adresse réseau est simplement les 3 premiers octets suivis de .0 (192.168.1.0).

🧩 Calcul d’adresse réseau avec masque non standard

Soit l’IP 192.168.10.132 et un masque de sous-réseau 255.255.255.224 (soit /27). Calculez l’adresse du réseau et l’adresse de broadcast.

🔎 Méthode : Appliquer l’opération ET entre IP et masque.
• 192.168.10.132 → binaire : 11000000.10101000.00001010.10000100
• Masque 255.255.255.224 → binaire : 11111111.11111111.11111111.11100000
• ET logique : partie hôte devient 10000000 (soit 128).
→ Adresse réseau : 192.168.10.128.
• Adresse broadcast : tous les bits hôte à 1 → dernier octet : 128 + (32-1) = 159 → 192.168.10.159.
Les hôtes valides : de 192.168.10.129 à 192.168.10.158.
🧠 Pour un masque /27, il y a 32 adresses par sous-réseau (dont réseau + broadcast). On peut avancer par paliers de 32.

🌐 Adressage dynamique (DHCP) vs Statique

Dans une salle informatique de lycée, on souhaite que les élèves obtiennent automatiquement une IP lorsqu’ils se connectent au réseau local. En revanche, le serveur de fichiers doit toujours avoir la même IP (192.168.1.10). Expliquez.

Solution :
• Pour les postes élèves → configurer le protocole DHCP : ils reçoivent une IP dynamique à chaque connexion. Cela simplifie l’administration.
• Pour le serveur → utiliser une adresse IP statique pour garantir une adresse fixe et prévisible, nécessaire aux services partagés.
À retenir : DHCP évite les conflits d’adresses IP et facilite la mobilité.
📌 Question typique du bac : “Citez deux avantages du DHCP” → automatisation, réduction des erreurs de saisie, utilisation optimale des adresses.

⚠️ Pièges courants & erreurs fréquentes

  • Confondre adresse IP publique et privée : Les adresses privées (ex: 192.168.x.x, 10.x.x.x, 172.16.x.x à 172.31.x.x) ne sont pas routables sur Internet. Utilisées en LAN.
  • Oublier que l’adresse de réseau et l’adresse de broadcast ne sont pas attribuables à un hôte. Ne jamais utiliser la première ni la dernière IP d’un sous-réseau pour une machine.
  • Erreur sur la classe d’adresse : 127.x.x.x est réservé (loopback), 0.0.0.0 est par défaut. 224-239 = multicast (non utilisée pour les hôtes classiques).
  • Ne pas savoir déterminer le masque par défaut → À mémoriser : Classe A = /8, B = /16, C = /24.
  • Confondre adresse IPv4 et IPv6 – Pour le baccalauréat au Cameroun, les exercices portent essentiellement sur IPv4.

🔎 Commande réseau associée

Sous Windows : ipconfig affiche la configuration IP (adresse, masque, passerelle). Sous Linux/Mac : ifconfig ou ip a. La commande ping permet de tester la connectivité (ex: ping 8.8.8.8). Ces commandes sont souvent évaluées (session 2023, partie I).
