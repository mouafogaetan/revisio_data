Leçon 18 : Insertion et modification des données (DML)

Informatique · Terminale C & D · Unité 4 – Implémentation des bases de données (SQL)

📌 Éléments clés à retenir

1. Langage DML (Data Manipulation Language)

Le DML est la partie du SQL qui permet de manipuler les données déjà présentes dans les tables : ajout, modification, suppression. Les trois commandes fondamentales sont :

  • INSERT INTO : Ajouter de nouvelles lignes (enregistrements) dans une table.
  • UPDATE : Modifier les valeurs d'une ou plusieurs lignes existantes.
  • DELETE : Supprimer des lignes (vue en complément, mais essentielle).
💡 À l'examen (session 2025, 2024), les requêtes INSERT INTO et UPDATE ... SET ... WHERE sont fréquentes.

2. INSERT INTO : syntaxe complète

Deux formes principales pour insérer des données :

  • Insérer toutes les colonnes (dans l'ordre de la table) :
    INSERT INTO table VALUES (valeur1, valeur2, ...);
  • Insérer seulement certaines colonnes :
    INSERT INTO table (colonneA, colonneC) VALUES (valA, valC); (les colonnes omises reçoivent NULL ou leur valeur par défaut)
  • Insertion multiple : depuis SQLite/MySQL récent, on peut chaîner : INSERT INTO table VALUES (…), (…), (…);

Contraintes : Respecter les types de données (INT, VARCHAR, DATE) et les clés primaires (unicité).

3. UPDATE : modifier des enregistrements

La commande UPDATE modifie des lignes existantes. La clause WHERE est cruciale pour cibler les bonnes lignes.

  • Syntaxe générale :
    UPDATE nom_table SET colonne1 = nouvelle_valeur, colonne2 = ... WHERE condition ;
  • Sans WHERE : toutes les lignes de la table sont modifiées → danger !
  • Exemple : UPDATE Etudiants SET moyenne = 15.5 WHERE id = 102;
⚠️ Toujours tester la condition avec un SELECT avant d'exécuter un UPDATE critique.

4. DELETE (rappel essentiel)

Bien que la leçon soit centrée sur INSERT/UPDATE, la manipulation inclut la suppression. DELETE FROM table WHERE condition ; supprime des lignes.

  • Sans WHERE : vide complètement la table (mais garde la structure).
  • La commande TRUNCATE est une variante plus rapide mais pas toujours au programme.

💡 Exemples de compréhension

Exemple 1 : Insertion dans une table "Élèves"

Soit la table Élèves définie par :
CREATE TABLE Eleves (id INT PRIMARY KEY, nom VARCHAR(50), prenom VARCHAR(50), age INT, classe CHAR(2));

Insertion classique (toutes colonnes) :

INSERT INTO Eleves VALUES (101, 'Djibril', 'Fatima', 18, 'C');

Insertion avec colonnes spécifiques (on omet l'âge) :

INSERT INTO Eleves (id, nom, prenom, classe)
VALUES (102, 'Tchouang', 'Kevin', 'D'); -- l'âge sera NULL (aucune valeur)
💬 Astuce : Pour insérer plusieurs élèves d’un coup : INSERT INTO Eleves VALUES (103,'Nana','Aicha',17,'D'), (104,'Mbarga','Paul',18,'C');

Exemple 2 : Mise à jour (UPDATE) avec condition

Dans la table Employes (id, nom, salaire, service). On souhaite augmenter le salaire de 10% pour tous les employés du service 'Informatique'.

UPDATE Employes
SET salaire = salaire * 1.10
WHERE service = 'Informatique';

Explication : La clause WHERE filtre les lignes concernées. Seuls les employés du service 'Informatique' verront leur salaire modifié.

💡 Pensez à vérifier avec un SELECT * FROM Employes WHERE service = 'Informatique'; avant de lancer l'UPDATE.

Exemple 3 : Mise à jour avec plusieurs colonnes

Table Produits (ref, nom, prix, stock). On veut modifier le prix et le stock du produit dont la référence est 'P101'.

UPDATE Produits
SET prix = 2500, stock = stock + 50
WHERE ref = 'P101';

On peut lister plusieurs colonnes séparées par des virgules dans la clause SET. Ici, le stock est incrémenté de 50 unités.

Examen type : Session 2025 (sujet adapté)

On vous donne la table Livres (code INT PK, titre VARCHAR, auteur VARCHAR, annee INT).
Écrire la requête SQL qui insère un nouveau livre : code=210, titre="Algorithmique avancée", auteur="Dupont", année=2024.

INSERT INTO Livres (code, titre, auteur, annee)
VALUES (210, 'Algorithmique avancée', 'Dupont', 2024);

Puis modifier l'année de ce livre pour la passer à 2025 :

UPDATE Livres SET annee = 2025 WHERE code = 210;
🎯 À l'examen, on attend une syntaxe précise : majuscules pour les mots-clés SQL, point-virgule final, guillemets simples pour les chaînes.

📊 Récapitulatif des commandes DML

CommandeRôleExemple minimal
INSERT INTOAjouter une ligneINSERT INTO Clients (nom, ville) VALUES ('Mbarga','Douala');
UPDATEModifier des lignesUPDATE Clients SET ville='Yaoundé' WHERE nom='Mbarga';
DELETESupprimer des lignesDELETE FROM Clients WHERE nom='Mbarga';

🔑 Règle d'or : toujours spécifier une condition WHERE pour UPDATE et DELETE sauf si on veut agir sur toutes les lignes.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier les guillemets simples pour les chaînes de caractères ou les dates. Ex : '2025-04-07' au lieu de 2025-04-07 (invalide).
  • Attention : Utiliser INSERT INTO table VALUES (…) sans préciser les colonnes → il faut fournir une valeur pour toutes les colonnes dans l'ordre défini.
  • Piège UPDATE : Ne pas mettre de WHERE → modification de toute la table ! Exemple catastrophique : UPDATE Eleves SET age = 20; (tous les élèves auront 20 ans).
  • Violation de clé primaire : Insérer une ligne avec un identifiant déjà existant → rejet par le SGBD (erreur). Toujours vérifier l'unicité.
  • Confusion DDL / DML : CREATE, ALTER, DROP sont DDL (structure). INSERT, UPDATE, DELETE sont DML (données).

✏️ Mini-exercice (auto-évaluation)

À vous de jouer !

Soit la table Stagiaires (matricule INT PRIMARY KEY, nom VARCHAR(30), formation VARCHAR(50), note_finale INT).
1. Insérez un stagiaire avec matricule 405, nom "Etonde", formation "Réseaux", note_finale 15.
2. Augmentez la note_finale de 2 points pour tous les stagiaires de la formation "Réseaux".
3. Modifiez la formation du stagiaire matricule 405 pour qu'elle devienne "Cybersécurité".

🔍 Voir les solutions
-- 1. Insertion
INSERT INTO Stagiaires (matricule, nom, formation, note_finale)
VALUES (405, 'Etonde', 'Réseaux', 15);

-- 2. Mise à jour conditionnelle
UPDATE Stagiaires SET note_finale = note_finale + 2 WHERE formation = 'Réseaux';

-- 3. Modification ciblée
UPDATE Stagiaires SET formation = 'Cybersécurité' WHERE matricule = 405;
💪 Pensez à toujours utiliser WHERE pour éviter des modifications massives involontaires.

🎯 Lien avec l’examen (Baccalauréat)

D’après les sessions 2025, 2024, 2022 : les questions de type DML sont fréquentes dans la Partie II (Bases de données). On demande souvent d’écrire des requêtes INSERT INTO ou UPDATE à partir d’un énoncé simple. Exemple typique (session 2024) : "Insérer un nouveau client dans la table Client" ou "Modifier le prix d’un produit dont l’identifiant est donné".

  • Maîtriser la syntaxe sans ambiguïté (parenthèses, guillemets, point-virgule).
  • Savoir que la clause WHERE peut utiliser plusieurs critères (AND, OR).
  • Ne pas confondre UPDATE et ALTER TABLE (qui modifie la structure).
