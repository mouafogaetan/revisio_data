📝 Exercice 1 – Algorithmique ▼

1. Valeurs affichées : A = 10, B = 2

Cheminement : A=5, B=2 → C=5+2=7 → Condition C>6 (7>6) vraie → A = 5×2 = 10. B reste inchangé. Affichage : 10 et 2.

2. Algorithme somme des entiers pairs :

Variables : N, i, somme : entiers
Début
    Lire(N)
    somme ← 0
    Pour i de 1 à N Faire
        Si i MOD 2 = 0 Alors
            somme ← somme + i
        Fin Si
    Fin Pour
    Afficher(somme)
Fin

3. Moyenne et maximum d'un tableau :

Variables : notes[5] : tableau réel, i : entier, somme, max, moyenne : réel
Début
    somme ← 0
    max ← notes[0]
    Pour i de 0 à 4 Faire
        somme ← somme + notes[i]
        Si notes[i] > max Alors
            max ← notes[i]
        Fin Si
    Fin Pour
    moyenne ← somme / 5
    Afficher("Moyenne =", moyenne, "Max =", max)
Fin

#include <stdio.h>

int main() {

📊 Exercice 3 – Tableaux en C ▼

1. Déclaration : float notes[20][3];

2. Fonction moyenne élève :

float calculerMoyenneEleve(float notes[][3], int eleve) {
    float somme = 0;
    for(int j = 0; j < 3; j++) {
        somme += notes[eleve][j];
    }
    return somme / 3;
}

3. Fonction meilleure matière :

int meilleureMatiere(float notes[][3], int eleve) {
    int meilleurIndice = 0;
    for(int j = 1; j < 3; j++) {
        if(notes[eleve][j] > notes[eleve][meilleurIndice]) {
            meilleurIndice = j;
        }
    }
    return meilleurIndice;  // 0: Maths, 1: Physique, 2: Info
}
🔄 Exercice 4 – Tri par sélection ▼

1. Déroulement du tri par sélection :

Tableau initial : [12, 7, 9, 3, 15, 8]

Passage 1 : minimum = 3 (indice 3) → échange avec indice 0 → [3, 7, 9, 12, 15, 8]
Passage 2 : minimum = 7 (indice 1) → déjà à sa place → [3, 7, 9, 12, 15, 8]
Passage 3 : minimum = 8 (indice 5) → échange avec indice 2 → [3, 7, 8, 12, 15, 9]
Passage 4 : minimum = 9 (indice 5) → échange avec indice 3 → [3, 7, 8, 9, 15, 12]
Passage 5 : minimum = 12 (indice 5) → échange avec indice 4 → [3, 7, 8, 9, 12, 15]
Tableau trié : [3, 7, 8, 9, 12, 15]

2. Intérêt de la recherche dichotomique : Dans un tableau trié, on peut diviser l'espace de recherche par 2 à chaque étape. Complexité en O(log n) au lieu de O(n) pour la recherche séquentielle. Principe : comparer l'élément recherché avec la valeur médiane, puis restreindre la recherche à la moitié gauche ou droite.
