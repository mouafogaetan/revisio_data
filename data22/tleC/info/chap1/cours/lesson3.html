📚 Leçon 3 : Boucles (Structures itératives)

Algorithmique · Terminale C & D · Session 2023-2024

🎯 Éléments clés à retenir

1. Qu'est-ce qu'une boucle ?

Une structure itérative (ou boucle) permet d'exécuter un bloc d'instructions plusieurs fois de suite, sans avoir à le réécrire. On parle d'itération. Chaque répétition est un tour de boucle. Une boucle doit toujours contenir une condition d'arrêt pour éviter les boucles infinies.

  • Utilité : Traitement de séries de données, calculs répétitifs, parcours de tableaux.
  • Composants : Initialisation, condition de poursuite, itération (incrémentation/décrémentation).
  • Représentation : En algorithmique, on utilise les mots-clés : Pour, Tant que, Répéter...Jusqu'à.

2. La boucle POUR (ou FOR)

Utilisée lorsque le nombre d'itérations est connu à l'avance. Elle s'appuie sur un compteur qui varie d'une valeur initiale à une valeur finale avec un pas.

Pour compteur ← valeur_initiale À valeur_finale [Pas pas] Faire
    // Instructions répétées
FinPour
  • Exemple : Afficher les entiers de 1 à 10 → Pour i de 1 à 10 faire Ecrire(i).
  • Incrémentation : Si le pas est omis, il vaut 1.
  • Ne jamais modifier le compteur manuellement à l'intérieur de la boucle.

3. La boucle TANT QUE (WHILE)

Boucle conditionnelle pré-testée : on vérifie la condition avant chaque itération. Si la condition est vraie, on exécute le bloc, puis on revérifie. Convient quand on ne connaît pas à l'avance le nombre d'itérations.

Tant que (condition) Faire
    // Instructions
    // Mise à jour des variables (pour que la condition devienne fausse un jour)
FinTantQue
  • Risque : Si la condition est toujours vraie → boucle infinie.
  • Cas typique : Somme d'entiers jusqu'à dépasser un seuil, saisie contrôlée.

4. La boucle RÉPÉTER... JUSQU'À (DO...WHILE)

Boucle post-testée : on exécute le bloc d'instructions une première fois, puis on vérifie la condition d'arrêt. Elle s'exécute donc au moins une fois.

Répéter
    // Instructions exécutées au moins une fois
Jusqu'à (condition_d_arrêt)
  • Dans Jusqu'à, la condition indique l'arrêt (contrairement à Tant que où elle indique la continuation).
  • Idéal pour les menus interactifs : on affiche le menu au moins une fois.

💡 Exemples de compréhension (Algorithmique)

Exemple 1 : Somme des entiers de 1 à N (Boucle POUR)

📌 Bac 2023 - Partie III

Écrire un algorithme qui lit un entier N > 0 et affiche la somme S = 1+2+...+N.

Variables : N, i, S : Entiers
Début
    Ecrire("Donner N : ")
    Lire(N)
    S ← 0
    Pour i ← 1 À N Faire
        S ← S + i
    FinPour
    Ecrire("La somme est : ", S)
Fin
💡 Astuce : La boucle POUR est idéale car le nombre de tours (N) est connu. Penser à initialiser S à 0 avant la boucle.

Exemple 2 : Saisie contrôlée (Boucle TANT QUE)

On veut lire une note comprise entre 0 et 20. Tant que l'utilisateur saisit une note invalide, on redemande.

Variables : note : Réel
Début
    Ecrire("Entrez une note (0-20) : ")
    Lire(note)
    Tant que (note < 0 OU note > 20) Faire
        Ecrire("Erreur, recommencez : ")
        Lire(note)
    FinTantQue
    Ecrire("Note valide : ", note)
Fin
⚠️ Important : La première saisie est lue avant la boucle, pour que la condition soit testée correctement.

Exemple 3 : Afficher un menu (RÉPÉTER... JUSQU'À)

Menu interactif : l'utilisateur choisit une option (1: Jouer, 2: Quitter). La boucle doit s'afficher au moins une fois.

Variables : choix : Entier
Début
    Répéter
        Ecrire("=== MENU ===")
        Ecrire("1. Jouer")
        Ecrire("2. Quitter")
        Lire(choix)
        Si choix = 1 Alors Ecrire("Démarrage jeu...") FinSi
    Jusqu'à (choix = 2)
    Ecrire("Au revoir")
Fin
💬 Particularité : La condition d'arrêt est testée à la fin, donc le bloc est exécuté au moins une fois même si l'utilisateur veut quitter immédiatement.

⚠️ Pièges courants & erreurs à éviter

  • Boucle infinie : Oublier de modifier la variable de condition dans un TANT QUE. Exemple : Tant que i < 10 Faire sans incrémenter i → bloc répété indéfiniment.
  • Confusion POUR / TANT QUE : Utiliser POUR quand le nombre d'itérations n'est pas connu à l'avance (ex: tant que l'utilisateur n'appuie pas sur 'Q').
  • Ne pas initialiser les accumulateurs : Avant une somme (S ← 0) ou un produit (P ← 1), toujours initialiser la variable.
  • Mauvaise interprétation de la condition dans Répéter...Jusqu'à : On s'arrête quand la condition est vraie, alors que dans Tant que on continue tant que la condition est vraie. Attention à la logique inverse.
  • Dépassement de bornes : Dans les tableaux, une boucle POUR qui va jusqu'à Taille inclus peut causer une erreur si l'indice maximum est taille-1.

🔁 Lien avec le Bac & Langage C

📖 D'après le programme officiel (Session 2023, 2024, 2025), la maîtrise des boucles est évaluée en algorithmique et en programmation C.

Transposition en langage C :

  • POUR → for (initialisation; condition; incrémentation) { ... }
  • TANT QUE → while (condition) { ... }
  • RÉPÉTER... JUSQU'À → do { ... } while (condition); (attention : en C, on continue tant que condition vraie, contrairement à "Jusqu'à" algorithmique où on s'arrête quand condition vraie → adapter la négation).
// Exemple en C : Somme des entiers de 1 à N (version for)
int somme = 0;
for (int i = 1; i <= N; i++) {
    somme += i;
}
printf("Somme = %d", somme);
🧠 À retenir pour l'écriture du code : i++ équivaut à i ← i+1 ; ne pas oublier les accolades pour les blocs de plusieurs instructions.
