Leçon 2 : Structures conditionnelles

Algorithmique · Terminale C & D (Scientifiques) · Baccalauréat

📌 Éléments clés à retenir

1. Principe des structures conditionnelles

Les structures conditionnelles permettent d'orienter le flux d'exécution d'un algorithme selon qu'une condition (expression logique) est vraie ou fausse. Elles introduisent des branchements (choix) dans le déroulement séquentiel.

  • Condition : expression booléenne (vraie ou fausse) utilisant des opérateurs de comparaison.
  • Alternative simple : Si condition Alors instruction(s) Fin Si.
  • Alternative double : Si condition Alors instruction(s) Sinon instruction(s) Fin Si.
  • Les instructions à l'intérieur d'un bloc peuvent être composées.

2. Opérateurs de comparaison et logiques

Pour écrire une condition, on utilise les opérateurs relationnels et éventuellement les opérateurs logiques.

  • Comparaison : = (égal), ≠ (différent), <, >, ≤, ≥.
  • Logiques : ET (les deux conditions vraies), OU (au moins une vraie), NON (inverse la valeur).
  • Exemple : (note ≥ 10) ET (note ≤ 20) → vérifie que note est entre 10 et 20.

3. Syntaxe algorithmique (pseudo-code)

En algorithmique, on utilise souvent une écriture structurée comme suit :

  • Alternative simple :
    Si condition Alors
        Instruction(s)
    Fin Si
  • Alternative double :
    Si condition Alors
        Instruction(s) vrai
    Sinon
        Instruction(s) faux
    Fin Si

💡 Exemples de compréhension

Exemple 1 : Alternative simple – Affichage d'un message

Écrire un algorithme qui lit un nombre entier N et affiche "POSITIF" si N est strictement supérieur à 0.

Solution :
Lire(N)
Si N > 0 Alors
    Ecrire("POSITIF")
Fin Si
💬 Astuce : Si la condition est fausse, rien ne s'affiche ; le programme continue après Fin Si.

Exemple 2 : Alternative double – Parité d'un nombre

Déterminer si un entier saisi est pair ou impair. Afficher "PAIR" ou "IMPAIR".

Algorithme :
Lire(nombre)
Si (nombre % 2 == 0) Alors
    Ecrire("PAIR")
Sinon
    Ecrire("IMPAIR")
Fin Si
Explication : L'opérateur modulo (%) donne le reste de la division par 2. Si reste 0 → pair.
💡 Penser aux parenthèses pour les conditions complexes : (a > 0) ET (b < 10).

Exemple 3 : Condition composée – Validation d'une note

Un professeur veut vérifier si une note saisie est valide (comprise entre 0 et 20). Si c'est le cas, afficher "Note valide", sinon afficher "Note invalide".

Solution :
Lire(note)
Si (note >= 0) ET (note <= 20) Alors
    Ecrire("Note valide")
Sinon
    Ecrire("Note invalide")
Fin Si
🧠 Une condition peut utiliser OU : par exemple, (age < 18) OU (age > 65) pour une réduction.

4. Alternatives multiples (Si ... Sinon Si ... Sinon)

Lorsqu'il y a plus de deux cas, on enchaîne plusieurs conditions avec Sinon Si.

  • Exemple : Afficher la mention selon une note :
    Si note < 10 Alors Ecrire("Insuffisant")
    Sinon Si note < 14 Alors Ecrire("Passable")
    Sinon Si note < 17 Alors Ecrire("Bien")
    Sinon Ecrire("Très bien")
    Fin Si
  • L'ordre des tests est important : dès qu'une condition est vraie, on exécute son bloc et on sort de la structure.

🎯 Entraînement type Bac (Session 2024, Partie III, Exercice 1)

Exercice : Calcul de réduction

Une librairie applique une réduction selon le montant total des achats :
- Si montant < 5000 FCFA → pas de réduction.
- Si montant entre 5000 et 10000 FCFA → réduction de 5%.
- Si montant > 10000 FCFA → réduction de 10%.
Écrire un algorithme qui lit le montant, calcule la réduction et affiche le montant à payer.

Correction :
Lire(montant)
Si montant < 5000 Alors
    reduction ← 0
Sinon Si montant ≤ 10000 Alors
    reduction ← montant * 5 / 100
Sinon
    reduction ← montant * 10 / 100
Fin Si
aPayer ← montant - reduction
Ecrire("Montant à payer : ", aPayer)
📌 Toujours structurer les Sinon Si pour éviter les redondances. Pensez aux cas limites (égalité).

⚠️ Pièges courants / À ne pas confondre

  • Confondre l'affectation (=) et la comparaison (==) : En algorithmique, on utilise souvent "=" pour la comparaison mais dans certains langages c'est "==". Attention au sens logique.
  • Oublier le "Fin Si" : Toute structure conditionnelle doit être fermée.
  • Conditions trop larges ou mal ordonnées : Par exemple, placer Si age > 18 avant Si age > 60 peut masquer le second cas. Toujours traiter les cas les plus spécifiques en premier.
  • Ne pas utiliser de parenthèses avec des opérateurs logiques (ET/OU) peut mener à des erreurs d’évaluation.
  • Erreur fréquente : Écrire Si a = 10 ET b = 20 au lieu de Si (a = 10) ET (b = 20) (bien que la logique soit la même, la clarté est essentielle).

✔️ Synthèse pour l'examen

  • Une structure conditionnelle répond à la question "Si ... alors ... sinon ...".
  • Les opérateurs de comparaison (>, <, =, ≠, ≥, ≤) construisent la condition.
  • Utilisez ET et OU pour combiner des sous-conditions.
  • N'oubliez pas d'indenter vos blocs pour une meilleure lisibilité.
  • Pensez aux cas d'égalité et aux valeurs limites (0, bornes supérieures).
