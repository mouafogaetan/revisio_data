🔧 Leçon 21 : Composants d’un système informatique

Informatique – Terminale C & D (Scientifiques) | Programme officiel 2025-2026

📌 Éléments clés à retenir

💻 1. Qu'est-ce qu'un système informatique ?

Un système informatique est un ensemble organisé de ressources matérielles (hardware) et logicielles (software) capable de traiter, stocker et communiquer des informations. Il repose sur une architecture de type Von Neumann qui regroupe processeur, mémoire, unités d’entrée/sortie et bus de communication.

  • Partie matérielle (hardware) : éléments physiques (carte mère, processeur, RAM, disque dur, périphériques).
  • Partie logicielle (software) : système d’exploitation (Windows, Linux), applications, pilotes.
  • Rôle central : exécuter des programmes, automatiser des tâches, faciliter la gestion de données.

🧠 2. Composants essentiels & caractéristiques

Chaque composant a une fonction précise et des caractéristiques techniques à connaître pour l’examen.

ComposantFonction principaleCaractéristiques clés
CPU (Processeur)Exécute les instructions, traite les données.Fréquence (GHz), nombre de cœurs, mémoire cache.
RAM (Mémoire vive)Stocke temporairement les données et programmes en cours d’exécution.Capacité (Go), vitesse (MHz), volatile.
Disque dur / SSDStockage permanent des données (OS, fichiers, applications).Capacité (To/Go), vitesse lecture/écriture, technologie (HDD/SSD).
Carte mèreCircuit principal qui relie tous les composants (bus, chipset, slots).Socket processeur, chipsets, connecteurs (RAM, PCIe, SATA).
Alimentation (PSU)Convertit le courant secteur en tensions stables pour les composants.Puissance (Watts), rendement (80+).
Carte graphique (GPU)Traite les images et accélère l’affichage (jeux, vidéo, 3D).Mémoire vidéo (VRAM), fréquence, interfaces.
💡 À retenir : La RAM est volatile (perte des données à l’extinction) alors que le disque dur conserve les données.

📏 3. Unités de mesure en informatique

Pour comparer les performances ou capacités des composants, on utilise des multiples d’octets (stockage) et de Hertz (fréquence).

  • Stockage : 1 Ko = 1 024 octets ; 1 Mo = 1 024 Ko ; 1 Go = 1 024 Mo ; 1 To = 1 024 Go.
  • Fréquence (CPU/RAM) : 1 Hz = 1 cycle/seconde ; 1 MHz = 10⁶ Hz ; 1 GHz = 10⁹ Hz.
  • Taille des registres : 32 bits vs 64 bits (capacité de traitement par cycle).
💾 Temps d’accès moyen = latence + taille_de_transfert / bande_passante

💡 Exemples de compréhension

🖥️ Exemple 1 : Comparer deux configurations pour un usage bureautique

Config A : Intel Core i3 2,4 GHz / 4 Go RAM / HDD 500 Go.
Config B : Intel Core i5 3,0 GHz / 8 Go RAM / SSD 256 Go.

Analyse : La config B possède un processeur plus rapide (fréquence + cœurs), deux fois plus de RAM (fluidité multitâche) et un SSD (démarrage et ouverture des logiciels bien plus rapide qu’un HDD). Elle est recommandée pour une utilisation polyvalente et réactive.

💬 Astuce : Pour les jeux ou montage vidéo, prioriser la carte graphique (GPU) et beaucoup de RAM (16 Go ou plus). Pour la bureautique, un SSD est l’amélioration la plus notable.

⚡ Exemple 2 : Pourquoi le processeur utilise-t-il plusieurs niveaux de cache ?

Un processeur cadencé à 3,5 GHz effectue 3,5 milliards d’opérations par seconde. La RAM met environ 50 ns (nanosecondes) pour répondre. Sans cache, le CPU attendrait souvent les données.

Explication : La mémoire cache (L1, L2, L3) est une mémoire ultra-rapide intégrée au processeur. Elle stocke les données et instructions les plus fréquemment utilisées. Ainsi, le CPU les récupère en quelques cycles au lieu d’attendre la RAM.
➜ Cache L1 : très petite (32 Ko), vitesse extrême.
➜ Cache L2/L3 : plus grande mais légèrement plus lente que L1.

💡 À retenir : Plus le cache est grand (surtout L3), meilleures sont les performances dans les tâches répétitives (jeux, bases de données).

📝 Examen type : Identifier un composant d’après sa description

« Je suis un circuit imprimé qui accueille le processeur, la RAM, les connecteurs de stockage et les périphériques. Sans moi, aucun composant ne peut communiquer. Qui suis-je ? »

Réponse : La carte mère (motherboard). Elle contient le chipset, les bus de données et les slots d’extension.

🧠 Piège fréquent : Ne pas confondre la carte mère avec le processeur. Le processeur est un composant posé sur la carte mère.

⚠️ Pièges courants / À ne pas confondre

  • Confusion mémoire vive (RAM) et mémoire de masse : La RAM est volatile et rapide, le disque dur est non volatile mais plus lent. Un programme en cours d’exécution réside en RAM.
  • Mégaoctet (Mo) vs Mégabit (Mb) : 1 octet = 8 bits. Les débits réseau sont souvent en Mbit/s, les capacités de stockage en Mo/Go. Attention aux conversions.
  • Fréquence seule ne définit pas la performance : Un processeur moderne avec plus de cœurs et une meilleure architecture peut être plus efficace qu’un ancien avec fréquence plus élevée.
  • Oublier les bus de communication : Même avec de bons composants, un bus lent (ex: PCIe 2.0 au lieu de 4.0) peut limiter les débits (carte graphique, SSD NVMe).

🔌 Les bus et l’interconnexion

🚌 Bus système et communication

Les composants échangent des données via des bus (ensemble de lignes électriques). On distingue :

  • Bus de données : transporte les données entre CPU, RAM, périphériques. Largeur (8, 16, 32, 64 bits) détermine la quantité transférée par cycle.
  • Bus d’adresses : véhicule l’adresse mémoire où lire/écrire.
  • Bus de contrôle : signaux de commande (lecture, écriture, interruption).

Exemple concret : un processeur 64 bits associé à un bus 64 bits peut lire 8 octets à la fois, accélérant les traitements.
