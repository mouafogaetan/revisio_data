UE 7 : Représentation des données dans l'ordinateur

Informatique · Classe de 3ème · Module 1 – Organisation des données

📌 Éléments clés à retenir

1. Le système binaire : le langage de l'ordinateur

L'ordinateur ne comprend que deux états électriques : 0 (absence de signal) et 1 (présence de signal). C'est ce qu'on appelle le système binaire. Chaque chiffre binaire (0 ou 1) est appelé un bit (binary digit). Toutes les données (nombres, textes, images, sons) sont converties en une suite de 0 et de 1 pour être traitées par la machine.

  • Un bit est l'unité d'information la plus petite.
  • Un octet (byte) = groupe de 8 bits. Exemple : 10110101 est un octet.
  • Multiples courants : 1 Ko (kilooctet) = 1024 octets ; 1 Mo = 1024 Ko ; 1 Go = 1024 Mo.
  • Formule de conversion décimal → binaire : on divise successivement par 2 et on lit les restes.

🔢 Exemple de conversion : Le nombre décimal 13 en binaire

13 ÷ 2 = 6 reste 1
6 ÷ 2 = 3 reste 0
3 ÷ 2 = 1 reste 1
1 ÷ 2 = 0 reste 1

Lecture des restes du dernier au premier : 1101₂ (soit 1×8 + 1×4 + 0×2 + 1×1 = 13).

2. Représentation des nombres entiers

En binaire, chaque position correspond à une puissance de 2 (1, 2, 4, 8, 16…). Les nombres entiers positifs sont représentés directement par leur équivalent binaire. L'ordinateur utilise un nombre fixe de bits (ex: 8 bits, 16 bits) pour coder un nombre.

  • Avec n bits, on peut représenter $2^n$ valeurs différentes (de 0 à $2^n - 1$).
  • Exemple : avec 4 bits → 16 valeurs possibles (0 à 15).
  • Valeur décimale = somme (bit × 2^{poids})

📌 Représentation des nombres négatifs : on utilise souvent la méthode du complément à deux (plus avancé, mais le principe : le bit de poids fort indique le signe).

3. Code ASCII : le langage des caractères

Pour représenter les lettres, chiffres et symboles, l'ordinateur utilise un code standard : ASCII (American Standard Code for Information Interchange). Chaque caractère est associé à un nombre (de 0 à 127 pour l'ASCII de base), puis converti en binaire.

A → 65 → 01000001
B → 66 → 01000010
a → 97 → 01100001
0 → 48 → 00110000
1 → 49 → 00110001
! → 33 → 00100001
  • L'ASCII étendu (256 caractères) permet d'ajouter des lettres accentuées et symboles.
  • De nos jours, Unicode (UTF-8) remplace ASCII pour gérer toutes les écritures du monde (plus de 100 000 caractères).
  • Le texte dans un ordinateur est donc une longue suite d'octets, chaque octet (ou plusieurs) représentant un caractère.

💡 Illustration : Chaque touche du clavier est convertie en code ASCII, puis en binaire pour le processeur.

4. Représentation binaire : pourquoi deux états ?

La logique binaire est fiable et économique pour les circuits électroniques : un interrupteur (transistor) peut être passant (1) ou bloqué (0). Moins de risque d'erreur qu'avec dix niveaux de tension (système décimal).

  • Bit = Binary digit : unité élémentaire de l'information numérique.
  • Octet = 8 bits : unité de stockage courante (un caractère ASCII tient sur 1 octet).
  • La mémoire et les disques durs sont organisés en octets.

💡 Exemples de compréhension

Exemple 1 : Convertir un nombre décimal en binaire

Convertir le nombre 25 (en base 10) en binaire sur 8 bits.

Résolution :
1️⃣ 25 ÷ 2 = 12 reste 1
2️⃣ 12 ÷ 2 = 6 reste 0
3️⃣ 6 ÷ 2 = 3 reste 0
4️⃣ 3 ÷ 2 = 1 reste 1
5️⃣ 1 ÷ 2 = 0 reste 1
On lit les restes du dernier au premier : 11001₂. Sur 8 bits, on ajoute des zéros devant : 00011001₂.

💬 Astuce : Vérification rapide : 16+8+0+0+1 = 25. La méthode des divisions successives fonctionne pour tout entier positif.

Exemple 2 : Interpréter un code ASCII

Le mot "3A" est stocké en mémoire sous forme binaire. Que signifie la séquence suivante : 00110011 01000001 ?

Analyse : Chaque octet correspond à un caractère ASCII.
- Premier octet : 00110011 = 51 en décimal → selon la table ASCII, 51 correspond au caractère '3'.
- Deuxième octet : 01000001 = 65 en décimal → caractère 'A'.
Donc la chaîne représente "3A".

📖 Mémorisez quelques codes utiles : 'A'=65, 'a'=97, '0'=48. L'ordre alphabétique est conservé.

Exemple 3 : Capacité de stockage et bits

Un fichier texte contient 500 caractères (lettres + espaces). Chaque caractère est codé sur 1 octet en ASCII. Quelle est la taille du fichier en bits ? En kilo-octets ?

Calcul :
- 1 caractère = 1 octet = 8 bits.
- 500 caractères = 500 octets.
- En bits : 500 × 8 = 4000 bits.
- En kilo-octets (1 Ko = 1024 octets) : 500 / 1024 ≈ 0,488 Ko.

⚡ Astuce : Un texte simple de quelques lignes pèse souvent quelques Ko. Les images et vidéos utilisent des millions d'octets.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Confondre bit et octet. Un octet = 8 bits. La taille des fichiers s'exprime souvent en octets (Ko, Mo) mais la vitesse de connexion en bits par seconde (Mbit/s).
  • Attention : La conversion binaire se lit du dernier reste au premier. Ne pas inverser l'ordre des chiffres.
  • Piège : ASCII ne gère pas tous les accents. Pour le français, il faut utiliser l'encodage UTF-8 (compatible Unicode) pour afficher correctement "é", "ç", etc.
  • À ne pas oublier : Un même nombre binaire peut représenter soit un entier, soit un caractère (selon le contexte d’interprétation).
  • Confusion unités : 1 Ko = 1024 octets et non 1000 octets (sauf dans certains contextes constructeurs).
