UE 24 : Écriture d'un algorithme simple

Informatique · Classe de 3ème · Algorithmique

📌 Éléments clés à retenir

1. Qu'est-ce qu'un algorithme ?

Un algorithme est une suite d'instructions précises et ordonnées qui décrit, étape par étape, la résolution d'un problème. Il doit être compréhensible par un humain (langage naturel) avant d'être traduit en langage de programmation.

  • Propriétés essentielles : précision, finitude (il doit se terminer), entrées et sorties définies.
  • Structure générale : un nom, des déclarations de variables, un corps d'instructions.
  • Syntaxe simplifiée : on utilise des mots-clés comme Début, Fin, Lire, Afficher, ← (affectation).

2. Les instructions de base

Pour écrire un algorithme simple, on utilise trois types d'instructions fondamentales.

  • Entrée (Lire) : permet de saisir une donnée depuis l'utilisateur.
    Lire variable ou Lire nom, age.
  • Sortie (Afficher) : permet d'afficher un message ou le contenu d'une variable à l'écran.
    Afficher "Bonjour" ou Afficher résultat.
  • Affectation (←) : permet de stocker une valeur (ou le résultat d'un calcul) dans une variable.
    prix ← 15.5 ou total ← prix * quantité.

3. Les variables et leur déclaration

Une variable est un espace mémoire nommé qui contient une valeur pouvant changer au cours de l'exécution. On doit préciser son type (ce qu'elle peut contenir) en début d'algorithme.

  • Type numérique : Entier (nombre sans virgule), Réel (nombre à virgule).
  • Type texte : Chaîne ou Caractère.
  • Type logique : Booléen (Vrai / Faux).
  • Exemple : Variable age en Entier, nom en Chaîne

💡 Exemples de compréhension

Exemple 1 : Calcul de la moyenne de deux notes

Problème : Écrire un algorithme qui demande deux notes à l'utilisateur, calcule leur moyenne, puis affiche le résultat.

Algorithme CalculMoyenne
// Déclaration des variables
Variable note1, note2, moyenne : Réel

Début
    // Saisie des notes
    Afficher "Entrez la première note : "
    Lire note1
    Afficher "Entrez la deuxième note : "
    Lire note2
    
    // Calcul de la moyenne
    moyenne ← (note1 + note2) / 2
    
    // Affichage du résultat
    Afficher "La moyenne est : ", moyenne
Fin
💬 Astuce : Utilisez des noms de variables explicites (note1, note2) pour rendre l'algorithme plus lisible. L'opérateur de division ( / ) fonctionne avec les réels.

Exemple 2 : Calcul de l'aire d'un rectangle

Problème : Concevoir un algorithme qui lit la longueur et la largeur d'un rectangle, calcule son aire (longueur × largeur), puis affiche le résultat.

Algorithme AireRectangle
Variable longueur, largeur, aire : Réel

Début
    Afficher "Entrez la longueur du rectangle : "
    Lire longueur
    Afficher "Entrez la largeur du rectangle : "
    Lire largeur
    aire ← longueur * largeur
    Afficher "L'aire du rectangle est : ", aire
Fin
💡 Penser à bien utiliser la multiplication (*) et à choisir un type réel si les mesures peuvent être décimales.

Exemple 3 : Conversion de minutes en heures

Problème : Écrire un algorithme qui demande un nombre de minutes à l'utilisateur et affiche l'équivalent en heures et minutes (ex: 130 minutes = 2 heures et 10 minutes).

Algorithme ConversionMinutes
Variable minutesTotal, heures, minutesRestantes : Entier

Début
    Afficher "Entrez le nombre de minutes : "
    Lire minutesTotal
    heures ← minutesTotal / 60    // Division entière
    minutesRestantes ← minutesTotal % 60    // Reste de la division
    Afficher minutesTotal, " minutes = ", heures, " heures et ", minutesRestantes, " minutes"
Fin
💬 Astuce : L'opérateur % (modulo) donne le reste de la division entière. Très utile pour séparer les unités.

✍️ À vous de jouer ! (Exercices d'application)

Exercice 1 : Périmètre d'un cercle

Écrire un algorithme qui demande le rayon d'un cercle, calcule son périmètre (formule : \( P = 2 \times \pi \times R \), avec \(\pi \approx 3.14\)) et affiche le résultat.

📖 Voir une solution possible
Algorithme PerimetreCercle
Variable rayon, perimetre : Réel
Constante PI = 3.14
Début
    Afficher "Entrez le rayon du cercle : "
    Lire rayon
    perimetre ← 2 * PI * rayon
    Afficher "Le périmètre du cercle est : ", perimetre
Fin

Exercice 2 : Échange de deux nombres

Écrire un algorithme qui lit deux nombres A et B, puis échange leurs valeurs (A prend la valeur de B et B prend la valeur de A) avant de les afficher.

📖 Voir une solution possible
Algorithme EchangeNombres
Variable A, B, temporaire : Entier
Début
    Afficher "Entrez la valeur de A : "
    Lire A
    Afficher "Entrez la valeur de B : "
    Lire B
    // Échange des valeurs
    temporaire ← A
    A ← B
    B ← temporaire
    Afficher "Après échange : A = ", A, " et B = ", B
Fin

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Confondre l'opérateur d'affectation (←) avec le signe égal mathématique. En algorithmique, x ← x + 1 est une instruction valide qui incrémente x de 1.
  • Attention : Oublier de déclarer une variable avant de l'utiliser. Toute variable doit être listée dans la section "Variable".
  • À ne pas confondre : Lire (entrée) et Afficher (sortie). Le premier récupère une donnée, le second affiche une information.
  • Piège : Ne pas respecter l'ordre des instructions. L'algorithme s'exécute séquentiellement de haut en bas.
  • Bon réflexe : Utiliser des noms de variables clairs (somme, age, note) pour que l'algorithme soit auto-documenté.
