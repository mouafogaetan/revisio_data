Leçon 18 : Regroupements en classes (d'égales amplitudes)

Mathématiques · Troisième · Statistiques

📌 Éléments clés à retenir

1. Pourquoi regrouper des données en classes ?

Lorsqu'une série statistique contient un grand nombre de valeurs différentes ou des valeurs continues (comme des tailles, des poids, des temps), il devient difficile de les détailler une par une. Le regroupement en classes permet de synthétiser l'information en créant des intervalles de valeurs. On parle alors de série statistique continue.

2. Classes d'égales amplitudes

Une classe est définie par un intervalle \([x_{min}, x_{max}[\) (ou fermé/ouvert selon le contexte). L'amplitude d'une classe est la différence entre sa borne supérieure et sa borne inférieure. Des classes sont dites d'égales amplitudes lorsque cet écart est le même pour toutes les classes.

  • Formule : Amplitude d’une classe = Borne supérieure − Borne inférieure.
  • Exemple : Classes [10 ; 20[, [20 ; 30[, [30 ; 40[ ont toutes une amplitude de 10.
  • Nombre de classes = (Valeur maximale - Valeur minimale) / Amplitude choisie (arrondi à l’entier supérieur).

3. Tableau d'effectifs et de fréquences

Une fois les classes définies, on construit un tableau qui récapitule pour chaque classe :

  • L’effectif : le nombre d’individus ou de données appartenant à la classe.
  • La fréquence : le rapport entre l’effectif de la classe et l’effectif total. fréquence = effectif de la classe / effectif total.
  • La fréquence peut s’exprimer en fraction, en nombre décimal ou en pourcentage.
  • La somme des fréquences est toujours égale à 1 (ou 100%).

💡 Exemples de compréhension

Exemple concret n°1 : Répartition des tailles

Dans une classe de 25 élèves, on mesure les tailles (en cm) : 145 ; 148 ; 152 ; 150 ; 147 ; 149 ; 153 ; 151 ; 155 ; 146 ; 148 ; 152 ; 154 ; 150 ; 149 ; 147 ; 151 ; 153 ; 148 ; 150 ; 152 ; 146 ; 149 ; 151 ; 155. Regrouper ces données en classes d’amplitude 3 cm à partir de 145 cm.

Résolution :
  • Étape 1 : Déterminer les classes. Amplitude = 3. Donc classes : [145 ; 148[, [148 ; 151[, [151 ; 154[, [154 ; 157[.
  • Étape 2 : Compter les effectifs.
    • Classe [145 ; 148[ → tailles 145, 147, 146, 147, 146 → effectif = 5.
    • Classe [148 ; 151[ → tailles 148, 150, 149, 148, 150, 149, 148, 150 → effectif = 8.
    • Classe [151 ; 154[ → tailles 152, 153, 151, 152, 151, 153, 152, 151 → effectif = 8.
    • Classe [154 ; 157[ → tailles 155, 154, 155 → effectif = 4.
  • Étape 3 : Calcul des fréquences. Total = 25.
    • Freq₁ = 5/25 = 0,20 = 20%.
    • Freq₂ = 8/25 = 0,32 = 32%.
    • Freq₃ = 8/25 = 0,32 = 32%.
    • Freq₄ = 4/25 = 0,16 = 16%.
💬 Astuce : Pour compter rapidement les effectifs, on peut ordonner les données ou utiliser un tableau de pointage (on barre chaque donnée).

Exemple d'application n°2 : Poids de colis

Une entreprise livre 40 colis. Leurs poids (en kg) sont compris entre 2 kg et 15 kg. On souhaite créer 5 classes d’égales amplitudes pour analyser la répartition.

Calcul de l’amplitude :
Amplitude = (Valeur max − Valeur min) / Nombre de classes = (15 - 2) / 5 = 13 / 5 = 2,6. On arrondit souvent à l’unité supérieure ou on choisit une amplitude pratique. Ici, prenons une amplitude de 3 kg.
Classes possibles : [2 ; 5[, [5 ; 8[, [8 ; 11[, [11 ; 14[, [14 ; 17[ (pour couvrir 15 kg).
Ensuite, on remplirait le tableau d’effectifs selon les données réelles des colis.
💡 Il est important que les classes soient disjointes (un individu ne peut appartenir à deux classes) et qu’elles couvrent toutes les valeurs possibles.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier de définir correctement les bornes des classes (incluse ou exclue). On utilise souvent [a ; b[ pour éviter les doubles comptes.
  • Attention : Des classes d’amplitudes différentes ne permettent pas de comparer directement les effectifs ou de construire un histogramme classique.
  • Piège : Confondre l’effectif cumulé croissant avec l’effectif simple. L’effectif cumulé se calcule en ajoutant les effectifs des classes précédentes.
  • À ne pas faire : Utiliser des classes qui ne se touchent pas (ex: [0-5] et [6-10]) sans raison valable, cela crée des trous dans l’analyse.

📊 Représentation graphique : L'histogramme

Lien avec le diagramme en bâtons (leçon 19)

Pour des données regroupées en classes d’égales amplitudes, on utilise un histogramme : chaque classe est représentée par un rectangle dont la largeur est proportionnelle à l’amplitude (égale ici) et la hauteur est proportionnelle à l’effectif (ou à la densité de fréquence). Cela permet de visualiser rapidement la distribution des données.

Contrairement au diagramme en bâtons utilisé pour des valeurs isolées (données discrètes), l’histogramme est adapté aux données continues regroupées en classes.
