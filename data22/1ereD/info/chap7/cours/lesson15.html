📊 Les tableaux en algorithmique et en langage C

Unité d’apprentissage 15 · Structures de données · Première C, D, E | Informatique

📌 Éléments clés à retenir

🔹 Définition : Qu'est-ce qu'un tableau ?

Un tableau est une structure de données qui permet de stocker plusieurs valeurs de même type (entiers, réels, caractères, etc.) sous un même nom. Chaque valeur est repérée par un indice (ou index). C’est comme une “boîte compartimentée” : chaque compartiment contient une donnée et porte un numéro.

  • Avantage : manipuler facilement une collection de données (notes d’une classe, températures, etc.).
  • Les cases sont contiguës en mémoire → accès direct et rapide.
  • En algorithmique, le premier indice est généralement 1 ; en langage C, le premier indice est 0.
📋 Exemple visuel (tableau d'entiers) :
[ indice 1 ] [ indice 2 ] [ indice 3 ] [ indice 4 ] [ indice 5 ]
| 17 | | 23 | | 45 | | 12 | | 8 |
💡 En algorithmique : tab[1] = 17 , tab[2] = 23 …
💻 En langage C : tab[0] = 17 , tab[1] = 23 … (décalage important !)

🔹 Déclarer un tableau

Avant d'utiliser un tableau, il faut indiquer son nom, son type et sa taille (nombre maximal d’éléments).

📐 Algorithmique

Variable: tabNote : tableau[1..10] d’entier;
→ tableau de 10 entiers, indices de 1 à 10.

Autre exemple :
Tableau notes[1..N] de réels (N connu)

⚙️ Langage C

int tabNote[10];
→ tableau de 10 entiers, indices de 0 à 9.

Exemple réel :
float moyenne[20]; (20 réels)

⚠️ La taille doit être une constante (connue à la compilation) en C classique. En algorithmique, on peut utiliser une variable pour la taille (ex: [1..N]).

🔹 Accéder et modifier les éléments

Pour lire ou écrire une valeur, on utilise le nom du tableau suivi de l’indice entre crochets.

  • Lire une valeur : x ← tab[3] (algo) ou x = tab[2]; (C, car indice 2 correspond à la 3e case)
  • Affecter une valeur : tab[5] ← 99 (algo) ou tab[4] = 99; (C)
  • Modifier une valeur existante : tabAge[10] ← tabAge[10] + 3; (ajouter 3 à la 10e case en algo)
💬 Piège fréquent : En langage C, l’indice commence à 0, donc le dernier élément est à l’indice taille-1. Si le tableau a 10 cases, les indices valides sont 0,1,2,...,9. L’indice 10 provoque une erreur (dépassement).

🔹 Parcourir un tableau avec une boucle

L’intérêt des tableaux apparaît avec les boucles : on peut traiter tous les éléments rapidement.

Exemple algorithmique – Afficher toutes les notes d’un tableau de 15 élèves :

Pour i allant de 1 à 15 faire
    Ecrire("Note de l'élève ", i, " : ", tabNote[i])
FinPour

Traduction en langage C :

for(i = 0; i < 15; i++) {
    printf("Note de l'élève %d : %.2f\n", i+1, tabNote[i]);
}
🧠 À retenir : la boucle for est l’outil idéal pour parcourir un tableau. En C, la condition d’arrêt est i < taille (jamais <= taille).

💡 Exemples de compréhension

📐 Exemple 1 : Somme des éléments d’un tableau

On dispose d’un tableau temp[1..7] contenant les températures relevées chaque jour de la semaine. Calculer la température moyenne.

Résolution pas à pas (algorithmique) :

  • Initialiser une variable somme ← 0.
  • Parcourir les indices de 1 à 7 : somme ← somme + temp[i].
  • Moyenne = somme / 7.
Variable: temp : tableau[1..7] de réel
Variable: somme, moyenne : réel
Début
    somme ← 0
    Pour i de 1 à 7 faire
        somme ← somme + temp[i]
    FinPour
    moyenne ← somme / 7
    Ecrire("Moyenne = ", moyenne)
Fin
💡 Astuce : en C, il faut veiller à ce que les indices aillent de 0 à 6. Si le tableau est float temp[7]; la boucle devient for(i=0; i<7; i++).

🔍 Exemple 2 : Rechercher la plus grande valeur

Un tableau notes[1..N] contient les notes sur 20 d’une classe. Écrire l’algorithme qui détermine la meilleure note.

Principe : On suppose que le premier élément est le maximum, puis on compare avec les suivants. 

max ← notes[1]
Pour i de 2 à N faire
    Si notes[i] > max alors
        max ← notes[i]
    FinSi
FinPour
Ecrire("Note maximale : ", max)
🎯 À retenir : l'initialisation avec la première valeur évite les erreurs de seuil. Si le tableau était vide, il faudrait un cas particulier.

✏️ Exemple 3 : Mise à jour d’une valeur

On a un tableau tabAge de 10 entiers représentant les âges de personnes. On souhaite augmenter de 3 ans l’âge de la 10ᵉ personne (indice 10 en algorithmique).

En algorithmique : tabAge[10] ← tabAge[10] + 3
En langage C (tableau de 10 cases, indices 0 à 9) : tabAge[9] = tabAge[9] + 3; ou tabAge[9] += 3;

🧩 La notation += est un raccourci fréquent en C : tabAge[9] += 3; équivaut à ajouter 3.

🔄 Traduction algorithmique ↔ Langage C

Correspondances essentielles

  • Déclaration algo : Tableau notes[1..20] de réels → C : float notes[20];
  • Accès en algo : notes[3] (3e élément) → C : notes[2] (décalage -1).
  • Parcours algo : Pour i de 1 à taille faire → C : for(i=0; i<taille; i++)
  • Lecture / écriture : idem syntaxe mais attention aux indices.
📌 Erreur classique en C : écrire for(i=0; i<=taille; i++) accède à l’indice taille qui n’existe pas (dépassement de mémoire).

⚠️ Pièges courants & erreurs à éviter

  • Confusion des indices : En algorithmique on commence à 1, en C à 0. L’élément d’indice 5 en algo correspond à l’indice 4 en C.
  • Dépassement de tableau (out of bounds) : Accéder à tab[taille] (C) provoque un comportement imprévisible.
  • Oublier de dimensionner : Un tableau doit avoir une taille fixe connue. En C, int t[]; sans taille est invalide (sauf initialisation).
  • Ne pas initialiser les valeurs : Un tableau local en C contient des valeurs aléatoires (poubelle). Penser à les initialiser avant lecture.
  • Utiliser une variable non entière comme indice : L’indice doit être un entier (ou expression entière).

🧠 Auto-évaluation (questions flash)

  1. Déclarer en algorithmique un tableau taille[1..12] d’entiers.
  2. En langage C, comment accéder au 7e élément d’un tableau nommé valeurs ?
  3. Que fait cette instruction en C : notes[3] = notes[2] + 5; ?
  4. Écrire la boucle en C qui parcourt un tableau temp[100] pour afficher chaque élément.
📌 Voir les réponses

1. Variable: taille : tableau[1..12] d’entier;
2. valeurs[6] (car 7e élément → indice 6 en C)
3. Elle ajoute 5 à la valeur de l’indice 2 (3e case) et affecte le résultat à l’indice 3 (4e case).
4. for(i=0; i<100; i++) { printf("%f ", temp[i]); }
