📚 UE 14 : Généralités sur les structures de données

Informatique · Première (Séries C, D, E) · Module 2 : Algorithmique & Programmation

📌 Éléments clés à retenir

🔹 Qu'est-ce qu'une structure de données ?

Une structure de données est une organisation logique des données en mémoire permettant de les stocker, de les manipuler et d’y accéder efficacement. C’est un peu comme un « contenant » structuré qui définit la façon dont les informations sont rangées et reliées entre elles.

  • Elle détermine comment les données sont agencées (en séquence, en arborescence, etc.).
  • Elle offre des opérations spécifiques : ajout, suppression, recherche, parcours.
  • Choisir la bonne structure de données améliore performance et clarté du programme.
📂 Analogies simples : une bibliothèque (rayonnages, étagères) organise les livres ; une structure de données organise les variables en mémoire.

🔹 Types de structures de données (à connaître)

En algorithmique et en programmation, on distingue plusieurs grandes familles. Chacune répond à un besoin spécifique.

  • Variable simple Types primitifs : entier, réel, caractère, booléen. Une seule valeur à la fois.
  • Tableau (array) Collection d’éléments de même type, accessibles par un indice (position).
  • Liste chaînée Éléments liés par des pointeurs (dynamique, insertion/suppression flexible).
  • Pile (stack) Principe LIFO (Last In, First Out) : le dernier arrivé est le premier sorti.
  • File (queue) Principe FIFO (First In, First Out) : premier arrivé, premier servi.
  • Arbre / graphe Relations hiérarchiques ou complexes (programmation avancée).

✅ À retenir pour le Probatoire : maîtriser les variables simples, les tableaux (UE 15), et savoir distinguer une pile d’une file.

🔹 Opérations de base sur les structures

Indépendamment de la structure choisie, certaines manipulations sont fondamentales :

  • Initialisation : créer la structure et lui donner un état initial (ex: tableau vide).
  • Ajout / Insertion : placer une nouvelle donnée.
  • Suppression : enlever une donnée existante.
  • Accès / Consultation : lire la valeur d’un élément.
  • Parcours / itération : visiter chaque élément (souvent avec une boucle).
  • Recherche : trouver si une valeur est présente.
💡 Conseil examen : On vous demandera souvent d’écrire un algorithme qui manipule un tableau ou une pile. Ayez toujours en tête les opérations de base.

💡 Exemples de compréhension

📌 Exemple 1 : Variables simples vs Tableau

Problème : On veut stocker les notes de 5 élèves dans un programme. Version 1 : utiliser 5 variables note1, note2, note3, note4, note5. Version 2 : utiliser un tableau notes[1..5] (en algo) ou notes[5] en C.

✅ Explication : Avec un tableau, on peut facilement parcourir toutes les notes avec une boucle POUR. La structure de données « tableau » permet de regrouper des données de même nature et d’y accéder par indice, ce qui rend le code plus concis et évolutif (si on a 100 élèves, la version simple devient impossible).

📌 Astuce : Dès que vous devez gérer plusieurs valeurs de même type et de même signification, pensez tableau.

📌 Exemple 2 : Pile de livres (Principe LIFO)

Une pile d’assiettes ou de livres : on empile le dernier livre au sommet. Lorsqu’on veut en retirer un, on prend d’abord celui du dessus (le dernier ajouté).

Opérations typiques d'une pile :
- Empiler (push) : ajouter un élément au sommet.
- Dépiler (pop) : retirer l’élément du sommet.
En algorithmique, on utilise souvent une pile pour annuler/revenir en arrière (Ctrl+Z), ou dans l’évaluation d’expressions.

🧱 État d’une pile : [Base] → livreA → livreB → livreC (sommet). Si on dépile, on récupère livreC. Si on empile livreD, il devient le nouveau sommet.

📌 Exemple 3 : File à la cantine (FIFO)

Dans une file d’attente (supermarché, billetterie), le premier arrivé est le premier servi.

Opérations d'une file :
- Enfiler (enqueue) : ajouter un élément à la fin de la file.
- Défiler (dequeue) : retirer l’élément en tête de file.
Les files sont utilisées dans la gestion des impressions (documents en attente), ou dans les files d’attente des processus en système d’exploitation.

🧠 Différence clé à mémoriser : Pile = LIFO (Last In, First Out) / File = FIFO (First In, First Out). Ne pas confondre !

📌 Exemple 4 : Variables simples (booléen, caractère)

Dans un algorithme de validation de mot de passe, on utilise une variable estValide de type booléen (Vrai/Faux). Pour un menu de choix, on peut utiliser une variable choix de type caractère ('A', 'B', 'C').

Les types simples sont la brique de base de toute structure de données. Exemple en langage C : int age = 17; (entier), float moyenne = 14.5; (réel), char sexe = 'F'; (caractère), int ok = 1; (booléen simulé).

⚠️ Pièges courants & erreurs fréquentes

  • Confondre Pile et File : Retenez "LIFO = Pile (Last In First Out)" et "FIFO = File (First In First Out)". Une astuce : pensez à une pile d’assiettes (dernière posée = première enlevée) ; une file de personnes à un guichet.
  • Oublier que les tableaux commencent à l’indice 0 en langage C : En algorithmique, on utilise souvent l’indice 1..N mais en C le premier élément est tab[0]. Cela génère des erreurs de décalage.
  • Ne pas distinguer variable simple et tableau : Une variable simple ne peut contenir qu’une seule valeur ; un tableau en contient plusieurs indexées. Utiliser une variable pour stocker 50 notes est impossible.
  • Méconnaître les opérations de base : Dans un exercice, si on demande d’empiler / dépiler, il faut bien utiliser la terminologie correcte (push/pop ou empiler/dépiler).
  • Penser que toutes les structures s’utilisent de la même manière : Une pile n’autorise l’accès qu’au sommet, alors qu’un tableau permet un accès direct à n’importe quel indice.

🔗 Prolongement : Lien avec UE 15 (Tableaux) & programmation

La leçon UE 15 approfondit les tableaux : déclaration, manipulation des indices, parcours. Les tableaux sont une structure de données fondamentale que vous utiliserez en algorithmique et en langage C. En JavaScript, les tableaux sont également omniprésents (ex : let notes = [12, 15, 08, 19];).

À retenir pour la suite : après avoir compris ce qu’est une structure de données, vous allez implémenter des algorithmes de tri, recherche et manipulation sur les tableaux. La notion de « structure » vous aidera à organiser vos données comme un vrai développeur.
