Leçon 19 : Introduction au JavaScript

Informatique · Première C, D, E (Probatoire) · Module 2 : Programmation en JavaScript

📌 Éléments clés à retenir

1. Qu’est-ce que JavaScript ?

JavaScript (JS) est un langage de programmation côté client qui permet de rendre les pages web interactives et dynamiques. Contrairement au HTML (structure) et CSS (style), JavaScript exécute des actions : calculs, validation de formulaires, animations, réactions aux clics, etc.

  • Rôle principal : dynamiser les pages web (réagir aux événements utilisateur).
  • Exécuté directement par le navigateur (pas besoin de compilation séparée).
  • Peut modifier le contenu HTML/CSS en temps réel.
  • Langage interprété, flexible et orienté objet.
💡 Analogies : HTML = squelette, CSS = peau/vêtements, JavaScript = muscles et mouvements.

2. Intégration de JS dans une page HTML

On utilise la balise <SCRIPT> pour insérer du code JavaScript directement dans une page web. La balise indique au navigateur que le code qu’elle contient est du JavaScript et doit être interprété.

<SCRIPT language="Javascript">
    // Code JavaScript ici
    alert("Bienvenue sur ma page !");
</SCRIPT>
  • On peut placer <SCRIPT> dans <head> ou <body>.
  • Attribut language="Javascript" (historique) ; on utilise souvent simplement <script>.
  • Possibilité de lier un fichier externe : <script src="monScript.js"></script> (bonne pratique).

3. Rôle de la balise <SCRIPT>

La balise <SCRIPT> est l’élément HTML qui délimite une zone de script exécutable. Son rôle est essentiel car :

  • ✅ Elle indique au navigateur d’interpréter le contenu comme du code (JavaScript par défaut).
  • ✅ Elle permet d’interagir avec le DOM (Document Object Model) et les éléments de la page.
  • ✅ Sans <script>, le navigateur afficherait le code comme du texte brut.

4. Premières instructions JavaScript

Voici quelques commandes fondamentales pour débuter en JavaScript :

  • alert("message") : affiche une boîte de dialogue avec un message.
  • console.log("info") : envoie un message dans la console du navigateur (débogage).
  • document.write("texte") : écrit directement dans le flux HTML de la page.
  • Les commentaires : // commentaire sur une ligne ou /* commentaire multi-lignes */.
<script>
    // Ceci est un commentaire
    alert("Bonjour, chers élèves du Cameroun !");
    console.log("Leçon 19 : introduction JS");
    document.write("<h2>JavaScript rend la page vivante</h2>");
</script>

💡 Exemples de compréhension

Exemple 1 : Intégration simple dans une page web

Une page HTML affiche "Bienvenue" et un message d’alerte au chargement.

<!DOCTYPE html>
<html>
<head><title>Ma première page JS</title></head>
<body>
  <h1>Apprentissage JavaScript</h1>
  <script>
    alert("Bienvenue sur ma page !");
    document.write("<p>JavaScript fonctionne parfaitement !</p>");
  </script>
</body>
</html>

Explication : Lorsque la page se charge, le navigateur rencontre la balise <script> et exécute les instructions : affiche une alerte, puis écrit un paragraphe dans la page.

💬 Astuce : L'alerte bloque l'affichage tant que l'utilisateur ne clique pas sur OK. Pour des affichages discrets, préférez console.log() ou document.write().

Exemple 2 : Interaction avec l'utilisateur (prompt)

On veut demander le nom de l'utilisateur et afficher un message personnalisé.

<script>
    let nom = prompt("Quel est ton nom ?");
    document.write("<p>Bonjour, " + nom + " ! Prêt(e) à coder en JS ?</p>");
</script>

Résolution : La fonction prompt() ouvre une boîte de saisie, stocke la valeur dans la variable nom, puis document.write() affiche un message personnalisé.

💡 À retenir : prompt() renvoie toujours une chaîne de caractères (string). Pensez à convertir avec parseInt() ou parseFloat() si besoin de nombres.

Exemple 3 : Différence entre fichier interne et externe

Organisation du code pour un projet propre : séparer le JavaScript dans un fichier script.js.

<!-- Fichier HTML -->
<script src="script.js"></script>
// Contenu du fichier script.js
alert("Ce code provient d'un fichier externe !");
console.log("Bonjour depuis le fichier externe");

Avantages : meilleure organisation, réutilisabilité du code, et possibilité de mise en cache par le navigateur.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier les parenthèses après alert ou prompt → alert seul ne déclenche rien, il faut alert().
  • Confusion : document.write() après le chargement complet de la page peut effacer le contenu existant (si utilisé dans un événement). Préférer d’autres méthodes (manipulation du DOM).
  • Attention : JavaScript est sensible à la casse : alert() fonctionne, Alert() ou ALERT() déclenchent une erreur.
  • Piège : Placer le script avant les éléments HTML qu’il veut modifier sans attendre le chargement (utiliser window.onload ou placer le script en fin de <body>).
  • À ne pas confondre : JavaScript ≠ Java (ce sont deux langages complètement différents).

🖥️ Récapitulatif : Interaction HTML & JavaScript

Cycle de vie d’un script dans une page web

1. Le navigateur charge le fichier HTML.
2. Il rencontre la balise <script> (ou src externe).
3. Le code JavaScript est interprété immédiatement (sauf si attribut defer/async).
4. JavaScript peut lire/modifier la structure HTML (DOM), répondre aux événements, etc.

📌 Exemple visuel conceptuel :
[Fichier HTML] → (balise <script>) → [Moteur JS du navigateur] → Exécution → (affichage dynamique, alertes, etc.)

📝 Mots-clés de la leçon à retenir

  • JavaScript : langage interactif côté client
  • Balise <script> : conteneur de code JS
  • alert() : boîte de dialogue d'information
  • prompt() : boîte de dialogue avec saisie
  • document.write() : écriture dans la page HTML
  • console.log() : débogage dans la console
  • Interprété : exécuté ligne par ligne par le navigateur
  • Événement (aperçu) : onClick, onLoad, etc. (vu en leçon 22)
