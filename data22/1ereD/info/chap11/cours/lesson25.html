📊 Les tableaux en C (Leçon 25)

Informatique · Première (Séries C, D, E) · Algorithmique et programmation

📌 Éléments clés à retenir

🔹 Définition et utilité d’un tableau en C

Un tableau est une structure de données qui permet de stocker plusieurs valeurs de même type dans des cases contiguës en mémoire. Chaque élément est accessible via un indice (ou index). En C, les indices commencent toujours à 0 et vont jusqu’à taille-1.

  • Avantage : éviter de déclarer des centaines de variables séparées (ex: note1, note2, … note100).
  • Déclaration type : type nomTableau[taille]; avec taille fixe (constante entière).
  • Exemple concret : float notes[30]; → 30 notes de type réel.
  • Mémoire : la taille totale = taille × sizeof(type) octets.

🔹 Déclaration et initialisation d’un tableau

On peut déclarer un tableau sans l’initialiser (contenant alors des valeurs aléatoires), ou l’initialiser lors de la déclaration.

// Déclaration simple int ages[10]; // tableau de 10 entiers (indices 0 à 9) // Initialisation avec une liste de valeurs double coefficients[5] = {1.5, 2.0, 3.2, 4.1, 5.0}; // Initialisation partielle : les autres sont mis à 0 int tableau[8] = {0}; // tous les 8 éléments valent 0 // Sans préciser la taille, elle est déduite char reponses[] = {'O', 'N', 'O'}; // taille 3
  • Astuce : int t[100] = {0}; met tous les éléments à zéro.
  • Les chaînes de caractères sont des tableaux de char terminés par '\0' (mais nous nous concentrons sur les tableaux numériques).

🔹 Accès et modification des éléments

On accède à un élément en utilisant son indice entre crochets [ ]. L’indice peut être une constante, une variable ou une expression. En langage C, l’indice du premier élément est 0 (contrairement à l’algorithmique où il commence parfois à 1).

int notes[5] = {12, 15, 10, 18, 14}; // Afficher la première note printf("%d", notes[0]); // affiche 12 // Modifier la troisième note (indice 2) notes[2] = 16; // maintenant le tableau vaut {12,15,16,18,14} // Parcours avec une boucle for for(int i = 0; i < 5; i++) { printf("Note %d : %d\n", i+1, notes[i]); }

⚠️ Piège fréquent : En C, on ne vérifie pas automatiquement les limites du tableau. Accéder à l’indice notes[5] (alors que la taille est 5) provoque un comportement indéfini (risque de crash ou données corrompues).

🔹 Tableaux et algorithmique : traduction du pseudo-code vers C

En algorithmique, on déclare souvent : Tableau Notes[1..N] de réels. En C, cela devient un tableau de taille N avec des indices allant de 0 à N-1. L’élément d’indice algorithmique i correspond à l’indice i-1 en C.

// Algo : Notes : tableau[1..10] d'entiers // En C : int Notes[10]; // indices 0 à 9 Notes[0] = 18; // correspond à Notes[1] en algo Notes[9] = 15; // correspond à Notes[10] en algo
  • Règle de traduction : pour une boucle pour i de 1 à N faire en algo, on écrit en C for(i=0; i<N; i++) et on accède à tab[i] pour l’élément algorithmique tab[i+1].
  • Il est conseillé d’utiliser une constante pour la taille : #define TAILLE 100 ou const int TAILLE = 100; afin de rendre le code plus maintenable.

🔹 Parcours et opérations classiques sur les tableaux

Les boucles for sont idéales pour parcourir tous les éléments d’un tableau. Voici quelques exemples typiques :

  • Calcul de la somme : int somme = 0; for(i=0; i<N; i++) somme += tab[i];
  • Recherche du maximum : initialiser max = tab[0], puis comparer avec chaque élément.
  • Affichage inversé : for(i=N-1; i>=0; i--).
  • Copie d’un tableau vers un autre : for(i=0; i<N; i++) copie[i] = original[i];
// Exemple : calcul de la moyenne des notes #define NB_NOTES 8 float notes[NB_NOTES] = {10.5, 12.0, 15.5, 8.0, 14.0, 16.5, 9.5, 13.0}; float somme = 0.0; for(int i = 0; i < NB_NOTES; i++) { somme = somme + notes[i]; } float moyenne = somme / NB_NOTES; printf("Moyenne de la classe : %.2f\n", moyenne);

💡 Exemples de compréhension

Exemple 1 : Saisie et affichage d’un tableau d’entiers

On souhaite écrire un programme C qui demande à l’utilisateur de saisir 5 entiers, les stocke dans un tableau, puis les affiche à l’envers.

Correction détaillée :

#include int main() { int T[5]; // déclaration du tableau de taille 5 // Saisie des éléments (indices 0 à 4) printf("Saisissez 5 entiers :\n"); for(int i = 0; i < 5; i++) { printf("T[%d] = ", i); scanf("%d", &T[i]); } // Affichage dans l'ordre inverse printf("Affichage inversé : "); for(int i = 4; i >= 0; i--) { printf("%d ", T[i]); } printf("\n"); return 0; }
💬 Astuce : Utilisez toujours une variable constante pour la taille (#define SIZE 5) afin de modifier facilement la taille du tableau sans tout réécrire.

Exemple 2 : Traduction d’un algorithme de tableau en C

Algorithme :
Variables : Notes : tableau[1..4] de réels ; i : entier ; somme : réel
Début
  Pour i de 1 à 4 faire
    Lire(Notes[i])
  FinPour
  Somme ← 0
  Pour i de 1 à 4 faire
    Somme ← Somme + Notes[i]
  FinPour
  Ecrire("La somme est : ", Somme)
Fin

Traduction en C :

#include int main() { float Notes[4]; // indices 0,1,2,3 → correspondent à Notes[1] à Notes[4] en algo int i; float somme = 0.0; // Lecture : i de 0 à 3 en C for(i = 0; i < 4; i++) { printf("Note %d : ", i+1); scanf("%f", &Notes[i]); } // Calcul somme for(i = 0; i < 4; i++) { somme = somme + Notes[i]; } printf("La somme est : %.2f\n", somme); return 0; }
💡 À retenir : l’indice algorithmique k correspond à l’indice C k-1. La boucle pour i de 1 à N faire devient for(i=0; i<N; i++).

Exemple 3 : Recherche de la plus grande valeur dans un tableau

Soit le tableau d’entiers valeurs[7] = {45, 12, 89, 23, 77, 56, 91}. Écrire un programme pour trouver le maximum et sa position (indice).

#include int main() { int valeurs[7] = {45, 12, 89, 23, 77, 56, 91}; int max = valeurs[0]; // on suppose que le premier est le max int position = 0; // indice du maximum for(int i = 1; i < 7; i++) { // on commence à i=1 if(valeurs[i] > max) { max = valeurs[i]; position = i; } } printf("La valeur maximale est %d à l'indice %d (position %d).\n", max, position, position+1); return 0; }
⚙️ Raisonnement : On initialise max avec le premier élément, puis on parcourt le reste. Si on trouve un élément plus grand, on met à jour max et l’indice.

⚠️ Pièges courants / À ne pas confondre

  • ❌ Erreur n°1 : Utiliser l’indice taille au lieu de taille-1 → accès hors limites. Ex : int t[5]; t[5]=10; est interdit (indices valides : 0..4).
  • ❌ Erreur n°2 : Confondre l’indice de départ en algorithmique (souvent 1) et en C (toujours 0). Penser à décaler systématiquement.
  • ❌ Erreur n°3 : Oublier d’initialiser les éléments avant de les lire ou de les utiliser (les valeurs par défaut sont indéterminées).
  • ❌ Erreur n°4 : Utiliser une variable non constante pour la taille du tableau lors de la déclaration (en C classique, la taille doit être une constante connue à la compilation). Pour les tailles dynamiques, on utilise l’allocation mémoire (malloc – vu plus tard).
  • 💡 Bonne pratique : Définir la taille avec #define TAILLE 100 en haut du programme pour éviter les “nombres magiques”.

⚡ Astuce mémoire : L’indice C = indice algorithmique − 1. La boucle commence à 0 et se termine à taille-1.
