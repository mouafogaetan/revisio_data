📥 La requête INSERT INTO

SQL · Ajout de données dans une table · Terminale A (Littéraire)

📌 Éléments clés à retenir

🔹 Rôle de INSERT INTO

La commande INSERT INTO permet d'ajouter une ou plusieurs nouvelles lignes (enregistrements) dans une table d'une base de données. C'est une instruction du langage SQL (Structured Query Language) utilisée pour insérer des données.

  • On l'utilise après avoir créé la structure de la table (grâce à CREATE TABLE).
  • Chaque insertion correspond à un nouvel enregistrement (ligne).
  • Les valeurs doivent respecter le type de chaque colonne (texte, nombre, date...).

🔹 Syntaxe de base (INSERT INTO)

La syntaxe la plus courante en examen : 

INSERT INTO NomTable (colonne1, colonne2, colonne3)
VALUES (valeur1, valeur2, valeur3);
  • NomTable : le nom de la table cible.
  • (colonne1, colonne2, …) : liste des colonnes dans lesquelles on souhaite insérer des données (ordre important).
  • VALUES : suivi des valeurs correspondantes, dans le même ordre que les colonnes.
  • Si on insère des valeurs pour toutes les colonnes (dans l'ordre de la table), on peut omettre la liste des colonnes :
    INSERT INTO Livres VALUES (5, 'Le manuscrit', 'A. Diallo', 2025);
💡 Bon réflexe : Toujours préciser les noms des colonnes pour éviter les erreurs si l’ordre de la table change.

🔹 Correspondance avec la table

Chaque colonne mentionnée reçoit une valeur. Les types doivent être respectés :

  • Texte (VARCHAR, TEXT) → entouré de guillemets simples 'exemple'.
  • Numérique (INT, DECIMAL) → valeur sans guillemets.
  • Date (DATE) → '2025-11-20' (format ISO).

💡 Exemples de compréhension (SQL INSERT)

📖 Exemple 1 : Bibliothèque d’un club de lecture

Table Ouvrages avec les colonnes : id_livre (INT, clé primaire), titre (TEXT), auteur (TEXT), annee_publication (INT). On souhaite ajouter un nouveau livre : "L’Afrique des idées" écrit par "Mboua S." publié en 2022.
✅ Requête SQL : 
INSERT INTO Ouvrages (id_livre, titre, auteur, annee_publication)
VALUES (14, 'L''Afrique des idées', 'Mboua S.', 2022);

Explication : Les valeurs sont dans le même ordre que les colonnes listées. Le titre contient une apostrophe → on la double 'L''Afrique' pour l’échapper.

💬 Astuce : Pour insérer un texte contenant une apostrophe, écrivez deux apostrophes consécutives (' ') ou utilisez des guillemets doubles selon le SGBD. En examen, on utilise la méthode simple avec double apostrophe.

📊 Exemple 2 : Ajout d'un adhérent dans une association

Table Adherents (id_adherent INT, nom VARCHAR(30), prenom VARCHAR(30), date_inscription DATE). Ajouter un membre : id=25, nom="TCHINDA", prenom="Gaëlle", date d’inscription = 6 avril 2026.
✅ Requête : 
INSERT INTO Adherents (id_adherent, nom, prenom, date_inscription)
VALUES (25, 'TCHINDA', 'Gaëlle', '2026-04-06');

La date est entourée de guillemets simples au format AAAA-MM-JJ.

📌 Pensez à respecter l'ordre des colonnes indiqué après le nom de la table. L'ordre des VALUES doit correspondre exactement.

🖥️ Exemple 3 : Insertion sans lister toutes les colonnes (si on connaît l'ordre)

Table Genres définie comme : id_genre INT, libelle VARCHAR(40). On souhaite ajouter un nouveau genre "Poésie".
✅ Insertion explicite (recommandée) : 
INSERT INTO Genres (id_genre, libelle) VALUES (8, 'Poésie');
✅ Insertion implicite (si toutes les colonnes sont fournies dans l'ordre) : 
INSERT INTO Genres VALUES (9, 'Théâtre');

⚠️ L’insertion implicite est plus risquée : si la structure de la table change (ajout d’une colonne), la requête échoue. Mieux vaut toujours préciser les colonnes.

🎯 Lien avec l'examen (Baccalauréat)

D’après les épreuves des sessions précédentes (2024 notamment – Partie III, question 3), l’écriture d’une requête INSERT INTO est exigible. On peut vous demander :

  • D’écrire la commande SQL permettant d’ajouter un enregistrement dans une table donnée.
  • De compléter une requête partielle (ex: INSERT INTO … VALUES (…);).
  • De comprendre l’utilité de l’insertion pour alimenter une base de données (gestion de bibliothèque, notes d’élèves, adhérents).
✅ Conseil examen : Relisez toujours la table et ses colonnes. N’oubliez pas les guillemets simples pour les chaînes de caractères et la virgule après chaque valeur.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier les guillemets simples autour des valeurs textuelles → INSERT INTO Eleves (nom) VALUES (DIOP); ❌ (manque les quotes) → 'DIOP' ✅.
  • Contrainte de clé primaire : On ne peut pas insérer deux fois la même valeur pour la clé primaire (id unique). Vérifier que l'identifiant n'existe pas déjà.
  • Ordre des colonnes : Ne pas inverser les valeurs. Exemple : INSERT INTO Livres (titre, auteur) VALUES ('Victor Hugo', 'Les Misérables'); → le titre devient 'Victor Hugo' et l'auteur 'Les Misérables', incohérent !
  • Valeurs manquantes : Si une colonne est définie comme NOT NULL (obligatoire) et qu’on ne lui fournit pas de valeur, l’insertion échoue.
  • Confusion avec UPDATE : INSERT ajoute une nouvelle ligne, UPDATE modifie des lignes existantes. À ne pas mélanger.

Exercice type bac

Soit la table Utilisateurs (id_utilisateur INT PRIMARY KEY, pseudo VARCHAR(30), email VARCHAR(50), date_creation DATE). Écrire la requête SQL permettant d’ajouter un utilisateur avec :
  • id_utilisateur = 101
  • pseudo = "Lectrice2026"
  • email = "lectrice@clublecture.cm"
  • date_creation = 15 mai 2026
🔍 Voir la correction 
INSERT INTO Utilisateurs (id_utilisateur, pseudo, email, date_creation)
VALUES (101, 'Lectrice2026', 'lectrice@clublecture.cm', '2026-05-15');
💡 La date au format ISO (AAAA-MM-JJ) est reconnue par tous les SGBD.

📋 En résumé : structure mentale

INSERT INTO nom_table (colonneA, colonneB, colonneC)
VALUES (valeurA, valeurB, valeurC);

✔ Chaque nouvelle ligne = un enregistrement supplémentaire.
✔ Les colonnes non listées reçoivent leur valeur par défaut (ou NULL si autorisé).
✔ Toujours tester l’ordre et les types de données.
