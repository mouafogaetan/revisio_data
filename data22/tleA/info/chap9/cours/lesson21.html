📌 Leçon 21 : La requête SELECT (SQL)

Bases de données · Terminale A (Littéraire) · Baccalauréat 2026

🔑 Éléments clés à retenir

🗄️ Qu’est-ce que la requête SELECT ?

La commande SELECT est l’instruction la plus utilisée en SQL. Elle permet d’interroger une base de données et d’extraire des informations contenues dans une ou plusieurs tables. Avec SELECT, on peut afficher toutes les colonnes d’une table ou seulement quelques-unes, et même filtrer les lignes grâce à la clause WHERE.

  • Syntaxe de base : SELECT colonne1, colonne2 FROM nom_table;
  • Afficher toutes les colonnes : SELECT * FROM nom_table; (l’astérisque signifie « toutes les colonnes »).
  • Avec condition : SELECT colonnes FROM table WHERE condition;
  • À retenir : SQL n’est pas sensible à la casse pour les mots-clés (SELECT = select), mais on écrit souvent en majuscules par convention.

🎯 La clause WHERE : filtrer les lignes

WHERE permet de sélectionner uniquement les enregistrements qui satisfont une condition (comme un filtre). On peut utiliser des opérateurs de comparaison : = , > , < , >= , <= , <> (différent de), et des opérateurs logiques : AND, OR, NOT.

  • Exemple : SELECT nom, prenom FROM eleves WHERE classe = 'Terminale A';
  • Filtrer avec AND : SELECT * FROM livres WHERE prix > 1000 AND genre = 'Roman';
  • Filtrer avec OR : SELECT * FROM clients WHERE ville = 'Douala' OR ville = 'Yaoundé';
💡 Piège fréquent : Ne pas confondre le signe = (comparaison) avec l’affectation. En SQL, = sert à tester l’égalité.

📝 Règles d’écriture en SQL (examen)

Au baccalauréat, on attend une syntaxe propre et correcte. Les noms de colonnes et de tables ne doivent pas contenir d’espaces (ou alors entre crochets ou guillemets, mais évitez). Les chaînes de caractères (texte) s’écrivent entre guillemets simples.

  • 🔹 SELECT : liste des colonnes séparées par des virgules.
  • 🔹 FROM : nom de la table source.
  • 🔹 WHERE : condition de filtrage (optionnel mais très utilisé).
  • 🔹 Terminer chaque requête par un point-virgule ; (bonne pratique).

💡 Exemples types du Baccalauréat

📖 Exemple 1 : Table "Élèves"

On dispose d’une table nommée eleves contenant les colonnes : id_eleve, nom, prenom, classe, moyenne. Question type : « Écrire la requête SQL qui affiche les noms et prénoms des élèves de la classe "Terminale A". »

SELECT nom, prenom
FROM eleves
WHERE classe = 'Terminale A';

Explication : On sélectionne uniquement les colonnes nom et prenom (pas besoin de l’identifiant ou de la moyenne). La condition WHERE classe = 'Terminale A' filtre les lignes où la classe correspond exactement à la valeur recherchée.
✅ Astuce : Vérifiez que la chaîne de caractères est entre quotes simples.

💬 À retenir : Si on veut toutes les colonnes d’une table pour une condition, on écrit SELECT * FROM eleves WHERE classe = 'Terminale A';

📚 Exemple 2 : Filtre avec comparaison numérique

Table livres : titre, auteur, prix, stock. Écrire la requête qui affiche le titre et le prix des livres dont le prix est supérieur à 5000 FCFA.

SELECT titre, prix
FROM livres
WHERE prix > 5000;

Résolution : L’opérateur > filtre les prix strictement supérieurs à 5000. Pas de guillemets pour les nombres.

🎓 Exemple 3 : Condition composée (AND)

Table employes : nom, service, salaire. Trouvez les employés du service « Comptabilité » dont le salaire est supérieur ou égal à 150 000 FCFA.

SELECT nom, salaire
FROM employes
WHERE service = 'Comptabilité' AND salaire >= 150000;
⚠️ Ne pas oublier que les deux conditions doivent être vraies simultanément. Si on avait utilisé OR, on aurait obtenu tous les employés de la compta ou tous ceux gagnant plus de 150k (même d’autres services).

⚠️ Pièges courants & Conseils bac

  • Erreur fréquente : Oublier les guillemets simples autour des chaînes de caractères dans le WHERE. Exemple : WHERE ville = Douala → incorrect, il faut WHERE ville = 'Douala'.
  • Attention : Confondre = (égalité) avec == (non valide en SQL standard). Toujours un seul = pour la comparaison.
  • Astuce examen : Lisez bien l’énoncé : parfois on demande « toutes les colonnes » → utilisez * ; parfois seulement certaines colonnes → listez-les.
  • Ne pas confondre : SELECT affiche des données, ne modifie rien. Pour ajouter/modifier/supprimer, on utilisera INSERT, UPDATE ou DELETE (autres leçons).
  • Conseil rédaction : Écrivez chaque mot-clé sur une ligne différente pour plus de clarté, et terminez par un point-virgule. Les correcteurs apprécient la lisibilité.

✍️ Mini-exercice d’entraînement

Testez-vous !

Une table films contient : titre, realisateur, annee_sortie, note (sur 10). Écrivez la requête SQL qui affiche le titre et le réalisateur des films dont la note est supérieure à 7 et sortis après 2010.

Voir la réponse
SELECT titre, realisateur
FROM films
WHERE note > 7 AND annee_sortie > 2010;

Explication : On filtre avec deux conditions : note supérieure à 7 ET année de sortie postérieure à 2010.
