🔑 Leçon 20 : Identification de la clé primaire

Informatique - Terminale A (Littéraire) | Base de données | Examen 2023-2025

📌 Qu'est-ce qu'une clé primaire ?

🔹 Définition & rôle À retenir absolument

Une clé primaire (ou primary key) est un champ (ou une combinaison de champs) dans une table de base de données qui permet d’identifier de manière unique chaque enregistrement (chaque ligne). Aucune ligne ne peut avoir la même valeur pour la clé primaire, et celle-ci ne peut jamais être vide (NULL).

  • Unicité : Deux enregistrements différents ont obligatoirement des valeurs distinctes pour la clé primaire.
  • Non nullité : Chaque enregistrement doit obligatoirement renseigner la clé primaire.
  • Stabilité : Idéalement, la valeur d'une clé primaire ne change jamais dans le temps (ex : numéro d'étudiant, code produit).
  • Dans une table, une seule clé primaire peut exister (mais elle peut être composée de plusieurs colonnes → clé primaire composite).
💡 Analogies pratiques : La clé primaire est comme le numéro de CNI d'une personne – unique pour chaque citoyen, jamais partagé. Ou encore le numéro de matricule d’un élève dans un lycée.

🔹 Pourquoi identifier la clé primaire ?

Identifier la clé primaire est essentiel pour :

  • 🔎 Retrouver rapidement une information spécifique (ex: la ligne d’un client grâce à son ID).
  • 🔗 Créer des relations entre plusieurs tables (clé étrangère fait référence à une clé primaire d’une autre table).
  • 🧹 Éviter les doublons dans la base de données.
  • 📌 Garantir l'intégrité des données : on ne peut pas avoir deux fois le même identifiant.

Lors des examens du Baccalauréat (sessions 2023, 2024), on demande souvent d'identifier le champ qui constitue la clé primaire dans une table donnée.

🧠 Comment repérer la clé primaire ?

✔️ Règles d'identification

Pour trouver la clé primaire dans une table, posez-vous ces questions :

  • 🔹 Y a-t-il un champ dont les valeurs sont toutes différentes (aucun doublon) ?
  • 🔹 Est-ce que ce champ est obligatoire (jamais vide) pour chaque ligne ?
  • 🔹 Est-ce que ce champ sert généralement d’identifiant unique (code, ID, numéro, matricule, référence) ?
  • 🔹 Dans les exercices, le concepteur souligne souvent ou nomme le champ "id", "code", "numéro" pour signaler la clé primaire.
💬 Astuce examen : Quand on vous donne la description d'une table, cherchez la colonne qui pourrait servir de « carte d’identité » pour chaque ligne. Si deux élèves peuvent avoir le même nom ou la même classe, seul le numéro d’inscription ou l’identifiant est unique.

📊 Exemple 1 : Table "Élèves"

Voici une table contenant des informations sur des élèves.

id_élèvenomprénomclassedate_naissance
101MbargaLéaTle A2006-05-12
102NgombePaulTle A2005-11-23
103MbargaJeanTle A2006-08-01
104DjoumessiSarahTle A2006-02-17

Analyse : Le champ id_élève contient des valeurs toutes différentes (101,102,103,104). Aucun doublon. Le nom "Mbarga" apparaît deux fois (Léa et Jean) → nom ne peut pas être clé primaire. La classe est identique pour tous → non unique. Donc la clé primaire est id_élève.

✨ À l'examen, on peut aussi vous montrer une table sans préciser la clé ; il faut justifier votre choix en expliquant l'unicité.

📊 Exemple 2 : Table "Livres" (Bibliothèque)

Une table qui référence des ouvrages.

ISBNTitreAuteurAnnéeÉditeur
978-2-123456-01-0Le Cœur des LittératuresE. Ngoa2022Presses Universitaires
978-2-123456-02-7Poésie camerounaiseF. Mboua2021Éditions CLE
978-2-123456-03-4Informatique pour littérairesJ. Tchinda2023Savoir numérique

Identification : L'ISBN (International Standard Book Number) est unique pour chaque livre. Même si le titre ou l’auteur peuvent se ressembler, l’ISBN ne se répète jamais. Donc ISBN est la clé primaire.

💡 Cas particulier : Parfois la clé primaire n'est pas évidente. Par exemple, dans une table "Inscriptions" avec les champs (id_élève, id_cours, date), la clé primaire peut être la combinaison (id_élève, id_cours) si un élève ne peut pas s’inscrire deux fois au même cours. Mais en Terminale A, on reste souvent sur une clé simple (un seul champ).

📝 Exercices fréquents au Baccalauréat

🎓 Exercice inspiré session 2023-2024

On donne la table suivante "CLIENTS" :
(NumClient, Nom, Adresse, Ville, Téléphone)

Question : Quel champ peut servir de clé primaire ? Justifiez.
Réponse type : Le champ NumClient car il identifie chaque client de façon unique (un numéro différent par client). Les autres champs (Nom, Adresse, Ville, Téléphone) peuvent comporter des doublons ou des valeurs vides.

⚠️ Attention : Le téléphone pourrait sembler unique, mais théoriquement deux clients pourraient partager le même numéro (ligne fixe partagée) ; donc on préfère un identifiant artificiel comme NumClient.

📌 Table "Employés" d'une entreprise

Matricule | Nom | Prénom | Service | Email

La clé primaire est Matricule car chaque employé reçoit un matricule unique (ex: E001, E002). L’email peut aussi être unique mais ce n’est pas toujours garanti (un employé peut ne pas avoir d’email professionnel). L’énoncé officiel considère généralement le matricule comme clé primaire.

🖼️ Illustration : Clé primaire dans une table

Image conceptuelle : la colonne "ID_Client" (clé primaire) ne contient aucun doublon et sert de référence unique pour chaque ligne.

⚠️ Pièges courants & erreurs à éviter

  • ❌ Erreur n°1 : Choisir un champ comme "nom" ou "prénom" comme clé primaire. Deux personnes peuvent porter le même nom → pas d'unicité garantie.
  • ❌ Erreur n°2 : Croire qu'une clé primaire peut être vide (NULL). C'est interdit ! Chaque enregistrement doit avoir une valeur.
  • ❌ Erreur n°3 : Confondre clé primaire et clé étrangère. La clé primaire est dans la table courante, la clé étrangère référence une clé primaire d’une autre table.
  • ⚠️ Attention examen : Lisez bien l'énoncé. Parfois la clé primaire est déjà indiquée (soulignée ou précisée). Si ce n'est pas le cas, cherchez la colonne avec des valeurs uniques.
  • 📌 Piège : Une table sans clé primaire explicite peut avoir une combinaison de champs qui rend chaque ligne unique (ex: (Date, Numéro_commande)). C'est une clé primaire composite, mais au programme de terminale on se concentre sur les clés simples.

📌 Mémo visuel pour l'examen

🔖 Comment rédiger une réponse type ?

Question : Identifiez la clé primaire de la table « Produits » (RefProd, Désignation, Prix, Quantité).
Réponse modèle : « La clé primaire est le champ RefProd car il permet d’identifier de manière unique chaque produit. Chaque référence produit est différente des autres et ne peut pas être nulle. »

✔️ Mots-clés à utiliser : "unique", "identifier chaque enregistrement", "pas de doublon", "obligatoire".
