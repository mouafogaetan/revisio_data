📖 Exercice 1 : Questions de cours (5 pts)
❶ / 1 pt
Définir les termes suivants : algorithme et variable.

✅ Réponse type

• Algorithme : suite ordonnée et finie d'instructions permettant de résoudre un problème.
• Variable : zone mémoire nommée pouvant contenir une donnée qui peut évoluer au cours de l'exécution d'un algorithme.

❷ / 2 pts
Citer les trois structures de contrôle fondamentales en algorithmique. Donner un exemple de mot-clé pour chacune.

✅ Corrigé

1. Structure séquentielle (instructions à la suite).
2. Structure alternative / conditionnelle : si... alors... sinon...
3. Structure répétitive / itérative : pour... faire... , tant que... faire...

❸ / 2 pts
Expliquer la différence entre une structure alternative et une structure répétitive. Illustrer par un cas concret (en français).

✅ Corrigé

• Alternative : exécute un bloc d'instructions selon qu'une condition est vraie ou fausse. Ex: "Si la moyenne ≥ 10 alors afficher 'admis' sinon 'non admis'".
• Répétitive : répète un bloc tant qu'une condition est vraie ou un nombre défini de fois. Ex: "Répéter 10 fois l'affichage de 'Bienvenue'".

⚙️ Exercice 2 : Algorithmique – Trace & organigramme (6 pts)
❶ / 3 pts
On considère l'algorithme suivant :
Variables : A, B, C : entiers
Début
   Lire A, B
   C ← A + B
   A ← B
   B ← C
   Afficher A, B
Fin
Faire la trace d'exécution pour A = 4 et B = 7. Qu'affiche l'algorithme ?

✅ Trace détaillée

Initial : A=4 , B=7
C ← 4+7 = 11
A ← 7 (nouvelle valeur)
B ← 11
Affichage : A = 7 , B = 11.

🌐 Exercice 3 : Programmation HTML (5 pts)
❶ / 1,5 pt
Donner la différence entre les balises <ul> et <ol> en HTML. Citer la balise utilisée pour chaque élément d'une liste.

✅ Bonne réponse

• <ul> = liste non ordonnée (à puces).
• <ol> = liste ordonnée (numérotée).
• Dans les deux cas, chaque élément est défini par la balise <li> (list item).

❷ / 2 pts
Compléter le code HTML suivant pour créer un formulaire simple avec un champ "Nom" (texte) et un champ "Email" (email) et un bouton d'envoi.
<form action="/traitement" method="post">
   <label>Nom : <input type="______" name="nom"></label><br>
   <label>Email : <input type="______" name="email"></label><br>
   <input type="______" value="Envoyer">
</form>

✅ Complétion correcte

<form action="/traitement" method="post">
   <label>Nom : <input type="text" name="nom"></label><br>
   <label>Email : <input type="email" name="email"></label><br>
   <input type="submit" value="Envoyer">
</form>

Remarque : type="submit" permet d'envoyer le formulaire.

❸ / 1,5 pt
Citez 3 balises HTML essentielles pour la structure d'une page web (hors <html>). Donnez leur rôle.

✅ Exemple de réponse

• <head> : contient les métadonnées, le titre et les liens CSS.
• <title> : définit le titre affiché dans l'onglet du navigateur.
• <body> : contient tout le contenu visible de la page (textes, images, liens).

🖼️ Exercice 4 : Infographie & Bonnes pratiques (4 pts)
❶ / 2 pts
Un élève souhaite utiliser une image trouvée sur Internet pour illustrer son site web. Quelles précautions doit-il prendre ? Donner au moins deux conseils liés aux droits d'usage et à la sécurité.

✅ Éléments de réponse

• Vérifier la licence (Creative Commons, domaine public) ou utiliser des banques d'images libres.
• Éviter d'exposer des données personnelles via les métadonnées des images.
• Télécharger depuis des sources fiables pour éviter les fichiers infectés.

❷ / 2 pts
Distinguer image matricielle (bitmap) et image vectorielle. Citer un format pour chaque type.

✅ Corrigé

• Matricielle (bitmap) : constituée de pixels ; perte de qualité en agrandissant. Exemple : JPEG, PNG.
• Vectorielle : constituée d'objets géométriques (courbes, formes) ; redimensionnable sans perte. Exemple : SVG.
