UE 13 : Les structures conditionnelles

Informatique · Première C, D, E (Probatoire) · Module 2 : Algorithmique et programmation

📌 Éléments clés à retenir

🔍 Définition : structure conditionnelle

Une structure conditionnelle (ou alternative) permet d’exécuter certaines instructions seulement si une condition est vraie (ou fausse). Elle introduit des choix dans l’exécution d’un algorithme ou d’un programme.

  • La condition est une expression logique (vraie ou fausse).
  • Les opérateurs de comparaison courants : = (égal), != (différent), <, >, <=, >=.
  • On peut combiner plusieurs conditions avec ET (&& en C) et OU (|| en C).

📐 Structure SI... ALORS... SINON (IF... ELSE)

La forme la plus utilisée en algorithmique et en programmation. Elle comporte obligatoirement un test et deux blocs possibles.

Syntaxe algorithmique :

Si (condition) Alors
    // instructions exécutées si condition vraie
Sinon
    // instructions exécutées si condition fausse
FinSi

Syntaxe en langage C :

if (condition) {
    // instructions si vrai
} else {
    // instructions si faux
}
  • Le bloc Sinon est optionnel : on peut écrire un Si sans Sinon.
  • On peut imbriquer des conditions les unes dans les autres.

🔀 Structure SELON... CAS (SWITCH)

Utile lorsqu’une variable peut prendre plusieurs valeurs précises et que l’on souhaite exécuter un bloc différent pour chaque valeur.

Exemple en algorithmique :

Selon jour_fait
    Cas 1: Ecrire("Lundi")
    Cas 2: Ecrire("Mardi")
    Cas 3: Ecrire("Mercredi")
    Sinon: Ecrire("Jour inconnu")
FinSelon

Exemple en C :

switch(jour) {
    case 1: printf("Lundi"); break;
    case 2: printf("Mardi"); break;
    case 3: printf("Mercredi"); break;
    default: printf("Jour inconnu");
}
💡 À retenir : Le break en C évite l’exécution des cas suivants. Le bloc default joue le rôle du Sinon.

🧠 Conditions complexes et opérateurs logiques

Parfois une condition dépend de plusieurs tests. On utilise les opérateurs logiques :

  • ET (&&) : vrai si toutes les sous-conditions sont vraies.
  • OU (||) : vrai si au moins une sous-condition est vraie.
  • NON (!) : inverse la valeur de vérité.

Exemple : « Si l’âge est compris entre 18 et 25 ans » → if (age >= 18 && age <= 25)

💡 Exemples de compréhension

Exemple 1 : Vérification de la parité d’un nombre

Écrire un algorithme et un programme en C qui lit un entier N et affiche « Pair » s’il est pair, sinon « Impair ».

Résolution : Un nombre est pair si son reste de division par 2 est 0. 

// Algorithme
Variable N : entier
Debut
    Ecrire("Donner un entier :")
    Lire(N)
    Si (N % 2 == 0) Alors
        Ecrire("Pair")
    Sinon
        Ecrire("Impair")
    FinSi
Fin
// Programme en C
#include <stdio.h>
int main() {
    int N;
    printf("Donner un entier : ");
    scanf("%d", &N);
    if (N % 2 == 0) {
        printf("Pair\n");
    } else {
        printf("Impair\n");
    }
    return 0;
}
💬 Astuce : L’opérateur modulo % donne le reste de la division entière. Très utile pour tester la divisibilité.

Exemple 2 : Structure imbriquée (notes et appréciation)

Un professeur souhaite afficher une mention selon la note d’un élève : note ≥ 16 → « Très bien » ; 14 ≤ note < 16 → « Bien » ; 12 ≤ note < 14 → « Assez bien » ; note < 12 → « Passable ».

Correction avec SI imbriqués : 

Si (note >= 16) Alors
    Ecrire("Très bien")
Sinon
    Si (note >= 14) Alors
        Ecrire("Bien")
    Sinon
        Si (note >= 12) Alors
            Ecrire("Assez bien")
        Sinon
            Ecrire("Passable")
        FinSi
    FinSi
FinSi

Alternative avec else if en C :

if (note >= 16) 
    printf("Très bien");
else if (note >= 14)
    printf("Bien");
else if (note >= 12)
    printf("Assez bien");
else
    printf("Passable");
📌 L’ordre des tests est important : on vérifie d’abord les conditions les plus restrictives.

Exemple 3 : Structure SELON / SWITCH

Afficher le nom du mois en fonction d’un numéro (1 = Janvier, 2 = Février, … 12 = Décembre).

Implémentation en C : 

#include <stdio.h>
int main() {
    int mois;
    printf("Entrez le numéro du mois (1-12) : ");
    scanf("%d", &mois);
    switch(mois) {
        case 1: printf("Janvier"); break;
        case 2: printf("Février"); break;
        case 3: printf("Mars"); break;
        case 4: printf("Avril"); break;
        case 5: printf("Mai"); break;
        case 6: printf("Juin"); break;
        case 7: printf("Juillet"); break;
        case 8: printf("Août"); break;
        case 9: printf("Septembre"); break;
        case 10: printf("Octobre"); break;
        case 11: printf("Novembre"); break;
        case 12: printf("Décembre"); break;
        default: printf("Numéro invalide");
    }
    return 0;
}
✅ Quand le nombre de cas est grand et que la variable est discrète, le switch rend le code plus lisible.

📊 Comparaison : SI/SINON vs SELON/CAS

SI... ALORS... SINON (IF) SELON... CAS (SWITCH)
Convient pour des conditions complexes (intervalles, combinaisons logiques). Utilisé uniquement pour tester l’égalité d’une variable avec des valeurs constantes.
Peut évaluer tout type d’expression booléenne. Ne fonctionne qu’avec des types entiers ou caractères (en C).
Permet des embranchements imbriqués. Structure plus claire pour de nombreux cas fixes (menu, code de commande).

⚠️ Pièges courants / À ne pas confondre

  • Confusion entre affectation et comparaison : En C, if (x = 5) est une affectation (toujours vraie) au lieu de if (x == 5). En algorithmique, on utilise = pour la comparaison, mais attention au passage au code.
  • Oublier le « Sinon » : Parfois on omet volontairement le sinon, mais il faut s’assurer que l’absence d’action par défaut est voulue.
  • Ne pas mettre d’accolades en C (blocs) : Si après un if on écrit deux instructions sans accolades, seule la première est conditionnelle.
  • Erreur fréquente : Utiliser = au lieu de == en C pour la comparaison.
  • Dans SWITCH : Oublier le break provoque l’exécution des cas suivants (effet "fall-through").

À toi de jouer !

Un magasin accorde une remise selon le montant total des achats :
- Moins de 20 000 F → pas de remise.
- Entre 20 000 et 50 000 F → remise de 5%.
- Plus de 50 000 F → remise de 10%.
Écrire l’algorithme (structure conditionnelle) qui lit le montant total et affiche le montant à payer après remise.

📖 Voir une proposition de correction 
Algorithme CalculRemise
Variables montant, remise, net : réels
Debut
    Ecrire("Entrez le montant total : ")
    Lire(montant)
    Si (montant < 20000) Alors
        net ← montant
    Sinon
        Si (montant <= 50000) Alors
            net ← montant - montant * 0.05
        Sinon
            net ← montant - montant * 0.10
        FinSi
    FinSi
    Ecrire("Montant à payer : ", net)
Fin
