📟 Invite de commandes : principes fondamentaux

Informatique · Première (Séries C, D, E) · Unité d’apprentissage 3

📌 Éléments clés à retenir

🔍 Qu’est-ce que l’invite de commandes ?

L’invite de commandes (ou Command Prompt sous Windows, terminal sous Linux/macOS) est une interface textuelle qui permet d’interagir avec le système d’exploitation en tapant des instructions. Contrairement à l’interface graphique, chaque action est déclenchée par une commande texte.

  • Rôle : exécuter des programmes, gérer les fichiers et dossiers, automatiser des tâches.
  • Accès sous Windows : `Win + R` → taper cmd → Entrée.
  • Principe : chaque commande suit une syntaxe précise (commande + options + arguments).
💻 Exemple visuel :
C:\Users\Eleve> dir
C:\Users\Eleve> cd Documents

Le prompt C:\Users\Eleve> indique le répertoire courant ; après la commande, le système exécute l’ordre.

⌨️ Commandes internes et externes

Les commandes de l’invite de commandes se divisent en deux familles :

  • Internes : intégrées directement à l’interpréteur (DIR, CD, COPY, DEL…). Disponibles sans fichier additionnel.
  • Externes : programmes indépendants (ex: format, diskpart, ping). Nécessitent un fichier exécutable.

📌 À retenir : l’invite de commandes est sensible à la casse ? Non, sous Windows les commandes s’écrivent généralement en majuscules ou minuscules indifféremment, mais par convention on utilise les MAJUSCULES.

📐 Syntaxe des commandes

Toute commande respecte une structure standard :

COMMANDE [options] [arguments]
  • Commande : action à réaliser (DIR, COPY, MD…).
  • Options (ou commutateurs) : modifient le comportement, souvent préfixées par / sous Windows (ex: DIR /W pour affichage large).
  • Arguments : cibles de la commande (noms de fichiers, chemins). Exemple : DEL note.txt → l’argument est note.txt.
💡 Aide immédiate : tapez COMMANDE /? pour afficher l’aide détaillée (ex: DIR /?).

🛠️ Commandes DOS de base (à maîtriser)

📂 DIR – Afficher le contenu d’un dossier

La commande DIR liste tous les fichiers et sous-répertoires du répertoire courant.

C:\> DIR
C:\> DIR /W (affichage en largeur)
C:\> DIR /P (affichage page par page)

➕ Astuce examen : DIR /? affiche toutes les options disponibles.

🚪 CD – Changer de répertoire (Change Directory)

Permet de naviguer dans l’arborescence des dossiers.

  • CD \ → aller à la racine du disque.
  • CD .. → remonter d’un niveau (dossier parent).
  • CD Documents → entrer dans le dossier Documents.

📁 MD / MKDIR – Créer un répertoire

MD (Make Directory) crée un ou plusieurs dossiers.

C:\Eleves> MD COURS
C:\Eleves> MKDIR Exercices_JS

✅ Exemple : MD PROJET C++ (créera le dossier "PROJET C++").

🗑️ RD / RMDIR – Supprimer un répertoire

Supprime un dossier (attention : le dossier doit être vide, sauf si on utilise /S).

C:\> RD Mathématiques
C:\> RMDIR /S DossierNonVide (supprime le dossier et son contenu)
⚠️ Piège fréquent : impossible de supprimer le répertoire courant ou un dossier contenant des fichiers sans l’option /S.

✏️ REN – Renommer fichier ou dossier

Change le nom d’un élément sans le déplacer.

C:\> REN Mathématiques Informatique
C:\> REN ancien.txt nouveau.txt

❌ DEL – Supprimer des fichiers

Efface un ou plusieurs fichiers (attention : suppression définitive, pas de corbeille).

C:\> DEL note.txt
C:\> DEL *.tmp (supprime tous les fichiers .tmp)

📋 COPY – Copier des fichiers

Duplique un fichier d’un emplacement vers un autre.

C:\> COPY rapport.docx D:\BACKUP\
C:\> COPY devoirs.txt copie_devoirs.txt

📄 TYPE – Afficher le contenu d’un fichier texte

Permet de visualiser rapidement le texte d’un fichier (sans l’ouvrir dans un éditeur).

C:\> TYPE config.ini
C:\> TYPE resultat.txt

💡 Utilitaire pour vérifier le contenu de scripts ou de fichiers de configuration.

💡 Exemples de compréhension

Exemple 1 : Organisation de dossiers pour un projet

Situation : Tu es dans le répertoire C:\Utilisateurs\Paul. Tu souhaites créer une arborescence pour ranger tes travaux d’informatique : un dossier principal “PROJET_C” contenant deux sous-dossiers “src” et “includes”.

Résolution pas à pas :
1️⃣ Créer le dossier principal : MD PROJET_C
2️⃣ Accéder à ce dossier : CD PROJET_C
3️⃣ Créer les sous-dossiers : MD src et MD includes
4️⃣ Vérifier la structure avec DIR → on voit src et includes.
✅ Résultat : arborescence prête à recevoir les fichiers.

🧠 Astuce : Utiliser MD PROJET_C\src PROJET_C\includes en une ligne (avec chemins complets).

Exemple 2 : Nettoyage et renommage rapide

Dans le répertoire C:\Epreuve, il y a deux fichiers : ancien.txt et a_supprimer.doc. On souhaite renommer ancien.txt en final.txt et supprimer a_supprimer.doc.

Commandes à taper :
- REN ancien.txt final.txt
- DEL a_supprimer.doc
Pour vérifier : DIR affiche uniquement final.txt.

Exemple 3 : Utiliser TYPE et COPY pour fusionner

On dispose de deux fichiers texte : intro.txt et conclusion.txt. On veut consulter le contenu du premier, puis copier les deux fichiers dans un nouveau fichier nommé complet.txt.

Procédure :
1️⃣ TYPE intro.txt → affiche le texte.
2️⃣ COPY intro.txt+conclusion.txt complet.txt → concatène les deux fichiers en un seul.
🔍 Vérification : TYPE complet.txt pour voir le résultat.

💬 À savoir : La concaténation avec + est très utile pour assembler des logs ou des rapports.

⚠️ Pièges courants / À ne pas confondre

  • Confondre RD et DEL : DEL supprime des fichiers, RD supprime des dossiers vides. Pour supprimer un dossier avec son contenu → RD /S dossier.
  • Oublier l’aide intégrée : beaucoup d’élèves ignorent DIR /? ou COPY /?. Utilisez-la en examen !
  • Espace mal placé : dans MD Mon Dossier → crée deux dossiers “Mon” et “Dossier”. Pour un nom avec espace, utiliser les guillemets : MD "Mon Dossier".
  • Se tromper de chemin avec CD : CD .. (deux points) monte d’un niveau ; CD . (un point) ne fait rien.
  • Suppression irréversible : DEL et RD n’envoient pas à la corbeille ⇒ réflexe de vérification avec DIR avant suppression.
