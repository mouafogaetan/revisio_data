📘 PARTIE 1 : ÉVALUATION DES RESSOURCES (8 points)
❓ Question 1 (1 pt)
Parmi ces structures algorithmiques, laquelle permet de répéter une séquence d'instructions un nombre de fois connu à l'avance ?
  • A. La structure conditionnelle SI...ALORS...SINON
  • B. La boucle TANT QUE (WHILE)
  • C. La boucle POUR (FOR)
  • D. La structure SELON...CAS (SWITCH)

    • ✅ Réponse correcte : C. La boucle POUR (FOR)

    🔹 La boucle POUR est utilisée quand le nombre d'itérations est connu à l'avance. La boucle TANT QUE est utilisée quand le nombre d'itérations dépend d'une condition.

    ❓ Question 2 (1 pt)
    En langage C, quelle est la bonne syntaxe pour déclarer un tableau pouvant contenir 50 nombres réels ?
  • A. int notes[50];
  • B. float notes[50];
  • C. tableau notes[50] de réel;
  • D. float[50] notes;

    • ✅ Réponse correcte : B. float notes[50];

    🔹 float est le type pour les nombres réels (décimaux) en C. int serait pour les entiers. La syntaxe correcte est : type nomTableau[taille];

    ❓ Question 3 (1 pt)
    En C, l'indice du premier élément d'un tableau est 1 (comme en algorithmique).
  • A. Vrai
  • B. Faux

    • ✅ Réponse correcte : B. Faux

    🔹 En langage C, les indices des tableaux commencent à 0 (et non à 1). Un tableau int T[10]; a des indices allant de 0 à 9. En algorithmique pédagogique, on commence souvent à 1.

    ❓ Question 4 (1 pt)
    Que fait l'opérateur ++ en langage C ?
  • A. Il compare deux valeurs
  • B. Il incrémente une variable de 1 (i++ équivaut à i = i + 1)
  • C. Il décrémente une variable de 1
  • D. Il déclare une variable

    • ✅ Réponse correcte : B. Il incrémente une variable de 1

    🔹 L'opérateur ++ est l'opérateur d'incrémentation. i++ est équivalent à i = i + 1.

    ❓ Question 5 (1 pt)
    Laquelle de ces affirmations sur les fonctions en C est correcte ?
  • A. Une fonction ne peut jamais retourner de valeur
  • B. Une fonction peut retourner une valeur grâce à l'instruction return
  • C. Une fonction ne peut pas avoir de paramètres
  • D. Une fonction doit obligatoirement être appelée main()

    • ✅ Réponse correcte : B. Une fonction peut retourner une valeur grâce à l'instruction return

    🔹 En C, une fonction peut retourner une valeur (fonction) ou ne rien retourner (procédure, indiquée par void). L'instruction return permet de renvoyer une valeur.

    ❓ Question 6 (1,5 pts)
    Citez les trois types de structures conditionnelles vus en cours, et donnez un exemple d'utilisation pour chacune.

    ✅ Éléments de réponse :

    1. SI...ALORS...SINON (IF...ELSE) : Exemple : Si (note >= 10) alors Ecrire("Admis") Sinon Ecrire("Non admis") Finsi

    2. SELON...CAS (SWITCH) : Exemple : Selon choix Faire 1: Ecrire("Lancer jeu") 2: Ecrire("Quitter") FinSelon

    3. SI...ALORS simple (sans SINON) : Exemple : Si (age >= 18) alors Ecrire("Majeur") Finsi

    ❓ Question 7 (1,5 pts)
    Quelle est la différence fondamentale entre une fonction et une procédure ? Donnez un exemple en langage C pour chacun.

    ✅ Éléments de réponse :

    Une fonction retourne une valeur (utilise return) alors qu'une procédure ne retourne rien (type void en C).

    Exemple de fonction :

    int carre(int x) {
    return x * x;
}

    Exemple de procédure :

    void afficherBonjour() {
    printf("Bonjour !");
}
    💻 PARTIE 2 : ÉVALUATION DES COMPÉTENCES (12 points)
    📋 SITUATION PROBLÈME (12 pts)

    Contexte : Au Lycée Moderne de Douala, le professeur d'informatique doit automatiser le traitement des notes d'une classe de 1ère D. Il souhaite écrire un programme en langage C qui permet de :

    1. Saisir les notes de 15 élèves (notes sur 20).
    2. Calculer et afficher la moyenne de la classe.
    3. Afficher le nombre d'élèves ayant obtenu une note supérieure ou égale à 10 (admis).
    4. Afficher la meilleure note et la moins bonne note.

    Travail demandé :

    1. (4 pts) Écrire l'algorithme correspondant à ce traitement en utilisant les structures appropriées (boucle POUR, structure conditionnelle SI).
    2. (4 pts) Traduire cet algorithme en langage C (code complet avec la fonction main()).
    3. (2 pts) Justifier le choix de la boucle POUR plutôt que TANT QUE dans ce contexte.
    4. (2 pts) À votre avis, pourquoi utilise-t-on un tableau pour stocker les notes plutôt que des variables simples ?

    ✅ CORRIGÉ

    1. Algorithme (4 pts) :

    Variable : tabNotes : tableau[1..15] de réel
           i, admis, meilleure, moinsBonne : entier
           somme, moyenne : réel

Début
    // Saisie des notes
    Pour i de 1 à 15 pas=1 faire
        Ecrire("Entrez la note de l'élève ", i)
        Lire(tabNotes[i])
    FinPour
    
    // Initialisation
    somme ← 0
    admis ← 0
    meilleure ← tabNotes[1]
    moinsBonne ← tabNotes[1]
    
    // Traitement
    Pour i de 1 à 15 pas=1 faire
        somme ← somme + tabNotes[i]
        
        Si (tabNotes[i] >= 10) Alors
            admis ← admis + 1
        FinSi
        
        Si (tabNotes[i] > meilleure) Alors
            meilleure ← tabNotes[i]
        FinSi
        
        Si (tabNotes[i] < moinsBonne) Alors
            moinsBonne ← tabNotes[i]
        FinSi
    FinPour
    
    moyenne ← somme / 15
    
    // Affichage des résultats
    Ecrire("Moyenne de la classe : ", moyenne)
    Ecrire("Nombre d'admis : ", admis)
    Ecrire("Meilleure note : ", meilleure)
    Ecrire("Moins bonne note : ", moinsBonne)
Fin

    2. Programme en langage C (4 pts) :

    #include <stdio.h>

int main() {
    float tabNotes[15];
    int i, admis = 0;
    float somme = 0, moyenne;
    float meilleure, moinsBonne;
    
    // Saisie des notes
    for(i = 0; i < 15; i++) {
        printf("Entrez la note de l'élève %d : ", i+1);
        scanf("%f", &tabNotes[i]);
    }
    
    // Initialisation
    meilleure = tabNotes[0];
    moinsBonne = tabNotes[0];
    
    // Traitement
    for(i = 0; i < 15; i++) {
        somme = somme + tabNotes[i];
        
        if(tabNotes[i] >= 10) {
            admis++;
        }
        
        if(tabNotes[i] > meilleure) {
            meilleure = tabNotes[i];
        }
        
        if(tabNotes[i] < moinsBonne) {
            moinsBonne = tabNotes[i];
        }
    }
    
    moyenne = somme / 15;
    
    // Affichage des résultats
    printf("Moyenne de la classe : %.2f\n", moyenne);
    printf("Nombre d'admis : %d\n", admis);
    printf("Meilleure note : %.2f\n", meilleure);
    printf("Moins bonne note : %.2f\n", moinsBonne);
    
    return 0;
}

    3. Justification du choix de la boucle POUR (2 pts) :
    La boucle POUR est la mieux adaptée car on connaît à l'avance le nombre d'élèves (15). Le nombre d'itérations est fixe et connu dès le départ, ce qui correspond parfaitement à l'utilisation de la structure POUR.

    4. Pourquoi utiliser un tableau ? (2 pts) :
    On utilise un tableau car on doit stocker 15 notes. Un tableau permet de regrouper plusieurs données de même type sous un même nom et d'y accéder via un indice. Avec des variables simples, il faudrait déclarer 15 variables différentes (note1, note2, ..., note15), ce qui est fastidieux et rend le code difficile à maintenir.
