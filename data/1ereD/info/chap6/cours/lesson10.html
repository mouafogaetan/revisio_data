UE 10 : Les caractéristiques d'une table

Bases de données · Première C, D, E · Module 1 - Unité d'apprentissage 6

📌 Éléments clés à retenir

1. Qu'est-ce qu'une table ?

Une table est une structure de données organisée en lignes et en colonnes. C'est le composant fondamental d'une base de données relationnelle. Elle permet de stocker des informations relatives à un même thème (ex: clients, produits, étudiants).

  • Chaque colonne représente une catégorie d'information (un attribut).
  • Chaque ligne représente un enregistrement (une entité spécifique).
  • L'intersection d'une ligne et d'une colonne contient une valeur unique.

2. Vocabulaire essentiel d'une table

Il est crucial de maîtriser les termes suivants pour décrire et manipuler une base de données.

  • Champ (ou colonne ou attribut) : Une catégorie d'information. Exemple : Nom, Prénom, Date_naissance.
  • Enregistrement (ou ligne ou tuple) : L'ensemble des informations concernant un seul élément. Exemple : les données de l'étudiant "Kamga Paul".
  • Clé primaire (Primary Key) : Un champ (ou une combinaison de champs) dont la valeur est unique pour chaque enregistrement et ne peut jamais être nulle. Elle sert d'identifiant unique.

3. Les contraintes d'intégrité

Ce sont des règles qui garantissent la cohérence et l'exactitude des données dans une table. Elles empêchent les erreurs et les anomalies.

  • Contrainte d'unicité : Un champ ne peut pas avoir la même valeur sur deux lignes différentes (ex: deux élèves ne peuvent pas avoir le même numéro de matricule). C'est le cas de la clé primaire.
  • Contrainte de non-nullité : Un champ doit toujours avoir une valeur, elle ne peut pas être vide (ex: le nom d'un client ne peut pas être inconnu).
  • Contrainte de format ou de domaine : Les valeurs d'un champ doivent être d'un certain type ou respecter un format (ex: la colonne Quantité doit contenir un nombre entier positif, la colonne Email doit contenir un '@').
💡 Astuce : Les contraintes d'intégrité sont comme les règles d'un jeu. Elles assurent que tout le monde joue de la même manière et que le tableau de score (la base de données) reste fiable.

💡 Exemples de compréhension

Exemple concret : La table "Étudiants"

Voici une table contenant des informations sur des étudiants.

Analyse du vocabulaire :
  • Champs (colonnes) : ID_Etudiant, Nom, Prenom, Age.
  • Enregistrement (ligne) : Toutes les informations concernant un étudiant. Exemple : "1, DIOP, Aminata, 19" est un enregistrement.
  • Clé primaire : ID_Etudiant. Chaque étudiant a un ID unique (1, 2, 3...). Aucun ID ne se répète et aucun étudiant ne peut être enregistré sans ID.
💬 Pourquoi l'ID est meilleur que le nom comme clé primaire ? Parce qu'il peut y avoir deux étudiants qui s'appellent "Jean" ! L'ID est un moyen sûr d'identifier chacun de manière unique.

Application des contraintes d'intégrité

On veut ajouter un nouvel étudiant à la table. Quelles contraintes doivent être vérifiées ?

Vérifications à faire :
  1. Clé primaire unique : L'ID_Etudiant du nouvel étudiant ne doit pas déjà exister dans la table.
  2. Non-nullité : Tous les champs doivent être remplis. On ne peut pas ajouter un étudiant sans son Nom ou son Prenom.
  3. Contrainte de domaine : La valeur dans la colonne Age doit être un nombre entier. On ne peut pas y mettre du texte comme "majeur".
💡 La violation d'une contrainte d'intégrité provoque une erreur par le SGBD, et l'opération (ajout, modification) est annulée.

🛠️ Les opérations fondamentales sur une table

Pour interagir avec les données d'une table, on utilise quatre opérations de base, issues de l'algèbre relationnelle.

  • Sélection (ou Restriction) : Choisir certains enregistrements (lignes) qui répondent à une condition donnée.
    Exemple : "Sélectionner tous les étudiants dont l'age est supérieur à 18 ans".
  • Projection : Choisir certains champs (colonnes) dans la table.
    Exemple : "Afficher uniquement les noms et prénoms de tous les étudiants".
  • Suppression : Retirer un ou plusieurs enregistrements de la table.
    Exemple : "Supprimer l'étudiant dont l'ID_Etudiant est 5".
  • Mise à jour (ou Modification) : Modifier les valeurs d'un ou plusieurs champs dans un ou plusieurs enregistrements.
    Exemple : "Changer l'age de l'étudiant avec l'ID 3 pour le passer à 20 ans".

⚠️ Pièges courants / À ne pas confondre

  • Ne pas confondre Champ et Enregistrement : Un champ est une colonne (catégorie), un enregistrement est une ligne (instance).
  • Clé primaire vs Champ simple : Tous les champs sont importants, mais seule la clé primaire a le pouvoir d'identifier un enregistrement de manière unique et infaillible.
  • NULL n'est pas zéro ou vide : Dans une base de données, NULL signifie "valeur inconnue" ou "absente". Ce n'est pas la même chose que le nombre 0 ou une chaîne de caractères vide (""). Une clé primaire ne peut jamais être NULL.
  • Confusion Sélection/Projection : La sélection réduit le nombre de lignes (elle filtre). La projection réduit le nombre de colonnes.

✅ Synthèse pour l'examen

  • Une table = lignes (enregistrements) + colonnes (champs).
  • Clé primaire = identifiant unique et non nul.
  • Contraintes d'intégrité = règles pour des données propres et cohérentes (unicité, non-nullité, format).
  • Les 4 opérations magiques : Sélectionner (lignes), Projeter (colonnes), Supprimer, Mettre à jour.
