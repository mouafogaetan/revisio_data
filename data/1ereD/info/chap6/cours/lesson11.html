📌 Fonctions essentielles d’une base de données

1. Interrogation (Requêtage)

Une base de données permet de poser des questions et d’obtenir des résultats précis à partir des données stockées. L’interrogation se fait généralement avec un langage comme SQL (Structured Query Language). On peut filtrer, trier, regrouper ou calculer des informations.

  • SELECT : choisir les colonnes à afficher.
  • WHERE : appliquer des conditions (filtres).
  • ORDER BY : trier les résultats.
  • GROUP BY : regrouper des enregistrements (ex: total par catégorie).
SELECT Nom, Prix FROM Produits
WHERE Prix > 1000 ORDER BY Prix DESC;

💡 À retenir : L’interrogation permet d’extraire uniquement l’information pertinente sans modifier les données.

2. Ajout, modification, suppression de données (Mise à jour)

Une base de données ne doit pas être figée. Les opérations CRUD (Create, Read, Update, Delete) sont fondamentales.

  • Ajouter un nouvel enregistrement → INSERT INTO
  • Modifier des valeurs existantes → UPDATE
  • Supprimer un ou plusieurs enregistrements → DELETE FROM
-- Ajouter un élève
INSERT INTO Eleves (Nom, Prenom, Classe)
VALUES ('Ndong', 'Marie', 'Première C');

-- Modifier le prix d’un article
UPDATE Produits SET Prix = 2500 WHERE Ref = 'ART102';

-- Supprimer les commandes antérieures à 2020
DELETE FROM Commandes WHERE DateCommande < '2020-01-01';
🔐 Attention : Une suppression sans condition (DELETE FROM table) efface TOUTES les lignes. Toujours utiliser WHERE pour cibler.

3. Génération d’états et de rapports

Un état (ou rapport) est une présentation structurée et souvent imprimable des données, sous forme de tableau, graphique ou document. Les bases de données modernes (ou SGBD comme Access, MySQL, PostgreSQL) intègrent des outils pour créer des rapports automatiques.

  • Synthèse : regrouper, totaliser, faire des moyennes (fonctions d’agrégation).
  • Mise en forme : titres, entêtes, pieds de page, logos.
  • Export : PDF, Excel, impression.
📄 Exemple de rapport : « Liste des élèves par classe »
📌 En-tête : Lycée de Douala – Année scolaire 2025
📌 Contenu : tableau avec matricule, nom, moyenne
📌 Pied de page : nombre total d’élèves + date d’édition.
Fonction SQL sous-jacente : SELECT Classe, COUNT(*) FROM Eleves GROUP BY Classe;

💡 Exemples d’application

🎓 Exemple 1 : Interroger la table « Élèves »

Contexte : Une base de données scolaire contient une table Eleves avec les champs : Matricule, Nom, Prenom, Classe, Moyenne.
Question : « Afficher les noms et moyennes des élèves de Première C dont la moyenne est supérieure ou égale à 12, triés par moyenne décroissante. »

Résolution / Requête SQL :

SELECT Nom, Moyenne
FROM Eleves
WHERE Classe = 'Première C' AND Moyenne >= 12
ORDER BY Moyenne DESC;
💬 Astuce : L’ordre des clauses est important : SELECT → FROM → WHERE → ORDER BY. On filtre avant de trier.

🛒 Exemple 2 : Mise à jour (augmentation des prix)

Un gérant souhaite augmenter de 10% le prix de tous les produits de la catégorie « Électronique » dans la table Produits.

Modification avec UPDATE :

UPDATE Produits
SET Prix = Prix * 1.10
WHERE Categorie = 'Électronique';

✅ Après cette commande, tous les prix des produits électroniques sont augmentés de 10%. On peut vérifier avec une requête SELECT.

⚠️ Toujours faire un SELECT avec les mêmes conditions avant un UPDATE/DELETE pour confirmer les enregistrements concernés.

📑 Exemple 3 : Générer un rapport des ventes par mois

Une entreprise veut un rapport indiquant le montant total des ventes pour chaque mois de l’année.

Requête d’agrégation (avec regroupement) :

SELECT MONTH(DateVente) AS Mois, SUM(Montant) AS TotalVentes
FROM Ventes
WHERE YEAR(DateVente) = 2025
GROUP BY MONTH(DateVente)
ORDER BY Mois;

Ce rapport peut être exporté vers un tableur ou affiché dans une application de gestion. Chaque ligne représente un mois avec le total correspondant.

🔧 Autres fonctions importantes d’un SGBD

🔐 Gestion des accès et sécurité

Une base de données permet de définir qui peut voir ou modifier quelles données (droits d’accès, utilisateurs, rôles). Exemple : un employé peut uniquement consulter les produits, mais le gestionnaire peut mettre à jour les prix.

  • GRANT : donner des privilèges.
  • REVOKE : retirer des privilèges.

🔄 Sauvegarde et restauration

Prévenir la perte de données : un SGBD offre des outils de sauvegarde (backup) et de restauration (restore). C’est une fonction essentielle pour la continuité du système d’information.

📀 En entreprise, on planifie des sauvegardes automatiques quotidiennes. En cas de panne, on peut restaurer la base à un état antérieur.

📊 Fonctions d’agrégation (synthèse de données)

Ces fonctions permettent de résumer des colonnes numériques : SUM (somme), AVG (moyenne), COUNT (nombre), MAX, MIN.

-- Calculer la moyenne des notes de la classe
SELECT AVG(Moyenne) AS MoyenneGenerale FROM Eleves WHERE Classe = 'Première D';

⚠️ Pièges courants & Bonnes pratiques

  • Oublier la clause WHERE : Une commande DELETE ou UPDATE sans condition affecte toutes les lignes ! Toujours vérifier avec un SELECT d’abord.
  • Confondre interrogation et modification : SELECT ne change rien, INSERT/UPDATE/DELETE modifient les données. Faire attention aux permissions.
  • Ne pas gérer les types de données : comparer une chaîne avec un nombre (ex: WHERE prix = '200' peut fonctionner mais moins performant).
  • Rapport sans regroupement : Pour une synthèse (total, moyenne), pensez à GROUP BY si vous affichez d’autres colonnes non agrégées.
  • Sauvegarde négligée : Une panne disque = perte définitive. Toujours prévoir des copies de sécurité régulières.

🌐 Lien avec le système d’information (UE7)

Une base de données est le cœur du système d’information automatisé. Les fonctions vues ci-dessus (interroger, mettre à jour, générer des rapports) permettent aux différents acteurs (système opérant, pilotage, information) de remplir leurs missions :

  • Le secrétaire saisit (ajout) les notes des élèves.
  • Le proviseur interroge la base pour avoir le classement.
  • Le conseil de classe génère un rapport des moyennes par matière.

Exemple concret : Dans une école, la fonction « générer un bulletin » est un rapport automatisé qui combine les données de plusieurs tables (élèves, notes, matières).
