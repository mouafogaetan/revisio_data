Exécution d'un programme C

Informatique · Première C, D, E · Unité d'Apprentissage 11 (UE 27)

📌 Éléments clés à retenir

Du code source au programme exécutable : les étapes fondamentales

Un programme écrit en langage C (fichier .c) doit être transformé en un fichier binaire exécutable par la machine. Ce processus comprend 4 étapes principales :

  • 1. Édition (ou écriture du code) : création du fichier source avec un éditeur de texte.
  • 2. Précompilation (prétraitement) : traitement des directives #include, #define, etc. Génère un code source “pur”.
  • 3. Compilation : traduction du code source en code assembleur puis en code objet (fichier .o ou .obj). Vérifie la syntaxe.
  • 4. Édition de liens (linkage) : fusion des différents fichiers objets avec les bibliothèques (ex: stdio.h) pour créer l'exécutable final (.exe sous Windows, ou fichier binaire sous Linux/macOS).

Compilateur C et rôles essentiels

Un compilateur est un programme qui traduit le code source (compréhensible par l'humain) en langage machine. Le compilateur le plus répandu est GCC (GNU Compiler Collection). Exemples de commandes :

  • gcc monprog.c -o monprog → compile et génère l'exécutable monprog (ou monprog.exe sur Windows).
  • gcc -c monprog.c → génère uniquement le fichier objet monprog.o (sans édition de liens).
  • gcc monprog.o -o monprog → édition de liens à partir du fichier objet.

D'autres compilateurs : Clang, Turbo C (ancien), MinGW (Windows). L'exécution du programme se fait via ./monprog (Linux/macOS) ou monprog.exe (invite de commandes Windows).

Erreurs fréquentes lors de l'exécution

On distingue plusieurs types d'erreurs qui empêchent la bonne exécution d'un programme C :

  • Erreurs de compilation : violation des règles de syntaxe (point-virgule oublié, variable non déclarée, parenthèse manquante). Le compilateur les détecte.
  • Erreurs d'édition de liens : fonction utilisée mais jamais définie, ou bibliothèque manquante (ex: undefined reference to 'fonction').
  • Erreurs d'exécution (run-time) : le programme compile mais échoue à l'exécution (division par zéro, accès à un pointeur nul, débordement de tableau).
  • Erreurs logiques : le programme tourne sans planter mais produit des résultats incorrects (mauvaise formule ou algorithme faux).

💡 Exemples de compréhension

Exemple 1 : Compilation et exécution pas à pas (GCC)

On dispose du fichier source bonjour.c contenant un programme simple qui affiche "Bonjour Cameroun !". Décrivez les commandes et les étapes pour obtenir l'exécutable.

Résolution / Explication :
  1. Ouvrir un terminal (ou invite de commandes).
  2. Se placer dans le dossier contenant bonjour.c (commande cd).
  3. Compiler : gcc bonjour.c -o bonjour
    → Le compilateur vérifie la syntaxe, puis génère bonjour (Linux/macOS) ou bonjour.exe (Windows).
  4. Exécuter : ./bonjour (Linux/macOS) ou bonjour.exe (Windows).
    → Affichage : Bonjour Cameroun !
💬 Astuce : Utilisez l'option -Wall pour afficher tous les avertissements (gcc -Wall bonjour.c -o bonjour). Cela aide à détecter des erreurs potentielles.

Exemple 2 : Détection et correction d'une erreur de compilation

Voici un code C erroné. Identifiez les erreurs et corrigez-les.

#include
int main()
{
   int a = 10
   printf("Valeur de a : %d\n", a)
   return 0;
}

Correction : Il manque les points-virgules (;) à la fin des deux instructions. De plus, la fonction printf nécessite une déclaration correcte. Version corrigée :
#include
int main() {
   int a = 10;
   printf("Valeur de a : %d\n", a);
   return 0;
}
Après correction, la compilation (gcc programme.c -o prog) réussit et l'exécution affiche Valeur de a : 10.
🔍 Rappel : Chaque instruction en C se termine par un point-virgule. L'absence de point-virgule est l'une des erreurs les plus fréquentes chez les débutants.

Exemple 3 : Erreur d'édition de liens (bibliothèque manquante)

Un étudiant a écrit un programme qui utilise la fonction sqrt() pour calculer une racine carrée. Il inclut bien #include mais la compilation échoue avec le message : undefined reference to 'sqrt'. Pourquoi ?

Explication : La bibliothèque mathématique (libm) n'est pas automatiquement liée par défaut avec GCC. Il faut explicitement lier la bibliothèque mathématique à l'aide de l'option -lm.
Commande correcte : gcc calcule.c -o calcule -lm.
Cela permet à l'éditeur de liens de résoudre la référence à la fonction sqrt.

⚠️ Pièges courants & Bonnes pratiques

  • Ne pas confondre compilation et exécution : Un programme peut compiler parfaitement mais planter à l'exécution (ex: division par zéro, boucle infinie). Il faut tester rigoureusement.
  • Extension de fichier : Toujours enregistrer les sources C avec l'extension .c. L'oubli peut empêcher le compilateur de reconnaître le langage.
  • Cas des IDE : Dans un environnement intégré (Code::Blocks, Dev-C++, VS Code), les étapes de compilation/édition de liens sont automatisées, mais il est essentiel de comprendre le processus sous-jacent.
  • Fichier objet vs exécutable : Le fichier .o (objet) n'est pas directement exécutable. Il doit subir l'édition de liens pour devenir un programme complet.
  • Attention au chemin d'accès : Sous Windows, si GCC n'est pas dans le PATH, la commande gcc ne sera pas reconnue. Solution : installer MinGW et configurer les variables d'environnement.

🛠️ Schéma récapitulatif & commandes pratiques

Cycle de développement en C

Commandes résumées (GCC) :

# Compilation simple avec exécutable 'programme'
gcc monfichier.c -o programme

# Génération du fichier objet uniquement
gcc -c monfichier.c → produit monfichier.o

# Édition de liens à partir de l'objet
gcc monfichier.o -o programme

# Compilation avec avertissements et debug info (-g)
gcc -Wall -g monfichier.c -o programme

# Exécution sous Linux / macOS
./programme

# Exécution sous Windows (cmd)
programme.exe
🖥️ Testez-vous : Que fait l'option -c ? Pourquoi utiliser -o ? (Réponse : -c arrête la compilation après la génération du code objet, sans édition de liens ; -o spécifie le nom du fichier de sortie.)

📚 Rôle des bibliothèques dans l'exécution

Les bibliothèques standards (comme stdio.h, stdlib.h, math.h) contiennent des fonctions prédéfinies. Lors de l'édition de liens, le compilateur relie les appels de fonctions à leur code réel. Si une bibliothèque n'est pas liée (ex: -lm pour math), l'exécutable ne pourra pas être généré.

Sous Windows, les fichiers .dll (Dynamic Link Libraries) sont utilisés à l'exécution. Sous Linux, les .so. Une bonne compréhension du linkage évite des erreurs mystérieuses.

🎯 À retenir pour l'examen (Probatoire)

  • ✔️ Le compilateur transforme le code source en code objet, puis l'éditeur de liens crée l'exécutable.
  • ✔️ GCC est le compilateur de référence : gcc source.c -o exec puis ./exec pour exécuter.
  • ✔️ Savoir reconnaître les trois grandes familles d'erreurs : compilation, édition de liens, exécution.
  • ✔️ Une fonction mathématique exige l'option -lm lors de la compilation.
  • ✔️ Un programme qui compile n'est pas nécessairement juste ; les tests sont indispensables.
