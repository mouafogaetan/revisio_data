⚙️ Exécution d'un programme C

Informatique · Première D · Cycle secondaire (Cameroun)

📌 Étapes clés de compilation & exécution

🔧 1. Du code source au programme exécutable

Un programme écrit en langage C doit passer par plusieurs transformations avant de pouvoir être exécuté par l'ordinateur. Ces étapes constituent le processus de compilation et d'édition de liens.

  • Prétraitement : Traite les directives #include, #define. Génère un code source modifié.
  • Compilation proprement dite : Transforme le code C en code assembleur (fichier .s ou .asm).
  • Assemblage : Convertit le code assembleur en code objet (fichier .obj ou .o).
  • Édition de liens (Linkage) : Combine les fichiers objets et les bibliothèques (ex: stdio.h) pour créer un exécutable (.exe sous Windows, a.out sous Linux).

🖥️ 2. Utilisation d’un compilateur C (exemple GCC)

Le compilateur le plus répandu est GCC (GNU Compiler Collection). Sous Linux ou dans un terminal (Windows avec MinGW), les commandes suivantes permettent de produire et exécuter un programme.

# Étape 1 : Compiler le fichier source monprogramme.c
gcc monprogramme.c -o monprogramme

# Étape 2 : Exécuter le programme généré
./monprogramme (sous Linux/macOS)
monprogramme.exe (sous Windows)

L'option -o permet de spécifier le nom de l'exécutable. Sans cette option, GCC produit un fichier nommé a.out (Linux) ou a.exe.

🧪 3. Environnements de développement intégré (IDE)

Au lieu d'utiliser la ligne de commande, on peut employer un IDE (Code::Blocks, Dev-C++, Visual Studio Code, Eclipse) qui automatise la compilation, le débogage et l’exécution.

  • L’IDE appelle le compilateur en arrière-plan via un simple clic sur "Build & Run".
  • Il permet d’identifier plus facilement les erreurs de syntaxe (lignes soulignées).
  • Débogueur intégré : exécution pas à pas, visualisation des variables.

💡 Exemples de compréhension

Exemple concret n°1 : Compilation d'un premier programme "Salut Cameroun"

Énoncé : Vous avez écrit un fichier bonjour.c contenant : #include <stdio.h> int main() { printf("Salut les D !\n"); return 0; } Décrivez les étapes pour obtenir l'exécution.

🔍 Démarche :
  1. Ouvrir un terminal dans le dossier contenant bonjour.c.
  2. Taper la commande : gcc bonjour.c -o bonjour (compilation + linkage).
  3. Si aucune erreur, un exécutable bonjour (Linux) ou bonjour.exe (Windows) est créé.
  4. Exécuter avec ./bonjour → affiche "Salut les D !".
💬 Astuce : Vérifiez toujours l'absence d'erreurs après compilation. Une erreur "undefined reference" signifie qu’il manque une bibliothèque ou une fonction.

Exemple d’application n°2 : Fichiers multiples (addition.c, main.c)

Deux fichiers : addition.c contient une fonction int additionner(int a, int b) ; principal.c appelle cette fonction. Comment compiler et lier ces deux fichiers ?

✅ Résolution :
On compile les deux fichiers ensemble en une seule commande :
gcc principal.c addition.c -o programme_complet
L'éditeur de liens combine les deux fichiers objets et résout la référence à additionner.
Exécution : ./programme_complet
💡 Pour des projets plus gros, on peut compiler séparément (gcc -c addition.c pour créer addition.o) puis lier (gcc principal.o addition.o -o prog).

Exemple n°3 : Interpréter les messages d'erreur

Lors de la compilation, GCC retourne : error: 'printf' was not declared in this scope. Quelle est la cause et la correction ?

Cause : On a oublié d'inclure la bibliothèque <stdio.h> qui contient la déclaration de printf.
Correction : Ajouter #include <stdio.h> en début de fichier.
🧠 Pensez toujours à inclure les headers nécessaires. #include <stdio.h> pour entrées/sorties ; #include <math.h> pour les fonctions mathématiques.

⚠️ Pièges courants (exécution & compilation)

  • Oublier de sauvegarder le fichier : On compile une version ancienne → les modifications ne sont pas prises en compte.
  • Confondre compilation et exécution : La compilation ne lance pas le programme. Il faut exécuter l’exécutable généré.
  • Ne pas respecter la casse : GCC distingue minuscules/majuscules. Main ≠ main.
  • Erreur de chemin : Tenter d’exécuter un fichier qui n’est pas dans le répertoire courant sans ./ (sous Linux/macOS) → command not found.
  • Utiliser une fonction sans inclure son header : Exemple : sqrt() nécessite #include <math.h> et parfois l’option -lm lors de la compilation.

🔁 Récapitulatif mnémotechnique : "P-C-A-L"

Prétraitement → Compilation → Assemblage → Link (édition de liens). Retenez cette chaîne pour décrire les phases de transformation du code C jusqu'à l'exécutable.

⌨️ Commandes pratiques & bonnes habitudes

📋 Résumé des commandes GCC (ligne de commande)

  • gcc fichier.c → produit a.out (Linux) ou a.exe (Windows)
  • gcc fichier.c -o nom_sortie → personnalise le nom de l'exécutable
  • gcc -Wall fichier.c -o prog → active tous les avertissements (aide à écrire un code plus propre)
  • ./prog (Linux/Mac) ou prog.exe (Windows) → exécution
  • gcc -c source.c → génère un fichier objet (.o) sans édition de liens

📍 À retenir pour les examens : Le rôle du compilateur est de traduire le code compréhensible par l'humain (C) en langage machine. L'exécution est la phase où le processeur lit les instructions binaires.

🧠 Auto-évaluation (flash)

Question 1 : Que se passe-t-il si vous compilez un programme C avec succès mais que vous ne voyez aucun résultat ?

Réponse : Vous avez probablement oublié d'exécuter le fichier produit. Après compilation, lancez ./nom_programme.

Question 2 : Quelle est la différence entre un fichier .c et un fichier .exe ?

Réponse : .c est le code source lisible. .exe est le fichier binaire exécutable généré après compilation + linkage.

✅ Pour réviser : pratiquez la compilation d’un programme simple "Hello World" sur votre machine (ou sur un compilateur en ligne) !
