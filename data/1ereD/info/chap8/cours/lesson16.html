UE 16 : Notions de procédure et fonction

Informatique · Première C, D, E (Probatoire) · Module 2 : Algorithmique et Programmation

📌 Définitions fondamentales

🔁 Procédure

Une procédure est un sous-programme qui exécute une série d'instructions sans retourner de valeur. Elle réalise une action (par exemple afficher un menu, trier un tableau, calculer et afficher un résultat). En algorithmique, on l’appelle souvent sous-programme ou sous-routine.

  • But : Effectuer une tâche spécifique, améliorer la lisibilité et la réutilisabilité.
  • Appel : nom_procedure(paramètres) seul.
  • Exemple en algorithmique : AfficherMenu() ou TrierTableau(tab, n).

⚙️ Fonction

Une fonction est un sous-programme qui retourne une seule valeur (entier, réel, booléen, etc.) à l’endroit où elle est appelée. Elle calcule un résultat à partir des paramètres fournis.

  • But : Effectuer un calcul et renvoyer le résultat pour une utilisation ultérieure.
  • Appel : La fonction s’utilise dans une expression (ex : moyenne ← calculerMoyenne(notes, nb)).
  • En algorithmique : Fonction carré(x : entier) : entier retourne x*x.

🌐 Variables globales vs locales

Comprendre la portée des variables est essentiel pour éviter les effets de bord.

  • Variable locale : Déclarée à l’intérieur d’une procédure/fonction. Accessible uniquement dans ce bloc. Elle est détruite à la fin de l’exécution du sous-programme.
  • Variable globale : Déclarée en dehors de toute procédure/fonction (souvent en tête d’algorithme). Accessible depuis n’importe quelle partie du programme.
  • 🔐 Bonnes pratiques : Privilégier les variables locales pour une meilleure modularité et éviter les modifications intempestives.

💡 Exemples illustrés (Algorithmique & C)

📐 Exemple 1 : Fonction – Calcul de la surface d’un rectangle

On souhaite créer une fonction qui reçoit la longueur et la largeur (réels) et retourne la surface.

// ALGORITHMIQUE
Fonction surfaceRectangle(L, l : réel) : réel
    Début
        Retourner L * l
    Fin
// LANGAGE C
#include 

float surfaceRectangle(float L, float l) {
    return L * l;
}

int main() {
    float longu = 5.5, larg = 3.2;
    float surf = surfaceRectangle(longu, larg);
    printf("Surface = %.2f\n", surf);
    return 0;
}

Analyse : La fonction surfaceRectangle reçoit deux paramètres et renvoie leur produit. Dans main(), le résultat est stocké dans la variable surf.

💬 Astuce : Une fonction peut être appelée directement dans un printf : printf("%f", surfaceRectangle(4,5));

🖨️ Exemple 2 : Procédure – Affichage d’un message de bienvenue

Créer une procédure qui affiche un message personnalisé sans retourner de valeur.

// ALGORITHMIQUE
Procédure afficherBienvenue(nom : chaîne)
    Début
        Ecrire("Bienvenue ", nom, " !")
    Fin
// LANGAGE C (procédure = fonction void)
#include 

void afficherBienvenue(char nom[]) {
    printf("Bienvenue %s !\n", nom);
}

int main() {
    afficherBienvenue("Marie");
    return 0;
}

Explication : La procédure ne renvoie rien (type void en C). Elle effectue simplement une action d’affichage.

💡 À retenir : En langage C, on utilise void pour indiquer qu’une fonction ne retourne pas de valeur (équivalent à une procédure).

🌍 Exemple 3 : Variable globale vs locale – Pièges fréquents

Dans cet exemple, on illustre la différence de portée.

// ALGORITHMIQUE
Variable globale : total ← 0

Procédure ajouter(valeur : entier)
    Variable locale : somme ← 0
    somme ← somme + valeur   // modifie la variable locale
    total ← total + valeur   // modifie la variable globale
    Ecrire("Somme locale : ", somme)
Fin

Programme principal:
    ajouter(5)
    ajouter(3)
    Ecrire("Total global : ", total)   // Affiche 8

Interprétation : somme est réinitialisée à chaque appel, alors que total garde sa valeur entre les appels. Les modifications sur les variables globales sont persistantes.

⚠️ Attention : L'usage excessif de variables globales peut rendre le programme difficile à déboguer. Préférez le passage de paramètres.

🛠️ Fonctions prédéfinies & définies par l'utilisateur

📚 Bibliothèques standards

De nombreux langages (C, Python, JavaScript) offrent des fonctions prêtes à l’emploi.

  • En C : sqrt() (racine carrée), pow() (puissance), printf() (affichage), scanf() (lecture).
  • En algorithmique : Ecrire() , Lire() , Aleatoire() sont des fonctions implicites.
  • Fonctions définies par l’utilisateur : Celles que le programmeur crée pour répondre à un besoin spécifique (exemple : factorielle() , estPair()).
🧠 Astuce : Réutiliser des fonctions évite la duplication de code et facilite la maintenance.

📬 Passage de paramètres (notions essentielles)

Paramètres formels vs effectifs

Lors de la définition d’un sous-programme, on déclare des paramètres formels (variables locales qui recevront les valeurs). À l’appel, on transmet des paramètres effectifs (ou arguments). En C, le passage se fait par valeur : une copie de la valeur est passée, donc la modification à l’intérieur n’affecte pas l’original. 

// Exemple : passage par valeur en C
void incrementer(int x) {
    x = x + 1;   // modification de la copie locale
    printf("Dans fonction : %d\n", x);
}

int main() {
    int a = 5;
    incrementer(a);
    printf("Dans main : %d\n", a);   // a reste 5
    return 0;
}

Résultat : a n’est pas modifié. Pour modifier une variable externe, on utilisera des pointeurs (niveau première : à connaître pour la suite).

⚠️ Pièges courants & Bonnes pratiques

  • Oublier le return dans une fonction : En C, cela génère un comportement indéfini. En algorithmique, la fonction doit impérativement retourner une valeur.
  • Confondre procédure et fonction : Une procédure ne retourne rien, une fonction retourne toujours une valeur (même void en C n’est pas une fonction mais une procédure).
  • Modifier une variable globale par accident : Cela rend le code moins prévisible. Toujours privilégier les paramètres.
  • Mauvaise gestion des types : Le type de retour de la fonction doit correspondre au type déclaré. Exemple : fonction déclarée entier mais qui retourne un réel → erreur de cohérence.
  • Ne pas initialiser les variables locales : En C, une variable locale non initialisée contient une valeur indéterminée (poubelle). Toujours initialiser avant usage.

🧩 Schéma récapitulatif

Schéma conceptuel (IA) : À gauche une fonction transforme des entrées en une sortie unique. À droite une procédure réalise une action sans retour.

🖥️ Traduction Algorithmique → C

En langage C, toute procédure est une fonction de type void (aucune valeur retournée). Les fonctions non-void doivent comporter une instruction return valeur;.

  • Algo : Fonction max(a,b : entier) : entier → C : int max(int a, int b) { return (a>b)? a : b; }
  • Algo : Procédure afficherSomme(x,y : entier) → C : void afficherSomme(int x, int y) { printf("%d", x+y); }
