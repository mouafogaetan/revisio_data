Leçon 7 : Structure d'un programme en C

Informatique · Terminale C & D · Programme officiel 2025-2026

📌 Éléments clés à retenir

1. Anatomie d'un programme C minimal

Tout programme écrit en langage C suit une structure obligatoire comprenant : l'inclusion de bibliothèques, la déclaration de la fonction principale main(), un corps d'instructions et une valeur de retour.

// Exemple du programme "Bonjour le monde"
#include <stdio.h> // Bibliothèque pour entrées/sorties

int main(void) {
    // Corps du programme : instructions
    printf("Bonjour les Terminales C & D !");
    return 0; // Fin normale du programme
}
  • #include : Directive préprocesseur qui intègre une bibliothèque (fichier d'en-tête .h).
  • int main(void) : Point d'entrée obligatoire. int indique le type de retour (entier).
  • return 0 ; : Signale au système d'exploitation que le programme s'est exécuté sans erreur.
  • {} : Les accolades délimitent le bloc d'instructions de la fonction.

2. Rôle des bibliothèques standard

Les bibliothèques fournissent des fonctions prêtes à l'emploi. Les deux plus utilisées en terminale :

  • <stdio.h> → STandard Input Output : printf() (affichage), scanf() (saisie utilisateur).
  • <stdlib.h> → Fonctions utilitaires (conversion, génération aléatoire, gestion mémoire).
  • <math.h> → Fonctions mathématiques (sqrt, pow, sin…).
💡 Astuce : #include <stdio.h> est indispensable pour toute interaction avec l'écran ou le clavier.

3. Fonction principale : signature et variations

La norme impose int main(void) ou int main() pour un programme sans arguments en ligne de commande. Une autre forme existe (int main(int argc, char *argv[])) pour récupérer des paramètres, mais elle n'est pas exigible au baccalauréat.

  • void entre parenthèses signifie que la fonction ne reçoit aucun argument.
  • Le return est obligatoire pour une fonction déclarée int.
  • Un programme correct se termine toujours par return 0; (succès).

💡 Exemples de compréhension

Examen type : Analyser la structure d'un programme C

Extrait d'un exercice (Bac session 2025, Partie III, Exercice 2) :
Énoncé : On donne le programme C suivant. Identifiez chaque partie (bibliothèque, fonction principale, corps, retour) et dites ce qu'il affiche.

#include <stdio.h>
int main(void) {
    printf("Année scolaire 2025-2026");
    return 0;
}

Résolution :
• #include <stdio.h> : directive d'inclusion pour utiliser printf.
• int main(void) : fonction principale sans paramètres.
• { ... } : bloc contenant l'instruction printf(...) puis return 0;.
• Affichage : Année scolaire 2025-2026 à l'écran.
• Le programme se termine correctement avec le code 0.

💬 À retenir : Toute instruction se termine par un point-virgule ; en C (sauf directives préprocesseur comme #include).

Programme avec saisie et affichage (mise en pratique)

On souhaite un programme qui demande l'âge de l'utilisateur et affiche "Tu as X ans". Compléter la structure manquante.

#include <stdio.h>

int main(void) {
    int age;
    printf("Entrez votre âge : ");
    scanf("%d", &age);
    printf("Tu as %d ans\n", age);
    return 0;
}

Explication :
1. Déclaration de la variable age de type int.
2. printf() affiche un message.
3. scanf() lit une valeur entrée au clavier et la stocke dans age (le & donne l'adresse de la variable).
4. Second printf() utilise le format %d pour insérer l'entier.
5. Le programme renvoie 0 : exécution réussie.

🧠 Note : Les commentaires (// ou /* */) aident à documenter la structure mais ne sont pas exécutés.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier le #include <stdio.h> → provoque une erreur de compilation car printf() ou scanf() ne sont pas reconnus.
  • Attention : Écrire main() sans type de retour (int) n'est plus autorisé par la norme actuelle et peut générer un avertissement.
  • Piège classique : Oublier le point-virgule ; après une instruction (ex: printf("...") sans ;).
  • Confusion possible : return 0 ; n'est pas une instruction d'affichage ; il termine la fonction. Ne pas écrire return 0 ; dans une fonction void (mais main est int).
  • À ne pas oublier : Les accolades { } doivent être équilibrées (ouverture et fermeture).

🧩 Schéma récapitulatif : structure type

Modèle générique à mémoriser

/* 1. Inclusion des bibliothèques */
#include <stdio.h> // obligatoire pour entrées/sorties
#include <stdlib.h> // facultatif selon besoins

/* 2. Fonction principale */
int main(void) {
    /* 3. Déclarations des variables locales */
    int a, b;
    float moyenne;
    
    /* 4. Corps du programme : entrées, traitements, sorties */
    printf("Saisir deux entiers : ");
    scanf("%d %d", &a, &b);
    moyenne = (a + b) / 2.0;
    printf("La moyenne est %.2f\n", moyenne);
    
    /* 5. Retour de la fonction */
    return 0;
}

Chaque programme C respecte cet enchaînement : Directives #include → Déclaration de main() → Variables → Instructions → return 0 ;

✅ Pour le Baccalauréat, savoir écrire un programme simple, reconnaître les blocs et expliquer le rôle de #include et return 0;.
