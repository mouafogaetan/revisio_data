📊 Leçon 6 : Algorithmes de tri (introduction)

Informatique · Terminale C & D · Tri par insertion & tri par sélection

📌 Éléments clés à retenir

🔁 Tri : définition et objectif

Un algorithme de tri organise les éléments d'une collection (tableau, liste) dans un ordre spécifique : croissant (du plus petit au plus grand) ou décroissant. Le tri facilite la recherche, l’analyse et l’affichage des données.

  • Ordre croissant : [3, 5, 7, 12, 19]
  • Ordre décroissant : [19, 12, 7, 5, 3]
  • Critère de stabilité : certains tris conservent l'ordre relatif des éléments égaux.

🃏 Tri par insertion (insertion sort)

Principe : construire progressivement un sous-tableau trié en insérant chaque nouvel élément à sa place correcte. Similaire à la façon dont on trie des cartes à jouer dans sa main.

Pseudo-code Pour i de 1 à n-1 faire:
    x ← T[i]
    j ← i-1
    Tant que j ≥ 0 et T[j] > x faire:
        T[j+1] ← T[j]
        j ← j-1
    T[j+1] ← x
  • ✅ Efficace pour des petites données ou données presque triées.
  • ⏱️ Complexité : O(n²) dans le pire cas, O(n) dans le meilleur cas (déjà trié).
  • 📦 Tri stable (ne change pas l'ordre des éléments égaux).

🎯 Tri par sélection (selection sort)

Principe : rechercher l'élément minimum (ou maximum) dans la partie non triée et l’échanger avec le premier élément de cette partie.

Pseudo-code Pour i de 0 à n-2 faire:
    min_idx ← i
    Pour j de i+1 à n-1 faire:
        Si T[j] < T[min_idx] alors min_idx ← j
    Échanger T[i] et T[min_idx]
  • ✅ Simple à comprendre, nombre d’échanges limité (au maximum n-1).
  • ⏱️ Complexité toujours O(n²) quel que soit l’ordre initial.
  • 📦 Non stable (peut changer l'ordre des éléments égaux).

⚖️ Comparaison visuelle

🔹 Insertion
Rapide si quasi-trié
🔸 Sélection
Peu d'échanges
📈 Complexité
O(n²) tous deux

Pour un tableau de taille 10, les deux fonctionnent bien. Pour des milliers d’éléments, on préférera des tris plus efficaces (fusion, rapide) vus en supérieur.

💡 Exemples de compréhension

📘 Exemple 1 : Tri par insertion croissant

Tableau initial : [8, 3, 6, 2, 9]. Appliquer l’algorithme d’insertion pas à pas.

Déroulement :
  • i=1 (élément 3) : 3 < 8 → décalage → [3, 8, 6, 2, 9]
  • i=2 (élément 6) : 6 < 8, pas < 3 → insertion → [3, 6, 8, 2, 9]
  • i=3 (élément 2) : 2 < 8,2<6,2<3 → décalage complet → [2, 3, 6, 8, 9]
  • i=4 (élément 9) : déjà bien placé → [2, 3, 6, 8, 9]

📘 Exemple 2 : Tri par sélection croissant

Tableau : [29, 10, 14, 37, 13]. Appliquer la sélection.

Itérations :
  • i=0 : minimum global = 10 (indice 1) → échange T[0] et T[1] → [10, 29, 14, 37, 13]
  • i=1 : minimum parmi [29,14,37,13] = 13 (indice4) → échange T[1] et T[4] → [10, 13, 14, 37, 29]
  • i=2 : minimum parmi [14,37,29] = 14 déjà à sa place → [10,13,14,37,29]
  • i=3 : minimum parmi [37,29] = 29 → échange → [10,13,14,29,37]
Résultat final : ✅ [10, 13, 14, 29, 37]

⌨️ Traduction en langage C (aperçu)

Extraits importants pour le baccalauréat : savoir implémenter les boucles.

// Tri par insertion en C
void triInsertion(int T[], int n) {
   for(int i=1; i<n; i++) {
      int x = T[i]; int j = i-1;
      while(j>=0 && T[j] > x) { T[j+1] = T[j]; j--; }
      T[j+1] = x;
   }
}

// Tri par sélection en C
void triSelection(int T[], int n) {
   for(int i=0; i<n-1; i++) {
      int minIdx = i;
      for(int j=i+1; j<n; j++)
         if(T[j] < T[minIdx]) minIdx = j;
      int tmp = T[i]; T[i] = T[minIdx]; T[minIdx] = tmp;
   }
}
📍 Ces fragments peuvent tomber à l’examen (session 2022/2023/2024) : savoir compléter ou expliquer la logique.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Confondre tri par insertion et tri par sélection. L’insertion décale les éléments, la sélection échange directement le minimum.
  • Attention aux indices : En C, un tableau de taille n va de 0 à n-1. Dans les boucles Pour i de 1 à n-1 correspond à l’indice 1 jusqu’au dernier élément.
  • Comparaison strictement supérieur/inférieur : Pour un tri croissant, si T[j] > x on décale ; pour décroissant, inverser le signe.
  • Tri par sélection : Oublier d’échanger les valeurs (perte de données). Toujours utiliser une variable temporaire.
  • Complexité : Ne pas dire que le tri par insertion est toujours quadratique, il peut être linéaire si le tableau est déjà trié (cas favorable).
