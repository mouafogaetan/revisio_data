Leçon 20 : Requêtes avec jointures

Informatique · Terminale C & D · Implémentation des bases de données (SQL)

📌 Éléments clés à retenir

1. Qu'est-ce qu'une jointure ?

Une jointure est une opération SQL qui permet de combiner des données provenant de plusieurs tables en se basant sur des colonnes communes (généralement la clé primaire d'une table et la clé étrangère d'une autre).

  • Les jointures évitent la redondance des données dans la base.
  • Elles permettent de répondre à des questions complexes nécessitant l'information de plusieurs tables.
  • La condition de jointure spécifie comment les tables sont reliées (ex: table1.cle = table2.cle_etrangere).

2. Types de jointures

Voici les jointures les plus utilisées en SQL :

  • INNER JOIN : Ne garde que les lignes qui ont une correspondance dans les deux tables.
  • LEFT JOIN (ou LEFT OUTER JOIN) : Toutes les lignes de la table de gauche, avec les correspondances de la table de droite (ou NULL si aucune).
  • RIGHT JOIN : Toutes les lignes de la table de droite, avec les correspondances de la gauche.
  • FULL JOIN : Toutes les lignes des deux tables, avec NULL là où il n'y a pas de correspondance.

3. Syntaxe générale

SELECT colonnes
FROM table1
[INNER | LEFT | RIGHT] JOIN table2
ON table1.colonne = table2.colonne;
[WHERE condition]
[ORDER BY colonne];

La condition ON est essentielle : elle définit la clé de jointure (souvent clé primaire / clé étrangère).

💡 Exemples de compréhension

Contexte : Base "École"

Deux tables : eleves(id_eleve, nom, classe) et notes(id_note, id_eleve, matiere, note). id_eleve est clé étrangère dans notes.

Exemple 1 : INNER JOIN
Question : Afficher le nom de chaque élève ainsi que sa note en Mathématiques.
SELECT eleves.nom, notes.note
FROM eleves
INNER JOIN notes ON eleves.id_eleve = notes.id_eleve
WHERE notes.matiere = 'Mathématiques';

Explication : Seuls les élèves ayant une note en Maths (correspondance dans notes) apparaissent. L'INNER JOIN filtre les élèves sans note.

Exemple 2 : LEFT JOIN
Question : Lister tous les élèves avec leur note en Physique, y compris ceux qui n'ont pas encore de note.
SELECT eleves.nom, notes.note
FROM eleves
LEFT JOIN notes ON eleves.id_eleve = notes.id_eleve AND notes.matiere = 'Physique';

Explication : Tous les élèves sont affichés. Ceux sans note en Physique auront NULL dans la colonne note.

💡 Astuce : Pour ne pas oublier la condition de jointure, posez-vous la question : "Quelle colonne relie logiquement ces deux tables ?" Utilisez des alias pour raccourcir les noms : FROM eleves e JOIN notes n ON e.id_eleve = n.id_eleve.

Jointure de plusieurs tables

Ajout d'une troisième table matieres(id_matiere, nom_matiere, coefficient). La table notes contient id_matiere au lieu de matiere (texte).

Requête : Obtenir le nom de l'élève, le nom de la matière et la note.
SELECT e.nom, m.nom_matiere, n.note
FROM eleves e
JOIN notes n ON e.id_eleve = n.id_eleve
JOIN matieres m ON n.id_matiere = m.id_matiere;

Explication : On enchaîne les jointures. JOIN seul est équivalent à INNER JOIN. L'ordre des tables peut être optimisé mais le résultat est le même.

📌 Pour l'examen : On attend souvent une jointure entre deux tables avec la clause ON bien précisée. N'oubliez pas les préfixes (nom_table.colonne) quand les colonnes ont le même nom.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier la condition ON → cela produit un produit cartésien (toutes les combinaisons), souvent très volumineux et faux.
  • Attention : Confondre LEFT JOIN et INNER JOIN. LEFT JOIN conserve toutes les lignes de la première table, INNER JOIN ne garde que les correspondances.
  • Piège : Ne pas utiliser d'alias et répéter de longs noms de table, ce qui rend la requête difficile à lire. Exemple : FROM employes AS e.
  • Souvent sous-estimé : Une jointure peut utiliser plusieurs conditions dans le ON (ex: ON table1.id = table2.id AND table1.annee = table2.annee).
