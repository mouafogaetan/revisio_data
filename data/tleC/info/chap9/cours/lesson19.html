Leçon 19 : Requêtes d'interrogation (SELECT)

Informatique · Terminale C & D · SQL - Langage de Manipulation de Données (DML)

📌 Éléments clés à retenir

1. Structure de base de la requête SELECT

La commande SELECT est utilisée pour extraire (interroger) des données d'une base de données. Sa structure de base est : SELECT [colonnes] FROM [nom_table] WHERE [condition] ORDER BY [colonne] [ASC/DESC];

  • Projection : Choisir les colonnes à afficher (avec SELECT).
  • Sélection : Filtrer les lignes (avec WHERE).
  • Tri : Ordonner les résultats (avec ORDER BY).
  • * : L'astérisque signifie "toutes les colonnes".

2. La clause WHERE et les opérateurs de comparaison

La clause WHERE permet de filtrer les enregistrements selon une condition logique. Les opérateurs de comparaison courants sont :

  • = : Égal à
  • > : Strictement supérieur à
  • < : Strictement inférieur à
  • >= : Supérieur ou égal
  • <= : Inférieur ou égal
  • <> ou != : Différent de
  • AND, OR, NOT : Opérateurs logiques pour combiner les conditions.
💡 Astuce : Pour les chaînes de caractères (VARCHAR, TEXT), les valeurs doivent être entourées de guillemets simples ' '. Exemple : WHERE nom = 'DIOP'.

3. La clause ORDER BY (Tri)

La clause ORDER BY permet de trier les résultats d'une requête.

  • ORDER BY colonne ASC : Tri par ordre croissant (par défaut, ASC peut être omis).
  • ORDER BY colonne DESC : Tri par ordre décroissant.
  • On peut trier sur plusieurs colonnes : ORDER BY colonne1 ASC, colonne2 DESC.

💡 Exemples de compréhension

Exemple 1 : Projection et sélection simple

Soit la table Eleves (id, nom, prenom, classe, moyenne). Écrire la requête pour afficher le nom et la moyenne des élèves de la classe 'Terminale C'.

Requête SQL :
SELECT nom, moyenne FROM Eleves WHERE classe = 'Terminale C';

💬 Explication : On projette les colonnes 'nom' et 'moyenne' depuis la table 'Eleves' et on ne sélectionne que les lignes où la colonne 'classe' vaut 'Terminale C'.

Exemple 2 : Utilisation d'opérateurs logiques (AND)

À partir de la même table Eleves, afficher l'identifiant et le nom des élèves ayant une moyenne supérieure à 12 et appartenant à la classe 'Terminale D'.

Requête SQL :
SELECT id, nom FROM Eleves WHERE moyenne > 12 AND classe = 'Terminale D';

💡 Penser à vérifier les types de données : 'moyenne' est un nombre (pas de guillemets), 'classe' est une chaîne (guillemets simples).

Exemple 3 : Tri des résultats (ORDER BY)

Afficher le nom, la classe et la moyenne de tous les élèves, triés par moyenne décroissante. En cas d'égalité, trier par ordre alphabétique du nom.

Requête SQL :
SELECT nom, classe, moyenne FROM Eleves ORDER BY moyenne DESC, nom ASC;

💬 Astuce : L'ordre des colonnes dans ORDER BY est important : on trie d'abord par moyenne DESC, puis par nom ASC.

Exemple 4 : Utilisation de l'astérisque (*) et de WHERE

Afficher toutes les informations des élèves dont la moyenne est strictement inférieure à 10 (ceux en dessous de la moyenne).

Requête SQL :
SELECT * FROM Eleves WHERE moyenne < 10;

💡 L'astérisque * est pratique mais en production, il est préférable de lister explicitement les colonnes pour des raisons de performance.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier les guillemets simples autour des valeurs de type texte (VARCHAR, CHAR). Exemple : WHERE ville = Yaoundé est faux, il faut WHERE ville = 'Yaoundé'.
  • Attention : Confondre le signe = (comparaison) avec = = (inexistant en SQL standard). Pour tester l'égalité, on utilise un seul =.
  • Piège : L'ordre des clauses est important : SELECT ... FROM ... WHERE ... ORDER BY .... On ne peut pas inverser WHERE et ORDER BY.
  • Ne pas confondre : La clause WHERE filtre les lignes avant le tri (ORDER BY), tandis que HAVING (non vu ici) filtre après regroupement.
  • Attention à la casse : En fonction du SGBD (MySQL, SQLite, PostgreSQL), les noms de colonnes et de tables peuvent être sensibles à la casse. Toujours vérifier l'orthographe exacte.

✏️ Exercice d'auto-évaluation

Entraînement : Base de données "Bibliothèque"

Table Livres : (id_livre, titre, auteur, annee_publication, prix, quantite).
Écrire les requêtes SQL suivantes :

  1. Afficher tous les titres et prix des livres de l'auteur 'Camara Laye'.
  2. Afficher les titres des livres publiés après 2010 et dont le prix est inférieur à 5000 FCFA.
  3. Afficher l'id, le titre et l'année de publication de tous les livres triés de l'année la plus récente à la plus ancienne.
  4. Afficher toutes les informations des livres dont la quantité est différente de 0.
Voir les solutions

1. SELECT titre, prix FROM Livres WHERE auteur = 'Camara Laye';
2. SELECT titre FROM Livres WHERE annee_publication > 2010 AND prix < 5000;
3. SELECT id_livre, titre, annee_publication FROM Livres ORDER BY annee_publication DESC;
4. SELECT * FROM Livres WHERE quantite <> 0;

💡 Après avoir écrit les requêtes, exécutez-les mentalement ou testez-les sur un logiciel comme SQLite pour vérifier votre compréhension.
