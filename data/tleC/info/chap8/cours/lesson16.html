Leçon 16 : Création de tables (DDL)

Bases de données - SQL (Langage de Définition des Données) · Terminale C & D

📌 Éléments clés à retenir

1. La commande CREATE TABLE

La commande CREATE TABLE appartient au langage de définition des données (DDL). Elle permet de créer une nouvelle table dans la base de données. On y définit le nom de la table, la liste des colonnes (attributs) avec leur type de données et éventuellement des contraintes d'intégrité.

  • Syntaxe générale : CREATE TABLE nom_table (colonne1 type [contraintes], colonne2 type, ...);
  • Chaque colonne est séparée par une virgule.
  • Les noms de tables et colonnes doivent être significatifs et sans espaces (on utilise souvent le snake_case).
  • Exemple concret : CREATE TABLE Eleve (id INT, nom VARCHAR(50), prenom VARCHAR(50));

2. Les types de données courants en SQL

Chaque colonne doit être associée à un type de données qui détermine la nature des valeurs qu'elle peut contenir. Voici les plus utilisés au baccalauréat :

Type SQLDescriptionExemple
INT ou INTEGERNombre entier (positif ou négatif)age, numéro, identifiant
VARCHAR(n)Chaîne de caractères de longueur variable (max n)nom, adresse, téléphone
CHAR(n)Chaîne de longueur fixe (n caractères)code postal (ex: '75000')
DATEStocke une date (format AAAA-MM-JJ)date_naissance, date_inscription
DECIMAL(p,s) ou FLOATNombre décimal (p = précision, s = échelle)moyenne, prix
BOOLEANVrai ou Faux (TRUE/FALSE)inscrit, valide

À retenir : Pour les textes courts (noms, prénoms), on utilise VARCHAR(50) ou VARCHAR(100). Pour les identifiants numériques, on utilise INT.

3. Contraintes d'intégrité (Clé primaire, Clé étrangère, NOT NULL)

Les contraintes garantissent la cohérence et l'intégrité des données. Elles se placent après le type de la colonne.

  • PRIMARY KEY : identifiant unique d'une table. Une table ne peut avoir qu'une seule clé primaire. Elle ne peut pas contenir de valeur NULL.
  • FOREIGN KEY : référence la clé primaire d'une autre table. Elle assure l'intégrité référentielle (pas d'orphelin).
  • NOT NULL : impose qu'une colonne ne puisse pas être vide (obligatoire).
  • UNIQUE : garantit que toutes les valeurs d'une colonne sont différentes (ex: email).
  • DEFAULT : définit une valeur par défaut si aucune valeur n'est fournie.
CREATE TABLE Etudiant (
   id_etudiant INT PRIMARY KEY,
   nom VARCHAR(50) NOT NULL,
   email VARCHAR(100) UNIQUE,
   date_naissance DATE
);

4. Définition des clés étrangères

Une clé étrangère se déclare au niveau de la colonne ou en fin de commande CREATE TABLE avec la syntaxe FOREIGN KEY (colonne) REFERENCES autre_table(colonne_ref).

CREATE TABLE Inscription (
   id_inscription INT PRIMARY KEY,
   date_inscription DATE,
   id_etudiant INT,
   FOREIGN KEY (id_etudiant) REFERENCES Etudiant(id_etudiant)
);

Cela signifie que id_etudiant dans la table Inscription doit exister dans la colonne id_etudiant de la table Etudiant.

💡 Exemples de compréhension (inspirés des sujets du Bac)

Examen blanc : Création d'une table "Professeur"

Énoncé : On souhaite créer une table Professeur pour stocker les informations suivantes : un identifiant unique (entier), le nom (chaîne de 50 caractères max), le prénom (VARCHAR(50)), la spécialité (VARCHAR(30)), et l'année de recrutement (entier). La colonne "nom" ne doit jamais être vide. L'identifiant est la clé primaire.

Requête SQL :
CREATE TABLE Professeur (
   id_prof INT PRIMARY KEY,
   nom VARCHAR(50) NOT NULL,
   prenom VARCHAR(50),
   specialite VARCHAR(30),
   annee_recrutement INT
);

Explication : La clé primaire id_prof garantit l'unicité de chaque professeur. La contrainte NOT NULL sur la colonne nom assure qu'on enregistre toujours le nom. Les autres colonnes peuvent être NULL si l'information est manquante.

💬 Astuce : Toujours nommer la clé primaire avec le préfixe "id_" suivi du nom de la table (ex: id_professeur) pour plus de clarté.

Modélisation MCD → MLD → SQL (Jointure implicite)

D'après le MCD suivant : une Commande est passée par un Client. Un client peut avoir plusieurs commandes, mais une commande appartient à un seul client. On a les attributs : Client(num_client, nom, ville) et Commande(num_cmd, date_cmd, total). La clé étrangère est num_client dans la table Commande.

Création des tables :
CREATE TABLE Client (
   num_client INT PRIMARY KEY,
   nom VARCHAR(50) NOT NULL,
   ville VARCHAR(50)
);

CREATE TABLE Commande (
   num_cmd INT PRIMARY KEY,
   date_cmd DATE NOT NULL,
   total DECIMAL(10,2),
   num_client INT,
   FOREIGN KEY (num_client) REFERENCES Client(num_client)
);

Analyse : La clé étrangère num_client dans la table Commande fait référence à la clé primaire de la table Client. Cela garantit qu'on ne peut pas créer une commande avec un client inexistant.

💡 Pensez à l'ordre de création : d'abord la table référencée (Client), puis celle qui contient la clé étrangère (Commande). Sinon, SQL génère une erreur.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier de déclarer la clé primaire ou la clé étrangère, ce qui fragilise l'intégrité des données. Toujours spécifier PRIMARY KEY pour l'identifiant unique.
  • Confusion : VARCHAR vs CHAR. VARCHAR économise de l'espace (taille variable) tandis que CHAR occupe toujours n caractères. Pour les noms, préférez VARCHAR.
  • Attention : Ne pas mettre de virgule après la dernière colonne ou la dernière contrainte dans la commande CREATE TABLE.
  • Piège : Créer une table avec une clé étrangère avant la table référencée → provoque une erreur. Ordre obligatoire : d'abord la table "parent", puis la table "enfant".
  • Oubli fréquent : Ne pas indiquer la taille pour VARCHAR (ex: VARCHAR seul au lieu de VARCHAR(50)). La taille est obligatoire dans la plupart des SGBD (MySQL, SQLite, etc.).

📊 Schéma récapitulatif : Structure d'une table SQL

Légende : Une table est un ensemble de lignes (enregistrements) et de colonnes (attributs). Chaque colonne possède un nom, un type et des contraintes. La clé primaire identifie chaque ligne de manière unique.
