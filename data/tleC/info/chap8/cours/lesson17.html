Leçon 17 : Modification de la structure d'une table (ALTER TABLE)

SQL - Langage de Définition des Données (DDL) · Terminale C & D · Baccalauréat

📌 Éléments clés à retenir

1. La commande ALTER TABLE

En SQL, une fois qu'une table a été créée avec CREATE TABLE, il est souvent nécessaire d'en modifier la structure sans perdre les données existantes. La commande ALTER TABLE permet d'ajouter, de modifier ou de supprimer des colonnes, ainsi que de renommer une table ou une colonne.

  • Syntaxe générale : ALTER TABLE nom_table action;
  • Principales actions : ADD (ajouter colonne), MODIFY / ALTER COLUMN (modifier type ou contrainte), DROP COLUMN (supprimer colonne).
  • Compatibilité : MySQL utilise MODIFY COLUMN ; PostgreSQL utilise ALTER COLUMN ... TYPE. Au baccalauréat, on retient ADD, MODIFY et DROP COLUMN.

2. Ajouter une colonne (ADD)

Pour ajouter une nouvelle colonne à une table existante, on utilise la clause ADD suivie de la définition de la colonne (nom, type, éventuellement contrainte).

ALTER TABLE eleves
ADD COLUMN email VARCHAR(100) NOT NULL;
  • La nouvelle colonne s'ajoute en dernière position (ou on peut préciser AFTER colonne dans certains SGBD).
  • On peut ajouter plusieurs colonnes en une seule commande : ADD (col1 type, col2 type).

3. Modifier une colonne (MODIFY / ALTER COLUMN)

Permet de changer le type de données, la taille ou une contrainte d'une colonne existante. Attention : certaines modifications peuvent échouer si les données existantes ne sont pas compatibles.

-- Exemple : augmenter la taille d'une colonne 'nom'
ALTER TABLE eleves
MODIFY COLUMN nom VARCHAR(150);

-- Changer le type d'une colonne 'age' en SMALLINT
ALTER TABLE eleves
MODIFY COLUMN age SMALLINT;
  • ⚠️ Précaution : Modifier un type (ex: de INT vers VARCHAR) peut entraîner une perte de données si la conversion est impossible.
  • On peut aussi ajouter ou supprimer une contrainte NOT NULL avec MODIFY.

4. Supprimer une colonne (DROP COLUMN)

Pour supprimer définitivement une colonne et toutes ses données. Opération irréversible sans sauvegarde.

ALTER TABLE eleves
DROP COLUMN adresse_temporaire;
  • La commande supprime à la fois la structure et les valeurs.
  • Certains SGBD n'autorisent pas la suppression d'une colonne référencée par une clé étrangère sans supprimer la contrainte d'abord.

5. Renommer une table ou une colonne

Pour renommer une table : RENAME TABLE ancien_nom TO nouveau_nom (MySQL) ou ALTER TABLE ancien_nom RENAME TO nouveau_nom.
Pour renommer une colonne : ALTER TABLE table CHANGE ancien_nom nouveau_nom type (MySQL) ou RENAME COLUMN (SQL standard).

-- Renommer une table
ALTER TABLE etudiants RENAME TO apprenants;

-- Renommer une colonne (MySQL)
ALTER TABLE apprenants CHANGE COLUMN telephone contact VARCHAR(20);

💡 Exemples de compréhension (inspirés du Bac)

Examen blanc : Modification d'une table "Produits"

Une entreprise possède une table nommée Produits avec les colonnes : id_produit INT PRIMARY KEY, nom_produit VARCHAR(50), prix_unitaire DECIMAL(10,2). L'entreprise souhaite :
1. Ajouter une colonne stock_disponible INT avec une valeur par défaut 0.
2. Modifier la colonne nom_produit pour qu'elle puisse contenir jusqu'à 100 caractères.
3. Supprimer la colonne prix_unitaire (devenu inutile car nouveau système de tarification).

Correction SQL :

-- 1. Ajout de la colonne stock_disponible
ALTER TABLE Produits
ADD COLUMN stock_disponible INT DEFAULT 0;

-- 2. Extension de la taille de nom_produit
ALTER TABLE Produits
MODIFY COLUMN nom_produit VARCHAR(100);

-- 3. Suppression de la colonne prix_unitaire
ALTER TABLE Produits
DROP COLUMN prix_unitaire;
💡 Astuce : Utilisez toujours DESC nom_table; après chaque modification pour vérifier la nouvelle structure.

Cas pratique : Ajout de contrainte d'intégrité

La table Commandes contient une colonne client_id de type INT. On souhaite que cette colonne soit obligatoire (NOT NULL) après coup, car on s'est rendu compte que toute commande doit être liée à un client.

Requête SQL :

ALTER TABLE Commandes
MODIFY COLUMN client_id INT NOT NULL;

Remarque : Si des lignes contiennent déjà des NULL dans cette colonne, la modification échouera. Il faudra d'abord mettre à jour ces enregistrements avec une valeur valide (UPDATE) avant d'appliquer ALTER TABLE.

⚠️ Piège fréquent au Bac : Oublier que MODIFY COLUMN peut supprimer des contraintes existantes si elles ne sont pas réécrites. Toujours spécifier toutes les caractéristiques souhaitées.

⚠️ Pièges courants & À ne pas confondre

  • Erreur fréquente : Utiliser ALTER TABLE ... DROP COLUMN sans sauvegarde préalable → perte définitive des données de cette colonne.
  • Confusion ADD / MODIFY : ADD sert à créer une nouvelle colonne ; MODIFY sert à modifier une colonne existante (type, taille). Ne pas les inverser.
  • Problème de dépendances : Si une colonne est référencée par une clé étrangère, il faut supprimer la contrainte avant de supprimer ou modifier la colonne.
  • Attention au type de SGBD : Oracle utilise MODIFY sans le mot COLUMN, SQL Server utilise ALTER COLUMN. Au baccalauréat, on utilise la syntaxe standard ALTER TABLE ... MODIFY COLUMN (MySQL).
  • Ne pas oublier : Une colonne ajoutée avec ADD reçoit la valeur NULL pour tous les enregistrements existants, sauf si une valeur par défaut est précisée (DEFAULT).

📖 Mémo SQL : Syntaxes ALTER TABLE (à connaître pour l'épreuve)

Action Syntaxe (MySQL/Standard)
Ajouter colonneALTER TABLE table ADD colonne type [contrainte];
Modifier colonneALTER TABLE table MODIFY COLUMN colonne nouveau_type;
Supprimer colonneALTER TABLE table DROP COLUMN colonne;
Renommer tableALTER TABLE ancien RENAME TO nouveau; ou RENAME TABLE ancien TO nouveau;
Renommer colonne (MySQL)ALTER TABLE table CHANGE ancien_nom nouveau_nom type;
🔁 Pensez à vérifier la structure avec DESCRIBE nom_table; ou SHOW COLUMNS FROM nom_table; avant et après les modifications.
