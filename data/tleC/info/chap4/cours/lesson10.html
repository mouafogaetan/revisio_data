Leçon 10 : Boucles en C

Informatique · Terminale C & D · Programmation en Langage C

📌 Éléments clés à retenir

Pourquoi des boucles ?

Les boucles permettent de répéter un bloc d'instructions plusieurs fois sans écrire le code en double. Elles sont essentielles pour traiter des séries de données, calculer des sommes, afficher des motifs ou attendre une condition.

La boucle for

Utilisée quand on connaît à l'avance le nombre d'itérations. Syntaxe : 

for (initialisation; condition; incrémentation) {
    // instructions à répéter
}
  • initialisation : exécutée une seule fois au début (ex: int i = 0;).
  • condition : testée avant chaque itération ; si vraie, la boucle continue.
  • incrémentation : exécutée après chaque itération (ex: i++ ou i = i + 1).
  • Exemple typique : parcourir un tableau de taille N.

La boucle while

Boucle conditionnelle avec test en tête. Le nombre d'itérations n'est pas forcément connu à l'avance. 

while (condition) {
    // instructions
    // penser à modifier la condition pour éviter la boucle infinie
}
  • La condition est évaluée avant chaque itération.
  • Si la condition est fausse dès le départ, le bloc n'est jamais exécuté.
  • Idéal pour : "Tant que l'utilisateur n'entre pas une valeur valide, redemander".

La boucle do...while

Variante avec test en sortie : le bloc est exécuté au moins une fois. 

do {
    // instructions
} while (condition);   // Notez le point-virgule final
  • Utile lorsqu'on veut garantir une exécution minimale (ex: afficher un menu avant de tester la sortie).
  • Attention à ne pas oublier le ; après le while(...).

💡 Exemples de compréhension

Exemple 1 : Somme des entiers avec for

Calculer la somme des entiers de 1 à N (N saisi par l'utilisateur).

#include <stdio.h>
int main() {
    int N, somme = 0;
    printf("Entrez N : ");
    scanf("%d", &N);
    for(int i = 1; i <= N; i++) {
        somme = somme + i;
    }
    printf("Somme = %d\n", somme);
    return 0;
}

Explication : La variable i est initialisée à 1, puis à chaque tour on ajoute i à somme et on incrémente i. La boucle s'arrête quand i dépasse N.

💡 Astuce : i++ équivaut à i = i + 1. On peut aussi décrémenter avec i--.

Exemple 2 : Validation de saisie avec while

Demander un nombre positif à l'utilisateur (redemander tant que la saisie est négative).

#include <stdio.h>
int main() {
    int nb;
    printf("Entrez un nombre positif : ");
    scanf("%d", &nb);
    while(nb < 0) {
        printf("Erreur, re-saisir : ");
        scanf("%d", &nb);
    }
    printf("Merci, vous avez saisi %d\n", nb);
    return 0;
}

Analyse : On teste d'abord la saisie. Si nb est négatif, on entre dans la boucle pour redemander. Si la première saisie est correcte, la boucle est ignorée.

💬 Pensez à toujours prévoir une issue à la boucle pour éviter les boucles infinies (modifier la variable testée).

Exemple 3 : do...while pour afficher un menu

Afficher un menu et effectuer une action au moins une fois.

#include <stdio.h>
int main() {
    int choix;
    do {
        printf("\n1. Jouer\n2. Quitter\nVotre choix : ");
        scanf("%d", &choix);
        if(choix == 1) printf("Début du jeu...\n");
    } while(choix != 2);
    return 0;
}

Pourquoi do...while ? Le menu doit s'afficher au moins une fois avant de tester si l'utilisateur veut quitter. Avec while, il faudrait dupliquer l'affichage.

🎯 À retenir : Le do...while se termine toujours par un point-virgule après la condition.

⚠️ Pièges courants / À ne pas confondre

  • Boucle infinie involontaire : Oublier l'incrémentation dans un while ou une condition toujours vraie (while(1) sans break).
  • Confondre = et == : Dans une condition, utiliser == pour comparer, = pour affecter. while(i = 5) est une boucle infinie et souvent une erreur logique.
  • Point-virgule après while(...); Écrire while(i<10); avec un point-virgule crée une boucle vide infinie (le bloc suivant n'est pas répété).
  • Ne pas initialiser le compteur : Une variable utilisée comme compteur doit être initialisée avant la boucle while ou dans la première partie du for.
  • Dépassement de tableau : Dans un for qui parcourt un tableau, la condition doit être i < taille et non i <= taille (indice valide de 0 à taille-1).
