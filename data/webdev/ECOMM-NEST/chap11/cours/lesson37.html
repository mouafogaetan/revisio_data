💡 Pourquoi ? Un e-commerce sans logs, c'est comme un avion sans boîte noire : impossible de diagnostiquer les pannes ou fraudes.

📦 Choisir son transporteur de logs

📘Winston

Plus mature, très configurable. Supporte plusieurs transports (fichier, console, HTTP, Sentry).

new winston.Logger({
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.Console()
  ]
})
⚡Pino

Ultra rapide, format JSON natif, faible overhead. Idéal pour les API à fort trafic.

const pino = require('pino')
const logger = pino({ level: 'info' })
logger.info({ user: 'admin' }, 'action réalisée')

🔍 Dans un projet NestJS, on utilise souvent @nestjs/common avec Winston personnalisé via un module dédié.

📊 Niveaux de logs (RFC 5424)

errorErreurs système, paiement échoué, panne BDD
warnStock bas, tentative d'accès non autorisé
infoCréation commande, connexion utilisateur, envoi email
debugDétails techniques (requêtes SQL, payloads) → développement uniquement
🚨 Actions critiques à logger OBLIGATOIREMENT : paiement réussi/refusé, suppression de compte, changement de rôle, modification de stock par admin, tentative de connexion échouée.

⚙️ Implémentation Winston dans un service NestJS

// logger.config.ts
import { WinstonModule } from 'nest-winston';
import * as winston from 'winston';

export const winstonConfig = WinstonModule.createLogger({
  transports: [
    new winston.transports.Console({ format: winston.format.simple() }),
    new winston.transports.File({ filename: 'logs/error.log', level: 'error' }),
    new winston.transports.File({ filename: 'logs/combined.log' }),
  ],
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
});

// Dans un service
@Injectable()
export class PaymentService {
  private readonly logger = new Logger(PaymentService.name);

  async processPayment(amount: number) {
    try {
      // logique paiement...
      this.logger.log(`Paiement de ${amount} FCFA effectué`);
    } catch (err) {
      this.logger.error(`Échec paiement: ${err.message}`, err.stack);
    }
  }
}

🗃️ Rotation & rétention des logs

Avec winston-daily-rotate-file : découpage automatique par jour/taille.

const DailyRotateFile = require('winston-daily-rotate-file');
new DailyRotateFile({
  filename: 'logs/application-%DATE%.log',
  datePattern: 'YYYY-MM-DD',
  maxSize: '20m',
  maxFiles: '14d'   // garder 14 jours
})
sécurité conformité audit

💾 Les logs fichiers doivent être protégés en écriture et sauvegardés régulièrement.

🚨 Détection d'erreurs avec Sentry

Sentry agrège les erreurs, capture le contexte et notifie les équipes.

// main.ts
import * as Sentry from '@sentry/node';
Sentry.init({ dsn: process.env.SENTRY_DSN, environment: 'production' });

// Interceptor global
@Injectable()
export class SentryInterceptor implements NestInterceptor {
  intercept(ctx: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(catchError((err) => {
      Sentry.captureException(err, { tags: { handler: ctx.getHandler().name } });
      return throwError(() => err);
    }));
  }
}
✅ Sentry capture automatiquement les erreurs non gérées et fournit une interface d'analyse.

🔒 Bonnes pratiques de logging

  • ❌ Ne jamais logguer des mots de passe, tokens ou données bancaires
  • ✅ Anonymiser les informations personnelles (email → haché partiel)
  • ✅ Ajouter un identifiant de corrélation (`correlationId`) pour tracer une requête utilisateur
  • ✅ Utiliser des variables d'environnement pour désactiver `debug` en production
  • ✅ Centraliser les logs via ELK (Elasticsearch, Logstash, Kibana) ou Datadog

🏪 Cas concret : logging sur une commande

async checkout(cartId: number, userId: number) {
  this.logger.log(`Début checkout - user ${userId}, panier ${cartId}`);
  try {
    const order = await this.orderService.createFromCart(cartId, userId);
    this.logger.info(`Commande ${order.id} créée, montant: ${order.total} FCFA`);
    // envoi email...
    return order;
  } catch (error) {
    this.logger.error(`Checkout échoué pour user ${userId}`, error.stack);
    throw new InternalServerErrorException();
  }
}
📈 Résultat : traçabilité complète, diagnostic rapide des pannes.
