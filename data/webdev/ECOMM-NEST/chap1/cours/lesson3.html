🔄 Modules partagés

Réutilisables dans toute l’application :

📁 CommonModule – gardes (JwtAuthGuard, RolesGuard), intercepteurs, filtres d’exception, décorateurs génériques.
🗄️ DatabaseModule – configuration TypeORM, repositories personnalisés, gestion des migrations.
⚙️ ConfigModule – variables d’environnement, validation Joi, chargement des fichiers .env.
✅ Ces modules sont importés dans le module principal (AppModule) et exportés vers les modules métier.

🏛️ Architecture en couches

Chaque domaine suit le pattern classique NestJS :

🎯 Controller

Gère les requêtes HTTP, validation des entrées, retourne les réponses. Décorateurs @Get, @Post, @Body().

🧠 Service

Logique métier : calculs, appels aux repositories, gestion des transactions.

📦 Repository

Interaction avec la base de données (TypeORM). Méthodes CRUD personnalisées.

📄 DTO

Data Transfer Object : définit la forme des données entrantes/sortantes, validation avec class-validator.

✅ Séparation claire : Contrôleur → Service → Repository → Entité

🔍 Exemple : module Products

// product.dto.ts
export class CreateProductDto {
  @IsString() name: string;
  @IsNumber() price: number;
  @IsOptional() @IsString() description?: string;
}

// product.controller.ts
@Controller('products')
export class ProductController {
  constructor(private productService: ProductService) {}
  @Post()
  create(@Body() dto: CreateProductDto) {
    return this.productService.create(dto);
  }
}

// product.service.ts (utilise le repository)
@Injectable()
export class ProductService {
  constructor(@InjectRepository(Product) private repo: Repository<Product>) {}
  async create(dto: CreateProductDto) {
    const product = this.repo.create(dto);
    return this.repo.save(product);
  }
}

📌 Le DTO valide les données, le contrôleur délègue au service, le service utilise le repository TypeORM.

✨ Pourquoi adopter cette architecture ?

🧩 Modularité – chaque domaine évolue indépendamment.
🔁 Réutilisabilité – les modules partagés évitent la duplication.
🧪 Testabilité – couches découplées, mocks faciles.
📈 Scalabilité – on peut ajouter des microservices plus tard.

🧠 À retenir :

Dans un projet e-commerce, la séparation en domaines (catalogue, panier, commande) + modules partagés (auth, config, base de données) facilite le travail en équipe et la maintenance à long terme.
