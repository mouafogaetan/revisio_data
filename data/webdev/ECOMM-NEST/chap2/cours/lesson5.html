Configuration de l'environnement

@nestjs/config, fichiers .env et validation Joi

📘 Lesson 5 · Mettre en place l'environnement de développement

🎯 Objectifs de la leçon

1️⃣ Mettre en place @nestjs/config : installer et configurer le module de configuration.
2️⃣ Créer les fichiers .env : structurer les environnements (dev, staging, prod).
3️⃣ Valider les variables avec Joi : garantir l'intégrité des configurations au démarrage.

🔐 Pourquoi gérer la configuration ?

Une API moderne doit s'adapter à différents environnements sans modifier le code source.

💻

Développement

Base de données locale, logs détaillés, debug activé

🧪

Staging (test)

Base de données de test, API externes sandbox

🚀

Production

Données réelles, sécurité maximale, performances

✅ Bonne pratique : Ne jamais stocker de secrets (clés API, mots de passe) dans le code. Utilisez .env et ajoutez ce fichier au .gitignore.

📦 Installation du module de configuration

@nestjs/config est le module officiel pour gérer les variables d'environnement.

npm install @nestjs/config

Importation dans le module principal :

// app.module.ts
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,  // Rend ConfigService disponible partout
      envFilePath: '.env',
    }),
  ],
})
export class AppModule {}

💡 isGlobal: true évite d'importer ConfigModule dans chaque module.

📁 Structure des fichiers d'environnement

Organisation recommandée pour un projet e-commerce :

📄 .env # développement local (non versionné)
📄 .env.example # modèle des variables (versionné)
📄 .env.staging # environnement de test
📄 .env.production # environnement de production

Exemple de .env pour une API e-commerce :

# Base de données
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=secret123
DB_NAME=ecommerce_db

# Authentification
JWT_SECRET=super_secret_key_2024
JWT_EXPIRES_IN=7d

# Paiements
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Email
SMTP_HOST=smtp.gmail.com
SMTP_USER=no-reply@example.com
SMTP_PASSWORD=app_password

# Application
PORT=3000
NODE_ENV=development
API_URL=http://localhost:3000

📋 Le fichier .env.example

Ce fichier est versionné (commité) et sert de documentation pour les autres développeurs.

✅ Validation des variables avec Joi

Joi permet de valider les types et de détecter les variables manquantes au démarrage.

npm install joi

Configuration avec validation :

// app.module.ts
import * as Joi from 'joi';

ConfigModule.forRoot({
  isGlobal: true,
  validationSchema: Joi.object({
    // Database
    DB_HOST: Joi.string().required(),
    DB_PORT: Joi.number().default(5432),
    DB_USER: Joi.string().required(),
    DB_PASSWORD: Joi.string().required(),
    DB_NAME: Joi.string().required(),
    
    // JWT
    JWT_SECRET: Joi.string().min(32).required(),
    JWT_EXPIRES_IN: Joi.string().default('7d'),
    
    // Stripe
    STRIPE_SECRET_KEY: Joi.string().required(),
    
    // Serveur
    PORT: Joi.number().default(3000),
    NODE_ENV: Joi.string()
      .valid('development', 'staging', 'production')
      .default('development'),
  }),
})

🔧 Utiliser ConfigService dans l'application

Injectez ConfigService pour accéder aux variables d'environnement.

// product.service.ts
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class ProductService {
  constructor(private configService: ConfigService) {}
  
  getDatabaseConfig() {
    const host = this.configService.get('DB_HOST');
    const port = this.configService.get('DB_PORT');
    // Typage sécurisé avec fallback
    const isProd = this.configService.get('NODE_ENV') === 'production';
    return { host, port, isProd };
  }
  
  // Récupérer une variable avec typage
  getJwtSecret(): string {
    return this.configService.get<string>('JWT_SECRET', 'fallback_secret');
  }
}
💡 Astuce : Utilisez ConfigModule.forRoot({ validationOptions: { abortEarly: false } }) pour afficher toutes les erreurs de validation.

🛡️ Bonnes pratiques de sécurité

  • .gitignore obligatoire : ajoutez .env, .env.*.local pour ne jamais commiter vos secrets.
  • Variables sensibles : utilisez des gestionnaires de secrets (HashiCorp Vault, AWS Secrets Manager) en production.
  • Validation stricte : Joi doit être configuré pour rejeter les variables inconnues (unknown: false).
  • Différenciation des environnements : NODE_ENV=production désactive les logs verbose et active les optimisations.
// Validation avancée
validationSchema: Joi.object({ ... }).unknown(false),  // Empêche les variables inattendues

// Chargement différencié
envFilePath: [`.env.${process.env.NODE_ENV}`, '.env'],
