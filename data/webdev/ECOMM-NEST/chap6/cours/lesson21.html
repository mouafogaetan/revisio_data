📋 Récupérer l’historique des commandes

Endpoint GET /orders/my-orders (authentifié) retourne les commandes de l’utilisateur connecté.

// order.controller.ts
@Get('my-orders')
@UseGuards(JwtAuthGuard)
async getUserOrders(@Req() req, @Query() paginationDto: PaginationDto) {
  return this.orderService.findByUser(req.user.id, paginationDto);
}

// order.service.ts
async findByUser(userId: number, { page, limit }: PaginationDto) {
  const [orders, total] = await this.orderRepo.findAndCount({
    where: { user: { id: userId } },
    relations: ['items', 'items.product'],
    order: { createdAt: 'DESC' },
    skip: (page - 1) * limit, take: limit
  });
  return { data: orders, total, page, limit };
}

📌 On inclut les items et le produit pour afficher les détails de la commande.

⏱️ Timeline des statuts

Ajouter des champs de date pour chaque transition de statut dans l’entité Order :

@Entity()
export class Order {
  @Column({ type: 'enum', enum: OrderStatus, default: OrderStatus.PENDING })
  status: OrderStatus;
  @Column({ nullable: true }) paidAt: Date;
  @Column({ nullable: true }) shippedAt: Date;
  @Column({ nullable: true }) deliveredAt: Date;
  @Column({ nullable: true }) cancelledAt: Date;
}

// Méthode de transition
async transitionTo(status: OrderStatus) {
  this.status = status;
  if (status === 'paid') this.paidAt = new Date();
  if (status === 'shipped') this.shippedAt = new Date();
  if (status === 'delivered') this.deliveredAt = new Date();
  if (status === 'cancelled') this.cancelledAt = new Date();
  await this.save();
}
💡 Timeline retournée au client : { pending: "2025-01-01", paid: "2025-01-01", shipped: "2025-01-03", delivered: null }

❌ Annulation d’une commande

Règles métier : seule une commande avec statut pending ou paid (non expédiée) peut être annulée. L’annulation doit restituer les stocks.

async cancelOrder(orderId: number, userId: number) {
  const order = await this.orderRepo.findOne({
    where: { id: orderId, user: { id: userId } },
    relations: ['items', 'items.product']
  });
  if (!order) throw new NotFoundException();
  if (order.status === 'shipped' || order.status === 'delivered') {
    throw new BadRequestException('Commande déjà expédiée, annulation impossible');
  }
  // Restituer les stocks
  for (const item of order.items) {
    item.product.stock += item.quantity;
    await this.productRepo.save(item.product);
  }
  order.status = 'cancelled';
  order.cancelledAt = new Date();
  return this.orderRepo.save(order);
}

⚠️ Si le paiement a déjà été effectué, il faut aussi déclencher un remboursement via Stripe/PayPal.

💰 Remboursement après annulation

Si la commande était déjà payée, on doit rembourser l’utilisateur via le fournisseur de paiement.

async refundOrder(order: Order) {
  if (order.paymentIntentId && order.status === 'paid') {
    await this.stripeService.refund(order.paymentIntentId);
    order.refundedAt = new Date();
  }
  return order;
}

✨ Bonne pratique :

Utiliser une transaction pour regrouper : restitution des stocks + mise à jour statut + remboursement (appel API externe). En cas d’échec, tout est annulé.

🔍 Filtres et tri avancés

Permettre à l’utilisateur de filtrer ses commandes par statut, période de date, etc.

// OrderFilterDto
export class OrderFilterDto {
  @IsOptional() @IsEnum(OrderStatus) status?: OrderStatus;
  @IsOptional() @IsDateString() startDate?: string;
  @IsOptional() @IsDateString() endDate?: string;
  @IsOptional() @IsString() sortBy?: 'createdAt' | 'total';
  @IsOptional() @IsIn(['ASC', 'DESC']) order?: 'ASC' | 'DESC';
}

// Construction dynamique du where
const where: any = { user: { id: userId } };
if (status) where.status = status;
if (startDate && endDate) {
  where.createdAt = Between(new Date(startDate), new Date(endDate));
}

🧪 Exercice :

Un client a passé une commande le 10 mars (statut "paid"). Le 12 mars, il demande l’annulation car il a changé d’avis. Le produit (stock initial 5, quantité commandée 2) n’a pas encore été expédié.

Décrivez les étapes techniques que l’API doit exécuter (séquences, transactions, appels externes).

🔍 Voir solution

1. Vérifier que la commande existe et appartient bien à l’utilisateur.
2. Vérifier que status = 'paid' (non expédié).
3. Démarrer une transaction.
4. Restituer le stock : product.stock += 2 (5 → 7).
5. Appeler Stripe pour remboursement (refund).
6. Mettre à jour order.status = 'cancelled', cancelledAt = now.
7. Commiter la transaction. Si une étape échoue, tout est annulé.
