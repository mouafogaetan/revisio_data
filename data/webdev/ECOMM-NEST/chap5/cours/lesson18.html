🔍 Vérification du stock avant ajout au panier

À chaque tentative d’ajout ou modification de quantité, on interroge le stock réel (variation de produit) et on bloque si quantité > stock disponible.

// Dans CartService
async addToCart(userId: number, variationId: number, quantity: number) {
  const variation = await this.variationRepo.findOne({ where: { id: variationId } });
  if (!variation) throw new NotFoundException('Variation inexistante');
  
  if (variation.stock < quantity) {
    throw new BadRequestException(`Stock insuffisant. Disponible : ${variation.stock}`);
  }
  
  // ... ajout au panier (pas encore de réservation permanente)
}

⚠️ Cette vérification simple évite d’ajouter plus que le stock, mais ne réserve pas le produit. Deux clients pourraient encore commander le même article simultanément.

🔒 Lock temporaire des articles (pendant checkout)

Lorsque l’utilisateur valide son panier (étape de paiement), on réserve le stock pour quelques minutes. Si le paiement échoue ou expire, on libère.

// Réservation temporaire avec Redis (TTL = 10 min)
async reserveStock(cart: Cart, ttlSeconds = 600) {
  for (const item of cart.items) {
    const key = `stock:lock:${item.productVariation.id}`;
    const reserved = await this.redisClient.incr(key);
    if (reserved > item.productVariation.stock) {
      // Pas assez de stock → annuler la réservation partielle
      await this.releaseStock(cart);
      throw new ConflictException('Stock insuffisant pour finaliser la commande');
    }
    await this.redisClient.expire(key, ttlSeconds);
  }
  // stock réservé avec succès
}

📌 Alternative sans Redis :

Ajouter une colonne reservedUntil dans la table cart_item ou product_variation, et un champ reservedQuantity. Plus complexe en production, Redis reste recommandé.

⏰ Expiration des paniers abandonnés

Un panier non finalisé après un certain délai (ex: 24h) doit être vidé et les réserves libérées.

// Entité Cart – ajout d'un champ expiresAt
@Entity()
export class Cart {
  @Column({ type: 'timestamp', nullable: true })
  expiresAt: Date;

  @BeforeUpdate()
  updateExpiration() {
    if (!this.isActive) this.expiresAt = null;
    else this.expiresAt = new Date(Date.now() + 24 * 60 * 60 * 1000);
  }
}
// Tâche cron (NestJS + @nestjs/schedule)
@Cron('0 0 * * *') // chaque nuit
async cleanExpiredCarts() {
  const expiredCarts = await this.cartRepo
    .createQueryBuilder('cart')
    .where('cart.expiresAt < :now', { now: new Date() })
    .getMany();

  for (const cart of expiredCarts) {
    await this.releaseStockForCart(cart);  // libère les locks Redis
    cart.items = [];
    cart.expiresAt = null;
    await this.cartRepo.save(cart);
  }
}

🎯 On peut aussi déclencher l’expiration après chaque action utilisateur (reset du timer) ou utiliser une file Bull/Redis.

🧠 Flux complet : panier → réservation → commande

1️⃣ Ajout produit → vérification stock (sans réserver)

2️⃣ Checkout (étape paiement) → appel à reserveStock() (lock Redis TTL 10–15 min)

3️⃣ Paiement réussi → on transforme la réservation en vente : décrémenter stock réel, supprimer les locks

4️⃣ Paiement échoué / panier expiré → libérer les locks (Redis DEL), panier redevient modifiable

5️⃣ Panier inactif > 24h → cron nettoie, libère les éventuels vestiges

// finalisation après paiement
async finalizeOrder(cartId: number) {
  const cart = await this.getCartWithItems(cartId);
  // 1. vérifier que les locks existent encore
  for (const item of cart.items) {
    const key = `stock:lock:${item.productVariation.id}`;
    const reserved = await this.redisClient.get(key);
    if (!reserved) throw new BadRequestException('Réservation expirée, recommencez');
  }
  // 2. décrémenter le stock réel
  for (const item of cart.items) {
    await this.variationRepo.decrement({ id: item.productVariation.id }, 'stock', item.quantity);
    await this.redisClient.del(`stock:lock:${item.productVariation.id}`);
  }
  // 3. créer la commande, vider le panier...
}

⚠️ Éviter les conditions de concurrence (race conditions)

Deux requêtes simultanées pourraient réserver plus que le stock. Utilisez des transactions atomiques ou des verrous optimistes.

// Exemple avec incr atomique de Redis (déjà fait)
const reserved = await this.redisClient.incr(key);
if (reserved > stock) {
  // rollback
}

// Alternative avec transaction SQL + SELECT FOR UPDATE
await this.dataSource.transaction(async manager => {
  const variation = await manager.findOne(ProductVariation, { 
    where: { id }, 
    lock: { mode: 'pessimistic_write' } 
  });
  if (variation.stock < quantity) throw new Error();
  variation.stock -= quantity;
  await manager.save(variation);
});
✅ Les locks Redis avec incr + TTL sont simples et efficaces pour les paniers.

async releaseExpiredLocks(variationId: number) {
  const key = `stock:lock:${variationId}`;
  const ttl = await this.redisClient.ttl(key);
  if (ttl <= 0) {
    await this.redisClient.del(key);
    return true;
  }
  return false;
}

📌 Bonnes pratiques à retenir

  • ✅ Toujours vérifier le stock avant d'ajouter au panier (sans réserver).
  • ✅ Réserver le stock le plus tard possible (étape de paiement) pour éviter de bloquer les produits trop longtemps.
  • ✅ Utiliser un TTL court (10–15 min) pour les locks de paiement.
  • ✅ Avoir un cron ou un worker qui nettoie les paniers abandonnés (24h).
  • ✅ Stocker l'expiration du panier en base pour faciliter le nettoyage.
  • ✅ Pour les sites à fort trafic, considérer un système de file d'attente (Bull/Redis) pour les réservations.

🎯 En résumé : Vérifier, Réserver temporairement, Confirmer, Libérer si abandon.
