➕ Créer & supprimer des utilisateurs

📌 useradd – ajouter un nouvel utilisateur

Syntaxe classique : sudo useradd -m -G groupe nom_utilisateur

  • -m : crée le répertoire personnel /home/nom
  • -G : ajoute l’utilisateur à des groupes supplémentaires
  • -s : définit le shell (ex: /bin/bash)

🗑️ userdel – suppression

sudo userdel -r nom_utilisateur → supprime le compte ET son home directory.

⚠️ Sans -r, les fichiers restent sur le système.

🧪 Exemple:
$ sudo useradd -m -G devops -s /bin/bash alice
➜ Alice peut maintenant se connecter

👥 Groupes : organisation & permissions

groupadd : créer un groupe sudo groupadd developpeurs

usermod : ajouter un utilisateur à un groupe secondaire
sudo usermod -aG developpeurs alice (option -aG pour ajouter sans retirer des groupes existants)

groupdel : supprimer un groupe sudo groupdel ancien_groupe

Un groupe principal (GID primaire) est assigné à chaque utilisateur. Le groupe principal est défini lors de la création.

✨ Astuce : vérifier les groupes d’un utilisateur avec groups nom_utilisateur ou id nom_utilisateur

📂 Comprendre les fichiers critiques

📄 /etc/passwd

Contient les comptes utilisateur (champ: nom:motdepasse( x ):UID:GID:commentaire:home:shell)

john:x:1001:1001:John Doe:/home/john:/bin/bash

⚠️ Le mot de passe est stocké dans /etc/shadow, le 'x' indique un shadowing.

🔒 /etc/shadow

Mots de passe chiffrés + politique de vieillissement (accès réservé root).

john:$6$salt$hash:19235:0:99999:7:::

👥 /etc/group

Définition des groupes et des membres.

devops:x:1002:alice,bob,charlie

⚙️ Cas pratiques & commandes essentielles

passwd alice – changer le mot de passe d’un utilisateur
chage -l alice – voir l’expiration du mot de passe
useradd -m -G sudo etudiant – créer un étudiant avec accès sudo
usermod -L nom – verrouiller un compte (Lock)
userdel -r ancien_etudiant – supprimer proprement

🧠 À retenir : Chaque utilisateur possède un UID (User ID) unique, les groupes simplifient la gestion des droits. Les fichiers /etc/passwd, /etc/shadow et /etc/group sont la base de la sécurité locale.

📝 Synthèse & mini-exercice

  • useradd / userdel : créer / supprimer utilisateurs
  • groupadd, usermod -aG : gestion des groupes secondaires
  • Fichiers système : passwd (identifiants), shadow (mots de passe hashés), group (appartenance)
  • Bonnes pratiques : toujours utiliser -m pour créer un home, vérifier avec id

🧪 Exercice rapide (à faire sur votre VM Linux) :

  1. Créez un groupe "webdev"
  2. Créez un utilisateur "student_web" avec le shell bash et répertoire personnel
  3. Ajoutez student_web au groupe webdev
  4. Consultez /etc/passwd et /etc/group pour vérifier
  5. Changez le mot de passe de student_web

💡 Utilisez sudo si nécessaire !
