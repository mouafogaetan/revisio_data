Segmentation mémoire

Découper la mémoire en segments logiques : code, données, pile

🎯 Objectifs de la leçon

Comprendre le principe de segmentation (segments code, données, pile).

2

3

À la fin de cette leçon, vous saurez expliquer comment la mémoire est divisée en segments logiques et comment le matériel protège les accès.

🧩 Qu'est-ce que la segmentation ?

La segmentation découpe l'espace d'adressage d'un programme en segments de taille variable ayant une signification logique : code, données, pile, bibliothèques, etc.

✔️ Chaque segment est une zone mémoire indépendante.
✔️ L'adresse logique = [segment, déplacement]
✔️ L'unité de gestion mémoire (MMU) traduit en adresse physique.

💡 Exemple concret : Quand un programme s'exécute, le système peut mettre le segment CODE en lecture seule, le segment DONNÉES en lecture/écriture, et la PILE pour les appels de fonction.

📋 Table des segments

Le système d'exploitation maintient une table des segments par processus. Chaque entrée contient :

ChampRôle
BaseAdresse physique de début du segment en RAM
LimiteTaille du segment (pour vérifier les accès hors limites)
DroitsLecture, écriture, exécution (protection mémoire)

🔍 Adressage : Adresse virtuelle = (numéro de segment, offset). La MMU vérifie que offset < limite, puis ajoute la base pour obtenir l'adresse physique. Si dépassement → exception (segmentation fault).

✅ Avantages de la segmentation

🔒

Protection fine

On peut attribuer des droits différents à chaque segment (ex: code en exécution seulement, données modifiables).

🔄

Partage efficace

Plusieurs processus peuvent partager un même segment (bibliothèque partagée) en référençant la même base.

📦

Pas de fragmentation interne

Chaque segment s'adapte exactement à la taille des données (pas de gaspillage à l'intérieur).

🧠

Logique naturelle

Correspond à la vision du programmeur (modules, fonctions).

⚠️ Fragmentation externe

Au fil du temps, la mémoire se retrouve avec des petits trous entre les segments. Même si la mémoire totale est suffisante, on ne peut pas charger un nouveau gros segment car aucun trou n'est assez grand.

⬆️ Petits espaces libres éparpillés → impossibilité de placer un gros segment.

🔧 Solutions possibles : Compaction (déplacer les segments pour regrouper l'espace libre) ou combiner avec la pagination (segmentation paginée).

Segmentation vs Pagination

CritèreSegmentationPagination
Taille unitéVariable (taille logique)Fixée (pages 4 Ko, 2 Mo)
FragmentationExterneInterne (dernière page)
Protection/partageNaturelle (par segment)Plus complexe (par page)
Utilisation moderneSouvent combinée (segmentation + pagination ex: x86)Très répandue (Linux, Windows)
💻 Sur les architectures modernes (x86-64), on utilise souvent une segmentation limitée avec une pagination pour une flexibilité maximale.

📌 En résumé

  • La mémoire est divisée en segments logiques : code, données, pile, etc.
  • Une table des segments associe à chaque segment : base, limite, droits.
  • Avantages : protection, partage, vision claire pour le développeur.
  • Inconvénient : fragmentation externe qui peut nécessiter une compaction.
  • Implémentation historique dans Intel 8086, puis segmentation paginée.

🧪 Petite question pour vous :

Un processus possède 3 segments : code (base=200, limite=80), données (base=600, limite=150). L'adresse logique <segment données, offset 140> est-elle valide ? Pourquoi ?

Voir la réponse

Oui, car offset 140 < limite 150 → accès valide, l'adresse physique = base(600) + 140 = 740.
