🔔 Les interruptions : signal d'alerte pour le CPU

Une interruption est un signal envoyé au processeur pour interrompre son exécution courante et forcer l'exécution d'une routine spécifique (gestionnaire d'interruption).

🖥️ Interruptions matérielles

Provenant des périphériques : clavier, souris, disque dur, timer interne.

Exemple : appuyer sur une touche → interruption IRQ → le système lit le code scanné.

💿 Interruptions logicielles

Déclenchées par une instruction logicielle (int en assembleur).

Utilisées pour les appels système (ex: lecture de fichier).

📡 Mécanisme simplifié : périphérique → CPU → gestionnaire

📋 Table des vecteurs d'interruption (IDT)

L'IDT (Interrupt Descriptor Table) est une structure de données utilisée par le processeur pour faire le lien entre un numéro d'interruption et le gestionnaire correspondant. À chaque interruption, le CPU consulte cette table pour savoir quelle fonction exécuter.

┌────────────────────────────────────────────┐
│   Index (vecteur)  │  Adresse du handler   │
├────────────────────────────────────────────┤
│   0 (division par zéro) → 0x8A4F2000       │
│   1 (debug)         → 0x8A4F2140           │
│   2 (NMI)           → ...                  │
│   32 (IRQ0 timer)   → scheduler_handler    │
│   33 (clavier)      → keyboard_handler     │
│   ...                                       │
└────────────────────────────────────────────┘

Sur les systèmes modernes (x86), l'IDT peut contenir jusqu'à 256 entrées. Les 32 premières sont réservées aux exceptions processeur.

🔍 Interruption vs Exception vs Trap

Bien qu'étant toutes des “événements” qui perturbent le flot normal, leurs causes et traitements diffèrent :

Type Origine Comportement Exemples
Interruption Matérielle (périphériques) ou logicielle (instruction INT) Asynchrone, survient à tout moment Clavier, souris, timer, appel système (trap volontaire)
Exception Erreur interne du processeur (situation anormale) Synchrone avec l'instruction en cours Division par zéro, page fault, accès mémoire invalide
Trap Instruction spéciale (souvent utilisée pour les appels système) Synchrone, volontaire, "portes dérobées" vers le noyau `syscall` sur Linux, `int 0x80` sur anciens x86
⚡ À retenir : Une interruption matérielle est asynchrone, l'exception est synchrone et souvent une erreur, le trap est synchrone et délibéré.
Exemple de trap: un programme utilisateur demande la lecture d'un fichier → instruction spéciale → noyau exécute la requête.

🎬 Mise en situation dans un OS

⌨️
Interruption clavier : Vous tapez "A". Le contrôleur du clavier envoie une interruption IRQ1 → le CPU suspend son programme, appelle le handler du clavier qui lit la touche et la place dans un buffer, puis reprend le programme.
🚫
Exception (division par zéro) : Un programme tente de diviser 10 par 0 → le CPU déclenche l'exception #DE. Le système peut tuer le processus ou afficher une erreur.
🖥️
Trap (Appel système) : En Python, `open()` génère un appel système. L'instruction `syscall` piège le CPU en mode noyau, exécute la demande, puis retourne au mode utilisateur.

📌 À retenir pour l'examen :

  • L'IDT permet la réaction rapide aux événements.
  • Les interruptions matérielles sont cruciales pour l'interactivité.
  • Les traps sont le pont entre mode utilisateur et mode noyau.
