📄 Qu'est-ce qu'un fichier ?

Un fichier est une collection nommée d'informations stockées sur un support (disque dur, SSD, clé USB). C'est l'unité logique de stockage utilisée par le système d'exploitation.

Fichier (données)

🔹 Le système d'exploitation voit le fichier comme une suite d'octets.
🔹 Chaque fichier possède des attributs (métadonnées) gérés par le système de fichiers.

Exemple : un document "cours.pdf", une image "photo.jpg", un programme "script.sh" sont tous des fichiers.

🏷️ Attributs d'un fichier

Chaque fichier possède des informations hors contenu appelées métadonnées. Voici les principales :

📛 Nom – identifiant lisible (ex: rapport.txt)
📏 Taille – nombre d'octets (Bytes)
🔐 Permissions – droits de lecture/écriture/exécution
📅 Dates – création, dernière modification, dernier accès
👤 Propriétaire – utilisateur et groupe propriétaire
🔗 Nombre de liens – combien de noms pointent vers ce fichier
📌 Exemple sous Linux : la commande stat fichier.txt affiche tous les attributs.

🛠️ Opérations de base sur les fichiers

Les systèmes d'exploitation offrent des appels système (syscalls) pour manipuler les fichiers :

➕
Création – create() ou open() avec O_CREAT
📖
Lecture – read() : transfert de données disque → mémoire
✍️
Écriture – write() : sauvegarde des données modifiées
🗑️
Suppression – unlink() (Linux) / DeleteFile() (Windows)
🔄
Renommage / Déplacement – rename()
📟 Exemple en C: FILE *f = fopen("data.txt", "r"); fread(...); fclose(f);

🗂️ Les différents types de fichiers

Tout n'est pas "fichier utilisateur" classique. Unix/Linux considère presque tout comme des fichiers :

TypeDescriptionExemple
📄 RégulierFichier standard (texte, binaire, image)document.pdf, script.sh
📁 RépertoireContient d'autres fichiers/dossiers/home/etudiant
🔗 Lien symboliquePointe vers un autre fichier (raccourci)lien -> cible.txt
📟 Périphérique (device)Représente un matériel (disque, souris, terminal)/dev/sda, /dev/tty
🔌 Tube (FIFO)Communication entre processuspipe nommé
💡 Sous Linux, la commande ls -l affiche un code (ex: -rw-r--r-- pour fichier régulier, d pour répertoire).

🧠 Analogie & Exemples pratiques

(Illustration: fichier = fiche)

📌 Imaginez un classeur avec des feuilles (fichiers). Chaque feuille a :

  • un nom (titre de la feuille)
  • une taille (nombre de mots)
  • des permissions (qui peut la lire/modifier)
  • des dates (création, dernière révision)

✅ À retenir : les attributs sont stockés dans la structure du système de fichiers (inode sous Linux).

🔎 Exercice mental :

Si un fichier texte fait 2048 octets, combien de kilo-octets ? (2 Ko) — La commande ls -lh permet de voir la taille.

📌 Récapitulons !

🎓 Ce qu'il faut maîtriser :

  • Un fichier = suite d'octets + métadonnées
  • Attributs : nom, taille, permissions, dates, propriétaire
  • Opérations : créer, lire, écrire, supprimer, renommer
  • Types : réguliers, répertoires, liens, périphériques

🧪 Mini‑quiz mental :

1️⃣ Quel attribut indique la date de dernière modification ? (timestamp mtime)
2️⃣ Sous Linux, quel caractère dans ls -l indique un répertoire ? (d)
3️⃣ La commande pour supprimer un fichier dans le terminal ? (rm fichier)
