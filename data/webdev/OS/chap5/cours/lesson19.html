Structure hiérarchique & chemins

Le système de fichiers UNIX est organisé en arborescence unique depuis la racine /. Chaque fichier ou répertoire est identifié par un chemin.

🔐 Permissions standard (rwx)

Chaque fichier/répertoire possède 3 catégories d'utilisateurs : propriétaire (u), groupe (g), autres (o). Pour chacune : lecture (r), écriture (w), exécution (x).

-rw-r--r--  1 alice  staff   1024 Mar 15 10:00 fichier.txt
drwxr-x---  2 bob    admin   4096 Mar 14 09:00 dossier/
r (4)
Lire le contenu
w (2)
Modifier le fichier
x (1)
Exécuter / traverser

Le mode octal combine les bits : chmod 754 fichier → propriétaire:7 (rwx), groupe:5 (r-x), autres:4 (r--).

🛠️ Commandes essentielles

chmod [options] mode fichier

Modifie les permissions. Exemple : chmod u+x script.sh (ajoute exécution au propriétaire). Mode octal : chmod 755 app.py.

chown propriétaire[:groupe] fichier

Change le propriétaire (et optionnellement le groupe). chown alice:staff data.txt.

chgrp groupe fichier

Change uniquement le groupe propriétaire. chgrp developers src/.

⚠️ Seul le superutilisateur (root) peut changer le propriétaire d’un fichier. Les utilisateurs normaux peuvent changer les permissions de leurs propres fichiers.

🔑 ACL : permissions avancées

Les ACL permettent de donner des permissions à des utilisateurs ou groupes spécifiques au-delà du modèle propriétaire/groupe/autres.

# Donner l'accès en lecture à l'utilisateur "kamdem"
setfacl -m u:kamdem:r-- fichier.conf

# Voir les ACL
getfacl fichier.conf

Utile dans les environnements multi-utilisateurs (serveurs, entreprises). Supporté par ext4, XFS, ZFS sous Linux.

📌 Les ACL sont une extension du modèle POSIX, très utilisées sur les serveurs partagés.

📌 Cas pratiques & bonnes pratiques

755Répertoire public : propriétaire peut tout, groupe/autres peuvent lire et traverser (drwxr-xr-x).
700Répertoire privé : seul le propriétaire a accès (drwx------).
644Fichier texte standard : propriétaire peut lire/écrire, autres seulement lecture.
111Exécutable pour tous (mal vu pour la sécurité).

✅ Règles de base :

  • Ne jamais donner 777 (sauf cas très spécifiques).
  • Utiliser le principe du moindre privilège.
  • Vérifier les permissions avec ls -l.
  • Pour les scripts, chmod +x est pratique mais vérifiez le contenu.
