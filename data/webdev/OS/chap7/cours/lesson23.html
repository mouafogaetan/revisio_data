🧩 Définition : Virtualisation des ressources

La virtualisation est une technique qui permet de créer une version virtuelle (abstraite) d’une ressource physique comme un serveur, un système d’exploitation, du stockage ou un réseau.

L'élément clé est l'hyperviseur (ou VMM), qui gère l'accès des machines virtuelles au matériel. Chaque VM pense posséder son propre matériel dédié.

⚡ Hyperviseurs : Type 1 (Bare-metal) vs Type 2 (Hébergé)

Type 1 — Bare metal

S'exécute directement sur le matériel, sans OS hôte. Très performant et sécurisé, utilisé dans les datacenters.

Exemples : VMware ESXi, Microsoft Hyper-V, KVM (noyau Linux), Xen.

Type 2 — Hébergé

Installé comme une application sur un OS hôte (Windows, Linux, macOS). Idéal pour tests et usage personnel.

Exemples : Oracle VirtualBox, VMware Workstation, QEMU.

📢 À retenir : Type 1 = plus efficace, utilisé en entreprise. Type 2 = pratique pour l'apprentissage, plus de latence.

💡 Pourquoi c'est important ? La virtualisation est la base du cloud computing (AWS, Azure, Google Cloud). Elle permet de mieux utiliser les ressources, d'isoler les applications et de simplifier la gestion.

✅ Pourquoi virtualiser ? Les avantages majeurs

✔️Meilleur taux d'occupation – plus de VMs sur un serveur physique.
✔️Isolation – un plantage dans une VM n'affecte pas les autres.
✔️Portabilité – les VMs peuvent migrer d’un serveur à l’autre.
✔️Tests & développement – créer des environnements sans risque.

📌 Récapitulons & Vérifions la compréhension

🧪 Question éclair :

"Je souhaite installer plusieurs machines virtuelles sur mon ordinateur portable Windows pour tester Linux. Quel type d'hyperviseur est le plus adapté ?"

🎓 À retenir absolument :

  • Virtualisation = créer des versions virtuelles du matériel.
  • Hyperviseur Type 1 : direct sur le matériel (performance max).
  • Hyperviseur Type 2 : sur un OS hôte (pratique pour la formation).
  • Exemples : ESXi (Type1), VirtualBox (Type2), KVM (Type1).
