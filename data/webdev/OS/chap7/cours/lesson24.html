🧠 Problématique : Mémoire virtuelle + Virtualisation

Dans un OS classique, la MMU traduit adresses virtuelles → physiques via tables des pages. Avec la virtualisation, l'hyperviseur gère la vraie mémoire physique tandis que chaque VM croit posséder sa propre mémoire physique. Deux approches majeures :

📄 Shadow Page Tables

L'hyperviseur maintient une copie « shadow » de la table des pages invitée. À chaque modification, il synchronise. Accès mémoire (GVA → GPA → HPA) devient rapide mais coûteux en mémoire et surcharge.

⚡ EPT / NPT (hardware)

Intel EPT (Extended Page Tables) / AMD NPT (Nested Page Tables) : matériel dédié pour effectuer la traduction en deux étapes directement. Gain de performance, moins d'interceptions.

     VM (invité)                  Hyperviseur            Matériel réel
 [Adresse virtuelle]    ->   [GPA -> HPA] via EPT    -> [Adresse physique]
     (GVA)                     (traduction 2 niveaux)        (HPA)

Figure: Avec EPT/NPT, le CPU parcourt deux tables imbriquées : une pour la VM, une pour l'hyperviseur.

📑 Shadow Page Tables (tables miroirs)

Principe : L'hyperviseur construit une table "shadow" qui mappe directement GVA (Guest Virtual Address) → HPA (Host Physical Address). La VM modifie ses propres tables (GVA→GPA), mais l'hyperviseur intercepte ces écritures pour mettre à jour la shadow table.

  • Avantages : Bonne performance une fois la shadow table à jour, compatible avec anciens processeurs.
  • Inconvénients : Coût mémoire élevé (table par VM), “page faults” fréquents (coût de synchronisation).

📌 Exemple concret : VMware ESXi (avant extensions) et Xen en mode shadow. Maintenant remplacé par EPT/NPT sur le matériel récent.

🚀 EPT (Intel) / NPT (AMD) : Accélération matérielle

Extensions de pagination imbriquée introduites autour de 2008. Le CPU gère deux niveaux de traduction en une seule itération matérielle :

1. Table des pages invitée
GVA → GPA (Guest Physical Address)
+
2. Extended Page Table (EPT)
GPA → HPA (Host Physical Address)

Conséquence : moins d'interceptions vers l'hyperviseur, réduction du overhead mémoire, performances proches du natif. Utilisé par KVM, Hyper-V, VMware.

✅ Anecdote : Sur Linux, vérifiez la présence avec cat /proc/cpuinfo | grep ept (Intel) ou npt (AMD).

⚙️ Virtualisation CPU : VMX (Intel) & SVM (AMD)

La virtualisation pure par logiciel (binary translation) était lourde. Dès les années 2000, les fondeurs intègrent des instructions dédiées :

Intel VT-x (VMX)

Modes VMX root (hyperviseur) et VMX non-root (invité). Instructions VMLAUNCH, VMRESUME pour passer d'un contexte à l'autre sans pièges.

AMD-V (SVM)

Secure Virtual Machine, jeu d'instructions similaire avec VMRUN. Gère les transitions rapides entre hôte et invité.

Ces extensions permettent d'exécuter directement le code privilégié de l'invité sur le CPU tout en interceptant les opérations sensibles via des "VM-Exit". L'hyperviseur n'a plus à traduire chaque instruction.

[Hyperviseur]  -- VMLAUNCH -->  [VM non-root] -- instruction sensible -->  VM-Exit --> Hyperviseur

🔁 Comparaison : 3 grands types de virtualisation

🧩 Full virtualization (virt. complète)

OS invité non modifié, hyperviseur émule le matériel. Ex: VMware Workstation ancien, VirtualBox avec extensions. Moins performant sans accélération.

📦 Para-virtualisation (PV)

L'invité est modifié (kernel adapté) avec des hypercalls à la place d'instructions non virtualisables. Ex: Xen (mode PV). Améliore les performances, mais nécessite des drivers.

⚡ Virtualisation assistée (Hardware-Assisted)

Utilise les extensions CPU (VT-x/AMD-V) et la virtualisation mémoire (EPT/NPT). Standard moderne : KVM, VMware ESXi, Hyper-V. Performance quasi-native.

💡 Actuellement, la plupart des cloud providers (AWS, GCP) utilisent KVM + hardware-assisted virtualization.

🧩 Synthèse : Comment CPU et mémoire collaborent

   +-------------------+       +---------------------------+
   |     VM (invité)   |       |    Hyperviseur (KVM/ESXi) |
   |   Code applicatif |       |                           |
   +--------|----------+       |  Gère les VM-Exit / EPT   |
            |                  |   +-------------------+   |
            v (GVA)            |   |  Shadow/EPT tables |   |
   [CPU avec VMX/SVM]  ----->  |   +-------------------+   |
            |                  +------------|--------------+
            |(EPT traduction)                |
            v                               v
   [Mémoire physique réelle] <----- (HPA)  [MMU matérielle]

Avec l'accélération matérielle, les accès mémoire et l'exécution CPU sont extrêmement rapides, rendant la virtualisation transparente pour les serveurs modernes.
