Les boucles en JavaScript

for, while, do...while

👆 glissez ou naviguez avec les boutons

🎯 Objectifs de la leçon

  • • Utiliser la boucle for avec ses trois expressions (initialisation, condition, incrémentation).
  • • Maîtriser la boucle while (condition vérifiée avant exécution).
  • • Utiliser la boucle do...while (exécution au moins une fois).
  • • Savoir quand choisir chaque type de boucle.
⚡ "Une boucle répète une action tant qu'une condition est vraie. C'est le carburant de l'automatisation !"

🔄 1. La boucle for

Idéale quand on sait combien de fois répéter une action. Syntaxe classique :

for (initialisation; condition; incrémentation) {
    // bloc à répéter
}

🔍 Exemple concret : afficher les nombres de 1 à 5.

for (let i = 1; i <= 5; i++) {
    console.log("Itération numéro " + i);
}
💡 Explication : let i = 1 (départ) → tant que i <= 5 → exécute le bloc → après chaque tour, i++ (incrémente).
✅ À retenir : for convient aux boucles à compteur (tableaux, séquences).

♾️ 2. La boucle while

Vérifie la condition avant chaque exécution. Si la condition est fausse dès le départ, la boucle ne s'exécute pas.

while (condition) {
    // instructions
}

📘 Exemple : saisie utilisateur jusqu'à obtenir un nombre valide (simulation).

let nombre = 0;
while (nombre < 1 || nombre > 10) {
    nombre = prompt("Donnez un nombre entre 1 et 10");
}
🧠 Important : pensez à mettre à jour la variable dans la boucle pour éviter une boucle infinie !
⚠️ Si la condition est toujours vraie → boucle infinie (plante le navigateur).

🔁 3. La boucle do...while

Exécute le bloc au moins une fois, puis vérifie la condition pour continuer ou s'arrêter.

do {
    // bloc exécuté au moins une fois
} while (condition);

🎲 Exemple : proposer un menu à un utilisateur (garantir l'affichage une fois).

let reponse;
do {
    reponse = prompt("Choisissez 1: Jouer, 2: Quitter");
} while (reponse !== "2");
⭐ Différence clé : while peut ne rien exécuter, do...while exécute toujours au moins 1 fois.

⚖️ Comparaison & astuces

BoucleQuand l'utiliser ?
forNombre d'itérations connu (parcourir tableau, compter).
whileCondition dépend d'une action extérieure, on ne sait pas combien de tours.
do...whileBesoin d'exécuter le code au moins une fois avant de tester (ex: demander une saisie).
📌 Schéma mental : Si tu as un compteur → for. Si tu attends une condition qui peut être fausse initialement → while. Si tu as besoin d'exécuter avant de valider → do...while.

📝 Récapitulatif :

  • for : structure compacte (init; cond; incrément).
  • while : idéal pour boucles conditionnées avec risque zéro exécution.
  • do...while : garantie une exécution, parfait pour les menus ou validation.
  • Maîtriser les boucles rend le code puissant et évite les répétitions !
