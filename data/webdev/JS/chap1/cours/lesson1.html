🌐 Qu'est-ce que JavaScript ?

JavaScript est un langage de programmation qui permet de rendre une page web interactive et dynamique. À l'origine, il était utilisé uniquement dans les navigateurs pour manipuler le contenu HTML, réagir aux clics, valider des formulaires ou animer des éléments.

// Exemple simple : afficher une alerte

alert("Bienvenue sur ma page web !");

Avec JavaScript, le navigateur exécute du code directement sur l’ordinateur de l’utilisateur, sans recharger la page.

📄 HTML (structure) + 🎨 CSS (style) + ⚡ JS (interaction) = 🌟 Site vivant

🖥️ Client vs Serveur (Node.js)

📱 JavaScript Client

Exécuté dans le navigateur (Chrome, Firefox...). Permet d’interagir avec le DOM, gérer événements, animations, requêtes HTTP.

✅ Accès au window, document
⚙️ JavaScript Serveur (Node.js)

Exécuté sur le serveur. Permet de créer des APIs, accéder à des fichiers, bases de données, etc.

✅ Accès au système de fichiers, réseau

À retenir : Dans ce cours, on se concentre sur le JavaScript côté client, pour animer nos pages web !

📜 Évolution d'ECMAScript

ECMAScript (ES) est le standard sur lequel JavaScript est basé. Voici les grandes étapes :

  • ES5 (2009) : support étendu, méthodes comme forEach(), JSON. Très répandu.
  • ES6 / ES2015 : révolution ! let/const, fonctions fléchées, classes, template literals, promesses...
  • ES2020+ : Nullish coalescing (??), optional chaining (?.), BigInt...
🧠 Les navigateurs modernes supportent les fonctionnalités ES6+, nous les utiliserons dans ce cours.

🔌 Intégration de JavaScript dans HTML

On utilise la balise <script> de deux manières principales :

1. JavaScript interne (dans la page)

<script>
  console.log("Hello depuis l'interne");
</script>

2. JavaScript externe (fichier .js) - recommandé

<script src="monScript.js"></script>

Avantage : code réutilisable et plus clair.

💡 Bonne pratique : placer les scripts avant la fermeture de </body> ou utiliser defer/async.

⏳ Ordre d'exécution & attributs defer/async

Par défaut, le navigateur charge et exécute les scripts dans l’ordre où ils apparaissent. Cela peut bloquer l’affichage de la page.

normal
Chargement HTML → script exécuté immédiatement → suite HTML
defer
Téléchargement en parallèle, exécuté après le parsing HTML → ordre préservé
async
Téléchargement parallèle, exécuté dès que possible (sans ordre)
<script src="app.js" defer></script>
<script src="analytics.js" async></script>
✅ Le plus souvent pour nos projets, on utilise defer quand on a besoin d'accéder au DOM complet.

🛠️ Outils de débogage : la console

La console du navigateur (F12 → onglet Console) est votre meilleure amie pour tester du code et voir les erreurs.

console.log("Message informatif");
console.error("Une erreur est survenue");
console.warn("Attention ⚠️");

Syntaxe de base : les instructions se terminent par un point-virgule ; (facultatif mais recommandé). JavaScript est sensible à la casse.

📌 Commentaires : // pour une ligne, /* ... */ pour plusieurs lignes.

💬 Interactions utilisateur : alert, prompt, confirm

JavaScript permet des boîtes de dialogue simples pour communiquer avec l'utilisateur :

  • 🔔 alert("message") → affiche une boîte d'information.
  • ✏️ prompt("question", "valeur par défaut") → demande une saisie à l'utilisateur.
  • ❓ confirm("Confirmez-vous ?") → retourne true (OK) ou false (Annuler).

Essayez d'utiliser la console plus souvent que alert pour déboguer, mais prompt/confirm sont utiles pour des petites démos.

📚 Récapitulons la leçon 1

  • JavaScript rend les pages interactives et dynamiques.
  • Il s’exécute côté client (navigateur) ou côté serveur (Node.js).
  • ECMAScript est le standard : ES6 a apporté des fonctionnalités majeures.
  • On intègre JS via <script> interne ou externe, avec defer/async pour contrôler le chargement.
  • Utilisez la console (console.log) et les boîtes de dialogue pour débuter.
