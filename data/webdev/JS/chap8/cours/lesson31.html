Leçon 31 – Chapitre 8

🌐 Fetch API

Requêtes HTTP (GET, POST, PUT, DELETE) et gestion des réponses JSON

Durée : 50 min Niveau : Intermédiaire Communication avec API

"fetch() est la manière moderne de faire des requêtes réseau en JavaScript."

🎯 Objectifs de la leçon

  • Comprendre l'API fetch() pour faire des requêtes HTTP.
  • Utiliser les méthodes GET, POST, PUT, DELETE.
  • Gérer les réponses JSON et les erreurs réseau.
  • Construire des en-têtes (headers) et envoyer des données.
fetch() remplace XMLHttpRequest et offre une syntaxe basée sur les Promesses, plus simple et plus puissante.

📡 Qu'est-ce que fetch() ?

fetch() est une fonction JavaScript moderne pour effectuer des requêtes réseau (AJAX). Elle retourne une Promesse qui se résout en un objet Response.

// Syntaxe de base d'une requête GET
fetch('https://api.exemple.com/posts')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Erreur:', error));
Important : fetch() ne rejette pas la Promesse pour les erreurs HTTP (404, 500). Il faut vérifier response.ok.

📖 Requête GET (lire des données)

GET est la méthode par défaut de fetch(). Elle sert à récupérer des données depuis un serveur.

async function getPosts() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts');
    if (!response.ok) throw new Error(`Erreur ${response.status}`);
    const data = await response.json();
    console.log('Posts reçus:', data);
  } catch (error) {
    console.error('Erreur réseau:', error);
  }
}

✏️ POST, PUT, DELETE

🔹 POST (créer une ressource)

const newPost = { title: 'Mon article', body: 'Contenu...', userId: 1 };
const response = await fetch('https://api.exemple.com/posts', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(newPost)
});

🔹 PUT (remplacer une ressource)

const updatedPost = { title: 'Titre modifié', body: 'Nouveau contenu', userId: 1 };
const response = await fetch('https://api.exemple.com/posts/1', {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(updatedPost)
});

🔹 DELETE (supprimer une ressource)

const response = await fetch('https://api.exemple.com/posts/1', {
  method: 'DELETE'
});

⚠️ Gestion des erreurs avec fetch()

fetch() ne rejette pas la Promesse pour les erreurs HTTP. Il faut vérifier response.ok.

async function secureFetch(url, options = {}) {
  try {
    const response = await fetch(url, options);
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    return await response.json();
  } catch (error) {
    console.error('Erreur fetch:', error.message);
    return null;
  }
}
Bonnes pratiques : Toujours utiliser try/catch avec async/await, vérifier response.ok, et afficher des messages utilisateur adaptés.

✏️ Exercice & Conclusion

Exercice guidé :

Créez une petite application qui utilise l'API JSONPlaceholder :

  1. Faites une requête GET pour récupérer les posts (https://jsonplaceholder.typicode.com/posts).
  2. Affichez les titres des posts dans la console.
  3. Ajoutez un nouveau post avec POST (titre et body).
  4. Gérez les erreurs avec try/catch et vérification de response.ok.
Voir une solution
async function exerciceFetch() {
  try {
    // 1. GET tous les posts
    const resGet = await fetch('https://jsonplaceholder.typicode.com/posts');
    if (!resGet.ok) throw new Error(`GET error: ${resGet.status}`);
    const posts = await resGet.json();
    console.log('Titres:', posts.map(p => p.title));

    // 2. POST un nouveau post
    const newPost = { title: 'Mon nouveau post', body: 'Contenu test', userId: 1 };
    const resPost = await fetch('https://jsonplaceholder.typicode.com/posts', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(newPost)
    });
    if (!resPost.ok) throw new Error(`POST error: ${resPost.status}`);
    const created = await resPost.json();
    console.log('Post créé:', created);
  } catch (error) {
    console.error('Erreur:', error.message);
  }
}

Récapitulatif :

  • fetch() = fonction moderne pour les requêtes HTTP.
  • Méthodes : GET (lire), POST (créer), PUT (remplacer), DELETE (supprimer).
  • Utiliser async/await ou .then().
  • Toujours gérer les erreurs (try/catch + vérification response.ok).
  • Convertir la réponse en JSON avec response.json().
