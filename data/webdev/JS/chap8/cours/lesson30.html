🔥 Le problème : "Callback Hell"

JavaScript exécute certaines tâches de manière asynchrone (setTimeout, requêtes). Avec les callbacks imbriqués, le code devient illisible :

// Exemple callback hell (pyramide de la mort)
setTimeout(() => {
  console.log("Étape 1");
  setTimeout(() => {
    console.log("Étape 2");
    setTimeout(() => {
      console.log("Étape 3");
      // difficile à gérer, peu maintenable
    }, 500);
  }, 500);
}, 500);

😵 Difficultés : erreurs compliquées, enchaînements moches, problème d'inversion de contrôle.

💡 Les Promesses (Promise) apportent une solution élégante.

🤝 Les Promesses : un engagement futur

Une promesse représente une valeur qui sera disponible maintenant, plus tard ou jamais.

const maPromesse = new Promise((resolve, reject) => { ... });
  • resolve(valeur) : succès → déclenche .then()
  • reject(erreur) : échec → déclenche .catch()
  • .finally() : toujours exécuté à la fin
// Exemple concret
const chargerDonnees = new Promise((resolve, reject) => {
  let succes = true;
  setTimeout(() => {
    if(succes) resolve("📦 Données chargées");
    else reject("❌ Erreur réseau");
  }, 1000);
});

chargerDonnees
  .then(result => console.log(result))
  .catch(err => console.error(err))
  .finally(() => console.log("Requête terminée"));

On peut chaîner plusieurs .then() pour des opérations séquentielles.

⚡ Async / Await : du sucre syntaxique

Les mots-clés async et await rendent le code asynchrone aussi clair que du code synchrone.

async function maFonction() { ... }

→ transforme la fonction en une fonction qui retourne toujours une Promesse.

const resultat = await promesse;

→ attend la résolution de la promesse et renvoie sa valeur (doit être utilisé dans une fonction async).

// Exemple avec async/await (plus propre !)
async function afficherUtilisateur() {
  try {
    let reponse = await fetch('https://jsonplaceholder.typicode.com/users/1');
    let utilisateur = await reponse.json();
    console.log(utilisateur.name);
  } catch (erreur) {
    console.error("Problème:", erreur);
  }
}
afficherUtilisateur();
✅ Avantages : plus besoin de chaînage .then(), gestion d'erreur avec try/catch classique.
