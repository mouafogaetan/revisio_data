Syntaxe moderne, return implicite et comportement de this lexical

🏹 Fonction fléchée : la base

Introduite avec ES6 (ECMAScript 2015), la fonction fléchée permet d'écrire des fonctions de façon plus compacte.

// Syntaxe classique

const maFonction = function(param) {
  return param * 2;
};

// Fonction fléchée

const maFlechee = (param) => {
  return param * 2;
};

✨ Si un seul paramètre, les parenthèses sont optionnelles.

const double = x => x * 2;   // return implicite (on verra juste après)
⚡ Règle : param => expression équivaut à function(param) { return expression; }

📦 Return implicite & parenthèses

Quand le corps de la fonction est une seule expression, on peut omettre return et les accolades.

// Avec return explicite
const addition = (a, b) => {
  return a + b;
};

// Return implicite (plus court !)
const additionCourte = (a, b) => a + b;

console.log(additionCourte(5, 3)); // 8

🔁 Pour retourner un objet littéral, il faut le parenthéser :

const obtenirPersonne = (nom, age) => ({ nom, age });
// sans parenthèses : erreur car les accolades sont interprétées comme bloc
✅ Astuce : retour implicite = code plus clair pour les petites fonctions (map, filter, etc.)

🧠 Le super-pouvoir : this lexical

Contrairement aux fonctions classiques, les fonctions fléchées ne redéfinissent pas leur propre this. Elles héritent du this du contexte parent (portée englobante).

Exemple classique (problème) avec function() :

const utilisateur = {
  nom: "Claude",
  direBonjour: function() {
    setTimeout(function() {
      console.log("Bonjour " + this.nom); // this = window / undefined
    }, 100);
  }
};
utilisateur.direBonjour(); // "Bonjour undefined"

✅ Solution avec fonction fléchée :

const utilisateur2 = {
  nom: "Aicha",
  direBonjour: function() {
    setTimeout(() => {
      console.log("Bonjour " + this.nom); // this = utilisateur2
    }, 100);
  }
};
utilisateur2.direBonjour(); // "Bonjour Aicha"

💡 Conclusion : Les fléchées sont parfaites pour les callbacks (setTimeout, événements, promesses) afin de conserver le contexte parent.

⚠️ Attention : Les fonctions fléchées ne doivent pas être utilisées comme méthodes si vous voulez accéder à l’objet via this (car this = fenêtre ou undefined en mode strict). Mais elles sont géniales pour les fonctions internes !

🧪 Petit exercice pratique

Teste ta compréhension ! Complète la fonction fléchée pour doubler un nombre.

const double = (n) => ___ ;

✅ La bonne réponse : const double = (n) => n * 2; (return implicite)

✨ Challenge : Transformer cette fonction classique en fléchée

function direBonjour(nom) {
  return "Salut " + nom + " !";
}
const direBonjour = nom => `Salut ${nom} !`; (template string + return implicite)

📌 Résumons-nous !

  • Les fonctions fléchées offrent une syntaxe plus courte : param => expression
  • Le return implicite évite les accolades pour une seule instruction.
  • this lexical : idéal pour les callbacks, méthodes de tableau, promesses.
  • Ne pas utiliser pour les constructeurs, ni comme méthode d'objet si on utilise this sur l'objet.
  • Pas d'objet arguments → préférer les paramètres REST : (...args) =>
