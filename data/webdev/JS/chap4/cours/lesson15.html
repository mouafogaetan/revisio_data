Les closures (fermetures)

Fonctions qui gardent un souvenir de leur environnement

Glisser ou cliquer sur les boutons →

🎯 Objectifs de la leçon

  • 1. Comprendre la portée lexicale (lexical scoping) : comment JavaScript recherche les variables.
  • 2. Définir une closure : une fonction qui accède à des variables d'une fonction parente même après que celle-ci ait fini son exécution.
  • 3. Utiliser les closures pour créer des variables privées et encapsuler des données.

💡 Une closure, c'est comme une "boîte à souvenirs" : la fonction enfant se souvient des variables de sa mère.

🧠 1. La portée lexicale (Lexical Scoping)

En JavaScript, la portée d'une variable est déterminée par l'endroit où elle est déclarée dans le code, pas au moment de l'exécution. Les fonctions imbriquées ont accès aux variables de leur fonction parent.

function externe() {
  let message = "Je suis parent !";
  
  function interne() {
    console.log(message); // ✅ Accès autorisé
  }
  
  interne();
}
externe(); // Affiche "Je suis parent !"

✅ Règle simple : une fonction voit toutes les variables déclarées dans les fonctions qui la contiennent (et globales).

🔒 2. Qu'est-ce qu'une closure ?

Une closure se produit quand une fonction interne est retournée ou utilisée en dehors de sa fonction parente, mais qu'elle garde l'accès aux variables de cette fonction parente, même après que celle-ci ait terminé son exécution.

🎬 Exemple classique :

function creerCompteur() {
  let count = 0;   // variable "privée"
  return function() {
    count++;
    return count;
  };
}
const monCompteur = creerCompteur();
console.log(monCompteur()); // 1
console.log(monCompteur()); // 2
// count n'est pas accessible directement, mais la closure s'en souvient !

✨ Explication : La fonction retournée « ferme » la variable count dans son environnement → closure.

🛡️ 3. Variables privées grâce aux closures

Avant les classes modernes (ES6), on utilisait les closures pour encapsuler des données et imiter la vie privée (comme dans d'autres langages).

function Personne(nom) {
  let _age = 0;   // variable privée
  
  return {
    prendreAge: function() { return _age; },
    feterAnniversaire: function() { _age++; }
  };
}
let alice = Personne("Alice");
alice.feterAnniversaire();
alice.feterAnniversaire();
console.log(alice.prendreAge()); // 2
// alice._age n'existe pas -> on ne peut pas y toucher directement !

✔️ Avantage : Protège l'intégrité des données. Seules les fonctions exposées (méthodes) peuvent modifier _age.

💻 4. Exercice interactif : compteur privé

Observez une closure en live : cette fonction crée un compteur indépendant pour chaque bouton. Chaque compteur garde sa propre variable compteur privée.

📦 Compteur A

0

closure créée par makeCounter()

📦 Compteur B

// Code de la closure :
function makeCounter() { let count = 0; return function() { count++; return count; } }

✨ Remarque : Les deux compteurs sont indépendants car chaque appel à makeCounter() crée une nouvelle closure avec son propre count.

⚠️ 5. Attention aux closures dans les boucles

Un problème classique : quand on crée des fonctions dans une boucle, elles partagent la même variable si on utilise var. Avec let ou une closure immédiate, on peut résoudre.

// Problème (si var i)
for (var i = 1; i <= 3; i++) {
  setTimeout(function() { console.log(i); }, 100);
} // Affiche 4,4,4 ❌

// Solution avec let (ES6) ou closure IIFE
for (let i = 1; i <= 3; i++) {
  setTimeout(() => console.log(i), 100);
} // Affiche 1,2,3 ✅

💡 Chaque itération avec let crée une nouvelle liaison, donc chaque closure capture la bonne valeur.

🧩 Résumé closure : C'est une fonction + l'environnement lexical dans lequel elle a été créée.
