Leçon 12 – Chapitre 4

📦 Fonctions en JavaScript

Déclaration, expression de fonction, hoisting

Durée : 45 min Niveau : Débutant Fonctions essentielles

"Une fonction est un bloc de code réutilisable qui exécute une tâche spécifique."

🎯 Objectifs de la leçon

  • Déclarer une fonction avec le mot-clé function (déclaration de fonction).
  • Utiliser les expressions de fonction (fonctions anonymes).
  • Comprendre la différence entre déclaration et expression de fonction.
  • Maîtriser le concept de hoisting pour les déclarations de fonction.
Les fonctions sont les briques fondamentales de la programmation JavaScript. Elles permettent de structurer, réutiliser et organiser le code.

📝 Déclaration de fonction

La syntaxe classique pour créer une fonction nommée :

function nomFonction(param1, param2) {
  // corps de la fonction
  return résultat;
}
Caractéristique importante : Les déclarations de fonction sont hoistées (remontées en haut de leur portée). On peut donc appeler la fonction avant sa déclaration dans le code.
// ✅ Ceci fonctionne grâce au hoisting !
direBonjour("Alice"); // Affiche "Bonjour Alice"

function direBonjour(prenom) {
  console.log("Bonjour " + prenom);
}

🔧 Expression de fonction

On stocke une fonction anonyme dans une variable :

const maFonction = function(param1, param2) {
  // corps de la fonction
  return résultat;
};
Attention : Les expressions de fonction ne sont PAS hoistées. On ne peut pas les appeler avant leur définition.
// ❌ Ceci génère une erreur (ReferenceError)
direBonjour("Bob"); // Erreur : Cannot access before initialization

const direBonjour = function(prenom) {
  console.log("Bonjour " + prenom);
};

Les expressions de fonction peuvent être anonymes (sans nom) ou nommées (utile pour le débogage).

Règle générale : préférez les déclarations de fonction pour le code principal, et les expressions lorsque vous avez besoin d'assigner une fonction à une variable conditionnellement.

🏗️ Le Hoisting en détail

Le hoisting (remontée) est un mécanisme JavaScript qui déplace les déclarations (variables et fonctions) en haut de leur portée avant l'exécution du code.

Comment JavaScript interprète le code :

// Ce que vous écrivez
maFonction();
function maFonction() { console.log("OK"); }

// Ce que JavaScript comprend (phase de compilation)
function maFonction() { console.log("OK"); }
maFonction();
Important : Seule la déclaration est remontée, pas l'initialisation. Les fonctions fléchées et les expressions de fonction ne bénéficient pas du hoisting.

✏️ Exercice & Conclusion

Exercice guidé :

Créez successivement :

  1. Une déclaration de fonction addition(a, b) qui retourne la somme de a et b.
  2. Une expression de fonction soustraction qui retourne la différence.
  3. Testez l'appel de addition avant sa déclaration (hoisting).
  4. Testez l'appel de soustraction avant son expression (que se passe-t-il ?).
Voir une solution
// ✅ Hoisting fonctionne
console.log(addition(5, 3)); // Affiche 8

function addition(a, b) {
  return a + b;
}

// ❌ Erreur : Cannot access before initialization
// console.log(soustraction(10, 4));

const soustraction = function(a, b) {
  return a - b;
};

console.log(soustraction(10, 4)); // ✅ Affiche 6

Récapitulatif :

  • Déclaration de fonction = function nom() {} → hoistée, appel avant définition possible.
  • Expression de fonction = const f = function() {} → non hoistée.
  • Le hoisting remonte les déclarations mais pas les initialisations.
  • Préférez les déclarations pour les fonctions principales, les expressions pour les cas spécifiques.
