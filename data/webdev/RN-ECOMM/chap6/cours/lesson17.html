💡 Pourquoi c'est important ? Dans une application e-commerce, un formulaire mal sécurisé (adresse, email, carte bancaire) peut entraîner des pertes de données ou des failles.

📝 React Hook Form : la base

React Hook Form est une bibliothèque légère qui optimise les rendus et simplifie la gestion des formulaires. Contrairement à d'autres solutions, elle utilise des refs non contrôlées pour éviter les re-rendus inutiles.

Installation
npm install react-hook-form zod @hookform/resolvers

Exemple minimal d'un formulaire d'inscription :

import { useForm } from "react-hook-form";

function RegisterForm() {
  const { register, handleSubmit, formState: { errors } } = useForm();
  
  const onSubmit = (data) => console.log(data);

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register("email", { required: true })} placeholder="Email" />
      {errors.email && <span>Ce champ est requis</span>}
      
      <input {...register("password", { minLength: 6 })} type="password" />
      {errors.password && <span>6 caractères minimum</span>}
      
      <button type="submit">S'inscrire</button>
    </form>
  );
}
⚠️ En React Native, remplacez <form> par <View> et utilisez TextInput.

✅ Zod : validation déclarative et typée

Zod est une bibliothèque de validation de schémas avec une intégration parfaite TypeScript. Elle permet de définir des règles complexes et d'en déduire automatiquement les types.

Exemple de schéma pour un compte client :

import { z } from "zod";

const userSchema = z.object({
  email: z.string().email("Email invalide"),
  password: z.string().min(8, "Au moins 8 caractères"),
  age: z.number().int().positive().min(18, "Vous devez avoir 18 ans"),
  newsletter: z.boolean().default(false)
});

// TypeScript inferé automatiquement
type UserFormData = z.infer<typeof userSchema>;

Combinaison avec React Hook Form grâce au résolveur @hookform/resolvers :

import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { z } from "zod";

const schema = z.object({
  email: z.string().email(),
  password: z.string().min(8)
});

function MyForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(schema)
  });
  // ... suite
}
🧠 Astuce: Les messages d'erreur de Zod sont entièrement personnalisables et supportent l'internationalisation.

⏳ Gestion des erreurs & états de chargement

Une bonne expérience utilisateur nécessite des retours clairs : chargement, succès, erreur.

📌 État de chargement

const { formState: { isSubmitting } } = useForm();

<button disabled={isSubmitting}>
  {isSubmitting ? "Envoi..." : "Valider"}
</button>

⚠️ Erreurs globales

const { setError } = useForm();

try {
  await api.login(data);
} catch (err) {
  setError("root", { 
    message: "Identifiants incorrects" 
  });
}

Affichage des erreurs par champ :

<TextInput />
{errors.email && (
  <Text className="text-red-500 text-sm">
    {errors.email.message}
  </Text>
)}

📱 Bonnes pratiques mobiles : Utilisez des toasts (react-native-toast-message) pour les erreurs globales et des messages sous les champs pour les erreurs locales.

📦 Cas concret : formulaire de livraison

Voici un exemple complet pour un écran de saisie d'adresse dans une application e-commerce.

import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { z } from "zod";
import { View, TextInput, Text, Button, ActivityIndicator } from "react-native";

const addressSchema = z.object({
  fullName: z.string().min(3, "Nom complet requis"),
  phone: z.string().regex(/^(6|7)[0-9]{8}$/, "Téléphone camerounais invalide (ex: 6XXXXXXXX)"),
  city: z.string().min(2, "Ville requise"),
  address: z.string().min(5, "Adresse détaillée"),
  isDefault: z.boolean().optional()
});

type AddressForm = z.infer<typeof addressSchema>;

export default function DeliveryForm() {
  const { control, handleSubmit, formState: { errors, isSubmitting }, setError } = useForm<AddressForm>({
    resolver: zodResolver(addressSchema),
    defaultValues: { isDefault: false }
  });

  const onSubmit = async (data: AddressForm) => {
    try {
      await saveAddress(data);
      // Afficher un toast de succès
    } catch (error) {
      setError("root", { message: "Erreur réseau, réessayez" });
    }
  };

  return (
    <View>
      <TextInput placeholder="Nom complet" />
      {errors.fullName && <Text>{errors.fullName.message}</Text>}
      
      <TextInput placeholder="Téléphone" keyboardType="phone-pad" />
      {errors.phone && <Text>{errors.phone.message}</Text>}
      
      <TextInput placeholder="Ville" />
      {errors.city && <Text>{errors.city.message}</Text>}
      
      <TextInput placeholder="Adresse rue, quartier" />
      
      <Button title="Enregistrer" onPress={handleSubmit(onSubmit)} disabled={isSubmitting} />
      {isSubmitting && <ActivityIndicator />}
    </View>
  );
}
✨ La validation par regex pour le téléphone tient compte des indicatifs camerounais (6 ou 7 suivi de 8 chiffres).

📌 Récapitulatif & bonnes pratiques

  • ✅ Préférez toujours react-hook-form pour ses performances et sa simplicité.
  • ✅ Utilisez Zod pour la validation : code déclaratif, typage automatique, réutilisable.
  • ✅ Affichez les erreurs en temps réel (au changement ou à la soumission) pour guider l'utilisateur.
  • ✅ Protégez contre la double soumission avec isSubmitting ou un état local.
  • ✅ Sanitisez les entrées même si Zod valide (ex: trim(), suppression des caractères dangereux).
