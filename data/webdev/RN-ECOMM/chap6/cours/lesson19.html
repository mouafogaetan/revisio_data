Pourquoi c'est important ? Au Cameroun comme ailleurs, les tokens JWT volés peuvent permettre à un attaquant de prendre le contrôle du compte utilisateur. SecureStore protège ces données même sur un appareil rooté ou jailbreaké.

1. Installation de expo-secure-store

expo-secure-store est une bibliothèque Expo qui permet de stocker des données sensibles de manière chiffrée. Sur Android, elle utilise le Keystore système ; sur iOS, le Keychain.

# Installation dans un projet Expo
npx expo install expo-secure-store

# Pour un projet React Native CLI (hors Expo)
npm install react-native-encrypted-storage
Android : Keystore + AES-GCM
iOS : Keychain (kSecAttrAccessibleWhenUnlocked)
Attention : SecureStore ne remplace pas AsyncStorage pour les données non sensibles (thème, préférences). Utilisez-le uniquement pour tokens, mots de passe, informations bancaires.

2. API de expo-secure-store

import * as SecureStore from 'expo-secure-store';

// Stocker une valeur (chaîne de caractères)
await SecureStore.setItemAsync('accessToken', 'jwt_token_ici');

// Lire une valeur
const token = await SecureStore.getItemAsync('accessToken');

// Supprimer une valeur
await SecureStore.deleteItemAsync('accessToken');

// Supprimer TOUTES les valeurs (déconnexion)
await SecureStore.deleteItemAsync('refreshToken');
await SecureStore.deleteItemAsync('userData');
Les méthodes sont asynchrones (retournent une Promise). Toujours utiliser async/await ou .then().
Fonctionnement interne : Les données sont chiffrées avec une clé dérivée du matériel. Même en rootant le téléphone, les tokens restent inaccessibles.

3. Intégration avec Zustand (authStore)

On combine SecureStore avec Zustand pour une gestion d'état persistante et sécurisée.

// stores/authStore.js
import { create } from 'zustand';
import * as SecureStore from 'expo-secure-store';

const useAuthStore = create((set, get) => ({
  user: null,
  accessToken: null,
  isLoading: true,

  login: async (userData, token) => {
    await SecureStore.setItemAsync('accessToken', token);
    await SecureStore.setItemAsync('user', JSON.stringify(userData));
    set({ user: userData, accessToken: token });
  },

  logout: async () => {
    await SecureStore.deleteItemAsync('accessToken');
    await SecureStore.deleteItemAsync('user');
    set({ user: null, accessToken: null });
  },

  restoreSession: async () => {
    const token = await SecureStore.getItemAsync('accessToken');
    const userJson = await SecureStore.getItemAsync('user');
    if (token && userJson) {
      set({ 
        accessToken: token, 
        user: JSON.parse(userJson),
        isLoading: false 
      });
    } else {
      set({ isLoading: false });
    }
  }
}));

export default useAuthStore;
Appeler restoreSession() dans le composant racine (useEffect) pour recharger la session au démarrage.

4. Déconnexion : effacer les données

À la déconnexion, il faut supprimer toutes les données sensibles pour éviter qu'un autre utilisateur du même téléphone accède au compte.

// Composant ProfileScreen
const handleLogout = async () => {
  // 1. Nettoyer le store Zustand
  useAuthStore.getState().logout();
  
  // 2. Supprimer d'autres données (panier, favoris si nécessaire)
  await AsyncStorage.removeItem('cartStore');
  await AsyncStorage.removeItem('favoritesStore');
  
  // 3. Optionnel : appeler API de déconnexion (blacklist token)
  await api.post('/auth/logout');
  
  // 4. Rediriger vers login
  navigation.reset({ index: 0, routes: [{ name: 'Login' }] });
};
Bonnes pratiques : Toujours effacer TOUS les tokens (access + refresh). Ne jamais laisser de données orphelines.
Statistique : 34% des failles de sécurité mobiles viennent de tokens mal effacés (OWASP).

5. SecureStore vs AsyncStorage

Critère SecureStore AsyncStorage
🔒 Chiffrement✅ Oui (matériel)❌ Non (texte clair)
⚡ PerformanceLégèrement plus lentTrès rapide
📦 CapacitéQuelques KoPlusieurs Mo
🎯 UsageTokens, mots de passePréférences, cache
Règle d'or : Données sensibles → SecureStore. Données non sensibles → AsyncStorage.

Les clés de chiffrement ne quittent jamais l'environnement sécurisé matériel.

Conforme aux normes de sécurité mobile (OWASP MASVS)

Bonnes pratiques & Récapitulatif

  • ✅ Ne stockez JAMAIS de tokens en clair dans AsyncStorage.
  • ✅ Utilisez des clés distinctes : accessToken, refreshToken, userData.
  • ✅ À la déconnexion, effacez TOUTES les données SecureStore + AsyncStorage.
  • ✅ Gérez les erreurs de lecture/écriture (try/catch).
  • ✅ Pour React Native CLI, utilisez react-native-encrypted-storage.
Formule de sécurité : $$ \text{Sécurité}_{mobile} = \text{SecureStore} + \text{JWT court} + \text{Refresh Token} $$

Les tokens stockés dans SecureStore résistent aux attaques par extraction de fichiers (root/jailbreak).
