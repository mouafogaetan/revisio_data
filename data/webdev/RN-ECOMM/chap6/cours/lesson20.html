🔐 Configuration de Google Sign-In

Pour intégrer Google Sign-In avec Expo, on utilise le package expo-auth-session et expo-auth-session/providers/google.

# Installation
npx expo install expo-auth-session expo-random expo-web-browser

# Configuration dans app.json
{
  "expo": {
    "scheme": "monapp",
    "ios": {
      "bundleIdentifier": "com.monapp.ecommerce",
      "config": {
        "google": {
          "iosClientId": "VOTRE_IOS_CLIENT_ID"
        }
      }
    },
    "android": {
      "package": "com.monapp.ecommerce",
      "config": {
        "google": {
          "androidClientId": "VOTRE_ANDROID_CLIENT_ID"
        }
      }
    }
  }
}
// Composant GoogleSignInButton
import * as Google from 'expo-auth-session/providers/google';

const GoogleSignInButton = () => {
  const [request, response, promptAsync] = Google.useAuthRequest({
    iosClientId: 'VOTRE_IOS_CLIENT_ID',
    androidClientId: 'VOTRE_ANDROID_CLIENT_ID',
  });

  useEffect(() => {
    if (response?.type === 'success') {
      const { authentication } = response;
      // Envoyer le token au backend
      handleGoogleLogin(authentication.accessToken);
    }
  }, [response]);

  return (
    <TouchableOpacity onPress={() => promptAsync()}>
      <Text>Se connecter avec Google</Text>
    </TouchableOpacity>
  );
};

🍎 Configuration Apple Sign-In

Apple Sign-In est disponible via expo-apple-authentication. Nécessite un compte développeur Apple.

# Installation
npx expo install expo-apple-authentication

# Configuration app.json
{
  "expo": {
    "ios": {
      "usesAppleSignIn": true,
      "bundleIdentifier": "com.monapp.ecommerce"
    }
  }
}
import AppleAuthentication from 'expo-apple-authentication';

const AppleSignInButton = () => {
  const handleAppleSignIn = async () => {
    try {
      const credential = await AppleAuthentication.signInAsync({
        requestedScopes: [
          AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
          AppleAuthentication.AppleAuthenticationScope.EMAIL,
        ],
      });
      // credential.user, credential.email, credential.identityToken
      await handleAppleLogin(credential);
    } catch (error) {
      console.log('Apple Sign-In annulé ou échoué', error);
    }
  };

  return (
    <AppleAuthentication.AppleAuthenticationButton
      buttonType={AppleAuthentication.AppleAuthenticationButtonType.SIGN_IN}
      buttonStyle={AppleAuthentication.AppleAuthenticationButtonStyle.BLACK}
      cornerRadius={8}
      onPress={handleAppleSignIn}
    />
  );
};
⚠️ Note : Apple Sign-In ne fonctionne que sur un vrai appareil iOS (pas sur simulateur sans configuration spécifique).

🔄 Gestion des callbacks et création de compte

Après réception du token, on l'envoie à notre backend pour créer ou connecter l'utilisateur.

// Service d'authentification sociale
const handleSocialLogin = async (provider, token, userInfo) => {
  try {
    const response = await axios.post('/auth/social', {
      provider,      // 'google' ou 'apple'
      token,
      email: userInfo.email,
      name: userInfo.name,
    });
    
    const { user, accessToken } = response.data;
    // Stocker dans Zustand store
    useAuthStore.getState().setUser(user, accessToken);
    
    // Redirection vers l'accueil
    navigation.replace('Home');
  } catch (error) {
    if (error.response?.status === 409) {
      // Compte existant avec email - proposition de lier
      showLinkingDialog();
    } else {
      showToast('Erreur lors de la connexion');
    }
  }
};

✅ Bonne pratique : Toujours gérer le cas où l'email est déjà utilisé avec un mot de passe classique. Proposez de lier les comptes.

🔗 Lier plusieurs comptes sociaux

Permettre aux utilisateurs déjà connectés d'ajouter une méthode de connexion supplémentaire.

// Dans l'écran des paramètres
const LinkSocialAccount = ({ provider }) => {
  const { user, linkAccount } = useAuthStore();
  
  const handleLink = async () => {
    const token = await getSocialToken(provider);
    await axios.post('/auth/link', {
      provider,
      token,
      userId: user.id
    });
    showToast(`${provider} lié avec succès`);
  };
  
  if (user?.linkedProviders?.includes(provider)) {
    return <Text>✓ {provider} déjà lié</Text>
  }
  
  return <Button onPress={handleLink} title={`Lier ${provider}`} />
};

📌 Pourquoi lier les comptes ? L'utilisateur peut se connecter indifféremment avec Google ou Apple au même compte, évitant les doublons.

📝 Exercices d'application

Exercice 1 (compétence – medium)

Implémentez un composant SocialLoginButtons qui affiche les boutons Google et Apple, gère les états de chargement, et redirige vers l'accueil après succès.

🔍 Voir corrigé 
const SocialLoginButtons = () => {
  const [loading, setLoading] = useState(false);
  const [googleRequest, googleResponse, googlePrompt] = Google.useAuthRequest({...});
  
  useEffect(() => {
    if (googleResponse?.type === 'success') {
      setLoading(true);
      handleSocialLogin('google', googleResponse.authentication.accessToken);
    }
  }, [googleResponse]);
  
  return (
    <View className="gap-3">
      <TouchableOpacity 
        onPress={() => googlePrompt()}
        disabled={loading}
        className="bg-white border border-gray-300 py-3 rounded-lg flex-row justify-center items-center"
      >
        {loading ? <ActivityIndicator /> : <>🅶 Se connecter avec Google</>}
      </TouchableOpacity>
      <AppleAuthenticationButton onPress={handleAppleSignIn} />
    </View>
  );
};

✅ Bonnes pratiques & récapitulatif

  • Toujours gérer le cas où l'utilisateur annule l'authentification
  • Stocker les tokens dans expo-secure-store (pas AsyncStorage)
  • Pour iOS, Apple Sign-In est obligatoire si vous utilisez Google Sign-In
  • Testez sur des vrais appareils (simulateur limité pour ces auth)
  • Utilisez des schémas URL uniques dans app.json pour le retour de l'auth
  • Proposez toujours un fallback (connexion email/mot de passe)
