Préparer les assets (icônes, captures), configurer App Store Connect, soumettre à l'Apple Review.

🎯 Objectifs de la leçon

1️⃣
Préparer les assets iOS : icônes (App Icon) et captures d'écran pour toutes les tailles d'écran.
2️⃣
Configurer App Store Connect : fiche produit, tarification, confidentialité.
3️⃣
Générer le bundle IPA avec EAS Build (ou Xcode) et le soumettre au TestFlight / Review.
4️⃣
Comprendre les critères de l'Apple Review et les pièges à éviter.
📌 Une publication sur l'App Store nécessite un compte développeur Apple (99$/an).

🎨 Icônes et assets

Apple exige une icône d'application au format PNG avec des tailles précises (20x20, 29x29, 40x40, 58x58, 60x60, 76x76, 80x80, 87x87, 120x120, 152x152, 167x167, 180x180 pixels, etc.). Sous Expo, la configuration est simplifiée.

// app.json - configuration des icônes
{
  "expo": {
    "icon": "./assets/icon.png",      // icône principale 1024x1024
    "ios": {
      "icon": "./assets/icon-ios.png" // spécifique iOS (optionnel)
    }
  }
}

💡 Bonne pratique : Utiliser un outil comme App Icon Generator (online) pour générer toutes les tailles à partir d'une image 1024x1024. Éviter la transparence.

📱

Icône 1024x1024 px → génère automatiquement les déclinaisons

📸 Captures d'écran

App Store Connect exige des captures d'écran pour chaque taille d'iPhone/iPad. Tailles recommandées : 6.5" (iPhone 15 Pro Max) : 1290x2796 px, 5.5" (iPhone 8 Plus) : 1242x2208 px.

📋 Checklist des captures :

  • Minimum 3 captures (jusqu'à 10)
  • Format JPEG ou PNG, pas de transparence
  • Montrer l'application en action (UI réelle)
  • Pas de contenu trompeur ou non conforme
  • Pour iPad : captures obligatoires si l'app supporte l'iPad
📱 Sous Expo, on peut générer les captures via un émulateur iOS (Commande + S) et les redimensionner.

📋 Configuration App Store Connect

📝 Informations générales

Nom, sous-titre (30 caractères), catégorie, âge minimum.

💰 Tarification

Gratuit, payant (Prix tiers Apple) ou abonnement.

🔒 Confidentialité

Renseigner les types de données collectées (localisation, contacts, etc.) et le lien vers la politique de confidentialité.

📱 TestFlight

Permet de distribuer une version bêta à des testeurs avant publication officielle.

💡 À savoir : Apple peut rejeter une application si le lien de politique de confidentialité est invalide ou si les permissions demandées ne sont pas justifiées.

🏗️ Génération du bundle IPA (EAS Build)

EAS Build d'Expo permet de compiler l'application iOS sans avoir besoin d'un Mac physique (serveurs Expo).

# Commandes EAS pour iOS
eas build --platform ios --profile production
eas submit --platform ios
🔐 Certificats et profils – EAS gère automatiquement les signing certificates (Apple Developer Portal).
📦 Fichier .ipa – Le bundle final est prêt à être soumis via Transporter ou directement via `eas submit`.

Note : Il faut avoir lié son compte Apple Developer dans EAS.

🚀 TestFlight & soumission

Avant la publication publique, on peut distribuer la version à des testeurs internes/externes via TestFlight. Cela permet de détecter les bugs en conditions réelles.

🧪 Test interne

Jusqu'à 100 testeurs (membres de l'équipe). Sans validation d'Apple.

🌍 Test externe

Nécessite une première validation par Apple (Beta App Review). Jusqu'à 10 000 testeurs.

✅ Soumission finale : Une fois la version prête, on choisit “Soumettre à l’App Review”. Apple examine généralement en 24-48h. Attention aux périodes de congés (fins d’année).

⚠️ Pièges fréquents de l'Apple Review

❌
Fonctionnalités incomplètes – Ex: bouton "Se connecter" qui ne fonctionne pas en mode démo.
❌
Demande de permissions non justifiée – Expliquer pourquoi l'app a besoin de la localisation.
❌
Interface utilisateur non terminée – Éléments mal alignés, texte coupé, boutons invisibles.
❌
Liens brisés ou contenu inapproprié – Politique de confidentialité invalide, contenu offensant.
💡 Apple fournit un rapport de rejet détaillé (Resolution Center). Il faut corriger et soumettre à nouveau.

🏁 Récapitulatif

✅ Générer les icônes 1024x1024 et configurer app.json
✅ Captures d'écran pour iPhone et iPad obligatoires
✅ Remplir la fiche App Store Connect (tarif, catégorie, confidentialité)
✅ Build IPA avec EAS → TestFlight → soumission finale
✅ Anticiper les critères Apple Review pour éviter les rejets
