📣 Notifications push avec Expo

Chapitre 10 : Engager l'utilisateur

Apprenez à configurer et envoyer des notifications push pour fidéliser vos clients e-commerce.

expo-notifications Push notifications iOS & Android Engagement client

🎯 Objectifs de la leçon

1

Configurer expo-notifications dans un projet Expo

2

Gérer les permissions sur iOS et Android correctement

3

Recevoir et afficher les notifications de statut de commande

4

Gérer les notifications lorsque l'application est au premier plan ou en arrière-plan

💡 Pourquoi c'est important ? Les notifications push permettent d'informer vos clients en temps réel : confirmation de commande, mise à jour du statut de livraison, offres promotionnelles. Elles augmentent le taux de retour dans l'application.

📦 Installation et configuration

Pour utiliser les notifications push avec Expo, installez la bibliothèque expo-notifications.

Commande d'installation
npx expo install expo-notifications expo-device

Pour que les notifications fonctionnent sur Android, ajoutez dans app.json :

{
  "expo": {
    "android": {
      "googleServicesFile": "./google-services.json",
      "useNextNotificationsApi": true
    }
  }
}
📌 Note importante : Sur iOS, vous devez avoir un compte Apple Developer actif et configurer les certificats push via App Store Connect.

🔔 Gestion des permissions

Avant d'envoyer des notifications, vous devez demander l'autorisation à l'utilisateur. Voici comment procéder :

import * as Notifications from 'expo-notifications';
import * as Device from 'expo-device';

async function registerForPushNotificationsAsync() {
  let token;
  
  // Vérifier si l'appareil est physique (les simulateurs ne supportent pas les push)
  if (Device.isDevice) {
    const { status: existingStatus } = await Notifications.getPermissionsAsync();
    let finalStatus = existingStatus;
    
    if (existingStatus !== 'granted') {
      const { status } = await Notifications.requestPermissionsAsync();
      finalStatus = status;
    }
    
    if (finalStatus !== 'granted') {
      alert('Permission refusée !');
      return;
    }
    
    token = await Notifications.getExpoPushTokenAsync();
    console.log('Token Expo :', token);
  } else {
    alert('Utilisez un vrai appareil pour tester les push');
  }
  
  return token;
}
✅ Sur Android, les permissions sont automatiquement ajoutées. Sur iOS, une boîte de dialogue s'affiche.

📲 Recevoir et afficher les notifications

Pour gérer les notifications reçues, on utilise des listeners :

// Configurer le comportement des notifications
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: true,
    shouldSetBadge: true,
  }),
});

// Écouter les notifications reçues (app ouverte)
const subscription = Notifications.addNotificationReceivedListener(notification => {
  console.log('Notification reçue :', notification);
});

// Écouter les clics sur notification
const responseSubscription = Notifications.addNotificationResponseReceivedListener(response => {
  const data = response.notification.request.content.data;
  console.log('Utilisateur a cliqué, data:', data);
  // Exemple : naviguer vers l'écran de détail commande
  if (data.orderId) {
    navigation.navigate('OrderDetails', { orderId: data.orderId });
  }
});
🧠 Bon à savoir : Les notifications peuvent contenir des données personnalisées (orderId, productId, etc.) pour orienter l'utilisateur.

📡 Envoi de notifications depuis votre serveur

Une fois le token Expo récupéré côté mobile, vous pouvez l'envoyer à votre backend. Pour envoyer une notification, utilisez l'API Expo :

// Côté serveur (Node.js)
const fetch = require('node-fetch');

async function sendPushNotification(expoPushToken, title, body, data = {}) {
  const message = {
    to: expoPushToken,
    sound: 'default',
    title: title,
    body: body,
    data: data,
    badge: 1
  };
  
  await fetch('https://exp.host/--/api/v2/push/send', {
    method: 'POST',
    headers: {
      'Accept': 'application/json',
      'Accept-Encoding': 'gzip, deflate',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(message),
  });
}

// Exemple : notification de statut commande
sendPushNotification(
  token,
  'Commande expédiée ! 🚚',
  'Votre commande #1234 est en route vers Douala',
  { orderId: '1234', screen: 'OrderDetails' }
);
⚡ Astuce : Pour les applications e-commerce, envoyez des notifications lors du changement de statut commande (confirmée, expédiée, livrée).

📱 États de l'application

Les notifications se comportent différemment selon l'état de l'application :

🟢 Premier plan (Foreground)

La notification est reçue mais n'apparaît pas automatiquement. Vous devez la gérer via le NotificationHandler ou afficher un toast.

🔵 Arrière-plan (Background)

La notification s'affiche dans le centre de notifications. L'application peut réagir au clic via addNotificationResponseReceivedListener.

🟠 App fermée (Quit/Closed)

La notification s'affiche. Au clic, l'application se lance et vous pouvez accéder aux données via les paramètres initiaux.

📌 Récapitulatif & bonnes pratiques

  • ✅ Toujours demander la permission et respecter le refus de l'utilisateur.
  • ✅ Stocker le token Expo sur votre serveur associé à l'utilisateur (pour envoyer des notifications personnalisées).
  • ✅ Ne pas spammer : envoyez des notifications pertinentes (statut commande, promo limitée, rappel panier abandonné).
  • ✅ Gérer les erreurs de token (token invalide → supprimer du backend).
  • ✅ Tester sur appareils physiques car les simulateurs/émulateurs ne supportent pas les notifications push.

🧪 Cas pratique pour le Cameroun :

Informez vos clients de l'arrivée de leurs colis à Douala, Yaoundé ou Garoua. Utilisez les notifications pour les alertes de flash sales pendant les fêtes de fin d'année !
