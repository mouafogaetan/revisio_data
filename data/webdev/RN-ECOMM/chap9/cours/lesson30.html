👤 Affichage du profil utilisateur

Les données utilisateur sont généralement stockées dans un store Zustand après connexion.

// Récupération du profil depuis le store
const ProfileScreen = () => {
  const { user, isLoading, fetchUser } = useAuthStore();
  
  useEffect(() => {
    if (!user) fetchUser();
  }, []);
  
  if (isLoading) return <ActivityIndicator size="large" />;
  
  return (
    <View className="p-4">
      <Image source={{ uri: user?.avatar }} className="w-24 h-24 rounded-full" />
      <Text className="text-xl font-semibold">{user?.name}</Text>
      <Text className="text-gray-500">{user?.email}</Text>
      <Text className="text-gray-500">📞 {user?.phone || 'Non renseigné'}</Text>
      <TouchableOpacity className="mt-4 bg-blue-500 p-3 rounded-lg">
        <Text className="text-white text-center">Modifier le profil</Text>
      </TouchableOpacity>
    </View>
  );
};

📌 Bonne pratique : Utilisez un skeleton loader pendant le chargement pour une meilleure UX.

✏️ Formulaire de modification

Avec React Hook Form et Zod pour la validation.

import { useForm, Controller } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';

const profileSchema = z.object({
  name: z.string().min(2, 'Nom trop court'),
  email: z.string().email('Email invalide'),
  phone: z.string().regex(/^[0-9]{9}$/, 'Téléphone invalide (9 chiffres)'),
});

const EditProfileForm = ({ user, onSave }) => {
  const { control, handleSubmit, formState: { errors, isSubmitting } } = useForm({
    resolver: zodResolver(profileSchema),
    defaultValues: user,
  });
  
  const onSubmit = async (data) => {
    await onSave(data);
  };
  
  return (
    <View>
      <Controller control={control} name="name" render={({field}) => (
        <TextInput {...field} placeholder="Nom complet" />
      )} />
      {errors.name && <Text className="text-red-500">{errors.name.message}</Text>}
      {/* Champs email et phone similaires */}
      <Button title="Enregistrer" onPress={handleSubmit(onSubmit)} disabled={isSubmitting} />
    </View>
  );
};

📸 Changer la photo de profil

expo-image-picker permet de sélectionner une image dans la galerie ou de prendre une photo.

# Installation
npx expo install expo-image-picker

# Utilisation
import * as ImagePicker from 'expo-image-picker';

const changeAvatar = async () => {
  // Demander la permission
  const { status } = await ImagePicker.requestMediaLibraryPermissionsAsync();
  if (status !== 'granted') return;
  
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    allowsEditing: true,
    aspect: [1, 1],
    quality: 0.8,
    base64: true,
  });
  
  if (!result.canceled) {
    const imageUri = result.assets[0].uri;
    const base64 = result.assets[0].base64;
    // Upload vers le backend
    await updateProfilePicture(imageUri, base64);
    // Mettre à jour le store local
    useAuthStore.getState().setUser({ ...user, avatar: imageUri });
  }
};
⚠️ Note : Sur iOS, il faut ajouter les permissions dans `app.json` sous `ios.infoPlist`.

🔄 Synchronisation avec l'API

Après modification locale, on envoie les données au backend pour persistance.

// Service de mise à jour du profil
const updateProfile = async (userId, data) => {
  const formData = new FormData();
  formData.append('name', data.name);
  formData.append('email', data.email);
  formData.append('phone', data.phone);
  
  if (data.avatar && data.avatar !== user.avatar) {
    formData.append('avatar', {
      uri: data.avatar,
      type: 'image/jpeg',
      name: 'avatar.jpg',
    });
  }
  
  const response = await axios.put(`/users/${userId}`, formData, {
    headers: { 'Content-Type': 'multipart/form-data' }
  });
  
  return response.data;
};

// Dans le composant
const handleSave = async (data) => {
  setLoading(true);
  try {
    const updatedUser = await updateProfile(user.id, data);
    useAuthStore.getState().setUser(updatedUser);
    showToast('Profil mis à jour');
    navigation.goBack();
  } catch (error) {
    showToast('Erreur lors de la mise à jour');
  } finally {
    setLoading(false);
  }
};

const AvatarPicker = ({ currentAvatar, onAvatarChange }) => {
  const showImagePicker = async (useCamera = false) => {
    const pickerMethod = useCamera ? ImagePicker.launchCameraAsync : ImagePicker.launchImageLibraryAsync;
    const result = await pickerMethod({

✅ Bonnes pratiques & récapitulatif

  • Validez toujours les données côté client (Zod) et côté serveur
  • Affichez des skeletons pendant le chargement initial du profil
  • Utilisez expo-image-manipulator pour redimensionner les images avant upload
  • Stockez l'avatar en base64 seulement si les images sont petites, sinon utilisez un service de stockage (Cloudinary, AWS S3)
  • Après modification, mettez à jour à la fois le store local et le cache React Query
  • Permettez à l'utilisateur d'annuler les modifications non sauvegardées
