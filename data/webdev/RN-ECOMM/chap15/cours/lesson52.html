📴 Architecture du mode hors ligne

Pour qu'une application e-commerce fonctionne sans connexion, il faut :

1️⃣ Stockage local

AsyncStorage / SQLite / WatermelonDB pour persister le catalogue.

2️⃣ Synchronisation

NetInfo + background fetch pour mettre à jour en arrière-plan.

3️⃣ File d'attente

Stockage des actions (ajouts panier, commandes) pour replay.

4️⃣ UI adaptative

Indicateur "hors ligne" + icône de synchronisation.


// Détection de connexion avec NetInfo
import NetInfo from '@react-native-community/netinfo';

const [isConnected, setIsConnected] = useState(true);

useEffect(() => {
  const unsubscribe = NetInfo.addEventListener(state => {
    setIsConnected(state.isConnected);
    if (state.isConnected) syncPendingOrders();
  });
  return unsubscribe;
}, []);

💾 Persistance du catalogue produit

Utilisation de Zustand avec middleware persist pour sauvegarder les produits.


// store/productStore.js
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

export const useProductStore = create(
  persist(
    (set, get) => ({
      products: [],
      isLoading: false,
      lastSync: null,
      fetchProducts: async () => {
        set({ isLoading: true });
        try {
          const response = await api.get('/products');
          set({ products: response.data, lastSync: Date.now() });
        } catch (error) {
          // En cas d'erreur, on garde le cache existant
          console.log('Mode hors ligne, utilisation du cache');
        } finally {
          set({ isLoading: false });
        }
      },
      getProductByBarcode: (barcode) => {
        return get().products.find(p => p.barcode === barcode);
      }
    }),
    {
      name: 'products-storage',
      storage: createJSONStorage(() => AsyncStorage),
    }
  )
);
💡 Astuce : Au premier lancement, téléchargez tout le catalogue (si < 5000 produits). Sinon, utilisez la pagination + mise en cache progressive.

📷 Scan de codes-barres pour ajout rapide

Permet d'ajouter un produit au panier en scannant son code EAN-13.


import { BarCodeScanner } from 'expo-barcode-scanner';
import { useState, useEffect } from 'react';

export default function BarcodeScanner({ onProductFound }) {
  const [hasPermission, setHasPermission] = useState(null);
  const [scanned, setScanned] = useState(false);

  useEffect(() => {
    (async () => {
      const { status } = await BarCodeScanner.requestPermissionsAsync();
      setHasPermission(status === 'granted');
    })();
  }, []);

  const handleBarCodeScanned = ({ type, data }) => {
    setScanned(true);
    // data = code-barres scanné (ex: 9782070542341)
    const product = useProductStore.getState().getProductByBarcode(data);
    if (product) {
      onProductFound(product);
    } else {
      Alert.alert('Produit non trouvé', 'Ce code-barres n\'existe pas dans notre catalogue');
    }
    setTimeout(() => setScanned(false), 2000);
  };

  if (hasPermission === null) return <Text>Demande de permission...</Text>;
  if (!hasPermission) return <Text>Permission refusée</Text>;

  return (
    <BarCodeScanner
      onBarCodeScanned={scanned ? undefined : handleBarCodeScanned}
      style={{ width: '100%', height: 400 }}
    />
  );
}

🎤 Commandes vocales avec expo-speech

Permet aux utilisateurs de rechercher des produits ou d'ajouter au panier par la voix.


import * as Speech from 'expo-speech';
import Voice from '@react-native-voice/voice'; // Reconnaissance vocale

// Synthèse vocale (lire le texte)
const speakProduct = (productName) => {
  Speech.speak(`Voici ${productName}`, {
    language: 'fr-FR',
    pitch: 1,
    rate: 0.9
  });
};

// Reconnaissance vocale
useEffect(() => {
  Voice.onSpeechResults = (e) => {
    const command = e.value[0].toLowerCase();
    if (command.includes('recherche')) {
      const query = command.replace('recherche', '').trim();
      searchProducts(query);
    } else if (command.includes('panier')) {
      navigation.navigate('Cart');
    }
  };
  return () => Voice.destroy().then(Voice.removeAllListeners);
}, []);

📦 File d'attente des commandes hors ligne

Les commandes passées sans connexion sont stockées et synchronisées automatiquement.


// store/orderQueueStore.js
export const useOrderQueueStore = create((set, get) => ({
  pendingOrders: [],
  
  addPendingOrder: (order) => {
    set(state => ({
      pendingOrders: [...state.pendingOrders, { ...order, id: Date.now(), status: 'pending' }]
    }));
  },
  
  syncPendingOrders: async () => {
    const { pendingOrders } = get();
    if (!isConnected || pendingOrders.length === 0) return;
    
    for (const order of pendingOrders) {
      try {
        await api.post('/orders', order);
        set(state => ({
          pendingOrders: state.pendingOrders.filter(o => o.id !== order.id)
        }));
      } catch (error) {
        console.log('Échec synchro, réessaie plus tard');
      }
    }
  }
}));

// Appel automatique au retour de connexion
NetInfo.addEventListener(state => {
  if (state.isConnected) useOrderQueueStore.getState().syncPendingOrders();
});

📱 Interface utilisateur hors ligne

Indicateurs visuels pour que l'utilisateur comprenne l'état de synchronisation.

🟢 En ligne

Catalogue à jour. Les commandes sont envoyées immédiatement.

🔴 Hors ligne

Catalogue disponible. Les commandes sont mises en file d'attente.

const NetworkStatus = () => {

  if (isConnected === null) return null;

      </Text>
    </View>
  );
};

❓ Question ressource (niveau medium)

Expliquez pourquoi le mode hors ligne est particulièrement important pour les applications e-commerce en Afrique subsaharienne.

Voir la réponse

En Afrique subsaharienne, la couverture réseau est inégale et les données mobiles restent chères. Le mode hors ligne permet : 1) de consulter le catalogue sans connexion (dans le bus, en zone rurale), 2) de passer commande même sans internet (synchro ultérieure), 3) de réduire la consommation de données, 4) d'améliorer l'expérience utilisateur dans les zones blanches.
