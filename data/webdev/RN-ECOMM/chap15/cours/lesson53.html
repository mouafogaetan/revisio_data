📦 Installation et configuration

Dans un projet Expo, installer le module Firebase via expo-firebase-analytics (ou le SDK Web).

# Installation des dépendances
expo install firebase
expo install @react-native-firebase/app @react-native-firebase/analytics

# Configuration firebase.js
import { initializeApp } from 'firebase/app';
import { getAnalytics, logEvent } from 'firebase/analytics';

const firebaseConfig = {
  apiKey: "VOTRE_API_KEY",
  projectId: "VOTRE_PROJECT_ID",
  // ... autres infos
};

const app = initializeApp(firebaseConfig);
export const analytics = getAnalytics(app);
export { logEvent };
⚠️ Sur iOS, il faut ajouter le fichier `GoogleService-Info.plist` (téléchargé depuis Firebase Console).

📊 Événements personnalisés

Firebase recommande d'utiliser les noms d'événements standardisés pour l'e-commerce (view_item, add_to_cart, begin_checkout, purchase).

// Tracker la vue d'un produit
import { analytics, logEvent } from './firebase';

const trackViewProduct = (product) => {
  logEvent(analytics, 'view_item', {
    currency: 'XAF',
    value: product.price,
    items: [{
      item_id: product.id,
      item_name: product.name,
      price: product.price,
      quantity: 1
    }]
  });
};

// Tracker l'ajout au panier
const trackAddToCart = (product, quantity) => {
  logEvent(analytics, 'add_to_cart', {
    currency: 'XAF',
    value: product.price * quantity,
    items: [{
      item_id: product.id,
      item_name: product.name,
      price: product.price,
      quantity: quantity
    }]
  });
};

💰 Événement d'achat (purchase)

L'événement `purchase` est crucial pour mesurer le chiffre d'affaires et le ROI des campagnes.

// Après confirmation de paiement (webhook Stripe)
const trackPurchase = (order) => {
  logEvent(analytics, 'purchase', {
    transaction_id: order.id,
    currency: 'XAF',
    value: order.totalAmount,
    tax: order.tax,
    shipping: order.shippingCost,
    items: order.items.map(item => ({
      item_id: item.productId,
      item_name: item.name,
      price: item.price,
      quantity: item.quantity
    }))
  });
};
💡 Le paramètre `transaction_id` évite les doublons dans les rapports Firebase.

📐 Taux de conversion (entonnoir)

On mesure l'efficacité du parcours d'achat avec le taux de conversion :

$$Taux_{conversion} = \frac{N_{achats}}{N_{vues\\_produit}} \times 100$$

Exemple : 150 achats pour 3000 vues produit → 5% de conversion.

🔎 Exemple d'entonnoir Firebase :

  • Étape 1 : `view_item` (1000 événements)
  • Étape 2 : `add_to_cart` (300 événements) → 30%
  • Étape 3 : `begin_checkout` (180 événements) → 18%
  • Étape 4 : `purchase` (120 événements) → 12% global

🐞 Debug View (Firebase)

Activer le mode debug sur l'appareil pour voir les événements en temps réel dans la console Firebase.

# Sur Android (terminal)
adb shell setprop debug.firebase.analytics.app com.monapp.id

# Sur iOS (simulateur ou appareil)
# Dans Xcode, ajouter l'argument -FIRDebugEnabled
✅ Événements visibles en direct
⚠️ Désactiver le debug avant publication

👤 Propriétés utilisateur

On peut associer des attributs (segment, niveau d'activité) à l'utilisateur pour mieux analyser les comportements.

// Définir des propriétés utilisateur
import { getAnalytics, setUserProperties } from 'firebase/analytics';

setUserProperties(analytics, {
  favorite_category: 'electronics',
  is_premium_user: isPremium ? 'yes' : 'no',
  signup_method: 'google'
});
💡 Les propriétés utilisateur sont utiles pour filtrer les rapports (ex: comparer le comportement des utilisateurs premium vs gratuits).
