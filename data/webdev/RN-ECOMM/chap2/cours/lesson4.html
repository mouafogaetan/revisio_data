Tailwind CSS pour React Native

Expo + NativeWind Classes utilitaires Responsive mobile

Apprenez à styliser vos applications React Native avec la puissance et la simplicité de Tailwind CSS.

1

💡 À la fin, vous saurez styliser rapidement n'importe quel composant React Native avec la syntaxe Tailwind.

📱 Qu'est-ce que NativeWind ?

NativeWind est une bibliothèque qui permet d'utiliser Tailwind CSS dans React Native.

  • Transforme les classes Tailwind en styles React Native natifs
  • Support des pseudo-classes (hover, active) et dark mode
  • Fonctionne avec Expo et React Native CLI
  • Recommandé par la communauté Expo

📦 Alternative : tailwind-react-native-classnames mais moins puissant.

// Avant (StyleSheet classique)
<View style={{ padding: 16, backgroundColor: 'blue' }}>
  <Text style={{ fontSize: 18, color: 'white' }}>Bonjour</Text>
</View>

// Avec NativeWind
<View className="p-4 bg-blue-500">
  <Text className="text-lg text-white">Bonjour</Text>
</View>

⚙️ Installation dans un projet Expo

Étape 1 : Créer un projet Expo

npx create-expo-app MonApp --template blank-typescript
cd MonApp

Étape 2 : Installer NativeWind

npm install nativewind tailwindcss-react-native
npm install --save-dev tailwindcss

Étape 3 : Initialiser Tailwind

npx tailwindcss init

Crée un fichier tailwind.config.js.

✅ Important : Pour Expo SDK 49+, utiliser tailwindcss-react-native.

🔧 Configuration avancée

tailwind.config.js : personnalisation du thème

module.exports = {
  content: ["./App.{js,jsx,ts,tsx}", "./src/**/*.{js,jsx,ts,tsx}"],
  theme: {
    extend: {
      colors: {
        primary: '#FF6B35',
        secondary: '#1E3A8A'
      },
      fontFamily: {
        'sans': ['Inter', 'system-ui']
      }
    }
  },
  plugins: []
}

babel.config.js : ajouter le plugin

module.exports = {
  presets: ['babel-preset-expo'],
  plugins: ['nativewind/babel']
}

💡 Astuce : Utilisez `extend` pour ne pas écraser les couleurs par défaut de Tailwind.

🎨 Utilisation des classes utilitaires

NativeWind supporte la plupart des classes Tailwind :

Layout

flex, flex-row, justify-between, items-center, p-4, m-2

Typographie

text-lg, font-bold, text-center, text-gray-700

Couleurs & fonds

bg-blue-500, bg-red-100, border border-gray-300

Dimensions

w-full, h-32, w-1/2, max-w-md, min-h-screen 
// Exemple : Carte produit
<View className="bg-white rounded-xl shadow-md p-4 m-2">
  <Image className="w-full h-40 rounded-lg" source={{ uri: product.image }} />
  <Text className="text-lg font-bold mt-2">{product.name}</Text>
  <Text className="text-teal-600 font-semibold">{product.price} XAF</Text>
  <TouchableOpacity className="bg-teal-500 p-3 rounded-lg mt-2">
    <Text className="text-white text-center">Ajouter au panier</Text>
  </TouchableOpacity>
</View>

📱 Responsive avec les breakpoints

NativeWind supporte les breakpoints Tailwind pour adapter l'interface à la taille de l'écran :

  • sm: → 640px (iPhone SE, petits téléphones)
  • md: → 768px (tablettes portrait)
  • lg: → 1024px (tablettes paysage / petits desktop)
  • xl: → 1280px (grands écrans)
<View className="flex-col sm:flex-row">
  <Text className="text-sm md:text-base lg:text-lg">
    Ce texte grossit sur les grands écrans
  </Text>
  <View className="w-full sm:w-1/2 lg:w-1/3">
    <!-- Largeur adaptative -->
  </View>
</View>

📌 Bon à savoir : Sur mobile, on part toujours du principe "mobile first" (style par défaut pour petit écran, puis md: et lg: pour agrandir).

🌙 Dark mode & Pseudo-classes

NativeWind supporte les variantes :

  • active: → lorsqu'un élément est pressé
  • dark: → mode sombre (basé sur le thème système ou personnalisé)
  • hover: → (limité sur mobile mais utile pour web)
// Bouton avec feedback actif
<TouchableOpacity className="bg-blue-500 active:bg-blue-700 p-3 rounded-lg">
  <Text className="text-white">Appuyez-moi</Text>
</TouchableOpacity>

// Support du dark mode
<View className="bg-white dark:bg-gray-900">
  <Text className="text-black dark:text-white">
    Thème adaptatif
  </Text>
</View>

💡 Pour le dark mode, il faut configurer le thème dans tailwind.config.js avec darkMode: 'class' et gérer le changement via un contexte.

✨ Bonnes pratiques

  • Créer des composants réutilisables : évitez la répétition de longues chaînes de classes.
  • Utiliser cn() (class-variance-authority) pour les variantes conditionnelles.
  • Ne pas surcharger : une ligne trop longue de classes nuit à la lisibilité.
  • Organiser le thème : personnalisez les couleurs dans tailwind.config.js plutôt que d'utiliser des valeurs arbitraires.

📘 Exemple : créez un composant Button qui accepte variant="primary" ou "secondary" et applique les classes correspondantes.

// Composant personnalisé
function CustomButton({ children, variant }) {
  const base = "px-4 py-2 rounded-lg font-semibold";
  const variants = {
    primary: "bg-teal-500 text-white",
    secondary: "bg-gray-200 text-gray-800"
  };
  return <TouchableOpacity className={`${base} ${variants[variant]}`}>
    <Text>{children}</Text>
  </TouchableOpacity>
}
