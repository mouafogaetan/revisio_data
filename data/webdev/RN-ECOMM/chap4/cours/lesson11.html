📦 Installation de Zustand

Pourquoi Zustand ? C'est une petite bibliothèque d'état global qui utilise les hooks React. Pas de boilerplate, pas de Provider obligatoire, et elle fonctionne parfaitement avec TypeScript.

# Installation dans un projet Expo / React Native
npm install zustand

# Pour la persistance (AsyncStorage)
npm install @react-native-async-storage/async-storage

💡 Info : Zustand ne nécessite pas de contexte. On crée un store avec create() et on le réutilise directement.

🔐 Créer le store d'authentification

Le authStore gère l'utilisateur connecté, le token JWT, et les actions de connexion/déconnexion.

// stores/authStore.js
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

const useAuthStore = create(
  persist(
    (set) => ({
      user: null,
      token: null,
      isLoading: false,
      login: async (email, password) => {
        set({ isLoading: true });
        try {
          // Appel API simulé
          const response = await fakeApiLogin({ email, password });
          set({ user: response.user, token: response.token, isLoading: false });
        } catch (error) {
          set({ isLoading: false });
          throw error;
        }
      },
      logout: () => set({ user: null, token: null }),
    }),
    {
      name: 'auth-storage', // clé dans AsyncStorage
    }
  );
);

export default useAuthStore;
📌 Utilisation dans un composant : 
const { user, login, logout } = useAuthStore();
🧠 Explication :

Le middleware persist sauvegarde automatiquement le store dans AsyncStorage. Au redémarrage, l'utilisateur reste connecté.

🛒 Panier d'achat (CartStore)

Le cartStore contient les articles, les quantités, et permet d'ajouter/supprimer/vider le panier. On active aussi la persistance pour que le panier survive à la fermeture de l'app.

// stores/cartStore.js
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

const useCartStore = create(
  persist(
    (set, get) => ({
      items: [],     // { id, name, price, quantity, image }
      total: 0,
      addItem: (product) => {
        const existing = get().items.find(i => i.id === product.id);
        let newItems;
        if (existing) {
          newItems = get().items.map(i =>
            i.id === product.id ? { ...i, quantity: i.quantity + 1 } : i
          );
        } else {
          newItems = [...get().items, { ...product, quantity: 1 }];
        }
        const newTotal = newItems.reduce((sum, i) => sum + (i.price * i.quantity), 0);
        set({ items: newItems, total: newTotal });
      },
      removeItem: (id) => {
        const newItems = get().items.filter(i => i.id !== id);
        const newTotal = newItems.reduce((sum, i) => sum + (i.price * i.quantity), 0);
        set({ items: newItems, total: newTotal });
      },
      clearCart: () => set({ items: [], total: 0 }),
    }),
    {
      name: 'cart-storage',
      storage: createJSONStorage(() => AsyncStorage),
    }
  )
);

export default useCartStore;
✅ Astuce : On calcule le total dynamiquement à chaque modification pour garantir la cohérence.

⚡ Utilisation dans un composant React

Avec Zustand, on importe le store et on sélectionne les données/actions nécessaires. Le composant se met à jour automatiquement.

import useAuthStore from './stores/authStore';
import useCartStore from './stores/cartStore';

function ProductScreen({ product }) {
  const { user } = useAuthStore();           // seulement l'utilisateur
  const { addItem, items } = useCartStore(); // actions du panier

  const handleAddToCart = () => {
    if (!user) {
      alert('Veuillez vous connecter');
      return;
    }
    addItem(product);
  };

  return (
    <View>
      <Text>{product.name}</Text>
      <Button title="Ajouter au panier" onPress={handleAddToCart} />
      <Text>🛒 {items.length} article(s)</Text>
    </View>
  );
}

💾 Persistance et AsyncStorage

Le middleware persist permet de sauvegarder automatiquement le state dans le stockage local. Très utile pour garder le panier ou le token.

📱 Avantages

  • Données conservées après redémarrage
  • Mode offline partiel
  • Configuration simple : une ligne name

⚠️ Limitations

  • Ne pas stocker d'infos ultra sensibles (préférer SecureStore)
  • Éviter les données volumineuses (limite perf)

Le total du panier se calcule par :

$$ \text{Total} = \sum_{i=1}^{n} (\text{prix}_i \times \text{quantité}_i) $$

🧹 Organisation recommandée

📁 stores/
├── authStore.js
├── cartStore.js
└── index.js (export centralisé)

✨ Règles d'or

  • Un store par domaine fonctionnel
  • Actions simples et immutables
  • Ne pas mettre de données dérivées dans le store (calculer dans le composant ou selector)
📘 Rappel : Zustand fonctionne aussi bien avec TypeScript, on peut typer le store avec State & Actions pour une meilleure autocomplétion.
