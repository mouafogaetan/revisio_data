🔧 React Native Debugger

React Native Debugger est une application standalone qui combine :

  • Redux DevTools (inspecteur d'état)
  • React DevTools (arbre des composants)
  • Network inspecteur (requêtes API)
  • Console JavaScript
# Installation (macOS)
brew install --cask react-native-debugger

# Ou téléchargement direct depuis GitHub

Pour l'utiliser :

# Lancer l'app en mode debug
npm run ios -- --simulator="iPhone 14"

# Dans le simulateur : Cmd + D → Debug Remote JS
# (Se connecte automatiquement à React Native Debugger)

💡 Astuce : Redux DevTools permet d'inspecter l'état du store, de dispatcher des actions manuellement, et de faire du time-travel debugging.

🐬 Flipper - Plateforme de débogage Meta

Flipper (anciennement Sonar) est l'outil officiel de Meta pour déboguer les apps React Native.

# Installation (macOS)
brew install --cask flipper

# Plugins recommandés :
# - React DevTools
# - Network (inspecter les requêtes)
# - Layout (inspecter la hiérarchie native)
# - Hermes Debugger (pour le moteur Hermes)
# - Redux Inspector

📱 Pour Expo : Flipper n'est pas supporté nativement. Utiliser Expo DevTools (expo start --dev-client) ou React Native Debugger.

// Activer Flipper dans une app RN CLI (iOS/Android)
// Rien à faire, c'est inclus depuis RN 0.62

⚡ Profilage des performances avec Hermes

Hermes est le moteur JavaScript par défaut depuis React Native 0.70. Il permet le profilage directement.

// Activer Hermes dans metro.config.js
module.exports = {
  transformer: {
    hermesParser: true,
  },
};

Pour profiler :

  • Ouvrir Chrome DevTools (chrome://inspect)
  • Onglet "Performance"
  • Lancer un enregistrement
  • Identifier les "long tasks" (> 16ms)
  • Optimiser les composants lents
// Mesurer le temps d'exécution
console.time('renderProductList');
// ... code
console.timeEnd('renderProductList');

🎯 Objectif : Maintenir 60fps (16ms par frame) pour une expérience fluide.

🚨 Sentry - Surveillance des crashs en production

Sentry capture automatiquement les erreurs JavaScript, les crashs natifs, et fournit des rapports détaillés.

npm install @sentry/react-native
npx sentry-wizard -i reactNative
// App.tsx
import * as Sentry from '@sentry/react-native';

Sentry.init({
  dsn: 'https://votre-dsn@sentry.io/projet',
  tracesSampleRate: 1.0,
  environment: process.env.NODE_ENV,
  release: 'monapp@1.0.0',
});

// Capture manuelle d'une erreur
try {
  // code risqué
} catch (error) {
  Sentry.captureException(error);
}

// Ajouter du contexte utilisateur
Sentry.setUser({ id: userId, email: userEmail });

📊 Sentry - Fonctionnalités avancées

  • Performance monitoring : mesure la durée des transactions (chargement écran, appel API).
  • Breadcrumbs : suivi des actions utilisateur avant le crash (navigation, clics).
  • Source maps : désobfuscation des stack traces (upload automatique avec `sentry-cli`).
  • Release health : taux de crash par version, sessions utilisateur.
// Transaction manuelle
const transaction = Sentry.startTransaction({ name: 'cart-checkout' });
// ... traitement
transaction.finish();

// Breadcrumb
Sentry.addBreadcrumb({
  category: 'navigation',
  message: 'Navigué vers produit 123',
  level: 'info',
});

📌 Bonnes pratiques : Ne pas capturer les erreurs 4xx (utilisateur), seulement les erreurs 5xx ou les inattendues.

Chrome DevToolsPerformance, Console, Sources✅ Oui
SentryCrashs production, monitoring✅ Oui
Expo DevToolsLogs, métadonnées, reload✅ Oui (natif)

📋 Bonnes pratiques de débogage

  • Développement : utiliser React Native Debugger + console.log stratégique.
  • Performance : profiler régulièrement avec Flipper ou Chrome DevTools.
  • Production : activer Sentry dès le début du projet.
  • Source maps : toujours uploader les source maps pour déchiffrer les stack traces.
  • Logs : ne pas laisser de `console.log` en production (les désactiver avec un plugin babel).

💡 Plugin babel pour supprimer console.log : `npm install babel-plugin-transform-remove-console --save-dev` puis ajouter dans babel.config.js.
