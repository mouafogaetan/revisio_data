🧹 Éviter les fuites mémoire avec cleanup

Les fuites mémoire surviennent quand un composant est démonté mais qu'une opération asynchrone (setTimeout, abonnement, listener) continue de s'exécuter.

❌ Mauvaise pratique :

useEffect(() => {
  const interval = setInterval(() => {
    console.log('Tick');
  }, 1000);
  // Pas de cleanup → intervalle continue après démontage
}, []);

✅ Bonne pratique :

useEffect(() => {
  const interval = setInterval(() => {
    console.log('Tick');
  }, 1000);
  
  return () => {
    clearInterval(interval); // Nettoyage !
  };
}, []);

📌 Cas courants : annuler des requêtes API (AbortController), se désabonner des événements, fermer des WebSockets, supprimer des listeners d'animation.

🛑 Annuler les requêtes API avec AbortController

useEffect(() => {
  const abortController = new AbortController();
  
  const fetchData = async () => {
    try {
      const response = await fetch('/api/products', {
        signal: abortController.signal,
      });
      const data = await response.json();
      setProducts(data);
    } catch (error) {
      if (error.name !== 'AbortError') {
        console.error(error);
      }
    }
  };
  
  fetchData();
  
  return () => {
    abortController.abort(); // Annule la requête si composant démonté
  };
}, []);

💡 Avec Axios : `const source = axios.CancelToken.source();` puis `source.cancel();`

💡 Avec React Query : le cache gère automatiquement l'annulation.

📦 useCallback : stabiliser les fonctions

useCallback retourne une version mémoïsée d'une fonction. Elle ne change que si ses dépendances changent.

// Sans useCallback → nouvelle fonction à chaque rendu
const handlePress = () => {
  console.log(count);
};
// Avec useCallback → fonction stable
const handlePress = useCallback(() => {
  console.log(count);
}, [count]); // Recrée seulement si count change

🎯 Utilité : Évite que des composants enfants mémoïsés (avec React.memo) ne se re-rendent inutilement.

🧠 useMemo : mémoïser des valeurs calculées

useMemo met en cache le résultat d'un calcul coûteux et ne le recalcute que si les dépendances changent.

// Sans useMemo → recalculé à chaque rendu
const expensiveValue = expensiveCalculation(data);

// Avec useMemo → recalculé seulement si data change
const expensiveValue = useMemo(() => {
  return expensiveCalculation(data);
}, [data]);

📊 Exemples concrets :

  • Filtrage/tri d'une grande liste de produits
  • Calcul du total du panier
  • Formatage de dates ou de nombres

🛡️ React.memo : éviter les re-rendus inutiles

React.memo est un HOC qui empêche un composant de se re-rendre si ses props n'ont pas changé (comparaison superficielle).

// Composant enfant
const ProductCard = React.memo(({ product, onPress }) => {
  return (
    <TouchableOpacity onPress={() => onPress(product.id)}>
      <Text>{product.name}</Text>
    </TouchableOpacity>
  );
});

⚠️ Attention : Si on passe une fonction sans useCallback, React.memo est inefficace car la fonction est recréée à chaque rendu parent → les props changent toujours.

// Bonne combinaison : React.memo + useCallback
const handlePress = useCallback((id) => {
  console.log(id);
}, []);

<ProductCard product={product} onPress={handlePress} />

🛍️ Exemple : Optimisation d'une liste de produits

const ProductList = ({ products, category }) => {
  // useMemo : filtre seulement si category ou products change
  const filteredProducts = useMemo(() => {
    console.log('Filtrage...');
    return products.filter(p => p.category === category);
  }, [products, category]);
  
  // useCallback : fonction stable pour React.memo
  const handleAddToCart = useCallback((productId) => {
    cartStore.addItem(productId);
  }, []);
  
  return (
    <FlatList
      data={filteredProducts}
      renderItem={({ item }) => (
        <ProductItem 
          product={item} 
          onAddToCart={handleAddToCart}
        />
      )}
      keyExtractor={item => item.id}
    />
  );
};

// Composant enfant optimisé
const ProductItem = React.memo(({ product, onAddToCart }) => {
  return (
    <View className="p-4 border-b">
      <Text>{product.name}</Text>
      <Button title="Ajouter" onPress={() => onAddToCart(product.id)} />
    </View>
  );
});

📋 Bonnes pratiques récapitulatives

  • Cleanup : toujours annuler les abonnements, timers et requêtes dans le return du useEffect.
  • useCallback : pour les fonctions passées à des composants enfants mémoïsés ou dans des effets.
  • useMemo : pour les calculs coûteux (filtres, tris, transformations de données).
  • React.memo : pour les composants qui se re-rendent souvent avec les mêmes props.
  • Ne pas sur-optimiser : useMemo/useCallback ont un coût. À réserver aux cas où le gain est visible.

📘 Règle d'or : Mesurez avant d'optimiser. Utilisez le profiler React DevTools pour identifier les re-rendus superflus.

🏁 Récapitulatif

✅ Acquis :

  • Cleanup des effets (intervalles, requêtes, listeners)
  • useCallback pour les fonctions stables
  • useMemo pour les valeurs calculées
  • React.memo pour éviter les re-rendus enfants

📌 À retenir :

Optimiser une app mobile ne se fait pas à l'aveugle : combinez ces outils avec parcimonie et mesurez l'impact réel.
