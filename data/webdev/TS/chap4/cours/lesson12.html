📐 Définition

Interface : un contrat de forme

Une interface en TypeScript décrit la forme qu’un objet doit avoir. Elle liste les noms de propriétés et leurs types. Ce n’est pas une classe, elle disparaît à la compilation.

interface Utilisateur {
  nom: string;
  age: number;
}

const alice: Utilisateur = { nom: "Alice", age: 30 };
TypeScript vérifie que l’objet respecte bien la structure définie. Une propriété manquante ou de type incorrect déclenche une erreur à la compilation.

📌 Propriétés obligatoires et optionnelles

Par défaut, une propriété est obligatoire. Pour la rendre optionnelle, ajoutez ? après son nom.

interface Produit {
  id: number;          // obligatoire
  nom: string;         // obligatoire
  description?: string; // optionnelle
}

const p1: Produit = { id: 1, nom: "Livre" }; // valide
const p2: Produit = { id: 2, nom: "Stylo", description: "Bille bleue" }; // valide aussi
⚠️ Une propriété optionnelle peut être absente ou valoir undefined.

🔒 Readonly : propriétés immuables

Avec readonly, une propriété ne peut être modifiée après l’initialisation de l’objet.

interface Carte {
  readonly id: number;
  valeur: string;
}

let carte: Carte = { id: 101, valeur: "As" };
carte.valeur = "Roi"; // ✅ OK
// carte.id = 102;     // ❌ Erreur : id est en lecture seule
📌 Idéal pour les identifiants, dates de création, ou champs qui ne doivent jamais changer après création.

🤔 Interface vs Type alias

Interface
  • Déclaration fusion (merge)
  • Spécialisée pour les objets/classes
  • Extensible avec extends
Type alias
  • Peut représenter unions, primitifs, tuples
  • Pas de fusion automatique
  • Plus polyvalent mais moins orienté objet
type ID = string | number;   // union possible
type TuplePoint = [number, number]; // tuple

interface Animal { nom: string; }
interface Chien extends Animal { aboyer(): void; }
✨ Règle simple : préférez interface pour les objets (extensibilité), type pour le reste.

🌳 Héritage d’interface : extends

Une interface peut hériter d’une autre, ce qui évite la duplication.

interface Personne {
  nom: string;
  age: number;
}

interface Employe extends Personne {
  salaire: number;
}

const emp: Employe = { nom: "Jean", age: 35, salaire: 45000 };
✅ Employe possède nom, age et salaire. L’héritage est multiple possible (ex: interface C extends A, B).

🚀 Exemple : formulaire utilisateur

Une interface permet de typer les données d’un formulaire et d’éviter les erreurs.

interface Inscription {
  email: string;
  motDePasse: string;
  age?: number; // optionnel
  readonly dateInscription: Date;
}

function enregistrer(data: Inscription) {
  console.log(data.email, data.motDePasse);
}

enregistrer({
  email: "test@example.com",
  motDePasse: "secret",
  dateInscription: new Date()
});
🎯 Grâce à l’interface, le compilateur vérifie que les champs obligatoires sont présents et du bon type.

✅ Bonnes pratiques avec les interfaces

  • Nommez les interfaces avec un nom clair (PascalCase).
  • Privilégiez interface pour les objets que vous étendez (API, props React).
  • Utilisez readonly pour les identifiants, dates de création.
  • Utilisez les propriétés optionnelles ? pour les données facultatives.
  • Exportez les interfaces depuis des modules pour les réutiliser.
💡 Une interface bien conçue documente directement le code.
