🧬 1. Étendre une interface (extends)

Une interface peut hériter d’une ou plusieurs autres interfaces. Elle reprend toutes leurs propriétés et peut en ajouter de nouvelles.

// Interface de base
interface Personne {
    nom: string;
    age: number;
}

// Étendre avec "extends"
interface Etudiant extends Personne {
    matricule: string;
    filiere: string;
}

// Utilisation
let etudiantCameroun: Etudiant = {
    nom: "Mbouombouo",
    age: 21,
    matricule: "TS2025",
    filiere: "Informatique"
};
✨ L’interface Etudiant possède désormais nom, age, matricule et filiere.

🏛️ 2. Implémenter une interface (implements)

Une classe peut implémenter une ou plusieurs interfaces. Elle doit fournir une implémentation concrète pour toutes les propriétés et méthodes déclarées.

interface Vehicule {
    marque: string;
    rouler(): void;
}

class Voiture implements Vehicule {
    marque: string;
    constructor(marque: string) {
        this.marque = marque;
    }
    rouler(): void {
        console.log(`La ${this.marque} roule au Cameroun 🚗`);
    }
}
const maVoiture = new Voiture("Toyota");
maVoiture.rouler(); // "La Toyota roule au Cameroun 🚗"
⚠️ Si une méthode manque, TypeScript génère une erreur. L'interface est un contrat obligatoire.

🔗 3. Combiner plusieurs interfaces (héritage multiple)

TypeScript permet à une interface d’étendre plusieurs interfaces à la fois en les séparant par des virgules.

interface Volant {
    voler(): void;
}
interface Nageant {
    nager(): void;
}
// Interface composite
interface Canard extends Volant, Nageant {
    couleur: string;
}

class CanardColvert implements Canard {
    couleur: string = "vert";
    voler(): void { console.log("Le canard vole !"); }
    nager(): void { console.log("Le canard nage !"); }
}

let canard = new CanardColvert();
canard.voler(); canard.nager();

✔️ La classe CanardColvert doit implémenter toutes les méthodes de Volant, Nageant et la propriété couleur.

📌 4. extends (interface) vs implements (classe)

📐 Interface + extends

Crée un nouveau type / contrat à partir d’un autre. Seulement pour les types.

interface A { a: string }
interface B extends A { b: number }

🏗️ Classe + implements

Une classe s’engage à respecter la forme d’une interface. Fournit du code concret.

class MaClasse implements MonInterface {
  // implémentation
}

🎓 Résumé : extends entre interfaces sert à composer/étendre des types. implements sert à forcer une classe à respecter un contrat.

🌍 Cas pratique : Gestion d’employés

interface Identifiable {
    id: number;
    afficherId(): string;
}

// Étendre l'interface
interface Employe extends Identifiable {
    nom: string;
    poste: string;
}

// Classe implémente l'interface étendue
class Ingenieur implements Employe {
    constructor(public id: number, public nom: string, public poste: string) {}
    afficherId(): string {
        return `ID: ${this.id} - ${this.nom}`;
    }
    travailler(): void {
        console.log(`${this.nom} développe des solutions TS au Cameroun`);
    }
}
let dev = new Ingenieur(101, "Armelle", "Dev Fullstack");
console.log(dev.afficherId());
dev.travailler();
🎯 L'interface Employe hérite de Identifiable et la classe Ingenieur les implémente toutes les deux.

Que doit faire la classe Avion pour implémenter correctement l’interface suivante ?

interface MachineVolante {
    vitesseMax: number;
    decoller(): void;
    atterrir(): void;
}
✅ La classe Avion doit déclarer la propriété vitesseMax: number et implémenter les méthodes decoller() et atterrir(). Exemple :
class Avion implements MachineVolante { vitesseMax = 900; decoller() { } atterrir() { } }
💡 Une classe peut implémenter plusieurs interfaces : class A implements X, Y, Z {}
