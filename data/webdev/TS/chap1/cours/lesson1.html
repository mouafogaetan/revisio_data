🤔 Qu'est-ce que TypeScript ?

TypeScript (TS) est un sur-ensemble de JavaScript développé par Microsoft en 2012.

  • ✅ Tout code JavaScript valide est aussi du TypeScript.
  • ✅ Il ajoute un système de typage statique optionnel.
  • ✅ Il se compile en JavaScript pur (compatible navigateur/Node.js).
TypeScript (fichier .ts) → Compilation (tsc) → JavaScript (fichier .js) → Exécution (navigateur/Node)

⚠️ Les faiblesses de JavaScript

JavaScript est dynamique et faiblement typé, ce qui peut causer des bugs silencieux :

function add(a, b) {
    return a + b;
}

console.log(add(5, "10"));  // → "510" 😱 pas 15
console.log(add(5, null));  // → "5null" 😱

Aucune erreur avant exécution. En production, cela peut planter l'application.

🐛 Les bugs de type sont fréquents dans les grandes bases de code JS.

✅ La solution : TypeScript

function add(a: number, b: number): number {
    return a + b;
}

// add(5, "10"); → ERREUR de compilation : Argument de type 'string' non assignable à 'number'
console.log(add(5, 10));  // → 15 ✅

L'erreur est détectée au moment de la compilation, avant même d'exécuter le code.

🔧 Meilleur feedback pour le développeur, moins de bugs en production.

⚙️ Compilation : du TS au JS

Le navigateur ne comprend pas le TypeScript. Il faut le transpiler en JavaScript.

📁 fichier.ts → tsc (TypeScript Compiler) → 📁 fichier.js 
// TypeScript (fichier.ts)
let message: string = "Hello TypeScript";

// JavaScript généré (fichier.js)
let message = "Hello TypeScript";

Le typage disparaît à la compilation : c'est une couche de vérification uniquement.

🌟 Les avantages de TypeScript

🔒 Typage statique

Détecte les erreurs de type avant l'exécution.

📝 Auto-complétion

Meilleure assistance dans l'éditeur (VS Code, WebStorm).

🔄 Refactoring sécurisé

Renommer une variable/fonction avec confiance.

📚 Documentation vivante

Les types servent de documentation automatique.

  • Microsoft : VS Code, Teams, Office 365.

  • Framework modernes : Next.js, NestJS, Vue 3, Svelte (optionnel).

❌ Ce que TypeScript n'est pas

  • ❌ Un nouveau langage : c'est JavaScript avec des annotations.
  • ❌ Un langage compilé bas-niveau : il transpile vers JS, pas vers du binaire.
  • ❌ Une sécurité absolue : les types sont vérifiés à la compilation, pas à l'exécution.
  • ❌ Réservé aux experts : on peut l'adopter progressivement.
⚠️ TS ne protège pas contre les erreurs logiques ou les failles de sécurité, uniquement contre les erreurs de type.

📝 Premier exemple concret

// Une fonction simple avec TypeScript
function greet(name: string): string {
    return `Bonjour ${name} !`;
}

// TypeScript infère le type de la variable
let userName = "Paul";  // type string

console.log(greet(userName));   // ✅ Bonjour Paul !
// console.log(greet(123));     // ❌ Erreur : 123 n'est pas une string

Le compilateur TS nous protège contre l'appel avec un nombre.

🧠 Inférence de type

TypeScript devine souvent le type tout seul :

let age = 25;        // TypeScript infère 'number'
let name = "Alice";  // infère 'string'

// Pas besoin de préciser le type tout le temps
let result = age + 10;  // infère 'number'

L'inférence fonctionne pour les variables, les retours de fonctions, etc.

💡 On laisse TS inférer quand c'est évident, on précise quand on veut contrôler le contrat.

⚖️ JS vs TS : comparaison rapide

JavaScript

function multiply(a, b) {
  return a * b;
}
multiply(5, "chat"); // → NaN (pas d'erreur)

TypeScript

function multiply(a: number, b: number): number {
  return a * b;
}
multiply(5, "chat"); // ❌ Erreur à la compilation
