Premiers pas avec le compilateur tsc, configuration tsconfig.json et mode watch

TypeScript tsc tsconfig.json

Lesson 3 · Chapitre 1 · Installation et compilation

🎯 Objectifs de la leçon

1. Écrire une fonction TypeScript simple avec typage des paramètres.
2. Compiler un fichier .ts en JavaScript avec tsc et exécuter le résultat.
3. Utiliser les options du compilateur : --watch, --noEmitOnError.
4. Créer et configurer un fichier tsconfig.json pour un projet.
💡 Rappel : TypeScript se compile en JavaScript pur, compréhensible par les navigateurs et Node.js.

✍️ Écrire sa première fonction typée

Voici une fonction qui additionne deux nombres. On ajoute des types aux paramètres et à la valeur de retour.

// fichier: addition.ts
function addition(a: number, b: number): number {
    return a + b;
}

const resultat = addition(5, 3);
console.log(`Résultat : ${resultat}`);
✅ TypeScript vérifie que les arguments sont bien des nombres. Essayez addition(5, "3") → erreur de compilation !

⚙️ Compiler avec tsc

Après avoir installé TypeScript (npm install -g typescript), on compile :

# Compilation simple
tsc addition.ts

# Exécution du fichier JS généré
node addition.js

💡 Astuce :

Le compilateur génère un fichier addition.js dans le même dossier. Vous pouvez ensuite l'exécuter avec Node.js ou l'inclure dans une page HTML.

🔧 Options utiles de tsc

tsc --watch (ou -w)

Surveille les fichiers .ts et recompile automatiquement à chaque modification.

tsc --noEmitOnError

Ne génère pas de fichier .js si une erreur de type est détectée.

tsc --target ES2020

Définit la version ECMAScript cible (ES5, ES6, ES2020, etc.).

tsc --outDir ./dist

Place les fichiers compilés dans un dossier spécifique (ex: dist/).

📄 Fichier tsconfig.json

Au lieu d'écrire les options à chaque fois, on crée un fichier de configuration.

// tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",          // Version JS cible
    "module": "commonjs",        // Système de modules
    "strict": true,              // Vérifications strictes
    "outDir": "./dist",          // Dossier de sortie
    "rootDir": "./src",          // Dossier source
    "noEmitOnError": true        // Pas de JS si erreur
  },
  "include": ["src/**/*"]
}
👉 Ensuite, il suffit de taper tsc (sans arguments) pour compiler tout le projet.

🚀 Mise en pratique

Créons un petit projet :

mon-projet/
├── src/
│   └── index.ts
├── dist/
└── tsconfig.json

Fichier src/index.ts :

function saluer(nom: string): string {
    return `Bonjour ${nom} !`;
}
console.log(saluer("Cameroun"));
🔁 Lancez tsc --watch. Désormais, à chaque sauvegarde, le fichier dist/index.js est automatiquement regénéré.

🏁 Vérification et exécution

Après compilation, exécutez le JavaScript généré :

node dist/index.js
# Affiche : Bonjour Cameroun !

📌 Points clés à retenir :

  • Le navigateur/Node.js n'exécute que du JavaScript → il faut toujours compiler le TypeScript.
  • tsc valide les types avant de générer le JS.
  • tsc --watch accélère le développement.
  • Le fichier tsconfig.json centralise les options du projet.
