Tableaux, Tuples & Readonly

Typer des collections et garantir l'immutabilité

👨‍🏫 Classe Développement Web • Cameroun

🎯 Objectifs du cours

  • ✓ Typer un tableau : string[], Array<number>
  • ✓ Utiliser les tuples (taille fixe avec types spécifiques) : [string, number]
  • ✓ Comprendre les tableaux en lecture seule (readonly)
💡 À la fin, vous saurez manipuler des collections typées et protéger vos données contre les modifications accidentelles.

📋 Typer un tableau

Deux syntaxes possibles pour déclarer un tableau typé :

// Syntaxe 1 : type[]
let fruits: string[] = ["pomme", "banane", "orange"];

// Syntaxe 2 : Array<type>
let ages: Array<number> = [18, 25, 30, 42];

Le compilateur vérifiera que chaque élément du tableau correspond au type déclaré.

// ❌ Erreur : 'number' n'est pas assignable à 'string'
fruits.push(123);

🔢 Tableaux homogènes

Un tableau TypeScript ne peut contenir que des éléments d'un même type (sauf union).

let mix: (string | number)[] = ["hello", 42, "world", 100];
// ✅ Autorisé grâce à l'union string | number

let onlyNumbers: number[] = [1, 2, 3];
// ❌ onlyNumbers.push("quatre"); // Erreur
💡 Si vous avez besoin de types différents, utilisez les tuples (taille fixe) ou les unions.

🧠 Inférence des tableaux

TypeScript infère automatiquement le type d'un tableau à partir de ses éléments :

let colors = ["rouge", "vert", "bleu"];  // TypeScript infère : string[]
let scores = [10, 20, 30];               // Infère : number[]

// ✅ Ajout d'un élément compatible
colors.push("jaune");

// ❌ Erreur : type 'number' non assignable à 'string'
colors.push(100);

⚙️ Opérations sur les tableaux typés

let nombres: number[] = [1, 2, 3, 4, 5];

// Lecture
let premier: number = nombres[0];     // TypeScript sait que c'est un number

// Ajout
nombres.push(6);                      // OK
// nombres.push("sept");              // ❌ Erreur

// Méthodes fonctionnelles
let doubles: number[] = nombres.map(n => n * 2);
let pairs: number[] = nombres.filter(n => n % 2 === 0);
let somme: number = nombres.reduce((acc, n) => acc + n, 0);

🔷 Les tuples : tableau à taille fixe

Un tuple est un tableau dont la taille est fixée et chaque position a un type spécifique.

// Tuple représentant un couple (nom, âge)
let personne: [string, number] = ["Alice", 30];

// Accès typé
let nom: string = personne[0];   // ✅ TypeScript sait que c'est une string
let age: number = personne[1];   // ✅ TypeScript sait que c'est un number

// ❌ Erreur : type 'boolean' non assignable à 'string'
personne[0] = true;

📌 Cas d'usage des tuples

// Coordonnées GPS (latitude, longitude)
let position: [number, number] = [48.8566, 2.3522];

// Réponse d'API (status, message)
let apiResult: [number, string] = [200, "Succès"];

// État d'un formulaire (champ, valeur, estValide)
let formField: [string, any, boolean] = ["email", "alice@example.com", true];

// Plusieurs types différents (taille fixe)
let mixte: [string, number, boolean] = ["résultat", 42, true];
📦 Les tuples sont parfaits pour retourner plusieurs valeurs d'une fonction.

🔘 Tuples optionnels et paramètres rest

// Éléments optionnels (avec ?)
let tupleOptional: [string, number?] = ["Alice"];
tupleOptional = ["Bob", 25];  // Ok aussi

// Paramètre rest (éléments en trop)
let tupleRest: [string, ...number[]] = ["Alice", 25, 30, 35];
// Premier élément : string, les suivants : number (autant qu'on veut)

let tupleMixte: [boolean, ...string[], number] = [true, "a", "b", 10];

🛡️ Tableaux en lecture seule (readonly)

readonly empêche la modification du tableau après sa création.

let nombres: readonly number[] = [1, 2, 3];

// ❌ Erreur : la propriété 'push' n'existe pas sur 'readonly number[]'
nombres.push(4);

// ❌ Erreur : impossible d'assigner à un élément d'un tableau readonly
nombres[0] = 10;

// On peut toujours lire
console.log(nombres[0]);  // ✅ Ok
console.log(nombres.length); // ✅ Ok

📖 ReadonlyArray<T>

Le type générique ReadonlyArray<T> est équivalent à readonly T[].

let fruits: ReadonlyArray<string> = ["pomme", "banane"];

// ❌ fruits.push("orange"); // Erreur
// ❌ fruits[0] = "poire";   // Erreur

// On peut créer un nouveau tableau à partir d'un readonly
let copie: string[] = [...fruits];  // ✅ Copie modifiable
🔒 Utilisez readonly pour les paramètres de fonction qui ne doivent pas être modifiés.

🎯 Exemple concret : fonction avec tuple

// Fonction retournant un tuple
function diviser(a: number, b: number): [number, number] {
    const quotient = Math.floor(a / b);
    const reste = a % b;
    return [quotient, reste];
}

const [q, r] = diviser(17, 5);
console.log(`Quotient: ${q}, Reste: ${r}`);  // Quotient: 3, Reste: 2

// Fonction acceptant un tableau readonly
function afficherTous(items: readonly string[]): void {
    items.forEach(item => console.log(item));
    // items.push("nouveau"); // ❌ Erreur
}

afficherTous(["a", "b", "c"]);

✅ Bonnes pratiques

  • Utilisez readonly pour les paramètres qui ne doivent pas être mutés.
  • Préférez les tuples pour les retours multiples d'une fonction.
  • Évitez les tuples trop longs (préférez une interface ou une classe).
  • Laissez TypeScript inférer le type des tableaux si l'inférence est claire.
  • Utilisez const pour les tableaux immutables (mais const ne bloque que la réaffectation, pas les modifications internes → utilisez readonly).
