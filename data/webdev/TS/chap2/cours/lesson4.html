📝 Pourquoi typer ses variables ?

TypeScript ajoute des types à JavaScript pour détecter les erreurs avant l'exécution.

❌ JavaScript (problème)

function add(a, b) {
  return a + b;
}
add(5, "10"); // "510" (concaténation !)

✅ TypeScript (sécurisé)

function add(a: number, b: number): number {
  return a + b;
}
add(5, "10"); // ❌ Erreur de compilation

🔢 Les types primitifs : number, string, boolean

number

Nombres entiers ou décimaux

let age: number = 25;
let prix: number = 19.99;
let hex: number = 0xFF;   // 255
let bin: number = 0b1010; // 10
string

Chaînes de caractères (guillemets simples, doubles, backticks)

let nom: string = "Alice";
let message: string = 'Bonjour';
let template: string = `Bienvenue ${nom}`;
boolean

Vrai ou faux

let estConnecte: boolean = true;
let estAdmin: boolean = false;

❓ null et undefined

null

Absence intentionnelle de valeur

let valeurVide: null = null;
let utilisateur: string | null = null;
undefined

Variable déclarée mais non initialisée

let nonDefini: undefined = undefined;
let x: number; // x est undefined

⚠️ strictNullChecks : Avec l'option activée, null et undefined ne sont pas assignables aux autres types.

🚫 Le type void

void représente l'absence de type. On l'utilise principalement pour les fonctions qui ne retournent rien.

function afficherMessage(message: string): void {
    console.log(message);
    // Pas de return (ou return;)
}

let resultat: void = undefined; // void = undefined ou null

Une fonction avec : void ne doit pas retourner de valeur explicite.

⚠️ Le type any

any désactive la vérification des types – à éviter autant que possible !

let valeur: any = "texte";
valeur = 42;         // OK
valeur.toUpperCase(); // ⚠️ Erreur à l'exécution !

Utilisation uniquement : migration progressive de JS vers TS, ou valeurs vraiment dynamiques (rare).

🔒 Le type unknown

unknown est une alternative sécurisée à any. On ne peut pas l'utiliser sans vérifier son type d'abord.

let data: unknown = "Hello";

// ❌ Erreur : data n'est pas forcément une chaîne
// console.log(data.toUpperCase());

// ✅ OK : vérification du type
if (typeof data === "string") {
    console.log(data.toUpperCase());
}

unknown = any avec sécurité (nécessite un narrowing)

💻 Exemple complet

// Déclaration typée
let username: string = "Jean";
let age: number = 28;
let isPremium: boolean = true;
let error: null = null;
let temp: undefined = undefined;

// Fonction avec void
function greet(user: string): void {
    console.log(`Bonjour ${user}`);
}

// Gestion d'unknown sécurisé
function processInput(input: unknown) {
    if (typeof input === "string") {
        return input.toUpperCase();
    }
    if (typeof input === "number") {
        return input * 2;
    }
    return "Type non supporté";
}
