💡 Activer "strict": true dans tsconfig.json est la meilleure pratique pour tout nouveau projet TypeScript.

🛡️ 1. Le mode strict

L'option "strict": true active un ensemble de vérifications pour rendre TypeScript plus sûr. C'est le mode recommandé pour tous les projets.

{
  "compilerOptions": {
    "strict": true,   // Active toutes les vérifications strictes
    "target": "ES2020",
    "outDir": "./dist",
    "rootDir": "./src"
  }
}
✅ Bonne pratique : Toujours activer strict pour les nouveaux projets.
⚠️ Sur un projet existant, activez-le progressivement.

🚫 2. strictNullChecks

Quand activé, null et undefined ne sont plus automatiquement assignables aux autres types.

// Avec strictNullChecks: true
let nom: string;
nom = null;        // ❌ Erreur : null n'est pas assignable à string
nom = undefined;   // ❌ Erreur

let age: number | null = null;  // ✅ OK car union avec null
age = 25;                        // ✅ OK

function afficherLongueur(texte: string | null) {
    // Nécessite une vérification avant d'accéder à .length
    if (texte !== null) {
        console.log(texte.length);
    }
    // Ou utiliser l'optional chaining
    console.log(texte?.length);
}
💡 Cela évite les fameuses erreurs Cannot read property of null à l'exécution.

❓ 3. noImplicitAny

Empêche TypeScript de déduire implicitement le type any quand il ne peut pas inférer le type.

// Avec noImplicitAny: true

function maFonction(param) {  // ❌ Erreur : param a un type 'any' implicite
    return param;
}

// Solution : typer explicitement
function maFonction(param: string): string {
    return param;
}

// ou si vraiment nécessaire, utiliser any explicitement
function maFonction(param: any): any {
    return param;
}
⚠️ Forcer le typage explicite rend le code plus lisible et évite les bugs silencieux.

🔄 4. strictFunctionTypes

Applique une vérification plus stricte sur la compatibilité des paramètres de fonctions.

// Avec strictFunctionTypes: true

type Animal = { nom: string };
type Chien = Animal & { race: string };

let ajouterAnimal = (a: Animal) => {};
let ajouterChien = (c: Chien) => {};

ajouterAnimal = ajouterChien;  // ✅ OK : Chien est un sous-type d'Animal
ajouterChien = ajouterAnimal;  // ❌ Erreur : Animal n'a pas 'race'

// Cela empêche les assignations dangereuses entre fonctions
📌 Garantit que les fonctions ne reçoivent pas plus de paramètres qu'elles ne peuvent traiter.

🔧 5. Autres options strictes

strictPropertyInitialization

Vérifie que toutes les propriétés de classe sont initialisées dans le constructeur.

strictBindCallApply

Vérifie les types des arguments pour bind, call, apply.

noImplicitReturns

Tous les chemins d'une fonction doivent retourner une valeur.

noFallthroughCasesInSwitch

Empêche les "fallthrough" accidentels dans les switch.

📁 6. paths et baseUrl (alias d'import)

Simplifiez vos imports en évitant les chemins relatifs complexes (../../../utils/helpers).

// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",           // Répertoire de base pour la résolution
    "paths": {
      "@utils/*": ["src/utils/*"],
      "@components/*": ["src/components/*"],
      "@models/*": ["src/models/*"]
    }
  }
}

// Dans votre code
// Avant : import { helper } from '../../../utils/helpers';
// Après :
import { helper } from '@utils/helpers';
import { Bouton } from '@components/Bouton';
🚀 Avantage : Code plus propre, réorganisation des dossiers facilitée.

🧪 Testez vos connaissances

Avec "strictNullChecks": true, quelle est la manière correcte d'appeler cette fonction ?

function afficherNom(nom: string | null): void {
    console.log(nom?.toUpperCase() ?? "Anonyme");
}
A) afficherNom(undefined);
B) afficherNom(null);
C) afficherNom("Marie");
D) B et C sont correctes.
✅ Réponse D : Avec strictNullChecks, la fonction accepte string ou null (pas undefined). Les appels avec "Marie" ou null sont valides.
