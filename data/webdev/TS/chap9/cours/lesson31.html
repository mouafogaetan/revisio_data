📦 Typer les props d'un composant

On utilise une interface pour décrire la forme des propriétés attendues.

interface Props {
  nom: string;
  age: number;
  estEtudiant?: boolean; // optionnel
}

const CarteUtilisateur: React.FC<Props> = ({ nom, age, estEtudiant = false }) => {
  return (
    <div>
      <h2>{nom}</h2>
      <p>Âge : {age} ans</p>
      {estEtudiant && <p>Statut : Étudiant</p>}
    </div>
  );
};

🔍 Bonnes pratiques : nommer l'interface avec "Props" + nom du composant.

⚛️ useState typé

TypeScript infère souvent le type, mais on peut le préciser avec useState<Type>.

// Inférence automatique
const [compteur, setCompteur] = useState(0); // number

// Typage explicite pour des structures complexes
interface Article {
  id: number;
  titre: string;
  prix: number;
}

const [articles, setArticles] = useState<Article[]>([]);

// État avec null possible
const [utilisateur, setUtilisateur] = useState<User | null>(null);

✅ L'inférence fonctionne pour les types simples. Pour les objets, arrays ou unions, le typage explicite est recommandé.

🔗 useRef avec TypeScript

useRef permet d'accéder directement à un élément du DOM.

import { useRef } from 'react';

const InputFocus = () => {
  // Typage : l'élément référencé est un HTMLInputElement ou null
  const inputRef = useRef<HTMLInputElement>(null);

  const focusInput = () => {
    if (inputRef.current) {
      inputRef.current.focus();
    }
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Mon input" />
      <button onClick={focusInput}>Focus</button>
    </div>
  );
};

🎯 Types courants : HTMLInputElement, HTMLDivElement, HTMLButtonElement...

🖱️ Typer les événements React

React fournit des types génériques pour tous les événements DOM.

import { ChangeEvent, MouseEvent } from 'react';

const MonForm = () => {
  const handleChange = (e: ChangeEvent<HTMLInputElement>) => {
    console.log(e.target.value); // typé automatiquement
  };

  const handleClick = (e: MouseEvent<HTMLButtonElement>) => {
    e.preventDefault();
    alert('Clic détecté');
  };

  return (
    <form>
      <input onChange={handleChange} />
      <button onClick={handleClick}>Envoyer</button>
    </form>
  );
};
📌 ChangeEvent<HTMLInputElement>
📌 MouseEvent<HTMLButtonElement>
📌 FormEvent<HTMLFormElement>
📌 KeyboardEvent<HTMLInputElement>

🧩 React.FC & les alternatives modernes

React.FC (FunctionComponent) inclut automatiquement les enfants (children).

// Avec React.FC
const Container: React.FC<{ bgColor: string }> = ({ bgColor, children }) => {
  return <div style={{ backgroundColor: bgColor }}>{children}</div>;
};

// Alternative (sans FC) : typage explicite des props
interface Props { bgColor: string; children?: React.ReactNode; }
const ContainerAlt = ({ bgColor, children }: Props) => {
  return <div style={{ backgroundColor: bgColor }}>{children}</div>;
};
💡 Depuis React 18, React.FC n'inclut plus children automatiquement. Mieux vaut typer children explicitement si nécessaire.

✏️ Exercice d'application

Consigne :

Créez un composant ListeTaches avec :

  • Props : taches: string[] et titre?: string
  • État local : une nouvelle tâche (useState<string>)
  • un champ input typé avec ChangeEvent
  • un bouton pour ajouter la tâche à la liste
Voir une solution possible 
interface Props { taches: string[]; titre?: string; }
const ListeTaches: React.FC<Props> = ({ taches: initialTaches, titre = "Mes tâches" }) => {
  const [taches, setTaches] = useState(initialTaches);
  const [nouvelleTache, setNouvelleTache] = useState("");
  const handleChange = (e: ChangeEvent<HTMLInputElement>) => setNouvelleTache(e.target.value);
  const ajouter = () => { if(nouvelleTache.trim()) setTaches([...taches, nouvelleTache]); setNouvelleTache(""); };
  return (<div><h2>{titre}</h2><input value={nouvelleTache} onChange={handleChange} /><button onClick={ajouter}>+</button><ul>{taches.map((t,i)=> <li key={i}>{t}</li>)}</ul></div>);
};
🧪 Testez ce code dans un projet React + TypeScript pour valider votre compréhension.
