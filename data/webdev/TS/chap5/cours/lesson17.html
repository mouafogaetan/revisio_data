Modificateurs classiques

🔓 public & 🔒 private

Par défaut, tout est public en TypeScript. private restreint l'accès à l'intérieur de la classe uniquement.

class CompteBancaire {
  public titulaire: string;     // accessible partout
  private solde: number;        // accessible seulement dans la classe

  constructor(titulaire: string, montantInitial: number) {
    this.titulaire = titulaire;
    this.solde = montantInitial;
  }

  deposer(montant: number) {
    this.solde += montant;      // ✅ OK (dans la classe)
  }

  getSolde() {
    return this.solde;          // accessible via méthode publique
  }
}

const compte = new CompteBancaire("Alice", 1000);
console.log(compte.titulaire);   // ✅ "Alice"
// compte.solde ❌ Erreur : propriété 'solde' est privée
console.log(compte.getSolde());  // ✅ 1000
💡 Bonnes pratiques : Cacher les détails internes (private) et exposer uniquement ce qui est nécessaire.
Héritage & immutabilité

🛡️ protected & readonly

protected : accessible dans la classe et ses sous-classes.
readonly : ne peut être modifié après l'initialisation.

class Animal {
  public name: string;
  protected age: number;   // accessible dans les classes filles
  readonly espece: string;

  constructor(name: string, age: number, espece: string) {
    this.name = name;
    this.age = age;
    this.espece = espece;
  }
}

class Chien extends Animal {
  public afficherAge() {
    return `${this.name} a ${this.age} ans`; // ✅ protected accessible
  }
}

const rex = new Chien("Rex", 3, "Canin");
rex.name = "Rexou";   // ✅ OK
// rex.age = 4;       // ❌ erreur : protected
// rex.espece = "Felin"; // ❌ erreur : readonly (immuable)
console.log(rex.afficherAge());
✅ readonly → sécurité pour les données constantes (identifiant, date création).
✅ protected → partage maîtrisé avec les classes enfants.
ECMAScript 2022+

🔒 Champs privés #

Le véritable champ privé au niveau JavaScript (même à l'exécution). Supporté par TypeScript moderne.

class Voiture {
  #kilometrage = 0;    // champ privé natif
  public marque: string;

  constructor(marque: string) {
    this.marque = marque;
  }

  rouler(km: number) {
    this.#kilometrage += km;
  }

  getKilometrage() {
    return this.#kilometrage;
  }
}

const tesla = new Voiture("Tesla");
tesla.rouler(150);
console.log(tesla.getKilometrage()); // 150
// console.log(tesla.#kilometrage);  // ❌ erreur syntaxique : privé
⚡ Différence avec private : private TypeScript est une vérification à la compilation. # est une vraie protection à l'exécution.
Encapsulation élégante

🔄 Accesseurs get / set

Permettent de contrôler l'accès aux propriétés, d'ajouter de la validation ou du calcul.

class Utilisateur {
  private _age: number = 0;

  get age(): number {
    return this._age;
  }

  set age(valeur: number) {
    if (valeur < 0 || valeur > 150) {
      throw new Error("Âge invalide");
    }
    this._age = valeur;
  }

  get estMajeur(): boolean {
    return this._age >= 18;
  }
}

const user = new Utilisateur();
user.age = 25;          // appel setter
console.log(user.age);  // appel getter → 25
console.log(user.estMajeur); // true
// user.age = -5; ❌ Error: Âge invalide
📢 Les accesseurs se manipulent comme des propriétés (pas de parenthèses). Idéal pour la validation.

📌 Récapitulatif visuel

public

Accessible partout (interne / externe / héritage).

private

Uniquement dans la classe qui le définit.

protected

Dans la classe + classes filles.

readonly

Lecture seule, assignation unique (constructeur).

✨ Règle d'or : Par défaut, utilisez private ou # pour les données internes, exposez via get/set ou méthodes publiques.

🧪 Testez votre compréhension

Complétez la classe Banque pour que :

  • Le solde soit privé (modificateur classique)
  • Le titulaire soit en lecture seule (readonly)
  • Un accesseur solde (getter) permette de lire le solde sans le modifier
class Banque {
  // À compléter ici
  constructor(public readonly titulaire: string, private _solde: number) {}
  
  get solde(): number {
    return this._solde;
  }
  
  deposer(montant: number) {
    this._solde += montant;
  }
}
