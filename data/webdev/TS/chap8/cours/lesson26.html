💡 Saviez-vous ? Les modules permettent de découper son code en fichiers indépendants et réutilisables.

🔍 Pourquoi les modules ?

Sans modules, tout le code partage le même espace global → conflits, difficulté de maintenance. Les modules ES6 apportent :

  • Encapsulation : chaque fichier est un module privé par défaut
  • Réutilisabilité via export / import
  • Meilleure organisation pour les projets d'équipe

📌 À retenir :

Un fichier TypeScript devient un module dès qu'il contient au moins une instruction export ou import.

export const addition = (a: number, b: number) => a + b;

📥 Importer des modules

Les différentes syntaxes d'importation :

// Importer des exports nommés
import { addition, Operation } from './maths.js';

// Importer avec alias
import { addition as add } from './maths.js';

// Importer tout le module
import * as Maths from './maths.js';

// Importer l'export par défaut
import VERSION from './maths.js';

// Importer uniquement les types (supprimé à la compilation)
import type { Operation } from './maths.js';
🎯 import type est supprimé en JavaScript final, idéal pour éviter des références circulaires inutiles.

🔄 Réexporter et agréger

On peut regrouper plusieurs modules dans un point d'entrée unique :

// 📁 index.ts
export { addition } from './maths.js';
export { default as Logger } from './logger.js';
export type { Operation } from './maths.js';

// On peut aussi tout réexporter d'un autre module
export * from './utils.js';

Astuce : export * from réexporte tous les exports nommés sauf l'export default.

🆚 CommonJS vs ES Modules

📜 CommonJS (Node.js classique)

// importer
const fs = require('fs');
// exporter
module.exports = { maFonction };

✨ ES Modules (TypeScript moderne)

import { readFile } from 'fs';
export const maFonction = () => {};

⚠️ En 2025, on recommande d'utiliser ES Modules ("module": "ESNext") pour tous les nouveaux projets.

✅ Bonnes pratiques avec les modules

  • Privilégier les exports nommés plutôt qu'un export default géant (meilleur autocomplete et refactoring).
  • Utiliser import type pour les types/interfaces afin d'éviter des imports superflus à l'exécution.
  • Organiser vos modules par fonctionnalité (dossier models/, services/, utils/).
  • Éviter les chemins trop longs en configurant paths dans tsconfig.json.

📖 Règle d'or : Un module = une responsabilité (Single Responsibility Principle).
