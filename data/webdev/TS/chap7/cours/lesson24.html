Partial, Required, Readonly, Pick, Omit, Exclude, Extract, NonNullable

📘 Lesson 24 · Maîtriser les transformations de types

🎯 Objectifs de la leçon

🔧 Utiliser Partial<T> et Required<T> pour rendre toutes les propriétés optionnelles ou obligatoires.
📖 Maîtriser Readonly<T>, Pick<T, K> et Omit<T, K> pour sélectionner ou exclure des propriétés.
⚡ Utiliser Exclude<T, U>, Extract<T, U> et NonNullable<T> pour manipuler les unions.

📦 Partial<T> & Required<T>

Partial<T> rend toutes les propriétés de T optionnelles. Required<T> fait l'inverse.

interface Utilisateur {
  nom: string;
  age?: number;
  email: string;
}

// Toutes les propriétés deviennent optionnelles
type PartialUser = Partial<Utilisateur>;
// équivaut à { nom?: string; age?: number; email?: string; }

// Toutes les propriétés deviennent obligatoires
type RequiredUser = Required<Utilisateur>;
// age n'est plus optionnel → { nom: string; age: number; email: string; }
💡 Cas pratique : Partial est idéal pour les mises à jour partielles d’un objet (PATCH API).

🛡️ Readonly<T> , Pick<T, K> , Omit<T, K>

interface Article {
  id: number;
  titre: string;
  contenu: string;
  date: Date;
}

// Readonly : empêche toute modification
type ArticleImmutable = Readonly<Article>;

// Pick : sélectionne uniquement certaines clés
type ArticlePreview = Pick<Article, "id" | "titre">;

// Omit : exclut certaines clés
type ArticleSansContenu = Omit<Article, "contenu">;
🔍 Différence clé : Pick garde les champs souhaités, Omit supprime ceux qu’on ne veut pas.

🔀 Exclude, Extract, NonNullable

Ces utilitaires agissent sur les types unions.

type T = "a" | "b" | "c";
type U = "a" | "d";

// Exclude : retire les types présents dans U
type ResultEx = Exclude<T, U>;   // "b" | "c"

// Extract : garde l'intersection
type ResultEx2 = Extract<T, U>;   // "a"

type AvecNull = string | null | undefined;
type SansNull = NonNullable<AvecNull>;   // string
✅ NonNullable est très utile pour nettoyer des types après des opérations risquées.

🌍 Exemple : formulaire avec mise à jour partielle

interface Profile {
  name: string;
  age: number;
  email: string;
  newsletter: boolean;
}

// Pour la mise à jour : tous les champs deviennent optionnels
type ProfileUpdate = Partial<Profile>;

function updateProfile(id: number, data: ProfileUpdate) {
  // seules les propriétés fournies seront modifiées
  console.log("Mise à jour partielle", data);
}

// Pick pour un affichage restreint
type PublicProfile = Pick<Profile, "name" | "age">;
🧩 Combinaison possible : Omit<Profile, "newsletter"> pour cacher certaines infos sensibles.

📋 Récapitulatif rapide

UtilitaireEffet
Partial<T>Toutes les propriétés deviennent optionnelles
Required<T>Toutes les propriétés deviennent obligatoires
Readonly<T>Propriétés non modifiables
Pick<T, K>Crée un type avec seulement les clés K
Omit<T, K>Crée un type sans les clés K
Exclude<T, U>Retire de l'union T les membres assignables à U
Extract<T, U>Garde l'intersection de T et U
NonNullable<T>Enlève null et undefined de T

🧪 Vérifie ta compréhension

Soit le type :

type User = { id: number; name: string; password: string };

Quel utilitaire utiliserais-tu pour exposer un type PublicUser contenant uniquement id et name ?

Réponse : Pick<User, "id" | "name"> ou Omit<User, "password">.

Comment rendre toutes les propriétés d’un objet readonly ?

Réponse : Readonly<MonType> transforme récursivement chaque propriété en lecture seule.
