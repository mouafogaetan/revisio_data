Mapped Types & Template Literals

Créer des types dynamiquement, transformer des structures et manipuler des chaînes au niveau type.

🔄 Mapped Types ➕ Modificateurs 📝 Template Literal Types
👆 Glissez ou utilisez les boutons 👆

🎯 Objectifs

  • ✔️ Créer des mapped types personnalisés : [P in keyof T]: T[P]
  • ✔️ Ajouter des modificateurs (+?, -?, +readonly)
  • ✔️ Utiliser les template literal types (${T}) pour combiner des chaînes
  • ✔️ Comprendre la puissance des transformations de types à la compilation
💡 Pourquoi ? Les mapped types permettent d'éviter la duplication de code et d'adapter des types existants de manière flexible.
Fondamentaux

🔄 Mapped Types : [P in keyof T]: T[P]

Un mapped type parcourt les propriétés d'un type existant pour en créer un nouveau.

type Utilisateur = {
  nom: string;
  age: number;
  email: string;
};

// Rendre toutes les propriétés optionnelles (comme Partial intégré)
type MesOptions = {
  [P in keyof Utilisateur]?: Utilisateur[P];
};
// Résultat : { nom?: string; age?: number; email?: string; }

// Rendre toutes les propriétés en lecture seule (comme Readonly)
type ImmutableUtilisateur = {
  readonly [P in keyof Utilisateur]: Utilisateur[P];
};
📘 Syntaxe : [P in keyof T] itère sur chaque clé P du type T.
Contrôle fin

➕ Modificateurs : +readonly, -?

On peut ajouter (+) ou enlever (-) les modificateurs readonly et ? (optionnel).

type Todo = {
  readonly titre: string;
  description?: string;
  termine: boolean;
};

// Enlever readonly de toutes les propriétés
type Mutable<T> = {
  -readonly [P in keyof T]: T[P];
};

// Rendre toutes les propriétés obligatoires (enlever le ?)
type RequiredProps<T> = {
  [P in keyof T]-?: T[P];
};

type TodoMutable = Mutable<Todo>;  // titre n'est plus readonly
type TodoComplet = RequiredProps<Todo>; // description devient obligatoire
-readonly → supprime l'immutabilité
-? → rend la propriété obligatoire
+readonly → ajoute readonly (par défaut)
+? → rend optionnelle (par défaut)
Cas concret

✨ Transformer un type en un autre

interface Produit {
  id: number;
  nom: string;
  prix: number;
}

// Type qui transforme chaque propriété en fonction getter
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

// Résultat : 
// {
//   getId: () => number;
//   getNom: () => string;
//   getPrix: () => number;
// }

type ProduitGetters = Getters<Produit>;
⚡ Remarque : as dans un mapped type permet de renommer les clés (ex: as `get${Capitalize<K>}`). Nous reverrons les template literals après.
Manipulation de chaînes

📝 Template Literal Types

On peut construire des types de chaînes dynamiquement, comme les templates JavaScript mais au niveau des types.

type Direction = "nord" | "sud" | "est" | "ouest";
type Deplacement = `aller ${Direction}`;
// Résultat : "aller nord" | "aller sud" | "aller est" | "aller ouest"

type Couleur = "rouge" | "vert" | "bleu";
type Taille = "petit" | "moyen" | "grand";
type Article = `${Couleur}-${Taille}`;
// "rouge-petit" | "rouge-moyen" | ... | "bleu-grand"

// Combinaison avec keyof
type Evenement = "click" | "change";
type EvenementCallback = `on${Capitalize<Evenement>}`;
// "onClick" | "onChange"
🎯 Utilité : Générer automatiquement des noms de props CSS, des types d'événements, des routes API, etc.
