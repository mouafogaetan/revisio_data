📌 1. Typer paramètres & retour

En TypeScript, on précise le type de chaque paramètre ainsi que le type de retour après les parenthèses : (param: type): typeRetour.

// Exemple de base
function additionner(a: number, b: number): number {
    return a + b;
}

const resultat: number = additionner(5, 3); // 8

// Avec inférence, TS devine le retour
function direBonjour(nom: string) {
    return `Bonjour ${nom}`; // inféré comme string
}

💡 Bonne pratique :

Toujours typer explicitement le retour pour les fonctions publiques — cela évite les surprises et améliore la documentation.

⚙️ 2. Paramètres optionnels & valeurs par défaut

Le point d'interrogation ? rend un paramètre facultatif. Les paramètres optionnels doivent être placés après les obligatoires.

// Paramètre optionnel (peut être undefined)
function afficherNom(prenom: string, nom?: string): string {
    if (nom) return `${prenom} ${nom}`;
    return prenom;
}

console.log(afficherNom("Ahmad"));     // Ahmad
console.log(afficherNom("Ahmad", "Camara"));

// Valeur par défaut (similaire ES6)
function saluer(utilisateur: string, message: string = "Bienvenue") : string {
    return `${message}, ${utilisateur}!`;
}
console.log(saluer("Fatima"));          // Bienvenue, Fatima!
console.log(saluer("Paul", "Bonjour")); // Bonjour, Paul!
🔸 Optionnel → nom?: type ⇒ peut être undefined
🔸 Défaut → param = valeur ⇒ fourni une valeur par défaut

🧩 3. Typer les paramètres REST

Le paramètre rest permet de rassembler un nombre variable d'arguments dans un tableau. On le type comme un tableau.

// Somme de tous les nombres passés
function sommeTous(...nombres: number[]): number {
    return nombres.reduce((acc, val) => acc + val, 0);
}
console.log(sommeTous(1, 2, 3, 4)); // 10

// Mélanger types fixes + rest
function description(prefix: string, ...valeurs: (string | number)[]): string {
    return `${prefix} : ${valeurs.join(", ")}`;
}
console.log(description("Résultats", 98, 76, "absent"));

📌 Important :

Le paramètre rest doit toujours être le dernier de la liste. TypeScript s'assure que les arguments restants correspondent au type du tableau.

🔍 Inférence de type & Bonnes pratiques

TypeScript peut déduire les types, mais il est parfois préférable d'être explicite pour la clarté et la sécurité.

✅ Inféré (simple)

const double = (x: number) => x * 2;
// retour inféré: number

✅ Explicite (recommandé)

const double = (x: number): number => x * 2;

🎯 Exemple complet : fonction utilitaire

function formaterPrix(prix: number, devise: string = "FCFA", optionnel?: boolean): string {
    const valeur = optionnel ? `${prix} ${devise} (TTC)` : `${prix} ${devise}`;
    return valeur;
}
console.log(formaterPrix(2500));        // 2500 FCFA
console.log(formaterPrix(3500, "$"));   // 3500 $
