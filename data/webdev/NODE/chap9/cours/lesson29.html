🧪 Supertest : Tester les requêtes HTTP

Supertest est une bibliothèque qui permet de simuler des requêtes HTTP sur une application Express (ou tout autre serveur Node) sans avoir à écouter sur un port réel. Idéal pour les tests d'intégration.

const request = require('supertest');
const app = require('../app');

describe('GET /api/users', () => {
  it('retourne la liste des utilisateurs', async () => {
    const res = await request(app)
      .get('/api/users')
      .expect(200);
    expect(res.body).toBeInstanceOf(Array);
  });
});
✅ Avantage : Pas besoin de serveur réel
⚡ Rapide et facile à intégrer avec Jest

🗄️ Base de données de test

Pour ne pas polluer votre base de développement, on utilise une base dédiée (par exemple mydb_test).

// config/testDB.js (avec Mongoose) const mongoose = require('mongoose'); const { MongoMemoryServer } = require('mongodb-memory-server'); let mongoServer; beforeAll(async () => { mongoServer = await MongoMemoryServer.create(); const uri = mongoServer.getUri(); await mongoose.connect(uri); }); afterAll(async () => { await mongoose.disconnect(); await mongoServer.stop(); });

💡 Alternative : Une base de données PostgreSQL de test gérée avec dotenv et NODE_ENV=test.

✨ Astuce : Utilisez .env.test pour charger des variables spécifiques aux tests.

🔄 Cycle complet : Authentification + CRUD

Exemple de test d'une API de gestion de tâches : création d'un utilisateur → connexion → création d'une tâche → récupération.

describe('Todo API - Cycle complet', () => {
  let token;
  let userId;

  test('Inscription utilisateur', async () => {
    const res = await request(app).post('/api/register').send({
      email: 'test@example.com',
      password: '123456'
    });
    expect(res.statusCode).toBe(201);
    userId = res.body.id;
  });

  test('Connexion et récupération du token JWT', async () => {
    const res = await request(app).post('/api/login').send({
      email: 'test@example.com',
      password: '123456'
    });
    token = res.body.token;
    expect(token).toBeDefined();
  });

  test('Créer une tâche avec le token', async () => {
    const res = await request(app)
      .post('/api/todos')
      .set('Authorization', `Bearer ${token}`)
      .send({ title: 'Apprendre Supertest', completed: false });
    expect(res.statusCode).toBe(201);
    expect(res.body.title).toBe('Apprendre Supertest');
  });
});

🧹 Nettoyage entre les tests

Jest propose des hooks (beforeEach, afterEach) pour remettre la base dans un état connu.

beforeEach(async () => { // Supprimer toutes les collections avant chaque test const collections = mongoose.connection.collections; for (const key in collections) { await collections[key].deleteMany(); } }); afterAll(async () => { await mongoose.connection.close(); });
✅ beforeEach : exécuté avant chaque test → base propre
✅ afterAll : fermeture des connexions après tous les tests

✏️ Exercice : Tester une route protégée

Consigne : Écrire un test complet pour une route GET /api/profile qui nécessite un JWT valide.

  • 1. Créer un utilisateur via /register
  • 2. Récupérer le token via /login
  • 3. Tester /api/profile avec le bon token → statut 200
  • 4. Tester /api/profile sans token → statut 401
it('devrait retourner le profil utilisateur avec un token valide', async () => {
  // 1. Inscription
  await request(app).post('/api/register').send({ email: 'paul@mail.com', password: 'secret' });
  // 2. Login
  const loginRes = await request(app).post('/api/login').send({ email: 'paul@mail.com', password: 'secret' });
  const token = loginRes.body.token;
  // 3. Récupération du profil
  const profileRes = await request(app)
    .get('/api/profile')
    .set('Authorization', `Bearer ${token}`);
  expect(profileRes.statusCode).toBe(200);
  expect(profileRes.body.email).toBe('paul@mail.com');
  
  // 4. Sans token
  const failRes = await request(app).get('/api/profile');
  expect(failRes.statusCode).toBe(401);
});
