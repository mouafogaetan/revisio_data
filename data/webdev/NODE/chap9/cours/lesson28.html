🧪 Pourquoi écrire des tests ?

Les tests unitaires permettent de vérifier chaque petite brique de votre application. Ils évitent les régressions et facilitent la collaboration en équipe.

📦 Installation de Jest

npm init -y
npm install --save-dev jest

Ajoutez dans package.json :

"scripts": {
  "test": "jest"
}
🧪 Tests unitaires 🔄 Intégration continue

✍️ Structure d’un test Jest

describe : groupe de tests / it (ou test) : cas individuel / expect : assertion

// sum.js
function sum(a, b) { return a + b; }
module.exports = sum;

// sum.test.js
const sum = require('./sum');

describe('Fonction sum', () => {
  it('additionne 1 + 2 pour donner 3', () => {
    expect(sum(1, 2)).toBe(3);
  });

  test('additionne -1 + 1 = 0', () => {
    expect(sum(-1, 1)).toBe(0);
  });
});

▶️ Lancez npm test pour voir le résultat vert ✅

🔍 Les matchers (assertions) utiles

.toBe(valeur) → égalité stricte (primitifs)
.toEqual(obj) → égalité profonde (objets/tableaux)
.toBeTruthy() / .toBeFalsy()
.toHaveLength(n) → longueur d'un tableau/chaîne
.toContain(item) → présence dans un tableau
.toThrow() → une fonction lance une erreur
expect([1,2,3]).toContain(2);
expect(() => { throw new Error('aie') }).toThrow();

🎭 Les mocks : isoler le code de l'extérieur

Quand une fonction utilise une API externe ou une base de données, on utilise jest.mock().

// api.js
const fetchData = () => fetch('https://api.exemple').then(r => r.json());
module.exports = { fetchData };

// app.js
const { fetchData } = require('./api');
async function getUser() { return await fetchData(); }

// test
jest.mock('./api');
test('getUser retourne un utilisateur mocké', async () => {
  const { fetchData } = require('./api');
  fetchData.mockResolvedValue({ id: 1, name: 'Alice' });
  const user = await getUser();
  expect(user.name).toBe('Alice');
});

⚡ mockResolvedValue simule une promesse résolue.

👀 Espionner avec jest.spyOn

Permet de surveiller qu'une méthode a bien été appelée, combien de fois, avec quels arguments.

const logger = {
  info: (msg) => console.log(msg)
};

test('vérifie que logger.info est appelé', () => {
  const spy = jest.spyOn(logger, 'info');
  logger.info('Hello');
  expect(spy).toHaveBeenCalledTimes(1);
  expect(spy).toHaveBeenCalledWith('Hello');
  spy.mockRestore(); // restaure la fonction originale
});
✅ Utile pour les logs, appels à des fonctions annexes, ou valider un comportement sans exécuter l'effet réel.

🏦 Cas pratique : service de paiement

On mock un module de paiement externe et on espionne une fonction de logging.

// paymentService.js
const pay = async (amount) => { /* appel API Stripe */ };
module.exports = { pay };

// order.js
const { pay } = require('./paymentService');
const logger = require('./logger');
async function processOrder(amount) {
  logger.info(`Tentative paiement ${amount}`);
  return await pay(amount);
}

// order.test.js
jest.mock('./paymentService');
test('processOrder appelle pay et log', async () => {
  const { pay } = require('./paymentService');
  pay.mockResolvedValue('success');
  const spyLogger = jest.spyOn(require('./logger'), 'info');
  const result = await processOrder(100);
  expect(pay).toHaveBeenCalledWith(100);
  expect(spyLogger).toHaveBeenCalledWith(expect.stringContaining('100'));
  expect(result).toBe('success');
});

✨ Bonnes pratiques avec Jest

  • Un fichier test par module : module.test.js
  • Utiliser beforeEach / afterEach pour nettoyer les mocks
  • Ne pas tester des détails d'implémentation, mais le comportement observable
  • Isoler les appels réseau : TOUJOURS mocker les requêtes HTTP
  • Lancer npm test -- --coverage pour la couverture de code
📈 Un projet bien testé = 70% de bugs en moins en production

🧠 Mini exercice à réaliser

Vous avez une fonction isAdult(age) qui renvoie true si age >= 18.

// adult.js
function isAdult(age) { return age >= 18; }
module.exports = isAdult;

Questions :

  1. Écrivez un test unitaire Jest pour valider que 20 est adulte.
  2. Écrivez un test pour vérifier que 16 n'est pas adulte.
  3. Créez un mock simple pour éviter un appel à une vraie DB (conceptuel).
🔎 Voir solution 
const isAdult = require('./adult');
test('20 est adulte', () => { expect(isAdult(20)).toBe(true); });
test('16 n’est pas adulte', () => { expect(isAdult(16)).toBe(false); });
// Pour le mock : jest.mock('./database');

🚀 Défi : valider un système d'envoi d'email

Vous développez une API d'inscription. À chaque nouvel utilisateur, on envoie un email de bienvenue. Vous devez tester la logique d'enregistrement sans envoyer de vrai mail.

emailService.js : envoiMail(destinataire, sujet)

Écrire un test Jest qui :

  • Mock le service d'email (jest.mock)
  • Vérifie que la fonction d'enregistrement appelle bien le service d'email
  • Vérifie le contenu du sujet (grâce à un spy ou mock)
💡 Indice : mock.calls ou toHaveBeenCalledWith permettent de valider les arguments.
