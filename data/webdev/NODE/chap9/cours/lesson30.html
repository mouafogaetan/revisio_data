ESLint : Le gardien du code

ESLint analyse votre code JavaScript/TypeScript pour détecter :

  • Les variables non utilisées
  • Les erreurs de syntaxe cachées
  • Les incohérences de style (guillemets, points-virgules)
  • Les pratiques dangereuses (ex: eval())
// Exemple de fichier .eslintrc.json
{
  "env": { "node": true, "es2021": true },
  "extends": "eslint:recommended",
  "rules": {
    "semi": ["error", "always"],
    "quotes": ["error", "single"],
    "no-unused-vars": "warn"
  }
}

💻 Commande : npx eslint . --fix corrige automatiquement les problèmes simples.

✨

Prettier : Élégance standardisée

Prettier reformate tout votre code selon des règles prédéfinies. Finis les débats sur les points-virgules ou les espaces !

Avant (sans Prettier)

function hello(name){return "Hello "+name}
console.log(hello('Cameroun'))

Après Prettier

function hello(name) {
  return `Hello ${name}`;
}
console.log(hello("Cameroun"));
// .prettierrc
{
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5"
}

⚡ Astuce : Intégrez Prettier à VS Code (format on save) + ESLint ensemble avec eslint-config-prettier.

🐶

Husky : Les gardes-frontières

Husky permet d'exécuter des scripts (comme ESLint ou Prettier) avant chaque git commit ou git push. Ainsi, aucun code sale ne peut entrer dans le dépôt.

# Installation

npm install --save-dev husky lint-staged
npx husky init

# .husky/pre-commit

#!/usr/bin/env sh
npx lint-staged

# package.json (lint-staged)

"lint-staged": {
  "*.js": ["eslint --fix", "prettier --write"]
}
"Un commit ne sera validé que si le code passe les tests de linting et de formatage."

🛠️ Workflow complet

Voici les étapes pour mettre en place une chaîne de qualité sur un projet Node.js :

  1. Initialiser npm : npm init -y
  2. Installer ESLint + Prettier : npm i -D eslint prettier
  3. Configurer les règles (fichiers .eslintrc.json et .prettierrc)
  4. Ajouter des scripts dans package.json : "lint": "eslint .", "format": "prettier --write ."
  5. Installer Husky et lint-staged pour automatiser avant chaque commit.
$ npm run lint    # Vérifie le code
$ npm run format  # Corrige le formatage
$ git add .
$ git commit -m "feat: code de qualité"  # => Husky exécute lint-staged
