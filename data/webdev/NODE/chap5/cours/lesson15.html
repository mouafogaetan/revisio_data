📀 1. Qu'est-ce que MongoDB ?

MongoDB est une base de données NoSQL orientée documents. Les données sont stockées sous forme de JSON (BSON), ce qui le rend parfait pour JavaScript et Node.js.

{
  "_id": ObjectId("..."),
  "nom": "Cameroun Dev",
  "email": "contact@example.com",
  "competences": ["Node.js", "MongoDB"]
}

⚡ Installation de MongoDB

  • Local : Télécharge MongoDB Community Edition et démarrez le service mongod
  • Cloud (Atlas) : Créez un cluster gratuit sur MongoDB Atlas et récupérez l'URI de connexion.
  • Module Node.js : npm install mongoose
🧪 Astuce Cameroun : MongoDB Atlas propose un serveur gratuit très accessible même avec connexion mobile.

🔌 2. Connexion et schémas Mongoose

Mongoose est un ODM (Object Document Mapper) qui facilite l’interaction avec MongoDB.

// connexion à MongoDB (ex: Atlas)
const mongoose = require('mongoose');
mongoose.connect('mongodb+srv://user:pass@cluster.mongodb.net/maBase')
  .then(() => console.log('✅ Connecté à MongoDB'))
  .catch(err => console.error(err));

📐 Définir un schéma et un modèle

const userSchema = new mongoose.Schema({
  nom: { type: String, required: true },
  email: { type: String, unique: true },
  age: Number,
  createdAt: { type: Date, default: Date.now }
});
const User = mongoose.model('User', userSchema);

💡 Le modèle permet de créer, lire, mettre à jour et supprimer des documents.

✍️ 3. CRUD - Create & Read

➕ CREATE

// méthode 1 : new + save()
const user = new User({ nom: "Alice", email: "alice@dev.cm" });
await user.save();

// méthode 2 : create()
await User.create({ nom: "Bob", email: "bob@dev.cm" });

📖 READ

// tous les documents
const users = await User.find();
// avec filtre
const adulte = await User.find({ age: { $gte: 18 } });
// un seul document par ID
const user = await User.findById("id123");
🧠 Le `find()` retourne toujours un tableau, `findById` retourne un objet ou null.

🔄 4. Update & Delete

✏️ Mettre à jour

// Mise à jour d'un document trouvé par ID
await User.findByIdAndUpdate(id, { nom: "Nouveau Nom" }, { new: true });

// Mise à jour conditionnelle
await User.updateOne({ email: "bob@dev.cm" }, { age: 30 });

🗑️ Supprimer

// Supprimer un seul document
await User.findByIdAndDelete(userId);

// Supprimer plusieurs correspondant à un filtre
await User.deleteMany({ age: { $lt: 13 } });
⚠️ Toujours vérifier l'existence du document avant update/delete dans vos API.

🌐 Exemple pratique : API avec Express + Mongoose

// app.js - route pour créer un utilisateur
app.post('/api/users', async (req, res) => {
  try {
    const newUser = await User.create(req.body);
    res.status(201).json(newUser);
  } catch (err) {
    res.status(400).json({ erreur: err.message });
  }
});

// obtenir tous les utilisateurs
app.get('/api/users', async (req, res) => {
  const users = await User.find();
  res.json(users);
});

Avec cette approche, vous construisez une API REST complète. Mongoose valide automatiquement les champs requis et les types.

💡 Rappel important :

Mongoose traduit vos opérations JavaScript en commandes MongoDB natives. Vous pouvez aussi utiliser les méthodes comme .populate() pour les relations entre collections (avancé).

🏆 Bonnes pratiques & Variables d'environnement

  • Ne jamais hardcoder l'URI MongoDB : utiliser dotenv et process.env.MONGO_URI.
  • Gérer la connexion au démarrage de l'application avec mongoose.connect() et async/await.
  • Utiliser des schémas pour contraindre les données (ex: email unique, validations).
  • Ajouter des index pour améliorer les performances sur les champs fréquemment recherchés.
# .env
MONGO_URI=mongodb+srv://admin:pass@cluster.mongodb.net/campus
// app.js
require('dotenv').config();
mongoose.connect(process.env.MONGO_URI);
