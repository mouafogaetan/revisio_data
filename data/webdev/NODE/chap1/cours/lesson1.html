🌐 Qu'est-ce que Node.js ?

Node.js n’est pas un langage, ni un framework. C’est un environnement d’exécution qui permet d’exécuter du code JavaScript en dehors d’un navigateur, directement sur le serveur ou l’ordinateur.

Il est basé sur le moteur V8 de Google Chrome (le même qui fait tourner JavaScript dans le navigateur) mais enrichi avec des capacités serveur : accès au système de fichiers, réseau, bases de données, etc.

💻 Avant Node.js : JavaScript uniquement côté front-end (navigateur)

✅ Avec Node.js : JavaScript aussi côté back-end (serveur, scripts, outils)

[Client Web] → (requête HTTP) → [Serveur Node.js] → (réponse JSON/HTML) → [Client]

⚙️ Modèle asynchrone & Event Loop

Node.js utilise une architecture mono-thread mais non-bloquante. Il ne reste jamais "coincé" à attendre une opération lente.

🔁 Cycle de l'Event Loop (simplifié)

📥 Tâches (callbacks)
→
🔄 File d'attente
→
⚡ Event Loop
→
✅ Exécution

Pendant qu'une lecture de fichier ou requête réseau a lieu, Node.js peut traiter d'autres requêtes. Résultat: performances élevées et meilleure gestion de la charge.

⛔ Bloquant (synchrone)
Une tâche lente bloque tout le programme.
✅ Non-bloquant (Node.js)
On lance une tâche et on continue, on est prévenu quand elle finit.
