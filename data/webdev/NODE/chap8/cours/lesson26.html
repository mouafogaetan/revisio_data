🔐 Qu'est-ce qu'une variable d'environnement ?

Une variable d'environnement est une valeur stockée au niveau du système d'exploitation ou du processus, qui peut être lue par votre application. Elle permet de paramétrer le comportement sans modifier le code source.

// Exemple d'accès en Node.js

console.log(process.env.PORT); // 3000

console.log(process.env.DB_PASSWORD); // monSuperMotDePasse

Avantages
  • Sécurité des secrets
  • Configuration portable
  • Facile à changer selon l’environnement
Sans variables d'environnement
  • Mots de passe en clair dans le code
  • Risques de fuite de données
  • Code moins flexible

📦 Installation de dotenv

Dotenv est un package npm qui charge les variables depuis un fichier .env dans process.env.

npm install dotenv

Configuration minimale :

// À placer au début de votre fichier principal (ex: app.js ou server.js)
require('dotenv').config();

const port = process.env.PORT || 3000;
console.log(`Serveur démarré sur le port ${port}`);

✅ Toujours appeler require('dotenv').config() le plus tôt possible dans votre application, avant tout autre code qui utilise process.env.

📄 Créer les fichiers .env et .env.example

Le fichier .env contient les vraies valeurs (sensible) et ne doit JAMAIS être commit sur Git. Le fichier .env.example sert de modèle public.

📁 .env (à ajouter dans .gitignore)

PORT=5000
DB_HOST=localhost
DB_USER=admin
DB_PASS=secret123
JWT_SECRET=supersecurekey

📁 .env.example (partagé sur GitHub)

PORT=3000
DB_HOST=localhost
DB_USER=your_db_user
DB_PASS=change_me
JWT_SECRET=your_jwt_secret_here
⚠️ Règle d'or : Ajoutez .env dans votre .gitignore et partagez uniquement .env.example pour que les autres développeurs sachent quelles variables sont nécessaires.

🌍 Gérer plusieurs environnements

Vous pouvez avoir différents fichiers .env.development, .env.production, .env.test. Dotenv permet de les charger selon NODE_ENV.

// Chargement conditionnel
const envFile = `.env.${process.env.NODE_ENV || 'development'}`;
require('dotenv').config({ path: envFile });

console.log(`Chargement de la config : ${envFile}`);
EnvironnementFichierUtilité
💻 Développement.env.developmentBase de données locale, logs détaillés
🧪 Test.env.testBase de données de test, JWT court
🚀 Production.env.productionHaute sécurité, URLs réelles, performances

✨ Bonnes pratiques et cas concret

  • Ne jamais hardcoder de secret 🔑
  • Utiliser des valeurs par défaut : process.env.PORT ?? 3000
  • Valider les variables critiques au démarrage (ex: JWT_SECRET)
  • Pour les équipes, documenter toutes les variables dans README

💻 Exemple réel : Configuration d'une connexion MongoDB avec dotenv

// .env
MONGODB_URI=mongodb://user:pass@localhost:27017/mydb

// db.js
require('dotenv').config();
const mongoose = require('mongoose');
mongoose.connect(process.env.MONGODB_URI)
  .then(() => console.log('✅ DB connectée'))
  .catch(err => console.error(err));
