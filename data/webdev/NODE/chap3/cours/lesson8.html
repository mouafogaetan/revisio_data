📦 package.json : présentation

Le fichier package.json est un document JSON qui contient les métadonnées de votre projet Node.js. Il permet de gérer les dépendances, les scripts, la version, et rend votre application portable.

🔧 Utilités principales
  • Lister les paquets installés
  • Définir des scripts personnalisés
  • Indiquer le point d'entrée de l'app
  • Partager facilement un projet (sans node_modules)
📌 Rôle clé

Avec npm install, npm lit ce fichier pour installer toutes les dépendances nécessaires.

Sans package.json, un projet Node.js est difficile à maintenir.

🛠️ Créer package.json : npm init

$ mkdir mon-projet
$ cd mon-projet
$ npm init

npm init lance un assistant interactif pour renseigner : nom, version, description, point d'entrée, etc.

⚡ Mode rapide : npm init -y génère un fichier avec les valeurs par défaut. 
{
  "name": "mon-projet",
  "version": "1.0.0",
  "description": "Premier projet Node",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Erreur: pas de test spécifié\" && exit 1"
  },
  "author": "",
  "license": "ISC"
}

👉 Ce fichier est modifiable à tout moment.

📋 Champs essentiels de package.json

"name" – Nom du projet (unique, sans majuscules, sans espaces)
"version" – Version sémantique (ex: 1.0.0)
"scripts" – Commandes personnalisables (start, dev, build)
"dependencies" – Paquets nécessaires en production
"devDependencies" – Outils de développement seulement (tests, linters…)
"main" – Point d'entrée de l'application (index.js par défaut)
💡 Une bonne connaissance de ces champs vous rend autonome dans tout projet Node.js !

🧩 dependencies vs devDependencies

📦 dependencies

Nécessaires pour que l'application fonctionne en production.

npm install express --save
Exemples: express, mongoose, jsonwebtoken, bcrypt
🛠️ devDependencies

Outils utiles uniquement pendant le développement (tests, build, rechargement).

npm install nodemon --save-dev
Exemples: jest, nodemon, eslint, prettier
✨ Quand vous déployez, installez uniquement les dépendances de production avec npm install --production.

⚙️ Scripts : automatisez vos tâches

Dans la section "scripts", on peut définir des commandes exécutables avec npm run <nom>.

"scripts": {
  "start": "node server.js",
  "dev": "nodemon server.js",
  "test": "jest",
  "build": "webpack --mode production"
}
npm start → lance node server.js
npm run dev → nodemon pour dev
💡 Le script "start" est spécial : on peut taper directement npm start sans "run".

🚀 Exemple concret : initialiser un vrai projet

# Étape 1 : Créer dossier
mkdir mon-api-rest
cd mon-api-rest
npm init -y

# Étape 2 : Installer express (dépendance)
npm install express

# Étape 3 : Installer nodemon (dev)
npm install --save-dev nodemon

# Étape 4 : Modifier scripts dans package.json
"scripts": {
  "start": "node app.js",
  "dev": "nodemon app.js"
}

📌 Résultat :

Le fichier package.json contient express dans "dependencies" et nodemon dans "devDependencies".

✔️ Pour lancer en développement : npm run dev

✔️ En production : npm start

npm install → package.json mis à jour automatiquement

🔁 Récapitulons ensemble

1️⃣ npm init permet de générer package.json.
2️⃣ dependencies = essentiel en production, devDependencies = outils de développement.
3️⃣ scripts facilite l'exécution de tâches récurrentes.
4️⃣ package-lock.json verrouille les versions exactes (notion complémentaire).
🧠 Astuce: Un bon package.json rend votre projet professionnel et partageable.
