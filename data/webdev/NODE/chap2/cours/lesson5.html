📂 Module path : construire des chemins universels

Sous Windows on utilise \ et sous Linux/Mac /. Le module path résout ce problème pour nous.

🔥 Fonctions essentielles :

  • path.join() : assemble des segments (séparateur automatique)
  • path.resolve() : résout un chemin absolu à partir de segments
  • path.basename() : extrait le nom du fichier
  • path.dirname() : récupère le dossier parent
  • path.extname() : donne l'extension du fichier
// Exemples d'utilisation (CommonJS)
const path = require('path');
const fullPath = path.join('dossier', 'sous-dossier', 'fichier.txt');
console.log(fullPath); 
// Linux: dossier/sous-dossier/fichier.txt
// Windows: dossier\sous-dossier\fichier.txt

const absolute = path.resolve('src', 'app.js');
console.log(absolute); // chemin absolu

🧭 Analyser un chemin : basename, dirname, extname

const filePath = '/home/user/docs/rapport.pdf';

  • 📄 basename → 'rapport.pdf'
  • 📁 dirname → '/home/user/docs'
  • 🔖 extname → '.pdf'

📁 Représentation

📂 home/ 📂 user/ 📂 docs/ 📄 rapport.pdf → basename dirname → /home/user/docs 
const path = require('path');
const testPath = 'C:\\Projets\\node\\app\\server.js'; // Windows
console.log(path.basename(testPath));   // server.js
console.log(path.dirname(testPath));    // C:\Projets\node\app
console.log(path.extname(testPath));    // .js

✨ resolve transforme un chemin relatif en absolu en partant du répertoire courant.

🖥️ Module os : connaître la machine

Très utile pour adapter son code (logs, monitoring, ou décisions techniques).

const os = require('os');

console.log('Plateforme :', os.platform());     // win32, linux, darwin
console.log('Architecture CPU :', os.arch());   // x64, arm64
console.log('Nombre de CPU :', os.cpus().length);
console.log('Mémoire totale (Go) :', (os.totalmem() / 1e9).toFixed(2));
console.log('Mémoire libre (Go) :', (os.freemem() / 1e9).toFixed(2));
console.log('Uptime (heures) :', (os.uptime() / 3600).toFixed(1));
✅ Exemple concret : sur un serveur, surveiller la mémoire libre et déclencher une alerte si freemem() est faible.

🔐 Variables d'environnement : process.env

Elles permettent de configurer l'application sans coder en dur : mots de passe, ports, clés API.

$ export NODE_ENV=production // sous Linux/Mac
$ set NODE_ENV=production // Windows cmd
$env:NODE_ENV="production" // PowerShell

// Dans notre script Node.js
const port = process.env.PORT || 3000;
const dbUrl = process.env.DATABASE_URL;

console.log(`Mode : ${process.env.NODE_ENV}`);
console.log(`Serveur sur le port ${port}`);
⚠️ Ne jamais versionner un fichier avec secrets. Utilisez .env (via dotenv) en développement.

🛠️ Mise en pratique : script utilitaire

Imaginons un outil qui enregistre des logs dans un dossier logs/ relatif, et affiche des infos système.

const path = require('path');
const os = require('os');
const fs = require('fs');

const logDir = path.join(__dirname, 'logs');
if (!fs.existsSync(logDir)) fs.mkdirSync(logDir);

const logFile = path.join(logDir, `system-${Date.now()}.txt`);
const info = `
Plateforme: ${os.platform()}
CPUs: ${os.cpus().length}
Mémoire totale: ${(os.totalmem() / 1e9).toFixed(2)} Go
Env: ${process.env.NODE_ENV || 'development'}
`;
fs.writeFileSync(logFile, info);
console.log(`Log système enregistré: ${logFile}`);

🎯 Ce script fonctionne sur tous systèmes et crée un fichier unique avec les données.

✨ Astuces & bonnes pratiques

  • Préférez path.join() plutôt que la concaténation manuelle (+ '/' +)
  • __dirname : répertoire du fichier courant, très utile avec path.join(__dirname, 'public')
  • Utilisez os.EOL pour un saut de ligne compatible (Windows \r\n vs Linux \n)
  • process.cwd() donne le dossier où la commande node a été lancée, différent de __dirname.
