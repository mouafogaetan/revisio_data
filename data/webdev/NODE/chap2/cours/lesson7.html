EventEmitter, Buffers & Streams

Comprendre la puissance des événements, des données binaires et des flux continus dans Node.js

Durée estimée : 35 min Niveau : Intermédiaire

Appuyez sur ➡️ pour commencer

Objectifs d'apprentissage

  • Utiliser EventEmitter – Créer et écouter des événements personnalisés.
  • Comprendre les Buffers – Manipuler des données binaires brutes.
  • Maîtriser les Streams – Lire/écrire des flux (readable, writable, transform).
  • Appliquer les concepts – Exemples concrets: logger, traitement de fichiers volumineux.

À la fin de cette leçon, vous serez capable de créer des architectures basées sur les événements et de traiter des données efficacement.

1. EventEmitter (module 'events')

Le pattern Observer natif de Node.js. Beaucoup d'objets internes (Streams, Serveur HTTP) en héritent.

Créer un émetteur :

const EventEmitter = require('events');
class MonLogger extends EventEmitter {}
const logger = new MonLogger();

// Écouter un événement
logger.on('message', (data) => {
  console.log(`Nouveau message : ${data}`);
});

// Émettre
logger.emit('message', 'Hello Node.js !');

✔️ Méthodes : on(), once(), emit(), removeListener()

À retenir : Les événements sont parfaits pour découpler les composants (ex: notifier une action utilisateur).

Cas pratique EventEmitter

Simulons un serveur de commandes qui émet des événements "order".

const EventEmitter = require('events');
const commandHandler = new EventEmitter();

commandHandler.on('order:pizza', (size) => {
  console.log(`🍕 Préparer une pizza ${size}`);
});
commandHandler.on('order:drink', (drink) => {
  console.log(`🥤 Servir ${drink}`);
});

// Déclencher
commandHandler.emit('order:pizza', 'familiale');
commandHandler.emit('order:drink', 'Coca-cola');

2. Buffer : données binaires

Un Buffer est un espace mémoire temporaire (tableau d'octets) utilisé pour traiter des flux bruts (fichiers, réseau, crypto).

Création :

Buffer.from('Cameroun')
Buffer.alloc(10)

Conversion :

buf.toString('utf8')
buf.toJSON() 
const buf = Buffer.from('Node.js 🚀');
console.log(buf); // 
console.log(buf.toString('utf8')); // Node.js 🚀
console.log(buf[0]); // 78 (code ASCII de 'N')

Utile pour les fichiers binaires, les web sockets, et les protocoles réseau.

3. Les Streams : traitements à la volée

Les streams permettent de lire ou écrire des données morceau par morceau, sans tout charger en mémoire. Idéal pour fichiers volumineux ou requêtes réseau.

Readable Stream : source de données (ex: lecture fichier).
Writable Stream : destination (ex: écriture fichier, réponse HTTP).
Duplex / Transform : lecture + écriture / modification des données.

Schéma conceptuel :

📁 Fichier → createReadStream → 🔁 transformation → 🖥️ affichage

📂 Lire et écrire avec des streams

Copier un gros fichier sans surcharger la RAM :

const fs = require('fs');
const readStream = fs.createReadStream('gros_fichier.log');
const writeStream = fs.createWriteStream('copie.log');

readStream.on('data', (chunk) => {
  console.log(`Reçu ${chunk.length} octets`);
  writeStream.write(chunk);
});
readStream.on('end', () => {
  writeStream.end();
  console.log('Copie terminée avec succès');
});

Node.js émet des événements 'data', 'end', 'error' automatiquement.

Streams de transformation

Utilisez stream.Transform pour modifier les données en transit.

const { Transform } = require('stream');
const upperCaseTransform = new Transform({
  transform(chunk, encoding, callback) {
    this.push(chunk.toString().toUpperCase());
    callback();
  }
});
process.stdin.pipe(upperCaseTransform).pipe(process.stdout);

Testez mentalement : taper "hello" → affiche "HELLO". Très pratique pour des pipelines de données.
