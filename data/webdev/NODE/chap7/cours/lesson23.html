✅ Validation des données Joi / Zod

Avant d’utiliser des données utilisateur, on doit s’assurer qu’elles respectent un schéma (type, format, présence).

📦 Installation de Joi

npm install joi

Exemple de middleware de validation :

const Joi = require('joi');

const userSchema = Joi.object({
  email: Joi.string().email().required(),
  age: Joi.number().integer().min(0).max(120),
  password: Joi.string().min(6).required()
});

function validateUser(req, res, next) {
  const { error } = userSchema.validate(req.body);
  if (error) return res.status(400).json({ message: error.details[0].message });
  next();
}

⚡ Alternative avec Zod (plus lisible TypeScript)

npm install zod

const z = require('zod');
const UserSchema = z.object({
  email: z.string().email(),
  age: z.number().int().min(0).max(120).optional(),
  password: z.string().min(6)
});
✔️ Toujours valider body, paramètres (req.params) et query strings (req.query). Sécurité immédiate.

🔍 Valider body, params & query

Exemple complet pour une route /api/users/:id

const Joi = require('joi');

// Schéma pour l'ID (paramètre)
const idSchema = Joi.object({
  id: Joi.string().pattern(/^[0-9a-fA-F]{24}$/).required() // MongoDB ObjectId
});

// Schéma pour query (filtres)
const querySchema = Joi.object({
  page: Joi.number().integer().min(1).default(1),
  limit: Joi.number().integer().max(100)
});

// Validation en middleware
function validateParams(req, res, next) {
  const { error } = idSchema.validate(req.params);
  if (error) return res.status(400).json({ erreur: "ID invalide" });
  next();
}

// pour GET /users?page=2&limit=10
function validateQuery(req, res, next) {
  const { error, value } = querySchema.validate(req.query);
  if(error) return res.status(400).json({erreur: error.details[0].message});
  req.validQuery = value;
  next();
}
❌ Requête invalide → 400 Bad Request avec message clair
✅ Requête valide → on passe au contrôleur
{ Client } → [Validation Middleware] → (schéma ok ?) → Contrôleur → API ↗️ si erreur → retour erreur 400 avec détails

📁 Gérer les uploads avec Multer

Multer est un middleware pour gérer multipart/form-data (fichiers).

npm install multer

Configuration simple (stockage disque)

const multer = require('multer');
const storage = multer.diskStorage({
  destination: (req, file, cb) => cb(null, 'uploads/'),
  filename: (req, file, cb) => cb(null, Date.now() + '-' + file.originalname)
});
const upload = multer({ storage });

// Route pour upload unique
app.post('/upload', upload.single('avatar'), (req, res) => {
  res.json({ file: req.file });
});
⚠️ Pensez à créer le dossier "uploads" et à gérer les erreurs (taille, type de fichier).

🧹 Filtres de fichiers & validation d'images

📌 Filtrer par type MIME

const fileFilter = (req, file, cb) => {
  if (file.mimetype === 'image/jpeg' || file.mimetype === 'image/png') {
    cb(null, true);
  } else {
    cb(new Error('Format non supporté (seulement JPEG/PNG)'), false);
  }
};

const upload = multer({ storage, fileFilter, limits: { fileSize: 2 * 1024 * 1024 } }); // 2Mo

☁️ Upload vers le cloud (Cloudinary)

const cloudinary = require('cloudinary').v2;
const { CloudinaryStorage } = require('multer-storage-cloudinary');

const storageCloud = new CloudinaryStorage({
  cloudinary: cloudinary,
  params: { folder: 'avatars', allowed_formats: ['jpg', 'png'] }
});
const uploadCloud = multer({ storage: storageCloud });

Middleware → stockage local ou cloud

🔧 Cas concret : créer un profil avec photo et validation

// Express route avec validation Joi + multer
const profileSchema = Joi.object({
  nickname: Joi.string().alphanum().min(3).required(),
  bio: Joi.string().max(200)
});

app.post('/profile', upload.single('photo'), (req, res) => {
  // 1. Validation du body texte
  const { error, value } = profileSchema.validate(req.body);
  if (error) {
    // supprimer fichier uploadé si erreur (optionnel)
    return res.status(400).json({ erreur: error.details[0].message });
  }
  if (!req.file) return res.status(400).json({ erreur: 'Photo requise' });
  
  // 2. Tout est valide : sauvegarde en BDD
  res.status(201).json({ message: 'Profil créé', fichier: req.file.filename, data: value });
});
💡 Bonnes pratiques : Toujours valider les métadonnées avant d'enregistrer le fichier. Utilisez des middlewares séparés pour la validation et l'upload.
✅ Limiter la taille des fichiers ✅ Nettoyer les fichiers orphelins ✅ Gérer les erreurs asynchrones

📢 Messages d'erreur compréhensibles

Un bon validateur retourne des messages explicites pour le développeur front-end ou l'utilisateur final.

❌ Mauvais message : "Validation failed"

→ Pas assez informatif

✅ Bon message : "Le champ 'email' doit être une adresse email valide."

→ Joi génère ce genre de détails via error.details[0].message

Personnalisation avec Joi :

email: Joi.string().email().required().messages({
  'string.email': 'Veuillez fournir un email valide (ex: nom@domaine.com)',
  'any.required': "L'email est obligatoire"
})
