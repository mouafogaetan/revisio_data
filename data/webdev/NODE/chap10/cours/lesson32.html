🔌 Socket.IO : Le pont du temps réel

Socket.IO permet une communication bidirectionnelle et en temps réel entre le navigateur (client) et le serveur Node.js. Contrairement à HTTP, la connexion reste ouverte.

Communication bidirectionnelle continue

// Côté serveur (Node.js) const io = require('socket.io')(server); io.on('connection', (socket) => { console.log('Nouvel utilisateur connecté'); socket.emit('welcome', 'Bienvenue !'); });

👋 Gérer connexion et déconnexion

Socket.IO émet automatiquement les événements connection et disconnect. On peut ajouter notre logique :

io.on('connection', (socket) => { console.log(`✅ Socket ${socket.id} connecté`); socket.on('disconnect', () => { console.log(`❌ Socket ${socket.id} déconnecté`); io.emit('user_left', socket.id); }); // Envoyer la liste des utilisateurs connectés socket.emit('user_list', getConnectedUsers()); });

💡 Chaque socket possède un identifiant unique (socket.id). On peut stocker les pseudos associés.

🔐 Messages privés avec Rooms

Socket.IO permet de créer des rooms (salons). Chaque utilisateur rejoint une room privée avec son socket.id pour recevoir des messages privés.

// Client demande un message privé socket.on('private_message', ({ toId, message }) => { const senderId = socket.id; // Envoyer au destinataire io.to(toId).emit('private_message', { from: senderId, text: message }); // Envoyer également à l'expéditeur pour confirmation socket.emit('private_message_sent', { to: toId, text: message }); }); // Pour rejoindre une room spécifique socket.join(`room_${userId}`);

✨ Astuce : On peut associer un pseudo unique à chaque socket.id pour faciliter l'envoi.

💾 Persister les messages (MongoDB)

Pour que les messages ne disparaissent pas après déconnexion, on les stocke en base. À chaque nouveau message, on enregistre dans MongoDB.

// Modèle Message (Mongoose) const messageSchema = new mongoose.Schema({ pseudo: String, content: String, room: String, timestamp: { type: Date, default: Date.now } }); // Lors de la réception d'un message public socket.on('chat_message', async (data) => { const newMsg = new Message({ pseudo: data.pseudo, content: data.msg, room: 'general' }); await newMsg.save(); io.emit('chat_message', data); // diffusion }); // Charger l'historique à la connexion socket.on('request_history', async () => { const lastMessages = await Message.find({ room: 'general' }).limit(50).sort('-timestamp'); socket.emit('history', lastMessages.reverse()); });

✅ Avantage : Les nouveaux arrivants voient les anciens messages !

📝 Mini-chat : code minimal

Serveur (app.js)

const express = require('express'); const http = require('http'); const socketIo = require('socket.io'); const app = express(); const server = http.createServer(app); const io = socketIo(server); io.on('connection', (socket) => { socket.on('chat message', (msg) => { io.emit('chat message', msg); }); }); server.listen(3000);

Client (index.html)

<script src="/socket.io/socket.io.js"></script> <script> const socket = io(); $('form').submit(() => { socket.emit('chat message', $('#m').val()); $('#m').val(''); return false; }); socket.on('chat message', (msg) => { $('#messages').append($('<li>').text(msg)); }); </script>

✨ Ce mini chat diffuse tous les messages à tous les utilisateurs connectés.

  • 🔹 Étape 5 (bonus) : Implémenter un message privé via /w pseudo message.
// Stocker pseudo et socket.id
const users = new Map(); // key = socket.id, value = pseudo

socket.on('set_pseudo', (pseudo) => {
  users.set(socket.id, pseudo);
  io.emit('user_joined', `${pseudo} a rejoint le chat`);
});

// Message privé
socket.on('private_msg', ({ toPseudo, msg }) => {
  for (let [id, p] of users.entries()) {
    if (p === toPseudo) io.to(id).emit('private', msg);
  }
});
