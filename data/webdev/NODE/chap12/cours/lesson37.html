🔐 Authentification avec JWT

JSON Web Token (JWT) permet une authentification stateless. Le serveur ne stocke pas de session : le client envoie un token signé à chaque requête.

// Génération du token après login
const jwt = require('jsonwebtoken');
const token = jwt.sign(
  { userId: user.id, role: user.role },
  process.env.JWT_SECRET,
  { expiresIn: '7d' }
);
// Middleware de vérification
const auth = (req, res, next) => {
  const token = req.header('Authorization')?.replace('Bearer ', '');
  if (!token) return res.status(401).json({ error: 'Accès refusé' });
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (err) { res.status(401).json({ error: 'Token invalide' }); }
};

📝 CRUD & Validation des données

Pour une ressource "Article", on implémente les opérations Create, Read, Update, Delete avec validation.

// validation avec Joi
const Joi = require('joi');
const articleSchema = Joi.object({
  title: Joi.string().min(3).max(100).required(),
  content: Joi.string().min(10).required(),
  category: Joi.string().valid('tech', 'sante', 'education')
});

// Route POST /articles (protégée)
app.post('/articles', auth, (req, res) => {
  const { error } = articleSchema.validate(req.body);
  if (error) return res.status(400).json({ message: error.details[0].message });
  // sauvegarde en BD ...
  res.status(201).json({ message: 'Article créé' });
});

⚠️ Gestion d'erreurs & Async

Un middleware spécifique capture toutes les erreurs et évite les crashs.

// middleware d'erreur global (après toutes les routes)
app.use((err, req, res, next) => {
  console.error(err.stack);
  const status = err.status || 500;
  res.status(status).json({
    error: err.message || 'Erreur interne du serveur'
  });
});

// Pour les routes async, on utilise express-async-errors
require('express-async-errors');
app.get('/profile', async (req, res) => {
  const user = await User.findById(req.user.id);
  if (!user) throw new Error('Utilisateur introuvable');
  res.json(user);
});

📜 Logs structurés (Winston)

Les logs permettent de tracer l’activité de l’API et de diagnostiquer les pannes en production.

const winston = require('winston');
const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' }),
    new winston.transports.Console({ format: winston.format.simple() })
  ]
});
// Utilisation
app.get('/secure-data', auth, (req, res) => {
  logger.info(`Accès aux données par ${req.user.userId}`);
  res.json({ secret: 'data' });
});

En production, centralisez les logs avec des outils comme Datadog ou Papertrail.

🛡️ Sécurisation avancée

  • Helmet : protège les en-têtes HTTP (XSS, clickjacking).
  • CORS : restreint les origines autorisées.
  • express-rate-limit : bloque les attaques par force brute / DDoS.
  • Bcrypt : hachage des mots de passe avec salage.
const rateLimit = require('express-rate-limit');
const limiter = rateLimit({ windowMs: 15*60*1000, max: 100 });
app.use('/api/login', limiter);
app.use(require('helmet')());
app.use(require('cors')({ origin: 'https://monfrontend.com' }));
