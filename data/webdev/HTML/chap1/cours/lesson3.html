Balises, attributs et bonnes pratiques

Maîtriser la syntaxe HTML, distinguer les types de balises et écrire un code propre

🎯 Chapitre 1 : Structure du HTML ⏱️ Durée estimée : 45 min
🎯

Objectifs de la leçon

1. Distinguer balises orphelines et balises en paires — Comprendre la différence entre une balise qui s'ouvre et se ferme, et celles qui sont auto-fermantes.
2. Utiliser les attributs universels (id, class, style, title) — Apprendre à ajouter des informations supplémentaires aux éléments HTML.
3. Appliquer les bonnes pratiques d'indentation et de commentaires — Écrire un code HTML lisible, maintenable et professionnel.

💡 À la fin de cette leçon, vous saurez construire n'importe quelle page avec une syntaxe correcte et organisée !

🧩 1. Balises orphelines vs balises en paires

En HTML, presque tout élément s'écrit avec une balise ouvrante et une balise fermante. Mais certaines balises n'ont pas de contenu textuel : ce sont les balises orphelines (ou auto-fermantes).

✔️ Balises en paires

  • <p> ... </p> → paragraphe
  • <h1>Titre</h1> → titre
  • <strong>gras</strong>
  • <div>bloc</div>

Elles ont une ouverture et une fermeture explicite avec / .

✨ Balises orphelines

  • <br> ou <br/> → retour à la ligne
  • <hr> → ligne horizontale
  • <img src="..." alt="..."> → image
  • <input type="text"> → champ formulaire

Elles n'ont pas de contenu interne, pas besoin de balise fermante !

📌 En HTML5, on peut écrire <br> sans slash, mais laisser le slash est autorisé. L'important est de comprendre la logique.

🏷️ 2. Attributs universels (id, class, style, title)

Les attributs s'ajoutent dans la balise ouvrante pour donner plus d'informations. Les attributs universels marchent sur tous les éléments HTML.

id="nom_unique"

Identifiant unique dans la page. Utile pour cibler un élément précis en CSS ou JavaScript. Une page ne doit pas avoir deux fois le même id.

class="nom_classe"

Classe(s) CSS partageables. On peut appliquer la même classe à plusieurs éléments pour leur donner un style commun.

style="couleur: bleu; font-size:20px;"

Ajoute du CSS directement dans la balise (inline). À utiliser avec parcimonie, réservé aux exceptions.

title="info bulle"

Affiche une infobulle au survol de la souris. Pratique pour donner des explications supplémentaires.

<p id="intro" class="texte-important" style="color: #2c3e50;" title="Définition HTML">
  Le HTML est le squelette du web.
</p>
✨ Petit dessin d'attribut : 🎨 [balise] + attributs = super pouvoir !

📐 3. Indentation & commentaires : code propre et lisible

Un code bien organisé est plus facile à lire, à corriger et à partager avec d'autres développeurs.

📌 Règles d'indentation

  • À chaque nouveau bloc (balise parente), on décale le contenu intérieur de 2 ou 4 espaces (ou une tabulation).
  • Les balises au même niveau doivent avoir le même décalage.
  • Un code bien indenté évite les erreurs de balises mal fermées.
<div>
  <h1>Titre principal</h1>
  <p>Un paragraphe correctement indenté</p>
</div>

💬 Les commentaires HTML

Les commentaires ne sont pas affichés à l'écran mais restent dans le code source. Excellents pour laisser des notes, expliquer une section ou désactiver temporairement du code.

<!-- Ceci est un commentaire -->
<!-- TODO : ajouter le formulaire de contact plus tard -->

Astuce : utilisez les commentaires pour organiser votre page (ex: <!-- HEADER START -->).

✍️ Mettons en pratique !

Voici un petit extrait de code qui illustre les notions vues : balises orphelines, attributs universels, indentation et commentaire.

<!-- Exemple complet de la leçon 3 -->
<section id="exemple" class="demo">
  <h2 title="Titre explicatif">Balises et attributs</h2>
  <p>Ligne 1<br>Ligne suivante avec balise orpheline <strong>&lt;br&gt;</strong></p>
  <img src="" alt="dessin attribut universel" style="max-width:100px;">
  <hr>
  <div class="note">Fin de l'aperçu.</div>
</section>
🔍 À retenir : Balises orphelines → pas de balise fermante.
