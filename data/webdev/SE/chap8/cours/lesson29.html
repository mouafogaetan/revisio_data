🤔

Qu'est-ce que le refactoring ?

Le refactoring est le processus de restructuration du code source existant, sans modifier son comportement observable. On améliore la structure interne, la lisibilité et la maintenabilité.

💩 Code sale
(duplication, longues méthodes)
🔧✨ ➡️
✨ Code propre
(DRY, nom clair, petites fonctions)

✅ même résultat fonctionnel ✅

"Refactorer, c'est comme ranger sa chambre : on ne change pas la pièce, mais on rend tout plus accessible."

📖 Extrait de Martin Fowler : "Refactoring is a disciplined technique for improving the design of existing code."

🐢 Pourquoi refactorer ? (Dette technique)

Chaque raccourci, duplication ou code mal organisé augmente la dette technique. Comme une dette financière, elle accumule des "intérêts" :

  • 🐞 Plus de bugs cachés
  • ⏱️ Ajout de fonctionnalités devient lent
  • 🤯 Difficulté pour les nouveaux développeurs
  • 💸 Coût de maintenance élevé

Exemple de dette technique croissante (70% de complexité évitable)

✅ Refactoring régulier → réduit la dette, code plus robuste.

🔧 Techniques de refactoring (3 fondamentales)

1. Extraction de méthode

Transformer un bloc de code en une méthode séparée, avec un nom explicite.

// Avant : long bloc dans une fonction
function afficherFacture() { /* 30 lignes de calcul + affichage */ }

// Après : extraire calcul
function calculerTotal() { ... }
function afficherFacture() { calculerTotal(); ... }

2. Renommage (significatif)

Variables, fonctions, classes doivent avoir des noms qui révèlent l'intention.

let d -> let dateLimiteInscription

3. Déplacement de méthode / champ

Déplacer une méthode vers la classe où elle est le plus utile (meilleure cohésion).

🛠️ Outils de refactoring automatique

Les IDE modernes intègrent des fonctionnalités de refactoring sécurisées. Exemples :

🔁 VS Code
Extraire méthode, renommage symbolique, organise imports
☕ IntelliJ / Eclipse
Refactor menu puissant : Pull up, Push down, Inline, Change signature
💻 PyCharm
Extract variable, extract method, rename, safe delete
⚡ WebStorm
JavaScript/TS refactoring (extract component, convert to arrow)
💡 Astuce : Utilisez le raccourci clavier Ctrl+Shift+R (ou F6/Ctrl+T) pour explorer les refactorings courants.

📐 Petit schéma : refactoring "Extract Method"

function afficherPanier(articles) {
  let total = 0;
  for(let a of articles) total += a.prix * a.qte;
  console.log("total:", total);
  let tva = total * 0.2;
  console.log("TVA:", tva);
}

🐙 Avant : trop de responsabilités

function calculerTotal(articles) { ... }
function calculerTVA(total) { ... }
function afficherPanier(articles) {
  let total = calculerTotal(articles);
  let tva = calculerTVA(total);
  console.log(...);
}

✨ Après : méthodes réutilisables, claires

✅ Refactoring + test unitaire = 0 régression

Le refactoring systématique améliore la lisibilité et facilite les tests.

👃 Indices qu'il faut refactorer (Code Smells)

📏 Méthode trop longue → plus de 20-30 lignes
📦 Classe paresseuse ou trop grosse (god class)
🔁 Code dupliqué (violation DRY)
🧩 Jalousie de fonctionnalité (méthode qui utilise trop les données d'une autre classe)
🤷‍♂️ Noms peu explicites (var x, fonction traiter)
⚙️ Chaînes de conditions complexes (if/else nested profond)
💚 Appliquer DRY (Don't Repeat Yourself) et KISS (Keep It Simple, Stupid) réduit les mauvaises odeurs.
🧪

Refactoring + Tests unitaires = duo gagnant

Avant de refactorer, assurez-vous d'avoir une suite de tests solide (tests unitaires, intégration). Après refactoring, exécutez les tests pour vérifier qu'aucune fonctionnalité n'a été cassée.

🔁 Refactoring → 🧪 Lancer les tests → ✅ Succès
Si tests échouent → corriger ou annuler le refactoring.
✨ Meilleure pratique : Faire des petites étapes de refactoring et relancer les tests fréquemment.

⌨️ Refactoring automatique en action (IDE)

Regardez comment IntelliJ/VSCode propose l'extraction :

// 1. Sélectionner un bloc de code

let prix = article.prix * article.quantite;

let remise = prix * 0.1;

// 2. Clic droit → Refactor → Extract Method

→ Génère : function calculerPrixAvecRemise(article) { ... }

🔧 Raccourci VS Code : Ctrl+Shift+R puis "Extract method" 🚀 Eclipse : Alt+Shift+M

Utilisez ces outils pour éviter les erreurs manuelles.

📝 En résumé : Le refactoring au quotidien

  • Refactorer souvent (à chaque fois qu'on comprend mieux le code).
  • S'appuyer sur les tests pour ne pas casser l'existant.
  • Utiliser les commandes automatiques des IDE (gain de temps et sécurité).
  • Améliorer la lisibilité pour soi-même et pour l'équipe.
  • Réduire la dette technique → projets plus pérennes.
⭐ "Laissez le code plus propre que vous ne l'avez trouvé" — Boy Scout Rule

Mini exercice mental

Identifiez un code smell dans ce snippet :

function calcule(a,b,c){
   let r = a*b;
   let t = c * 0.2;
   let result = r + t;
   console.log("resultat:"+result);
   return result;
}

👉 Réponse : nom de fonction peu explicite, paramètres génériques, affichage log mélangé → extraire l'affichage, renommer la fonction.
