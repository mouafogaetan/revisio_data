📘 Observer & Strategy

👀 OBSERVER

Définit une dépendance un-à-plusieurs : quand un objet (sujet) change d'état, tous ses dépendants (observateurs) sont notifiés automatiquement.

Exemple : Système de notifications, mise à jour d'interface utilisateur (Twitter, listeners d'événements).

🎯 STRATEGY

Permet de définir une famille d'algorithmes, les encapsuler et les rendre interchangeables. Le comportement peut changer à l'exécution.

Exemple : Modes de paiement (carte, PayPal, mobile money), tri d'une liste selon différentes stratégies.

💡 Pourquoi c'est utile ?

Ces patrons évitent les conditionnels monstres et respectent le principe Ouvert/Fermé. Observer découple l'émetteur des récepteurs. Strategy évite de modifier une classe pour chaque nouvel algorithme.

⚙️ Command & Template Method

📟 COMMAND

Transforme une requête en objet (commande). Permet de paramétrer des clients avec différentes requêtes, de faire des files d'attente, annulations (undo/redo).

Structure : Invoker → Command → Receiver

📐 TEMPLATE METHOD

Définit le squelette d'un algorithme dans une méthode, en laissant certaines étapes à des sous-classes. Réutilise la structure sans dupliquer le code.

Exemple concret : Construire une maison (fondations, murs, toit) où chaque type de maison personnalise certaines étapes.

📌 Rappel en programmation web : Les commandes sont utiles pour implémenter des macros ou historiques d'actions utilisateur. Template Method est très utilisé dans les frameworks (méthodes abstraites appelées par un workflow).

🔄 State, Chain of Responsibility & Mediator

🔁 STATE (État)

Permet à un objet de modifier son comportement lorsque son état interne change. L'objet paraît changer de classe. Exemple: un lecteur audio (Play, Pause, Stop) change de comportement selon le mode.

⛓️ CHAIN OF RESPONSIBILITY

Évite de coupler l'émetteur d'une requête à son receveur en donnant à plusieurs objets la chance de traiter la requête. Chaîne de responsabilité. Ex: système d'aide, validation de formulaires en cascade.

📢 MEDIATOR

Réduit les dépendances chaotiques entre objets en introduisant un médiateur qui centralise la communication. Exemple: une tour de contrôle d'aéroport, ou un gestionnaire de chat entre composants UI.

⭐ Ces patrons aident à rendre le code plus flexible, réutilisable et adapté aux changements fréquents.

🎨 Petit schéma: Strategy vs State

Un même objet "Personnage" peut changer de stratégie de déplacement (marche, course, nage) selon le contexte — Strategy. Alors que State gère les états (endormi, réveillé, en combat).

📝 Code imaginaire : 

class Personnage {
  setDeplacementStrategy(strategy) { this.strategy = strategy; }
  seDeplacer() { this.strategy.deplacer(); }
}

💻 Observer en JavaScript (exemple réel)

// Sujet (Observable)
class Newsletter {
  constructor() { this.suscribers = []; }
  subscribe(observer) { this.suscribers.push(observer); }
  notify(message) { this.suscribers.forEach(obs => obs.update(message)); }
}
// Observateur
class Abonne {
  constructor(name) { this.name = name; }
  update(msg) { console.log(`${this.name} a reçu : ${msg}`); }
}
const newsletter = new Newsletter();
const alice = new Abonne("Alice");
newsletter.subscribe(alice);
newsletter.notify("Nouveau cours sur les patterns !"); // Alice reçoit

Ce patron est partout: addEventListener, RxJS, stores Vue/React.

🌍 Analogie pratique (Cameroun)

Dans une gare routière :

  • Observer : Les passagers (observateurs) attendent l’annonce du départ. Une fois l’autocar prêt, le gare routière (sujet) les informe tous.
  • Strategy : Paiement du ticket par Orange Money, MTN Mobile Money ou espèces → différentes stratégies interchangeables.
  • Chain of responsibility : Réclamation client passant du guichet au contrôleur puis au chef de gare.
  • Mediator : La plateforme de covoiturage qui coordonne conducteurs et passagers.
💡 Ces patterns rendent les applications web plus faciles à maintenir et à faire évoluer.

🧪 Vérifions votre compréhension

1️⃣ Quel patron utiliser pour annuler des actions dans un éditeur de texte ?

Réponse : Command (permet de stocker l'état/action inverse).

2️⃣ Un système de feux de circulation change de comportement (vert, orange, rouge) → quel patron ?

Réponse : State (le comportement varie selon l'état du feu).

3️⃣ Pour un calculateur d'itinéraire (voiture, marche, vélo), quel patron s'impose ?

Réponse : Strategy, on peut changer l'algorithme de calcul dynamiquement.
