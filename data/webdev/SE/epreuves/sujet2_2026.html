🏷️ Exercice 2 – Classification d'exigences (4 pts)

4 pts

Classez les exigences suivantes selon la matrice : Fonctionnelle / Non fonctionnelle et Utilisateur / Système.

ExigenceFonctionnelle / Non fonctionnelleUtilisateur / Système
"L'application doit afficher le catalogue de produits."
"Le système doit supporter 1000 connexions simultanées."
"L'utilisateur veut recevoir un email de confirmation après commande."
"Le temps de chargement de la page d'accueil ne doit pas dépasser 2 secondes."
Exemple : Fonctionnelle = F, Non fonctionnelle = NF ; Utilisateur = U, Système = S

📝 Éléments de correction

  • QCM : 1.1 Non fonctionnelle / 1.2 Should have / 1.3 Observation / 1.4 SRS / 1.5 User story bien formulée (option 2).
  • Classification : Afficher catalogue → F/U ; 1000 connexions → NF/S ; Email confirmation → F/U ; Temps chargement → NF/S.
  • User story : Format standard exigé : « En tant que [rôle], je veux [action] afin de [bénéfice] ».
  • Include : S'authentifier est obligatoire avant de prendre rendez-vous ; inclusion réutilisable.
  • MoSCoW : Must have → indispensables ; Won't have → exclues du scope.
