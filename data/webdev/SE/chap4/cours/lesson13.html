Définition & utilité

Le diagramme de classes est le diagramme structurel le plus important en UML. Il décrit les classes (entités) du système, leurs attributs (données) et méthodes (comportements), ainsi que les liens entre elles.

✨ "Montre la structure statique : ce qui existe, et non comment ça bouge."

✔️ Analyser le domaine métier
✔️ Générer du code (squelette)
✔️ Documenter l'architecture
✔️ Faciliter la communication équipe

📦 Une classe, comment la dessiner ?

Une classe se représente par un rectangle divisé en 3 compartiments :

  • 1. Nom de la classe (en gras, centré)
  • 2. Attributs : visibilité nom : type
  • 3. Méthodes : visibilité nom(paramètres) : typeRetour

Visibilités : + public , - privé , # protégé

+----------------------+
|      Utilisateur     |
+----------------------+
| - id : int           |
| - nom : String       |
| - email : String     |
+----------------------+
| + seConnecter()      |
| + modifierEmail(nouveau) |
+----------------------+

Exemple : classe "Utilisateur"

💡 Bon à savoir : Les attributs et méthodes statiques sont soulignés en UML.

🔗 Les relations entre classes

Association simple ——>

Lien structurel entre classes (ex: Professeur → Cours)

Agrégation ◇—— (diamant vide)

Relation "tout-partie" faible. Le tout existe sans la partie. Ex: Département contient des Professeurs (prof peut exister sans département).

Composition ◆—— (diamant plein)

Relation forte : la partie n'existe pas sans le tout. Ex: Commande contient des LignesCommande (une ligne n'a pas de sens sans commande).

Héritage / Généralisation ◁—— (triangle vide)

"est-un" : une classe enfant hérite de la classe parente. Ex: Voiture hérite de Véhicule.

🔢 Multiplicités & cardinalités

La multiplicité indique combien d'instances participent à la relation.

0..1
1
0..*
1..*

Exemple fréquent : [Client] 1 -----> 0..* [Commande] → un client peut avoir plusieurs commandes.

Un étudiant suit plusieurs cours, un cours est suivi par plusieurs étudiants.

🛠️ Outils UML & bonnes pratiques

Outils gratuits : Draw.io, Lucidchart (version gratuite), PlantUML (code → diagramme), StarUML.

Conseils : Commencez simple, nommez clairement les relations, évitez les surcharges inutiles.
