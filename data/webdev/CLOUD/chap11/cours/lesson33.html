1. Choisir un projet cloud

Pour mettre en pratique, choisissez un cas qui vous motive. Trois idées fréquentes :

🌐

Application Web

Blog, e-commerce vitrine, site avec sessions utilisateur et base de données relationnelle.

⚙️

API RESTful

Backend pour mobile ou SPA, authentification JWT, stockage JSON.

📊

Traitement de données

Pipeline ETL, analyse de logs, transformation d’images ou vidéos.

🧠 À retenir :

Chaque projet va influencer le choix des services cloud : serveurs web (VM ou conteneurs), bases de données managées, files d’attente, etc.

🏛️

2. Architecture type 3 tiers

Une architecture cloud classique sépare les couches :

  • VPC (Virtual Private Cloud) : réseau isolé dans le cloud.
  • Sous-réseaux publics/privés : DMZ pour load balancer, sous-réseau privé pour les serveurs applicatifs, sous-réseau data pour bases de données.
  • Load Balancer (ALB/ELB) : distribue le trafic vers plusieurs instances.
  • Base de données managée (RDS, Cloud SQL) : haute disponibilité, backups automatiques.

3. Estimer les coûts & scalabilité

Le cloud facture à l’usage. Pour maîtriser votre budget, évaluez :

⚡ Calcul (VM/Conteneurs)

Type d’instance (vCPU, RAM), nombre d’heures d’exécution, réservée ou à la demande.

🗄️ Stockage

Volume (Go), type SSD/HDD, IOPS, transferts de données sortants.

🌐 Réseau

Data Transfer (sortie d’Internet généralement facturée), Load Balancer heures.

📦 Services managés

Base de données (RDS), serverless (Lambda) : coût par requête ou par Go-seconde.

📊 Exemple de calcul rapide (petite app web) :

  • 2 VM t2.micro (Linux) – 24h/7j ≈ 15$ par mois (on-demand)
  • RDS db.t3.micro (20 Go) ≈ 12$ / mois
  • Stockage S3 pour assets ≈ 0.50$ (faible volume)
  • Data transfer (50 Go sortant) ≈ 4$
  • Total estimé ~ 32$/mois (prix variables selon région)

ℹ️ Utilisez AWS Pricing Calculator / Azure Pricing Calc pour affiner.

📈 Auto-scaling : réduit les coûts hors-pointe 💸 Instances Spot / réservées pour économiser
