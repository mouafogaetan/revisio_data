💡 Pourquoi c'est important ? Une mauvaise gestion des coûts cloud peut faire exploser la facture. Maîtriser la facturation, c'est allier performance et économies.

💸 Les 3 grands modèles tarifaires

⏱️

À la demande (On-Demand)

Paiement à l'heure ou à la seconde. Sans engagement, souple mais plus cher pour les charges continues.

Ex: dev/test, pic imprévisible.
📆

Instances réservées (RI)

Engagement 1 ou 3 ans → réduction jusqu'à 72%. Paiement intégral/partiel/mensuel.

Charge de travail stable (serveurs web, base de données).
⚡

Instances Spot / Low-priority

Capacité excédentaire à prix cassé (jusqu'à -90%). Peut être interrompue à tout moment.

Jobs batch, conteneurs tolérants, big data.
Chaque fournisseur (AWS, Azure, GCP) propose des noms similaires.

📈 Savings Plans & Instances réservées

AWS a introduit les Savings Plans : plus flexible que les RI classiques.

🔹 Compute Savings Plans

S'applique automatiquement à toute instance (EC2, Fargate, Lambda). Change de famille/region.

🔹 EC2 Instance Savings Plans

Similaire aux RI, engagement sur une famille d'instance dans une région. Réduction plus élevée.

🧠 À retenir : Les Savings Plans réduisent la facture tout en gardant une certaine élasticité. Sur Azure : "Azure Reservations", sur GCP : "Committed Use Discounts".

🧮 Exemple concret : Comparaison tarifaire

Prenons une instance t3.medium (AWS) tournant 730h/mois (toujours allumée).

  • 🔹 On-Demand : 0,0416 $/h → 730 × 0,0416 ≈ 30,37 $/mois
  • 🔹 Instance réservée (1 an, paiement partiel) : ~0,026 $/h → 730 × 0,026 ≈ 18,98 $/mois → Économie ~37%
  • 🔹 Instance Spot (prix moyen ~0,012 $/h) : 730 × 0,012 ≈ 8,76 $/mois → Économie jusqu'à 71% (risque d'interruption).
💡 Mathématiquement : \(\text{Coût Spot} \ll \text{Coût RI} < \text{Coût On-Demand}\). Pour les charges flexibles, le Spot est roi. Pour une base de données critique, les instances réservées sont idéales.

✅ Stratégies pour maîtriser la facture

📌 Analyser régulièrement avec AWS Cost Explorer, Azure Cost Management.
📌 Mettre en place des budgets et alertes (éviter les dépassements).
📌 Supprimer les ressources inutilisées (volumes, snapshots, IPs élastiques).
📌 Utiliser l'auto-scaling pour adapter la capacité à la charge réelle.
📌 Planifier l’arrêt des environnements hors production (ex: nuit/week-end).
📌 Opter pour du serverless (Lambda/Functions) pour les charges sporadiques.

📣 Citation : "Le cloud n'est pas cher, ce sont les mauvaises pratiques qui coûtent cher."
