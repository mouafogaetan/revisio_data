📁 Stockage partagé
NFS / SMB dans le cloud

Comprendre comment plusieurs serveurs peuvent accéder aux mêmes fichiers comme sur un disque réseau partagé.

🎯 Objectifs de la leçon

  • 1 Comprendre le stockage partagé NFS/SMB : principe, cas d’usage (cloud, VMs, clusters).
  • 2 Monter un système de fichiers cloud partagé sur plusieurs VMs (machine virtuelle).
  • 3 Comparer stockage objet (S3), bloc (EBS) et stockage partagé (EFS / Azure Files).
  • 4 Identifier les services cloud managés : AWS EFS, Azure Files, GCP Filestore.
💡 Pourquoi c'est important ? Les applications web, bases de données ou clusters ont besoin d'un stockage commun accessible depuis plusieurs serveurs.

🧠 Définition : stockage partagé réseau

Le stockage partagé permet à plusieurs machines (serveurs, VMs) de lire et écrire dans les mêmes fichiers via le réseau. Deux protocoles principaux existent :

🐧 NFS

Network File System – Très utilisé dans les environnements Linux/Unix. Rapide et standard dans le cloud.

🪟 SMB/CIFS

Server Message Block – Natif Windows, compatible avec Linux via Samba. Idéal pour entreprises mixtes.

🔁 Plusieurs VMs montent le même point de montage réseau → données centralisées.

🔧 Monter un partage cloud sur plusieurs VMs

Exemple avec AWS EFS (NFS) ou Azure Files (SMB). Étapes générales :

  1. Créer un système de fichiers partagé dans la console cloud (EFS / Azure Files / GCP Filestore).
  2. Obtenir le point de montage (ex: fs-12345.efs.eu-west-1.amazonaws.com:/).
  3. Sur chaque VM Linux: installer le client NFS (sudo apt install nfs-common).
  4. Créer un dossier local et monter : sudo mount -t nfs4 endpoint:/ /mnt/cloudshare.
  5. Rendre le montage persistant via /etc/fstab.
💡 Résultat : Toutes les VMs écrivent dans le même espace → parfait pour une application PHP qui stocke les fichiers uploads, ou des logs centralisés.

⚖️ Comparaison : Objet vs Bloc vs Partagé

TypeExemple cloudAccès simultanéCas typique
📦 ObjetAWS S3, GCSvia API HTTP, pas montage standardArchivage, images statiques, sites web
🔲 Bloc (EBS)AWS EBS, Azure Disk1 seule VM à la foisBase de données, système d'exploitation
🔄 Partagé (NFS/SMB)AWS EFS, Azure Files, GCP Filestore✅ Multi-VM simultanéContenu partagé, clusters, home directories

Le stockage partagé est idéal pour la collaboration entre services, contrairement au stockage bloc qui est comme un disque dur virtuel dédié à une seule instance.

☁️ Solutions cloud managées

AWS EFS

NFS version 4.1, élastique, facturation à l'usage. Parfait pour les clusters EC2.

Azure Files

SMB et NFS, compatible Windows/Linux, intégration Active Directory.

GCP Filestore

NFSv3 haute performance, utilisé pour GKE, Compute Engine.

Open Source

GlusterFS, Ceph – mais demande plus d’administration.

📌 Avec ces services, plus besoin de gérer un serveur NFS manuellement : le cloud s'occupe de la réplication, haute disponibilité et sauvegarde.

✅ Avantages & ⚠️ limites du stockage partagé cloud

✨ Points forts

  • Accès simultané depuis plusieurs VMs / conteneurs
  • Mise à l'échelle automatique (pas de gestion de quota complexe)
  • Protocoles standards (NFS / SMB) compatibilité maximale
  • Sauvegarde intégrée (snapshots)

🧐 Limitations

  • Latence réseau légèrement supérieure à un disque local
  • Certains workloads très haute performance préfèrent le stockage bloc (IOPS dédiées)
  • Quelques protocoles (NFSv3) moins sécurisés que versions récentes
🚀 Exemple concret : Un site e-commerce tourne sur 3 serveurs web. Tous doivent accéder aux images des produits et fichiers de configuration. Avec un volume partagé NFS, chaque serveur voit les mêmes fichiers → synchronisation instantanée.

🖥️ Exemple d'utilisation : montage NFS sur deux VMs Linux

# Sur VM1 et VM2 (Ubuntu 22.04)
$ sudo apt update && sudo apt install nfs-common -y

# Créer un dossier de montage
$ sudo mkdir -p /mnt/cloud-data

# Monter le partage EFS (remplacer par votre endpoint)
$ sudo mount -t nfs4 fs-12345678.efs.eu-west-3.amazonaws.com:/ /mnt/cloud-data

# Tester l'écriture depuis VM1
$ echo "Donnée partagée" | sudo tee /mnt/cloud-data/test.txt

# Depuis VM2, vérifier :
$ cat /mnt/cloud-data/test.txt
Donnée partagée ✅

# Montage automatique (fstab)
fs-12345678.efs.eu-west-3.amazonaws.com:/ /mnt/cloud-data nfs4 defaults 0 0

✅ Le même fichier est accessible depuis toutes les VMs, comme un disque réseau ultra simple.

📝 Vérification des acquis

1️⃣ Quel protocole est le plus utilisé pour le partage sous Linux dans le cloud ?
✔ Réponse : NFS (Network File System)
2️⃣ Vrai ou faux ? Le stockage bloc (EBS) peut être monté simultanément sur 5 EC2 différentes.
❌ Faux, un volume bloc est attaché à une seule instance à la fois.
3️⃣ Citez deux services cloud de fichiers partagés managés.
✔ AWS EFS, Azure Files, GCP Filestore.

🎓 Résumé de la leçon 13 :

Le stockage partagé NFS/SMB rend possible l'accès concurrent à des fichiers depuis plusieurs VMs. Dans le cloud, les solutions managées comme AWS EFS simplifient énormément la tâche. Contrairement au stockage objet (S3) et bloc (EBS), il est le bon choix pour les applications nécessitant un système de fichiers classique distribué.
