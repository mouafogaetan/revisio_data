📀 Définition : Stockage Bloc

Le stockage bloc découpe les données en blocs de taille fixe. Chaque bloc possède une adresse unique, mais aucune métadonnée structurée. Il se comporte comme un disque dur virtuel que l'on attache à une instance (VM).

🧠 Exemple simple : C'est comme un disque externe que vous branchez à un ordinateur. Vous pouvez le formater (NTFS, ext4) et y écrire des fichiers.

✔️ Avantages
  • Performances élevées (faible latence)
  • Idéal pour bases de données, systèmes de fichiers
  • Persistance indépendante de la VM
⚠️ Limites
  • Ne peut être partagé simultanément (sauf solutions spécialisées)
  • Gestion manuelle des snapshots / backup

Schéma : Un volume bloc attaché à une machine virtuelle

⚙️ Types de disques blocs

Les fournisseurs cloud proposent plusieurs familles selon les performances et le coût :

SSD (Solid State Drive)
⚡ Ultra-rapide, faible latence (< 1ms). Idéal pour bases de données, systèmes à forte sollicitation IOPS. Ex: gp3 (AWS), Premium SSD (Azure).
HDD (Disque magnétique)
🐢 Économique, bonne capacité, mais latence plus élevée. Pour charges batch, logs, archivage. Ex: st1 (AWS), Standard HDD (Azure).
IOPS Provisionnées
🎯 Vous réservez un nombre garanti d'opérations entrée/sortie par seconde. Essentiel pour applications critiques (ERP, gros volume transactionnel). Coût plus élevé mais performances prévisibles.
📘 IOPS = Input/Output Operations Per Second. Plus c'est élevé, plus le disque traite rapidement de petites requêtes.

📸 Snapshots : La sauvegarde incrémentale

Un snapshot est une copie à un instant T du volume bloc. Il est stocké dans le cloud de façon redondante.

✨ Avantages
  • Restauration rapide après panne ou erreur humaine
  • Créer de nouveaux volumes depuis un snapshot
  • Souvent incrémental : seule la différence occupe de l'espace
  • Utilisé pour la migration entre régions
📌 Bonne pratique
  • Planifier des snapshots automatiques (via AWS Backup, Azure Backup)
  • Ne pas oublier de supprimer les anciens snapshots pour maîtriser les coûts

"Un snapshot, c'est comme une machine à remonter le temps pour vos données !" ⏪💾

💡 Le stockage bloc est parfait pour les charges transactionnelles (base de données). Pour des fichiers statiques ou une bibliothèque multimédia, préférez le stockage objet (S3).

🛠️ En pratique : services cloud populaires

  • ☁️ AWS : Elastic Block Store (EBS) – volumes SSD/HDD, snapshots vers S3.
  • ☁️ Azure : Managed Disks – Ultra Disk, Premium SSD, Standard HDD.
  • ☁️ GCP : Persistent Disks – zonal ou régional, snapshots automatiques.

📝 Exemple d'utilisation (console AWS) :

# Créer un volume EBS de 100 Go type gp3
aws ec2 create-volume --size 100 --region eu-west-3 --volume-type gp3

# Attacher à une instance EC2
aws ec2 attach-volume --volume-id vol-xxxx --instance-id i-xxxx --device /dev/sdf

(commande illustrative, les élèves utiliseront la console ou Terraform)

📊 Optimisation & bonnes pratiques

✓ Choisir le bon type de disque selon votre workload (SSD pour base de données, HDD pour logs).
✓ Planifier des snapshots réguliers pour la reprise après sinistre.
✓ Surveiller les IOPS et le volume utilisé pour éviter le surdimensionnement.
✓ Utiliser des politiques de cycle de vie pour supprimer les anciens snapshots.
🧠 À retenir : Un volume bloc est persistant même si la VM est éteinte ou supprimée (sauf si on choisit la suppression à la termination). C'est parfait pour les données critiques.

✔️ Réponse : Sauvegardes incrémentales et restauration rapide.

✔️ Réponse : Pour garantir des performances constantes et élevées pour les applications critiques.
