🧠 React.memo : éviter les re-rendus inutiles

React.memo est un HOC qui ne re-rend le composant que si ses props changent.

const Button = React.memo(({ title, onPress }) => {
  console.log('Rendu Button');
  return <TouchableOpacity onPress={onPress}><Text>{title}</Text></TouchableOpacity>;
});

// Utilisation : le composant ne se re-rendra pas si les props sont identiques
⚠️ Attention : memo compare superficiellement les objets. Pour des props complexes, utilisez une fonction de comparaison personnalisée (deuxième argument).

💾 useMemo : mémoriser des valeurs calculées

useMemo recalcule la valeur uniquement quand les dépendances changent.

// Sans useMemo → recalcul à chaque rendu
const expensiveValue = heavyComputation(data);

// Avec useMemo → recalcul uniquement si 'data' change
const expensiveValue = useMemo(() => heavyComputation(data), [data]);

// Exemple concret : filtrage d'une grande liste
const filteredList = useMemo(() => {
  return items.filter(item => item.name.includes(searchTerm));
}, [items, searchTerm]);

📌 Utile pour les transformations coûteuses (tri, filtrage, calculs mathématiques).

🔄 useCallback : stabiliser les fonctions

Empêche la recréation d'une fonction à chaque rendu, utile pour les props passées à des composants memoïsés.

// Sans useCallback → nouvelle fonction à chaque rendu
const handlePress = () => { doSomething(id); };

// Avec useCallback → même fonction tant que les dépendances ne changent pas
const handlePress = useCallback(() => {
  doSomething(id);
}, [id]);

// Combiné avec React.memo → évite des re-rendus en chaîne
Utilisez toujours useCallback quand vous passez une fonction à un composant enfant optimisé avec memo.

📱 Optimisation de FlatList

Pour des listes longues (centaines d'éléments), ces props sont essentielles :

  • getItemLayout : optimise le calcul de hauteur des éléments (améliore le scroll).
  • initialNumToRender : nombre d'éléments rendus au départ (défaut: 10).
  • maxToRenderPerBatch : éléments chargés par lot (défaut: 10).
  • windowSize : hauteur virtuelle de rendu (défaut: 21).
  • removeClippedSubviews : démonte les vues hors écran (économise la mémoire).
<FlatList
  data={data}
  renderItem={renderItem}
  keyExtractor={item => item.id}
  getItemLayout={(data, index) => ({ length: 80, offset: 80 * index, index })}
  initialNumToRender={8}
  removeClippedSubviews={true}
/>

🔍 Outils de profiling

React DevTools : onglet "Profiler" pour enregistrer les rendus et voir les composants qui se re-rendent trop souvent.
Flipper (React Native Debugger) : monitorer les performances, réseau, layout.
console.log + Performance.now() pour mesurer le temps d'exécution.
🚀 Activez le mode "Highlight updates" dans React DevTools pour visualiser les re-rendus en temps réel.

⚠️ Problèmes courants de performance

  • Fonctions anonymes dans les props → utiliser useCallback
  • Objets créés à chaque rendu → utiliser useMemo
  • Grandes listes sans getItemLayout → scroll saccadé
  • setState dans des composants profonds → remonter l'état ou utiliser Context avec précaution
  • Images non optimisées → utiliser des dimensions fixes, cache

✏️ Exercice : optimisez ce code

// Code problématique : beaucoup de re-rendus
const Parent = () => {
  const [count, setCount] = useState(0);
  const [items, setItems] = useState([...]);
  const filtered = items.filter(i => i.active); // recalcul à chaque rendu
  const handlePress = () => console.log('pressed'); // nouvelle fonction
  return <Child items={filtered} onPress={handlePress} />;
};

Proposez une version optimisée avec useMemo, useCallback, et React.memo sur Child.

🧩 Voir la solution
const filtered = useMemo(() => items.filter(i => i.active), [items]);
const handlePress = useCallback(() => console.log('pressed'), []);
const ChildMemo = React.memo(Child);
return <ChildMemo items={filtered} onPress={handlePress} />;
