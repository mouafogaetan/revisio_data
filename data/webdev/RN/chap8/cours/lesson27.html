💡 À retenir : Redux Toolkit simplifie énormément Redux classique. Plus besoin d'écrire des actions types, des créateurs d'actions et des reducers séparément.

📦 Qu'est-ce que Redux Toolkit ?

Redux Toolkit (RTK) est la façon officielle et recommandée d'écrire la logique Redux. Il résout trois problèmes majeurs :

🔧

Configuration simplifiée

Plus besoin de combiner manuellement les reducers avec combineReducers.

⚡

Immutabilité intégrée

Utilise Immer pour écrire des mutations "mutables" qui deviennent immutables.

🧩

Thunk intégré

Gestion des actions asynchrones sans bibliothèque supplémentaire.

📌 Composants clés :

  • configureStore() : crée le store Redux avec les bons defaults
  • createSlice() : génère automatiquement actions et reducers
  • createAsyncThunk : pour les appels API asynchrones

🛠️ Installation

Dans un projet React Native (Expo ou CLI), ouvrez un terminal et exécutez :

npm install @reduxjs/toolkit react-redux

Ou avec yarn :

yarn add @reduxjs/toolkit react-redux

✅ Note : react-redux est le binding officiel pour connecter React à Redux. Il fournit les hooks useSelector et useDispatch.

✂️ Créer un Slice (exemple compteur)

Un slice regroupe l'état initial, les reducers et les actions générées automatiquement.

// store/counterSlice.js
import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  value: 0
};

const counterSlice = createSlice({
  name: 'counter',
  initialState,
  reducers: {
    increment: (state) => {
      state.value += 1;
    },
    decrement: (state) => {
      state.value -= 1;
    },
    incrementByAmount: (state, action) => {
      state.value += action.payload;
    }
  }
});

export const { increment, decrement, incrementByAmount } = counterSlice.actions;
export default counterSlice.reducer;

✨ Grâce à Immer, on peut écrire state.value += 1 comme si c'était mutable.

🏪 Configurer le Store

Le store centralise tout l'état de l'application.

// store/index.js
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from './counterSlice';

export const store = configureStore({
  reducer: {
    counter: counterReducer
  }
});

🔍 Provider : Dans App.js, enveloppez votre application avec <Provider store={store}> pour que tous les composants puissent accéder au store.

🎣 Hooks : useSelector & useDispatch

Dans un composant, lire l'état et envoyer des actions :

// components/Counter.js
import React from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement } from '../store/counterSlice';

const Counter = () => {
  const count = useSelector((state) => state.counter.value);
  const dispatch = useDispatch();

  return (
    <View style={styles.container}>
      <Text style={styles.count}>{count}</Text>
      <Button title="Incrémenter" onPress={() => dispatch(increment())} />
      <Button title="Décrémenter" onPress={() => dispatch(decrement())} />
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  count: { fontSize: 48, fontWeight: 'bold', marginBottom: 20 }
});

export default Counter;
📖 useSelector souscrit au store et se réexécute quand l'état change. useDispatch renvoie la fonction pour envoyer des actions.

🌐 Actions asynchrones avec createAsyncThunk

Pour récupérer des données depuis une API :

// store/userSlice.js
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';
import axios from 'axios';

export const fetchUser = createAsyncThunk(
  'user/fetchUser',
  async (userId) => {
    const response = await axios.get(`https://jsonplaceholder.typicode.com/users/${userId}`);
    return response.data;
  }
);

const userSlice = createSlice({
  name: 'user',
  initialState: { data: null, loading: false, error: null },
  reducers: {},
  extraReducers: (builder) => {
    builder
      .addCase(fetchUser.pending, (state) => {
        state.loading = true;
      })
      .addCase(fetchUser.fulfilled, (state, action) => {
        state.loading = false;
        state.data = action.payload;
      })
      .addCase(fetchUser.rejected, (state, action) => {
        state.loading = false;
        state.error = action.error.message;
      });
  }
});

export default userSlice.reducer;

⚡ Les trois états : pending (chargement), fulfilled (succès), rejected (erreur).

📝 Résumé & Bonnes pratiques

✅

Toujours utiliser Redux Toolkit pour tout nouveau projet Redux.

✅

Organiser les slices par fonctionnalité (ex: userSlice.js, productsSlice.js).

✅

Utiliser createAsyncThunk pour les appels API (évite du boilerplate).

✅

Ne stocker que l'état global dans Redux (préférences utilisateur, authentification, données partagées).
