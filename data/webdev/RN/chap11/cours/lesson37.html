📱 Tester sur plusieurs appareils

Une application doit fonctionner sur une variété d'appareils !

Émulateurs / Simulateurs

Android Studio (AVD) • Xcode Simulator (iOS) • Expo Go

Appareils physiques

Tester sur Android (Samsung, Xiaomi, Pixel) et iOS (iPhone, iPad)

Services de test cloud

BrowserStack, Firebase Test Lab, Sauce Labs

Conseil : Testez sur au moins un petit écran et un grand écran !

🐛 Outils de debugging

React Native Debugger

Debugger standalone avec Redux DevTools

Flipper

Outil de debugging Meta (logs, réseau, layouts)

Chrome DevTools

Debugger le JavaScript via http://localhost:8081/debugger-ui

Console logs

console.log, console.warn, console.error

⚡ Optimisation des performances

useMemo et useCallback – Éviter les recalculs inutiles
FlatList avec getItemLayout et initialNumToRender
React.memo – Éviter les re-rendus superflus
Optimisation des images – Formats modernes (WebP), compression, lazy loading
Code splitting avec React.lazy et Suspense

Utilisez le Profiler de React DevTools pour identifier les goulets d'étranglement !

🖼️ Préparation des assets

📱 Icône de l'application

Préparer des icônes en plusieurs résolutions (48x48, 72x72, 96x96, 144x144, 192x192, 512x512)

🖼️ Splash screen

Image de lancement pour Android et iOS (outil : expo-splash-screen)

📸 Captures d'écran

Pour les stores : 2-8 captures (format portrait et/ou paysage)

// app.json - configuration des assets
{
  "expo": {
    "icon": "./assets/icon.png",
    "splash": {
      "image": "./assets/splash.png",
      "resizeMode": "contain"
    }
  }
}

🤖 Génération APK / AAB (Android)

📦 Avec EAS Build (recommandé)

eas build --platform android --profile production

📦 Avec Expo (ancienne méthode)

expo build:android -t app-bundle  # AAB pour Play Store
expo build:android -t apk          # APK pour test

Important :

Conservez précieusement votre keystore (fichier .jks) ! Sans lui, impossible de publier des mises à jour.

🍎 Préparation iOS & App Store

Adhérer à l'Apple Developer Program (99$/an)
Générer un certificat de distribution et un provisioning profile
Compiler avec eas build --platform ios ou Xcode
Tester avec TestFlight avant publication 
# Générer un build iOS avec EAS
eas build --platform ios --profile production

# Soumettre automatiquement
eas submit --platform ios

📤 Soumission aux stores

Google Play Console

  • Créer une fiche application
  • Télécharger l'AAB
  • Remplir la fiche (description, captures)
  • Déploiement interne → alpha → bêta → production

App Store Connect

  • Créer une fiche application
  • Uploader l'IPA via Transporter
  • Remplir les métadonnées
  • Soumettre à validation Apple

Délais : Google (2h-24h), Apple (24h-48h en moyenne)

✅ Checklist de pré-lancement

Tester sur Android + iOS (appareils physiques)
Vérifier les permissions (caméra, localisation, etc.)
Optimiser les performances (rendu, mémoire)
Préparer les icônes et le splash screen
Configurer les numéros de version (versionName + versionCode)
Supprimer les logs de debug et les console.log inutiles
Signer l'application (keystore Android / certificat iOS)
Rédiger une description attrayante et des mots-clés
