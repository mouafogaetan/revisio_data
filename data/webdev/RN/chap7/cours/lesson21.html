📌 Qu'est-ce qu'une modale ?

Une modale est une fenêtre qui s'affiche par-dessus l'écran principal, forçant l'utilisateur à interagir avec elle avant de revenir à l'application.

🎨 Exemple concret :

  • Formulaire de connexion rapide
  • Message de confirmation avant suppression
  • Paramètres rapides
  • Affichage d'une image en grand
┌─────────────────────────────┐ │ Écran principal │ │ ┌───────────────┐ │ │ │ MODALE │ │ │ │ (par dessus) │ │ │ └───────────────┘ │ └─────────────────────────────┘

🛠️ Le composant Modal

Propriétés essentielles :

  • visible : booléen (true = affiché, false = caché)
  • onRequestClose : callback (Android obligatoire pour bouton retour)
  • animationType : 'slide', 'fade', 'none'
  • transparent : arrière-plan transparent
import { Modal, View, Text, Button } from 'react-native';

const [modalVisible, setModalVisible] = useState(false);

<Modal
  visible={modalVisible}
  animationType="slide"
  transparent={true}
  onRequestClose={() => setModalVisible(false)}>
  <View style={{ flex:1, justifyContent:'center', backgroundColor:'rgba(0,0,0,0.5)'}}>
    <View style={{ backgroundColor:'white', margin:20, padding:20, borderRadius:10 }}>
      <Text>Contenu modal !</Text>
      <Button title="Fermer" onPress={() => setModalVisible(false)}/>
    </View>
  </View>
</Modal>

⚠️ Alertes système natives

Utilisez Alert.alert() pour des popups rapides avec boutons.

import { Alert } from 'react-native';

// Alerte simple
Alert.alert('Titre', 'Message important');

// Avec boutons
Alert.alert(
  'Confirmation',
  'Voulez-vous supprimer cet élément ?',
  [
    { text: 'Annuler', style: 'cancel' },
    { text: 'Supprimer', onPress: () => console.log('Supprimé'), style: 'destructive' }
  ]
);
🔔 Sur iOS, les Alertes sont natives, sur Android également. Elles bloquent l'interaction avec l'arrière-plan.

🔄 Gestion des états d'ouverture

Avec useState, on contrôle la visibilité de la modale.

const [isModalOpen, setIsModalOpen] = useState(false);

const openModal = () => setIsModalOpen(true);
const closeModal = () => setIsModalOpen(false);

// Dans le JSX
<Button title="Ouvrir Modale" onPress={openModal} />
<Modal visible={isModalOpen} onRequestClose={closeModal}>
  <!-- contenu -->
</Modal>

✅ Toujours prévoir un moyen de fermeture (bouton Fermer ou geste de retour).

🎨 Modale transparente & design

En mettant transparent={true} et un fond semi-transparent, on peut créer des modales "centrées".

<Modal transparent={true} visible={showModal}>
  <View style={{ flex:1, justifyContent:'center', alignItems:'center', backgroundColor:'rgba(0,0,0,0.5)' }}>
    <View style={{ backgroundColor:'white', borderRadius:20, padding:20, width:'80%' }}>
      <Text style={{ fontSize:18, fontWeight:'bold' }}>🎉 Super Modale</Text>
      <Button title="Fermer" onPress={closeModal} />
    </View>
  </View>
</Modal>
✨ Astuce : On peut insérer des champs TextInput ou des listes dans une modale.

📱 Mini-exemple : Modale de contact

const ContactModal = () => {
  const [modalVisible, setModalVisible] = useState(false);
  return (
    <View>
      <Button title="Nous contacter" onPress={() => setModalVisible(true)}/>
      <Modal visible={modalVisible} animationType="slide" transparent onRequestClose={() => setModalVisible(false)}>
        <View style={{ flex:1, justifyContent:'center', backgroundColor:'rgba(0,0,0,0.7)' }}>
          <View style={{ margin:20, backgroundColor:'white', borderRadius:15, padding:20 }}>
            <Text style={{ fontSize:20, fontWeight:'bold' }}>Envoyer un message</Text>
            <TextInput placeholder="Votre email" style={{ borderWidth:1, marginVertical:10, padding:8 }}/>
            <Button title="Envoyer" onPress={() => { Alert.alert('Merci !'); setModalVisible(false); }}/>
            <Button title="Annuler" onPress={() => setModalVisible(false)} color="red"/>
          </View>
        </View>
      </Modal>
    </View>
  );
};

💡 Bonnes pratiques

  • Toujours fournir onRequestClose sur Android pour capturer le bouton retour physique.
  • Éviter les modales trop grandes (préférer le scroll avec ScrollView).
  • Pour les confirmations rapides, privilégier Alert.alert.
  • Ne pas superposer plusieurs modales (risque de confusion).
  • Accessibilité : ajouter aria-label ou accessibilityLabel.
🌟 Règle d'or : la modale doit avoir un moyen clair de fermeture (bouton "Fermer" ou croix).

🎓 Ce qu'il faut retenir

✓ Modal pour des interfaces riches
✓ Alert.alert pour des messages rapides
✓ Gérer la visibilité avec useState
✓ transparent + fond semi-transparent = overlay
