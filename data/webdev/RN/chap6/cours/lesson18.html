Leçon 18 – fetch() dans React Native

Apprenez à récupérer des données depuis un serveur, gérer le chargement et les erreurs.

🌐 fetch API ⏳ États (loading/error) 📋 FlatList

Objectifs de la leçon

  • ✓ Récupérer des données avec fetch() dans useEffect.
  • ✓ Gérer les états de chargement (loading) et d'erreur.
  • ✓ Afficher les données dans une FlatList performante.
💡 Une bonne gestion des requêtes API est essentielle pour des applications mobiles professionnelles.

🤔 Pourquoi fetch dans useEffect ?

Les requêtes API sont des effets de bord. Elles doivent être exécutées après le rendu du composant, pas pendant. useEffect est l'endroit parfait.

📌 Rappel :

  • Le rendu doit être pur (pas d'appel réseau).
  • useEffect s'exécute après que React a mis à jour l'interface.
  • On utilise un tableau de dépendances vide [] pour charger les données une seule fois au montage.
// Structure de base dans React Native
useEffect(() => {
  fetch('https://api.exemple.com/data')
    .then(response => response.json())
    .then(data => setData(data));
}, []);

⏳ Afficher un indicateur de chargement

Pendant que les données arrivent, montrez un ActivityIndicator ou un message à l'utilisateur.

function ListeUtilisateurs() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => {
        setUsers(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <ActivityIndicator size="large" color="#4f46e5" />;
  if (error) return <Text>Erreur : {error}</Text>;
  return (
    <FlatList
      data={users}
      keyExtractor={(item) => item.id.toString()}
      renderItem={({ item }) => <Text>{item.name}</Text>}
    />
  );
}

⚠️ Gestion des erreurs HTTP (404, 500)

fetch() ne rejette pas la promesse pour les erreurs HTTP. Il faut vérifier response.ok.

useEffect(() => {
  const fetchData = async () => {
    setLoading(true);
    try {
      const response = await fetch('https://api.exemple.com/data');
      if (!response.ok) {
        throw new Error(`HTTP ${response.status}: ${response.statusText}`);
      }
      const data = await response.json();
      setData(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };
  fetchData();
}, []);
🔴 Important : Toujours gérer le cas où le serveur répond avec une erreur (404, 500, etc.).

📋 FlatList : le composant roi des listes

Pour afficher de grandes quantités de données, FlatList est bien plus performant que ScrollView car il rend uniquement les éléments visibles.

<FlatList
  data={users}
  keyExtractor={(item) => item.id.toString()}
  renderItem={({ item }) => (
    <View style={styles.item}>
      <Text style={styles.name}>{item.name}</Text>
      <Text style={styles.email}>{item.email}</Text>
    </View>
  )}
  ItemSeparatorComponent={() => <View style={styles.separator} />}
  ListEmptyComponent={<Text>Aucun utilisateur</Text>}
/>
✨ Optimisation : utilisez getItemLayout si tous les éléments ont la même hauteur pour des performances maximales.

📱 Exemple complet : Actualités

import React, { useState, useEffect } from 'react';
import { View, Text, FlatList, ActivityIndicator, StyleSheet } from 'react-native';

function ActualitesScreen() {
  const [articles, setArticles] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch('https://jsonplaceholder.typicode.com/posts')
      .then(res => {
        if (!res.ok) throw new Error('Erreur réseau');
        return res.json();
      })
      .then(data => {
        setArticles(data.slice(0, 20));
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <ActivityIndicator style={styles.center} size="large" />;
  if (error) return <Text style={styles.center}>Erreur : {error}</Text>;

  return (
    <FlatList
      data={articles}
      keyExtractor={item => item.id.toString()}
      renderItem={({ item }) => (
        <View style={styles.card}>
          <Text style={styles.title}>{item.title}</Text>
          <Text>{item.body.slice(0, 100)}...</Text>
        </View>
      )}
    />
  );
}

✅ Bonnes pratiques à retenir

  • Toujours utiliser useEffect avec les dépendances adaptées.
  • Ne pas appeler fetch() directement dans le corps du composant.
  • Gérer systématiquement les 3 états : loading, data, error.
  • Vérifier response.ok pour les erreurs HTTP.
  • Utiliser FlatList pour les listes, pas ScrollView.
  • Extraire la logique de fetch dans des hooks personnalisés pour la réutiliser.

⚠️ Attention :

N'oubliez pas d'ajouter keyExtractor dans FlatList avec une clé unique (souvent l'id).

function MeteoScreen() {
  const [meteo, setMeteo] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch('https://api.open-meteo.com/v1/forecast?latitude=4.05&longitude=9.7¤t_weather=true')
      .then(res => res.json())
      .then(data => {
        setMeteo(data.current_weather);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <ActivityIndicator />;
  if (error) return <Text>Erreur: {error}</Text>;
  return (
    <View>
      <Text>Température: {meteo.temperature}°C</Text>
      <Text>Vent: {meteo.windspeed} km/h</Text>
    </View>
  );
}
