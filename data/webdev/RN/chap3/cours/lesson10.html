useEffect

Cycle de vie et effets secondaires dans React Native

Maîtrisez les moments clés : montage, mise à jour et démontage des composants.

🎯

Objectifs de la leçon

🔄 useEffect
Exécuter du code après le rendu du composant.
📦 Tableau de dépendances
Contrôler quand l'effet se réexécute.
🧹 Cleanup function
Nettoyer les abonnements, timers et requêtes.
⏱️ Cycle de vie
Comprendre montage, mise à jour, démontage.
💡 useEffect est le hook qui remplace les méthodes de cycle de vie des composants de classe.
🤔

Qu'est-ce que useEffect ?

useEffect permet de réaliser des effets secondaires dans vos composants :

  • Appels API (fetch)
  • Abonnements (WebSocket, listeners)
  • Manipulation directe du DOM (ou du composant natif)
  • Timers (setTimeout, setInterval)
  • Mise à jour du titre de la page
import { useEffect } from 'react';

function MonComposant() {
  useEffect(() => {
    // Code exécuté APRÈS le rendu
    console.log('Effet exécuté !');
    
    // Optionnel : fonction de nettoyage
    return () => {
      console.log('Nettoyage avant démontage');
    };
  }, []); // Tableau de dépendances
  
  return <View><Text>Composant</Text></View>;
}

📋 Tableau de dépendances

Il contrôle quand l'effet se réexécute :

[]

Exécuté une seule fois après le montage

équivalent à componentDidMount

[dep1, dep2]

Exécuté au montage et à chaque changement de dep1 ou dep2

(pas de tableau)

Exécuté après chaque rendu (⚠️ risqué !)

function Example({ userId }) {
  // 1 seul fois au montage
  useEffect(() => { console.log('Montage'); }, []);
  
  // À chaque changement de userId
  useEffect(() => { fetchUser(userId); }, [userId]);
  
  // À chaque rendu (déconseillé)
  useEffect(() => { console.log('Rerender'); });
}

🧹 La fonction de nettoyage

La fonction retournée par useEffect s'exécute :

  • Avant le démontage du composant
  • Avant la ré-exécution de l'effet (pour éviter les fuites mémoire)
useEffect(() => {
  const timer = setInterval(() => {
    console.log('Tick');
  }, 1000);
  
  // Nettoyage : on arrête le timer
  return () => {
    clearInterval(timer);
    console.log('Timer arrêté');
  };
}, []);
⚠️ Attention : Sans nettoyage, les timers et abonnements continuent de s'exécuter en arrière-plan, causant des fuites mémoire !

📡 Exemple : Appel API avec useEffect

function UserList() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    let isMounted = true; // Flag pour éviter setState après démontage
    
    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => {
        if (isMounted) {
          setUsers(data);
          setLoading(false);
        }
      })
      .catch(err => console.error(err));
    
    return () => { isMounted = false; }; // Nettoyage
  }, []); // Une seule fois au montage

  if (loading) return <Text>Chargement...</Text>;
  return <FlatList data={users} renderItem={...} />;
}
✨ Astuce : Toujours annuler les requêtes asynchrones ou utiliser un flag au démontage pour éviter les "memory leaks".

📊 useEffect vs composants de classe

Composant fonctionnel (moderne)

useEffect(() => {
  // componentDidMount
  return () => { /* componentWillUnmount */ };
}, []);

useEffect(() => {
  // componentDidUpdate
}, [dep]);

Composant de classe (ancien)

componentDidMount() {}
componentDidUpdate() {}
componentWillUnmount() {}
✅ Avantage : useEffect regroupe logique liée, contrairement aux méthodes de cycle de vie qui fragmentent le code.

⚠️ Pièges à éviter

❌ Boucle infinie

useEffect(() => {
  setCount(count + 1); // setState dans l'effet
}); // pas de dépendances → rerender infini !

⚠️ Dépendances manquantes

useEffect(() => {
  fetchUser(userId); // userId utilisé mais pas dans les dépendances
}, []); // eslint-disable-next-line (mauvaise pratique)

❌ SetState après démontage

// Solution : utiliser un flag isMounted ou AbortController
