Partie II

Correction & Refactorisation (6 pts)

On donne l'extrait suivant d'un écran React Native :

// Composant problématique
import { View, Text, ScrollView, Image } from 'react-native';

export default function ProductList({ items }) {
  return (
    <ScrollView>
      {items.map(product => (
        <View key={product.id} style={{ marginBottom: 20, borderWidth: 1 }}>
          <Image source={{ uri: product.image }} style={{ width: 100, height: 100 }} />
          <Text>{product.name} - {product.price}€</Text>
        </View>
      ))}
    </ScrollView>
  );
}

a) (2 pts) Identifiez au moins deux limitations ou problèmes de performance dans ce code. Proposez des corrections en utilisant les composants appropriés.

b) (2 pts) Réécrivez le composant en utilisant FlatList et en respectant les bonnes pratiques (key, séparation du style, rendu performant).

c) (2 pts) En vous basant sur le cours, expliquez comment rendre les éléments interactifs (ajouter un bouton "Ajouter au panier") avec un retour visuel natif (opacité/ripple). Nommez le composant adapté.

Partie III

Mini-projet – Carte de visite interactive (9 pts)

🧩 Contexte :

Vous devez développer un composant BusinessCard qui affiche les informations d'un contact (nom, métier, email, avatar). La carte doit être stylisée avec StyleSheet, utiliser Image, Text, View, et un TouchableOpacity pour afficher une alerte lorsqu'on clique sur l'email. Gérez également un champ TextInput pour modifier dynamiquement le métier.

❶ (2 pts) Définissez le style avec StyleSheet.create(). Créez un conteneur principal avec ombrage (shadow pour iOS / elevation pour Android), bords arrondis, et un padding intérieur.

❷ (2.5 pts) Structure JSX : une View parent contient une Image (avatar circulaire), une View pour les textes (nom, métier, email). Le métier doit être lié à un état (useState) modifiable via un TextInput.

❸ (2 pts) Gérez l'interaction : lorsque l'utilisateur clique sur l'email, déclenchez Alert.alert() (simulez l'envoi). Utilisez TouchableOpacity pour un retour visuel.

❹ (2.5 pts) Justifiez l'utilisation de SafeAreaView sur les appareils modernes (iPhone X+). Expliquez comment Platform.select() permet d'adapter les ombres entre iOS et Android. [citation:4][citation:8]

✨ À retenir : les composants React Native se comportent comme du React, mais le rendu est natif. View → div, Text → span/p, et le style utilise la casse camelCase. 📱 Bon courage !
