🚀 Lancer l'application avec Expo

Expo est la façon la plus simple de démarrer. Une fois le projet créé avec npx create-expo-app mon-app :

📱 Commandes :

cd mon-app
npm start  # ou npx expo start

Un QR code s'affiche dans le terminal. Scannez-le avec l'application Expo Go sur votre téléphone (Android ou iOS).

💡 Astuce : Appuyez sur a dans le terminal pour ouvrir sur un émulateur Android, i pour iOS (si Xcode est installé).

⚙️ Lancer avec React Native CLI

Pour les projets nécessitant des bibliothèques natives, on utilise le CLI traditionnel :

npx react-native init MonApp
cd MonApp
npx react-native start   # démarre le Metro bundler
# Dans un autre terminal :
npx react-native run-android   # ou run-ios
⚠️ Prérequis : Android Studio (avec SDK) pour Android, Xcode pour iOS (Mac uniquement).

📁 Structure d'un projet React Native

📄 Fichiers importants

  • App.js : Composant racine de l'application
  • app.json : Configuration de l'app (nom, icône, splash screen)
  • package.json : Dépendances et scripts
  • metro.config.js : Configuration du bundler Metro
  • babel.config.js : Configuration Babel

📁 Dossiers

  • assets/ : Images, polices, icônes
  • src/ : Code source (components, screens, navigation)
  • android/ : Projet Android natif (CLI uniquement)
  • ios/ : Projet iOS natif (CLI uniquement)
mon-app/
├── App.js                # Point d'entrée
├── app.json              # Configuration Expo/CLI
├── assets/               # Images et ressources
│   └── icon.png
├── src/                  # Code source (à organiser)
│   ├── components/
│   ├── screens/
│   └── navigation/
└── package.json

🛠️ Outils de développement

Metro Bundler

Serveur de développement React Native. Ouvre dans le navigateur sur http://localhost:8081. Permet de voir les logs, recharger, ouvrir dev menu.

Expo Go (Expo)

Application mobile pour tester rapidement vos applis Expo sans build. Scannez le QR code, l'app se charge instantanément.

React Native Debugger

Outil tout-en-un : React DevTools, Redux DevTools, network inspection. À installer séparément.

Dev Menu (secouer l'app ou Cmd+D / Ctrl+M)

Accès rapide : Reload, Debug, Enable Fast Refresh, Show Perf Monitor.

⚡ Fast Refresh

Fast Refresh est la fonctionnalité qui recharge instantanément votre code après modification, tout en préservant l'état du composant.

✅ Ce qui fonctionne :

  • Changements dans des composants fonctionnels
  • Modifications de styles
  • Corrections de bugs mineurs

⚠️ Ce qui force un rechargement complet :

  • Modification dans un fichier non-React
  • Erreur de syntaxe
  • Changement de hooks ou de l'ordre des hooks
🐞 Pour déboguer : Dans le Dev Menu, activez "Debug JS Remotely". Chrome DevTools s'ouvre pour déboguer le code JavaScript.

📌 Récapitulatif

Expo (recommandé débutant)

  • Pas de build native
  • Test instantané avec Expo Go
  • Idéal pour apprendre et prototyper

React Native CLI (avancé)

  • Contrôle total du code natif
  • Nécessite Android Studio/Xcode
  • Pour apps avec librairies natives complexes
