Expo vs React Native CLI

Choisir la bonne approche pour vos projets mobiles

📱 Expo Go ⚙️ React Native CLI

  • 1 Créer une application avec Expo Go (npx create-expo-app)

💡 "Expo accélère le développement, React Native CLI donne la liberté totale"

📱 React Native en bref

React Native permet de créer des applications mobiles natives (iOS et Android) avec JavaScript/React.

Deux façons de développer :

  • Expo : framework qui simplifie le développement
  • React Native CLI : outils officiels, plus de contrôle

🚀 Expo : la voie express

Expo est une boîte à outils qui inclut :

  • Un client mobile Expo Go pour prévisualiser immédiatement
  • Des bibliothèques pré-intégrées (appareil photo, localisation, notifications)
  • Une ligne de commande expo et eas pour les builds
# Créer un projet Expo
npx create-expo-app MonApp

# Démarrer le serveur de développement
cd MonApp
npx expo start

Scannez le QR code avec Expo Go (Android/iOS) pour voir l'application en direct !

📲 Expo Go en action

1. Installez l'app Expo Go depuis Google Play ou App Store.

2. Lancez npx expo start dans votre projet.

3. Scannez le QR code avec Expo Go.

✅ Résultat : votre application s'affiche immédiatement sur votre téléphone !

Le code est exécuté via le bundler Metro, les modifications sont rechargées à chaud.

⚙️ React Native CLI

L'approche officielle, plus proche du « metal ».

  • Nécessite Android Studio (Android) et/ou Xcode (macOS uniquement pour iOS)
  • Vous compilez l'application nativement sur votre machine
  • Contrôle total sur les bibliothèques natives
# Installer React Native CLI
npm install -g react-native-cli

# Créer un projet
npx react-native init MonAppCLI

# Lancer sur Android (émulateur ou téléphone)
npx react-native run-android

# Lancer sur iOS (Mac uniquement)
npx react-native run-ios

🔧 Prérequis techniques

📱 Android

  • Android Studio (SDK, émulateur)
  • Java JDK 17
  • Variables d'environnement (ANDROID_HOME)
  • Appareil physique ou émulateur

🍎 iOS (Mac uniquement)

  • Xcode (dernière version)
  • Command Line Tools
  • CocoaPods (sudo gem install cocoapods)
  • Simulateur iOS ou appareil

⚠️ L'installation peut prendre plusieurs heures et nécessite un espace disque important !

⚖️ Expo vs React Native CLI

CritèreExpoReact Native CLI
🚀 Facilité d'installation✅ Très simple (Node.js uniquement)❌ Complexe (Android Studio/Xcode)
📲 Test sur appareil✅ QR Code avec Expo Go✅ Connexion USB ou build
🔌 Modules natifs personnalisés⚠️ Limité (expo-build-properties, config plugins)✅ Illimité (lien natif complet)
📦 Taille de l'applicationPlus lourde (inclus expo SDK)Optimisable
📤 Publication sur Stores✅ via EAS Build✅ Manuellement ou CI/CD
💰 CoûtGratuit (EAS Build payant pour les grosses équipes)Gratuit

✅ Expo est idéal pour :

  • Débuter en React Native (courbe d'apprentissage douce)
  • Prototyper rapidement une idée
  • Projets sans modules natifs exotiques (utilisez les bibliothèques Expo)
  • Équipes sans expérience mobile native
  • Démonstrations client, hackathons, formations

💡 Expo SDK fournit plus de 50 modules (camera, location, maps, haptics, etc.)

🔧 React Native CLI est préférable si :

  • Vous avez besoin d'un module natif non supporté par Expo (ex: bibliothèque Bluetooth spécifique)
  • Vous souhaitez optimiser la taille de l'application au maximum
  • Vous intégrez un SDK propriétaire (bancaire, IoT)
  • Vous voulez un contrôle fin sur le processus de build (gradle, xcode)
  • Vous avez déjà une expérience mobile native

🔁 Expo prebuild : hybride puissant

Expo permet d'éjecter le projet si besoin avec expo prebuild.

# Générer les dossiers natifs (ios/ et android/)
npx expo prebuild

# Vous pouvez ensuite ajouter des modules natifs classiques
# Puis build avec EAS ou localement

Cela donne la flexibilité de React Native CLI tout en gardant la simplicité d'Expo au début.

✨ Expo Application Services (EAS) : builds cloud, mises à jour OTA (Over‑The‑Air).

🧪 Exercice : Créez votre première app Expo

  1. Installez Node.js (si ce n'est pas fait).
  2. Dans votre terminal : npx create-expo-app MonPremierProjet.
  3. cd MonPremierProjet
  4. npx expo start
  5. Téléchargez Expo Go sur votre téléphone.
  6. Scannez le QR code.
  7. Modifiez App.js (changez le texte) et observez le rafraîchissement automatique.
🔎 Question : Comment passer de Expo à React Native CLI si nécessaire ?

Utilisez npx expo prebuild pour générer les dossiers natifs, puis vous pouvez travailler avec npx react-native run-android ou run-ios. La migration reste possible, mais une fois que vous ajoutez du code natif personnalisé, vous ne pourrez plus utiliser Expo Go (vous devrez compiler l'application vous-même).

📌 Points clés à retenir

📱 Expo – rapide, simple, QR code, SDK riche
⚙️ CLI – contrôle total, modules natifs personnalisés
📲 Expo Go – test instantané sans build
🔧 EAS Build – builds cloud et mises à jour OTA

Choisissez Expo pour débuter et prototyper, passez au CLI pour des besoins avancés ! 🚀
