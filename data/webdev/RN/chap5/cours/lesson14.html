À la fin de ce cours, vous saurez naviguer entre plusieurs écrans dans votre application mobile.

📱 Pourquoi React Navigation ?

React Navigation est la bibliothèque de navigation la plus populaire pour React Native. Elle permet de gérer les transitions entre écrans avec une expérience utilisateur fluide et native.

Pourquoi l’utiliser ?

  • Gestes natifs (swipe retour)
  • Animations fluides
  • Deep linking
  • Large communauté

Types de navigation

  • Stack (pile)
  • Tab (onglets)
  • Drawer (menu latéral)
  • Bottom Tabs

⚙️ Installation

Pour commencer, installez les packages nécessaires dans votre projet Expo ou React Native CLI :

# Installation de base
npm install @react-navigation/native

# Dépendances natives (Expo)
npx expo install react-native-screens react-native-safe-area-context

# Pour la Stack Navigation
npm install @react-navigation/native-stack
Note : Si vous utilisez React Native CLI, suivez la documentation pour l’installation sur iOS (pod install) et Android (autolinking).

🥞 Stack Navigation (pile)

La Stack Navigation fonctionne comme une pile : chaque nouvel écran est ajouté au sommet, et le bouton retour le retire.

Écran Accueil (Home)
Écran Détails (Details)
Écran Profil (Profile)

← Le bouton retour enlève l’écran du haut

Exemple : Accueil → Détails → Profil. En appuyant sur retour, on revient de Profil à Détails puis à Accueil.

📦 NavigationContainer

NavigationContainer est le composant racine qui gère l’état de navigation. Il doit envelopper toute l’application.

import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

const Stack = createNativeStackNavigator();

function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator>
        <Stack.Screen name="Accueil" component={HomeScreen} />
        <Stack.Screen name="Details" component={DetailsScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}

Chaque Stack.Screen représente un écran de votre application.

🚀 Exemple complet

// HomeScreen.js
function HomeScreen({ navigation }) {
  return (
    <View>
      <Text>Écran Accueil</Text>
      <Button
        title="Voir détails"
        onPress={() => navigation.navigate('Details')}
      />
    </View>
  );
}

// DetailsScreen.js
function DetailsScreen() {
  return (
    <View>
      <Text>Écran Détails</Text>
    </View>
  );
}
navigation.navigate('NomDeLEcran') permet de passer d’un écran à l’autre.

🎨 Personnalisation du header

<Stack.Navigator
  screenOptions={{
    headerStyle: { backgroundColor: '#4f46e5' },
    headerTintColor: '#fff',
    headerTitleStyle: { fontWeight: 'bold' }
  }}
>
  <Stack.Screen 
    name="Accueil" 
    component={HomeScreen}
    options={{ title: 'Bienvenue' }}
  />
</Stack.Navigator>
  • screenOptions : applique les mêmes options à tous les écrans
  • options : personnalise un écran spécifique
  • Possibilité de cacher le header avec headerShown: false

💡 Question : Quelle est la différence entre navigation.navigate('Ecran') et navigation.push('Ecran') ?

Réponse : navigate() retourne à un écran existant dans la pile, push() ajoute toujours un nouvel écran (même si déjà présent).
