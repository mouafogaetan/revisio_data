📦 View et Text : les conteneurs universels

View est le composant le plus fondamental. C'est un conteneur qui supporte la mise en page (Flexbox), le style, les gestes tactiles. Text est le seul composant capable d'afficher du texte (contrairement au web où on peut mettre du texte directement dans une div).

📱 React Native

import { View, Text } from 'react-native'; function App() { return ( <View style={{ padding: 20 }}> <Text>Bonjour React Native !</Text> <Text>Texte sur plusieurs lignes</Text> </View> ); }

🌐 Équivalent web

<div style="padding: 20px;"> <p>Bonjour React !</p> <p>Texte sur plusieurs lignes</p> </div>

💡 À retenir : Tout texte DOIT être dans un composant Text, jamais directement dans une View.

🎨 StyleSheet.create() : organiser ses styles

En React Native, on n'écrit pas de fichiers CSS. On utilise StyleSheet.create() pour créer un objet de styles, ce qui apporte validation et meilleures performances.

import { StyleSheet, View, Text } from 'react-native'; function MonComposant() { return ( <View style={styles.container}> <Text style={styles.titre}>Mon Application</Text> <Text style={styles.paragraphe}>Ceci est un texte stylisé.</Text> </View> ); } const styles = StyleSheet.create({ container: { flex: 1, backgroundColor: '#f0f0f0', padding: 20 }, titre: { fontSize: 24, fontWeight: 'bold', color: '#333', marginBottom: 10 }, paragraphe: { fontSize: 16, color: '#666', lineHeight: 24 } });

✅ Avantages : validation des noms de propriétés, auto-complétion, meilleures performances (création unique).

🔄 Différences avec le CSS web

1. CamelCase pour les propriétés CSS

En React Native, on écrit backgroundColor, fontSize, marginTop (pas de tirets).

// CSS web : background-color, font-size, margin-top
// React Native : backgroundColor, fontSize, marginTop

2. Pas d'unités (sauf pour certaines propriétés)

Les dimensions sont en points (device-independent pixels). On écrit width: 100 et non width: '100px'.

// En React Native : fontSize: 16, width: 200, padding: 10

3. Certaines propriétés n'existent pas (boxShadow, border-radius...)

⚠️ boxShadow n'existe pas, mais on a shadowColor, shadowOffset, shadowOpacity, shadowRadius (iOS) et elevation (Android).

👪 Héritage des styles : particularité importante

Contrairement au web (où les styles CSS se transmettent souvent aux enfants), en React Native, les styles ne sont pas hérités sauf pour la famille de police (fontFamily) et la couleur (color) sur les composants Text.

// ✅ Le texte hérite de la couleur et de la police <Text style={{ color: 'blue', fontFamily: 'Arial' }}> <Text>Ce texte sera aussi bleu</Text> </Text> // ❌ Le padding ne sera PAS hérité par les enfants <View style={{ padding: 20 }}> <Text>Ce texte n'aura PAS de padding</Text> </View>

📌 Règle : Il faut appliquer les styles directement sur chaque composant qui en a besoin, ou créer des composants réutilisables.

🎭 Styles conditionnels et composition

On peut combiner plusieurs styles avec un tableau, et rendre les styles conditionnels.

function Bouton({ isPrimary, label }) { return ( <View style={[styles.bouton, isPrimary && styles.primary]}> <Text style={[styles.text, isPrimary && styles.textPrimary]}> {label} </Text> </View> ); } const styles = StyleSheet.create({ bouton: { padding: 12, borderRadius: 8, backgroundColor: '#gray' }, primary: { backgroundColor: '#007AFF' }, text: { color: '#333' }, textPrimary: { color: '#fff' } });

💡 Astuce : Utilisez le spread operator ou les tableaux pour combiner les styles. Le dernier style écrasera les précédents.

✏️ Exercice : Créer une carte de profil

Consigne : Créez une carte de profil qui affiche :

  • 🔹 Un nom (texte en gras, taille 20)
  • 🔹 Une profession (texte gris, taille 14)
  • 🔹 Une description (texte normal)
  • 🔹 Carte avec bordure arrondie, ombre (iOS) / élévation (Android), padding
const styles = StyleSheet.create({ card: { backgroundColor: 'white', borderRadius: 12, padding: 16, margin: 16, shadowColor: '#000', shadowOffset: { width: 0, height: 2 }, shadowOpacity: 0.1, shadowRadius: 4, elevation: 3 // Pour Android }, nom: { fontSize: 20, fontWeight: 'bold', marginBottom: 4 }, profession: { fontSize: 14, color: '#666', marginBottom: 8 }, description: { fontSize: 16, color: '#333', lineHeight: 22 } });

📌 Ce qu'il faut retenir

✅ View = conteneur universel (comme <div>).
✅ Text = obligatoire pour afficher du texte.
✅ StyleSheet.create() = organiser ses styles, meilleures performances.
✅ Propriétés en camelCase : backgroundColor, fontSize...
✅ Pas d'unités : width: 100 au lieu de width: '100px'.
