Images, ScrollView & FlatList

Afficher du contenu dynamique dans React Native

Image • ScrollView • FlatList • Performances

Niveau: Débutant Durée: 50 min

🎯 Objectifs de la leçon

  • Afficher des images locales et distantes avec le composant Image
  • Utiliser ScrollView pour du contenu défilant simple
  • Maîtriser FlatList pour des listes performantes avec de grands volumes de données

À la fin, vous saurez choisir le bon composant pour afficher du contenu scrollable !

🖼️ Le composant Image

Le composant Image permet d'afficher des images locales ou distantes.

import { Image } from 'react-native';

// Image distante (URL)
<Image
  source={{ uri: 'https://picsum.photos/200' }}
  style={{ width: 200, height: 200 }}
/>

// Image locale (dans le projet)
import logo from './assets/logo.png';
<Image source={logo} style={{ width: 100, height: 100 }} />

// Image avec paramètres additionnels
<Image
  source={{ uri: 'https://...' }}
  style={{ width: '100%', height: 200 }}
  resizeMode="cover"  // cover, contain, stretch, repeat, center
  blurRadius={5}      // flou
/>

resizeMode : "cover" (remplit, recadre), "contain" (intègre entièrement), "stretch" (étire), "center" (centre sans redimensionner)

🌐 Images distantes : gérer les dimensions

Pour les images distantes, React Native a besoin de connaître les dimensions à l'avance.

// ✅ Bon : dimensions fixes
<Image
  source={{ uri: 'https://api.example.com/photo.jpg' }}
  style={{ width: 300, height: 200 }}
/>

// ❌ Mauvais : dimensions auto (ne fonctionne pas en RN)
<Image source={{ uri: '...' }} style={{ width: 'auto', height: 'auto' }} />

// ✅ Solution : utiliser getSize pour obtenir les dimensions
Image.getSize(myUri, (width, height) => {
  setDimensions({ width, height });
});

Astuce : utilisez getSize() pour connaître les dimensions réelles d'une image distante avant de l'afficher.

📜 ScrollView : Contenu défilant simple

ScrollView permet de faire défiler du contenu qui dépasse l'écran. Il rend tous ses enfants immédiatement.

import { ScrollView, Text, View } from 'react-native';

function MyScrollView() {
  return (
    <ScrollView
      showsVerticalScrollIndicator={true}
      pagingEnabled={false}
      horizontal={false}  // true = défilement horizontal
      contentContainerStyle={{ padding: 20 }}
    >
      <View style={{ height: 200, backgroundColor: 'red' }} />
      <View style={{ height: 300, backgroundColor: 'blue' }} />
      <View style={{ height: 150, backgroundColor: 'green' }} />
      <Text>Contenu supplémentaire</Text>
    </ScrollView>
  );
}

Attention : ScrollView charge TOUS ses enfants même s'ils ne sont pas à l'écran. À éviter pour les longues listes (100+ éléments).

⚡ FlatList : La liste haute performance

FlatList est optimisée pour les longues listes : elle ne rend que les éléments visibles à l'écran (lazy loading).

import { FlatList, Text, View } from 'react-native';

const DATA = [
  { id: '1', title: 'Premier élément' },
  { id: '2', title: 'Deuxième élément' },
  // ... des centaines d'éléments
];

function MyFlatList() {
  return (
    <FlatList
      data={DATA}
      keyExtractor={(item) => item.id}
      renderItem={({ item }) => (
        <View style={{ padding: 20, borderBottomWidth: 1 }}>
          <Text>{item.title}</Text>
        </View>
      )}
      // Options de performance
      initialNumToRender={10}     // Nb d'éléments rendus initialement
      maxToRenderPerBatch={10}    // Nb d'éléments rendus par lot
      windowSize={5}              // Nombre de "fenêtres" d'écran à conserver
      removeClippedSubviews={true} // Optimisation mémoire
    />
  );
}

📌 FlatList : Header, Footer, Empty state

function MyAdvancedList() {
  return (
    <FlatList
      data={data}
      renderItem={({ item }) => <Item title={item.title} />}
      keyExtractor={item => item.id}
      
      // En-tête
      ListHeaderComponent={() => (
        <View style={{ padding: 20, backgroundColor: '#f0f0f0' }}>
          <Text>Liste des produits</Text>
        </View>
      )}
      
      // Pied de page
      ListFooterComponent={() => (
        <View style={{ padding: 10, alignItems: 'center' }}>
          <Text>Fin de la liste</Text>
        </View>
      )}
      
      // Affiché si data est vide
      ListEmptyComponent={() => (
        <View style={{ padding: 50, alignItems: 'center' }}>
          <Text>Aucun élément à afficher</Text>
        </View>
      )}
      
      // Séparateur entre les éléments
      ItemSeparatorComponent={() => (
        <View style={{ height: 1, backgroundColor: '#ccc' }} />
      )}
    />
  );
}

📸 Exemple : Galerie d'images avec FlatList

import { FlatList, Image, View, Text, StyleSheet } from 'react-native';

const IMAGES = [
  { id: '1', uri: 'https://picsum.photos/id/101/200', title: 'Paysage' },
  { id: '2', uri: 'https://picsum.photos/id/104/200', title: 'Nature' },
  // ... 50 autres images
];

function ImageGallery() {
  const renderItem = ({ item }) => (
    <View style={styles.card}>
      <Image source={{ uri: item.uri }} style={styles.image} />
      <Text style={styles.title}>{item.title}</Text>
    </View>
  );

  return (
    <FlatList
      data={IMAGES}
      renderItem={renderItem}
      keyExtractor={item => item.id}
      numColumns={2}  // Affichage en grille
      contentContainerStyle={{ padding: 10 }}
      showsVerticalScrollIndicator={false}
    />
  );
}

const styles = StyleSheet.create({
  card: {
    flex: 1,
    margin: 5,
    backgroundColor: '#fff',
    borderRadius: 10,
    overflow: 'hidden',
    elevation: 3,
  },
  image: { width: '100%', height: 150 },
  title: { padding: 8, textAlign: 'center' },
});

💡 Bonnes pratiques

Images

  • Spécifier width et height
  • Utiliser resizeMode="cover" pour le remplissage
  • Optimiser les images (taille, format WebP)

ScrollView

  • Réservé aux contenus courts (formulaires)
  • Éviter pour les listes dynamiques

FlatList

  • keyExtractor obligatoire (id unique)
  • Utiliser initialNumToRender (10-15)
  • Définir windowSize pour la mémoire
  • Ne pas utiliser d'index comme clé

Performance

  • Mémoriser les fonctions avec useCallback
  • React.memo pour les items complexes
  • removeClippedSubviews={true}

🧪 Exercice : Application de recettes

Créez une application qui affiche une liste de recettes avec les fonctionnalités suivantes :

  1. Afficher une liste de recettes (nom, image, temps de préparation)
  2. Utiliser FlatList pour gérer au moins 20 recettes
  3. Chaque recette doit être un composant séparé
  4. Ajouter un en-tête avec le titre "Nos Recettes"
  5. Afficher un message personnalisé si la liste est vide
// Structure d'une recette
const RECIPES = [
  { id: '1', name: 'Poulet DG', image: '...', time: '45 min' },
  // ...
];

<FlatList
  data={RECIPES}
  keyExtractor={item => item.id}
  ListHeaderComponent={() => <Text style={{fontSize: 24}}>Nos Recettes</Text>}
  ListEmptyComponent={() => <Text>Aucune recette disponible</Text>}
  renderItem={({item}) => <RecipeCard recipe={item} />}
/>

// Composant RecipeCard
function RecipeCard({ recipe }) {
  return (
    <View style={{ flexDirection: 'row', padding: 10 }}>
      <Image source={{ uri: recipe.image }} style={{ width: 80, height: 80 }} />
      <View>
        <Text>{recipe.name}</Text>
        <Text>⏱️ {recipe.time}</Text>
      </View>
    </View>
  );
}

📌 Ce qu'il faut retenir

Image

Spécifiez toujours width/height. Utilisez resizeMode pour contrôler l'affichage.

ScrollView

Pour les petits contenus statiques (formulaires, petit nombre d'éléments).

FlatList

Pour les grandes listes dynamiques. Rendu lazy + performances optimisées.

Règle d'or : ScrollView pour les petits contenus, FlatList pour les grandes listes !
