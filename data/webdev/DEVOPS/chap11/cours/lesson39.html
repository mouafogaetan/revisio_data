📊 1. Monitoring avec Prometheus + Grafana

🔍 Qu'est-ce que Prometheus ?

Prometheus est un système de monitoring open-source qui collecte des métriques (CPU, mémoire, requêtes HTTP, etc.) à partir de vos services via un modèle "pull".

📈 Grafana : visualisation puissante

Grafana se connecte à Prometheus (et d'autres sources) pour créer des dashboards interactifs, des alertes et suivre l'état de santé de l'infrastructure en temps réel.

prometheus.yml → cible les exporters
datasource Grafana → pointe vers Prometheus

🖥️ Installation & premiers pas

⚙️ Étapes clés :

  • Télécharger Prometheus : wget https://.../prometheus.tar.gz puis lancer ./prometheus
  • Accéder à l'interface Prometheus sur http://localhost:9090
  • Installer Grafana : sudo apt install grafana (ou docker) puis démarrer le service
  • Se connecter à Grafana (admin/admin) et ajouter Prometheus comme source de données
  • Importer un dashboard prédéfini (ID 1860 pour Node Exporter) ou créer ses propres graphiques

Flux : Applications → Prometheus (collecte) → Grafana (visualisation)

📜 Centralisation des logs : architecture ELK

ELK stack = Elasticsearch (stockage/recherche) + Logstash (traitement) + Kibana (visualisation). On ajoute souvent Filebeat pour alléger l'envoi.

🔄 Flux typique :
Application → logs → Filebeat → Logstash (filtrage) → Elasticsearch → Kibana

Filebeat
collecte légère
Logstash
enrichissement
Elasticsearch
indexation rapide

💡 Kibana permet de faire des recherches full-text, créer des visualisations et détecter des anomalies dans les logs applicatifs.

⚡ Configuration rapide (docker-compose ELK)

Exemple de fichier docker-compose.yml pour démarrer ELK rapidement :

version: '3'
services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.10.0
    environment: - discovery.type=single-node
    ports: - "9200:9200"
  logstash:
    image: docker.elastic.co/logstash/logstash:8.10.0
    volumes: - ./logstash.conf:/usr/share/logstash/pipeline/logstash.conf
  kibana:
    image: docker.elastic.co/kibana/kibana:8.10.0
    ports: - "5601:5601"

Une fois lancé, Kibana est accessible sur http://localhost:5601, et les logs centralisés sont consultables.

🎯 Objectif final : Voir tous les logs de votre application (backend, base de données) dans une seule interface !

🚀 Présentation du projet DevOps complet

Pour boucler la formation, chaque étudiant devra présenter son projet final incluant :

  • 🌐 Application containerisée (Docker) avec un Dockerfile multi-stage
  • ⚙️ Pipeline CI/CD (GitHub Actions ou GitLab CI) test + build + push d'image
  • ☸️ Déploiement Kubernetes (Minikube ou cloud) avec Deployments, Services, ConfigMaps
  • 📡 Monitoring : Prometheus + Grafana pour observer les métriques
  • 📃 Centralisation logs : ELK stack (ou alternative) pour les logs applicatifs
  • 📄 Infrastructure as Code (Terraform pour le cloud ou démo locale)

🎤 À l'oral : expliquez l'architecture, les choix d'outils, démontrez un dashboard Grafana et une recherche Kibana.

📌 Bonnes pratiques & livrables

📈

Dans Grafana

Créez des graphiques pour : utilisation CPU, mémoire, nombre de requêtes HTTP, latence. Utilisez PromQL : rate(http_requests_total[5m])

🔎

Dans Kibana

Créez des "Discover" pour filtrer les logs par niveau (ERROR, INFO). Sauvegardez des visualisations et tableaux de bord.

🏆 Défi personnel

Mettez en place un mini-projet avec les étapes suivantes :

  1. Lancer une application simple (ex: API Node.js) qui écrit des logs dans un fichier.
  2. Déployer Prometheus et Grafana via Docker, et exporter les métriques de l'API.
  3. Configurer Filebeat pour envoyer les logs de l'application vers Elasticsearch.
  4. Créer un dashboard Grafana avec au moins trois panneaux et une alerte simple.
  5. Écrire un court rapport ou enregistrement vidéo présentant votre architecture.
💡 Astuce : Utilisez Prometheus client libraries dans votre app pour exposer des métriques custom.
