Objectifs de la leçon

📂 Panorama

Les familles d'outils DevOps

Chaque phase du cycle de vie DevOps s'appuie sur des outils spécialisés. Voici les catégories clés :

⚙️ CI/CD

Intégration & déploiement continus : Jenkins, GitLab CI, GitHub Actions, CircleCI.

🐳 Containerisation

Docker, Podman, containerd — pour empaqueter et isoler les applications.

☸️ Orchestration

Kubernetes (K8s), Docker Swarm, OpenShift — gestion de conteneurs à grande échelle.

📊 Monitoring

Prometheus, Grafana, Datadog, ELK Stack — observabilité et alertes.

💡 À retenir : Chaque outil répond à un besoin précis, mais ils se combinent pour former un pipeline complet.

🔗 La chaîne d'outils DevOps classique

📌 Git : versionnement du code et collaboration.
⚡ Jenkins / CI : automatisation des builds & tests.
🐋 Docker : conteneurisation portable.
🚢 Kubernetes : orchestration et scaling.
📈 Prometheus & Grafana : collecte de métriques et dashboards.
🧠

Comment choisir ses outils DevOps ?

Il n'y a pas de solution unique. Le choix dépend de :

  • 📐 Taille de l’équipe : une petite équipe préférera GitHub Actions (simple) plutôt que Jenkins (lourd).
  • 🏗️ Infrastructure cible : cloud (AWS, Azure) ou on-premise.
  • ⚙️ Langages / technologies : compatibilité des outils avec l'environnement existant.
  • 💰 Budget et courbe d’apprentissage : solutions open source (Prometheus) vs SaaS (Datadog).

✨ Règle d’or :

"Commence simple, automatise progressivement, et mesure l'impact" — évite la surcharge d'outils trop complexes au début.

🏁 Atelier mental

Pipeline complet : exemple fictif

1. Développeur push sur Git → 2. GitHub Actions déclenche les tests
3. Build Docker image → 4. Push image vers registre (Docker Hub)
5. Mise à jour sur Kubernetes (via kubectl) → 6. Prometheus collecte métriques
7. Grafana affiche statuts & alertes en cas d’erreur

📦 On reconnaît les briques essentielles : Git (versioning), CI (tests), Docker (conteneur), K8s (orchestrateur), Prometheus/Grafana (monitoring).

✅ Pour les débutants : commencez avec Docker + GitHub Actions + un petit cluster local (Minikube). Les outils les plus populaires ont une large communauté.
