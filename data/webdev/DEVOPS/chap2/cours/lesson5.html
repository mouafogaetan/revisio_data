Naviguer dans le système de fichiers

pwd

Affiche le chemin absolu du répertoire courant.

$ pwd
/home/etudiant

ls -la

Liste les fichiers (y compris cachés) avec détails.

$ ls -la
drwxr-xr-x 2 user user 4096 févr. 12 .config

cd /var/log

Se déplacer vers le dossier des logs.

mkdir / rmdir

Créer / supprimer un dossier vide.

💡 Astuce : cd ~ ramène au dossier personnel, cd - alterne entre deux dossiers.

🔐 Permissions Linux : rwx

Trois droits : Lecture (r), Écriture (w), Exécution (x) pour propriétaire, groupe, autres.

-rwxr-xr--  1  alice  devops  1024  fév 12 10:30  script.sh
│└─┬┘│└─┬┘│
│  │   │   └── autres (r-- : lecture seule)
│  │   └────── groupe (r-x : lecture+exécution)
│  └────────── propriétaire (rwx : tous droits)
└──────────── type (- fichier, d dossier)
chmod – modifier les droits
chmod u+x script.sh (ajoute exécution au user)
chmod 755 fichier → rwxr-xr-x
chown – changer propriétaire/groupe
sudo chown root:admin fichier
⚠️ Seul le propriétaire ou root peut changer les permissions.

Liens : symboliques vs physiques

ln -s cible lien_symbolique → raccourci comme un "alias" de fichier.

✨ Lien symbolique (soft link)

Pointe vers un chemin. Si le fichier original est supprimé, le lien est cassé.

$ ln -s /var/log/syslog ma_log
$ ls -l ma_log
lrwxrwxrwx ... ma_log -> /var/log/syslog

🧱 Lien physique (hard link)

Même inœud, même contenu. Supprimer l’original n’efface pas les données.

$ ln original.txt dur_lien

👉 Les liens symboliques sont très utilisés pour les versions d’applications ou configurations.

🧪 Exemples concrets & bonnes pratiques

📌 Créer une arborescence de projet :

mkdir -p devops/{projet1,projet2,scripts}
touch devops/scripts/deploy.sh
chmod +x devops/scripts/deploy.sh

📌 Sauvegarde rapide avec droits préservés :

cp -rp documents/ backup_docs/ # -p = conserver permissions

📌 Sécurité : supprimer récursivement mais vérifier avant :

ls -la dossier_a_supprimer/ && rm -ri dossier_a_supprimer/
⚠️ rm -rf est dangereux : toujours double-vérifier le chemin !
