📝 1. Structure d’un script Bash

Un script shell commence par un #!/bin/bash (shebang) et peut contenir variables, tests et boucles.

#!/bin/bash
# Mon premier script
nom="Étudiant DevOps"
age=25
echo "Bonjour $nom, tu as $age ans"

if [ $age -ge 18 ]; then
    echo "✅ Majeur"
else
    echo "❌ Mineur"
fi

🧠 Conditions & boucles : if-then-else, for, while.

#!/bin/bash
for fichier in *.txt; do
    echo "📄 Traitement de $fichier"
done

⚙️ 2. Fonctions et arguments

Les fonctions rendent le script clair et réutilisable. Accédez aux arguments avec $1, $2, $#.

#!/bin/bash
saluer() {
    local nom=$1
    echo "Bonjour $nom depuis la fonction"
}

saluer "Cameroun DevOps"
echo "Nombre d'arguments fournis au script : $#"

if [ $# -lt 1 ]; then
    echo "Usage: $0 <votre_nom>"
    exit 1
fi
🔧 Astuce : utilisez local pour les variables internes aux fonctions, évitez les effets de bord.

💾 3. Exemple : Script de sauvegarde automatique

Un script qui archive un dossier avec la date du jour et supprime les anciennes sauvegardes.

#!/bin/bash
SOURCE="/home/user/documents"
DEST="/backups"
DATE=$(date +%Y-%m-%d_%H%M%S)
    tar -czf "$DEST/backup_$DATE.tar.gz" "$SOURCE"
echo "✅ Sauvegarde créée : backup_$DATE.tar.gz"
# Supprimer les backups de plus de 7 jours
find "$DEST" -name "*.tar.gz" -mtime +7 -delete
echo "🧹 Anciennes sauvegardes nettoyées"

📌 À retenir : on peut lancer ce script manuellement ou le planifier avec cron pour sauvegardes quotidiennes.

🚢 4. Déploiement automatisé (exemple simplifié)

Idéal pour mettre à jour une application web: pull du code + redémarrage service.

#!/bin/bash
REPO_DIR="/var/www/monapp"
SERVICE_NAME="monapp.service"

cd $REPO_DIR || exit
git pull origin main
npm install # si nodejs
sudo systemctl restart $SERVICE_NAME
echo "✅ Application redéployée avec succès à $(date)" >> /var/log/deploy.log
📦 Intégration continue simplifiée 🔁 zéro erreur manuelle

⏰ 5. Planification avec Cron

Cron est un utilitaire Linux pour exécuter des scripts à intervalles réguliers. Éditez le crontab : crontab -e

# Exemples de lignes cron
# Sauvegarde chaque jour à 2h du matin
0 2 * * * /home/user/scripts/sauvegarde.sh
# Déploiement toutes les heures (en dev)
0 * * * * /home/user/scripts/deploy.sh >> /var/log/deploy.log 2>&1
# Nettoyage chaque lundi à 5h
0 5 * * 1 /home/user/scripts/nettoyage.sh

🧪 Format cron : minute heure jour mois jour_semaine commande. Super pratique pour la maintenance serveur !

📢 Petit dessin d’horloge conceptuelle :
🕒 → ⚙️ script → ✅ action
Les tâches cron tournent en arrière-plan, idéal pour l'administration DevOps.

🧠 6. Bonnes pratiques Bash / DevOps

  • Toujours utiliser set -e au début : arrête le script si une commande échoue.
  • Vérifier les codes retour : $? pour connaître le succès de la dernière commande.
  • Préférer des chemins absolus ou définir PATH explicitement.
  • Ajouter une aide (--help) pour vos scripts d’automatisation.
  • Pour les déploiements critiques, utiliser des logs et des notifications (email/slack).
🌟 Exemple concret (stage) : Script qui vérifie l’espace disque et envoie un avertissement si > 80% d’utilisation.

✍️ 7. Mettez la main à la pâte !

Exercice : Écrire un script nommé monitor_cpu.sh qui :

  • Affiche l'utilisation CPU actuelle avec top -bn1 ou mpstat.
  • Si l'utilisation dépasse 75%, enregistre un message d'alerte dans /var/log/cpu_alert.log.
  • Utilise une fonction pour l'affichage.
  • Planifier ce script chaque 5 minutes via cron.
