Docker Compose : définition

Docker Compose est un outil qui permet de définir et de lancer plusieurs conteneurs Docker en même temps. Grâce à un fichier docker-compose.yml, vous décrivez :

  • Les services (ex: backend, base de données, cache Redis, serveur web)
  • Les réseaux entre les conteneurs
  • Les volumes pour persister les données
  • Les variables d’environnement, ports, dépendances

✨ Avantage clé : Plus besoin de lancer manuellement docker run ... pour chaque conteneur. Un simple docker compose up démarre toute l'infrastructure !

📄 Fichier docker-compose.yml

Le fichier utilise le format YAML. Voici une structure typique :

version: '3.8'
services:
  webapp:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - db
  db:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: example
    volumes:
      - pgdata:/var/lib/postgresql/data
volumes:
  pgdata:

✔️ services : chaque conteneur s'appelle un service. volumes : pour la persistance.

⚙️

Commandes essentielles

docker compose up

Lance tous les services (mode attaché). Ajoutez -d pour détaché (arrière-plan).

docker compose down

Arrête et supprime les conteneurs, réseaux (par défaut).

docker compose logs

Affiche les logs de tous les services. Ajoutez -f pour suivre en direct.

docker compose exec <service> ...

Exécute une commande dans un conteneur actif (ex: bash).

💡 Petit plus : docker compose up --build reconstruit les images avant de démarrer.

📦 Exemple réel : Webapp + PostgreSQL + Redis

version: '3.8'
services:
  app:
    build: ./app
    ports:
      - "8080:80"
    environment:
      - DB_HOST=db
      - REDIS_HOST=cache
    depends_on:
      - db
      - cache
  db:
    image: postgres:14-alpine
    environment:
      POSTGRES_USER: devops
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: myapp
    volumes:
      - postgres_data:/var/lib/postgresql/data
  cache:
    image: redis:alpine
    ports:
      - "6379:6379"
volumes:
  postgres_data:

✔️ Les services communiquent via leurs noms (app ↔ db).
✔️ Les données de la base sont persistées avec un volume nommé.

Réseaux & Volumes

Par défaut, Compose crée un réseau pour vos services. Vous pouvez personnaliser :

🔌 Réseaux personnalisés

Isoler certaines stacks ou définir une passerelle spécifique.

networks: frontend: backend:

💾 Volumes

Volumes nommés (persistants) ou bind mounts (dossier local).

volumes: - ./data:/var/lib/mysql
🔁 Conteneur ⇄ volume nommé ⇄ Données sauvegardées

✅ Bonnes pratiques avec Compose

  • Utiliser .env pour les variables sensibles (ne pas versionner les secrets).
  • Spécifier des healthcheck pour s'assurer qu'un service est prêt.
  • Préférer les images officielles et versionnées (ex: postgres:15-alpine).
  • Utiliser les profiles pour lancer des services optionnels (tests, monitoring).
  • Garder le docker-compose.yml simple et documenté.

🏁 En équipe, Docker Compose garantit que tout le monde a le même environnement de développement !

🧪 Mini atelier

Créez un fichier docker-compose.yml qui lance :

  • Un service web avec l'image nginx:alpine, port exposé 8080:80
  • Un service api avec l'image httpd:alpine, port 8081:80
  • Un réseau personnalisé pour qu'ils communiquent entre eux.
📌 Voir un exemple de solution 
version: '3'
services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    networks:
      - appnet
  api:
    image: httpd:alpine
    ports:
      - "8081:80"
    networks:
      - appnet
networks:
  appnet:

▶️ Testez avec docker compose up -d puis visitez localhost:8080 et 8081
