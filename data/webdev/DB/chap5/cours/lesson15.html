GROUP BY & HAVING

Grouper et filtrer des groupes pour analyser efficacement vos données.

GROUP BY : Division par groupes

La clause GROUP BY permet de regrouper des lignes ayant les mêmes valeurs dans une ou plusieurs colonnes, puis d'appliquer des fonctions d'agrégation (COUNT, SUM, AVG, MIN, MAX) sur chaque groupe.

📖 Syntaxe de base :

SELECT colonne, fonction_agrégation(*)
FROM table
GROUP BY colonne;

✨ Exemple concret :

SELECT statut, COUNT(*) AS nombre_etudiants
FROM etudiants
GROUP BY statut;

👉 Compte combien d'étudiants par statut (ex: "actif", "suspendu")

✅ Chaque groupe produit une ligne dans le résultat.
🔍

HAVING : Filtrer après GROUP BY

WHERE filtre les lignes individuelles avant regroupement. HAVING filtre les groupes après le GROUP BY.

🧠 Règle d’or :

  • WHERE → s’applique sur chaque ligne brute (pas sur agrégats).
  • HAVING → s’applique sur les résultats des fonctions d’agrégation.

-- Exemple : ne montrer que les catégories ayant plus de 5 produits
SELECT categorie, COUNT(*) AS total_produits
FROM produits
GROUP BY categorie
HAVING COUNT(*) > 5;

💡 On ne peut pas utiliser HAVING sans GROUP BY (dans la majorité des SGBD).
⚖️

WHERE vs HAVING en pratique

CritèreWHEREHAVING
Quand s'applique-t-il ?Avant GROUP BYAprès GROUP BY
Peut utiliser agrégats ?❌ Non (ex: SUM, AVG)✅ Oui
Filtre sur colonnes brutes✅ Oui❌ Uniquement colonnes groupées ou agrégats

Exemple combiné :

SELECT ville, AVG(age) as age_moyen
FROM clients
WHERE age >= 18 -- ligne par ligne
GROUP BY ville
HAVING AVG(age) > 25; -- sur groupe

👉 D'abord on filtre les clients majeurs, puis on regroupe par ville et on garde villes où âge moyen > 25

🏆

Cas pratique & astuces

📊 Scénario : Ventes par département

SELECT departement, SUM(montant) as CA_total, COUNT(*) as nb_ventes
FROM ventes
WHERE montant > 0
GROUP BY departement
HAVING SUM(montant) > 10000
ORDER BY CA_total DESC;
  • ✅ WHERE montant > 0 : élimine les ventes à 0 ou négatives avant le groupement.
  • ✅ GROUP BY departement : regroupe les lignes par département.
  • ✅ HAVING SUM(montant) > 10000 : ne garde que les départements avec CA supérieur à 10k.

💡 À retenir :

HAVING est souvent utilisé avec COUNT, SUM, AVG. On peut y mettre plusieurs conditions avec AND/OR.

🧪 Mini exercice : Écrire une requête qui affiche les années (colonne année) où le nombre de commandes dépasse 100, depuis la table "commandes".
👉 Indice / solution SELECT année, COUNT(*) FROM commandes GROUP BY année HAVING COUNT(*) > 100;
