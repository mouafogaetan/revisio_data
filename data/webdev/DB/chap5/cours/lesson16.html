🔗 Les jointures SQL

INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN
et jointures multi-tables

Cliquez sur suivant pour commencer

Objectifs de la leçon

1. Comprendre INNER JOIN pour les intersections entre tables.
2. Maîtriser LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN et leurs cas d'usage.
3. Savoir joindre plusieurs tables (2, 3 tables ou plus).
4. Appliquer ces jointures dans des requêtes réelles pour analyser des données.
💡 Pourquoi les jointures sont-elles essentielles ? Les bases de données relationnelles organisent les données en tables séparées. Les jointures permettent de rassembler ces informations et d'obtenir une vue complète.

🔗 clé étrangère (client_id) référence id_client

Pour connaître le nom du client pour chaque commande, on joint les deux tables grâce à la relation.

📌 SQL va rapprocher les lignes où la clé étrangère correspond à la clé primaire.

📌 INNER JOIN

Renvoie uniquement les lignes qui ont des correspondances dans les deux tables.

SELECT clients.nom, commandes.produit
FROM clients
INNER JOIN commandes ON clients.id_client = commandes.client_id;
✅ Résultat : Alice (Livre), Bob (Stylo), Alice (Cahier).
Tous les clients qui ont au moins une commande apparaissent.

⬅️ LEFT JOIN / RIGHT JOIN

LEFT JOIN

Toutes les lignes de la table de gauche + correspondances de droite (sinon NULL).

SELECT clients.nom, commandes.produit
FROM clients
LEFT JOIN commandes ON clients.id_client = commandes.client_id;

👉 Si un client n'a pas commandé, produit = NULL mais client apparaît quand même.

RIGHT JOIN

Symétrique : toutes les lignes de la table de droite.

SELECT clients.nom, commandes.produit
FROM clients
RIGHT JOIN commandes ON clients.id_client = commandes.client_id;

(Moins courant; on utilise surtout LEFT JOIN en inversant l'ordre)

🔎 Cas pratique : lister tous les clients, même ceux sans commande → LEFT JOIN.

🔄 FULL OUTER JOIN

Combine LEFT JOIN et RIGHT JOIN : toutes les lignes des deux tables, avec correspondances si existent, sinon NULL de l'autre côté.

SELECT * FROM tableA
FULL OUTER JOIN tableB ON tableA.key = tableB.key;
⚠️ MySQL ne supporte pas directement FULL OUTER JOIN, mais on peut l'émuler avec UNION de LEFT JOIN et RIGHT JOIN. En PostgreSQL, c'est natif.

📚 Joindre plusieurs tables (3 tables ou plus)

On enchaîne les JOIN. Exemple avec trois tables : clients, commandes, produits.

SELECT clients.nom, commandes.date_cmd, produits.libelle
FROM clients
JOIN commandes ON clients.id_client = commandes.client_id
JOIN produits ON commandes.produit_id = produits.id_produit;

Chaque JOIN crée une table virtuelle étendue. L'ordre des JOIN influence parfois les performances, mais le résultat final est le même s'ils sont bien conditionnés.

✏️ Exemple pas à pas

Scénario : école et étudiants

étudiants
id_etudiant, nom, classe_id
classes
id_classe, nom_classe
notes
id_note, etudiant_id, valeur
SELECT e.nom, c.nom_classe, n.valeur
FROM étudiants e
LEFT JOIN classes c ON e.classe_id = c.id_classe
INNER JOIN notes n ON e.id_etudiant = n.etudiant_id
WHERE n.valeur > 10;

🔍 Requête : noms des étudiants avec leurs classes (peut être NULL si pas de classe) et notes >10 uniquement pour ceux qui ont des notes.

⚠️ Erreurs fréquentes & astuces

  • Oublier la condition de jointure (ON) → produit un produit cartésien (toutes les combinaisons).
  • Confondre WHERE et ON : les filtres sur la table secondaire dans un LEFT JOIN doivent être dans la clause ON, sinon LEFT JOIN se comporte comme INNER JOIN.
  • Alias de table : utilisez des alias courts (ex : c pour clients, cmd pour commandes).
  • Jointures multi-tables : toujours vérifier les cardinalités.
💡 Astuce : Dessinez vos tables et les flèches avant d'écrire la requête ! Cela clarifie les relations.
