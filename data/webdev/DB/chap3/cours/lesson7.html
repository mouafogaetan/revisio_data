🔢 1. Les types numériques

Chaque type a sa précision, sa taille et son usage.

TypeDescriptionExemple SQL
INTNombre entier (de -2^31 à 2^31-1)âge INT
BIGINTTrès grands entierspopulation BIGINT
DECIMAL(p,s)Nombre exact à virgule fixe (prix, notes)prix DECIMAL(10,2)
FLOATApproximation virgule flottante (scientifique)pi FLOAT
SERIALAuto-incrémentation (MySQL AUTO_INCREMENT)id SERIAL PRIMARY KEY

💡 Bon à savoir :

Pour les montants d'argent, utilisez DECIMAL pour éviter les erreurs d'arrondi. SERIAL est parfait pour les clés primaires auto-incrémentées.

🔤 2. Types chaînes de caractères

Stocker du texte, des noms, descriptions…

CHAR(n)

Longueur fixe (n caractères). Rempli d'espaces. Idéal pour codes fixes (ex: code pays, sexe).

code CHAR(2) -- 'CM'
VARCHAR(n)

Longueur variable (jusqu'à n). Économise de l'espace. Parfait pour noms, emails.

nom VARCHAR(100)
TEXT

Très longs textes (articles, commentaires). Pas de limite de taille définie mais peut impacter performance.

description TEXT

⚠️ Règle pratique : utilisez VARCHAR pour la plupart des textes courts (moins de 255 caractères) et TEXT pour les longs contenus. CHAR est rare sauf pour données rigides.

📆 3. Types date et heure

Gérer le temps dans vos applications : naissance, création, mise à jour...

TypeFormatExemple concret
DATE'AAAA-MM-JJ'2025-12-25 (date de naissance)
TIME'HH:MM:SS'14:30:00 (heure de cours)
DATETIME'AAAA-MM-JJ HH:MM:SS'2025-12-25 14:30:00 (date complète)
TIMESTAMPidem DATETIME + fuseau horairegénéralement utilisé pour enregistrement automatique (created_at)
🕒 Fonctions utiles : NOW(), CURDATE(), DATEDIFF()
Exemple SQL : INSERT INTO eleves (nom, date_inscription) VALUES ('Awa', NOW());

📌 Guide de choix : quel type pour quelle donnée ?

Un bon schéma de base de données commence par des types adaptés.

✅ Recommandations
  • Âge, quantité → INT ou SMALLINT
  • Prix, salaire → DECIMAL(10,2)
  • Nom, prénom, ville → VARCHAR(50-100)
  • Description longue → TEXT
  • Date de création → TIMESTAMP ou DATETIME
  • Code postal fixe 5 chiffres → CHAR(5)
❌ Erreurs fréquentes
  • Stocker des prix dans FLOAT → arrondis bizarres
  • Utiliser VARCHAR pour des valeurs de oui/non (mieux BOOLEAN)
  • Mettre des dates en VARCHAR → perte des fonctions SQL
  • Texte très long dans VARCHAR(255) → troncature
📖 Petit exemple de création de table avec types appropriés :
CREATE TABLE etudiants (
    id SERIAL PRIMARY KEY,
    nom VARCHAR(60) NOT NULL,
    email VARCHAR(150) UNIQUE,
    age INT CHECK (age >= 16),
    moyenne DECIMAL(4,2),
    date_naissance DATE,
    inscrit_le TIMESTAMP DEFAULT NOW()
);

🧪 Petite vérification

Cliquez sur les questions pour révéler la réponse (apprenez en jouant).

🔹 Quel type SQL utiliser pour enregistrer le prix d'un produit en CFA avec 2 décimales ?▼
✔ Réponse : DECIMAL(10,2) (stockage exact pour l'argent).
🔹 Doit-on stocker une biographie d’un auteur (environ 2000 caractères) dans CHAR, VARCHAR ou TEXT ?▼
✔ Réponse : TEXT ou VARCHAR(3000). CHAR est inapproprié (taille fixe).
🔹 Pour stocker uniquement la date d’embauche (jour/mois/année) sans heure, quel type ?▼
✔ Réponse : DATE, économique et adapté.
🔹 Quelle type pour une colonne "id" qui s'incrémente automatiquement ?▼
✔ Réponse : SERIAL (PostgreSQL) ou INT AUTO_INCREMENT (MySQL).
