Le cycle de vie dans NestJS

Contrôler l'initialisation et l'arrêt de votre application

OnModuleInit → OnApplicationBootstrap → ... → OnApplicationShutdown

Objectifs pédagogiques

Hooks de cycle de vie – OnInit, OnDestroy
OnModuleInit – Initialisation des modules
OnApplicationBootstrap – Après le démarrage
Shutdown – Arrêt gracieux

Maîtriser le cycle de vie permet d'exécuter du code au bon moment (connexion DB, chargement de config, nettoyage).

🔄 Séquence du cycle de vie NestJS

Démarrage → OnModuleInit → OnApplicationBootstrap → Application prête
Signal SIGTERM → OnApplicationShutdown → Arrêt

NestJS expose plusieurs interfaces que vos classes peuvent implémenter pour exécuter du code à des moments précis du cycle de vie.

import { Injectable, OnModuleInit, OnApplicationBootstrap, OnApplicationShutdown } from '@nestjs/common';

@Injectable()
export class DatabaseService implements OnModuleInit, OnApplicationBootstrap, OnApplicationShutdown {
  async onModuleInit() { console.log('Module initialisé'); }
  async onApplicationBootstrap() { console.log('Application démarrée'); }
  async onApplicationShutdown(signal?: string) { console.log('Arrêt:', signal); }
}

🚀 Les hooks de démarrage

OnModuleInit

Exécuté une fois après l'initialisation du module (mais avant la résolution complète des dépendances).

Cas d'usage : connexion à une base de données, chargement de configuration externe.

OnApplicationBootstrap

Exécuté après le démarrage complet de l'application, tous les modules/services sont prêts.

Cas d'usage : insertion de données de test, démarrage de workers périodiques, vérification de santé.

// Exemple concret
@Injectable()
export class SeederService implements OnApplicationBootstrap {
  async onApplicationBootstrap() {
    const count = await this.userRepository.count();
    if (count === 0) {
      await this.userRepository.save([{ name: 'Admin', email: 'admin@afrimarket.cm' }]);
      console.log('✅ Données de test insérées');
    }
  }
}

🛑 Arrêt gracieux de l'application

NestJS permet de nettoyer les ressources avant l'arrêt (fermeture des connexions DB, flush des logs, etc.).

import { Injectable, OnApplicationShutdown } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

@Injectable()
export class PrismaService extends PrismaClient implements OnApplicationShutdown {
  async onApplicationShutdown(signal?: string) {
    console.log(`Arrêt demandé (signal: ${signal}) - fermeture de Prisma...`);
    await this.$disconnect();
    console.log('✅ Connexion DB fermée');
  }
}
Important : Pour que OnApplicationShutdown soit appelé, activez app.enableShutdownHooks() dans main.ts. 
// main.ts
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.enableShutdownHooks();  // Active la détection SIGTERM/SIGINT
  await app.listen(3000);
}

📊 Ordre d'exécution des hooks

Application NestJS démarrée
1. OnModuleInit (Modules racine → enfants)
2. OnApplicationBootstrap (tous les modules)
--- Application en cours d'exécution ---
3. Réception d'un signal d'arrêt (SIGTERM)
4. OnApplicationShutdown (exécution parallèle)
Application arrêtée

À retenir :

OnModuleInit → OnApplicationBootstrap → (vie) → OnApplicationShutdown.

💡 Cas pratique : Module de monitoring

@Injectable()
export class MonitoringService implements OnModuleInit, OnApplicationBootstrap, OnApplicationShutdown {
  private cronJob: NodeJS.Timeout;

  onModuleInit() {
    console.log('🔧 MonitoringService - Module initialisé');
  }

  onApplicationBootstrap() {
    console.log('📊 MonitoringService - Application prête, démarrage du cron');
    this.cronJob = setInterval(() => this.checkHealth(), 60000);
  }

  async onApplicationShutdown(signal: string) {
    console.log(`🛑 Arrêt monitoring (signal: ${signal})`);
    clearInterval(this.cronJob);
    await this.flushMetricsToDatabase();
    console.log('✅ Metrics sauvegardées');
  }

  private async flushMetricsToDatabase() { /* ... */ }
  private checkHealth() { console.log('Health check OK'); }
}

✨ Bonnes pratiques du cycle de vie

  • ✔️ Ne pas bloquer les hooks avec des opérations synchrones longues → utilisez async/await.
  • ✔️ OnModuleInit pour les connexions externes (DB, Redis, API).
  • ✔️ OnApplicationBootstrap pour les opérations qui nécessitent que TOUT soit prêt.
  • ✔️ enableShutdownHooks indispensable pour les environnements conteneurisés (Docker/K8s).
  • ✔️ Éviter les appels réseau dans OnApplicationShutdown (peu de temps).

Impact en production :

Des hooks bien utilisés = démarrage fiable + arrêt sans perte de données.

🎯 Synthèse & mini-projet

Interfaces à retenir :

  • OnModuleInit → après l'initialisation du module
  • OnApplicationBootstrap → après le démarrage complet
  • OnModuleDestroy → avant la destruction du module
  • OnApplicationShutdown → lors de l'arrêt de l'application

Mini-projet :

Créez un service CacheService qui charge les paramètres depuis Redis au démarrage (OnModuleInit), effectue une vérification périodique (OnApplicationBootstrap avec setInterval), et vide le cache à l'arrêt (OnApplicationShutdown).

Maîtriser le cycle de vie, c'est garantir la fiabilité de votre API NestJS.
