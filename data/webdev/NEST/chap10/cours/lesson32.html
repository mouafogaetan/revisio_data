📦 Installation de @nestjs/swagger

Pour générer automatiquement la documentation OpenAPI, installez les dépendances suivantes :

npm install @nestjs/swagger swagger-ui-express

Ensuite, configurez Swagger dans le fichier main.ts :

import { NestFactory } from '@nestjs/core';
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  // Configuration Swagger
  const config = new DocumentBuilder()
    .setTitle('API E-commerce')
    .setDescription('Documentation de l\'API de notre plateforme e-commerce')
    .setVersion('1.0')
    .addTag('products', 'Gestion des produits')
    .addTag('orders', 'Gestion des commandes')
    .addBearerAuth() // Pour l'authentification JWT
    .build();

  const document = SwaggerModule.createDocument(app, config);
  SwaggerModule.setup('api/docs', app, document); // Interface dispo sur /api/docs

  await app.listen(3000);
}
bootstrap();

📌 L'interface sera accessible à l'adresse : http://localhost:3000/api/docs

🏷️ Documenter les contrôleurs et méthodes

Utilisez @ApiTags() pour regrouper les endpoints par catégorie et @ApiOperation() pour décrire chaque opération.

import { Controller, Get, Post, Body, Param } from '@nestjs/common';
import { ApiTags, ApiOperation, ApiResponse } from '@nestjs/swagger';
import { CreateProductDto } from './dto/create-product.dto';

@ApiTags('products')  // Groupe "products" dans Swagger
@Controller('products')
export class ProductsController {

  @Get()
  @ApiOperation({ summary: 'Récupérer tous les produits' })
  @ApiResponse({ status: 200, description: 'Liste des produits retournée avec succès' })
  findAll() {
    return this.productService.findAll();
  }

  @Post()
  @ApiOperation({ summary: 'Créer un nouveau produit' })
  @ApiResponse({ status: 201, description: 'Produit créé avec succès' })
  @ApiResponse({ status: 400, description: 'Données invalides' })
  create(@Body() createProductDto: CreateProductDto) {
    return this.productService.create(createProductDto);
  }

  @Get(':id')
  @ApiOperation({ summary: 'Récupérer un produit par son ID' })
  @ApiResponse({ status: 200, description: 'Produit trouvé' })
  @ApiResponse({ status: 404, description: 'Produit non trouvé' })
  findOne(@Param('id') id: string) {
    return this.productService.findOne(id);
  }
}

🔍 Documenter les paramètres

Pour une documentation précise, utilisez :

📌 @ApiParam() - Paramètres de route

@Get(':id')
@ApiParam({ name: 'id', type: Number, description: 'Identifiant unique du produit' })
findOne(@Param('id') id: number) {}

📌 @ApiQuery() - Paramètres de requête (filtres, pagination)

@Get()
@ApiQuery({ name: 'page', required: false, type: Number, description: 'Numéro de page' })
@ApiQuery({ name: 'limit', required: false, type: Number, description: 'Nombre d\'éléments par page' })
findAll(@Query('page') page: number, @Query('limit') limit: number) {}

📌 @ApiBody() - Corps de la requête (pour décrire un DTO personnalisé)

@Post()
@ApiBody({ type: CreateProductDto, description: 'Données du produit à créer' })
create(@Body() createProductDto: CreateProductDto) {}

🔐 @ApiResponse et @ApiBearerAuth

Documentez les codes de retour et sécurisez vos endpoints avec l'authentification JWT.

📌 @ApiResponse() - Décrire les codes HTTP

@Post()
@ApiResponse({ status: 201, description: 'Ressource créée' })
@ApiResponse({ status: 400, description: 'Validation échouée' })
@ApiResponse({ status: 401, description: 'Non authentifié' })
@ApiResponse({ status: 403, description: 'Accès refusé' })
create() {}

📌 @ApiBearerAuth() - Protection JWT

Active l'authentification par token Bearer dans Swagger :

// Dans main.ts, ajouter .addBearerAuth()
const config = new DocumentBuilder()
  .addBearerAuth()
  .build();

// Dans le contrôleur
@Post('admin')
@ApiBearerAuth()  // Nécessite un token JWT
@UseGuards(AuthGuard('jwt'))
adminOnly() {}

📄 Documenter les DTO avec @ApiProperty

Les DTO (Data Transfer Objects) définissent la structure des données échangées. Utilisez @ApiProperty() pour les décrire dans Swagger.

import { ApiProperty, ApiPropertyOptional } from '@nestjs/swagger';

export class CreateProductDto {
  @ApiProperty({ example: 'Smartphone XYZ', description: 'Nom du produit' })
  name: string;

  @ApiProperty({ example: 499.99, description: 'Prix en euros' })
  price: number;

  @ApiPropertyOptional({ example: 'Smartphone dernière génération', description: 'Description optionnelle' })
  description?: string;

  @ApiProperty({ enum: ['electronics', 'clothing', 'food'], example: 'electronics' })
  category: string;
}

// Pour les réponses
export class ProductResponseDto {
  @ApiProperty({ example: 1 })
  id: number;

  @ApiProperty({ example: '2024-01-15T10:00:00Z' })
  createdAt: Date;
}
✨ Astuce : Utilisez @ApiPropertyOptional() pour les champs optionnels, et l'option enum pour les valeurs autorisées.

⚡ Bonnes pratiques & astuces avancées

✔️ À faire

  • Grouper les endpoints avec @ApiTags()
  • Décrire les codes d'erreur (@ApiResponse)
  • Utiliser des exemples concrets (exemple dans @ApiProperty)
  • Mettre à jour la documentation en même temps que le code

❌ À éviter

  • Documenter uniquement les succès (oublier les erreurs)
  • Laisser des descriptions vides ou trop vagues
  • Exposer des endpoints sensibles sans @ApiBearerAuth()

🧠 À retenir :

Une bonne documentation OpenAPI permet à vos clients (frontend, mobile, autres services) de comprendre et d'utiliser votre API sans avoir à lire le code source.
