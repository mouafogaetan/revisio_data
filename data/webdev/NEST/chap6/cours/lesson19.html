🏛️ Hiérarchie des exceptions NestJS

Toutes les exceptions héritent de la classe de base HttpException.

Error (JS natif)
  └── HttpException (NestJS)
       ├── BadRequestException      (400)
       ├── UnauthorizedException    (401)
       ├── ForbiddenException       (403)
       ├── NotFoundException        (404)
       ├── ConflictException        (409)
       ├── UnprocessableEntityException (422)
       ├── InternalServerErrorException (500)
       └── ... (plus de 20 exceptions)
💡 Astuce : Utilisez toujours les exceptions spécifiques plutôt que HttpException brut pour un code plus lisible et un statut HTTP explicite.

🎯 Lancer des exceptions standards

Dans un service ou un contrôleur, on lance une exception pour interrompre le flux normal.


import { Injectable, NotFoundException, BadRequestException } from '@nestjs/common';

@Injectable()
export class UserService {
  async findOne(id: number) {
    const user = await this.userRepository.findOne({ where: { id } });
    
    if (!user) {
      // Lève une exception 404 avec un message personnalisé
      throw new NotFoundException(`Utilisateur avec l'id ${id} non trouvé`);
    }
    
    return user;
  }
  
  async create(createUserDto: CreateUserDto) {
    const existing = await this.userRepository.findOne({ 
      where: { email: createUserDto.email } 
    });
    
    if (existing) {
      // Lève une exception 409 (Conflit)
      throw new ConflictException('Cet email est déjà utilisé');
    }
    
    return this.userRepository.save(createUserDto);
  }
}
⚠️ Important : Quand vous lancez une exception, le code après le `throw` ne s'exécute pas. NestJS intercepte automatiquement l'exception et envoie la réponse HTTP appropriée.

⚙️ Personnaliser les exceptions

On peut passer un objet personnalisé comme second paramètre pour ajouter une cause ou une description.


// Exemple basique avec message personnalisé
throw new BadRequestException('Le mot de passe doit contenir au moins 8 caractères');

// Exemple avec cause et description
throw new BadRequestException({
  message: 'Validation échouée',
  errors: [
    { field: 'email', message: 'Format email invalide' },
    { field: 'password', message: 'Mot de passe trop court' }
  ]
});

// Avec cause d'erreur (stack trace)
throw new InternalServerErrorException('Erreur base de données', { 
  cause: new Error('Connection timeout') 
});

// Définir une réponse personnalisée complète
throw new HttpException({
  status: 429,
  error: 'Trop de requêtes',
  retryAfter: 60
}, 429);

🔧 Filtre d'exception global personnalisé

Un filtre permet de capturer toutes les exceptions et de formater la réponse comme on le souhaite.


// http-exception.filter.ts
import { ExceptionFilter, Catch, ArgumentsHost, HttpException } from '@nestjs/common';
import { Request, Response } from 'express';

@Catch(HttpException)
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const request = ctx.getRequest<Request>();
    const status = exception.getStatus();
    const exceptionResponse = exception.getResponse();

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      path: request.url,
      method: request.method,
      message: typeof exceptionResponse === 'string' 
        ? exceptionResponse 
        : (exceptionResponse as any).message,
      error: (exceptionResponse as any).error || 'Error',
    });
  }
}

📋 Enregistrer le filtre globalement

Plusieurs façons d'enregistrer un filtre d'exception.

1️⃣ Au niveau de l'application (main.ts)

app.useGlobalFilters(new HttpExceptionFilter());

2️⃣ Avec injection de dépendances (main.ts)

app.useGlobalFilters(app.get(HttpExceptionFilter));

Nécessite que le filtre soit un provider (avec @Injectable())

3️⃣ Par module (filtre au niveau contrôleur)

@Controller('users')
@UseFilters(new HttpExceptionFilter())
export class UsersController {}

4️⃣ Global via module

@Module({
  providers: [{ provide: APP_FILTER, useClass: HttpExceptionFilter }]
})

Cette méthode permet l'injection de dépendances dans le filtre

🛡️ Capturer TOUTES les exceptions

Pour capturer aussi les erreurs JavaScript natives (ex: TypeError), on omet le décorateur `@Catch()` ou on utilise `@Catch()` vide.


// catch-all-exception.filter.ts
import { ExceptionFilter, Catch, ArgumentsHost } from '@nestjs/common';

@Catch()  // Pas de paramètre = capture TOUTES les exceptions
export class AllExceptionsFilter implements ExceptionFilter {
  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const request = ctx.getRequest();

    let status = 500;
    let message = 'Erreur interne du serveur';

    if (exception instanceof HttpException) {
      status = exception.getStatus();
      message = exception.message;
    } else if (exception instanceof Error) {
      message = exception.message;
      // En prod, on peut logger mais ne pas exposer les détails
      console.error('Erreur non gérée:', exception.stack);
    }

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      path: request.url,
      message,
    });
  }
}
⚠️ Attention : Ne révélez jamais les stacks d'erreur en production. Utilisez `process.env.NODE_ENV !== 'production'` pour les inclure conditionnellement.

📝 Exercices pratiques

🎯 Exercice 1 - Lancer une exception (niveau easy)

Dans un service de gestion de produits, écrivez la méthode delete(id: number) qui lance une NotFoundException si le produit n'existe pas.

Voir la solution 
import { Injectable, NotFoundException } from '@nestjs/common';

@Injectable()
export class ProductService {
  async delete(id: number) {
    const product = await this.productRepository.findOne({ where: { id } });
    
    if (!product) {
      throw new NotFoundException(`Produit avec l'ID ${id} non trouvé`);
    }
    
    return this.productRepository.delete(id);
  }
}

🎯 Exercice 2 - Filtre personnalisé (niveau medium)

Créez un filtre d'exception ValidationExceptionFilter qui transforme les erreurs de validation en format : { field, message }[].

Voir la solution 
@Catch(BadRequestException)
export class ValidationExceptionFilter implements ExceptionFilter {
  catch(exception: BadRequestException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const status = exception.getStatus();
    const exceptionResponse = exception.getResponse() as any;

    const errors = exceptionResponse.message || [];

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      errors: Array.isArray(errors) 
        ? errors.map(msg => ({ message: msg }))
        : [{ message: errors }]
    });
  }
}

🎯 Exercice 3 - Cas concret (niveau hard)

Implémentez un filtre global qui formate TOUTES les erreurs selon le format suivant :

{
  "success": false,
  "error": "NOT_FOUND",
  "message": "Ressource non trouvée",
  "timestamp": 1678901234567
}
Voir la solution 
@Catch()
export class GlobalResponseFilter implements ExceptionFilter {
  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    
    let status = 500;
    let errorCode = 'INTERNAL_ERROR';
    let message = 'Erreur interne du serveur';

    if (exception instanceof HttpException) {
      status = exception.getStatus();
      message = exception.message;
      errorCode = this.getErrorCode(status);
    } else if (exception instanceof Error) {
      message = exception.message;
    }

    response.status(status).json({
      success: false,
      error: errorCode,
      message: message,
      timestamp: Date.now()
    });
  }

  private getErrorCode(status: number): string {
    const codes = {
      400: 'BAD_REQUEST',
      401: 'UNAUTHORIZED',
      403: 'FORBIDDEN',
      404: 'NOT_FOUND',
      409: 'CONFLICT',
      500: 'INTERNAL_ERROR'
    };
    return codes[status] || 'UNKNOWN_ERROR';
  }
}
