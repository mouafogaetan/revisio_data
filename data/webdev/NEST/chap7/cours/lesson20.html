Gestion de configuration avec @nestjs/config

NestJS – Lesson 20 | Chapitre 7 : Configuration de l'application

@nestjs/config ConfigModule ConfigService .env

🎯 Objectifs de la leçon

  • ✓ Installer le package @nestjs/config.
  • ✓ Configurer ConfigModule.forRoot() pour charger les variables d'environnement.
  • ✓ Injecter et utiliser ConfigService dans n'importe quel composant.
  • ✓ Comprendre l'utilité des fichiers .env pour la sécurité et la portabilité.
Ne jamais stocker de secrets (clés API, mots de passe) en dur dans le code ! Les variables d'environnement sont la solution.

📦 Installation et première configuration

Dans un projet NestJS, exécutez :

npm install @nestjs/config

Ensuite, importez ConfigModule dans votre module principal (généralement AppModule) :

// app.module.ts
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,   // Rendre ConfigService disponible partout
      envFilePath: '.env',
    }),
  ],
})
export class AppModule {}
isGlobal: true évite de réimporter ConfigModule dans chaque module.

📄 Fichier .env et ConfigService

Créez un fichier .env à la racine du projet :

PORT=3000
DATABASE_URL=postgresql://user:pass@localhost:5432/mydb
JWT_SECRET=superSecretKey123

Injectez ConfigService dans n'importe quel service ou contrôleur :

// app.service.ts
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class AppService {
  constructor(private configService: ConfigService) {}

  getPort(): number {
    return this.configService.get<number>('PORT', 3000);
  }
}
✅ Bonnes pratiques : Utilisez .gitignore pour exclure le fichier .env. Fournissez un fichier .env.example pour l'équipe.

🛡️ Valider les variables d'environnement

Avec le package joi, vous pouvez valider que toutes les variables nécessaires sont présentes et du bon type.

npm install joi
// app.module.ts
import * as Joi from 'joi';

ConfigModule.forRoot({
  validationSchema: Joi.object({
    PORT: Joi.number().default(3000),
    DATABASE_URL: Joi.string().required(),
    JWT_SECRET: Joi.string().min(8).required(),
  }),
})
⚠️ Si une variable requise est manquante, l'application refusera de démarrer → sécurité renforcée.

⚙️ Configuration asynchrone avec useFactory

Parfois, vous devez configurer un module (ex: TypeORM) en utilisant les variables d'environnement chargées. Exemple avec TypeORM :

// app.module.ts
import { TypeOrmModule } from '@nestjs/typeorm';
import { ConfigModule, ConfigService } from '@nestjs/config';

TypeOrmModule.forRootAsync({
  imports: [ConfigModule],
  inject: [ConfigService],
  useFactory: (configService: ConfigService) => ({
    type: 'postgres',
    url: configService.get('DATABASE_URL'),
    synchronize: true,
  }),
})

Cela évite d'appeler ConfigService avant que le module ne soit prêt.

✅ Pourquoi @nestjs/config change la donne ?

Sécurité

Plus de secrets en dur dans le code.

Portabilité

Un même code tourne en dev, test, prod sans modification.

Centralisation

Toutes les variables au même endroit.

Validation

Détection précoce des erreurs de configuration.
