🤔 Qu'est-ce que NestJS ?

NestJS est un framework Node.js progressif pour construire des applications côté serveur efficaces, fiables et évolutives. Il utilise TypeScript par défaut (tout en restant compatible JavaScript pur) et s'inspire fortement de l'architecture d'Angular (modules, décorateurs, injection de dépendances).

🏛️ Philosophie : L'influence d'Angular

🧩 Modules (@Module)

Organiser l'application en blocs fonctionnels (users, products, auth). Chaque module encapsule ses contrôleurs, services et providers.

🎯 Décorateurs (@Controller, @Injectable)

Déclarer les classes comme des contrôleurs ou des services via des décorateurs TypeScript.

💉 Injection de dépendances (IoC)

Le conteneur NestJS gère les dépendances : on injecte un service dans le constructeur d'un contrôleur.

📁 Structure par défaut

Fichiers .controller.ts, .service.ts, .module.ts – similaire à la structure Angular.

✅ Les avantages de NestJS

🔷
TypeScript première classe – Support natif, typage fort, meilleure IDE.
🔷
Modularité – Code organisé, réutilisable, facile à maintenir.
🔷
Testabilité – Injection de dépendances et outils de test (Jest) intégrés.
🔷
Large écosystème – Modules officiels pour TypeORM, Mongoose, GraphQL, WebSockets, etc.
🔷
Performance – Basé sur Express (par défaut) mais peut utiliser Fastify (plus rapide).

🌍 Que peut-on construire avec NestJS ?

📡 API REST

Avec validation, authentification JWT, pagination.

📊 GraphQL

Module @nestjs/graphql + Apollo ou Mercurius.

🔌 Microservices

TCP, Redis, NATS, RabbitMQ, gRPC.

📡 WebSockets

Chat temps réel, notifications, jeux.

🏗️ CLI puissante

`nest generate` pour créer modules, services, contrôleurs.

🧪 Tests

Unitaires avec Jest, e2e avec Supertest.

📝 Premier aperçu : contrôleur NestJS

Un contrôleur gère les routes HTTP. Voici un exemple simple :

// app.controller.ts
import { Controller, Get } from '@nestjs/common';

@Controller('hello')
export class HelloController {
  @Get()
  sayHello(): string {
    return 'Bonjour depuis NestJS !';
  }
}
✨ Notez l'utilisation du décorateur @Controller('hello') qui définit le préfixe de route /hello.
