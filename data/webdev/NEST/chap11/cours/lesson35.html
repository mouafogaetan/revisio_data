1. Validation automatique avec ValidationPipe

NestJS fournit un pipe puissant qui utilise class-validator et class-transformer. Activez-le globalement dans main.ts :

import { ValidationPipe } from '@nestjs/core';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalPipes(new ValidationPipe({
    whitelist: true,           // supprime les propriétés non définies dans le DTO
    forbidNonWhitelisted: true, // rejette les requêtes avec des champs inconnus
    transform: true,           // transforme automatiquement les types (ex: string -> number)
  }));
  await app.listen(3000);
}

Exemple de DTO avec décorateurs :

import { IsEmail, IsString, MinLength } from 'class-validator';
import { ApiProperty } from '@nestjs/swagger';

export class CreateUserDto {
  @ApiProperty()
  @IsEmail()
  email: string;

  @ApiProperty()
  @IsString()
  @MinLength(6)
  password: string;
}
Astuce pédagogique : La validation prévient 80% des erreurs d'entrée et protège des attaques.

2. Authentification JWT & Passport

Installation : npm install @nestjs/jwt @nestjs/passport passport passport-jwt bcrypt

🔐 AuthModule & AuthService 
// auth.service.ts
import { Injectable } from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';
import * as bcrypt from 'bcrypt';

@Injectable()
export class AuthService {
  constructor(private jwtService: JwtService) {}
  
  async validateUser(email: string, pass: string): Promise {
    const user = await this.userService.findByEmail(email);
    if (user && await bcrypt.compare(pass, user.password)) {
      const { password, ...result } = user;
      return result;
    }
    return null;
  }
  
  login(user: any) {
    const payload = { sub: user.id, email: user.email };
    return { access_token: this.jwtService.sign(payload) };
  }
}
🛡️ JwtStrategy (validation du token) 
// jwt.strategy.ts
import { ExtractJwt, Strategy } from 'passport-jwt';
import { PassportStrategy } from '@nestjs/passport';

export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor() {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      ignoreExpiration: false,
      secretOrKey: process.env.JWT_SECRET,
    });
  }
  async validate(payload: any) {
    return { userId: payload.sub, email: payload.email };
  }
}

3. Guards : Contrôle d'accès

Les guards déterminent si une requête doit être traitée. Utilisez @UseGuards(AuthGuard('jwt')) ou un guard personnalisé pour les rôles.

// roles.guard.ts (personnalisé avec métadonnées)
import { CanActivate, ExecutionContext, Injectable } from '@nestjs/common';
import { Reflector } from '@nestjs/core';

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}
  canActivate(context: ExecutionContext): boolean {
    const roles = this.reflector.get<string[]>('roles', context.getHandler());
    if (!roles) return true;
    const request = context.switchToHttp().getRequest();
    const user = request.user;
    return roles.some(role => user.roles?.includes(role));
  }
}

// Décorateur associé
export const Roles = (...roles: string[]) => SetMetadata('roles', roles);

Utilisation dans un contrôleur :

@Post('admin/delete')
@UseGuards(AuthGuard('jwt'), RolesGuard)
@Roles('admin')
async deleteUser(@Body() data) { ... }
📘 Les guards s'exécutent après les middlewares mais avant les pipes et interceptors.

4. Interceptors : Logique transverse

Les interceptors permettent de modifier la réponse, mesurer le temps d'exécution ou ajouter des en-têtes.

⏱️ Interceptor de temps d'exécution 
@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const start = Date.now();
    return next.handle().pipe(
      tap(() => console.log(`Temps: ${Date.now() - start}ms`))
    );
  }
}
🔄 Transformation des réponses 
@Injectable()
export class ExcludePasswordInterceptor implements NestInterceptor {
  intercept(ctx: ExecutionContext, next: CallHandler) {
    return next.handle().pipe(
      map(data => {
        if (data?.password) delete data.password;
        return data;
      })
    );
  }
}

Bon à savoir On les applique globalement (app.useGlobalInterceptors()) ou par contrôleur avec @UseInterceptors().

5. Exception Filters

NestJS a une couche d'exceptions intégrée (NotFoundException, BadRequestException…). Créez un filtre global pour uniformiser les réponses d'erreur.

// http-exception.filter.ts
import { ExceptionFilter, Catch, ArgumentsHost, HttpException } from '@nestjs/common';

@Catch(HttpException)
export class HttpErrorFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const status = exception.getStatus();
    const message = exception.message || 'Internal server error';
    
    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      path: ctx.getRequest().url,
      message: message,
    });
  }
}

// Dans main.ts :
app.useGlobalFilters(new HttpErrorFilter());
✔️ Meilleure expérience API ✔️ Log centralisé possible ✔️ Sécurité (évite les fuites de stack)

Cas pratique : Mise en œuvre complète

Créez un module TasksModule avec validation, JWT, guard admin, interceptor de log et filtre d'exception.

// tasks.controller.ts
@Controller('tasks')
@UseInterceptors(LoggingInterceptor)
export class TasksController {
  @Post()
  @UseGuards(AuthGuard('jwt'))
  create(@Body(ValidationPipe) createTaskDto: CreateTaskDto) {
    return this.tasksService.create(createTaskDto);
  }

  @Delete(':id')
  @Roles('admin')
  @UseGuards(AuthGuard('jwt'), RolesGuard)
  remove(@Param('id') id: string) { ... }
}

Question réflexe (Bloom - Analyser) :

"Pourquoi est-il conseillé d'utiliser whitelist: true et un filtre d'exception global ? Quels risques évitent-ils ?"

Voir la réponse

Réponse : whitelist empêche l'injection de propriétés non autorisées (mass assignment), et un filtre global évite d'exposer les erreurs internes du serveur aux clients tout en uniformisant le format JSON. Les deux améliorent la sécurité et la maintenabilité.
