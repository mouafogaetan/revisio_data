Créer des entités et relations avec TypeORM

NestJS – Lesson 11 | Chapitre 4 : Base de données relationnelle

@Entity @Column @PrimaryGeneratedColumn @OneToMany @ManyToOne

🎯 Objectifs de la leçon

  • ✓ Créer des entités avec @Entity() dans NestJS/TypeORM.
  • ✓ Définir les colonnes avec @Column, @PrimaryGeneratedColumn.
  • ✓ Gérer les relations : @OneToMany, @ManyToOne, @OneToOne.
  • ✓ Comprendre le schéma automatique et les jointures.
Une entité correspond à une table SQL. Chaque propriété décorée @Column devient une colonne.

📄 Créer une entité basique

Exemple : une table User avec id auto-incrémenté, nom, email.

// user.entity.ts
import { Entity, Column, PrimaryGeneratedColumn } from 'typeorm';

@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column({ unique: true })
  email: string;

  @Column({ default: true })
  isActive: boolean;
}
🔑 PrimaryGeneratedColumn → clé primaire auto-incrémentée 📌 Column → options : length, nullable, unique, default

🔗 Relation OneToMany / ManyToOne

Un utilisateur peut avoir plusieurs articles (posts). C’est la relation la plus courante.

User (côté "un")

@OneToMany(() => Post, post => post.author)
posts: Post[];

Post (côté "plusieurs")

@ManyToOne(() => User, user => user.posts)
@JoinColumn()
author: User;
@JoinColumn() est obligatoire du côté @ManyToOne pour définir la clé étrangère.

💍 Relation OneToOne

Un utilisateur a un unique profil (ex: photo de profil, adresse).

// user.entity.ts
@OneToOne(() => Profile, profile => profile.user)
@JoinColumn()  // la clé étrangère se trouve dans la table User
profile: Profile;

// profile.entity.ts
@OneToOne(() => User, user => user.profile)
user: User;

Astuce : @JoinColumn() définit la table qui possède la colonne de relation (côté propriétaire).

⚙️ Options avancées des colonnes

  • @Column({ type: 'varchar', length: 255 }) → spécifie le type SQL.
  • @Column({ nullable: true }) → champ optionnel en base.
  • @Column({ update: false }) → la colonne n’est pas modifiable après création.
  • @CreateDateColumn() et @UpdateDateColumn() → dates automatiques.
✅ Exemple concret : @CreateDateColumn() createdAt: Date; sera rempli automatiquement par TypeORM.

🗂️ Schéma simplifié (User ↔ Post)

users

id (PK)

name

email

→ posts[] (OneToMany)

⟷

posts

id (PK)

title

content

authorId (FK) → users.id

Cette organisation garantit l'intégrité référentielle et permet de faire des requêtes avec relations dans TypeORM.
