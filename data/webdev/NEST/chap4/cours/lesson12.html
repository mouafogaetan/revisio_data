🔌 1. Injection du Repository

Injecter le repository avec @InjectRepository()

Dans NestJS, chaque entité possède un Repository TypeORM. Pour l’utiliser dans un service, on utilise le décorateur @InjectRepository(Entité).

// user.service.ts
import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { User } from './user.entity';

@Injectable()
export class UserService {
  constructor(
    @InjectRepository(User)
    private userRepository: Repository<User>,
  ) {}

  async findAll(): Promise<User[]> {
    return this.userRepository.find();
  }
}

📌 Rappel : Le module doit importer TypeOrmModule.forFeature([User]) pour rendre le repository disponible.

📝 Opérations essentielles du Repository

Les méthodes standards permettent un accès rapide à la BDD sans écrire de SQL.

💾 save()

Insère ou met à jour une entité.

await repo.save(user);

🔍 find() / findOne()

Récupérer plusieurs ou un seul enregistrement.

repo.find({ where: { role: 'admin' } });

✏️ update()

Mettre à jour sans charger l’entité.

repo.update(id, { name: 'nouveau' });

🗑️ delete()

Supprimer par identifiant ou critère.

repo.delete(id); 
// Exemple concret dans un service
async createUser(createUserDto) {
  const newUser = this.userRepository.create(createUserDto);
  return this.userRepository.save(newUser);
}

async findUserByEmail(email: string) {
  return this.userRepository.findOne({ where: { email } });
}
⚡⚡ QueryBuilder

Requêtes complexes avec createQueryBuilder

Le QueryBuilder permet d’écrire des requêtes SQL élégantes et sécurisées, idéal pour les jointures, sous-requêtes, agrégations.

// Trouver les utilisateurs avec leurs commandes récentes
const usersWithOrders = await this.userRepository
  .createQueryBuilder('user')
  .leftJoinAndSelect('user.commandes', 'commande')
  .where('user.isActive = :active', { active: true })
  .andWhere('commande.date > :date', { date: '2025-01-01' })
  .orderBy('user.createdAt', 'DESC')
  .getMany();
🔧 Méthodes utiles : .select() .addSelect() .where() .andWhere() .leftJoin() .getRawMany()
📊 Agrégation : .select('AVG(user.age)') .getRawOne()

🛠️ Mise en pratique – Service complet

Imaginons une entité Produit avec des relations et des recherches filtrées.

// produit.service.ts extrait
async findProductsWithFilters(category?: string, minPrice?: number) {
  const qb = this.produitRepository.createQueryBuilder('p');
  
  if(category) {
    qb.andWhere('p.categorie = :cat', { cat: category });
  }
  if(minPrice) {
    qb.andWhere('p.prix >= :prix', { prix: minPrice });
  }
  
  return qb.leftJoinAndSelect('p.fournisseur', 'fournisseur')
           .orderBy('p.prix', 'ASC')
           .getMany();
}

async updateStock(productId: number, newStock: number) {
  return this.produitRepository.update(productId, { stock: newStock });
}

✅ Bonnes pratiques : Utiliser le QueryBuilder pour les filtres dynamiques et les relations chargées. Éviter les N+1 requêtes avec leftJoinAndSelect.

🧠 Ce qu'il faut retenir

  • ✔️ @InjectRepository(Entity) → injecte le repository dans le service.
  • ✔️ Méthodes save, find, update, delete = CRUD basique.
  • ✔️ QueryBuilder = puissance et flexibilité pour requêtes élaborées.
  • ✔️ Toujours importer l’entité dans le module via TypeOrmModule.forFeature().
