👂 Sniffing – Écoute passive

Attaque passive

Interception des données circulant sur le réseau.

  • Utilisation d'outils : Wireshark, tcpdump
  • Réseau non commuté (hub) ou ARP spoofing
  • Capture : mots de passe, emails, clés API

Protection : Chiffrement (HTTPS, SSH, VPN), switch (pas hub)

Ordinateur malveillant capture tous les paquets

PC1 ←→ Switch ←→ PC2 ←→ Attaquant (sniff)

🎭 Spoofing – Usurpation d'identité

ARP Spoofing

L'attaquant prétend être la passerelle. Capture du trafic (MITM).

DNS Spoofing

Redirection vers un faux site web (phishing).

IP Spoofing

Fausse adresse IP source pour cacher l'attaquant.

# Exemple ARP spoofing (commande malveillante)
arpspoof -t 192.168.1.1 192.168.1.10

🤝 MITM – Attaque de l'homme du milieu

Client ←→ Attaquant ←→ Serveur

L'attaquant intercepte, écoute et modifie les communications

Exemples concrets :

  • Faux WiFi public (Evil Twin)
  • ARP spoofing + forwarding
  • Détournement de session HTTP

Protections : Certificats TLS, double authentification, VPN, HSTS

💥 DoS / DDoS – Déni de service

DoS (une seule source)

Un seul ordinateur envoie des requêtes en masse pour saturer le serveur.

DDoS (plusieurs sources)

Des milliers de machines (botnet) attaquent simultanément.

Exemple : Attaque par amplification DNS, NTP, SYN flood.

🌊 SYN Flood (attaque du 3-way handshake)

L'attaquant envoie des paquets SYN mais ne répond jamais au SYN-ACK.

  • Le serveur garde des connexions semi-ouvertes
  • Épuisement de la file d'attente (backlog)
  • Refus de service légitime

Attaquant → Serveur : SYN (seq=x)

Serveur → Attaquant : SYN-ACK

❌ Pas de réponse ACK

Protection : SYN cookies, réduction timeout

📢 Attaques par amplification

L'attaquant envoie une petite requête à un serveur vulnérable qui répond avec une très grosse réponse vers la victime (IP spoofée).

DNS
Facteur x50
NTP
Facteur x500
Memcached
Facteur x10000

Mitigation : Anti-spoofing (BCP38), rate limiting, Cloudflare/Akamai.
