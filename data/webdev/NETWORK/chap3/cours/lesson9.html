Lesson 9 : IPv6 - La relève d'Internet

🌐 IPv6

Le protocole qui sauve Internet

128 bits, adresses presque infinies, sécurité intégrée, et nouvelles fonctionnalités

$2^{128}$ adresses IPSec natif Plus simple

🎯 Objectifs de la leçon

Comprendre la nécessité d'IPv6 (épuisement IPv4)
Maîtriser le format IPv6 (128 bits, 8 groupes hexadécimaux)
Connaître les types d'adresses (unicast, multicast, anycast)
Savoir compresser/étendre une adresse IPv6
💡 IPv6 n'est pas juste "IPv4 avec plus de bits" — c'est une refonte complète du protocole réseau.

⚠️ Pourquoi IPv6 ? L'épuisement d'IPv4

IPv4 :

  • $32$ bits → $2^{32} \approx 4,3 \text{ milliards d'adresses}$
  • Créé en 1981 (époque des premiers PCs)
  • Derniers blocs d'adresses attribués en 2011
  • Solutions de contournement : NAT, CIDR, adresses privées

Conséquences :

  • NAT complexifie les communications (P2P difficile)
  • Pas de vraie bout-en-bout (end-to-end)
  • Fragmentation d'Internet

📊 En 2024, ~45% des utilisateurs accèdent à Google en IPv6 (source : Google IPv6 stats).

🔢 Format de l'adresse IPv6

2001:0db8:0000:0000:0000:ff00:0042:8329

128 bits = 8 groupes de 16 bits (4 chiffres hexadécimaux chacun)

Règles de compression :

  • Supprimer les zéros non significatifs d'un groupe : 0042 → 42
  • Remplacer la plus longue série de groupes nuls par :: (une seule fois)
2001:0db8:0000:0000:0000:ff00:0042:8329 → 2001:db8:0:0:0:ff00:42:8329 → 2001:db8::ff00:42:8329

🏷️ Types d'adresses IPv6

🟢 Unicast

Une seule interface. Types courants :

  • 2000::/3 → Global Unicast (Internet)
  • fe80::/10 → Link-local (réseau local)
  • fc00::/7 → Unique Local (équivalent privé)
  • ::1/128 → Loopback

🟣 Multicast

Un groupe d'interfaces. Préfixe ff00::/8. Ex: ff02::1 (tous les nœuds du lien).

🟢 Anycast

Une adresse attribuée à plusieurs machines. Le routeur envoie le paquet à la plus proche (ex: serveurs DNS anycast).

📡 Broadcast

N'existe pas en IPv6 ! Remplacé par multicast ff02::1.

✨ Avantages d'IPv6

Espace d'adressage immense : $2^{128}$ adresses
Pas de NAT → vrai bout-en-bout
Configuration automatique (SLAAC, DHCPv6)
IPSec intégré (authentification, chiffrement)
En-tête simplifié → routage plus rapide
Multicast étendu + Anycast natif
Extension headers flexibles
Pas de checksum au niveau IP (gain perf)

📌 Adresses IPv6 importantes

::1/128 → Loopback (équivalent 127.0.0.1)
:: → Adresse non spécifiée (équivalent 0.0.0.0)
2001:db8::/32 → Documentation (exemples, ne jamais utiliser sur Internet)
fe80::/10 → Link-local (communication sur le même lien, pas routée)
ff02::1 → Tous les nœuds du lien (multicast broadcast-like)
ff02::2 → Tous les routeurs du lien

✏️ Exercice : Compression IPv6

Compressez les adresses IPv6 suivantes :

  1. fe80:0000:0000:0000:0200:5eff:fe00:1234
  2. 2001:0db8:0000:0000:abcd:0000:0000:1234
  3. ff02:0000:0000:0000:0000:0000:0000:0001

Inversement, étendez : 2001:db8::1

🧩 Voir les réponses
1. fe80::200:5eff:fe00:1234
2. 2001:db8::abcd:0:0:1234 ou 2001:db8:0:0:abcd::1234
3. ff02::1
Extension : 2001:0db8:0000:0000:0000:0000:0000:0001

📌 Récapitulatif

IPv6 : 128 bits, $3.4 \times 10^{38}$ adresses
Format : 8 groupes hexadécimaux de 16 bits
Compression : suppression zéros non significatifs + ::
Types : Unicast (global, link-local, unique local), Multicast, Anycast
Plus de NAT, plus de broadcast → communication bout-en-bout retrouvée
