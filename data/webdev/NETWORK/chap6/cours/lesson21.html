Le Routeur & NAT/PAT

Comprendre le rôle du routeur, ses types et la traduction d'adresses

👨‍🏫 Classe Développement Web • Cameroun

🎯 Objectifs du cours

  • ✓ Comprendre le rôle d'un routeur (niveau 3, table de routage)
  • ✓ Distinguer routeur domestique et routeur professionnel
  • ✓ Comprendre le NAT (Network Address Translation) et PAT
💡 À la fin, vous saurez expliquer comment votre box Internet permet à plusieurs appareils de partager une seule IP publique.

📡 Qu'est-ce qu'un routeur ?

Un routeur est un équipement réseau qui connecte plusieurs réseaux entre eux et achemine les paquets (niveau 3 du modèle OSI). Il utilise une table de routage pour décider par où envoyer chaque paquet.

┌─────────┐ ┌─────────┐ ┌─────────┐ │ Réseau A │ ─── │ Routeur │ ─── │ Réseau B │ │192.168.1.0│ │ │ │10.0.0.0 │ └─────────┘ └─────────┘ └─────────┘

Exemple : un routeur relie votre réseau local (192.168.1.x) à Internet.

🗺️ La table de routage

Le routeur consulte sa table de routage pour chaque paquet. Elle contient des entrées comme :

Réseau destinationMasquePasserelle / Interface
192.168.1.0255.255.255.0eth0 (direct)
0.0.0.00.0.0.0203.0.113.1 (FAI)
  • La route par défaut (0.0.0.0/0) est utilisée quand aucune autre route ne correspond.
  • Le routeur choisit toujours la route la plus spécifique (masque le plus long).

🏠 vs 🏢 Routeur domestique vs professionnel

Routeur domestique (box)

  • 4 ports Ethernet + WiFi
  • NAT/PAT intégré (partage d'IP publique)
  • Serveur DHCP, pare-feu basique
  • Configuration simple (interface web)
  • Exemple : Livebox, Freebox, Bbox

Routeur professionnel

  • Matériel plus puissant (plusieurs cœurs)
  • Protocoles dynamiques (OSPF, BGP)
  • VPN, VLAN, QoS avancée
  • Redondance (HSRP, VRRP)
  • Exemples : Cisco, MikroTik, Juniper

⚠️ Le problème : pas assez d'adresses IP publiques

IPv4 ne permet que 4,3 milliards d'adresses. Avec des milliards d'appareils connectés, c'est insuffisant.

🌍 Solution : NAT (Network Address Translation) – partager une seule IP publique entre plusieurs machines privées.

Les adresses privées (RFC 1918) ne sont pas routables sur Internet :

  • 10.0.0.0 – 10.255.255.255
  • 172.16.0.0 – 172.31.255.255
  • 192.168.0.0 – 192.168.255.255

🔄 Comment fonctionne le NAT ?

Le routeur NAT remplace l'adresse IP privée source par son adresse IP publique dans les paquets sortants. À la réception, il fait l'opération inverse.

PC (192.168.1.10) ──▶ Routeur NAT (203.0.113.50) ──▶ Serveur (8.8.8.8) [source: 192.168.1.10] [source: 203.0.113.50] [destination: 8.8.8.8]

Mais comment savoir à quel PC renvoyer la réponse ? Le routeur utilise une table de traduction.

🔢 PAT : plusieurs appareils, un seul port public

Le PAT (ou NAT overloading) utilise les numéros de port pour distinguer les connexions. Plusieurs machines derrière une même IP publique.

Table PAT (exemple) :
192.168.1.10:12345 → 203.0.113.50:55001
192.168.1.11:12345 → 203.0.113.50:55002
192.168.1.12:80 → 203.0.113.50:55003

Chaque connexion sortante se voit attribuer un port source unique sur l'IP publique. La réponse est redirigée vers la bonne machine grâce à ce port.

🏠 Exemple : votre box Internet

Smartphone (IP privée 192.168.1.20) veut visiter google.com

  1. Le smartphone envoie la requête vers la box (192.168.1.1).
  2. La box NAT : remplace l'IP source par son IP publique (81.64.x.x), choisit un port libre (ex: 55001).
  3. Google répond à la box (IP publique).
  4. La box consulte sa table PAT et renvoie la réponse au smartphone.

📌 L'ordinateur du voisin a aussi 192.168.1.20, mais derrière sa propre box. Pas de conflit car les boîtes sont différentes.

⚠️ Inconvénients du NAT

  • Impossible de recevoir des connexions entrantes sans configuration manuelle (redirection de ports).
  • Certains protocoles (FTP, SIP, IPSec) ont du mal à traverser le NAT (nécessitent des helpers ALG).
  • IPv6 a été créé pour supprimer le besoin de NAT (chaque appareil a une IP publique).
  • Ajoute de la latération et de la complexité.
💡 Solution : redirection de ports (port forwarding) pour autoriser l'accès depuis l'extérieur (ex: serveur web, jeu en ligne).

📊 Les 3 types de NAT

NAT statique : 1 IP privée ↔ 1 IP publique fixe (bijection). Utile pour un serveur visible de l'extérieur.
NAT dynamique : un pool d'IP publiques est attribué dynamiquement aux IP privées (moins courant aujourd'hui).
PAT (NAT overloading) : plusieurs IP privées partagent une seule IP publique via des ports. C'est ce qu'utilisent les box Internet.

🖥️ Exemple de configuration (Cisco)

! Définir les interfaces
interface eth0
 ip address 192.168.1.1 255.255.255.0
 ip nat inside
!
interface eth1
 ip address 203.0.113.50 255.255.255.248
 ip nat outside
!
! Configurer le PAT
ip nat inside source list 1 interface eth1 overload
!
! Liste des IP privées autorisées
access-list 1 permit 192.168.1.0 0.0.0.255
!
! Voir les traductions actives
show ip nat translations

La commande show ip nat translations affiche la table PAT.

🔓 Redirection de ports (port forwarding)

Permet d'accéder à un service interne depuis Internet. Exemple : héberger un site web chez soi.

Règle : port 80 (TCP) externe → 192.168.1.100:80 (serveur web local)

Dans l'interface de votre box :

  • Application : Serveur Web (ou personnalisé)
  • Port externe : 80
  • IP interne : 192.168.1.100
  • Port interne : 80
  • Protocole : TCP
⚠️ Attention à la sécurité : n'ouvrez que les ports nécessaires !

📌 Ce qu'il faut retenir

✓ Routeur = aiguillage au niveau réseau
✓ Table de routage + route par défaut
✓ NAT = traduit IP privées → IP publique
✓ PAT = partage d'une IP via les ports
✓ Routes domestique : tout-en-un (NAT, WiFi, DHCP)
✓ Port forwarding pour services entrants
🌟 Sans NAT, Internet serait à court d'adresses IP depuis longtemps !
