💡 Packet Tracer est le logiciel idéal pour se former aux réseaux sans matériel physique. Utilisé dans les formations Cisco (CCNA).

📦 Qu'est-ce que Packet Tracer ?

Packet Tracer est un simulateur réseau développé par Cisco Systems. Il permet de :

  • Créer des topologies réseau (LAN, WAN)
  • Configurer des équipements (routeurs, switches, PC, serveurs)
  • Simuler le trafic (Ping, traceroute, HTTP, DNS, Email)
  • Visualiser les protocoles en temps réel (mode simulation)

🔹 Disponible gratuitement pour les étudiants (après inscription à Cisco NetAcad)

Windows, Linux, macOS (via Wine) et version Web limitée.

🖥️ Découverte de l'interface

Zone de travail : l'espace où l'on place les équipements et on les câble.
Panneau inférieur : catégories d'équipements (routeurs, switches, terminaux, câbles).
Barre d'outils : sélection, supprimer, annuler, mode réel/simulation.
Mode Realtime : les paquets circulent en temps réel (comme sur un vrai réseau).
Mode Simulation : on peut observer chaque paquet étape par étape.
📌 Ctrl+Z annule une action. Le Device Template Manager permet d'ajouter des équipements personnalisés.

🔌 Principaux équipements dans PT

CatégorieExemplesRôle
RouteursRouter 1941, 2911, ISR4331Connecter des réseaux différents, NAT, routage
SwitchesSwitch 2960, 3650, 9300Connecter des périphériques sur le même réseau (couche 2)
TerminauxPC, Laptop, Serveur, IoTTest et services (DHCP, DNS, HTTP)
Moyens de transmissionCuivre (Copper), Fibre, Sans filLiaison entre équipements

💡 On peut ajouter des modules (cartes d'interface) aux routeurs et switches pour étendre les ports.

🏗️ Construisons notre premier réseau

Objectif : connecter 2 PC à un switch, puis un routeur pour simuler un réseau simple.

Étapes :
1. Glisser un Switch 2960 sur la zone de travail.
2. Glisser deux PC (PC0 et PC1).
3. Choisir l'outil "Câble cuivre droit" (Copper Straight-Through).
4. Relier PC0 au port FastEthernet 0/1 du switch.
5. Relier PC1 au port FastEthernet 0/2 du switch.
6. Ajouter un routeur 1941 et le relier au switch (port Gig0/0 → switch port 0/3).
🧪 Test rapide : les LEDs des ports doivent passer du rouge au vert (liaison active).

⚙️ Configurer les adresses IP

Cliquez sur un PC → onglet Desktop → IP Configuration.

Exemple de configuration simple :

  • PC0 : 192.168.1.10 / 255.255.255.0, passerelle 192.168.1.1
  • PC1 : 192.168.1.11 / 255.255.255.0, passerelle 192.168.1.1
  • Routeur (interface Gi0/0) : 192.168.1.1 / 24
$ ping 192.168.1.11
Pinging 192.168.1.11 with 32 bytes of data: Reply from 192.168.1.11: bytes=32 time<1ms TTL=128 Reply from 192.168.1.11: bytes=32 time<1ms TTL=128

⌨️ Utiliser la CLI des routeurs/switches

Pour configurer un routeur, cliquer sur l'équipement → onglet CLI.

Router> enable
Router# configure terminal
Router(config)# interface gigabitethernet 0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit
Router(config)# exit
Router# show ip interface brief
Router# copy running-config startup-config

✅ no shutdown active l'interface (très important !). show running-config affiche la configuration courante.

🧪 Vérification avec ping et traceroute

Depuis un PC (Desktop → Command Prompt)

ping 192.168.1.1
ping 8.8.8.8

Depuis le routeur (CLI)

Router# ping 192.168.1.10
Router# traceroute 192.168.1.11

📊 Interprétation :

✔️ "!!!!!" ou "Reply from" = succès.
❌ "....." ou "Request timed out" = échec (vérifier câbles, IP, masque, no shutdown).

🐞 Mode simulation : analyser les paquets

Le mode simulation permet de voir chaque paquet traverser le réseau pas à pas.

  1. Passer en mode Simulation (en bas à droite).
  2. Créer un scénario : "Ping de PC0 vers PC1".
  3. Cliquer sur "Play" et observer la PDU (Packet Data Unit).
  4. Cliquer sur un paquet pour voir les couches OSI (ICMP, IP, Ethernet).
💡 C'est très pédagogique pour comprendre l'encapsulation des données et le rôle de chaque couche.

🏋️ Exercice pratique à réaliser

  1. Installer Packet Tracer (version 8.x).
  2. Construire le réseau suivant :
    2 switches (Switch0 et Switch1) – 1 routeur (Router0).
    Switch0 connecté à Router0 (Gi0/0).
    Switch1 connecté à Router0 (Gi0/1).
    3 PC sur Switch0 (IP : 192.168.1.10-12/24).
    2 PC sur Switch1 (IP : 192.168.2.10-11/24).
  3. Activer les interfaces, configurer les IP des PC.
  4. Vérifier que les PC d'un même switch se pingent.
  5. Ajouter deux routes statiques sur le routeur pour que les deux réseaux communiquent.
  6. Tester le ping de 192.168.1.10 vers 192.168.2.10.
💡 Si vous bloquez, utilisez la commande show ip route sur le routeur pour vérifier les routes.

📝 À retenir

  • Packet Tracer est un simulateur réseau idéal pour s'entraîner sans matériel coûteux.
  • On peut y construire des topologies LAN/WAN complexes, configurer routeurs, switches, serveurs.
  • Les équipements se connectent avec des câbles : cuivre droit (PC→switch), croisé (switch→switch), console (accès CLI).
  • La configuration se fait via GUI (PC) ou CLI (routeurs/switches).
  • Le mode simulation permet de visualiser chaque paquet et de comprendre l'encapsulation.

🎓 Pour aller plus loin :

Essaie de configurer un serveur DHCP sur un routeur, puis de connecter un PC en IP automatique. Tu peux aussi ajouter un serveur HTTP et y accéder depuis un navigateur du simulateur.
