🖥️ Cisco Packet Tracer

Packet Tracer est un simulateur réseau développé par Cisco. Il permet de construire des topologies, configurer des équipements (routeurs, switches, serveurs) et simuler des protocoles sans matériel réel.

Avantages : gratuit pour les étudiants, interface graphique, grand choix d'équipements
Limites : certaines fonctionnalités avancées manquantes, ne remplace pas un vrai équipement
Il est utilisé dans de nombreuses formations (Cisco CCNA, BTS, Licences).

📡 Serveur DHCP

DHCP (Dynamic Host Configuration Protocol) attribue automatiquement des adresses IP aux machines du réseau.

Configuration dans Packet Tracer (serveur générique) :
1. Ajouter un serveur et un switch avec des PCs
2. Sur le serveur, onglet Services → DHCP
3. Activer le service DHCP
4. Définir le pool : 
   - Nom du pool : "LAN"
   - Adresse de départ : 192.168.1.1
   - Masque : 255.255.255.0
   - Passerelle par défaut : 192.168.1.254
   - DNS Server : 192.168.1.10 (ou 8.8.8.8)
💡 Les clients en DHCP obtiendront automatiquement IP, masque, passerelle et DNS.

🔍 Serveur DNS

DNS (Domain Name System) résout les noms de domaine en adresses IP.

Configuration dans Packet Tracer (serveur DNS) :
1. Onglet Services → DNS
2. Activer le service DNS
3. Ajouter des enregistrements :
   - Nom : www.monsite.cm
   - Adresse : 192.168.1.100
   - Type : A Record
4. Ajouter éventuellement un CNAME :
   - Nom : intranet
   - Adresse : www.monsite.cm (alias)
5. Sur les clients, paramétrer l'adresse du serveur DNS
💡 Test : depuis un PC, ouvrir l'invite de commandes et taper ping www.monsite.cm

🌐 Serveur Web (HTTP)

Packet Tracer intègre un serveur web simple avec pages personnalisables.

Configuration :
1. Onglet Services → HTTP
2. Activer HTTP (et HTTPS si besoin)
3. Créer/modifier des fichiers HTML :
   - Fichier index.html : page d'accueil
   - Fichier contact.html : autre page
4. Les clients peuvent accéder via navigateur :
   - http://192.168.1.100
   - http://www.monsite.cm (si DNS configuré)
⚠️ Packet Tracer ne supporte pas le PHP ou les bases de données, seulement du HTML/CSS statique.

🏗️ Topologie type pour TP

    [Serveur DHCP/DNS/Web]
            |
        [Switch]
         /  |  \
      [PC1][PC2][PC3]
    
    Adresses :
    - Serveur : 192.168.1.10 (statique)
    - Pool DHCP : 192.168.1.100 - 192.168.1.200
    - Passerelle : 192.168.1.254 (routeur fictif ou switch couche 3)

Les clients en DHCP obtiennent IP + DNS. Ils peuvent naviguer sur le serveur web via son nom DNS.

✅ Vérifications sur les clients

ipconfig /all (Windows) ou ifconfig (Linux) → vérifier IP obtenue, serveur DNS
ping 192.168.1.10 → tester connectivité avec serveur
ping www.monsite.cm → tester résolution DNS
nslookup www.monsite.cm → interroger DNS directement
Navigateur → taper http://www.monsite.cm → afficher page web

📌 Bonnes pratiques dans Packet Tracer

  • Nommer clairement les équipements (ex: SRV-DHCP, PC-Bureau1)
  • Utiliser des couleurs différentes pour les VLANs ou services
  • Sauvegarder régulièrement (.pkt)
  • Vérifier les câbles (type : droit, croisé) – Packet Tracer gère automatiquement mais attention aux liaisons croisées entre switches
  • Tester au fur et à mesure avec ping avant de passer aux services
  • Utiliser le mode Simulation pour observer les paquets DHCP (DORA : Discover, Offer, Request, Ack)
