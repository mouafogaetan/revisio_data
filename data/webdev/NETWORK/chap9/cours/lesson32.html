Wireshark : Analyse de paquets

Capturer, filtrer et analyser le trafic réseau

👨‍🏫 Classe Développement Web • Cameroun

🎯 Objectifs du cours

  • ✓ Installer Wireshark et capturer des paquets
  • ✓ Filtrer les paquets (ip.addr, tcp.port, http)
  • ✓ Analyser une conversation TCP et un flux HTTP
💡 À la fin, vous saurez utiliser Wireshark pour diagnostiquer des problèmes réseau.

🦈 Qu'est-ce que Wireshark ?

Wireshark est un analyseur de protocoles réseau (sniffer) qui capture et affiche le trafic circulant sur une interface réseau (Ethernet, WiFi, loopback...).

  • Open-source et gratuit.
  • Disponible sur Windows, macOS, Linux.
  • Permet d'inspecter chaque paquet (trames, IP, TCP, UDP, HTTP, DNS...).
  • Outils de filtrage puissants.
⚠️ La capture nécessite des droits administrateur (root). Utilisez toujours légalement, sur vos propres réseaux.

⚙️ Installation de Wireshark

Windows / macOS : Télécharger depuis wireshark.org et exécuter l'installateur. Sous Windows, installez aussi Npcap (pour capturer).

Linux (Debian/Ubuntu) :

sudo apt update
sudo apt install wireshark -y

Autorisation (Linux) : Ajouter votre utilisateur au groupe wireshark :

sudo usermod -aG wireshark $USER

Redémarrez la session pour que les droits soient effectifs.

🖥️ Interface principale

3 panneaux essentiels :

  • Liste des paquets : chaque ligne = un paquet (n° temps source destination protocole info).
  • Détail du paquet : arborescence des couches (Ethernet, IP, TCP, HTTP...).
  • Données brutes (hexadécimal + ASCII) : le contenu du paquet en binaire.
No. Time Source Destination Protocol Info 1 0.000 192.168.1.10 8.8.8.8 DNS Standard query A google.com 2 0.045 8.8.8.8 192.168.1.10 DNS Standard query response A 142.250.185.46 3 0.051 192.168.1.10 142.250.185.46 TCP 52000 → 80 [SYN] 4 0.067 142.250.185.46 192.168.1.10 TCP 80 → 52000 [SYN, ACK] 5 0.068 192.168.1.10 142.250.185.46 TCP 52000 → 80 [ACK]

🎬 Démarrer une capture

  1. Choisir l'interface réseau (Wi-Fi, Ethernet, Loopback).
  2. Cliquer sur le bouton « Démarrer la capture » (icône requin 🦈).
  3. Générer du trafic (naviguer, pinger...).
  4. Cliquer sur « Arrêter » (carré rouge).
💡 Astuce : Utilisez un filtre de capture avant de commencer pour limiter la taille du fichier. Exemple : host 8.8.8.8

🔍 Les filtres d'affichage

Pour ne voir que les paquets intéressants.

FiltreSignification
ip.addr == 192.168.1.10Tous paquets avec cette IP (source ou dest)
tcp.port == 80Trafic HTTP (port 80)
udp.port == 53Trafic DNS
httpTous les paquets contenant HTTP
tcp.flags.syn == 1Paquets SYN (début connexion)
icmpPing (ICMP Echo request/reply)

🔗 Combinaison de filtres

Utilisez and, or et parenthèses.

ip.src == 192.168.1.10 and tcp.port == 443 → trafic HTTPS sortant
http or dns → paquets HTTP ou DNS
tcp.port == 80 and ip.dst == 1.1.1.1 → HTTP vers 1.1.1.1
!(arp or icmp) → tout sauf ARP et ICMP
🎯 Exemple concret : ip.addr == 8.8.8.8 and tcp → tous les échanges TCP avec Google DNS.

📜 Suivre un flux TCP

Pour reconstruire une conversation complète (requête + réponse).

  1. Sélectionner un paquet d'une connexion TCP (ex: HTTP).
  2. Clic droit → Follow → TCP Stream.
  3. Une nouvelle fenêtre affiche la conversation brute.
GET / HTTP/1.1 Host: example.com User-Agent: Mozilla/5.0 HTTP/1.1 200 OK Content-Type: text/html Content-Length: 1256 <html><body>Example Domain</body></html>

Utile pour déboguer une API, voir les en-têtes HTTP, ou extraire des fichiers.

🤝 Observer le 3-way handshake

Filtre : tcp.flags.syn == 1 or tcp.flags.ack == 1 puis chercher une connexion.

1. 192.168.1.10 → 93.184.216.34 TCP [SYN] Seq=0 2. 93.184.216.34 → 192.168.1.10 TCP [SYN, ACK] Seq=0 Ack=1 3. 192.168.1.10 → 93.184.216.34 TCP [ACK] Seq=1 Ack=1
  • SYN : le client demande une connexion.
  • SYN+ACK : le serveur accepte.
  • ACK : le client confirme.

On peut aussi voir la fenêtre TCP (window size) et les options comme MSS.

🌐 Analyser du trafic HTTP

Filtre : http ou tcp.port == 80.

GET /index.html HTTP/1.1 Host: www.example.com HTTP/1.1 200 OK Cache-Control: max-age=604800 Content-Type: text/html; charset=UTF-8 Content-Length: 1270

Wireshark décode : méthode GET, code 200, taille du contenu. Vous pouvez aussi voir les cookies ou User-Agent.

⚠️ HTTP n'est pas chiffré – sur WiFi public, quelqu'un peut lire vos requêtes. Préférez HTTPS !

📊 Outils statistiques

  • Statistics → Protocol Hierarchy : répartition du trafic par protocole.
  • Statistics → Conversations : endpoints qui échangent le plus.
  • Statistics → IO Graph : visualisation du débit dans le temps.
  • Statistics → Flow Graph : diagramme de séquence (TCP).
Exemple : identifier quel hôte génère le plus de trafic ou détecter un pic anormal.

🎨 Coloration conditionnelle

Wireshark colore automatiquement les paquets par défaut (TCP SYN en bleu, retransmissions en noir...).

Vous pouvez créer vos propres règles :

View → Coloring Rules → New
Exemple : colorer en rouge les paquets vers un IP suspect
ip.dst == 185.130.5.253

💾 Exporter et sauvegarder

  • File → Save As : sauvegarde la capture en .pcapng (format standard).
  • Export Packet Dissections → en texte, JSON, CSV.
  • Export Objects → HTTP : extrait tous les fichiers téléchargés (images, HTML).
🔐 Attention : les captures peuvent contenir des données sensibles (mots de passe en clair si HTTP). Manipulez-les avec précaution.

🩺 Exemple : ralentissement réseau

Scénario : un utilisateur se plaint que le site web met du temps à charger.

  1. Capturer le trafic (host site.com).
  2. Filtrer tcp.analysis.retransmission → voir s'il y a des pertes.
  3. Observer le Time to first byte (TTFB) : délai entre requête et premier octet de réponse.
  4. Si beaucoup de retransmissions → problème de perte de paquets.
  5. Si TTFB long → serveur ou latence réseau élevée.
