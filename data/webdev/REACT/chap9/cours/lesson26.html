Leçon 26 – Consommer une API avec fetch()

Apprenez à interroger des serveurs, afficher des données et gérer les erreurs comme un pro.

🌐 fetch API ⏳ useEffect 🛑 AbortController

Objectifs de la leçon

  • ✓ Récupérer des données avec fetch() dans useEffect.
  • ✓ Gérer les états de chargement (loading) et d'erreur.
  • ✓ Annuler les requêtes avec AbortController pour éviter les fuites mémoire.
💡 Une bonne gestion des requêtes API est essentielle pour des applications React professionnelles.

🤔 Pourquoi fetch dans useEffect ?

Les requêtes API sont des effets de bord. Elles doivent être exécutées après le rendu du composant, pas pendant. useEffect est l'endroit parfait pour cela.

📌 Rappel :

  • Le rendu doit être pur (pas d'appel réseau).
  • useEffect s'exécute après que React a mis à jour le DOM.
  • On utilise un tableau de dépendances vide [] pour charger les données une seule fois au montage.
// Structure de base
useEffect(() => {
  fetch('https://api.exemple.com/data')
    .then(response => response.json())
    .then(data => setData(data));
}, []); // ← tableau vide = une seule fois

⏳ Afficher un indicateur de chargement

Pendant que les données arrivent, montrez un spinner ou un message à l'utilisateur.

function ListeUtilisateurs() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => {
        setUsers(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <div className="loader">Chargement...</div>;
  if (error) return <div>Erreur : {error}</div>;
  return <ul>{users.map(user => <li key={user.id}>{user.name}</li>)}</ul>;
}

⚠️ Gestion des erreurs HTTP (404, 500, etc.)

fetch() ne rejette pas la promesse pour les erreurs HTTP (ex: 404). Il faut vérifier response.ok.

useEffect(() => {
  const fetchData = async () => {
    setLoading(true);
    try {
      const response = await fetch('https://api.exemple.com/data');
      if (!response.ok) {
        throw new Error(`HTTP ${response.status}: ${response.statusText}`);
      }
      const data = await response.json();
      setData(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };
  fetchData();
}, []);
🔴 Important : Toujours gérer le cas où le serveur répond avec une erreur (404, 500, etc.).

🛑 AbortController : annuler les requêtes

Si l'utilisateur quitte la page avant la fin de la requête, on annule pour éviter les warnings et les fuites mémoire.

useEffect(() => {
  const abortController = new AbortController();

  const fetchData = async () => {
    try {
      const response = await fetch('https://api.exemple.com/data', {
        signal: abortController.signal  // ← lien avec le controller
      });
      const data = await response.json();
      setData(data);
    } catch (err) {
      if (err.name !== 'AbortError') {
        setError(err.message);
      }
    }
  };
  fetchData();

  return () => abortController.abort(); // Nettoyage
}, []);
✅ Bon à savoir : Le nettoyage empêche setState sur un composant démonté.

🔁 Requête avec dépendances (recherche dynamique)

Quand l'utilisateur tape dans un champ, on relance la requête.

function RechercheProduits() {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState([]);
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    if (query.length < 2) return;
    const controller = new AbortController();
    setLoading(true);

    fetch(`https://api.mercadolibre.com/sites/MLC/search?q=${query}`, {
      signal: controller.signal
    })
      .then(res => res.json())
      .then(data => {
        setResults(data.results || []);
        setLoading(false);
      })
      .catch(err => {
        if (err.name !== 'AbortError') setLoading(false);
      });

    return () => controller.abort();
  }, [query]); // ← se relance à chaque changement de query

  return (
    <div>
      <input value={query} onChange={e => setQuery(e.target.value)} />
      {loading && <p>Recherche...</p>}
      <ul>{results.map(item => <li key={item.id}>{item.title}</li>)}</ul>
    </div>
  );
}

✅ Bonnes pratiques à retenir

  • Toujours utiliser useEffect avec les dépendances adaptées.
  • Ne pas appeler fetch() directement dans le corps du composant.
  • Utiliser AbortController pour les requêtes qui peuvent être interrompues.
  • Ne pas oublier de gérer loading et error.
  • Extraire la logique de fetch dans des hooks personnalisés (ex: useFetch) pour la réutiliser.

⚠️ Anti-pattern à éviter :

Ne pas mettre l'appel fetch dans une fonction asynchrone déclarée dans l'effet sans gérer le cleanup.
