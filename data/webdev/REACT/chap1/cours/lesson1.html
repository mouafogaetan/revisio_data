📌 React, c'est quoi ?

React est une bibliothèque JavaScript développée par Facebook (Meta). Elle sert à construire des interfaces utilisateur (UI) interactives et performantes.

🎯 Idée principale :

On découpe l'écran en composants indépendants et réutilisables. Chaque composant gère sa propre logique et son apparence.

// Exemple d'un petit composant React
function Bienvenue() {
  return <h1>Bonjour Douala !</h1>;
}

⚠️ React n'est pas un framework complet (comme Angular), mais une bibliothèque centrée sur la vue.

🖥️ Single Page Application (SPA)

Une SPA est une application web qui charge une seule page HTML, puis met à jour dynamiquement le contenu sans recharger toute la page.

❌ Traditionnel (MPA)
Clic → rechargement complet → attendre → afficher
✅ React (SPA)
Clic → React modifie juste la partie nécessaire → instantané
🚀 Avantages : Rapidité, fluidité, expérience utilisateur proche d'une application mobile.
Exemples connus : Facebook, Gmail, Twitter, Netflix.

⚛️ Virtual DOM : la magie de React

Manipuler le DOM réel (celui du navigateur) est lent. React crée une copie légère en mémoire : le Virtual DOM.

🔁 Comment ça marche ?

  1. Quand l'état change, React met à jour le Virtual DOM.
  2. Il compare l'ancienne version avec la nouvelle (diffing).
  3. Il applique uniquement les changements nécessaires au DOM réel (reconciliation).
[ Virtual DOM ] → diffing → [ DOM réel modifié intelligemment ]

✅ Résultat : des mises à jour ultra-rapides et des applications fluides.

🌟 Les avantages de React

✔️Composants réutilisables → code propre
✔️Grande communauté et écosystème riche
✔️Performance grâce au Virtual DOM
✔️Support mobile avec React Native
✔️JSX : mélanger HTML et JavaScript facilement
✔️Outillage moderne (Vite, Next.js)
💼 Employabilité : React est très demandé par les entreprises au Cameroun et dans le monde.

👨‍💻 React vs JavaScript pur

Avec JS pur : manipulation directe du DOM lourde. Avec React : on décrit l'interface, React fait le reste.

// JavaScript pur : manipulation manuelle
const div = document.createElement('div');
div.textContent = 'Bonjour';
document.body.appendChild(div);

// React : déclaratif
function App() {
  return <div>Bonjour</div>;
}

React est déclaratif : on dit quoi afficher, pas comment le faire.

📝 Ce qu'il faut retenir

  • React = bibliothèque JS pour construire des UI.
  • SPA = une seule page HTML, navigation fluide sans rechargement.
  • Virtual DOM = copie légère qui optimise les mises à jour.
  • Composants = briques réutilisables.

🧪 Petit test pour vous :

1. React est un framework complet ? (Non, c'est une bibliothèque).
2. Le Virtual DOM est plus lent que le DOM réel ? (Non, il est plus rapide).
3. Une SPA recharge toute la page à chaque clic ? (Non, seulement le contenu change).
