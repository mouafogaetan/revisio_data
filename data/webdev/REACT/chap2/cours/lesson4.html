🧩 Un composant, c'est quoi ?

Un composant React est une fonction (ou une classe) qui retourne du JSX (du HTML amélioré). Chaque composant représente une partie de l'interface utilisateur.

🎯 Analogie simple :

Comme des Lego 🧱. Chaque brique (composant) est indépendante, on peut l'utiliser plusieurs fois et les assembler pour construire un grand château (l'application).

// Exemple de composant simple
function Bienvenue() {
  return <h1>Bonjour Douala !</h1>;
}

⚠️ Un composant doit toujours retourner un élément JSX (une seule balise parente).

📛 Règle du nommage : PascalCase

Le nom d'un composant doit commencer par une lettre majuscule et utiliser la convention PascalCase (chaque mot commence par une majuscule).

✅ Corrects :
Menu, ProfileCard, UserAvatar
❌ Incorrects :
menu, profileCard, user_avatar
🔍 Pourquoi ? React reconnaît les composants grâce à la majuscule. Si vous écrivez <menu />, React pense que c'est une balise HTML standard.

⚙️ Créer son premier composant

Dans un fichier séparé (ex: Salutation.jsx) :

// Salutation.jsx
function Salutation() {
  return (
    <div className="p-4 bg-blue-100 rounded">
      <h2>Bonjour les étudiants !</h2>
      <p>Bienvenue dans React</p>
    </div>
  );
}

export default Salutation;
📌 export default permet d'importer ce composant ailleurs.

📥 Importer un composant

Dans App.jsx (ou un autre composant), on importe notre composant :

// App.jsx
import Salutation from './Salutation';

function App() {
  return (
    <div>
      <Salutation />
      <Salutation />  {/* on peut le réutiliser ! */}
    </div>
  );
}

export default App;
📁 Salutation.jsx → export default
⇢ import ... from
✅ Utilisation <Salutation />

🔗 La composition : assembler des composants

On peut imbriquer des composants les uns dans les autres pour construire une interface complète.

// Header.jsx
function Header() { return <header>Mon site</header>; }

// Footer.jsx
function Footer() { return <footer>© 2025</footer>; }

// PageAccueil.jsx
function PageAccueil() {
  return (
    <div>
      <Header />
      <main>Contenu principal</main>
      <Footer />
    </div>
  );
}
🌟 Avantage : chaque composant est indépendant, facile à tester et à réutiliser.

📤 Export nommé vs export par défaut

Il existe deux façons d'exporter :

export default

// composant.js
export default MonComposant;

// import
import MonComposant from './composant';

export nommé

// utils.js
export function formatDate() {}
export const API_URL = "...";

// import avec accolades
import { formatDate, API_URL } from './utils';

💡 Pour un composant principal, on utilise souvent export default. Pour des fonctions utilitaires, on utilise l'export nommé.

📁 Organisation d'un projet React

Voici une structure simple pour vos composants :

src/
├── components/
│   ├── Header.jsx
│   ├── Footer.jsx
│   ├── Card.jsx
│   └── Button.jsx
├── App.jsx
└── main.jsx
  • Un fichier = un composant (par convention).
  • Nom du fichier = nom du composant (PascalCase).
  • Les composants réutilisables vont dans le dossier components/.
🧠 À retenir : "Sépare pour mieux régner" – découpez votre interface en petites fonctions.
