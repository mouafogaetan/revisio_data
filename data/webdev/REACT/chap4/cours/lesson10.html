💡 Le rendu conditionnel = afficher différentes interfaces selon l'état de l'application !

🔀 if/else en dehors du JSX

La manière la plus simple : on utilise une condition classique avant l'instruction return.

function WelcomeMessage({ isLoggedIn }) { let message; if (isLoggedIn) { message = <h1>Bienvenue !</h1>; } else { message = <h1>Veuillez vous connecter</h1>; } return <div>{message}</div>; }
✅ Avantage : lisible, facile à comprendre
⚠️ Inconvénient : un peu verbeux pour les petites conditions

❓ L'opérateur ternaire : condition ? vrai : faux

Idéal pour des conditions simples, directement dans le JSX.

function Status({ isOnline }) { return ( <div> {isOnline ? ( <span className="text-green-500">● En ligne</span> ) : ( <span className="text-gray-500">● Hors ligne</span> )} </div> ); }

💡 Astuce : On peut aussi ternaires imbriqués, mais attention à la lisibilité !

⚡ L'opérateur && : afficher OU rien

On utilise condition && <JSX> quand on veut afficher un élément seulement si la condition est vraie, sinon rien.

function Notification({ message }) { return ( <div> {message && <div className="alert">{message}</div>} <p>Contenu principal</p> </div> ); }

🔍 Comment ça marche ?

En JavaScript, condition && expression : si condition est vraie, l'expression est retournée ; si fausse, JavaScript retourne false et React ne rend rien.

⚠️ Attention aux valeurs falsy !

En JavaScript, certaines valeurs sont considérées comme "fausses" : 0, "", null, undefined, NaN, false.

❌ Piège classique

// Si count = 0, on affiche "0" à l'écran ! {count && <p>Vous avez {count} messages</p>}

Car 0 est falsy, mais React affiche la valeur 0.

✅ Solution

// Transformer en booléen explicite {count > 0 && <p>Vous avez {count} messages</p>}

Maintenant 0 > 0 = false, rien ne s'affiche.

📌 Règle d'or : Utilisez toujours une condition qui retourne un booléen explicite avec && !

🏗️ Exemple complet : Profil utilisateur

function UserProfile({ user }) { // if/else avant return pour des cas complexes if (!user) { return <div>Chargement...</div>; } return ( <div className="profile"> <h2>{user.name}</h2> {/* Ternaire pour le statut */} <p>Statut : {user.isAdmin ? "Administrateur" : "Utilisateur"}</p> {/* && pour afficher le badge VIP */} {user.isVip && <span className="vip-badge">⭐ VIP</span>} {/* Affichage conditionnel d'une section */} {user.bio ? ( <div><h3>Bio</h3><p>{user.bio}</p></div> ) : ( <p>Aucune biographie</p> )} </div> ); }

✏️ Exercice : Panier d'achat conditionnel

Consigne : Créez un composant ShoppingCart qui affiche :

  • 🔹 Si le panier est vide → message "Votre panier est vide"
  • 🔹 Si le panier a des articles → afficher la liste des articles
  • 🔹 Si l'utilisateur est connecté → afficher un bouton "Passer commande"
  • 🔹 Si le total dépasse 50€ → afficher "Livraison gratuite !"
function ShoppingCart({ items, isLoggedIn, total }) { if (items.length === 0) { return <div>🛒 Votre panier est vide</div>; } return ( <div> <ul>{items.map(item => <li key={item.id}>{item.name} - {item.price}€</li>)}</ul> <p>Total : {total}€</p> {total > 50 && <p className="text-green-600">🎉 Livraison gratuite !</p>} {isLoggedIn ? ( <button className="bg-blue-500 px-4 py-2 rounded">Passer commande</button> ) : ( <p>Connectez-vous pour commander</p> )} </div> ); }

📌 Ce qu'il faut retenir

✅ if/else : à utiliser avant le return pour des conditions complexes.
✅ Opérateur ternaire : condition ? vrai : faux pour des conditions simples dans le JSX.
✅ && : parfait pour afficher "quelque chose ou rien".
⚠️ Attention aux valeurs falsy : 0, "", null avec && peuvent causer des bugs.
🎯 Le rendu conditionnel = des interfaces plus intelligentes et adaptatives !
