🔄 Le problème : les re-rendus inutiles

En React, un composant se re-rend chaque fois que son état ou ses props changent. Parfois, ces re-rendus sont inutiles et ralentissent l'application.

Exemple problématique : Un calcul coûteux (filtrage de grande liste, tri, etc.) est refait à chaque rendu, même si les données n'ont pas changé. 
// Ce calcul est refait à chaque rendu, même si 'items' n'a pas changé
const filteredItems = items.filter(item => 
    item.name.toLowerCase().includes(searchTerm.toLowerCase())
);

💾 useMemo : mémoriser une valeur

useMemo permet de mettre en cache le résultat d’un calcul coûteux et de ne le recalculer que si ses dépendances changent.

import { useMemo } from 'react';

const filteredItems = useMemo(() => {
    console.log('Calcul coûteux exécuté');
    return items.filter(item =>
        item.name.toLowerCase().includes(searchTerm.toLowerCase())
    );
}, [items, searchTerm]); // Recalcul uniquement si items ou searchTerm change
Avantage : Les calculs lourds ne sont plus exécutés à chaque rendu.

📦 useCallback : mémoriser une fonction

useCallback retourne une version mémorisée de la fonction, qui ne change que si les dépendances changent.

import { useCallback } from 'react';

// Sans useCallback : une nouvelle fonction est créée à chaque rendu
const handleClick = () => { console.log(count); };

// Avec useCallback : la fonction reste la même entre les rendus
const handleClick = useCallback(() => {
    console.log(count);
}, [count]); // Recrée la fonction uniquement si count change

Utile quand on passe des fonctions à des composants enfants optimisés avec React.memo.

⚖️ useMemo vs useCallback

useMemo

Mémorise une valeur (résultat d'un calcul).

const result = useMemo(() => 
    computeExpensiveValue(a, b), [a, b]
);

useCallback

Mémorise une fonction (l'identité de la fonction).

const fn = useCallback(() => {
    doSomething(a, b);
}, [a, b]);
Astuce : useCallback(fn, deps) équivaut à useMemo(() => fn, deps).

📝 Exemple : Filtrage de liste (useMemo)

function ProductList({ products, searchTerm }) {
    // ⚠️ Sans useMemo : recalcul à chaque rendu
    const filtered = products.filter(p =>
        p.name.includes(searchTerm)
    );

    // ✅ Avec useMemo : recalcul uniquement si products ou searchTerm change
    const filteredOptimized = useMemo(() => {
        console.log('Filtrage exécuté');
        return products.filter(p => p.name.includes(searchTerm));
    }, [products, searchTerm]);

    return <div>{filteredOptimized.map(p => <Product key={p.id} {...p} />)}</div>;
}

🎛️ Exemple : Callback évite les re-rendus enfants

function Parent() {
    const [count, setCount] = useState(0);

    // ❌ Nouvelle fonction créée à chaque rendu → Child se re-rend
    const handleClick = () => console.log('clicked');

    // ✅ Fonction mémorisée → Child ne se re-rend pas inutilement
    const handleClickMemo = useCallback(() => {
        console.log('clicked');
    }, []);

    return <Child onClick={handleClickMemo} />;
}

const Child = React.memo(({ onClick }) => {
    console.log('Child rendu');
    return <button onClick={onClick}>Click</button>;
});

⚠️ Bonnes pratiques & pièges

  • N'optimisez pas trop tôt : Commencez sans useMemo/useCallback, n'optimisez qu'en cas de problème de performance mesuré.
  • Ne mettez pas tout en useMemo : La mémorisation a un coût (mémoire). Utilisez-la pour les calculs vraiment coûteux.
  • Dépendances complètes : Toujours inclure toutes les valeurs utilisées dans le tableau de dépendances.
  • useCallback pour les props : Indispensable quand le composant enfant est enveloppé dans React.memo.
À éviter : useMemo pour des calculs triviaux (addition, concaténation simple).

🏋️ Exercice & Synthèse

À réaliser

Vous avez un composant qui affiche une liste d'utilisateurs filtrée et un compteur. À chaque clic sur le compteur, le filtre se recalcule inutilement.

  • Identifiez le problème de performance.
  • Utilisez useMemo pour mémoriser le résultat du filtre.
  • Utilisez useCallback pour qu'un bouton de réinitialisation ne provoque pas de re-rendu enfant inutile.
