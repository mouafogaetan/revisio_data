💡 Pourquoi c'est important ? useRef vous permet de manipuler le DOM directement (comme en JavaScript pur) tout en restant dans l'écosystème React, et de garder des données sans provoquer de rendus inutiles !

📦 Qu'est-ce que useRef ?

useRef est un hook React qui retourne un objet mutable avec une propriété .current. Sa valeur persiste entre les rendus sans déclencher de re-rendu lorsqu'elle est modifiée.

// Syntaxe de base

import { useRef } from 'react';

const maRef = useRef(valeurInitiale);

// Lecture : maRef.current

// Écriture : maRef.current = nouvelleValeur

✨ Caractéristiques

  • L'objet ref est le même à chaque rendu
  • Modifier .current ne cause pas de re-rendu
  • Utile pour les références DOM et les variables persistantes

⚠️ À ne pas confondre

  • useRef ≠ useState (useState déclenche un re-rendu)
  • useRef ≠ createRef (createRef est pour les classes)

🎯 Accéder au DOM avec useRef

On peut attacher une ref à un élément JSX via l'attribut ref. Ensuite, on accède à l'élément DOM avec maRef.current.

function ChampAutoFocus() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus();  // Auto-focus au montage
  }, []);

  return <input ref={inputRef} type="text" placeholder="Je suis focus automatiquement" />
}

✅ Cas d'usage fréquents : focus automatique sur un champ, mesure de la taille d'un élément, scroll vers un élément, intégration avec des bibliothèques tierces.

💾 Stocker une valeur persistante (sans re-rendu)

Contrairement à useState, modifier ref.current ne déclenche pas de nouveau rendu. Idéal pour les compteurs d'instances, timers ID, ou valeurs qui n'affectent pas l'interface.

function CompteurDeRendus() {
  const renderCount = useRef(0);
  
  // À chaque rendu, on incrémente la ref
  renderCount.current = renderCount.current + 1;
  
  return <p>Ce composant a été rendu {renderCount.current} fois</p>
}

⚠️ Important : La modification d'une ref n'affiche pas la nouvelle valeur à l'écran automatiquement. Si l'affichage doit changer, utilisez useState !

⏱️ Exemple concret : Chronomètre avec intervalle

On stocke l'ID de setInterval dans une ref pour pouvoir l'arrêter proprement.

function Chronometre() {
  const [secondes, setSecondes] = useState(0);
  const intervalRef = useRef(null);

  const demarrer = () => {
    if (intervalRef.current) return;
    intervalRef.current = setInterval(() => {
      setSecondes(prev => prev + 1);
    }, 1000);
  };

  const arreter = () => {
    clearInterval(intervalRef.current);
    intervalRef.current = null;
  };

  const reset = () => {
    arreter();
    setSecondes(0);
  };

  return (
    <div>
      <p>Temps : {secondes} secondes</p>
      <button onClick={demarrer}>Démarrer</button>
      <button onClick={arreter}>Arrêter</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
}

⚖️ useRef vs useState

CritèreuseRefuseState
Déclenche un re-rendu❌ Non✅ Oui
Valeur persistante✅ Oui✅ Oui
Mise à jour asynchrone❌ Non (synchrone)✅ Oui (batch)
Utilité principaleRéférences DOM, valeurs sans renduDonnées qui affectent l'UI

💡 Règle simple : Si la modification doit changer ce que l'utilisateur voit → useState. Si la modification est interne (timer ID, compteur de rendu, référence DOM) → useRef.

📜 Scroll vers un élément

Utilisez useRef pour faire défiler la page jusqu'à un élément spécifique.

function ScrollDemo() {
  const sectionRef = useRef(null);

  const scrollToSection = () => {
    sectionRef.current?.scrollIntoView({ behavior: 'smooth' });
  };

  return (
    <div>
      <button onClick={scrollToSection}>Aller à la section</button>
      <div style={{ height: '100vh' }}></div>
      <div ref={sectionRef} className="bg-blue-100 p-4">
        🎯 Je suis la section cible !
      </div>
    </div>
  );
}

📌 Récapitulatif et bonnes pratiques

  • • useRef(initialValue) retourne { current: initialValue }
  • • Attribuez la ref à un élément JSX avec ref={maRef}
  • • Modifiez .current directement (pas de fonction setter)
  • • Ne pas utiliser useRef pour des données qui doivent s'afficher à l'écran
  • • Parfait pour : focus, scroll, timers, compteurs de rendu, intégrations tierces
