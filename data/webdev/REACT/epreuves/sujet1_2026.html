Partie 2 : Vrai ou Faux (10 pts)

Les props sont mutables à l’intérieur du composant enfant.
L'attribut `key` dans les listes aide React à identifier les éléments modifiés.
useState ne peut stocker que des chaînes de caractères ou des nombres.
Une SPA (Single Page Application) recharge entièrement la page à chaque clic sur un lien.
En JSX, on utilise `class` pour définir les classes CSS, comme en HTML.

Partie 3 : Mise en pratique (70 pts)

Exercice 1(20 pts) – Composant Produit et Props

Créez un composant fonctionnel ProductItem qui reçoit via props : nom, prix et disponible (booléen). Il doit afficher une carte avec le nom, le prix et une indication “En stock” ou “Rupture”. Puis dans le composant principal App, affichez une liste d’au moins trois produits différents en passant les props.

// Exemple de structure attendu (pseudo-code)
function ProductItem({ nom, prix, disponible }) {
  return (
    <div>
      <h3>{nom}</h3>
      <p>Prix : {prix} FCFA</p>
      <p>{disponible ? "✅ En stock" : "❌ Rupture"}</p>
    </div>
  );
}

✔️ Évaluation : bonne destructuration, rendu conditionnel, réutilisabilité.

Exercice 2(25 pts) – Compteur interactif avec useState

Implémentez un composant Counter qui utilise le hook useState pour gérer un compteur. Ajoutez trois boutons : Incrémenter, Décrémenter, Réinitialiser. Affichez la valeur du compteur en grand. Gérez le cas où le compteur ne passe pas en dessous de 0.

🎯 Objectif : maîtriser la mise à jour d’état, immutabilité, gestionnaires d’événements.
Aide rapide (squelette) 
const [count, setCount] = useState(0);
const increment = () => setCount(prev => prev + 1);
Exercice 3(25 pts) – Liste d'étudiants et rendu conditionnel

À partir d'un tableau d'objets (nom, age, moyenne), affichez une liste non ordonnée des étudiants. Pour chaque étudiant : si sa moyenne est ≥ 12, ajoutez un badge "Admis", sinon "Non admis". Utilisez map() et une key correcte (id ou nom+age).

Données fournies :

const students = [ { id: 1, nom: "Fatima", age: 20, moyenne: 15 }, { id: 2, nom: "Claude", age: 22, moyenne: 10 }, { id: 3, nom: "Yannick", age: 19, moyenne: 13 } ];

💡 N'oubliez pas l'attribut key unique et la gestion des opérateurs logiques pour le badge.
