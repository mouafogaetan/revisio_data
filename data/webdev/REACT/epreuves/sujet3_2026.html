Partie II

Analyse de code – Effets & État (5 pts)

function ProductPage() {
  const [product, setProduct] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    fetch('https://api.example.com/product/42')
      .then(res => res.json())
      .then(data => setProduct(data))
      .catch(err => setError(err.message))
      .finally(() => setLoading(false));
  }, []); // Dépendances vides

  if (loading) return <div>Chargement...</div>;
  if (error) return <div>Erreur : {error}</div>;
  return <div>{product?.name}</div>;
}

a) (1.5 pt) Expliquez le rôle du tableau de dépendances [] dans ce code. À quel moment la requête fetch est-elle exécutée ?

Partie III

Exercice pratique – To-Do List avec gestion d’effets (9 pts)

🧪 Contexte :

Vous devez développer un composant TaskManager qui gère une liste de tâches. Les tâches sont stockées dans le localStorage pour persister après rafraîchissement. Vous utiliserez useState pour l'état local et useEffect pour synchroniser avec le stockage. Le composant doit permettre d'ajouter, basculer le statut (complétée / active) et supprimer une tâche.

❶ (2 pts) Écrivez la structure de l'état et le chargement initial depuis localStorage avec useState + useEffect. Assurez-vous de gérer le cas où localStorage est vide. (Extrait de code commenté)

❷ (2.5 pts) Implémentez la fonction addTask qui respecte l'immutabilité. Chaque tâche doit avoir un id unique, un text et une propriété completed (booléenne). Montrez le JSX pour le formulaire simple (input + bouton).

❸ (2 pts) Écrivez le rendu de la liste avec map() et la gestion de la suppression (filter). Chaque élément doit afficher une checkbox pour toggle completed et un bouton supprimer.

❹ (2.5 pts) Expliquez comment vous utiliseriez useEffect pour sauvegarder automatiquement les tâches dans localStorage à chaque modification. Discutez des dépendances et des performances (sauvegarde trop fréquente).

✨ Rappel des règles des hooks : Les hooks doivent être appelés uniquement au niveau racine du composant et jamais dans des conditions, boucles ou fonctions imbriquées [citation:3][citation:7]. Bon courage ! 🚀
