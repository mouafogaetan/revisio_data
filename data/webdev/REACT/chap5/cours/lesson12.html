Les événements React

Leçon 12 – Rendre l'application interactive

Apprenez à capturer les clics, les changements de formulaire et à rendre vos composants réactifs.

🖱️ onClick 📝 onChange 📬 onSubmit

Objectifs de la leçon

  • ✓ Utiliser les événements React : onClick, onChange, onSubmit.
  • ✓ Comprendre la différence entre événements React et événements DOM natifs.
  • ✓ Passer des arguments aux gestionnaires d'événements.
💡 À la fin, vous saurez rendre vos composants React interactifs comme des applications modernes.

🎯 Les événements, c'est quoi ?

Les événements permettent à l'utilisateur d'interagir avec votre application : cliquer, taper du texte, soumettre un formulaire, etc.

📌 Différence clé avec HTML classique :

En HTML : <button onclick="alert('Clic!')">
En React : <button onClick={handleClick}>

⚠️ En React, on utilise camelCase (onClick et non onclick) et on passe une fonction entre accolades, pas une chaîne de caractères.

// Exemple simple
function MonBouton() {
  function handleClick() {
    alert('Bouton cliqué !');
  }
  return <button onClick={handleClick}>Clique-moi</button>;
}

📋 Les 3 événements incontournables

🖱️ onClick

Déclenché quand on clique sur un élément (bouton, div, lien, etc.).

<button onClick={() => console.log('clic')}>OK</button>

✏️ onChange

Déclenché à chaque modification d'un champ de formulaire (input, textarea, select).

<input onChange={(e) => setValeur(e.target.value)} />

📬 onSubmit

Déclenché quand un formulaire est soumis (touche Entrée ou clic sur bouton submit).

<form onSubmit={handleSubmit}> ... </form>

🧪 L'objet SyntheticEvent

React encapsule l'événement natif du navigateur dans un objet appelé SyntheticEvent. Il a la même interface que l'événement DOM, mais fonctionne de manière identique sur tous les navigateurs.

🔍 Exemple d'accès aux informations : 
function handleClick(e) {
  console.log(e.target);      // L'élément qui a reçu le clic
  console.log(e.currentTarget); // L'élément auquel l'événement est attaché
  e.preventDefault();         // Empêche le comportement par défaut (ex: rechargement)
  e.stopPropagation();        // Empêche la propagation de l'événement
}

⚠️ En React 17+, le pooling (réutilisation des objets événements) a été supprimé. Vous pouvez accéder à l'événement de manière asynchrone sans souci.

📨 Passer des arguments à un gestionnaire

Parfois, on a besoin de savoir quel élément a été cliqué (ex: l'id d'un produit).

Méthode 1 : fonction fléchée

<button onClick={() => supprimer(id)}>
  Supprimer
</button>

Méthode 2 : .bind

<button onClick={supprimer.bind(this, id)}>
  Supprimer
</button>

✨ Exemple concret avec une liste :

function ListeTaches({ taches, onSupprimer }) {
  return taches.map(tache => (
    <li key={tache.id}>
      {tache.nom}
      <button onClick={() => onSupprimer(tache.id)}>X</button>
    </li>
  ));
}

⚖️ React vs DOM natif

CaractéristiqueDOM natifReact
Nommageonclick (minuscules)onClick (camelCase)
ValeurChaîne de caractères ou fonctionFonction JavaScript (entre {})
Objet événementObjet natif, dépend du navigateurSyntheticEvent (normalisé)
Comportement par défautreturn false ou preventDefaultToujours appeler preventDefault() explicitement
💡 Bon à savoir : return false en React ne bloque pas le comportement par défaut. Utilisez toujours e.preventDefault().

📝 Exemple concret : formulaire géré

function FormulaireContact() {
  const [nom, setNom] = useState('');
  const [email, setEmail] = useState('');

  function handleSubmit(e) {
    e.preventDefault(); // empêche le rechargement
    alert(`Bienvenue ${nom}, email: ${email}`);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        placeholder="Votre nom"
        value={nom}
        onChange={(e) => setNom(e.target.value)}
      />
      <input
        type="email"
        placeholder="Email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
      />
      <button type="submit">Envoyer</button>
    </form>
  );
}
🧪 À vous d'essayer : ajoutez un bouton "Réinitialiser" avec onClick qui vide les champs.

💡 Bonnes pratiques

  • Définissez vos gestionnaires en dehors du JSX pour une meilleure lisibilité.
  • Utilisez des fonctions fléchées dans les props uniquement si vous devez passer des arguments. Sinon, passez directement la référence.
  • Pour éviter des recréations de fonctions inutiles, utilisez useCallback (vu plus tard).
  • N'oubliez jamais d'appeler e.preventDefault() pour les formulaires.
⚠️ Anti-pattern : <button onClick={handleClick()} /> appelle la fonction immédiatement au rendu ! Il faut passer la fonction, pas son résultat.

🧪 Exercice rapide

Créez un composant Compteur avec :

  • Un état count initialisé à 0.
  • Un bouton "Incrémenter" qui ajoute 1.
  • Un bouton "Décrémenter" qui enlève 1.
  • Un bouton "Réinitialiser" qui remet à 0.
📌 Voir la solution 
function Compteur() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>Compteur : {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
      <button onClick={() => setCount(count - 1)}>-</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}
