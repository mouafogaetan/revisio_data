🧩 SyntheticEvent : l'événement universel

En React, vous ne manipulez pas directement l'événement natif du navigateur (MouseEvent, KeyboardEvent…). Vous recevez un objet SyntheticEvent.

  • Il a la même interface que l'événement DOM natif (mêmes méthodes preventDefault, stopPropagation).
  • Il fonctionne de manière identique sur tous les navigateurs (Chrome, Firefox, Safari, Edge).
  • React réutilise ces objets pour des raisons de performance (pooling dans React < 17).
function handleClick(e) {
  console.log(e); // SyntheticEvent
  console.log(e.nativeEvent); // Événement natif du navigateur
}

🎯 target vs currentTarget

Deux propriétés essentielles pour comprendre la propagation :

e.target : l'élément qui a déclenché l'événement (le plus profond).

e.currentTarget : l'élément sur lequel l'écouteur est attaché (toujours l'élément du onClick).

function Parent() {
  return (
    <div onClick={(e) => console.log('currentTarget:', e.currentTarget)}>
      <button onClick={(e) => console.log('target:', e.target)}>Clique</button>
    </div>
  );
}
// Cliquer sur le bouton affiche : target: <button>, currentTarget: <div>

🚫 preventDefault()

Empêche le comportement par défaut du navigateur.

  • Empêcher la soumission d'un formulaire (rechargement de page).
  • Empêcher la navigation d'un lien <a href>.
  • Empêcher la sélection de texte, etc.
function Form() {
  const handleSubmit = (e) => {
    e.preventDefault(); // Plus de rechargement !
    console.log('Formulaire soumis sans rechargement');
  };
  return <form onSubmit={handleSubmit}><button>Envoyer</button></form>;
}

⛔ stopPropagation()

Empêche l'événement de remonter (bubbling) vers les parents.

Exemple :

function Parent() {
  const onParentClick = () => alert('Parent cliqué');
  const onChildClick = (e) => {
    e.stopPropagation(); // Le parent ne reçoit pas l'événement !
    alert('Enfant cliqué');
  };
  return <div onClick={onParentClick}>
           <button onClick={onChildClick}>Clique moi</button>
         </div>;
}

⏳ Pooling des événements

Avant React 17 : les objets SyntheticEvent étaient réutilisés (pooling). L'événement devenait null après le callback, impossible de le lire de manière asynchrone.

// Ancien comportement (React < 17) — à éviter !
function handleClick(e) {
  setTimeout(() => {
    console.log(e.target); // Erreur : e est null (réinitialisé)
  }, 100);
}
// Solution : e.persist() pour retirer l'événement du pool.

✅ Depuis React 17 : plus de pooling. Les événements sont stables, vous pouvez les utiliser de manière asynchrone sans e.persist().

🔄 Comparaison : natif vs synthétique

Événement DOM natif

– Pas de normalisation entre navigateurs
– Objet persistant
– Ajout avec addEventListener

SyntheticEvent

✓ Normalisé tous navigateurs
✓ (Historique) pooling avant 17
✓ Mêmes méthodes (preventDefault…)

Accéder à l'événement natif : e.nativeEvent

✨ Bonnes pratiques

  • Toujours appeler e.preventDefault() pour les formulaires si vous ne voulez pas recharger.
  • Utiliser stopPropagation() avec parcimonie (peut casser des comportements attendus).
  • En React moderne (>17), pas besoin de e.persist(), les événements sont stables.
  • Utiliser e.target pour lire ce que l'utilisateur a tapé (onChange).
  • Utiliser e.currentTarget pour se référer à l'élément qui porte l'écouteur.

📝 Exemple concret : formulaire avec SyntheticEvent

import { useState } from 'react';

function MonFormulaire() {
  const [nom, setNom] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();          // évite rechargement
    console.log('Nom soumis :', nom);
  };

  const handleChange = (e) => {
    setNom(e.target.value);      // target contient l'input
  };

  return (
    <form onSubmit={handleSubmit}>
      <input value={nom} onChange={handleChange} />
      <button type="submit">Envoyer</button>
    </form>
  );
}

⚠️ Pièges fréquents

❌ Oublier preventDefault : le formulaire recharge la page.

❌ Utiliser stopPropagation partout : rend le code fragile et peut bloquer des librairies externes.

❌ Dans React <17, stocker l'événement pour un setTimeout : il devient null sans e.persist().

✅ Bon réflexe : si vous avez besoin de l'événement de façon asynchrone, utilisez e.persist() (anciennes versions) ou mettez à jour React !

🧪 Mini exercice

Codez un composant React contenant :

  • Un bouton "Alerte" qui affiche un message alert() sans comportement supplémentaire.
  • Un lien <a href="https://google.com"> qui n'ouvre pas Google mais affiche un message dans la console (utilisez preventDefault).
  • Deux divs imbriquées : cliquer sur la div enfant ne doit pas déclencher le parent (utilisation de stopPropagation).
🔎 Voir solution 
function Exercice() {
  const handleChildClick = (e) => {
    e.stopPropagation();
    alert('Enfant seul');
  };
  const handleParentClick = () => alert('Parent');
  const handleLink = (e) => {
    e.preventDefault();
    console.log('Lien bloqué');
  };
  return (
    <div>
      <button onClick={() => alert('Bouton')}>Alerte</button>
      <a href="https://google.com" onClick={handleLink}>Lien bloqué</a>
      <div onClick={handleParentClick} style={{padding:20,background:'lightgray'}}>
        Parent
        <div onClick={handleChildClick} style={{padding:10,background:'white'}}>
          Enfant (stopPropagation)
        </div>
      </div>
    </div>
  );
}
