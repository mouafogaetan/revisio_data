“Une application React rapide ne rend que ce qui a vraiment changé.”

🤔 Pourquoi optimiser React ?

Par défaut, quand un composant parent se re-rend, TOUS ses enfants se re-rendent aussi, même si leurs props n'ont pas changé. Cela peut causer des problèmes de performance dans les applications complexes.

⚠️ Problème classique :

Une liste de 1000 éléments qui se re-rend entièrement à chaque saisie dans un champ de recherche, alors que seul le filtre change.

💡 Solution :

React.memo, useMemo et useCallback permettent de contrôler quand un composant doit se re-rendre.

📦 React.memo - Composant mémoïsé

React.memo est une fonction d'ordre supérieur (HOC) qui empêche un composant de se re-rendre si ses props n'ont pas changé (comparaison superficielle).

// Sans memo : se re-rend toujours
const MaListe = ({ items }) => (
  <ul>{items.map(item => <li key={item.id}>{item.nom}</li>)}</ul>
);

// Avec memo : re-rend uniquement si items change
const MaListeMemo = React.memo(({ items }) => (
  <ul>{items.map(item => <li key={item.id}>{item.nom}</li>)}</ul>
));
💡 Astuce : Exportez directement : export default React.memo(MonComposant);

🧮 useMemo - Mémoriser des valeurs calculées

useMemo mémorise le résultat d'un calcul coûteux et ne le recalcule que lorsque ses dépendances changent.

import { useMemo, useState } from 'react';

function ComposantLourd() {
  const [compteur, setCompteur] = useState(0);
  const [texte, setTexte] = useState('');
  
  // Calcul coûteux : filtrer 10 000 éléments
  const elementsFiltres = useMemo(() => {
    console.log('Recalcul coûteux...');
    return grosTableau.filter(el => el.includes(texte));
  }, [texte]); // Recalcule uniquement quand 'texte' change
  
  return (
    <div>
      <input value={texte} onChange={(e) => setTexte(e.target.value)} />
      <p>Compteur : {compteur}</p>
      <button onClick={() => setCompteur(c => c+1)}>Inc</button>
    </div>
  );
}

🔄 useCallback - Mémoriser des fonctions

useCallback est similaire à useMemo mais pour les fonctions. Il retourne une version mémoïsée de la fonction qui ne change que si les dépendances changent.

import { useCallback, useState } from 'react';

function Parent() {
  const [compteur, setCompteur] = useState(0);
  
  // Cette fonction ne change pas entre les rendus
  const handleClick = useCallback(() => {
    console.log('Clic reçu');
  }, []); // Dépendances vides = fonction stable
  
  return (
    <div>
      <button onClick={() => setCompteur(c => c+1)}>Inc ({compteur})</button>
      <Enfant onClick={handleClick} />
    </div>
  );
}

// React.memo empêche le re-rendu d'Enfant si ses props ne changent pas
const Enfant = React.memo(({ onClick }) => (
  <button onClick={onClick}>Enfant</button>
));

📊 Quand utiliser quoi ?

🎯 React.memo

Pour composants enfants qui reçoivent les mêmes props et se re-rendent trop souvent.

Ex: Liste, Card, Ligne de tableau

🧮 useMemo

Pour valeurs calculées coûteuses (tri, filtrage, agrégation).

Ex: total, liste filtrée, transformation de données

🔄 useCallback

Pour fonctions passées à des enfants mémoïsés.

Ex: gestionnaires d'événements, callbacks API

⚠️ Attention

N'optimisez pas prématurément ! Mesurez d'abord avec React DevTools.

🔍 Outils de profiling

React DevTools (extension navigateur) permet de visualiser les re-rendus et d'identifier les composants à optimiser.

⚙️ Comment faire :

  • Installer l'extension React DevTools
  • Onglet "Profiler" → Démarrer l'enregistrement
  • Effectuer des actions dans l'application
  • Arrêter et analyser les re-rendus colorés
💡 Règle d'or : N'optimisez que quand vous avez un problème de performance mesuré. Une optimisation inutile peut complexifier le code.
