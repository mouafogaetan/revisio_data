Manipulation des tableaux et objets

L'immutabilité dans React avec useState

Ajouter • Modifier • Supprimer • Spread operator • Immer

Niveau: Intermédiaire Durée: 50 min

🎯 Objectifs de la leçon

  • Ajouter, modifier et supprimer des éléments dans un tableau en respectant l'immutabilité.
  • Mettre à jour des objets avec l'opérateur spread (...).
  • Comprendre pourquoi l'immutabilité est cruciale dans React.
  • Utiliser Immer pour simplifier les mises à jour complexes.
À la fin, vous saurez gérer des listes dynamiques (todo list, panier d'achat) sans erreurs !

🧠 Pourquoi l'immutabilité ?

En React, on ne modifie JAMAIS directement l'état (state). On crée une nouvelle copie avec les modifications.

Mauvais exemple (mutation directe) :

// ❌ Ne faites JAMAIS ça !
const [users, setUsers] = useState(['Alice', 'Bob']);
users.push('Charlie');  // Mutation directe
setUsers(users);        // React ne détecte pas le changement

Bon exemple (immuable) :

// ✅ Création d'un nouveau tableau
const [users, setUsers] = useState(['Alice', 'Bob']);
setUsers([...users, 'Charlie']); // Nouveau tableau

React compare les références des objets. Si la référence n'a pas changé, le composant ne se re-rend pas !

➕ Ajouter un élément dans un tableau

Utiliser l'opérateur spread ... pour créer un nouveau tableau.

const [todos, setTodos] = useState(['Apprendre React', 'Faire les exercises']);

// Ajouter un élément à la fin
const addTodo = (nouveauTodo) => {
  setTodos([...todos, nouveauTodo]);
};

// Ajouter un élément au début
const addTodoAtStart = (nouveauTodo) => {
  setTodos([nouveauTodo, ...todos]);
};

Exemple concret :

Tableau initial : ['A', 'B']
Après setTodos([...todos, 'C']) → ['A', 'B', 'C']
Après setTodos(['Z', ...todos]) → ['Z', 'A', 'B']

❌ Supprimer un élément d'un tableau

Utiliser filter() qui retourne un nouveau tableau.

const [todos, setTodos] = useState(['A', 'B', 'C']);

// Supprimer l'élément avec l'index 1 ('B')
const deleteTodo = (indexASupprimer) => {
  setTodos(todos.filter((_, index) => index !== indexASupprimer));
};

// Supprimer par valeur
const deleteByValue = (valeur) => {
  setTodos(todos.filter(item => item !== valeur));
};

Exemple :

['A', 'B', 'C'] → suppression de l'index 1 → ['A', 'C']

✏️ Modifier un élément spécifique

Utiliser map() pour transformer le tableau.

const [todos, setTodos] = useState(['A', 'B', 'C']);

// Remplacer l'élément à l'index 1 par 'X'
const updateTodo = (index, nouvelleValeur) => {
  setTodos(todos.map((item, i) => 
    i === index ? nouvelleValeur : item
  ));
};

Résultat :

['A', 'B', 'C'] → modification index 1 en 'X' → ['A', 'X', 'C']

📦 Mettre à jour un objet

Utiliser le spread operator pour conserver les propriétés existantes.

const [user, setUser] = useState({ name: 'Alice', age: 25, city: 'Douala' });

// Modifier uniquement l'âge
const updateAge = () => {
  setUser({ ...user, age: 26 });
};

// Modifier plusieurs propriétés
const updateProfile = () => {
  setUser({ ...user, age: 27, city: 'Yaoundé' });
};
Attention : Les objets imbriqués nécessitent un spread à chaque niveau !

🪆 Objets imbriqués (nested objects)

const [state, setState] = useState({
  user: { name: 'John', address: { city: 'Douala', street: 'Avenue Kennedy' } }
});

// Modifier une propriété imbriquée
const updateCity = () => {
  setState({
    ...state,
    user: {
      ...state.user,
      address: {
        ...state.user.address,
        city: 'Kribi'
      }
    }
  });
};

Plus l'objet est profond, plus le code devient lourd → solution : Immer !

🪄 Immer : la simplicité retrouvée

Immer permet d'écrire des mutations comme si on modifiait directement l'état, mais crée automatiquement une copie immuable.

import { produce } from 'immer';

const [state, setState] = useState({
  user: { name: 'John', address: { city: 'Douala' } },
  todos: ['A', 'B']
});

// Avec Immer : on "modifie" directement !
const updateImmer = () => {
  setState(produce(draft => {
    draft.user.address.city = 'Yaoundé';
    draft.todos.push('C');
  }));
};
Installation : npm install immer

📋 Tableau d'objets (exemple Todo)

const [todos, setTodos] = useState([
  { id: 1, text: 'Apprendre React', done: false },
  { id: 2, text: 'Maîtriser Immer', done: false }
]);

// Ajouter
setTodos([...todos, { id: Date.now(), text: newTodo, done: false }]);

// Basculer done (toggle)
const toggleTodo = (id) => {
  setTodos(todos.map(todo =>
    todo.id === id ? { ...todo, done: !todo.done } : todo
  ));
};

// Supprimer
const deleteTodo = (id) => {
  setTodos(todos.filter(todo => todo.id !== id));
};

⚖️ Avec Immer vs Sans Immer

Sans Immer (spread)

setState({
  ...state,
  user: {
    ...state.user,
    address: {
      ...state.user.address,
      city: 'Kribi'
    }
  }
});

Avec Immer

setState(produce(draft => {
  draft.user.address.city = 'Kribi';
}));

Pour les structures profondes, Immer réduit le code de 70% et élimine les erreurs de spread oublié !

📌 Récapitulatif & Bonnes pratiques

  • Ne jamais muter directement un état (state.push(), state.age = 30)
  • Tableaux : [...ancien, nouveau], filter(), map()
  • Objets : { ...ancien, propriete: nouvelleValeur }
  • Objets imbriqués : spread à chaque niveau ou Immer
  • Pour les projets complexes : adopter Immer pour la lisibilité

Règle d'or : Toujours créer une NOUVELLE référence pour informer React du changement !
