“L'état (state) est la mémoire d'un composant React.”

🔍 useState : La mémoire du composant

useState est un Hook React qui permet d'ajouter un état réactif à un composant fonctionnel. Chaque fois que l'état change, React re-rend le composant automatiquement.

// Syntaxe de base
import { useState } from 'react';

function MonComposant() {
  const [valeur, setValeur] = useState(0); // 0 est la valeur initiale
  
  const incrementer = () => {
    setValeur(valeur + 1); // met à jour l'état
  };
  
  return <button onClick={incrementer}>{valeur}</button>
}
💡 Règle d'or : Ne modifiez jamais l'état directement (ex: valeur = 5). Utilisez toujours la fonction setter setValeur(...).

📦 useState avec plusieurs types de données

🔢 String / Number

const [nom, setNom] = useState("Jean");
const [age, setAge] = useState(25);

✅ Boolean

const [estConnecte, setEstConnecte] = useState(false);

📋 Tableau (Array)

const [todos, setTodos] = useState(["Apprendre React", "Créer un projet"]);

🧩 Objet

const [user, setUser] = useState({ name: "Alice", role: "dev" });

✨ Astuce : Pour les objets/tableaux, créez toujours une nouvelle copie pour conserver l'immutabilité.

⏳ Mise à jour asynchrone & prevState

La fonction setState est asynchrone. Si une nouvelle valeur dépend de l'ancienne, utilisez la forme fonctionnelle :

// ❌ À éviter (si plusieurs mises à jour rapides)
setCount(count + 1);
setCount(count + 1); // ne fonctionne pas comme prévu

// ✅ Version fonctionnelle (recommandée)
setCount(prevCount => prevCount + 1);
setCount(prevCount => prevCount + 1); // incrémente deux fois
🧠 Comprendre : prevCount représente l'état précédent garanti. Utile pour les compteurs, les toggles ou les calculs complexes.

⚠️ Pièges fréquents & conseils

  • Ne pas appeler useState conditionnellement – Les hooks doivent toujours être appelés au même ordre (en haut du composant).
  • Les setteurs ne fusionnent pas les objets : pour un objet, utilisez setUser({ ...user, nouveauChamp: valeur }).
  • State initial coûteux : si la valeur initiale nécessite un calcul long, passez une fonction : useState(() => calculLong()).
  • Plusieurs useState : vous pouvez utiliser plusieurs appels à useState dans un même composant.
🔥 Rappel : L'état est local à chaque composant. Deux composants identiques auront leurs propres états indépendants.

✏️ Mini-exercice

Créez mentalement un composant React avec :

  • Un état message initialisé à "Bonjour !"
  • Un état compteur initialisé à 0
  • Une fonction qui modifie message en "Nouveau message" et incrémente le compteur en une seule action.
const [message, setMessage] = useState('Bonjour !');
const [compteur, setCompteur] = useState(0);

const handleClick = () => {
  setMessage('Nouveau message');
  setCompteur(c => c + 1);
};
