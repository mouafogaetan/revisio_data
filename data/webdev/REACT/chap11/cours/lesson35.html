🏗️ Architecture d'un projet React professionnel

src/
├── components/       # Composants réutilisables (Button, Card, Modal...)
├── pages/           # Pages de l'application (Home, About, Dashboard...)
├── contexts/        # Contextes globaux (AuthContext, ThemeContext...)
├── hooks/           # Hooks personnalisés (useFetch, useLocalStorage...)
├── services/        # Appels API (api.js, authService.js...)
├── utils/           # Fonctions utilitaires (formatters, validators...)
├── styles/          # Fichiers CSS globaux
├── App.jsx          # Composant principal avec les routes
├── main.jsx         # Point d'entrée avec les providers
└── index.css        # Styles globaux

✅ Avantage : Cette structure facilite la maintenance, la collaboration et l'évolution du projet.

🧭 React Router + Context : Application exemple

// main.jsx - Configuration des providers
import { BrowserRouter } from 'react-router-dom';
import { AuthProvider } from './contexts/AuthContext';
import { ThemeProvider } from './contexts/ThemeContext';

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <AuthProvider>
      <ThemeProvider>
        <App />
      </ThemeProvider>
    </AuthProvider>
  </BrowserRouter>
);

// App.jsx - Routes
import { Routes, Route, Navigate } from 'react-router-dom';
import { useAuth } from './contexts/AuthContext';

function App() {
  const { user } = useAuth();
  return (
    <Routes>
      <Route path="/login" element={<Login />} />
      <Route path="/" element={user ? <Dashboard /> : <Navigate to="/login" />} />
      <Route path="/products" element={<ProductList />} />
      <Route path="/products/:id" element={<ProductDetail />} />
    </Routes>
  );
}

⚡ Consommer une API avec React Query

// Installation
// npm install @tanstack/react-query

// main.jsx - Ajouter le provider
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

const queryClient = new QueryClient();

<QueryClientProvider client={queryClient}>
  <App />
</QueryClientProvider>

// Composant qui utilise l'API
import { useQuery, useMutation } from '@tanstack/react-query';

function Products() {
  const { data: products, isLoading, error } = useQuery({
    queryKey: ['products'],
    queryFn: () => fetch('/api/products').then(res => res.json())
  });
  
  const mutation = useMutation({
    mutationFn: (newProduct) => fetch('/api/products', {
      method: 'POST',
      body: JSON.stringify(newProduct)
    }).then(res => res.json())
  });
  
  if (isLoading) return <p>Chargement...</p>;
  if (error) return <p>Erreur : {error.message}</p>;
  
  return <div>{/* Affichage des produits */}</div>
}

🚀 Déploiement sur Vercel (le plus simple !)

1
Installer Vercel CLI : npm i -g vercel
2
Se connecter : vercel login
3
Déployer : vercel (suivre les instructions)
4
Alternative sans CLI : Connecter votre dépôt GitHub sur vercel.com

🎉 Résultat : Une URL publique du type https://mon-projet.vercel.app

🌐 Déploiement sur Netlify (drag & drop)

1
Générer le build : npm run build (crée un dossier dist/ ou build/)
2
Aller sur netlify.com et créer un compte
3
Glisser-déposer le dossier build/ sur l'interface Netlify
4
Option : Connecter GitHub pour déploiement automatique

✅ Checklist avant déploiement

  • ✓ Variables d'environnement configurées (VITE_API_URL ou REACT_APP_API_URL)
  • ✓ Aucune erreur dans la console (nettoyer les warnings)
  • ✓ Toutes les routes fonctionnent (pas de page 404)
  • ✓ L'application est responsive (mobile friendly)
  • ✓ Les tokens/identifiants ne sont pas en clair dans le code
  • ✓ Le fichier .gitignore exclut node_modules/ et .env

⚠️ Attention : Vérifiez que votre API de développement n'est pas accessible en production ! Utilisez des variables d'environnement différentes.
