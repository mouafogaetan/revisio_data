🧭 React Router DOM : La navigation dans React

React Router DOM est la bibliothèque standard pour gérer la navigation dans les applications React. Elle permet de créer des applications multi-pages tout en restant une Single Page Application (SPA) : pas de rechargement de la page, uniquement le contenu qui change.

📦 Installation

npm install react-router-dom

Une fois installé, vous pouvez créer des routes et naviguer entre vos composants.

⚙️ BrowserRouter vs HashRouter

🌐 BrowserRouter

Utilise l'API HTML5 History (URL propres : /about)

✅ Recommandé pour la production
⚠️ Nécessite configuration serveur

#️⃣ HashRouter

Utilise le fragment d'URL (ex: /#/about)

✅ Fonctionne sans config serveur
⚠️ URLs moins esthétiques

💡 Conseil :

En développement, les deux fonctionnent. Pour un site en production sur GitHub Pages ou sans accès serveur, utilisez HashRouter.

🔧 Configuration de base

Dans votre fichier main.jsx ou App.jsx, vous devez envelopper l'application avec le Router choisi :

import { BrowserRouter } from 'react-router-dom';

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
);
⚠️ Important : Tous les composants qui utilisent les hooks de React Router (comme useNavigate) doivent être à l'intérieur du Router.

🗺️ Définir des Routes (v6)

Dans React Router v6, on utilise les composants Routes et Route :

import { Routes, Route } from 'react-router-dom';
import Accueil from './pages/Accueil';
import APropos from './pages/APropos';
import Contact from './pages/Contact';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Accueil />} />
      <Route path="/apropos" element={<APropos />} />
      <Route path="/contact" element={<Contact />} />
    </Routes>
  );
}
✨ Note : L'attribut element remplace l'ancien component ou render des versions antérieures.

🔗 Navigation avec Link et NavLink

Au lieu des balises <a href> qui rechargent la page, utilisez <Link> ou <NavLink> :

import { Link, NavLink } from 'react-router-dom';

function Menu() {
  return (
    <nav>
      <Link to="/">Accueil</Link>
      <Link to="/apropos">À propos</Link>
      
      <NavLink 
        to="/contact" 
        className={({ isActive }) => isActive ? "active" : ""}
      >
        Contact
      </NavLink>
    </nav>
  );
}
💡 NavLink ajoute automatiquement une classe active quand l'URL correspond, idéal pour les menus.

📱 Exemple complet : Mini-site

// App.jsx
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function Accueil() { return <h2>🏠 Bienvenue</h2>; }
function Services() { return <h2>⚙️ Nos services</h2>; }
function Contact() { return <h2>📞 Contactez-nous</h2>; }

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Accueil</Link> | 
        <Link to="/services">Services</Link> | 
        <Link to="/contact">Contact</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Accueil />} />
        <Route path="/services" element={<Services />} />
        <Route path="/contact" element={<Contact />} />
      </Routes>
    </BrowserRouter>
  );
}
🚀 Le résultat : une navigation fluide sans rechargement de page !
