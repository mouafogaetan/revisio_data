🧩 Routes dynamiques avec :

Dans React Router, on définit un paramètre d'URL avec deux-points :.

<Route path="/utilisateur/:id" element={<Profil />} />
<Route path="/article/:slug" element={<Article />} />

Exemples d'URL capturées :

  • /utilisateur/42 → id = "42"
  • /utilisateur/alice123 → id = "alice123"
  • /article/ma-super-recette → slug = "ma-super-recette"

📖 useParams()

Le hook useParams retourne un objet contenant tous les paramètres dynamiques de l'URL courante.

import { useParams } from 'react-router-dom';

function Profil() {
  const { id } = useParams();   // id correspond au :id de la route
  return <h1>Profil de l'utilisateur {id}</h1>;
}

💡 Plusieurs paramètres :

/produit/:categorie/:id → useParams() donne { categorie: "...", id: "..." }

🧭 useNavigate()

Le hook useNavigate retourne une fonction pour changer d'URL de manière programmatique (après un clic, une soumission de formulaire, etc.).

import { useNavigate } from 'react-router-dom';

function Login() {
  const navigate = useNavigate();

  const handleSubmit = () => {
    // ... vérification identifiants
    navigate('/dashboard');      // redirection simple
  };

  return <form onSubmit={handleSubmit}>...</form>;
}

⚙️ Options de navigate

// Remplacer l'entrée courante (pas d'ajout dans l'historique)
navigate('/accueil', { replace: true });

// Reculer / avancer dans l'historique
navigate(-1);   // retour en arrière
navigate(1);    // avancer

// Avec paramètres d'URL (dynamiques)
navigate(`/utilisateur/${userId}`);

// Avec query string (voir plus loin)
navigate('/recherche?q=react&page=2');
✅ useNavigate remplace useHistory de React Router v5.

🔍 useSearchParams()

Permet de lire et modifier les paramètres de requête (query string) après le ? dans l'URL.

import { useSearchParams } from 'react-router-dom';

function Recherche() {
  const [searchParams, setSearchParams] = useSearchParams();
  const q = searchParams.get('q') || '';    // lecture
  const page = searchParams.get('page') || 1;

  const handleSearch = (term) => {
    setSearchParams({ q: term, page: 1 });  // mise à jour
  };

  return <div>...</div>;
}

📄 Exemple : liste paginée

function Produits() {
  const [searchParams, setSearchParams] = useSearchParams();
  const page = parseInt(searchParams.get('page') || '1');
  const limit = 10;

  const nextPage = () => {
    setSearchParams({ page: page + 1 });
  };

  return (
    <div>
      <h1>Page {page}</h1>
      <button onClick={nextPage}>Suivant</button>
    </div>
  );
}

L'URL devient /produits?page=2, ?page=3... L'état est persistant (partageable via lien).

📦 Passer du state avec navigate

On peut transmettre des données à la route cible sans les exposer dans l'URL.

// Depuis le composant source
navigate('/detail', { state: { id: 42, from: 'home' } });

// Dans le composant destination
import { useLocation } from 'react-router-dom';

function Detail() {
  const location = useLocation();
  const data = location.state;   // { id: 42, from: 'home' }
  return <div>...</div>;
}

⚠️ L'état n'est pas persistant après rechargement de la page.

⚠️ Pièges fréquents

  • Oublier l'import : useParams nécessite import { useParams } from 'react-router-dom'.
  • Paramètre inexistant : useParams() retourne un objet vide si aucun paramètre n'est défini.
  • Nested routes : useParams renvoie aussi les paramètres des routes parents imbriquées.
  • useNavigate hors du Router : le composant doit être un descendant de <BrowserRouter>.
  • setSearchParams supprime les params existants : utilisez l'opérateur spread pour conserver les autres clés.
