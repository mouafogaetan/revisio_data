Leçon 23 : Membres statiques

Membres statiques

static : partager des données et comportements au niveau de la classe

⌨️ C# avancé 🧩 POO

🎯 Objectifs de la leçon

1. Créer des membres statiques (champs, méthodes, propriétés) en C#.
2. Utiliser des classes statiques comme Math et Console.
3. Comprendre le constructeur statique et son rôle.
4. Savoir quand utiliser static pour optimiser et partager des ressources.
💡 Pourquoi c'est important ? Les membres statiques appartiennent à la classe elle-même, non aux instances. Idéal pour des compteurs, des méthodes utilitaires ou des configurations globales.

🔧 Qu'est-ce qu'un membre statique ?

Un membre marqué avec le mot-clé static appartient à la classe elle-même et non à une instance particulière. On y accède directement via le nom de la classe.

public class Compteur
{
    // Champ statique : partagé par TOUTES les instances
    public static int nombreInstances = 0;
    
    // Champ d'instance
    private string nom;
    
    // Constructeur d'instance
    public Compteur(string nom)
    {
        this.nom = nom;
        nombreInstances++;  // incrémente le compteur statique
    }
    
    // Méthode statique
    public static void AfficherNbInstances()
    {
        Console.WriteLine($"Nombre total d'instances : {nombreInstances}");
    }
}

// Utilisation
Compteur.AfficherNbInstances();   // appel sans créer d'objet
Compteur c1 = new Compteur("A");
Compteur c2 = new Compteur("B");
Compteur.AfficherNbInstances();   // affiche 2
🔹 Accès : Classe.Membre (pas besoin d'instance)
🔹 Une seule copie en mémoire

📦 Classes statiques

Une classe statique ne peut pas être instanciée et ne contient que des membres statiques.
Exemples célèbres : System.Math, System.Console (bien que Console ait aussi des membres statiques, la classe n'est pas officiellement statique mais très proche).

✨ Exemples concrets

double racine = Math.Sqrt(64);      // 8
double puissance = Math.Pow(3, 4);  // 81
int max = Math.Max(45, 78);         // 78

// Console statique
Console.WriteLine("Hello static world!");

💡 À retenir : Une classe statique empêche l'héritage, tous les membres doivent être static. Parfaite pour des bibliothèques d'outils.

🏷️ Propriétés & champs statiques

On peut déclarer des propriétés statiques (avec get/set) pour contrôler l'accès à une donnée partagée.

public class Configuration
{
    private static string theme = "Clair";
    
    // Propriété statique
    public static string Theme
    {
        get { return theme; }
        set { theme = value; Console.WriteLine($"Thème changé en {value}"); }
    }
    
    // Champ statique en lecture seule (configuration)
    public static readonly string Version = "1.0.0";
}

// Accès via la classe
Console.WriteLine(Configuration.Version);   // "1.0.0"
Configuration.Theme = "Sombre";
🔒 readonly static : constante liée à la classe 🧠 Propriété statique = comme variable globale contrôlée

⚙️ Constructeur statique

Un constructeur statique est exécuté une seule fois, avant la première utilisation de la classe (instanciation ou accès à un membre statique).

public class DatabaseConnector
{
    public static string ConnectionString;
    
    // Constructeur statique - appelé automatiquement une fois
    static DatabaseConnector()
    {
        ConnectionString = "Server=localhost;Database=myDb";
        Console.WriteLine("Constructeur statique exécuté !");
    }
    
    public static void Connect()
    {
        Console.WriteLine($"Connexion avec {ConnectionString}");
    }
}

// Première utilisation : le static ctor est invoqué
DatabaseConnector.Connect();  // affiche d'abord "Constructeur statique exécuté !"

📌 Utilité : initialiser des champs statiques, charger des configurations depuis un fichier, logger un démarrage.

📌 Bonnes pratiques & pièges

✅ Quand utiliser ?

  • Méthodes utilitaires (calculs, conversions)
  • Compteurs d'instances globaux
  • Constantes partagées / configuration commune
  • Fabriques (Factory pattern)

⚠️ Précautions

  • L'état static est partagé entre tous les threads → risque de concurrence
  • Difficile à tester (mock)
  • Ne pas abuser : rompt l'encapsulation si trop global
🧠 Règle d'or : Static pour ce qui est intrinsèquement lié à la classe, pas à l'objet.

🏆 Exemple complet : Gestionnaire d'utilisateurs

public class Utilisateur
{
    private static int totalUtilisateurs = 0;
    public string Nom { get; set; }
    
    public Utilisateur(string nom)
    {
        Nom = nom;
        totalUtilisateurs++;
    }
    
    public static int GetTotalUtilisateurs() => totalUtilisateurs;
    
    public static void AfficherTotal()
    {
        Console.WriteLine($"👥 Total utilisateurs : {totalUtilisateurs}");
    }
}

// Programme
Utilisateur.AfficherTotal();  // 0
var u1 = new Utilisateur("Alice");
var u2 = new Utilisateur("Bob");
Utilisateur.AfficherTotal();  // 2

🧪 Mini exercice mental : Que se passe-t-il si on ajoute un constructeur statique qui affiche "Initialisation statique" ? Essaie de prévoir l'ordre d'exécution.

📌 Ce qu'il faut retenir

  • ✔️ static = membre de classe, partagé entre toutes les instances
  • ✔️ Classes statiques : Math, utilitaires personnalisés
  • ✔️ Constructeur statique : une seule exécution à la première utilisation
  • ✔️ Accès via NomClasse.Membre (pas via objet)
  • ✔️ Parfait pour des compteurs, constantes globales, helpers
