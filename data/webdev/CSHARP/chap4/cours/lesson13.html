📋 List<T> : La liste flexible

Contrairement aux tableaux classiques, List<T> s'agrandit automatiquement.

using System.Collections.Generic;

List<string> eleves = new List<string>();
eleves.Add("Amina");
eleves.Add("Kévin");
eleves.Insert(1, "Béatrice");   // Insère à l'index 1

Console.WriteLine(eleves[0]);    // Amina
eleves.Remove("Kévin");
foreach (var e in eleves)
    Console.WriteLine(e);
📌 Propriétés : Count, Capacity
⚙️ Méthodes : AddRange, RemoveAt, Contains, Sort, Find
[ "Amina" | "Béatrice" | "Kévin" ]
📈 Taille dynamique - pas de limite fixe

🗂️ Dictionary<TKey, TValue> : association clé ⇄ valeur

Recherche ultra-rapide par clé unique (comme un annuaire téléphonique).

Dictionary<int, string> etudiants = new();
etudiants.Add(101, "Fatima");
etudiants.Add(102, "Marc");
etudiants[103] = "Joseph";   // autre façon

string nom = etudiants[101];  // Fatima

if (etudiants.ContainsKey(104))
    Console.WriteLine("Trouvé");

foreach (var kvp in etudiants)
{
    Console.WriteLine($"{kvp.Key} : {kvp.Value}");
}
🔑 Clé unique
💾 Accès O(1)
📚 TryGetValue sécurisé
Exemple: {"101":"Fatima", "102":"Marc", "103":"Joseph"}

🟢 HashSet<T> : Ensemble sans doublons

// set contient {"Rouge", "Vert"} uniquement
set.Add("Bleu");

⏳ Queue<T> (FIFO)

Premier entré, premier sorti → file d'attente

Queue<string> file = new();
file.Enqueue("Client1");
file.Enqueue("Client2");
string suivant = file.Dequeue(); // Client1
📚 Stack<T> (LIFO)

Dernier entré, premier sorti → pile d'assiettes

Stack<int> pile = new();
pile.Push(10);
pile.Push(20);
int sommet = pile.Pop(); // 20

⚡ Quand utiliser quelle collection ?

CollectionIdéal pourAccès / Ordre
List<T>Liste modifiable, besoin indexIndexé, ordre d'insertion
Dictionary<K,V>Recherche par clé uniquePar clé, pas d'ordre garanti
HashSet<T>Éléments uniques, tests d’appartenanceNon ordonné, pas d'index
Queue<T>Traitement FIFO (impressions, messages)Ordre d’entrée
Stack<T>Annulation (Undo), navigationOrdre inverse
💡 Exemple concret : Dans un système de gestion d'étudiants, Dictionary<int, Etudiant> permet de retrouver un étudiant par son ID immédiatement.
🎒 List vs Dictionary: imaginez un classeur (List = feuilles numérotées) / Dictionary = index de mots-clés

⚠️ Pièges fréquents & bonnes pratiques

  • 🚫 Modifier une List pendant qu’on l’itère avec foreach → risque d'erreur. Utiliser une boucle for ou .ToList().
  • 🔑 Dictionary : ne pas utiliser dictionnaire[clé] si la clé n'existe pas → lève KeyNotFoundException. Préférer TryGetValue.
  • 🧹 HashSet : excellente performance pour les opérations ensemblistes (IntersectWith, UnionWith).
  • 📌 Queue/Stack : attention aux méthodes Peek() (ne supprime pas) vs Dequeue/Pop.

📖 Rappel générique : Les collections génériques (List<T>) évitent les conversions de type (boxing/unboxing) par rapport aux anciennes collections.
