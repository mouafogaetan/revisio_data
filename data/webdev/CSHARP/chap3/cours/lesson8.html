📘 Compétence : Contrôler le flux d'exécution

🔍 La structure if classique

Permet d'exécuter un bloc seulement si une condition booléenne est vraie.

int note = 85;
if (note >= 80)
{
    Console.WriteLine("Félicitations ! Mention bien.");
}
else if (note >= 60)
{
    Console.WriteLine("Assez bien, continue !");
}
else
{
    Console.WriteLine("Tu peux faire mieux, courage !");
}

💡 Bonnes pratiques : Toujours utiliser des accolades { } même pour une seule instruction, cela améliore la lisibilité.

C# évalue les conditions de haut en bas, dès qu'une condition est vraie, le bloc correspondant s'exécute puis sort de la structure.

⚙️ Switch & Pattern Matching (C# 7+)

Alternative élégante lorsque vous comparez une variable à plusieurs valeurs constantes ou motifs.

string jour = "Mardi";
string message = jour switch
{
    "Lundi" => "Début de semaine 💪",
    "Mardi" or "Mercredi" => "Milieu de semaine ⚡",
    "Jeudi" => "Bientôt le week-end 🎉",
    "Vendredi" => "Enfin vendredi ! 🥳",
    _ => "Week-end ou jour inconnu 😎"
};
Console.WriteLine(message);

// Switch classique avec pattern matching sur types
object valeur = 42;
switch (valeur)
{
    case int i when i > 10:
        Console.WriteLine($"Entier supérieur à 10 : {i}");
        break;
    case string s:
        Console.WriteLine($"Chaîne de caractères : {s}");
        break;
    case null:
        Console.WriteLine("Valeur nulle");
        break;
    default:
        Console.WriteLine("Type non géré");
        break;
}
📌 Depuis C# 8.0, les switch expressions (syntaxe avec =>) rendent le code plus concis et fonctionnel.

❓ L'opérateur ternaire ? :

Permet d'écrire une condition simple sur une seule ligne. Idéal pour assigner une valeur.

int age = 19;
string statut = (age >= 18) ? "Majeur" : "Mineur";
Console.WriteLine($"Vous êtes {statut}.");

// Exemple avec condition plus avancée
int heure = 14;
string salutation = heure < 12 ? "Bonjour" : (heure < 18 ? "Bon après-midi" : "Bonsoir");
Console.WriteLine(salutation);

✅ Utilisez le ternaire pour des affectations simples, mais évitez de l'imbriquer trop profondément (nuisible à la lisibilité).

🧠 À retenir :

condition ? valeur_si_vrai : valeur_si_faux

📝 Exemple concret : système de validation de notes

Fusionnons if/else et switch dans une mini application console.

// Calcul de mention avec if-else
int score = 92;
string mention;
if (score >= 90) mention = "Excellent";
else if (score >= 75) mention = "Très bien";
else if (score >= 60) mention = "Bien";
else if (score >= 50) mention = "Passable";
else mention = "Insuffisant";

// Transformation du niveau en appréciation (switch expression)
string appreciation = mention switch
{
    "Excellent" => "🏆 Félicitations, bourse possible !",
    "Très bien" => "🎓 Félicitations, solide travail",
    "Bien" => "👍 Encourageant, continuez",
    "Passable" => "📘 Quelques efforts supplémentaires",
    _ => "💪 Besoin de révisions intensives"
};
Console.WriteLine($"Note {score} → {mention} → {appreciation}");

➡️ Testez ces extraits dans votre environnement .NET pour bien comprendre les branchements.

🧹 Bonnes pratiques & pièges à éviter

  • Privilégier switch quand vous avez plus de 3 cas fixes (lisibilité).
  • Attention aux conditions mutuellement exclusives : l'ordre compte dans if-else if.
  • Éviter les ternaires trop longs ou imbriqués → privilégier if classique.
  • Utiliser when en switch pour des filtres supplémentaires (case int n when n > 0:).
  • Ne pas oublier break dans les branches switch classiques (C# n'autorise pas la chute implicite).

🔁 Rappel : la condition doit toujours être une expression booléenne.

✏️ Petit exercice / Vérification

Question :

Qu'affiche le code suivant ?

int x = 5;
string result = x switch
{
    < 0 => "Négatif",
    >= 0 and <= 10 => "Entre 0 et 10",
    _ => "Au-delà de 10"
};
Console.WriteLine(result);
Clique pour voir la réponse

✅ "Entre 0 et 10" car 5 correspond au motif >= 0 and <= 10.
