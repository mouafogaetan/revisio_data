🚪 L'instruction break

break permet de sortir immédiatement de la boucle (for, while, do-while, foreach ou d'un switch). Le programme continue après la boucle.

// Exemple : Trouver le premier nombre divisible par 7
for (int i = 1; i <= 20; i++)
{
    if (i % 7 == 0)
    {
        Console.WriteLine($"Premier multiple de 7 trouvé : {i}");
        break;   // sortie immédiate de la boucle
    }
    Console.WriteLine($"Essai : {i} non divisible");
}
// Output : Affiche jusqu'à 7 puis s'arrête.

📌 Cas d'usage :

  • Recherche d'un élément dans un tableau → on arrête dès qu'on le trouve.
  • Validation de saisie : sortie quand condition remplie.
  • Éviter des traitements inutiles après un résultat.

⏩ L'instruction continue

continue ignore le reste du corps de la boucle et passe immédiatement à l'itération suivante (incrémentation / condition).

// Afficher uniquement les nombres pairs entre 1 et 10
for (int i = 1; i <= 10; i++)
{
    if (i % 2 != 0)   // si impair
    {
        continue;     // on saute l'affichage
    }
    Console.Write(i + " ");   // 2 4 6 8 10
}

💡 Utilité :

  • Filtrer des données (ignorer les valeurs invalides).
  • Éviter trop de niveaux d'imbrication de if.
  • Alléger la logique dans les boucles.

🔀 goto : saut vers une étiquette

⚠️ À utiliser avec précaution !

goto peut rendre le code difficile à suivre. Cependant, dans quelques rares cas (sortie de boucles imbriquées), il peut être utile.

// Exemple : sortir de deux boucles imbriquées avec goto
for (int i = 0; i < 5; i++)
{
    for (int j = 0; j < 5; j++)
    {
        if (i + j == 6)
        {
            Console.WriteLine($"Trouvé à i={i}, j={j}");
            goto FinBoucles;   // saut vers l'étiquette
        }
    }
}
FinBoucles:
Console.WriteLine("Sortie des boucles imbriquées !");

📖 Alternative moderne : encapsuler la logique dans une méthode avec return. Néanmoins, le goto reste valide dans certains algorithmes.

🎯 Exercice mental :

Dans une boucle while(true) avec saisie utilisateur, on utilise break quand l'utilisateur tape "quit".

💻 Code intelligent :

Utilisez continue pour éviter d'écrire de grands blocs conditionnels.

📌 Points clés à retenir

1

break : "stop immédiat" → utile pour quitter une boucle prématurément.

2

continue : "sauter le tour" → ne pas exécuter le reste du code de l’itération courante.

3

goto : à utiliser avec parcimonie ; privilégier refactorisation avec méthodes ou drapeaux.
