💡 Les lambda rendent le code plus déclaratif et puissant, surtout avec les collections.

🔍 Définition

Une expression lambda est une fonction anonyme que vous pouvez utiliser pour créer des délégués ou des arbres d'expression.

(paramètres) => expression ou bloc d'instructions

Symbole => se lit "devient" ou "goes to".

Lambda expression
(int x) => x * x;
Bloc lambda
(x, y) => { return x + y; }

✍️ Syntaxe de base

// 1. Sans paramètre
Action bonjour = () => Console.WriteLine("Bonjour !");

// 2. Un seul paramètre (parenthèses optionnelles)
Func<int, int> carre = x => x * x;

// 3. Plusieurs paramètres
Func<int, int, int> somme = (a, b) => a + b;

// 4. Corps avec accolades (plusieurs instructions)
Func<int, string> evaluer = note => 
{
    if (note >= 10) return "Validé";
    else return "Échec";
};
📌 Remarque : si le corps contient une seule expression, les accolades et return sont implicites.

🧩 Utilisation avec Func<T> et Action

Func<T, TResult> : représente une fonction qui retourne une valeur.

Action<T> : représente une méthode qui ne retourne rien (void).

// Func: prendre un double et retourner son carré Func<double, double> puissance2 = x => Math.Pow(x, 2); Console.WriteLine(puissance2(5)); // 25 // Action: afficher un message Action<string> afficherMsg = msg => Console.WriteLine("INFO: " + msg); afficherMsg("Lambda en action !");

📚 Lambda + LINQ : combo gagnant

int[] notes = { 12, 8, 15, 6, 19, 10 };

// Filtrer les notes supérieures à 10
var bonnesNotes = notes.Where(n => n > 10);
// Résultat : 12, 15, 19

// Ordonner et projeter
var resultat = notes.OrderByDescending(n => n)
                    .Select(n => $"Note: {n}/20");

foreach(var item in resultat)
    Console.WriteLine(item);
Where, Select, OrderBy, Count, etc.

🪤 Capture de variables (Closure)

Une expression lambda peut capturer des variables locales ou des paramètres de la méthode englobante.

int facteur = 3;
Func<int, int> multiplier = x => x * facteur;
// facteur est capturé
Console.WriteLine(multiplier(10)); // 30

facteur = 10;
Console.WriteLine(multiplier(10)); // 100 (la variable est partagée)

⚠️ Attention : les variables capturées sont évaluées lors de l'exécution de la lambda, pas au moment de sa création.

✅ Bonnes pratiques & limites

  • Utilisez les lambda pour simplifier le code des délégués.
  • Privilégiez la lisibilité : évitez les expressions trop longues sans blocs.
  • Les lambda ne peuvent pas contenir de goto, break, continue à moins d'être dans un bloc d'instructions.
  • Ne capturez pas de variables modifiées dans des boucles sans comprendre les effets (pièges classiques).

💡 Les expressions lambda sont transformées par le compilateur en méthodes anonymes ou en délégués.
