Pourquoi c'est important ?

OWASP Top 10 : les failles d'injection et XSS sont parmi les plus critiques. Une API non sécurisée expose les données sensibles.

💉 Injection SQL : la menace

L'injection SQL permet à un attaquant d'exécuter des commandes SQL malveillantes. Entity Framework Core protège automatiquement grâce aux requêtes paramétrées.

❌ MAUVAIS : concaténation dangereuse

string query = $"SELECT * FROM Users WHERE Name = '{userInput}'";

✅ BON : EF Core paramètre automatiquement

var users = _context.Users.Where(u => u.Name == userInput).ToList();
Règle d'or : N'utilisez JAMAIS de requêtes SQL brutes avec concaténation. Préférez LINQ ou FromSqlRaw avec paramètres.

⚠️ Cross-Site Scripting (XSS)

Un attaquant injecte du JavaScript malveillant dans une page web. Dans une API REST (JSON), le risque est limité, mais il faut encoder les sorties HTML si vous servez du contenu dynamique.

// Si votre API renvoie du HTML (peu recommandé), échappez les données : using System.Text.Encodings.Web; var safeOutput = HtmlEncoder.Default.Encode(userInput);
Ne jamais insérer d'entrée utilisateur brute dans une réponse HTML Les API JSON sont naturellement moins vulnérables au XSS

🌐 CSRF : forge de requête intersite

Un site malveillant force le navigateur d'un utilisateur authentifié à envoyer une requête non autorisée à votre API.

Bonne nouvelle : Avec une API JWT (token dans le header Authorization: Bearer ...), les cookies ne sont pas utilisés, donc le CSRF n'est pas un problème.
Si vous utilisez l'authentification par cookie (mode traditionnel MVC), utilisez les anti-forgery tokens :
services.AddAntiforgery(); et [AutoValidateAntiforgeryToken].

🛡️ En-têtes HTTP de sécurité

Ajoutez des en-têtes pour prévenir les attaques courantes. Créez un middleware personnalisé :

public class SecurityHeadersMiddleware
{
  private readonly RequestDelegate _next;
  public SecurityHeadersMiddleware(RequestDelegate next) => _next = next;

  public async Task InvokeAsync(HttpContext context)
  {
    context.Response.Headers.Add("X-Content-Type-Options", "nosniff");
    context.Response.Headers.Add("X-Frame-Options", "DENY");
    context.Response.Headers.Add("X-XSS-Protection", "1; mode=block");
    context.Response.Headers.Add("Referrer-Policy", "strict-origin-when-cross-origin");
    context.Response.Headers.Add("Content-Security-Policy", "default-src 'self'");
    await _next(context);
  }
}

// Dans Program.cs : app.UseMiddleware<SecurityHeadersMiddleware>();

HSTS (HTTP Strict Transport Security) force HTTPS : app.UseHsts();

✅ Validation robuste des entrées

Utilisez les attributs de validation Data Annotations pour filtrer les entrées malveillantes ou mal formées.

public class CreateProductDto
{
  [Required(ErrorMessage = "Le nom est obligatoire")]
  [StringLength(100, MinimumLength = 2)]
  public string Name { get; set; }

  [Range(0, 9999.99)]
  public decimal Price { get; set; }

  [EmailAddress]
  public string ContactEmail { get; set; }
}
Le décorateur [ApiController] valide automatiquement les DTO et retourne une erreur 400 Bad Request.

🚨 Autres bonnes pratiques

Rate Limiting (dès .NET 7) : limite le nombre de requêtes par IP.
Ne pas exposer les erreurs détaillées en production (app.UseDeveloperExceptionPage() conditionnel).
Gérer les CORS : limiter les origines autorisées.
Scanner les dépendances pour les vulnérabilités connues (dotnet list package --vulnerable).
// Exemple : CORS restrictif builder.Services.AddCors(options =>
{
  options.AddPolicy("FrontendApp", policy =>
    policy.WithOrigins("https://monsite.com")
          .AllowAnyHeader().AllowAnyMethod());
});

✅ Checklist sécurité pour votre API .NET

☑️ Toujours utiliser EF Core ou des requêtes paramétrées (anti-SQL injection)
☑️ Activer l'authentification JWT (pas de cookies, donc pas de CSRF)
☑️ Ajouter les en-têtes de sécurité (CSP, X-Frame-Options, HSTS)
☑️ Valider tous les DTO avec Data Annotations
☑️ Désactiver les pages d'erreur détaillées en production
☑️ Configurer CORS pour autoriser uniquement les origines connues

Félicitations ! Vous pouvez maintenant sécuriser votre API contre les attaques les plus courantes.
