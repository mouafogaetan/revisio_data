💡 Pourquoi JWT ? Authentification sans état, idéal pour les API REST. Le client stocke le token et l’envoie dans l’en-tête Authorization: Bearer <token>.

📥 Étape 1 : Installer le package d'authentification JWT

Pour activer l’authentification par JWT, nous avons besoin du package officiel Microsoft :

dotnet add package Microsoft.AspNetCore.Authentication.JwtBearer

Ou via le NuGet Package Manager dans Visual Studio / Rider. Ce package contient les middlewares et les extensions pour valider les tokens JWT.

✔️ Vérification : après installation, le fichier .csproj contient une référence à JwtBearer.

🔧 dotnet CLI ➡️ JwtBearer installé ➡️ Authentification possible

⚙️ Étape 2 : Configurer l'authentification JWT dans Program.cs

Dans le fichier Program.cs, nous ajoutons le service d'authentification et nous précisons les règles de validation du token.

// Ajouter cette section après le builder
var key = Encoding.ASCII.GetBytes("VOTRE_CLE_SECRETE_SUPER_LONGUE_32+_CARACTERES_ICI");

builder.Services.AddAuthentication(options => 
{
    options.DefaultAuthenticateScheme = JwtBearerDefaults.AuthenticationScheme;
    options.DefaultChallengeScheme = JwtBearerDefaults.AuthenticationScheme;
})
.AddJwtBearer(options => 
{
    options.RequireHttpsMetadata = false; // ⚠️ en production = true
    options.SaveToken = true;
    options.TokenValidationParameters = new TokenValidationParameters
    {
        ValidateIssuerSigningKey = true,
        IssuerSigningKey = new SymmetricSecurityKey(key),
        ValidateIssuer = true,
        ValidIssuer = "https://votredomaine.com",
        ValidateAudience = true,
        ValidAudience = "monApiClient",
        ValidateLifetime = true,
        ClockSkew = TimeSpan.Zero
    };
});

builder.Services.AddAuthorization(); // optionnel mais recommandé

🔐 À retenir :

  • IssuerSigningKey : la clé secrète servant à signer le JWT.
  • ValidateIssuer / ValidateAudience : vérifie l'émetteur et le destinataire du token.
  • ValidateLifetime : rejette les tokens expirés.

🔑 Étape 3 : Comprendre Issuer, Audience & Clé secrète

🗝️

Clé secrète (Secret Key)

Une chaîne de caractères longue (min 32 caractères) utilisée pour signer et vérifier le token. Ne jamais la partager.

🏢

Issuer (Émetteur)

Identifie qui a émis le JWT. Exemple : https://monapi.com. Le client doit envoyer un token avec le même issuer.

📱

Audience

Destinataire prévu du token (ex: "monApplicationWeb", "monApiMobile"). Empêche qu’un token soit utilisé sur une autre API.

// Configuration typique stockée dans appsettings.json :

{
  "JwtSettings": {
    "Secret": "super_secrete_key_32_characters_minimum!",
    "Issuer": "https://api.monecole.cm",
    "Audience": "etudiantsWebDev"
  }
}

✅ Il est conseillé de lire ces valeurs depuis appsettings.json avec IConfiguration pour éviter les hardcodes.

🛡️ Étape 4 : Activer le middleware & protéger les endpoints

Après avoir configuré les services, nous devons activer l’authentification et l’autorisation dans le pipeline :

var app = builder.Build();
// ... autres middlewares
app.UseAuthentication();   // 🔐 Ajouter cette ligne
app.UseAuthorization();    // ⚡ Autorisation (vérifie [Authorize])
app.MapControllers();
app.Run();

Pour protéger un contrôleur ou une action, utiliser l’attribut [Authorize] :

[ApiController]
[Route("api/[controller]")]
[Authorize]  // 🔒 toute la classe nécessite authentification
public class DashboardController : ControllerBase
{
    [HttpGet("secret")]
    public IActionResult GetSecretData() => Ok("Données protégées");
}

// Autorisation par rôle : [Authorize(Roles = "Admin")]

✅ Si un client envoie une requête sans token (ou token invalide), l’API renvoie automatiquement une réponse 401 Unauthorized.

🧪 Tester l’authentification JWT (Swagger + Postman)

Une fois la configuration réalisée, vous pouvez générer un token (via un endpoint /api/auth/login) et l’utiliser dans Swagger UI ou un outil comme Postman.

📮 En-tête HTTP requis

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Pour Swagger, cliquez sur "Authorize" et entrez Bearer <votre_token>.

🔁 Cycle classique

  1. Client envoie login/mot de passe
  2. API génère un JWT signé
  3. Client stocke token (localStorage / mémoire)
  4. Client inclut token dans chaque requête protégée
💡 Astuce : Pour des tests rapides, générez un token sur jwt.io en utilisant la même clé et les mêmes paramètres que votre API.

📌 Récapitulatif : Les 3 piliers de la config JWT

1
Installer Microsoft.AspNetCore.Authentication.JwtBearer
2
Configurer les services avec AddAuthentication().AddJwtBearer() et paramètres de validation (clé, issuer, audience).
3
Activer app.UseAuthentication() + app.UseAuthorization() dans le pipeline, puis utiliser [Authorize].

⚠️ Recommandations sécuritaires : Toujours utiliser HTTPS en production, stocker la clé secrète dans les variables d’environnement, et générer des clés robustes (minimum 256 bits).

🛠️ Bonus : Générer un JWT (aperçu)

Pour que votre API puisse délivrer des tokens, un endpoint de login est nécessaire. Voici un extrait typique :

// Dans un contrôleur AuthController
[HttpPost("login")]
public IActionResult Login(LoginDto login)
{
    // Vérifier les identifiants (ex: user existant, hash mot de passe)
    var user = _userService.ValidateUser(login.Email, login.Password);
    if (user == null) return Unauthorized();
    
    var tokenHandler = new JwtSecurityTokenHandler();
    var key = Encoding.ASCII.GetBytes(_jwtSettings.Secret);
    var tokenDescriptor = new SecurityTokenDescriptor
    {
        Subject = new ClaimsIdentity(new Claim[] 
        {
            new Claim(ClaimTypes.NameIdentifier, user.Id.ToString()),
            new Claim(ClaimTypes.Email, user.Email),
            new Claim(ClaimTypes.Role, user.Role)
        }),
        Expires = DateTime.UtcNow.AddHours(2),
        Issuer = _jwtSettings.Issuer,
        Audience = _jwtSettings.Audience,
        SigningCredentials = new SigningCredentials(new SymmetricSecurityKey(key),
            SecurityAlgorithms.HmacSha256Signature)
    };
    var token = tokenHandler.CreateToken(tokenDescriptor);
    return Ok(new { token = tokenHandler.WriteToken(token) });
}

✨ Ce code sera détaillé dans la leçon 19. Il montre comment créer et retourner un token JWT.
