Leçon 11 – Chapitre 4

🗄️ EF Core : Accès aux données

Entity Framework Core – Code First, Migrations & CRUD

Durée : 45 min Niveau : Intermédiaire ORM moderne

"Entity Framework Core : l'ORM qui vous simplifie l'accès aux données."

🎯 Objectifs de la leçon

  • Installer les packages nécessaires pour EF Core avec SQL Server.
  • Comprendre l'approche Code First (créer la base depuis le code).
  • Créer des modèles (entités) et la classe DbContext.
  • Maîtriser les migrations (ajouter, appliquer, annuler).
  • Effectuer les opérations CRUD de base avec DbSet.
À la fin, vous pourrez connecter votre API .NET à une base de données réelle.

📦 Installation des packages

Pour utiliser EF Core avec SQL Server, installez ces packages NuGet :

dotnet add package Microsoft.EntityFrameworkCore
dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Tools

Ou via la console du gestionnaire de packages :

Install-Package Microsoft.EntityFrameworkCore.SqlServer
Astuce : Si vous utilisez PostgreSQL, remplacez SqlServer par Npgsql.

🏗️ Modèles et DbContext (Code First)

Approche Code First : on écrit d'abord les classes C#, EF Core génère la base.

🔹 Entité Product (exemple) :

public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
}

🔹 DbContext (pont entre entités et base) :

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }
    public DbSet<Product> Products { get; set; }
}

Chaque DbSet représente une table en base de données.

⚙️ Configuration et Migrations

Dans Program.cs ou appsettings.json, on ajoute la chaîne de connexion et on enregistre DbContext :

// Program.cs
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));

🛠️ Commandes de migration (terminal) :

dotnet ef migrations add InitialCreate
dotnet ef database update

La première commande crée un dossier Migrations, la seconde applique les changements à la base.

Résultat : la base de données et la table Products sont créées automatiquement.

📝 CRUD avec DbContext

1 Créer (Create) :

var product = new Product { Name = "Clavier", Price = 25.99m };
context.Products.Add(product);
await context.SaveChangesAsync();

2 Lire (Read) :

var product = await context.Products.FindAsync(1);
var allProducts = await context.Products.ToListAsync();

3 Mettre à jour (Update) :

product.Price = 29.99m;
context.Products.Update(product);
await context.SaveChangesAsync();

4 Supprimer (Delete) :

context.Products.Remove(product);
await context.SaveChangesAsync();

✨ Bonnes pratiques & récapitulatif

  • Toujours utiliser async/await (ToListAsync, SaveChangesAsync) pour ne pas bloquer le thread.
  • Ne jamais exposer DbContext directement dans les contrôleurs à long terme, utilisez un repository.
  • Gérer les migrations en équipe : ne pas supprimer le dossier Migrations sans précaution.
  • Pour le développement, utiliser UseInMemoryDatabase pour les tests.
  • Toujours valider les données avant d'appeler SaveChanges.

Exercice pratique :

Créez une entité "Category" (Id, Name) et configurez une relation One-to-Many avec Product. Générez une migration et insérez des catégories via l'API.

EF Core transforme vos classes C# en tables SQL – Puissant et intuitif.
