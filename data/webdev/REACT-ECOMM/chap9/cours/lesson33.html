Gestion des adresses (CRUD)

Créer, lire, modifier, supprimer des adresses de livraison et gérer l'adresse par défaut dans le compte utilisateur

➕ Ajouter / ✏️ Modifier ⭐ Adresse par défaut ✅ Validation Zod

🎯 Objectifs de la leçon

  • ✓ Implémenter les opérations CRUD (Create, Read, Update, Delete) pour les adresses d'un utilisateur
  • ✓ Permettre de définir une adresse par défaut pour accélérer le processus de commande
  • ✓ Assurer la validation des données (code postal, ville, téléphone) avec Zod et React Hook Form
  • ✓ Améliorer l'expérience utilisateur en proposant la sélection rapide d'une adresse existante lors du checkout
💡 Pourquoi c'est crucial ? Des adresses bien gérées = livraisons sans erreur, clients satisfaits et réduction des problèmes de retour.
📋

CRUD : Définition et modèle d'adresse

CRUD désigne les 4 opérations de base sur une ressource : Create, Read, Update, Delete. Pour une adresse, on stocke :

  • id : identifiant unique
  • nom (ex: "Maison", "Travail")
  • rue + complément (optionnel)
  • ville, codePostal, pays
  • téléphone (livraison)
  • estParDefaut (boolean)
// Schéma Zod pour validation
const addressSchema = z.object({
  nom: z.string().min(2, "Nom trop court"),
  rue: z.string().min(5, "Rue requise"),
  ville: z.string().min(2),
  codePostal: z.string().regex(/^\d{5}$/, "Code postal 5 chiffres"),
  pays: z.string().default("Cameroun"),
  telephone: z.string().regex(/^\d{9,13}$/, "Téléphone invalide"),
  estParDefaut: z.boolean().default(false)
});

Create : Ajouter une adresse

On utilise un formulaire contrôlé par React Hook Form et la validation Zod. À la soumission, on envoie les données à l'API.

import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';

function AjouterAdresse() {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(addressSchema)
  });

  const onSubmit = async (data) => {
    await axios.post('/api/addresses', data);
    // Rafraîchir la liste des adresses
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register("nom")} placeholder="Ex: Domicile" />
      {errors.nom && <span>{errors.nom.message}</span>}
      {/* autres champs ... */}
      <button type="submit">Ajouter</button>
    </form>
  );
}
✨ Astuce : Proposer un bouton "Utiliser mes coordonnées actuelles" pour pré-remplir le formulaire.

Update / Delete & Adresse par défaut

Chaque adresse dans la liste propose des actions : modifier, supprimer et définir par défaut.

// Mise à jour de l'adresse par défaut
const setDefaultAddress = async (addressId) => {
  await axios.patch(`/api/addresses/${addressId}/default`);
  refetchAddresses(); // recharger la liste
};

// Supprimer une adresse
const deleteAddress = async (addressId) => {
  if (confirm('Supprimer cette adresse ?')) {
    await axios.delete(`/api/addresses/${addressId}`);
    refetchAddresses();
  }
};
⭐ Adresse par défaut : mise en évidence visuelle
⚠️ Empêcher suppression si c'est la seule adresse
📌 Une seule adresse peut être par défaut. On désactive l'ancienne avant d'en définir une nouvelle.

Validation des adresses

Au-delà du format, on peut intégrer des vérifications côté client et serveur :

  • Code postal : regex pour 5 chiffres ($\d{5}$)
  • Téléphone : 9 à 13 chiffres, accepte +237
  • Ville : pas de caractères spéciaux interdits
  • Adresse unique : éviter les doublons en comparant champs sensibles
const addressSchema = z.object({
  codePostal: z.string().regex(/^\d{5}$/, "Code postal invalide (5 chiffres)"),
  telephone: z.string().regex(/^(\+237)?[0-9]{9}$/, "Téléphone invalide (ex: 6XXXXXXXX ou +237XXXXXXXXX)"),
  ville: z.string().min(2).max(50)
});
🔁 En production, on utilise aussi des API de géocodage pour vérifier l'existence de l'adresse.

Réutilisation dans le checkout

Pendant le paiement, on permet à l'utilisateur de choisir une adresse parmi ses adresses enregistrées ou d'en ajouter une nouvelle.

function LivraisonStep() {
  const { data: addresses } = useQuery(['addresses'], fetchAddresses);
  const [selectedAddress, setSelectedAddress] = useState(null);

  return (
    <div>
      <h3>Sélectionnez votre adresse de livraison</h3>
      {addresses?.map(addr => (
        <div key={addr.id} onClick={() => setSelectedAddress(addr)}>
          <input type="radio" checked={selectedAddress?.id === addr.id} />
          {addr.rue}, {addr.ville} - {addr.codePostal}
          {addr.estParDefaut && <span> (Défaut)</span>}
        </div>
      ))}
      <button>➕ Ajouter une nouvelle adresse</button>
    </div>
  );
}
🏆 L'adresse par défaut est automatiquement pré-sélectionnée pour fluidifier l'expérience.

🎁 Récapitulatif & bonnes pratiques

  • Implémenter un CRUD complet (Create, Read, Update, Delete) pour les adresses.
  • Permettre de définir une adresse par défaut et la mettre en avant.
  • Utiliser React Hook Form + Zod pour une validation robuste.
  • Proposer la sélection rapide des adresses existantes durant le checkout.
  • Ne pas autoriser la suppression si c'est la dernière adresse de l'utilisateur.
