📦 Historique des commandes

Liste · Détails · Suivi de statut

Liste des commandes Détails produits Suivi en temps réel

Donnez à vos clients la possibilité de consulter leurs achats passés et de suivre l'avancement de leurs commandes.

🎯 Objectifs de la leçon

  • Afficher la liste des commandes d’un utilisateur connecté (pagination, filtres).
  • Visualiser le détail d’une commande : produits, quantités, prix, adresse de livraison.
  • Implémenter le suivi de statut avec barre de progression et timeline.
  • Gérer les actions (annulation, re-commander, téléchargement facture).

Un bon suivi de commande réduit les appels au support client et augmente la confiance.

Entité Commande (Order)

Données typiques renvoyées par l'API backend :

interface Order {
  id: string;
  createdAt: string; // date de la commande
  totalAmount: number; // montant total TTC
  status: 'pending' | 'paid' | 'shipped' | 'delivered' | 'cancelled';
  shippingAddress: { street, city, zipCode, country };
  items: OrderItem[]; // produits commandés
  trackingNumber?: string; // numéro de suivi colis
  estimatedDelivery?: string;
  invoiceUrl?: string;
}

Les statuts évoluent dans le temps : pending (en attente) → paid (payée) → shipped (expédiée) → delivered (livrée).

Composant OrdersList

Affichage des commandes avec React Query et pagination.

const OrdersList = () => {
  const { data, isLoading } = useQuery({
    queryKey: ['userOrders'],
    queryFn: () => api.get('/orders/user')
  });

  if (isLoading) return <OrderSkeleton />;

  return (
    <div className="space-y-4">
      {data.orders.map(order => (
        <OrderCard key={order.id} order={order} />
      ))}
    </div>
  );
};

Actions sur une commande

  • Annuler : si la commande n'est pas encore expédiée (statut pending/paid).
  • Commander à nouveau : ajoute tous les produits au panier en une seule action.
  • Télécharger la facture (PDF) via un lien fourni par l'API.
  • Contacter le support : lien rapide vers le service client.
const cancelOrder = useMutation({
  mutationFn: () => api.post(`/orders/${orderId}/cancel`),
  onSuccess: () => queryClient.invalidateQueries(['order', orderId])
});

Exercice : Mise en pratique

🎯 À réaliser dans votre projet e-commerce :

  1. Créer une page /account/orders qui liste toutes les commandes de l'utilisateur (avec React Query).
  2. Chaque commande doit afficher : date, montant total, statut (badge coloré), et un lien vers les détails.
  3. Ajouter une pagination ou un bouton "Voir plus".
  4. Créer la page /account/orders/:orderId qui affiche les détails : produits, adresse, timeline de statut.
  5. Implémenter un bouton "Télécharger la facture" (PDF).
  6. Bonus : ajouter la fonctionnalité "Commander à nouveau" (remplit le panier).
