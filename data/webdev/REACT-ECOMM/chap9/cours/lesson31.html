Gestion du compte utilisateur

Afficher les informations du compte · Modifier le profil · Changer le mot de passe

📋 Affichage données ✏️ Modification profil 🔐 Sécurité mot de passe

🎯 Objectifs de la leçon

✓ Créer une page "Mon compte" affichant les informations utilisateur (nom, email, téléphone, avatar)
✓ Implémenter un formulaire de modification du profil avec validation
✓ Gérer le changement de mot de passe (ancien mot de passe, nouveau, confirmation)
✓ Utiliser React Hook Form + Zod pour la validation des formulaires
💡 Pourquoi c'est important ? Les utilisateurs doivent pouvoir gérer leurs données facilement et en toute sécurité.

📄 Affichage des informations du compte

On utilise un composant qui récupère les données utilisateur via le store Zustand ou l'API.

// composant ProfileInfo.jsx
import { useAuthStore } from '../stores/authStore';

export default function ProfileInfo() {
  const { user } = useAuthStore();
  if (!user) return <div>Chargement...</div>;

  return (
    <div className="bg-white p-6 rounded-xl shadow">
      <div className="flex items-center gap-4">
        <img src={user.avatar} alt="Avatar" className="w-20 h-20 rounded-full" />
        <div>
          <h3 className="text-xl font-bold">{user.firstName} {user.lastName}</h3>
          <p className="text-gray-500">{user.email}</p>
          <p className="text-gray-500">📞 {user.phone || 'Non renseigné'}</p>
        </div>
      </div>
    </div>
  );
}
✨ Astuce : on peut aussi afficher la date d'inscription et la dernière connexion.

✏️ Modifier le profil (React Hook Form + Zod)

// ProfileEditForm.jsx
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';
import { useAuthStore } from '../stores/authStore';
import api from '../services/api';

const profileSchema = z.object({
  firstName: z.string().min(2, 'Prénom trop court'),
  lastName: z.string().min(2, 'Nom trop court'),
  phone: z.string().regex(/^[0-9+\-\s]{8,}$/, 'Téléphone invalide'),
  email: z.string().email('Email invalide')
});

export default function ProfileEditForm() {
  const { user, setUser } = useAuthStore();
  const { register, handleSubmit, formState: { errors, isSubmitting } } = useForm({
    resolver: zodResolver(profileSchema),
    defaultValues: { firstName: user.firstName, lastName: user.lastName, phone: user.phone, email: user.email }
  });

  const onSubmit = async (data) => {
    const { data: updatedUser } = await api.patch('/users/me', data);
    setUser(updatedUser);
    alert('Profil mis à jour');
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)} className="space-y-4">
      <input {...register('firstName')} placeholder="Prénom" />
      {errors.firstName && <p>{errors.firstName.message}</p>}
      {/* autres champs similaires */}
      <button disabled={isSubmitting}>Enregistrer</button>
    </form>
  );
}
📌 On valide les données avant envoi, on met à jour le store local après succès.

🔒 Changer le mot de passe (sécurité)

const passwordSchema = z.object({
  currentPassword: z.string().min(1, 'Requis'),
  newPassword: z.string().min(8, '8 caractères min').regex(/[A-Z]/, 'Majuscule requise'),
  confirmPassword: z.string()
}).refine(data => data.newPassword === data.confirmPassword, {
  message: 'Mots de passe différents',
  path: ['confirmPassword']
});

const onSubmitPassword = async (data) => {
  await api.post('/users/change-password', data);
  alert('Mot de passe modifié');
};
⚠️ Ne jamais stocker ou logger les mots de passe en clair. Le backend doit les hacher (bcrypt).

✨ Bonnes pratiques UX pour la gestion du compte

🔔 Notifications
Afficher des toasts de succès/erreur après chaque action (profil modifié, mot de passe changé).
📱 Responsive
Adapter les formulaires en colonne sur mobile, champs pleine largeur.
🛡️ Confirmation
Demander le mot de passe actuel pour toute modification sensible (email ou mot de passe).
🧪 Validation en temps réel
Afficher les erreurs dès la saisie avec `mode: 'onChange'` dans React Hook Form.
💡 Pensez à l'accessibilité : labels explicites, messages d'erreur clairs.

🔄 Mise à jour du store après modification

// authStore.js (Zustand)
const useAuthStore = create((set) => ({
  user: null,
  setUser: (userData) => set({ user: userData }),
  updateProfile: async (updates) => {
    const { data } = await api.patch('/users/me', updates);
    set({ user: data });
    return data;
  }
}));

// Dans le composant
const updateProfile = useAuthStore(state => state.updateProfile);
await updateProfile({ firstName: 'Jean' });

Centraliser la logique dans le store évite la duplication et garantit que toute l'application utilise les données à jour.
