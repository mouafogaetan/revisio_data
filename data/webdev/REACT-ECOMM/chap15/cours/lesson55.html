📱 Mobile-first avec Tailwind

Tailwind utilise les breakpoints par défaut : sm (640px), md (768px), lg (1024px). Le style mobile s’écrit sans préfixe.

<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4">
  <!-- 1 colonne sur mobile, 2 sur tablette, 4 sur desktop -->
</div>

Exemple typique : les cartes produit occupent toute la largeur sur mobile, puis s’alignent en grille sur les écrans plus grands.

🍔 Menu burger responsive

Sur mobile, on cache la navigation horizontale et on affiche un bouton hamburger qui ouvre un menu latéral ou vertical.

MonShop 
const [isOpen, setIsOpen] = useState(false);

return (
  <div>
    <button onClick={() => setIsOpen(!isOpen)} className="md:hidden">
      ☰
    </button>
    <div className={`${isOpen ? 'block' : 'hidden'} md:flex`}>
      {/* Liens de navigation */}
    </div>
  </div>
);

Sur desktop (md:flex), le menu est toujours visible.

🛒 Panier optimisé mobile

  • Réduire la taille des images (vignettes).
  • Permettre de modifier les quantités avec des boutons + / - faciles à taper.
  • Afficher le total de manière visible.
  • Proposer un récapitulatif compact avec scrolling vertical.
<div className="flex flex-col gap-4">
  {cartItems.map(item => (
    <div className="flex gap-3 items-center">
      <img src={item.image} className="w-16 h-16 object-cover rounded" />
      <div className="flex-1">
        <h3 className="text-sm font-semibold">{item.name}</h3>
        <div className="flex items-center gap-2">
          <button className="px-2 py-1 bg-gray-200 rounded">-</button>
          <span>{item.quantity}</span>
          <button className="px-2 py-1 bg-gray-200 rounded">+</button>
        </div>
      </div>
      <span className="font-bold">{item.price} FCFA</span>
    </div>
  ))}
</div>

🧪 Tester la réactivité

Utiliser les DevTools de Chrome (icône mobile) ou des services comme BrowserStack.

  • Raccourci Chrome : Ctrl + Maj + M (Windows) / Cmd + Opt + M (Mac).
  • Tester sur iPhone SE, Pixel 5, iPad.
  • Vérifier les points de rupture : 375px, 768px, 1024px.
📌 Testez aussi les interactions tactiles (scroll, clics).

⚡ Optimisations avancées

  • Images responsives : utiliser srcSet ou des services CDN (Cloudinary).
  • Police adaptative : tailles de texte en rem plutôt que px.
  • Touch targets : boutons d’au moins 44×44px (recommandé Apple).
  • Menu déroulant : éviter les hover sur mobile, privilégier les clics.
<button className="p-3 min-w-[44px] min-h-[44px]">Ajouter</button>

🏗️ Layout responsive complet

export default function Layout() {
  return (
    <div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">
      <header className="flex flex-wrap justify-between items-center py-4">
        <Logo />
        <MobileMenu />
        <DesktopNav className="hidden md:flex" />
      </header>
      <main><Outlet /></main>
    </div>
  );
}

Le conteneur principal a une largeur max et des marges qui se réduisent sur mobile.

✏️ Exercice pratique

🧪 À réaliser :

  • Prenez votre projet e-commerce existant.
  • Transformez la navigation en menu burger sur mobile (état local useState).
  • Adaptez la grille des produits : 1 colonne sur mobile, 2 sur tablette, 4 sur desktop.
  • Rendez le panier responsive : images réduites, boutons tactiles.
  • Testez sur les tailles d’écran via Chrome DevTools.
💡 Utilisez les classes Tailwind flex, grid et les breakpoints sm:, md:, lg:.
