1. Installation de Vitest + RTL

Dans un projet Vite + React, on installe les dépendances nécessaires :

npm install -D vitest @testing-library/react @testing-library/jest-dom @testing-library/user-event jsdom

On configure vite.config.js :

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  test: {
    globals: true,
    environment: 'jsdom',
    setupFiles: './src/test/setup.js',
  },
})
📌 setup.js : import '@testing-library/jest-dom/vitest' pour les assertions supplémentaires.

2. Tester un composant isolé

Exemple : bouton ProductCard qui affiche un nom et un prix.

// ProductCard.jsx
export default function ProductCard({ name, price, onBuy }) {
  return (
    <div>
      <h3>{name}</h3>
      <p>{price} FCFA</p>
      <button onClick={onBuy}>Acheter</button>
    </div>
  );
}
// ProductCard.test.jsx
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import ProductCard from './ProductCard';

test('affiche le nom et le prix', () => {
  render(<ProductCard name="T-shirt" price={5000} />);
  expect(screen.getByText('T-shirt')).toBeInTheDocument();
  expect(screen.getByText('5000 FCFA')).toBeInTheDocument();
});

test('appelle onBuy quand on clique', async () => {
  const mockBuy = vi.fn();
  render(<ProductCard name="Livre" price={3000} onBuy={mockBuy} />);
  await userEvent.click(screen.getByRole('button', { name: /Acheter/i }));
  expect(mockBuy).toHaveBeenCalledTimes(1);
});
✨ Utilisez screen.getByRole pour les interactions accessibles.
🎣

3. Tester un hook personnalisé

On utilise renderHook depuis @testing-library/react.

// useCounter.js
import { useState } from 'react';
export function useCounter(initial = 0) {
  const [count, setCount] = useState(initial);
  const increment = () => setCount(c => c + 1);
  return { count, increment };
}
// useCounter.test.js
import { renderHook, act } from '@testing-library/react';
import { useCounter } from './useCounter';

test('devrait incrémenter la valeur', () => {
  const { result } = renderHook(() => useCounter(0));
  expect(result.current.count).toBe(0);
  act(() => { result.current.increment(); });
  expect(result.current.count).toBe(1);
});
⚠️ act() est nécessaire pour que les mises à jour d'état soient correctement traitées.
🧠

Bonnes pratiques de test

  • Tester le comportement visible (ce que l'utilisateur voit/fait), pas les détails internes.
  • Utiliser getByRole, getByText, getByLabelText plutôt que getByTestId quand possible.
  • Nettoyer les mocks après chaque test (vi.clearAllMocks()).
  • Un test = une assertion principale (ou quelques-unes logiquement liées).
  • Éviter de tester des bibliothèques externes (React lui-même, Axios…).
