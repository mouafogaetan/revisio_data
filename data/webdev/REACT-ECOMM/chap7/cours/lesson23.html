Construisons une interface de catalogue performante et agréable pour vos clients.

🎯 Objectifs de la leçon

  • Afficher les produits en mode grille ou liste (toggle).
  • Implémenter une sidebar de filtres (prix, catégories, notes).
  • Ajouter des options de tri (prix croissant/décroissant, popularité, nouveauté).
  • Combiner filtres + tri pour une recherche avancée côté client ou serveur.

Un bon système de filtrage augmente le taux de conversion et l'expérience utilisateur.

const [filters, setFilters] = useState({ priceRange: [0,500], categories: [], minRating: 0 });
const filteredProducts = useMemo(() => {
  return products.filter(p =>
    p.price >= filters.priceRange[0] && p.price <= filters.priceRange[1] &&
    (filters.categories.length===0 || filters.categories.includes(p.categoryId)) &&
    p.rating >= filters.minRating
  );
}, [products, filters]);

Pour un grand nombre de produits, préférez le filtrage côté API (avec React Query).

Options de tri

Trier par :
const [sortBy, setSortBy] = useState('price-asc');
const sortedProducts = useMemo(() => {
  const sorted = [...filteredProducts];
  switch(sortBy) {
    case 'price-asc': return sorted.sort((a,b) => a.price - b.price);
    case 'price-desc': return sorted.sort((a,b) => b.price - a.price);
    case 'rating': return sorted.sort((a,b) => b.rating - a.rating);
    default: return sorted;
  }
}, [filteredProducts, sortBy]);

⚡ Combiner useMemo pour éviter des recalculs inutiles à chaque render.

Permet de partager un lien filtré ou de conserver les filtres après rechargement.

Exercice : Mise en pratique

🎯 À réaliser (sur votre projet e-commerce) :

  1. Créer une page /catalogue qui récupère les produits depuis l'API.
  2. Ajouter un bouton pour basculer entre vue grille et vue liste (state local ou store).
  3. Implémenter une barre latérale avec : filtre par prix (slider), filtre par catégorie (checkbox) et filtre par note (select).
  4. Ajouter une liste déroulante pour trier (prix asc/desc, popularité).
  5. Utiliser useMemo pour optimiser le filtrage/tri si fait côté client.
  6. Bonus : persister les filtres dans l'URL avec useSearchParams.
Pour aller plus loin : Que se passe-t-il si le nombre de produits dépasse 1000 ? (Indice : filtrage côté serveur + pagination)
