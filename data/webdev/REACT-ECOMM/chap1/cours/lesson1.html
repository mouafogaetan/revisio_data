🎯 Objectifs de la leçon

  • ✓ Définir les écrans fonctionnels d'un e-commerce (home, catalogue, produit, panier, checkout, compte)
  • ✓ Créer des wireframes avec Figma (maquettes basse fidélité)
  • ✓ Définir une charte graphique : couleurs, typographie, composants UI
💡 Pourquoi ? Une bonne architecture UI facilite le développement, l'expérience utilisateur et la maintenance.

📱 Les 6 écrans essentiels d'un e-commerce

🏠 Home (Page d'accueil)

Hero banner, produits vedettes, catégories populaires, promotions

📋 Catalogue / Liste produits

Grille/liste, filtres (prix, catégorie), tri, recherche

🔍 Détail produit

Images, description, variations (taille/couleur), avis, quantité

🛒 Panier

Récapitulatif, modification quantités, suppression, sous-total

📦 Checkout (Commande)

Adresse livraison, modes livraison, paiement, validation

👤 Compte utilisateur

Profil, commandes passées, adresses, favoris, avis

+ écrans admin : Dashboard, gestion produits, commandes, utilisateurs

🔤 Typographie et hiérarchie

font-sans (Inter, Poppins) — Titres H1
text-2xl font-semibold — Sous-titres H2
text-base — Corps de texte (16px)
text-sm text-gray-500 — Légendes, mentions légales

Bonnes pratiques :

  • Éviter plus de 2 polices différentes
  • Taille de base : 16px pour une lisibilité optimale
  • Hauteur de ligne (line-height) : 1.5 pour le corps

🧩 Composants UI réutilisables

Atomic Design : atomes → molécules → organismes → templates → pages

Atomes : Button, Input, Label, Badge
Molécules : SearchBar, ProductCard, CartItem
Organismes : Navbar, ProductGrid, FilterSidebar
Templates : Layout principal, CheckoutLayout
// Exemple d'atome Button en React
const Button = ({ children, variant, onClick }) => (
  <button className={`px-4 py-2 rounded-lg font-semibold 
    ${variant === 'primary' ? 'bg-amber-500 text-white' : 'bg-gray-200'}`}
    onClick={onClick}>{children}</button>
);

📱 Mobile-first et accessibilité

Points de rupture responsives (Tailwind) :

  • sm:640px → tablettes
  • md:768px → petits écrans
  • lg:1024px → desktop
  • xl:1280px → grands écrans

♿ Accessibilité (a11y) :

  • Utiliser des balises sémantiques (main, nav, section)
  • Attributs ARIA pour les composants dynamiques
  • Contraste des couleurs (ratio 4.5:1 minimum)
  • Navigation au clavier (tabindex, focus visible)
