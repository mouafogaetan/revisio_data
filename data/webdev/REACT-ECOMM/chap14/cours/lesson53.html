1. Intégration de Sentry

Sentry capture les exceptions non catchées, les erreurs réseau et fournit un contexte détaillé (navigateur, version, stack trace).

npm install @sentry/react @sentry/tracing
// main.jsx ou index.js
import * as Sentry from "@sentry/react";
import { BrowserTracing } from "@sentry/tracing";

Sentry.init({
  dsn: "https://votre-dsn@sentry.io/projet",
  integrations: [new BrowserTracing()],
  tracesSampleRate: 1.0, // pourcentage des transactions capturées
  environment: import.meta.env.MODE, // 'development' ou 'production'
  beforeSend(event) {
    // Ne pas envoyer les erreurs en dev (optionnel)
    if (import.meta.env.DEV) return null;
    return event;
  }
});
⚠️ Le DSN est une variable d'environnement : VITE_SENTRY_DSN. Ne jamais le hardcoder.
🎣

Capturer des erreurs ciblées

Dans les blocs try/catch ou les promesses, on envoie manuellement des exceptions à Sentry.

import * as Sentry from "@sentry/react";

async function addToCart(productId) {
  try {
    await api.addToCart(productId);
    toast.success("Ajouté au panier");
  } catch (error) {
    // Capturer l'erreur avec contexte supplémentaire
    Sentry.captureException(error, {
      extra: { productId, userId: getCurrentUserId() },
      tags: { feature: "cart" }
    });
    toast.error("Erreur lors de l'ajout");
  }
}

// Ajouter du contexte utilisateur
Sentry.setUser({ email: user.email, id: user.id });
✅ Sentry regroupe les erreurs similaires et vous alerte (email, Slack, etc.).

2. Google Analytics 4 (GA4)

On utilise le package react-ga4 ou le snippet officiel.

npm install react-ga4
// src/lib/analytics.js
import ReactGA from "react-ga4";

const GA_MEASUREMENT_ID = import.meta.env.VITE_GA_MEASUREMENT_ID;

export const initGA = () => {
  if (GA_MEASUREMENT_ID) {
    ReactGA.initialize(GA_MEASUREMENT_ID);
  }
};

export const sendPageView = (path) => {
  ReactGA.send({ hitType: "pageview", page: path });
};

// App.jsx
useEffect(() => {
  initGA();
  sendPageView(location.pathname);
}, [location]);
📌 Alternative : utiliser le script gtag directement dans index.html. Mais react-ga4 offre plus de contrôle.
🎯

Tracking d'événements métier

On suit les actions clés : ajout panier, début checkout, paiement réussi.

// services/tracking.js
import ReactGA from "react-ga4";

export const trackAddToCart = (product) => {
  ReactGA.event({
    category: "Ecommerce",
    action: "add_to_cart",
    label: product.name,
    value: product.price,
    // Paramètres personnalisés
    items: [{
      item_id: product.id,
      item_name: product.name,
      price: product.price,
      quantity: 1
    }]
  });
};

export const trackPurchase = (order) => {
  ReactGA.event({
    category: "Ecommerce",
    action: "purchase",
    value: order.total,
    transaction_id: order.id,
    items: order.items
  });
};
✨ Ces données permettent de calculer le taux de conversion, le panier moyen, et d'optimiser le tunnel d'achat.
🔒

Conformité & bonnes pratiques

  • RGPD / Consentement : demander l'autorisation avant d'activer GA4 ou Sentry (bannière cookies).
  • Anonymiser les IPs dans GA4 (option anonymize_ip: true).
  • Ne jamais envoyer de données sensibles (mot de passe, carte bancaire) à Sentry ou GA.
  • Environnements : désactiver Sentry en dev pour ne pas polluer le dashboard.
✅ Consentement via react-cookie-consent
✅ Vérifier la politique de confidentialité

🏁 Synthèse & commandes clés

Sentry :

Capture automatique des erreurs + Sentry.captureException() pour les cas métier.

Google Analytics 4 :

Pageviews automatiques + événements personnalisés (add_to_cart, begin_checkout, purchase).

Bonnes pratiques :

Variables d'environnement, consentement utilisateur, anonymisation des IP.

🎯 Résultat : Vous êtes capable de surveiller votre application en production et d'analyser le comportement des utilisateurs pour améliorer continuellement votre site e-commerce.
