🛡️ ProtectedRoute : Redirection si non authentifié

Ce composant vérifie si l'utilisateur est connecté. Si non, il redirige vers /login tout en sauvegardant la page qu'il voulait visiter.

// components/ProtectedRoute.jsx
import { Navigate, Outlet, useLocation } from 'react-router-dom';
import useAuthStore from '../stores/authStore';

const ProtectedRoute = () => {
  const isAuthenticated = useAuthStore((state) => state.isAuthenticated);
  const location = useLocation();

  if (!isAuthenticated) {
    // Redirige vers login mais garde en mémoire la page demandée
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  return <Outlet />;
};

export default ProtectedRoute;
📌 Le state={{ from: location }} permet après connexion de revenir à la page initialement demandée.

↩️ Redirection si déjà connecté

Un utilisateur déjà authentifié ne doit pas avoir accès aux pages /login ou /register. Il doit être redirigé vers la page d'accueil ou le dashboard.

// components/PublicRoute.jsx (ou directement dans LoginPage)
import { Navigate } from 'react-router-dom';
import useAuthStore from '../stores/authStore';

const LoginPage = () => {
  const isAuthenticated = useAuthStore((state) => state.isAuthenticated);

  if (isAuthenticated) {
    return <Navigate to="/" replace />;
  }

  return (
    <div>
      {/* Formulaire de connexion */}
    </div>
  );
};

export default LoginPage;
✅ Cela évite qu'un utilisateur connecté tente de se reconnecter ou de créer un nouveau compte.

💾 Persistance de session avec Zustand

Pour que l'utilisateur reste connecté après un rafraîchissement, on utilise le middleware persist de Zustand avec localStorage.

// stores/authStore.js
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

const useAuthStore = create(
  persist(
    (set) => ({
      user: null,
      token: null,
      isAuthenticated: false,

      login: (userData, token) => set({
        user: userData,
        token: token,
        isAuthenticated: true
      }),

      logout: () => set({
        user: null,
        token: null,
        isAuthenticated: false
      }),
    }),
    {
      name: 'auth-storage', // clé dans localStorage
      getStorage: () => localStorage,
    }
  )
);

🎯 Après rafraîchissement, le store est rechargé depuis localStorage → l'utilisateur reste connecté.

🔄 Flux complet : connexion + redirection

Le composant ProtectedRoute sauvegarde la page demandée et y redirige après connexion.

↪️ Redirection après connexion réussie

// pages/LoginPage.jsx
import { useNavigate, useLocation } from 'react-router-dom';
import useAuthStore from '../stores/authStore';

const LoginPage = () => {
  const navigate = useNavigate();
  const location = useLocation();
  const login = useAuthStore((state) => state.login);

  const from = location.state?.from?.pathname || '/';

  const handleSubmit = async (email, password) => {
    try {
      const { user, token } = await authService.login(email, password);
      login(user, token);
      navigate(from, { replace: true }); // ← redirection vers la page d'origine
    } catch (error) {
      toast.error('Identifiants incorrects');
    }
  };

  // ...
};
💡 location.state?.from?.pathname récupère la page que l'utilisateur voulait visiter avant d'être redirigé vers login.

🚪 Déconnexion et nettoyage

// components/LogoutButton.jsx
import { useNavigate } from 'react-router-dom';
import useAuthStore from '../stores/authStore';
import useCartStore from '../stores/cartStore';

const LogoutButton = () => {
  const logout = useAuthStore((state) => state.logout);
  const clearCart = useCartStore((state) => state.clearCart);
  const navigate = useNavigate();

  const handleLogout = () => {
    logout();           // vide le store auth
    clearCart();        // vide le panier
    navigate('/login'); // redirige vers login
  };

  return <button onClick={handleLogout} className="...">Se déconnecter</button>;
};
⚠️ Important : vider également les données sensibles (panier, favoris) lors de la déconnexion.

👑 Redirection basée sur les rôles

// components/RoleBasedRoute.jsx
import { Navigate } from 'react-router-dom';
import useAuthStore from '../stores/authStore';

const RoleBasedRoute = ({ allowedRoles, children }) => {
  const user = useAuthStore((state) => state.user);
  const isAuthenticated = useAuthStore((state) => state.isAuthenticated);

  if (!isAuthenticated) return <Navigate to="/login" />;
  if (!allowedRoles.includes(user?.role)) return <Navigate to="/unauthorized" />;

  return children;
};

// Utilisation dans les routes
<Route path="/admin" element={
  <RoleBasedRoute allowedRoles={['admin']}>
    <AdminDashboard />
  </RoleBasedRoute>
} />

🎯 Les utilisateurs non autorisés sont redirigés vers une page d'erreur 403.

✅ Bonnes pratiques

  • Utiliser Navigate replace pour éviter de polluer l'historique
  • Sauvegarder la page d'origine (location.state.from) pour une redirection intelligente
  • Ne jamais stocker de données sensibles en clair (token OK car signé)
  • Vider le store et localStorage lors de la déconnexion
  • Protéger aussi les routes côté API (ne pas se fier uniquement au frontend)
  • Utiliser un PrivateRoute wrapper pour toutes les routes nécessitant auth
