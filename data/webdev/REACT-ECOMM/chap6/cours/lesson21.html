1. Installation et configuration

On utilise la bibliothèque officielle @react-oauth/google pour gérer le flux OAuth2.

npm install @react-oauth/google

Ensuite, il faut envelopper l'application avec GoogleOAuthProvider et fournir votre Client ID Google (créé dans Google Cloud Console).

import { GoogleOAuthProvider } from '@react-oauth/google';

function App() {
  return (
    <GoogleOAuthProvider clientId="VOTRE_CLIENT_ID.apps.googleusercontent.com">
      <Routes />
    </GoogleOAuthProvider>
  );
}
📌 Récupérez votre Client ID depuis la Google Cloud Console → API & Services → Identifiants. Activez l'API People si besoin.
🔘

2. Bouton Google et callback

Le composant GoogleLogin affiche un bouton prêt à l'emploi. On récupère le jeton d'identité (credential) et on l'envoie à notre backend.

import { GoogleLogin } from '@react-oauth/google';

function LoginPage() {
  const handleSuccess = async (response) => {
    const { credential } = response; // token JWT Google
    // Envoyer au backend pour vérification et création de session
    await axios.post('/api/auth/google', { token: credential });
  };

  const handleError = () => {
    console.log('Erreur de connexion Google');
  };

  return (
    <GoogleLogin
      onSuccess={handleSuccess}
      onError={handleError}
      useOneTap
    />
  );
}
✅ À retenir : Le token reçu est un JWT signé par Google. Notre backend doit le vérifier (vérifier l'audience et l'émetteur).

3. Création / Liaison de comptes

Deux cas possibles côté backend :

  • L'email n'existe pas → créer un nouvel utilisateur avec le nom et avatar Google.
  • L'email existe déjà mais sans Google → lier le compte social (ajouter googleId à l'utilisateur existant).
// Exemple de logique backend (Node.js)
const { OAuth2Client } = require('google-auth-library');
const client = new OAuth2Client(process.env.GOOGLE_CLIENT_ID);

async function verifyGoogleToken(token) {
  const ticket = await client.verifyIdToken({
    idToken: token,
    audience: process.env.GOOGLE_CLIENT_ID,
  });
  const payload = ticket.getPayload();
  const { email, name, sub: googleId, picture } = payload;

  let user = await User.findOne({ email });
  if (!user) {
    user = await User.create({ email, name, googleId, avatar: picture });
  } else if (!user.googleId) {
    user.googleId = googleId;   // liaison
    await user.save();
  }
  return user;
}
✨ Avantage : l'utilisateur pourra se connecter aussi bien par email/mdp que par Google après la liaison.

🛡️ Bonnes pratiques & résumé

✅ Sécurité

  • Toujours vérifier le token côté serveur.
  • Utiliser HTTPS en production.
  • Ne pas stocker le token Google dans localStorage sans raison.

🚀 UX & redirections

  • Après succès, redirigez vers tableau de bord.
  • Gérer le chargement (spinner).
  • Offrir la possibilité de lier les comptes dans les paramètres.
