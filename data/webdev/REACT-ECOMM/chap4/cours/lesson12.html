🎛️ Leçon 12

État des produits, filtres & pagination

Centraliser les produits, gérer la recherche, les filtres (prix, catégorie) et la pagination avec Zustand.

👨‍🏫 Classe: Développement Web | React · Interface e-commerce

🎯 Objectifs de la leçon

  • ✓ Gérer l'état des produits et catégories avec Zustand
  • ✓ Implémenter des filtres dynamiques (prix, catégorie, recherche, tri)
  • ✓ Mettre en place la pagination client-side ou via API
  • ✓ Persister les filtres dans l'URL pour le partage

💡 À la fin, vous aurez une expérience de filtrage fluide et performante comme sur les grands sites e-commerce !

📦 Store produits avec Zustand

Un store central qui contient : la liste des produits, les filtres actifs, et des actions pour les modifier.

// stores/productStore.ts
import { create } from 'zustand';

interface Filters {
  search: string;
  category: string;
  minPrice: number;
  maxPrice: number;
  sortBy: 'price_asc' | 'price_desc' | 'name_asc';
}

interface ProductState {
  products: any[];
  filters: Filters;
  currentPage: number;
  setProducts: (products: any[]) => void;
  updateFilters: (newFilters: Partial<Filters>) => void;
  resetFilters: () => void;
  setPage: (page: number) => void;
}

export const useProductStore = create<ProductState>((set) => ({
  products: [],
  filters: { search: '', category: '', minPrice: 0, maxPrice: 10000, sortBy: 'name_asc' },
  currentPage: 1,
  setProducts: (products) => set({ products }),
  updateFilters: (newFilters) => set((state) => ({ 
    filters: { ...state.filters, ...newFilters },
    currentPage: 1  // réinitialiser la page à chaque filtre
  })),
  resetFilters: () => set({ 
    filters: { search: '', category: '', minPrice: 0, maxPrice: 10000, sortBy: 'name_asc' },
    currentPage: 1
  }),
  setPage: (page) => set({ currentPage: page })
}));

Ce store sera utilisé dans les composants pour lire et modifier les filtres, déclenchant le re-filtrage.

⚡ Filtrer et trier les produits

On crée un hook personnalisé ou un selector pour obtenir les produits filtrés en temps réel.

// hooks/useFilteredProducts.ts
import { useMemo } from 'react';
import { useProductStore } from '@/stores/productStore';

export const useFilteredProducts = () => {
  const { products, filters } = useProductStore();
  
  const filtered = useMemo(() => {
    let result = [...products];
    // Filtre par recherche (nom)
    if (filters.search) {
      result = result.filter(p => p.name.toLowerCase().includes(filters.search.toLowerCase()));
    }
    // Filtre catégorie
    if (filters.category) {
      result = result.filter(p => p.category === filters.category);
    }
    // Filtre prix
    result = result.filter(p => p.price >= filters.minPrice && p.price <= filters.maxPrice);
    // Tri
    if (filters.sortBy === 'price_asc') result.sort((a,b) => a.price - b.price);
    if (filters.sortBy === 'price_desc') result.sort((a,b) => b.price - a.price);
    if (filters.sortBy === 'name_asc') result.sort((a,b) => a.name.localeCompare(b.name));
    return result;
  }, [products, filters]);
  
  return filtered;
};

Ce hook réagit aux changements de filtres et met à jour l'affichage sans effort.

📄 Pagination côté client

On ajoute la pagination dans le même composant en utilisant le store.

// Composant ProductsList.tsx (extrait)
import { useProductStore } from '@/stores/productStore';
import { useFilteredProducts } from '@/hooks/useFilteredProducts';

const ProductsList = () => {
  const filteredProducts = useFilteredProducts();
  const { currentPage, setPage } = useProductStore();
  const itemsPerPage = 9;
  const totalPages = Math.ceil(filteredProducts.length / itemsPerPage);
  const paginated = filteredProducts.slice(
    (currentPage - 1) * itemsPerPage,
    currentPage * itemsPerPage
  );

  return (
    <div>
      <div className="grid grid-cols-1 md:grid-cols-3 gap-4">
        {paginated.map(product => <ProductCard key={product.id} {...product} />)}
      </div>
      <Pagination current={currentPage} total={totalPages} onPageChange={setPage} />
    </div>
  );
};

La pagination est automatiquement réinitialisée à 1 quand les filtres changent (déjà géré dans le store).

🎨 Sidebar filtres interactive

Exemple de composant qui met à jour le store à chaque interaction.

// components/FilterSidebar.tsx
import { useProductStore } from '@/stores/productStore';

export const FilterSidebar = () => {
  const { filters, updateFilters, resetFilters } = useProductStore();
  
  return (
    <div className="space-y-4 p-4 bg-gray-50 rounded-xl">
      <input 
        type="text" placeholder="Rechercher..."
        value={filters.search}
        onChange={(e) => updateFilters({ search: e.target.value })}
        className="w-full border rounded p-2"
      />
      <select value={filters.category} onChange={(e) => updateFilters({ category: e.target.value })}>
        <option value="">Toutes catégories</option>
        <option value="electronics">Électronique</option>
        <option value="clothing">Vêtements</option>
      </select>
      <div>
        <label>Prix min: {filters.minPrice}</label>
        <input type="range" min="0" max="500" step="10"
          value={filters.minPrice}
          onChange={(e) => updateFilters({ minPrice: Number(e.target.value) })}
        />
      </div>
      <button onClick={resetFilters} className="bg-gray-200 px-4 py-2 rounded">Réinitialiser</button>
    </div>
  );
};

Chaque modification déclenche updateFilters et la liste se met à jour en temps réel !

🔗 Synchronisation avec l'URL

Permet de partager un lien avec les filtres actifs. On utilise useSearchParams de React Router.

// hooks/useSyncFiltersWithUrl.ts
import { useEffect } from 'react';
import { useSearchParams } from 'react-router-dom';
import { useProductStore } from '@/stores/productStore';

export const useSyncFiltersWithUrl = () => {
  const [searchParams, setSearchParams] = useSearchParams();
  const { filters, updateFilters } = useProductStore();

  // Lecture initiale depuis l'URL
  useEffect(() => {
    const urlFilters = {
      search: searchParams.get('search') || '',
      category: searchParams.get('category') || '',
      minPrice: Number(searchParams.get('minPrice')) || 0,
      maxPrice: Number(searchParams.get('maxPrice')) || 10000,
      sortBy: (searchParams.get('sortBy') as any) || 'name_asc'
    };
    updateFilters(urlFilters);
  }, []);

  // Écriture dans l'URL quand les filtres changent
  useEffect(() => {
    setSearchParams({
      search: filters.search,
      category: filters.category,
      minPrice: String(filters.minPrice),
      maxPrice: String(filters.maxPrice),
      sortBy: filters.sortBy
    });
  }, [filters]);
};

✅ Ainsi, un client peut envoyer un lien de recherche filtrée à un ami, parfait pour l'UX !

🚀 Optimisations & astuces

📌 Debounce sur recherche

Utilisez useDebounce pour éviter trop de ré-rendus lors de la frappe.

🧩 Mémorisation

useMemo pour les produits filtrés évite des recalculs inutiles.

🌐 Pagination API

Pour des milliers de produits, préférez une pagination côté serveur avec React Query.

💾 Persistance

Sauvegarder les filtres dans localStorage ou sessionStorage ? Utile pour retour utilisateur.

🧪 Résumé : Un store Zustand gère l'état, des hooks dérivent les données filtrées, et l'UI reste simple et réactive.

📌 Résumé – Ce qu'il faut retenir

  • ✅ Zustand centralise produits, filtres et pagination.
  • ✅ Les filtres (recherche, catégorie, prix, tri) sont réactifs et déclenchent le re-filtrage.
  • ✅ La pagination client-side est simple pour des catalogues de taille modérée.
  • ✅ Synchroniser l'URL avec les filtres améliore le partage et l'UX.
