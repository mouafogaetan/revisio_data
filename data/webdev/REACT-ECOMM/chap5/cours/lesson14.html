Axios & Intercepteurs

Installer Axios, créer une instance, configurer intercepteurs pour gérer les tokens JWT et le refresh automatique

🔌 HTTP Client 🔄 Interceptors 🔐 JWT Refresh

🎯 Objectifs de la leçon

✓ Installer Axios et créer une instance configurée (baseURL, timeout)
✓ Configurer des intercepteurs de requête et de réponse
✓ Gérer automatiquement l'injection du token JWT
✓ Implémenter le rafraîchissement automatique du token (refresh token)
💡 Pourquoi Axios ? Il simplifie les requêtes HTTP, gère les erreurs, supporte les promesses et les intercepteurs qui sont parfaits pour l'authentification.

📦 Installation & instance Axios

On commence par installer Axios, puis on crée une instance centralisée avec la configuration de base.

npm install axios
// src/services/api.js
import axios from 'axios';

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL || 'http://localhost:3000/api',
  timeout: 10000,
  headers: { 'Content-Type': 'application/json' }
});

export default api;
✨ L'instance évite de répéter la baseURL partout. On peut aussi y ajouter des paramètres par défaut.

🔌 Intercepteur de requête : Ajouter le token JWT

L'intercepteur capture chaque requête sortante et y injecte le token d'accès s'il existe.

// Intercepteur de requête
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('access_token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);
📌 Ce code se place juste après la création de l'instance. Toutes les requêtes passeront automatiquement par cet intercepteur.

🔄 Intercepteur de réponse & Refresh Token

Si une requête échoue avec 401 (token expiré), on tente de rafraîchir le token et de rejouer la requête.

let isRefreshing = false;
let failedQueue = [];

api.interceptors.response.use(
  (response) => response,
  async (error) => {
    const originalRequest = error.config;
    if (error.response?.status === 401 && !originalRequest._retry) {
      if (isRefreshing) {
        return new Promise((resolve, reject) => {
          failedQueue.push({ resolve, reject });
        }).then(() => api(originalRequest));
      }
      originalRequest._retry = true;
      isRefreshing = true;
      try {
        const refreshToken = localStorage.getItem('refresh_token');
        const { data } = await axios.post('/auth/refresh', { refreshToken });
        localStorage.setItem('access_token', data.access_token);
        api.defaults.headers.common['Authorization'] = `Bearer ${data.access_token}`;
        failedQueue.forEach(prom => prom.resolve());
        return api(originalRequest);
      } catch (refreshError) {
        failedQueue.forEach(prom => prom.reject(refreshError));
        // Rediriger vers login
        window.location.href = '/login';
        return Promise.reject(refreshError);
      } finally {
        isRefreshing = false;
        failedQueue = [];
      }
    }
    return Promise.reject(error);
});
⚠️ Ce pattern empêche les appels parallèles multiples lors du refresh.

⚛️ Utilisation dans un composant React

Une fois l'instance configurée, il suffit d'importer `api` et de faire des requêtes.

// ProductList.jsx
import { useEffect, useState } from 'react';
import api from '../services/api';

export default function ProductList() {
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchProducts = async () => {
      try {
        const { data } = await api.get('/products');
        setProducts(data);
      } catch (err) {
        console.error('Erreur:', err);
      } finally {
        setLoading(false);
      }
    };
    fetchProducts();
  }, []);

  if (loading) return <div>Chargement...</div>
  return <div>{/* affichage produits */}</div>
}
✅ Les intercepteurs s'appliquent automatiquement, token et refresh sont gérés de manière transparente.

🧠 Bonnes pratiques & astuces

🔐 Stockage sécurisé

Préférez `httpOnly` cookies pour le refresh token, mais si localStorage, attention au XSS.

⏱️ Timeout & retry

Configurer `timeout` et éventuellement un mécanisme de retry avec `axios-retry`.

🧪 Env variables

Utiliser `import.meta.env.VITE_API_URL` avec Vite pour switcher entre dev/prod.

📡 Logging

Ajouter un intercepteur pour logger les requêtes (utile en développement).

⚠️ Erreur fréquente : Oublier de retourner `config` dans l'intercepteur de requête → la requête est bloquée.

const api = axios.create({ baseURL: 'https://api.ton-ecommerce.com/v1' });
api.interceptors.request.use(config => { const token = localStorage.getItem('token'); if(token) config.headers.Authorization = `Bearer ${token}`; return config; });
