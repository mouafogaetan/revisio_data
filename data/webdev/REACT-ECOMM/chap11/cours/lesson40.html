Diviser son bundle pour charger plus vite — Améliorer l’expérience utilisateur

🧠 CONCEPT CLÉ

Pourquoi diviser son code ?

Dans une application e-commerce, le bundle initial peut rapidement devenir très lourd (bibliothèques UI, pages panier, checkout, dashboard admin). Le code splitting permet de ne charger que le code nécessaire pour la page affichée, réduisant ainsi le First Paint et le Time To Interactive.


à la demande

👉 On charge d’abord le strict nécessaire, puis les autres morceaux (chunks) au moment où l’utilisateur en a besoin.

⚛️ React.lazy & Suspense

React.lazy() est une fonction qui permet de charger dynamiquement un composant. On l’associe à Suspense qui affiche un fallback (spinner, skeleton) pendant le chargement.

// Au lieu d’importer normalement // import ProductPage from './pages/ProductPage'; // Chargement paresseux const ProductPage = React.lazy(() => import('./pages/ProductPage')); function App() { return ( <Suspense fallback={<Spinner />}> <ProductPage /> </Suspense> ); }
⚠️ Important : React.lazy ne fonctionne qu’avec les exports par défaut. Et il doit toujours être utilisé à l’intérieur d’un composant Suspense.

🗺️ Code splitting par route

Pratique recommandée

Dans un site e-commerce, on applique le lazy loading sur chaque page principale : catalogue, produit, panier, checkout, compte, admin. Ainsi le bundle initial reste léger.

import { BrowserRouter, Routes, Route } from 'react-router-dom'; import { lazy, Suspense } from 'react'; const Home = lazy(() => import('./pages/Home')); const Catalog = lazy(() => import('./pages/Catalog')); const Cart = lazy(() => import('./pages/Cart')); const Checkout = lazy(() => import('./pages/Checkout')); function Router() { return ( <Suspense fallback={<div className="p-8 text-center">Chargement...</div>}> <Routes> <Route path="/" element={<Home />} /> <Route path="/catalog" element={<Catalog />} /> <Route path="/cart" element={<Cart />} /> <Route path="/checkout" element={<Checkout />} /> </Routes> </Suspense> ); }

✅ Résultat : le code de la page "Checkout" (lourd, avec Stripe, formulaires) n’est chargé que lorsque l’utilisateur clique sur le bouton de validation.

⏩ Précharger au survol

Pour rendre la navigation instantanée, on peut démarrer le chargement d’un chunk avant que l’utilisateur ne clique, par exemple quand il survole un lien.

import { lazy } from 'react'; const LazyProductPage = lazy(() => import('./pages/ProductPage')); function ProductLink() { const handlePreload = () => { // déclenche le chargement anticipé import('./pages/ProductPage'); }; return ( <Link to="/product/123" onMouseEnter={handlePreload}> Voir le produit </Link> ); }
💡 Cette technique combinée à Suspense rend l’expérience quasi instantanée : le chunk est déjà prêt quand on clique.

📦 Outils : groupement & analyse

  • Vite (ou Webpack) génère automatiquement des chunks pour chaque lazy.
  • Utiliser rollup-plugin-visualizer ou vite-bundle-visualizer pour voir la taille des chunks.
  • Possibilité de nommer les chunks avec webpackChunkName : /* webpackChunkName: "checkout" */
  • Configurer Vite pour optimiser le code splitting des librairies externes via build.rollupOptions.output.manualChunks.
// Exemple avec Vite (vite.config.js) export default defineConfig({ build: { rollupOptions: { output: { manualChunks: { 'vendor-react': ['react', 'react-dom', 'react-router-dom'], 'ui-lib': ['@headlessui/react', 'lucide-react'] } } } } });

⚠️ Gestion des erreurs & fallbacks

Production-ready

Si un chunk échoue à charger (panne réseau, erreur serveur), il faut prévoir une UI de secours et un mécanisme de rechargement.

import { ErrorBoundary } from 'react-error-boundary'; function ErrorFallback({ error, resetErrorBoundary }) { return ( <div className="p-4 text-center"> <p>Erreur de chargement : {error.message}</p> <button onClick={resetErrorBoundary}>Réessayer</button> </div> ); } <ErrorBoundary FallbackComponent={ErrorFallback}> <Suspense fallback={<Spinner />}> <LazyComponent /> </Suspense> </ErrorBoundary>

🎯 On combine ainsi ErrorBoundary + Suspense pour une expérience robuste.
