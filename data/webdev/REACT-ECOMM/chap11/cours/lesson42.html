Optimisation des images

⚡ L'importance des images pour les performances

📊 Statistiques

Les images représentent en moyenne 60% du poids d'une page web. Sur mobile, chaque seconde de chargement supplémentaire peut réduire les conversions de 20%.

🏆 Core Web Vitals

Google prend en compte le LCP (Largest Contentful Paint) : l'image principale doit charger rapidement.

📱 Images responsives avec srcset

L'attribut srcset permet de fournir plusieurs versions d'une même image selon la largeur d'écran.

<img
  src="image-small.jpg"
  srcset="image-small.jpg 480w, image-medium.jpg 800w, image-large.jpg 1200w"
  sizes="(max-width: 600px) 480px, (max-width: 1200px) 800px, 1200px"
  alt="Produit e-commerce"
  loading="lazy"
/>

Le navigateur choisit la meilleure résolution selon la taille de l'écran et la densité de pixels.

✅ En React, on peut générer dynamiquement les srcset via un composant Image personnalisé.

⏳ Lazy loading : charger les images au besoin

// 1. Lazy loading natif HTML
<img src="photo.jpg" loading="lazy" alt="..." />

// 2. Avec React Intersection Observer (plus de contrôle)
import { useInView } from 'react-intersection-observer';

const LazyImage = ({ src, alt, className }) => {
  const { ref, inView } = useInView({ triggerOnce: true, rootMargin: '200px' });
  
  return (
    <div ref={ref} className={className}>
      {inView && <img src={src} alt={alt} loading="lazy" />}
      {!inView && <div className="bg-gray-200 animate-pulse w-full h-full" />}
    </div>
  );
};
skeleton
Affiche un placeholder (skeleton) tant que l'image n'est pas dans le viewport.

☁️ Utiliser un CDN d'images

Cloudinary, Imgix ou Uploadcare permettent de transformer les images à la volée : redimensionnement, qualité, format (WebP, AVIF).

// Exemple avec Cloudinary
// URL de base : https://res.cloudinary.com/demo/image/upload/

// Image originale : v1620000000/sample.jpg
// Redimensionnée en 300x300 : w_300,h_300,c_fill
// Qualité réduite : q_auto
// Format WebP : f_auto

const imageUrl = `https://res.cloudinary.com/demo/image/upload/w_300,h_300,c_fill,q_auto,f_auto/sample.jpg`;

// Composant React générique
const OptimizedImage = ({ publicId, width, height, alt }) => {
  const url = `https://res.cloudinary.com/demo/image/upload/w_${width},h_${height},c_fill,q_auto,f_auto/${publicId}`;
  return <img src={url} alt={alt} loading="lazy" className="w-full h-auto" />
};
📌 Avantage : transformation à la volée, cache global, format next-gen (WebP/AVIF) automatique.

🎨 Composant Image générique pour e-commerce

// components/ui/OptimizedImage.tsx
interface Props {
  src: string;
  alt: string;
  width?: number;
  height?: number;
  className?: string;
  sizes?: string;
}

export const OptimizedImage = ({ src, alt, width, height, className, sizes }: Props) => {
  // Génère les srcset pour différentes largeurs
  const generateSrcSet = () => {
    if (!src.includes('cloudinary.com')) return undefined;
    const baseUrl = src.split('/upload/')[0] + '/upload/';
    const publicId = src.split('/upload/')[1];
    const widths = [480, 800, 1200];
    return widths.map(w => `${baseUrl}w_${w},c_limit/${publicId} ${w}w`).join(', ');
  };

  return (
    <img
      src={src}
      srcSet={generateSrcSet()}
      sizes={sizes || '(max-width: 640px) 100vw, 50vw'}
      width={width}
      height={height}
      alt={alt}
      loading="lazy"
      className={className}
    />
  );
};

Ce composant peut être utilisé partout : catalogue, page produit, panier, etc.

📌 Résumé – Ce qu'il faut retenir

  • ✅ Images responsives : utiliser srcset et sizes pour adapter la résolution.
  • ✅ Lazy loading : loading="lazy" ou intersection observer + skeleton.
  • ✅ CDN d'images : Cloudinary/Imgix pour redimensionnement, optimisation et formats modernes.
  • ✅ Composant réutilisable : centraliser la logique d'optimisation dans toute l'app.
