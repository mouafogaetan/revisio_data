❓ Pourquoi ajouter des notifications ?

Une notification (toast) donne un feedback immédiat à l'utilisateur :

  • Confirmation d'action (article ajouté au panier ✅)
  • Erreur lors du paiement ❌
  • Information : promo appliquée, livraison offerte ℹ️
  • Promesse de chargement asynchrone (mise à jour)

📱 Sur mobile, les toasts discrets n’interrompent pas l’expérience.

+-----------------------------------+
|  🛒 1 produit ajouté au panier   ✖ |
+-----------------------------------+

Exemple visuel d’un toast qui apparaît en bas à droite (ou en haut sur mobile).

📦 Installation & Configuration

Option A : sonner (recommandé moderne)

npm install sonner

Ensuite dans main.jsx ou App.jsx :

import { Toaster } from 'sonner';

function App() {
  return (
    <>
      <Toaster position="top-right" />
      {/* vos routes */}
    </>
  );
}

Option B : react-hot-toast

npm install react-hot-toast
import { Toaster } from 'react-hot-toast';

function App() {
  return (
    <>
      <Toaster position="bottom-center" />
      ...
    </>
  );
}
✅ Une fois le Toaster placé, vous pouvez utiliser les hooks `toast()` partout.

🎨 Notifications : succès, erreur, info, promesse

Avec sonner ou react-hot-toast, la syntaxe est très simple :

import { toast } from 'sonner';

// Succès
toast.success('✅ Produit ajouté au panier !');

// Erreur
toast.error('❌ Échec du paiement, réessayez.');

// Info simple
toast.info('📦 Livraison gratuite dès 50€');

// Promesse (async)
toast.promise(
  fetch('/api/order'),
  {
    loading: 'Création de la commande...',
    success: 'Commande validée !',
    error: 'Une erreur est survenue',
  }
);

* Démo interactive (simulation de toast) → un message fictif apparaît ci-dessous

🛒 Notifications persistantes & actions

Pour un panier ou une commande, vous pouvez afficher un toast avec une durée illimitée ou un bouton d'action :

// sonner avec bouton
toast('Article ajouté', {
  action: {
    label: 'Voir le panier',
    onClick: () => router.push('/cart'),
  },
  duration: 5000, // 5 secondes
});

// Persistant jusqu'à action manuelle
toast.success('Commande confirmée !', {
  duration: Infinity,
});

✨ Conseil UX : Après l’ajout au panier, un toast avec lien direct vers le panier améliore le taux de conversion.

🔔 Notification persistante : "Votre commande #RE1004 est expédiée" reste affichée jusqu'à ce que l'utilisateur la ferme.

🎨 Personnalisation (style, position, thème)

Les deux bibliothèques offrent des options puissantes :

  • position : top-right, bottom-center, top-left...
  • duration : durée en ms (2000 par défaut)
  • icon : icône personnalisée (JSX)
  • style : CSS inline pour surcharger l'apparence
<Toaster 
  position="bottom-right"
  toastOptions={{
    style: { background: '#1f2937', color: '#fff' },
    duration: 3000,
  }}
/>

// Appel personnalisé
toast('🎉 Promotion -10%', {
  icon: '🏷️',
  style: { background: '#0f172a', color: '#facc15' }
});

Sous le capot, chaque toast est un composant React que vous pouvez même remplacer entièrement.

🛍️ Intégration dans une boutique React

Scénario typique : l'utilisateur clique sur "Ajouter au panier" → un toast apparaît.

const addToCart = (product) => {
  cartStore.addItem(product);
  toast.success(`${product.name} ajouté au panier`, {
    action: { label: 'Panier', onClick: () => navigate('/cart') }
  });
};

// Si erreur réseau
const checkout = async () => {
  try {
    await api.createOrder();
    toast.success('Commande validée !');
  } catch(err) {
    toast.error('Problème de connexion');
  }
};

💡 Bonnes pratiques : évitez les toasts intempestifs (pas plus de 2 en même temps). Utilisez des couleurs distinctes (vert → succès, rouge → erreur).
