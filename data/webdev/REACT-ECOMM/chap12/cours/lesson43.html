UX : Skeletons, Spinners & Fallback UI

Améliorez l'expérience utilisateur avec des indicateurs de chargement et des interfaces de secours élégantes

🦴 Skeleton screens ⏳ Spinners 🛡️ Fallback UI

🎯 Objectifs de la leçon

✓ Afficher des squelettes (skeleton screens) pendant le chargement des données
✓ Utiliser des spinners et barres de progression adaptés
✓ Gérer les erreurs avec des fallback UI (composant ErrorBoundary, messages)
✓ Implémenter des états de chargement optimistes (mutations)
💡 Pourquoi ? Une bonne UX réduit la perception du temps d'attente et guide l'utilisateur.

🦴 Skeleton screens : l'alternative élégante

Un skeleton montre la structure de la page avant que le contenu ne soit chargé. Il rassure l'utilisateur.

// ProductSkeleton.jsx
export default function ProductSkeleton() {
  return (
    <div className="animate-pulse space-y-3">
      <div className="bg-gray-200 h-48 rounded-xl"></div>
      <div className="h-4 bg-gray-200 rounded w-3/4"></div>
      <div className="h-4 bg-gray-200 rounded w-1/2"></div>
    </div>
  );
}

// Utilisation avec React Query
const { data, isLoading } = useQuery(...);
if (isLoading) return <ProductSkeleton />;

Exemple visuel : skeleton de carte produit.

⏳ Spinners & barres de progression

Pour les actions longues (upload, paiement), un spinner ou une barre de progression est indispensable.

// Composant Spinner
const Spinner = () => (
  <div className="flex justify-center items-center">
    <div className="animate-spin rounded-full h-8 w-8 border-b-2 border-indigo-600"></div>
  </div>
);

// Utilisation dans une mutation
const { mutate, isPending } = useMutation(...);
if (isPending) return <Spinner />;

🛡️ Error Boundaries & Fallback UI

Les erreurs (réseau, serveur) doivent être gérées avec une interface de secours claire.

// ErrorFallback.jsx
export default function ErrorFallback({ error, resetErrorBoundary }) {
  return (
    <div className="text-center p-8 bg-red-50 rounded-xl">
      <h3>Une erreur est survenue</h3>
      <p className="text-red-600">{error.message}</p>
      <button onClick={resetErrorBoundary} className="btn">Réessayer</button>
    </div>
  );
}

// Avec ErrorBoundary de react-error-boundary
<ErrorBoundary FallbackComponent={ErrorFallback}>
  <ProductList />
</ErrorBoundary>
⚠️ Les Error Boundaries capturent les erreurs lors du rendu, des cycles de vie et des constructeurs (pas les erreurs asynchrones ou les callbacks).

⚡ Mises à jour optimistes

Améliore la réactivité : on met à jour l'UI immédiatement, on annule en cas d'erreur.

// Exemple avec React Query
const mutation = useMutation({
  mutationFn: (newItem) => api.addToCart(newItem),
  onMutate: async (newItem) => {
    await queryClient.cancelQueries(['cart']);
    const previousCart = queryClient.getQueryData(['cart']);
    queryClient.setQueryData(['cart'], (old) => [...old, newItem]);
    return { previousCart };
  },
  onError: (err, newItem, context) => {
    queryClient.setQueryData(['cart'], context.previousCart);
    toast.error('Échec ajout panier');
  }
});

💡 Idéal pour les actions fréquentes : likes, ajout au panier, votes.

📌 Bonnes pratiques UX

🎯 Progresser par étapes
Afficher d'abord le squelette, puis le contenu réel. Éviter les flashs de contenu.
🎨 Skeletons cohérents
Ils doivent ressembler à la mise en page finale (couleurs, formes).
🔁 Retry explicite
Proposer un bouton "Réessayer" en cas d'erreur, pas seulement un message.
📱 Accessibilité
Ajouter `aria-label="Chargement"` sur les spinners, éviter les animations trop rapides.
