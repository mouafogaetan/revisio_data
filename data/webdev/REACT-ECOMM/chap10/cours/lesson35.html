Interface d'administration

Layout séparé, Sidebar dédiée et protection des routes admin.

📁 Layout admin 📌 Sidebar navigation 🔐 Vérification des rôles

🎯 Objectifs de la leçon

1️⃣

Layout séparé

Créer un layout différent pour l’espace admin (Dashboard, produits, commandes).

2️⃣

Sidebar navigation

Menu latéral avec liens vers les pages admin.

3️⃣

Vérification des rôles

Protéger les routes : redirection si l’utilisateur n’est pas admin.

💡 Un espace admin bien isolé renforce la sécurité et l’organisation du code.

🏗️ Layout séparé pour l’admin

On crée un composant AdminLayout qui contient la Sidebar + un espace pour le contenu (Outlet).

// layouts/AdminLayout.jsx
import { Outlet } from "react-router-dom";
import AdminSidebar from "../components/AdminSidebar";

export default function AdminLayout() {
  return (
    <div className="flex min-h-screen">
      <AdminSidebar />
      <main className="flex-1 p-6 bg-gray-100">
        <Outlet />
      </main>
    </div>
  );
}

Les routes admin sont placées sous ce layout dans createBrowserRouter.

🗺️ Routes admin dans le routeur

const router = createBrowserRouter([
  // ... routes publiques
  {
    path: "/admin",
    element: <AdminLayout />,
    children: [
      { index: true, element: <AdminDashboard /> },
      { path: "products", element: <AdminProducts /> },
      { path: "orders", element: <AdminOrders /> },
      { path: "users", element: <AdminUsers /> }
    ]
  }
]);

Toutes les routes sous /admin héritent du layout et de la sidebar.

🔒 Protection des routes admin

On crée un composant AdminRoute qui vérifie le rôle de l’utilisateur.

import { Navigate, Outlet } from "react-router-dom";
import { useAuthStore } from "../stores/authStore";

export default function AdminRoute() {
  const { user } = useAuthStore();
  
  if (!user) return <Navigate to="/login" replace />;
  if (user.role !== "admin") return <Navigate to="/" replace />;
  
  return <Outlet />;
}

On enveloppe le layout admin dans ce guard :

{ path: "admin", element: <AdminRoute />, children: [ ... ] }

👤 Stockage du rôle utilisateur

Le store auth doit contenir le rôle (admin/user).

const useAuthStore = create((set) => ({
  user: null,
  login: (userData) => set({ user: userData }),
  logout: () => set({ user: null })
}));

// Lors du login
const userFromApi = { id: 1, name: "Jean", role: "admin" };
useAuthStore.getState().login(userFromApi);

Le rôle est aussi dans le token JWT. Le backend le vérifie, mais le frontend s’en sert pour l’affichage.

✨ Améliorations possibles

  • Sidebar responsive (collapse sur mobile) avec un bouton hamburger.
  • Lazy loading des pages admin (React.lazy) pour réduire le bundle principal.
  • Gestion des permissions fines (ex: seul un super-admin peut voir la page Utilisateurs).
  • Ajout d’un message d’erreur 403 si l’utilisateur tente d’accéder directement à une route admin sans autorisation.
📌 Ne jamais se fier uniquement au frontend : le backend doit aussi vérifier le rôle.

✏️ Exercice pratique

🧪 À réaliser :

  • Créer un store Zustand avec un utilisateur ayant le rôle 'admin' ou 'user'.
  • Construire un composant AdminRoute qui redirige vers la page d’accueil si le rôle n’est pas admin.
  • Mettre en place un layout AdminLayout avec une sidebar simple (liens fictifs).
  • Ajouter deux routes sous /admin : Dashboard et Produits.
  • Tester la redirection avec un utilisateur normal.
💡 Utilisez les composants NavLink pour le surlignage de la page active.
