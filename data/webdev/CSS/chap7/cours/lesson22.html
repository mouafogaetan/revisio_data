Organiser et maintenir son CSS

Conventions de nommage, fichiers propres, variables CSS et bonnes pratiques pour un code professionnel.

Classe : Développement Web | Matière : Initiation au CSS
🎯

Objectifs de la leçon

  • ✓ Adopter une convention de nommage comme BEM, OOCSS ou SMACSS pour des classes claires.
  • ✓ Organiser son fichier CSS avec des commentaires et la règle @import.
  • ✓ Utiliser les variables CSS (custom properties) pour une maintenance facile et des thèmes dynamiques.
  • ✓ Comprendre l'importance d'un code propre et réutilisable en équipe.

📌 Pourquoi bien organiser son CSS ?

🧩

Problèmes courants

Fichiers monolithiques de milliers de lignes, sélecteurs trop spécifiques, `!important` à outrance, difficultés à ajouter des fonctionnalités sans casser l'existant.

🚀

Avantages d’une bonne structure

Maintenabilité, lisibilité, réutilisabilité, travail en équipe facilité, réduction des bugs CSS, et meilleures performances.

💡 Astuce : Un CSS bien organisé, c'est comme une maison rangée : on trouve chaque outil facilement !

🧱 Méthodologie BEM (Block, Element, Modifier)

BEM est une convention puissante pour créer des composants indépendants et éviter les conflits entre classes.

.block { }                /* composant principal */
.block__element { }       /* élément interne */
.block--modifier { }      /* variation du bloc */

.card → bloc

.card__title → élément titre

.card--dark → variante thème sombre

✅ Exemple concret : 
<div class="card card--dark">
  <h3 class="card__title">Titre</h3>
</div>

D'autres conventions : OOCSS (séparation structure/skin), SMACSS (catégorisation base/layout/module/état/thème).

📁 Organiser ses fichiers CSS & @import

Diviser son CSS en plusieurs fichiers thématiques : reset.css, variables.css, layout.css, components.css, responsive.css.

/* style principal - avec @import */
@import url("reset.css");
@import url("variables.css");
@import url("components/buttons.css");
@import url("layout/header.css");
/* attention : @import peut ralentir le chargement, privilégier la concaténation en production */

⚠️ Bonne pratique : En production, on minifie et on concatène les fichiers (ex: PostCSS, Vite, Webpack). Mais pour l’apprentissage, @import reste pratique pour la modularité.

📝 Structure recommandée (projet)

css/
├── base/
│   ├── reset.css
│   ├── typography.css
├── components/
│   ├── card.css
│   ├── button.css
├── layouts/
│   └── grid.css
├── utils/
│   └── variables.css
└── main.css   (avec @import)

🎨 Variables CSS : une révolution pour la maintenabilité

Les custom properties (--nom-variable) permettent de stocker des valeurs réutilisables, de créer des thèmes facilement, et d’améliorer la dynamique du CSS.

Déclaration

:root {
  --couleur-primaire: #3b82f6;
  --espacement: 1rem;
  --bordure-radius: 0.5rem;
}

Utilisation

.btn {
  background: var(--couleur-primaire);
  padding: var(--espacement);
  border-radius: var(--bordure-radius);
}

✨ Avantages :

  • Modification globale instantanée (ex: thème sombre)
  • Héritage et override facile via media queries ou JavaScript
  • Code plus propre, moins de répétitions

📝 Commentaires et documentation

Des commentaires bien placés aident à naviguer dans le code. Utiliser des sections pour séparer les parties.

/* ====================
   COMPOSANTS - BOUTONS
   ==================== */
.btn-primary { ... }

/* ----- Formulaire ----- */
.form-group { ... }

Autres bonnes pratiques :

  • Éviter les sélecteurs trop profonds (max 3 niveaux)
  • Préférer les classes aux IDs pour le style (sauf très rare)
  • Utiliser des linters (stylelint) et des formateurs (Prettier)
  • Nommer les classes de manière sémantique : `.error-message` plutôt que `.red-text`

💎 Règle d'or : Un développeur doit comprendre votre CSS sans avoir besoin de lire le HTML en parallèle.

🏆 À retenir (checklist)

✅ Utiliser une convention de nommage (BEM, SMACSS).
✅ Fractionner son CSS en modules + @import maîtrisé.
✅ Adopter les variables CSS pour les thèmes et valeurs récurrentes.
✅ Commenter et organiser logiquement le code.
✅ Appliquer une approche responsive et pensée composants.

🛠️ Mini exercice :

Reprenez une petite page HTML et restructurez son CSS en utilisant BEM et des variables pour les couleurs. Bonus : découpez le CSS en 2 fichiers (variables + composants).

/* variables.css */
:root { --primary: #0ea5e9; }
/* button.css */
.btn { background: var(--primary); }
🎉
