Sélecteurs avancés CSS

Cibler précisément chaque élément avec des combinateurs puissants

⬇️ Descendant ➡️ Enfant direct ➕ Frère adjacent 🔄 Frère général

🎯 Objectifs de la leçon

  • ✓ Utiliser les sélecteurs descendants (espace) et les sélecteurs enfants directs (>)
  • ✓ Maîtriser les sélecteurs frères adjacents (+) et frères généraux (~)
  • ✓ Combiner plusieurs sélecteurs avec la virgule pour appliquer les mêmes styles simplement
  • ✓ Savoir quand utiliser chaque combinateur dans des projets réels
💡 Pourquoi c'est important ? Pour écrire moins de CSS, plus propre et cibler dynamiquement des éléments selon leur position dans l'arborescence HTML.

1️⃣ Descendant (espace) vs Enfant direct (>)

nav a (descendant)

Cible TOUS les <a> à l'intérieur de <nav>, peu importe la profondeur.

nav a { color: blue; }

✔️ Sélectionne aussi les liens dans des sous-menus

nav > a (enfant direct)

Cible uniquement les <a> qui sont enfants directs de <nav>.

nav > a { font-weight: bold; }

🎯 Utile pour les menus principaux sans toucher les sous-liens

🧬 Exemple visuel :

nav
└─ a ✅ (desc + enfant)
└─ ul
└─ a ✅ (descendant seulement)

2️⃣ Sélecteurs de fratrie

+ adjacent (immédiatement après)

h2 + p → cible le premier <p> directement après un <h2>.

<h2>Titre</h2>
<p>Ce paragraphe est ciblé ✅</p>
<p>Non ciblé ❌</p>

~ général (tous les frères suivants)

h2 ~ p → cible tous les <p> qui suivent un h2 (même niveau et après).

<h2>Sujet</h2>
<p>Paragraphe 1 ✅</p>
<div></div>
<p>Paragraphe 2 ✅ (peu importe les éléments entre)</p>
✨ astuce : très utiles pour espacer des éléments ou styliser des messages de formulaire après une étiquette.

3️⃣ Grouper plusieurs sélecteurs : la virgule

Lorsque plusieurs éléments ou sélecteurs partagent les mêmes propriétés, on les sépare par une virgule pour éviter les répétitions.

/* au lieu d'écrire trois règles séparées */
h1, h2, .titre-special, #main-title {
  color: #2c3e66;
  font-family: 'Poppins', sans-serif;
  margin-bottom: 1rem;
}
✅ .btn, button, a.btn : styles cohérents pour tous les boutons.
✅ .error, .warning, .info : style commun des messages.

⚠️ Important : chaque sélecteur dans la liste est indépendant, la virgule ne crée pas de relation.

🧩 Mise en pratique : cas concrets

🎨 Menus déroulants :

nav > ul > li > a { ... } → cible uniquement les liens du premier niveau.

📝 Styles alternés entre article et commentaire :

.post + .comment { margin-top: 20px; border-top: 1px solid #ddd; }

🖼️ Icônes avant chaque paragraphe après un titre :

h3 ~ p::before { content: "✨ "; }
article
├─ h2
├─ p ← h2 + p (adjacent)
├─ div
└─ p ← h2 ~ p (général)

⚡ Conseils et pièges à éviter

  • Le sélecteur descendant consomme plus de ressources s'il est très profond ? pas de panique, mais préférez les enfants directs si possible.
  • + ne cible que l'élément immédiatement suivant : s'il y a du texte ou un commentaire ? ça fonctionne quand même.
  • La virgule ne modifie pas la spécificité, chaque sélecteur garde la sienne.
  • Pour les sélecteurs frères, l'ordre dans le DOM est important : ils doivent partager le même parent.
