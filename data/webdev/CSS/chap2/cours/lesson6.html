🌊 La cascade CSS : les règles d'or

Quand plusieurs règles s'appliquent au même élément, le navigateur suit un ordre précis :

1 Origine & importance – Règles du navigateur < utilisateur < auteur, et !important inversé.
2 Spécificité – Le sélecteur le plus précis gagne.
3 Ordre d’apparition – En cas d’égalité, c’est la dernière déclaration qui l’emporte.

💡 Exemple : CSS inline > ID > classe > élément. La feuille externe importée en dernier domine si même spécificité.

🔢 Calcul de la spécificité (système A - B - C - D)

Chaque sélecteur a un poids. On le représente comme (inline, ID, classe/pseudo-classe, éléments).

Type de sélecteurPoidsExemple
Style inline1,0,0,0style="color:red"
ID (#id)0,1,0,0#header
Classe, pseudo-classe, attribut0,0,1,0.btn, :hover, [type]
Élément, pseudo-élément0,0,0,1div, p, ::before

📌 Exemple : #sidebar .menu a:hover → (0,1,2,1) → ID(1) + classe(1) + pseudo-classe(1) + balise(1)

Plus le nombre est grand, plus le sélecteur est spécifique.

⚠️ !important & Héritage

🚨 !important

Il donne la priorité maximale à une règle, même face à des ID. À utiliser exceptionnellement (dépannage, override de librairie).

p { color: blue !important; }

🌳 Héritage naturel

Certaines propriétés (couleur, font-family) se transmettent aux enfants. D'autres (margin, border) non.

body { color: gray; } → tous les textes héritent sauf si redéfini.

📢 Rappel : inherit permet de forcer l’héritage, initial remet à la valeur par défaut.

🧪 Exemple interactif – Testez la spécificité

<div id="demo" class="box">Quelle couleur ?</div>

<style>
  div { color: black; }           /* spécificité (0,0,0,1) */
  .box { color: green; }         /* (0,0,1,0) → gagne sur div */
  #demo { color: red; }          /* (0,1,0,0) → gagne sur .box */
  /* Si !important sur .box : .box { color: orange !important; } alors orange */
</style>
Couleur finale = rouge (ID)
