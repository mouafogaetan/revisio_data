🏗️ 1. Constructeur const : le super-pouvoir silencieux

En Flutter, chaque fois qu’un widget est recréé, cela peut ralentir l’interface. Les widgets const sont immutables et Flutter peut les réutiliser sans les reconstruire.

❌ Non optimisé

→

✅ Optimisé

// Mauvaise pratique (rebuild à chaque setState)
Text('Bonjour', style: TextStyle(fontSize: 16))

// Bonne pratique (const)
const Text('Bonjour', style: TextStyle(fontSize: 16))

// Pour les widgets personnalisés, déclarez le constructeur const
class MyButton extends StatelessWidget {
  final String label;
  const MyButton(this.label, {super.key}); // rendre const
  @override
  Widget build(BuildContext context) => Text(label);
}

🌲 2. Arbre des widgets & Rebuilds

Flutter reconstruit uniquement les parties de l'interface qui ont changé. Cependant, un setState() déclenche le rebuild du sous-arbre entier. Pour limiter les rebuilds :

💡 Astuce : découper votre interface en petits widgets StatelessWidget & const pour éviter les cascades de rebuilds.

Lorsque Widget B change, seul lui et ses enfants se rebuildent, les widgets const sont ignorés. Exploitez cette mécanique.

🔧 3. Flutter DevTools & Profiling

DevTools est une suite de debuggage et performance. Pour y accéder, lancez votre app en mode debug, puis dans le terminal : flutter devtools .

📊 Performance Tab

Visualisez les FPS, détectez les "jank" (images saccadées).

🧩 Widget Inspector

Analysez le nombre de rebuilds et l'arbre des widgets.

⏱️ Timeline

Vérifiez les tâches asynchrones lourdes, les build trop longs.

🧪 Memory

Surveillez les fuites mémoire.

flutter run --profile

📱 Ouvrir DevTools → Onglet "Performance"

🔍 Enregistrer une trace → identifier rebuilds inutiles → Appliquer const et keys

📌 4. Optimisations avancées du quotidien

🖼️
Images : utilisez precacheImage et des formats adaptés (webp). Dimensions adéquates.
📜
Longues listes : privilégiez ListView.builder plutôt que ListView(children: [...]).
🔑
Keys : Utiliser ValueKey ou ObjectKey pour aider Flutter à identifier les widgets dans une liste dynamique.
🧩
Split widgets : extraire les parties qui changent souvent en StatefulWidget séparés pour éviter de rebuild tout l’écran.
✨ Exemple concret : Au lieu de tout reconstruire, découpez votre compteur en petit widget réutilisable.

🧠 Résumé & À pratiquer

  • const dans les widgets statiques réduit énormément les rebuilds.
  • Comprendre l’arbre des widgets et limiter les zones affectées par setState.
  • Utiliser DevTools pour mesurer les performances réelles de l’application.
  • ListView.builder, images optimisées, et clés uniques pour les collections.

Créez une application Flutter affichant une longue liste de 1000 éléments. Ajoutez une animation ou un compteur. Utilisez DevTools pour observer les rebuilds, puis appliquez const et des optimisations. Mesurez la différence de FPS.
