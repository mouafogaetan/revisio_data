💡 Pourquoi les tests d'intégration ? Ils simulent une vraie interaction utilisateur et valident que TOUT fonctionne ensemble (UI, navigation, état, API).

📦 Installation du package

Pour écrire des tests d'intégration robustes, ajoutez integration_test dans pubspec.yaml :

dev_dependencies:
  flutter_test:
    sdk: flutter
  integration_test:
    sdk: flutter
  # optionnel : pour driver
  flutter_driver:
    sdk: flutter

⚠️ Note importante :

Les tests d'intégration s'exécutent dans un environnement réel ou émulateur. Ils sont plus lents que les tests unitaires, mais indispensables pour les parcours critiques.

✍️ Écrire un premier test d'intégration

Structure d’un fichier de test integration_test/app_test.dart :

import 'package:flutter_test/flutter_test.dart';
import 'package:integration_test/integration_test.dart';
import 'package:mon_app/main.dart' as app;

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  group('Test de l\'application complète', () {
    testWidgets('Vérifier navigation et compteur', (tester) async {
      app.main();
      await tester.pumpAndSettle();

      // Vérifier écran d'accueil
      expect(find.text('0'), findsOneWidget);
      // clic sur bouton increment
      await tester.tap(find.byIcon(Icons.add));
      await tester.pumpAndSettle();
      expect(find.text('1'), findsOneWidget);
    });
  });
}

🔄 Tester le flux complet

Scénario typique : connexion → navigation vers un détail → retour arrière.

testWidgets('Flux complet connexion → profil', (tester) async {
  // Lancer l'app
  app.main();
  await tester.pumpAndSettle();

  // Remplir email + mot de passe
  await tester.enterText(find.byKey(Key('email')), 'etudiant@example.com');
  await tester.enterText(find.byKey(Key('password')), '123456');
  await tester.tap(find.text('Se connecter'));
  await tester.pumpAndSettle();

  // Vérifier accès écran profil
  expect(find.text('Bienvenue étudiant'), findsOneWidget);
  
  // Navigation vers détails
  await tester.tap(find.byIcon(Icons.person));
  await tester.pumpAndSettle();
  expect(find.text('Paramètres'), findsOneWidget);
});

📲 Exécuter les tests sur émulateur & appareil réel

🖥️ Terminal / CLI

# Sur émulateur Android/iOS
flutter test integration_test/app_test.dart

# Sur un appareil réel connecté
flutter run integration_test/app_test.dart

⚙️ Android Studio / VS Code

Créez une configuration “Test Intégration” et sélectionnez le fichier app_test.dart. Lancez avec F5.

💡 Astuce prof : Utilisez flutter drive --driver=test_driver/integration_test.dart --target=integration_test/app_test.dart pour des tests plus poussés.

🧰 Bonnes pratiques & astuces

  • 🔁 Utilisez pumpAndSettle() après chaque action pour attendre animations/requêtes.
  • 🧩 Isoler les tests : chaque scénario doit être indépendant.
  • 📁 Organisez les tests par fonctionnalité (auth_test, panier_test, etc).
  • 🤖 Combine avec mock API : utiliser MockClient pour éviter appels réels.
  • 📊 Utilisez --reporter expanded pour des logs clairs.
🔄 Fiabilité 📈 Coverage 🧪 CI/CD

🏁 Exemple concret : test de formulaire

Simuler un envoi de formulaire et vérifier les messages d'erreur :

testWidgets('Validation email incorrect', (tester) async {
  await tester.pumpWidget(MyApp());
  await tester.enterText(find.byType(TextField).first, 'mail_invalide');
  await tester.tap(find.text('Valider'));
  await tester.pump();
  expect(find.text('Email invalide'), findsOneWidget);
});

⚡ Intégration continue & rapports

Les tests d'intégration peuvent être automatisés sur GitHub Actions ou GitLab CI. Exemple de workflow :

jobs:
  integration_test:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - uses: subosito/flutter-action@v2
      - run: flutter pub get
      - run: flutter test integration_test

Les résultats au format JSON ou JUnit permettent d’analyser les échecs facilement.
