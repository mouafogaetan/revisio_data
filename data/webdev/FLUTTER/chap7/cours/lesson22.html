🎯 Leçon 22

JSON ↔ Objets Dart

Convertir des données JSON en objets Dart avec json_serializable

📦 package: json_annotation + build_runner ⚙️ génération de code automatique

👉 Glissez ou utilisez les boutons 👈

🎯

Objectifs de la leçon

1️⃣

Convertir JSON en objet Dart en utilisant jsonDecode.

2️⃣

Utiliser le package json_serializable pour générer automatiquement fromJson / toJson.

3️⃣

Créer des modèles Dart robustes avec annotations @JsonSerializable().

4️⃣

Exécuter build_runner pour générer les fichiers .g.dart et les intégrer.

💡 Pourquoi c'est essentiel ? Les API REST retournent du JSON ; pour manipuler des données typées en Dart, on a besoin de modèles convertisseurs. json_serializable évite l'écriture manuelle lourde et les erreurs.

📡

1. Décoder du JSON brut avec dart:convert

Avant la génération automatique, comprenons comment Dart parse du JSON via jsonDecode :

import 'dart:convert';

void main() {
  String jsonString = '{"nom":"Cameroun","capitale":"Yaoundé","habitants":2760000}';
  Map userMap = jsonDecode(jsonString);
  print(userMap['nom']); // Cameroun
}

⚠️ Limites de l'approche manuelle :

  • Pas de typage fort → risque d'erreurs à l'exécution.
  • Beaucoup de code répétitif pour chaque modèle (cast, vérification null).
  • Évolutivité difficile quand les API deviennent complexes.

💡 D'où l'utilité de json_serializable pour générer du code fiable !

📦 Installer json_serializable

📄 pubspec.yaml (dependencies & dev_dependencies)

dependencies:

🔧 Étape 1

Ajouter les packages puis exécuter flutter pub get

⚡ Étape 2

Créer un modèle avec annotations, puis lancer dart run build_runner build

🎯 build_runner génère un fichier nom_modele.g.dart contenant les sérialiseurs.
🏗️

Modèle Dart & annotations

Exemple : modèle Étudiant avec nom, age, et notes.

import 'package:json_annotation/json_annotation.dart';

part 'etudiant.g.dart';  // Fichier généré

@JsonSerializable()
class Etudiant {
  final String nom;
  final int age;
  @JsonKey(name: 'moyenne') // Si clé JSON différente
  final double noteMoyenne;

  Etudiant({required this.nom, required this.age, required this.noteMoyenne});

  // FromJson factory
  factory Etudiant.fromJson(Map<String, dynamic> json) => _$EtudiantFromJson(json);
  // ToJson method
  Map<String, dynamic> toJson() => _$EtudiantToJson(this);
}

✅ Après exécution de build_runner build, le fichier etudiant.g.dart apparaît avec l’implémentation concrète de fromJson et toJson.

🚀 Utilisation dans l'application

Une fois le modèle généré, convertir des réponses d’API devient élégant :

// Exemple avec requête http (simulée)
Future<List<Etudiant>> fetchEleves() async {
  final response = await http.get(Uri.parse('https://api.exemple.com/etudiants'));
  if (response.statusCode == 200) {
    List<dynamic> body = jsonDecode(response.body);
    return body.map((json) => Etudiant.fromJson(json)).toList();
  }
  throw Exception('Erreur chargement');
}

✨ Avantages clés :

  • Typage fort : plus d’erreurs de casting inattendues.
  • Gestion des champs optionnels / nullable automatique (supports null safety).
  • Facile à maintenir pour des gros projets Flutter.

🧠 Bonnes pratiques & objets imbriqués

📌

Utiliser @JsonKey(ignore: true) pour exclure un champ de la sérialisation.

📌

Pour les objets JSON imbriqués, créez aussi des classes avec @JsonSerializable() et référencez-les directement.

Exemple d'objet complexe :

class Profil {
  final String bio;
  final List<String> hobbies;
  Profil(this.bio, this.hobbies);
  factory Profil.fromJson(...)...
}
@JsonSerializable()
class Utilisateur {
  final String nom;
  final Profil profil; // automatiquement converti si Profil a fromJson
}

🎓 À retenir : json_serializable libère le développeur des tâches répétitives et garantit des conversions robustes.
