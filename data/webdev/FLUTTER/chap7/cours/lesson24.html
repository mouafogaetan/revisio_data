📀

Introduction à SQLite dans Flutter

SQLite est un moteur de base de données relationnelle embarqué. Avec le package sqflite, Flutter peut stocker des données structurées localement sur le téléphone.

📥 Installation

# 1. Ajouter dans pubspec.yaml
dependencies:
  sqflite: ^2.3.0
  path: ^1.8.0

# 2. Importer dans votre fichier Dart
import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';

🔍 Explication : sqflite fournit le moteur, path aide à localiser le fichier .db sur l'appareil.

⚠️ Note pour Android/iOS : Aucune permission supplémentaire n'est nécessaire pour SQLite, les bases sont stockées dans le répertoire privé de l'application.

🏗️ Créer la base et une table

Exemple d'une table "notes" avec id, titre, contenu et date de création.

Future<Database> initDatabase() async {
  String path = join(await getDatabasesPath(), 'app_notes.db');
  return openDatabase(
    path,
    version: 1,
    onCreate: (Database db, int version) async {
      await db.execute('''
        CREATE TABLE notes(
          id INTEGER PRIMARY KEY AUTOINCREMENT,
          titre TEXT NOT NULL,
          contenu TEXT,
          date_creation TEXT
        )
      ''');
    },
  );
}
✔️ PRIMARY KEY AUTOINCREMENT : id unique automatique
✔️ TEXT / INTEGER : types SQLite standards

💡 La méthode onCreate s'exécute uniquement si la base n'existe pas. C'est idéal pour définir vos tables initiales.

✍️ CRUD : Insérer & Lire

➕ INSERT (Créer)

Future<int> insertNote(Note note) async {
  Database db = await initDatabase();
  return await db.insert('notes', note.toMap());
}

📖 READ (Lire toutes les notes)

Future<List<Note>> getNotes() async {
  Database db = await initDatabase();
  final List<Map<String, dynamic>> maps = await db.query('notes');
  return List.generate(maps.length, (i) => Note.fromMap(maps[i]));
}

.insert() et .query() sont des méthodes asynchrones. Pensez à convertir vos objets Dart en Map.

✏️ Mettre à jour & Supprimer

🔁 UPDATE (modifier une note)

Future<int> updateNote(Note note) async {
  Database db = await initDatabase();
  return await db.update(
    'notes',
    note.toMap(),
    where: 'id = ?',
    whereArgs: [note.id],
  );
}

🗑️ DELETE (supprimer une note)

Future<int> deleteNote(int id) async {
  Database db = await initDatabase();
  return await db.delete(
    'notes',
    where: 'id = ?',
    whereArgs: [id],
  );
}
✨ Astuce : Utilisez toujours whereArgs pour éviter les injections SQL.

🔄 Migrations de schéma

Quand votre application évolue, vous ajoutez des colonnes ou tables. Augmentez la version et utilisez onUpgrade.

openDatabase(
  path,
  version: 2,
  onCreate: (db, version) => createTables(db),
  onUpgrade: (db, oldVersion, newVersion) async {
    if (oldVersion < 2) {
      await db.execute('ALTER TABLE notes ADD COLUMN priorite INTEGER DEFAULT 0');
    }
  },
);
📌

Les migrations permettent de conserver les données existantes tout en mettant à jour la structure. N'oubliez pas de tester les changements sur des versions antérieures.

Évolution par migrations

📱 Exemple d'utilisation en pratique

Dans un StatefulWidget ou avec un Provider, vous pouvez appeler ces méthodes :

// Ajouter une note
final newNote = Note(titre: "Cours SQLite", contenu: "CRUD terminé");
await insertNote(newNote);

// Lire les notes
final notes = await getNotes();
setState(() => maListe = notes);

// Supprimer
await deleteNote(notes[0].id);
✅ Bonnes pratiques
- Fermez la base après usage (close())
- Utilisez un singleton Database helper
⚡ Performances
- Évitez les requêtes dans le build()
- Utilisez batch pour plusieurs opérations
