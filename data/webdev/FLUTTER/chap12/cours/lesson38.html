Publier sur l'App Store

Soumettre votre application Flutter sur l'App Store d'Apple

Certificats & Profils Archive (.ipa) Transporter / Xcode
Suivez les étapes pour déployer sur iOS

🎯 Objectifs de la leçon

Configurer Xcode et les certificats de signature
Générer une archive avec flutter build ipa
Soumettre l'application avec Transporter ou Xcode
Comprendre le processus de validation et de publication
À savoir

La publication sur l'App Store nécessite un compte Apple Developer (99$/an), un Mac, et le respect des directives d'Apple.

📋 Prérequis avant publication

Mac avec Xcode installé
Compte Apple Developer (99 $/an)
Certificats (Development / Distribution)
App ID et Provisioning Profile pour la distribution
# Vérifier la configuration Flutter pour iOS
flutter doctor -v
# S'assurer que Xcode est bien installé et configuré
sudo xcode-select --switch /Applications/Xcode.app

⚙️ Configuration dans Xcode

Ouvrez le dossier ios/ de votre projet Flutter dans Xcode (open ios/Runner.xcworkspace).

Identifiant du bundle : doit être unique (ex: com.votrenom.monapp)
Version & Build : Version (CFBundleShortVersionString) = 1.0.0, Build (CFBundleVersion) = 1
Team : sélectionner votre compte Apple Developer
Provisioning Profile : choisir "iOS Distribution"
N'oubliez pas d'ajouter les icônes et le splash screen dans Assets.xcassets.

📦 Générer l'archive (.ipa)

# Nettoyer les anciens builds
flutter clean
# Récupérer les dépendances
flutter pub get
# Générer l'archive iOS (Release)
flutter build ipa --release
L'archive se trouve dans : build/ios/archive/Runner.xcarchive
Un fichier .ipa est également généré dans build/ios/ipa/.

Alternative : depuis Xcode (Product → Archive).

🚀 Soumettre à l'App Store

Méthode 1 : Transporter (application macOS)

  1. Téléchargez Transporter depuis l'App Store (gratuit).
  2. Connectez-vous avec votre identifiant Apple Developer.
  3. Glissez-déposez le fichier .ipa généré.
  4. Validez et envoyez.
# Alternative : via xcrun altool (ligne de commande)
xcrun altool --upload-app -f monapp.ipa -t ios -u "votre@email.com" -p "mot-de-passe-app-specific"
Méthode 2 : Xcode → Product → Archive → Distribute App → App Store Connect.

📱 App Store Connect & validation

Après soumission, connectez-vous à App Store Connect pour :

  • Remplir les informations (nom, description, mots-clés, catégorie).
  • Ajouter les captures d'écran (pour iPhone et iPad).
  • Configurer les prix et la disponibilité.
  • Soumettre à la revision Apple (délai : 24h à quelques jours).
Pièges fréquents : confidentialité (politique de confidentialité), compatibilité iPad, crashs au lancement, utilisation de permissions (justifications valides).

✅ Récapitulatif des étapes clés

1. Avoir un compte Apple Developer (99$/an)
2. Configurer Bundle ID, version, team dans Xcode
3. Générer l'archive (flutter build ipa)
4. Soumettre via Transporter ou Xcode
5. Remplir la fiche App Store Connect
6. Attendre la validation Apple

Félicitations ! Vous savez déployer votre application Flutter sur l'App Store.

Pensez à tester rigoureusement avant la soumission.
