🎯 Objectifs de la leçon

  • ✓ Comprendre la différence entre navigation impérative (Navigator) et déclarative (go_router).
  • ✓ Installer et configurer go_router dans une application Flutter.
  • ✓ Définir des routes avec paramètres, redirections et transitions personnalisées.
  • ✓ Naviguer avec context.push(), context.go(), context.pop().
💡 go_router est la solution recommandée par l’équipe Flutter pour la navigation moderne.

🤔 Pourquoi go_router plutôt que Navigator ?

⚠️ Navigation impérative (Navigator classique)

  • pushNamed, push, pop
  • Gestion manuelle des arguments
  • Difficile à tester
  • Pas de synchronisation URL (web)

✅ Navigation déclarative (go_router)

  • Routes définies en une seule fois
  • Paramètres automatiques (path, query)
  • Redirections intégrées
  • Support web prêt à l’emploi
// Installation flutter pub add go_router

⚙️ Configuration de go_router

// 1. Créer un fichier router.dart ou configurer directement dans main.dart import 'package:go_router/go_router.dart'; final router = GoRouter( initialLocation: '/', routes: [ GoRoute( path: '/', name: 'home', builder: (context, state) => HomeScreen(), ), GoRoute( path: '/profile/:userId', name: 'profile', builder: (context, state) { final userId = state.pathParameters['userId']!; return ProfileScreen(userId: userId); }, ), ], ); // 2. Dans MaterialApp.router MaterialApp.router( routerConfig: router, title: 'Mon App', )

🔍 Les paramètres dynamiques sont définis avec :paramName.

🧭 Naviguer avec go_router

Aller vers une route (push) :

context.push('/profile/123'); // ajoute l'écran à la pile context.pushNamed('profile', pathParameters: {'userId': '123'});

Remplacer l’écran courant (go) :

context.go('/settings'); // remplace toute la pile d'historique

Revenir en arrière :

context.pop(); // retourne à l'écran précédent
💡 push conserve l’historique, go le réinitialise.

🔗 Paramètres query et redirections

// Route avec paramètres query GoRoute( path: '/search', builder: (context, state) { final query = state.uri.queryParameters['q'] ?? ''; return SearchScreen(query: query); }, ), // Redirection (ex: utilisateur non connecté) GoRoute( path: '/dashboard', redirect: (context, state) { final isLogged = AuthService.isLogged(); return isLogged ? null : '/login'; }, builder: (context, state) => DashboardScreen(), ),

🔄 La redirection renvoie null si aucune redirection n’est nécessaire.

✨ Transitions animées personnalisées

GoRoute( path: '/details', pageBuilder: (context, state) => CustomTransitionPage( child: DetailsScreen(), transitionsBuilder: (context, animation, secondaryAnimation, child) { return FadeTransition(opacity: animation, child: child); }, ), ),

On peut utiliser SlideTransition, ScaleTransition, ou créer ses propres animations.

📱 go_router supporte aussi les onglets (StatefulShellRoute) et les routes imbriquées.
