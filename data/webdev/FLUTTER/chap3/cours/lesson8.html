Leçon 8 – Chapitre 3

🧩 StatelessWidget vs StatefulWidget

Comprendre les widgets statiques et dynamiques dans Flutter

Durée : 45 min Niveau : Débutant Widgets essentiels

"Tout est widget – mais certains évoluent, d'autres non."

🎯 Objectifs de la leçon

  • Comprendre la différence fondamentale entre un StatelessWidget et un StatefulWidget.
  • Savoir créer un widget statique immuable.
  • Savoir créer un widget dynamique avec son propre état interne.
  • Maîtriser la méthode setState() pour mettre à jour l'interface.
Un widget qui ne change jamais (texte, icône) = StatelessWidget. Un widget qui réagit à des actions utilisateur = StatefulWidget.

📊 Comparaison : Stateless vs Stateful

StatelessWidget

  • Immuable : ne change jamais après sa création
  • La méthode build() est appelée une seule fois
  • Pas de méthode setState()
  • Exemples : Text, Icon, Container (sans animation)

StatefulWidget

  • Mutable : peut changer pendant l'exécution
  • Possède une classe State associée
  • Utilise setState() pour rafraîchir l'UI
  • Exemples : Checkbox, TextField, compteur
La règle d'or : si l'apparence d'un widget dépend de données qui peuvent changer, utilisez StatefulWidget.

📦 Créer un StatelessWidget

Un widget statique : on redéfinit simplement la méthode build().

import 'package:flutter/material.dart';

class MaCarteStatique extends StatelessWidget {
  final String titre;
  final String sousTitre;

  const MaCarteStatique({required this.titre, required this.sousTitre});

  @override
  Widget build(BuildContext context) {
    return Card(
      child: ListTile(
        title: Text(titre),
        subtitle: Text(sousTitre),
        leading: Icon(Icons.info),
      ),
    );
  }
}

Les données sont passées par le constructeur et ne peuvent pas être modifiées de l'intérieur.

🔄 Créer un StatefulWidget

Un widget dynamique nécessite deux classes : le widget lui-même et sa classe State.

class CompteurWidget extends StatefulWidget {
  @override
  State<CompteurWidget> createState() => _CompteurWidgetState();
}

class _CompteurWidgetState extends State<CompteurWidget> {
  int _compteur = 0; // variable d'état

  void _incrementer() {
    setState(() {
      _compteur++; // notification de changement
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Compteur : $_compteur'),
        ElevatedButton(
          onPressed: _incrementer,
          child: Text('Incrémenter'),
        ),
      ],
    );
  }
}

⚡ setState() : comment ça fonctionne ?

Le rôle de setState() :

Elle indique à Flutter que l'état interne a changé et qu'il faut reconstruire la partie de l'arbre des widgets associée.

1. Modification

On modifie une variable d'état.

2. Notification

setState() notifie le framework.

3. Rebuild

La méthode build() est rappelée.

// ❌ MAUVAISE PRATIQUE : modification sans setState
_compteur = 5; // L'UI ne sera PAS mise à jour !

// ✅ BONNE PRATIQUE :
setState(() {
  _compteur = 5; // L'UI sera reconstruite
});

📋 Quand utiliser l'un ou l'autre ?

Critère StatelessWidget StatefulWidget
Données modifiables❌ Non✅ Oui (via setState)
Rebuild automatiqueNon (construit une fois)Oui (sur setState)
PerformancePlus légerLéger surcoût
Cas typiqueTexte, Image, mise en page fixeFormulaire, animation, compteur
Astuce : Commencez toujours par un StatelessWidget. Si vous avez besoin de modifier l'interface, transformez-le en StatefulWidget (l'IDE peut le faire automatiquement).

✏️ Exercice & Conclusion

Exercice guidé :

Créez un StatefulWidget qui affiche un texte "J'aime Flutter" et un bouton ❤️. À chaque clic, incrémentez un compteur de likes. Affichez le nombre de likes à côté du bouton.

Voir une solution possible
class LikeButton extends StatefulWidget {
  @override _LikeButtonState createState() => _LikeButtonState(); }
class _LikeButtonState extends State<LikeButton> {
  int _likes = 0;
  void _addLike() => setState(() => _likes++);
  @override Widget build(BuildContext context) {
    return Row(children: [
      IconButton(icon: Icon(Icons.favorite), onPressed: _addLike),
      Text('$_likes likes'),
    ]); } }

Récapitulatif :

  • StatelessWidget = widget figé (ne change pas).
  • StatefulWidget = widget qui peut évoluer grâce à son State.
  • setState() est la clé pour rafraîchir l'affichage.
  • Toujours placer les modifications de variables d'état à l'intérieur de setState().
