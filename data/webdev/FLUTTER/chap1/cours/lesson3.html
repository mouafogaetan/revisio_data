🎯 Objectifs de la leçon

  • Créer un projet Flutter avec flutter create mon_app
  • Comprendre la structure de dossiers (lib, pubspec.yaml, android, ios, etc.)
  • Lancer l’application avec flutter run et utiliser le Hot Reload 🔥
  • Savoir interagir avec l’émulateur ou un vrai téléphone

Pourquoi c’est important ? Avant d’écrire du code, il faut maîtriser l’environnement de travail. Ces bases vous permettront de démarrer n’importe quelle application Flutter.

💻 1. Créer un nouveau projet

Ouvrez votre terminal (VS Code intégré ou invite de commande) et exécutez :

flutter create mon_premiere_app

👉 Remplacez mon_premiere_app par le nom souhaité (sans espaces ni caractères spéciaux).

Après exécution, Flutter génère un dossier contenant tout le nécessaire :

  • Un projet de démonstration (compteur classique)
  • Les fichiers de configuration multiplateformes
  • Le fichier pubspec.yaml (gestion des dépendances)

📁 2. Structure du projet Flutter

Dossiers importants :

  • lib/ → tout le code Dart (l’application)
  • android/ & ios/ → code spécifique aux plateformes
  • test/ → tests unitaires
  • web/ (si activé) pour déployer sur navigateur

Fichiers clés :

  • pubspec.yaml : dépendances, assets (images, polices)
  • lib/main.dart : point d’entrée de l’app
  • README.md : documentation
Bon à savoir : le dossier lib contient tout le code Dart. C’est là que nous allons écrire nos interfaces (Widgets).

📄 3. Le fichier main.dart

Ouvrez lib/main.dart. Vous verrez une application simple avec un compteur. Décortiquons :

import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());  // Lance l'application
}

class MyApp extends StatelessWidget { ... }
class MyHomePage extends StatefulWidget { ... }
  • main() → fonction principale, point d’entrée obligatoire.
  • runApp() → attache le widget racine à l’écran.
  • MyApp → widget racine (StatelessWidget).

🏃 4. Lancer et recharger à chaud

Depuis le terminal, dans le dossier du projet :

flutter run

Attendez le build. Une fois l’app lancée sur l’émulateur ou le téléphone, modifiez du texte dans main.dart et sauvegardez.

Hot Reload (touche ‘r’ dans le terminal)

Permet de voir les changements instantanément sans perdre l’état de l’application. Génial pour développer rapidement !

Autres raccourcis : R pour Hot Restart (réinitialise l’état), q pour quitter.

📱 5. Choisir un appareil (émulateur / physique)

Lister les appareils disponibles :

flutter emulators

Lancer un émulateur Android :

flutter emulators --launch Pixel_4_API_30

Ou si vous avez un téléphone connecté (débogage USB activé), il apparaîtra automatiquement.

Flutter supporte Android, iOS, web et desktop !

📌 Récapitulatif & astuces

Commande essentielle : flutter create nom_projet
Structure : lib/ pour le code, pubspec.yaml pour les dépendances.
Lancement : flutter run (hot reload = r).
Fichier principal : main.dart avec void main() => runApp(...).
À retenir : Le hot reload est votre meilleur ami. N’hésitez pas à modifier les couleurs, textes pour voir la magie Flutter.
