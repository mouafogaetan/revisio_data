🔍 1. Collecte des preuves numériques

Lorsqu'une attaque a lieu, il est essentiel de préserver les preuves sans les altérer. Ordre d'importance :

🧠 Mémoire vive (RAM)

Contient les processus actifs, connexions réseau, mots de passe en clair. Se volatilise à l'extinction → à capturer en priorité.

💾 Disques durs (DD/SSD)

Données persistantes : fichiers systèmes, logs applicatifs, historique navigateur.

📜 Journaux (logs)

Serveurs, pare-feux, applications – traces de connexions, erreurs, accès suspects.

🌐 Trafic réseau

Capturé avec Wireshark ou tcpdump pour voir les communications malveillantes.

⚠️ Règle d'or : Ne jamais éteindre une machine avant d'avoir capturé la mémoire vive (sinon preuves volatiles perdues).

🛠️ 2. Analyse des artefacts & outils clés

Après la collecte, vient l’analyse des indices. Voici trois outils fondamentaux :

🕵️

Autopsy / The Sleuth Kit

Plateforme forensique pour analyser des images disque. Retrouve fichiers supprimés, historique, métadonnées. Interface graphique facile.

🧩

Volatility

Analyse de la mémoire RAM (dump). Permet de lister les processus cachés, connexions réseau, DLL injectées, clés de registre volatiles.

📡

Wireshark

Analyseur de paquets réseau. Capture et inspecte le trafic pour détecter des exfiltration de données, connexions C2, etc.

Flux d'investigation : collecte → analyse → corrélation

📂 3. Artefacts système sous Windows & Linux

🪟 Windows

  • Registre : clés SAM, SYSTEM
  • Fichiers : $MFT (Master File Table)
  • Journaux : Event Viewer (Sécurité, Application)
  • Prefetch, Amcache, USB history

🐧 Linux

  • Logs : /var/log/auth.log, syslog
  • Historique bash (.bash_history)
  • Processus : /proc, analyse des binaires
  • Super journal journalctl

# Exemple d'extraction de processus suspects sous Linux

$ ps aux | grep -i "malware\|reverse"

$ lsof -i :4444 # connexion suspecte sur port 4444

🧪 Les artefacts aident à comprendre le vecteur d'attaque, la persistance et l'exfiltration.

🔐 Chaîne de custodia & bonnes pratiques

📝

Documenter chaque manipulation

Qui, quand, quoi, pourquoi ?
🔒

Hacher les preuves (SHA-256)

Vérifier l'intégrité avant analyse
⛔

Travailler sur copie forensique

Jamais sur l'original

Un bon analyste préserve les preuves intactes pour qu'elles soient recevables en justice. La chaîne de custodia (chain of custody) retrace tous les transferts des preuves.

✨ À retenir : Les trois piliers de la collecte – Volatilité (commencer par RAM), Intégrité (hash, write-blocker), Traçabilité.
