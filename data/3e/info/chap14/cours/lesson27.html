UE 27 : Notions de base sur le développement logiciel

Informatique · Classe de 3ème · Module 4 : Initiation à l’algorithme et au développement logiciel

📌 Éléments clés à retenir

1. Qu'est-ce qu'un logiciel ?

Un logiciel est un ensemble de programmes, de données et d’instructions qui permettent à un ordinateur d’exécuter des tâches spécifiques. Il peut être de deux grandes familles :

  • Logiciel système : le système d’exploitation (Windows, Linux, macOS) qui gère les ressources matérielles.
  • Logiciel applicatif : programmes dédiés à des tâches utilisateur (navigateur web, tableur, jeux, etc.).

🔍 Schéma : couches logicielle et matérielle

2. Langage de programmation : haut niveau vs langage machine

Un programme est écrit dans un langage de programmation. Il existe plusieurs niveaux d'abstraction :

🔹 Langage de haut niveau

Proche du langage humain (anglais simplifié).
Exemples : Python, Java, C, JavaScript.

Facile à lire/écrire Indépendant de la machine

🔻 Langage machine

Suite de 0 et de 1 directement exécutable par le processeur.
Spécifique à chaque architecture.

Binaire Très difficile à écrire

Le code source (haut niveau) doit être transformé en langage machine pour être exécuté.

3. Compilateur et interpréteur : les traducteurs essentiels

Deux méthodes principales pour passer du code source au programme exécutable :

  • Compilateur : traduit tout le code source en une seule fois en langage machine. Génère un fichier exécutable (.exe, .out). Exemple : GCC pour C, javac pour Java (compilation en bytecode).
  • Interpréteur : exécute le code source ligne par ligne, sans créer de fichier exécutable indépendant. Exemples : Python, JavaScript (dans le navigateur).
💡 Résumé mnémotechnique : Le compilateur prépare tout le plat avant de le servir ; l’interpréteur cuisine et sert au fur et à mesure.

4. Différence entre programme, application et logiciel

Ces termes sont souvent utilisés de manière interchangeable, mais il existe des nuances :

  • Programme : suite d’instructions accomplissant une tâche précise.
  • Logiciel : ensemble de programmes + documentation + données. Plus large.
  • Application : logiciel destiné à l’utilisateur final (ex: application mobile, tableur).

💡 Exemples de compréhension

Exemple 1 : De l’algorithme au programme exécutable

Un élève écrit un algorithme en langage naturel pour calculer le périmètre d’un rectangle. Ensuite, il code ce programme en Python (langage interprété) et en C (langage compilé).

Python (interprété) : longueur = 5; largeur = 3; perimetre = 2*(longueur+largeur); print(perimetre)
→ L’interpréteur Python exécute chaque ligne directement sans créer un fichier .exe.

C (compilé) : int main() { int L=5, l=3; int p=2*(L+l); printf("%d", p); return 0; }
→ Le compilateur transforme le code source en un exécutable machine (ex: calcul.exe).

💬 Astuce : Python est idéal pour apprendre car il est interprété, donc on teste rapidement. Les langages compilés sont souvent plus rapides à l’exécution.

Exemple 2 : Identifier le type de langage

Observation de deux extraits de code :

  • Extrait A : MOV AX, 5 ; ADD AX, 2 (assembleur proche du langage machine)
  • Extrait B : print("Bienvenue en 3ème") (Python)

Analyse : L’extrait A est un langage de bas niveau (assembleur) quasiment du langage machine symbolique. L’extrait B est un langage de haut niveau (Python) très lisible. Un compilateur ou un interpréteur sera nécessaire pour passer du code B au langage machine.

Exemple 3 : Rôle du compilateur dans un jeu vidéo

Un studio développe un jeu en C++. Une fois le code terminé, ils utilisent un compilateur pour générer des versions pour Windows, macOS et Linux.

Pourquoi ? Le compilateur transforme le même code source en langage machine adapté à chaque processeur et système d’exploitation. Sans compilateur, le code ne pourrait pas être exécuté directement par l’ordinateur du joueur.

🎮 À retenir : le compilateur est un pont essentiel entre le langage humain et le langage de la machine.

⚠️ Pièges courants / À ne pas confondre

  • Confondre compilateur et interpréteur : Le compilateur traduit tout le programme avant exécution, l’interpréteur exécute ligne par ligne sans générer de fichier exécutable autonome.
  • Langage machine ≠ langage de programmation : Le langage machine est du binaire (0/1) alors que les langages de programmation utilisent des mots-clés lisibles (if, while, etc.).
  • Penser que tous les logiciels sont écrits en un seul langage : En réalité, de gros logiciels mélangent plusieurs langages (C++ pour les performances, Python pour l’interface, etc.).
  • Confondre code source et exécutable : Le code source (fichier .py, .c, .java) est éditable par l’humain, tandis que le fichier exécutable (.exe) est en langage machine illisible pour nous.

Processus de transformation : Code source → Traducteur → Programme exécutable / exécution directe

🧠 Pour aller plus loin : vocabulaire du développeur

API, bibliothèques, framework

Un développeur ne réinvente pas toujours la roue. Il utilise souvent :

  • Bibliothèque : ensemble de fonctions prêtes à l’emploi (ex: math.h en C).
  • API (Application Programming Interface) : interface permettant à des logiciels de communiquer entre eux.
  • Framework : structure qui facilite le développement d’applications (ex: Django pour Python).

Ces outils s’appuient sur des langages de haut niveau et sont utilisés pour créer des logiciels plus rapidement.

Exemple concret : développement d’une calculatrice

Pour créer une calculatrice scientifique, un développeur peut utiliser le langage Python (haut niveau) et l’interpréteur Python.

Étapes :

  1. Écrire le code source (Python) avec des fonctions d’addition, multiplication, etc.
  2. L’interpréteur Python lit le code et le convertit en instructions machine à la volée.
  3. L’utilisateur lance le script, l’ordinateur exécute les opérations.
Si le développeur avait choisi le C, il aurait dû compiler le code pour obtenir un exécutable indépendant.

🐍 À retenir : Python (interprété) est idéal pour prototyper, C (compilé) pour des logiciels nécessitant rapidité.

🌍 Applications dans la vie réelle

  • ✅ Chaque application mobile (WhatsApp, Instagram) : écrite en langage de haut niveau (Kotlin, Swift) puis compilée pour être installée sur smartphone.
  • ✅ Navigateurs web : ils intègrent des interpréteurs JavaScript pour exécuter du code directement dans la page.
  • ✅ Systèmes d’exploitation : développés majoritairement en C/C++ (compilés) car ils nécessitent une communication directe avec le matériel.
