UE 23 : Structure de base d’un algorithme

Informatique · Classe de 3ème · Module 4 : Initiation à l'algorithme

📌 Éléments clés à retenir

🔍 Qu'est-ce qu'un algorithme ?

Un algorithme est une suite d’instructions précises et ordonnées qui permet de résoudre un problème ou d’effectuer une tâche. C’est le « plan de cuisine » avant la réalisation du plat (le programme). En informatique, l’algorithme est indépendant du langage de programmation.

  • Caractéristiques : fini, précis, non ambigu, avec un début et une fin.
  • Objectif : transformer des données d'entrée en données de sortie.
  • Langage utilisé : langage naturel (français structuré), pseudo-code ou organigramme.
Figure 1 : Structure générale d’un algorithme

🏗️ Structure de base d’un algorithme

Tout algorithme se décompose en trois grandes parties obligatoires :

  • 1. En-tête (Nom) : Donne un nom explicite à l’algorithme. Exemple : Algorithme CalculMoyenne
  • 2. Zone des déclarations : On y déclare les variables et constantes utilisées. Une variable est une zone mémoire qui peut contenir une valeur qui évolue. Exemple : Variables : note1, note2, moyenne : réels
  • 3. Corps de l’algorithme : Ensemble des instructions (entrée, traitement, sortie) qui exécutent la tâche. Délimité par Début et Fin.
// Exemple de structure type
Algorithme ExempleStructure
Variables
   x, y : entier
Début
   // Instructions du corps
   x ← 5
   y ← x + 3
   Écrire(y)
Fin

⚙️ Instructions de base

Dans un algorithme, trois familles d'instructions simples sont essentielles :

  • 📥 Entrée (Lire) : Permet de saisir des données depuis l'utilisateur. Lire(variable)
  • 🧮 Affectation (← ou =) : Stocke une valeur dans une variable. resultat ← 10 + 5 ou nom ← "Jean"
  • 📤 Sortie (Écrire/Afficher) : Affiche un résultat ou un message. Écrire("La somme est : ", somme)

Remarque : L'affectation n'est pas une égalité mathématique ! compteur ← compteur + 1 signifie "prend la valeur actuelle de compteur, ajoute 1 et stocke le nouveau résultat".

💡 Exemples de compréhension

Exemple 1 : Calcul de la somme de deux nombres

Écrire un algorithme qui demande deux nombres à l'utilisateur, calcule leur somme et affiche le résultat.

Solution :
Algorithme SommeDeuxNombres
Variables
   a, b, s : réel
Début
   Écrire("Entrez le premier nombre : ")
   Lire(a)
   Écrire("Entrez le deuxième nombre : ")
   Lire(b)
   s ← a + b
   Écrire("La somme est : ", s)
Fin
💬 Astuce : Toujours bien déclarer toutes les variables avant de les utiliser. Ici, la variable s stocke le résultat.

Exemple 2 : Calcul d'une moyenne simple

Concevoir un algorithme qui lit trois notes, calcule leur moyenne et affiche "Moyenne : X".

Étapes :
  • 1. Déclarer les variables (note1, note2, note3, moyenne).
  • 2. Lire les trois notes.
  • 3. Calculer moyenne = (note1 + note2 + note3) / 3.
  • 4. Afficher le résultat.
Algorithme CalculMoyenne
Variables
   n1, n2, n3, moy : réel
Début
   Écrire("Saisir les trois notes :")
   Lire(n1, n2, n3)
   moy ← (n1 + n2 + n3) / 3
   Écrire("La moyenne est : ", moy)
Fin
💡 Pensez à utiliser des noms de variables explicites (note_français, note_maths) pour plus de clarté.

Exemple 3 : Échanger le contenu de deux variables

On dispose de deux variables A et B contenant des valeurs différentes. Écrire un algorithme qui échange leurs contenus (A prend la valeur de B et B prend la valeur de A).

Solution avec variable temporaire :
Algorithme EchangeValeurs
Variables
   A, B, temp : entier
Début
   A ← 10
   B ← 25
   Écrire("Avant échange : A=", A, " B=", B)
   // Échange
   temp ← A
   A ← B
   B ← temp
   Écrire("Après échange : A=", A, " B=", B)
Fin
⚙️ Sans une variable temporaire, on perdrait une valeur. Cette technique est classique en algorithmique.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier de déclarer une variable avant de l'utiliser dans le corps de l'algorithme.
  • Confusion : Le symbole d'affectation ← n’est pas une égalité mathématique. x ← x + 1 est correct en algorithme mais impossible en mathématiques.
  • Attention : Les instructions Lire et Écrire doivent respecter l'ordre logique. On ne peut pas afficher un résultat avant de l'avoir calculé.
  • Piège : Ne pas mettre les guillemets pour les chaînes de caractères dans les sorties (Écrire("Bonjour") est correct, Écrire(Bonjour) génère une erreur).

Mini-exercice : Périmètre d'un rectangle

Complète l’algorithme suivant pour calculer et afficher le périmètre d’un rectangle (formule : P = 2 × (longueur + largeur)).

Algorithme PerimetreRectangle
Variables
   L, l, P : réel
Début
   Écrire("Donner la longueur : ")
   ___________(L) // instruction manquante
   Écrire("Donner la largeur : ")
   Lire(l)
   P ← __________ // formule manquante
   ___________( "Le périmètre est : ", P) // instruction manquante
Fin
✅ Réponses : 1. Lire(L)   2. 2 * (L + l)   3. Écrire
