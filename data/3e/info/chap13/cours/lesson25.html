UE 25 : Structures alternatives et itératives

Algorithmique & Programmation · Classe de 3ème · Programme Cameroun

📌 Éléments clés à retenir

1. Structures alternatives (conditionnelles)

Les structures alternatives permettent d'exécuter différentes parties d'un algorithme selon qu'une condition est vraie ou fausse. Elles introduisent la logique de choix et de décision.

  • Structure SI ... ALORS ... SINON : exécute un bloc si la condition est VRAIE, un autre bloc si elle est FAUSSE.
  • Structure SI ... ALORS : version simplifiée (sans SINON), un bloc s'exécute seulement si la condition est vraie.
  • Condition : expression logique (comparaison, égalité, opérateurs ET/OU) qui donne un résultat booléen (VRAI ou FAUX).
  • Opérateurs de comparaison courants : = (égal), != (différent), < , > , <= , >= .
Exemple de syntaxe (langage naturel) :
SI (note >= 10) ALORS
    Afficher("Admis")
SINON
    Afficher("Non admis")
FIN SI

2. Structures itératives (boucles)

Les boucles permettent de répéter un bloc d'instructions plusieurs fois, soit un nombre déterminé de fois (POUR), soit tant qu'une condition est vérifiée (TANT QUE).

🔁 Boucle POUR

Utilisée quand on connaît à l'avance le nombre d'itérations.

POUR i DE 1 À N FAIRE
    // instructions répétées
FIN POUR

Exemple : afficher les nombres de 1 à 10, calculer la somme des notes d'une classe.

🔄 Boucle TANT QUE

Utilisée quand on ne connaît pas le nombre d'itérations à l'avance, mais une condition d'arrêt.

TANT QUE (condition) FAIRE
    // instructions répétées
FIN TANT QUE

Exemple : saisir un mot de passe jusqu'à ce qu'il soit correct.

  • Boucle infinie : si la condition d'une boucle TANT QUE n'est jamais fausse, le programme tourne indéfiniment. Toujours s'assurer que la condition évolue !
  • Itération : une seule exécution du corps de la boucle.
  • On peut imbriquer des structures alternatives à l'intérieur des boucles (et vice-versa).

3. Opérateurs logiques (ET, OU, NON)

Pour créer des conditions complexes, on combine plusieurs expressions avec des opérateurs logiques.

  • ET (AND) : le résultat est VRAI seulement si les deux conditions sont VRAIES.
  • OU (OR) : le résultat est VRAI si au moins une des conditions est VRAIE.
  • NON (NOT) : inverse la valeur logique (VRAI devient FAUX, FAUX devient VRAI).
Exemple combiné :
SI (age >= 18 ET age <= 65) ALORS
    Afficher("Éligible")
SINON
    Afficher("Non éligible")
FIN SI

💡 Exemples de compréhension

Exemple 1 : Mention d'un élève

Contexte : Un professeur saisit la moyenne d'un élève. L'algorithme doit afficher la mention correspondante : "Très bien" (moyenne ≥ 16), "Bien" (14 ≤ moyenne < 16), "Assez bien" (12 ≤ moyenne < 14), "Passable" (10 ≤ moyenne < 12), "Insuffisant" (moyenne < 10).

Résolution avec SI imbriqués :
ALGORITHME MentionEleve
VARIABLES moyenne : REEL
DEBUT
    ECRIRE("Entrez la moyenne : ")
    LIRE(moyenne)
    SI (moyenne >= 16) ALORS
        ECRIRE("Très bien")
    SINON SI (moyenne >= 14) ALORS
        ECRIRE("Bien")
    SINON SI (moyenne >= 12) ALORS
        ECRIRE("Assez bien")
    SINON SI (moyenne >= 10) ALORS
        ECRIRE("Passable")
    SINON
        ECRIRE("Insuffisant")
    FIN SI
FIN

💡 On utilise SINON SI pour tester plusieurs conditions exclusives. L'ordre est important : on part des cas les plus stricts.

💬 Astuce : Toujours penser à couvrir toutes les possibilités. Utiliser des bornes claires pour éviter les oublis.

Exemple 2 : Somme des 10 premiers entiers (POUR)

Calculer la somme des entiers de 1 à 10 à l'aide d'une boucle POUR.

ALGORITHME SommeEntiers
VARIABLES i, somme : ENTIER
DEBUT
    somme ← 0
    POUR i DE 1 À 10 FAIRE
        somme ← somme + i
    FIN POUR
    ECRIRE("La somme de 1 à 10 est : ", somme)
FIN

Explication : La variable somme accumule les valeurs de i à chaque itération. Au départ somme=0, puis après i=1 → somme=1, i=2 → somme=3, etc. À la fin on affiche 55.

Exemple 3 : Contrôle d'accès (TANT QUE)

On demande à l'utilisateur de saisir un code secret. Tant que le code saisi est différent du code attendu ("1234"), on répète la demande.

ALGORITHME AccesSecurise
VARIABLES code_saisi : CHAINE, code_attendu : CHAINE ← "1234"
DEBUT
    ECRIRE("Entrez le code d'accès : ")
    LIRE(code_saisi)
    TANT QUE (code_saisi != code_attendu) FAIRE
        ECRIRE("Code incorrect. Réessayez : ")
        LIRE(code_saisi)
    FIN TANT QUE
    ECRIRE("Accès autorisé. Bienvenue !")
FIN

📌 On commence par lire une première fois le code avant la boucle, puis la condition vérifie si on doit redemander.

⚠️ Attention à ne pas oublier la lecture à l'intérieur de la boucle, sinon la condition ne changera jamais → boucle infinie !

Exemple 4 : Compter les nombres pairs dans une liste

On saisit 5 nombres un par un. L'algorithme compte combien de ces nombres sont pairs (divisibles par 2).

ALGORITHME CompterPairs
VARIABLES i, nbre, compteur : ENTIER
DEBUT
    compteur ← 0
    POUR i DE 1 À 5 FAIRE
        ECRIRE("Donnez un nombre : ")
        LIRE(nbre)
        SI (nbre % 2 == 0) ALORS // % désigne le reste de la division
            compteur ← compteur + 1
        FIN SI
    FIN POUR
    ECRIRE("Nombre de nombres pairs : ", compteur)
FIN

Explication : On utilise une boucle POUR pour lire les 5 valeurs. À chaque itération, une condition SI vérifie si le nombre est pair, et incrémente le compteur si c'est le cas.

⚠️ Pièges courants & erreurs fréquentes

  • Confondre = et == : En algorithmique et en programmation, "=" est l'affectation, tandis que "==" est la comparaison d'égalité. Dans les conditions SI, on utilise toujours la comparaison (ou directement le test logique).
  • Boucle infinie oubliée : Dans une structure TANT QUE, oublier de modifier la variable de la condition à l'intérieur de la boucle provoque une répétition infinie. Vérifier toujours l'évolution.
  • Mauvais ordre des SI/SINON SI : Placer des conditions plus larges avant des conditions plus précises peut masquer des cas. Exemple : SI (note >= 10) ... SINON SI (note >= 16) ne fonctionnera jamais pour les notes ≥16.
  • Ne pas initialiser les accumulateurs : Avant une boucle qui calcule une somme ou un compteur, il faut initialiser la variable à 0 (ou la valeur de base).
  • Indentation & lisibilité : Ne pas bien structurer visuellement les blocs SI et les boucles rend le code difficile à comprendre et génère des erreurs logiques.
  • Oublier le FIN SI ou FIN POUR : Dans les algorithmes en langage naturel, bien fermer chaque structure permet d'éviter les ambiguïtés.
