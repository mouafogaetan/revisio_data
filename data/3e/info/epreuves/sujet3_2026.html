📌 Partie A – Exercice 1

Réponse 1.a Arborescence : structure hiérarchique de dossiers (ou répertoires) permettant d’organiser les fichiers de manière arborescente, avec une racine et des embranchements. [citation:2]

Réponse 1.b Chemin absolu : C:\Users\Nom\Documents\Scolaire\Troisieme\rapport_trimestre.txt (sous Windows) ou /home/nom/Documents/Scolaire/Troisieme/rapport_trimestre.txt (Linux).

Réponse 1.c Métadonnées : date de modification, taille du fichier, auteur. Elles aident à décrire le fichier sans ouvrir son contenu.

Réponse 1.d Un fichier contient des données (texte, image, programme). Un dossier contient des fichiers ou d’autres dossiers (structure logique).

💽 Exercice 2 – Conversions

2.a 1011011₂ = 1×64 + 0×32 + 1×16 + 1×8 + 0×4 + 1×2 + 1×1 = 64+16+8+2+1 = 91₁₀.

2.b 105₁₀ → 105/2=52 reste 1, 52/2=26 r0, 26/2=13 r0, 13/2=6 r1, 6/2=3 r0, 3/2=1 r1, 1/2=0 r1 => lecture inverse : 01101001 (8 bits).

2.c Code décimal 65 correspond au caractère 'A' (ASCII). Le codage ASCII permet de représenter des caractères (lettres, chiffres) en binaire pour que l’ordinateur les interprète.

🎨 Exercice 3 – Multimédia

3.a Bitmap : image matricielle formée de pixels (ex: JPEG, PNG). Vectorielle : formée d’objets géométriques (ex: SVG, AI).

3.b La résolution (en ppp) définit le nombre de pixels par pouce. Une haute résolution augmente le poids du fichier.

3.c Formats audio : MP3 (compressé avec perte), WAV (qualité supérieure, non compressé). Le WAV offre une meilleure fidélité.

🛠️ Partie B – Cas pratique

B.1 Maintenance préventive : actions pour éviter les pannes (nettoyage, mises à jour). Maintenance corrective : réparer après panne. Dans le contexte, effectuer une défragmentation/nettoyage est préventif. [citation:5]

B.2 Arborescence proposée : Racine → (Dossier "ECOLE" → sous-dossiers "Cours", "Exercices") et (Dossier "PERSO" → "Projets_photos", "Factures").

B.3 Méthodes pour récupérer fichier : utiliser la corbeille, restaurer une version précédente (historique des fichiers), ou un logiciel de récupération comme Recuva.

B.4 Espace restant = 500 Go – 350 Go = 150 Go. Conversion : 150 × 1024 = 153 600 Mo.

🧩 Partie C – Logique

C.1 Bit → Octet (8 bits) → Kilooctet (1024 o) → Mégaoctet (1024 Ko) → Gigaoctet (1024 Mo).

C.2 Algorigramme de recherche : (Début) → [Ouvrir dossier racine] → (Décision : fichier "examen.pdf" trouvé ?) → Oui → [Afficher chemin] → Fin ; Non → [Parcourir sous-dossiers].

C.3 L’arborescence permet un accès rapide, une organisation logique et une gestion efficace des droits/permissions.
