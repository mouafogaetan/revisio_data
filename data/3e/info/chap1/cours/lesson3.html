RISC et CISC : Deux philosophies de conception

Les architectures RISC (Reduced Instruction Set Computer) et CISC (Complex Instruction Set Computer) représentent deux approches différentes pour concevoir le jeu d'instructions d'un processeur. Cette distinction influence les performances, la consommation énergétique et la complexité du matériel.

  • RISC (Jeu d'instructions réduit) : Ensemble d'instructions simples, de taille fixe, exécutées en un seul cycle d'horloge. Privilégie la simplicité matérielle et l'efficacité énergétique.
  • CISC (Jeu d'instructions complexe) : Ensemble d'instructions variées et puissantes, de taille variable, pouvant prendre plusieurs cycles. Optimise le nombre d'instructions par programme.
  • Formule à retenir : RISC = Hardware simple + Software complexe et CISC = Hardware complexe + Software simple

Exemples de processeurs par architecture

🔷 RISC
  • ARM (Advanced RISC Machines) : Utilisé dans les smartphones (Android, iPhone), tablettes, Raspberry Pi, objets connectés. Dominant dans l'embarqué.
  • PowerPC : Historiquement utilisé par Apple (Mac G3, G4), consoles de jeux (GameCube, Wii, PS3).
  • MIPS : Utilisé dans certains routeurs, consoles (PlayStation 1, Nintendo 64).
  • RISC-V : Architecture open source en pleine expansion.
🔶 CISC
  • x86 (Intel, AMD) : Processeurs des PC de bureau, portables, serveurs. Exemples : Intel Core i5/i7/i9, AMD Ryzen.
  • Motorola 68000 : Utilisé dans les premiers Macintosh, Amiga, consoles (Mega Drive).
  • Zilog Z80 : Processeur légendaire des ordinateurs 8 bits (ZX Spectrum, Game Boy).

💡 Exemples de compréhension

Exemple concret n°1 : Pourquoi les smartphones utilisent du RISC ?

On souhaite concevoir un processeur pour un smartphone qui doit offrir une bonne autonomie et ne pas chauffer excessivement.

Raisonnement : L'architecture RISC (comme ARM) est idéale car :

  • Sa faible consommation énergétique préserve la batterie.
  • Elle produit moins de chaleur, ce qui est crucial dans un espace réduit sans ventilation.
  • La simplicité matérielle permet d'intégrer plusieurs cœurs (multicœur) pour gérer les tâches en parallèle.
💬 Astuce : Si l'appareil est mobile (smartphone, tablette, montre connectée), il y a de fortes chances qu'il utilise un processeur RISC (ARM) ! Les PC fixes et portables "lourds" utilisent plutôt du CISC (x86).

Exemple d'application n°2 : Compter les instructions

Pour ajouter deux nombres en mémoire, un processeur CISC peut utiliser une instruction unique : ADD [adresse1], [adresse2]. Un processeur RISC doit charger les données dans des registres, effectuer l'addition, puis les stocker en mémoire.

Comparaison :

  • CISC : 1 instruction → 1 ligne de code assembleur, mais plusieurs cycles d'horloge pour l'exécuter.
  • RISC : 3 instructions (LOAD → ADD → STORE) → 3 lignes de code, mais chaque instruction s'exécute en 1 cycle.

Cette différence montre que le RISC nécessite plus d'instructions (code plus long), mais l'exécution peut être plus rapide grâce au pipeline et à la fréquence élevée.

💡 À retenir : Le nombre d'instructions n'est pas le seul indicateur de performance. L'architecture influence aussi la vitesse d'exécution et la complexité du matériel.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Croire que RISC est "plus récent" ou "meilleur" que CISC. En réalité, les deux architectures cohabitent et évoluent. Les processeurs x86 modernes traduisent d'ailleurs les instructions CISC en micro-opérations RISC en interne.
  • Attention : Ne pas confondre "taille d'instruction fixe" avec "architecture 32/64 bits". Un processeur RISC peut être 32 bits ou 64 bits. La taille fixe signifie que toutes les instructions ont le même nombre de bits (ex: 32 bits).
  • Piège : Associer systématiquement performance brute = CISC. Aujourd'hui, les processeurs ARM RISC (M1, M2, M3 d'Apple) rivalisent et surpassent certains x86 en performance.

🖼️ Visualisation des architectures

Comment ça fonctionne en détail ?

Architecture RISC : Le processeur exécute une instruction par cycle grâce à un pipeline profond. Les instructions sont de taille fixe, ce qui simplifie le décodage et l'exécution en parallèle.

Architecture CISC : Le processeur utilise un microcode (un petit programme interne) pour décomposer chaque instruction complexe en plusieurs micro-opérations simples. Cela permet d'avoir un jeu d'instructions riche mais complexifie le matériel.

🧠 Moyen mnémotechnique

Pour retenir facilement

RISC = Rapide, Instructions Simples et Consommation réduite
CISC = Complexe, Instructions Spécialisées et Consommation plus élevée

🧩 Astuce visuelle : Imaginez un couteau suisse (CISC) qui fait beaucoup de choses avec un seul outil, mais qui est encombrant. Imaginez un ensemble de couteaux spécialisés (RISC) : chaque outil ne fait qu'une chose, mais on peut les utiliser très rapidement et efficacement.
