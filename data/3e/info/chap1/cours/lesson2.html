🏛️ Architecture de Von Neumann

UE 2 · Architecture d’un microordinateur · Troisième (3ᵉ) · Programme officiel Cameroun

📌 Éléments clés à retenir

🔹 Le modèle « programme enregistré »

Proposé par John von Neumann en 1945, ce modèle est à la base de la majorité des ordinateurs modernes. Son principe fondamental est que les données ET le programme (instructions) sont stockés dans la même mémoire. Cela permet à l’ordinateur d’exécuter des séquences d’instructions sans intervention humaine continue.

  • Mémoire unique : contient à la fois les données à traiter et les instructions du programme.
  • Cycle fondamental : « Fetch – Decode – Execute » (lecture de l’instruction, décodage, exécution).
  • Avantage : souplesse et programmabilité – on peut changer de programme sans modifier le matériel.
  • Limite : « goulot d’étranglement de Von Neumann » (bus unique ralentit l’accès simultané aux données et instructions).

🔹 Les composants de l’architecture Von Neumann

L’architecture se compose de 5 blocs interconnectés par un bus de communication :

  • 📀 Mémoire (RAM) : Stocke les données et les instructions. Chaque emplacement possède une adresse.
  • 🧠 Unité de Contrôle (UC) : Coordonne les échanges, décode les instructions et génère les signaux de commande.
  • ⚙️ Unité Arithmétique et Logique (UAL) : Effectue les calculs (+, −, ×, ÷) et les opérations logiques (ET, OU, NON).
  • ⌨️ Entrées : Périphériques permettant d’introduire des données (clavier, souris, scanner…).
  • 🖥️ Sorties : Périphériques qui restituent les résultats (écran, imprimante, haut-parleurs).
📐 Schéma simplifié : les échanges se font via le bus système (adresses, données et contrôle).

🔹 Cycle d’exécution d’une instruction

Le processeur répète sans cesse un cycle en trois phases :

  • 1. Lecture (Fetch) : L’unité de contrôle va chercher l’instruction suivante dans la mémoire (à l’adresse indiquée par le compteur ordinal).
  • 2. Décodage (Decode) : L’instruction est analysée pour comprendre l’opération à réaliser et les données concernées.
  • 3. Exécution (Execute) : L’UAL effectue l’opération (calcul, comparaison…) ou un transfert de données. Le résultat peut être stocké en mémoire ou envoyé en sortie.

💡 Ce cycle se répète des milliards de fois par seconde (fréquence du processeur).

💡 Exemples de compréhension

🧮 Exemple : Calcul de la somme 5 + 3

Un programme simple est chargé en mémoire : « charger 5 dans registre A, charger 3 dans registre B, additionner, afficher résultat ». Déroulement selon l’architecture Von Neumann.

Étapes :
1️⃣ Fetch : L’unité de contrôle lit l’instruction "CHARGER 5" à l’adresse mémoire #100.
2️⃣ Decode : L’instruction est décodée → opération = chargement d’une donnée vers un registre interne.
3️⃣ Execute : L’UAL (via l’unité de contrôle) transfère la valeur 5 dans un registre.
🔁 Le cycle se répète pour "CHARGER 3" puis pour l’instruction "ADD". Enfin, "AFFICHER" envoie le résultat (8) vers un périphérique de sortie.
💡 Astuce : C’est le programme en mémoire qui dicte l’ordre des opérations. On peut changer le programme sans changer le circuit électronique.

🍳 Analogie de la recette de cuisine

Pour mieux comprendre le rôle de chaque composant, comparons l’ordinateur à une cuisine.

  • 📖 Livre de recettes (Mémoire) : contient la recette (programme) et la liste des ingrédients (données).
  • 👨‍🍳 Chef cuisinier (Unité de contrôle) : lit la recette étape par étape et donne des ordres.
  • 🔪 Batterie de cuisine / Robot (UAL) : exécute les actions : couper, mélanger, cuire (calculer, comparer).
  • 🍅 Ingrédients entrants (Entrées) : ce qu’on apporte du marché.
  • 🍽️ Plat final (Sortie) : résultat présenté à table.

Le chef utilise la même fiche recette (mémoire) pour chaque plat ; il suit la séquence d’instructions.

🔁 Le « goulot » dans une cuisine : le chef doit souvent revenir lire la recette (accès mémoire) – c’est le goulot de Von Neumann.

⚠️ Pièges courants & idées reçues

  • ❌ « La mémoire RAM contient uniquement les données de l’utilisateur » : Faux, elle contient aussi les instructions du système d’exploitation et des programmes.
  • ❌ « L’UAL est la même chose que le processeur » : Le processeur (CPU) regroupe l’UAL ET l’unité de contrôle. L’UAL n’est qu’une partie.
  • ❌ « Dans Von Neumann, les entrées/sorties sont gérées uniquement par le CPU » : Elles communiquent via le bus, mais certains ordinateurs modernes utilisent des contrôleurs spécialisés ; le modèle de base reste valable.
  • ❌ « Le programme enregistré signifie qu’on ne peut exécuter qu’un seul programme à la fois » : Dans les ordinateurs actuels, le système d’exploitation permet le multitâche (partage du temps) mais la base conceptuelle reste la même.
  • ⚠️ Attention : Ne pas confondre Architecture de Von Neumann et Architecture Harvard (dans Harvard, instructions et données ont des bus et mémoires séparés). Au programme, on retient le modèle classique.

🔌 Rôle du bus système

Dans l’architecture de Von Neumann, tous les échanges entre la mémoire, l’unité de contrôle et l’UAL passent par un ensemble de lignes appelé bus. On distingue :

  • Bus de données : transporte les informations (valeurs, instructions).
  • Bus d’adresses : indique l’emplacement mémoire concerné.
  • Bus de contrôle : envoie les signaux de commande (lecture/écriture, horloge).

Ce bus unique crée un goulot d’étranglement (le processeur peut être ralenti car il ne peut pas lire instruction et données simultanément). C’est un point faible du modèle.

🧠 À mémoriser absolument :

  • ✔️ Von Neumann = mémoire unique contenant données + programme.
  • ✔️ 5 composants : Mémoire, UC, UAL, Entrées, Sorties.
  • ✔️ Cycle : Fetch – Decode – Execute.
  • ✔️ C’est le modèle des ordinateurs actuels (PC, smartphones, serveurs).
