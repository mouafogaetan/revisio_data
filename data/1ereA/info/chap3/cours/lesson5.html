Leçon 5 : Structures de contrôle

Algorithmique et programmation · Première A, ABI, AC, SH · UE9

📌 Éléments clés à retenir

1. Notions fondamentales

  • Algorithme : Suite finie et ordonnée d'instructions qui résout un problème.
  • Variable : Zone mémoire nommée pouvant contenir une valeur qui peut changer au cours de l'exécution.
  • Instruction : Action élémentaire exécutée par l'ordinateur (calcul, saisie, affichage...).
  • Instruction d'affectation : Action de donner une valeur à une variable. variable ← valeur ou variable = valeur
  • Instruction d'affichage : Action d'afficher un résultat à l'écran. Afficher variable ou Écrire variable

2. Les structures alternatives (Conditionnelles)

Permettent d'exécuter une instruction ou un bloc d'instructions seulement si une condition est vraie.

  • Structure Si... Alors : Si condition Alors instruction Fin Si
  • Structure Si... Alors... Sinon : Si condition Alors instruction1 Sinon instruction2 Fin Si
  • La condition est une expression logique (comparaison : =, >, <, >=, <=, ≠).

3. Les structures répétitives (Boucles)

Permettent de répéter un bloc d'instructions un certain nombre de fois.

  • Boucle Pour (itération définie) : Pour variable de début à fin Faire instruction(s) Fin Pour
    On connaît à l'avance le nombre de répétitions.
  • Boucle Tant que (itération indéfinie) : Tant que condition Faire instruction(s) Fin Tant que
    On répète tant que la condition est vraie. Le nombre de répétitions n'est pas connu à l'avance.

4. Trace d'exécution

Technique qui consiste à suivre pas à pas l'évolution des variables d'un algorithme pour comprendre son fonctionnement ou vérifier son résultat.

  • On dresse un tableau avec une colonne pour chaque variable.
  • On note les changements de valeurs à chaque instruction.
  • On termine par la (ou les) valeur(s) affichée(s).

💡 Exemples de compréhension

Exemple 1 : Algorithme avec alternative (permutation de deux variables)

Soit l'algorithme suivant :
Variables A, B, C : entiers
Début
  A ← 5
  B ← 3
  Afficher "A = ", A, " B = ", B
  Si A < B Alors
    C ← A
    A ← B
    B ← C
  Fin Si
  Afficher "A = ", A, " B = ", B
Fin

Trace d'exécution :
- Étape 1 : A=5, B=3 → affichage : "A = 5 B = 3"
- Étape 2 : Test Si A < B ? 5 < 3 ? Faux → on ne fait rien.
- Étape 3 : affichage : "A = 5 B = 3" (les valeurs n'ont pas changé car la condition était fausse)
Résultat : Les valeurs restent 5 et 3.

💬 Astuce : Pour échanger deux variables, on a besoin d'une variable temporaire (C). Sans elle, on perdrait la valeur initiale.

Exemple 2 : Calcul de moyenne avec boucle POUR

Algorithme qui calcule la moyenne de 3 notes saisies par l'utilisateur :
Variables somme, note, moyenne : réels ; i : entier
Début
  somme ← 0
  Pour i de 1 à 3 Faire
    Afficher "Entrer la note ", i
    Lire note
    somme ← somme + note
  Fin Pour
  moyenne ← somme / 3
  Afficher "Moyenne = ", moyenne
Fin

Exemple de déroulement avec notes : 12 , 15 , 10
- i=1 : somme = 0 + 12 = 12
- i=2 : somme = 12 + 15 = 27
- i=3 : somme = 27 + 10 = 37
- moyenne = 37 / 3 = 12,33
- Affichage : "Moyenne = 12.33"

💡 La variable somme doit toujours être initialisée à 0 avant la boucle. Sinon, le résultat est faux.

Exemple 3 : Structure conditionnelle (comparaison de deux nombres)

Algorithme qui affiche le plus grand de deux nombres A et B :
Variables A, B : entiers
Début
  Lire A, B
  Si A > B Alors
    Afficher A, " est le plus grand"
  Sinon
    Si B > A Alors
      Afficher B, " est le plus grand"
    Sinon
      Afficher "Les deux nombres sont égaux"
    Fin Si
  Fin Si
Fin

Test avec A=7, B=3 : La condition A>B (7>3) est vraie → affichage : "7 est le plus grand".
Test avec A=4, B=9 : A>B est faux → on va dans Sinon → test B>A (9>4) vrai → affichage : "9 est le plus grand".
Test avec A=5, B=5 : A>B faux → Sinon → B>A faux → Sinon → affichage : "Les deux nombres sont égaux".

💬 Astuce : Une structure conditionnelle peut contenir d'autres conditions imbriquées (Si... Sinon Si...).

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Confondre l'affectation = (ou ←) avec la comparaison d'égalité. En algorithmique, x ← 5 signifie "x prend la valeur 5" et non "x est égal à 5".
  • Attention : Oublier d'initialiser une variable (comme somme) avant une boucle conduit à des résultats aléatoires.
  • Piège : Dans une boucle Tant que, il faut s'assurer que la condition devienne fausse un jour, sinon la boucle est infinie (l'algorithme ne s'arrête jamais).
  • À ne pas confondre : = pour l'affectation (prend la valeur) et == ou = dans la condition selon les langages. En algorithmique, on utilise souvent ← pour l'affectation et = pour la comparaison.

📊 Les organigrammes (UE10)

Symboles à connaître

  • Début/Fin : Ovalle ou Rectangle aux bords arrondis
  • Instruction (traitement) : Rectangle
  • Entrée/Sortie (Lire/Afficher) : Parallélogramme
  • Test conditionnel (alternative) : Losange (deux branches : VRAI et FAUX)
  • Flèches : indiquent l'ordre d'exécution
