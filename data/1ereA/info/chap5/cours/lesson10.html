📐 Leçon 10 : Les organigrammes

Informatique · Première A, ABI, AC, SH · Représentation graphique des algorithmes

📌 Éléments clés à retenir

🔷 Qu'est-ce qu'un organigramme ?

Un organigramme (ou logigramme) est une représentation graphique d’un algorithme. Il utilise des symboles normalisés reliés par des flèches qui indiquent l’ordre d’exécution des instructions. Cela permet de visualiser facilement la logique du programme, les décisions et les répétitions.

  • Avantage : lecture plus intuitive qu’un algorithme textuel, utile pour la conception ou la documentation.
  • Normalisation : les symboles suivent la norme ISO 5807 ou la norme AFNOR.

📐 Les principaux symboles

⬭ ELLIPSE / OVALE
Début / Fin
Représente le point d’entrée et de sortie de l’algorithme.
▭ PARALLÉLOGRAMME
Entrée / Sortie
Lecture (Lire) ou affichage (Écrire).
▯ RECTANGLE
Traitement / Action
Calcul, affectation, instruction simple.
◇ LOSANGE
Test conditionnel
Structure alternative (si ... alors ... sinon).
⬄ FLÈCHE
Liens / Flux
Enchaînement des étapes.

On utilise parfois des cercles ou connecteurs pour éviter des flèches trop longues ou pour continuer sur une autre page.

🧠 Structures de contrôle en organigramme

  • Structure séquentielle : actions exécutées les unes après les autres (enchaînement rectangulaire).
  • Structure alternative (SI...ALORS...SINON) : un losange avec deux branches (Vrai / Faux).
  • Structure répétitive (boucle) :
    • Boucle POUR : on utilise un compteur avec un test de fin.
    • Boucle TANT QUE : test en tête (losange) qui renvoie vers une action.
    • Boucle RÉPÉTER JUSQU’À : test en pied (répéter l’action puis tester).

💡 Exemples de compréhension

Exemple 1 : Algorithme de comparaison de deux nombres

Texte Algorithme textuel :

Début
    Lire A, B
    Si A > B Alors
        Afficher "A est plus grand"
    Sinon
        Afficher "B est plus grand ou égal"
    Fin Si
Fin

Organigramme Version graphique correspondante :

💡 Le losange est indispensable pour toute alternative. La branche "Vrai" et "Faux" doivent être clairement indiquées.

Exemple 2 : Boucle POUR (somme de 1 à N)

Contexte : Calculer la somme des entiers de 1 jusqu’à un entier N saisi par l’utilisateur.

Début
    Lire N
    S ← 0
    Pour i de 1 à N faire
        S ← S + i
    Fin Pour
    Afficher S
Fin

Organigramme de la boucle POUR :

On représente généralement la boucle POUR par un test sur le compteur (i ≤ N) et un retour en arrière après incrémentation.

Exemple 3 : Boucle TANT QUE (validation d’un mot de passe)

L'utilisateur doit saisir un mot de passe. Tant que le mot de passe est différent de "secret", on redemande.

Début
    Lire mdp
    Tant que mdp ≠ "secret" faire
        Afficher "Erreur, ressaisir"
        Lire mdp
    Fin Tant que
    Afficher "Accès autorisé"
Fin
⚙️ En organigramme, le test (losange) est placé avant les actions répétées. La flèche "Vrai" revient vers le début de la boucle, la flèche "Faux" sort vers la suite.

✏️ Savoir lire et tracer un organigramme

Méthode pour tracer l’exécution

La trace d'exécution consiste à suivre manuellement chaque étape en notant l'évolution des variables.

  • Repérer le symbole Début (ovale).
  • Suivre les flèches dans l’ordre.
  • Pour un losange, évaluer la condition et choisir la branche correspondante.
  • Dans une boucle, revenir en arrière tant que la condition est vraie.
  • S’arrêter au symbole Fin.

Exemple de trace : Soit l’organigramme qui lit A, B puis calcule C = A + B et affiche C. Trace : A=5, B=3 → C=8 → affiche 8.

⚠️ Pièges courants à éviter

  • ❌ Ne pas utiliser les bons symboles : Par exemple, mettre un rectangle pour un test conditionnel.
  • ❌ Oublier le sens des flèches : Les flèches doivent toujours être orientées (sinon ambiguïté).
  • ❌ Absence de branche "Sinon" : Dans un losange, même si rien n’est fait dans le cas Faux, il faut le préciser.
  • ❌ Boucle infinie : Si la condition de sortie n’est jamais vraie (ex: incrémentation oubliée).
  • ❌ Confondre le parallélogramme (E/S) et le rectangle (traitement).

📝 Exercice type (examen probatoire)

Sujet : Organigramme et calcul de moyenne

On considère l’organigramme ci-dessous (à décrire). Celui-ci lit 3 notes N1, N2, N3, calcule la somme S puis la moyenne M. Si M ≥ 10, il affiche "Admis" sinon "Non admis".

Question 1 : Dessiner l’organigramme correspondant (symboles normalisés).
Question 2 : Faire la trace pour N1=12, N2=8, N3=15.

💬 Correction rapide :
• Début → Lire N1,N2,N3 → S ← N1+N2+N3 → M ← S/3 → Losange M ≥ 10 ? → Si vrai afficher "Admis" sinon "Non admis" → Fin.
• Trace : S = 12+8+15=35 → M = 35/3 ≈ 11.67 → 11.67 ≥ 10 → affiche "Admis".

🧠 Astuce pour mémoriser les symboles :
Ovale = "O" comme Ouverture/Fermeture.
Parallélogramme = forme de "Poche" pour entrer/sortir les données.
Rectangle = "R" comme Réaliser une action.
Losange = "L" comme Logique (décision).
