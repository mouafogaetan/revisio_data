📄 Généralités sur le HTML

Informatique · Première (A, ABI, AC, SH) · Module 2 – Programmation web

📌 Éléments clés à retenir

🌐 Définir : site web, page web, formulaire

Avant de coder, il est essentiel de maîtriser le vocabulaire de base du web.

  • Site web : ensemble de pages web reliées entre elles par des hyperliens, stockées sur un serveur (ex: www.cameroon-education.cm).
  • Page web : document unique, écrit en HTML, affiché dans un navigateur (Chrome, Firefox, etc.). Elle peut contenir du texte, des images, des vidéos, des liens.
  • Formulaire HTML : zone interactive permettant aux visiteurs de saisir et envoyer des données (nom, email, recherche). Balisé par <form>.
💡 À retenir : Un site web = plusieurs pages web reliées. Un formulaire sert à collecter des informations.

🛠️ Outils nécessaires à la création d'une page web

Pour fabriquer une page web en HTML, on a besoin de deux catégories d'outils.

  • Éditeur de texte : logiciel pour écrire le code HTML. Exemples : Bloc-notes (Windows), Notepad++, Visual Studio Code, Sublime Text.
  • Navigateur web : pour visualiser et tester la page. Exemples : Google Chrome, Mozilla Firefox, Microsoft Edge, Opera.
  • Optionnel mais utile : un serveur local (XAMPP, WampServer) pour simuler un vrai hébergement.
📌 Méthode simple : Écrire le code HTML dans un éditeur → enregistrer avec l’extension .html → ouvrir le fichier avec un navigateur.

🏗️ Structure de base d'une page HTML

Toute page HTML suit un squelette obligatoire. Les balises sont les mots-clés encadrés par < >.

<!DOCTYPE html>
<html>
    <head>
        <title>Titre de l'onglet</title>
    </head>
    <body>
        <h1>Bienvenue sur ma page</h1>
        <p>Ceci est un paragraphe.</p>
    </body>
</html>
  • <!DOCTYPE html> : déclare la version HTML5 (la plus récente).
  • <html> : racine du document.
  • <head> : contient des infos sur la page (titre, liens CSS).
  • <title> : texte affiché dans l’onglet du navigateur.
  • <body> : contient tout le contenu visible par l’utilisateur (textes, images, vidéos…).

💡 Exemples de compréhension

🔍 Exemple 1 – Analyser une page web simple

Observe le code HTML ci-dessous. Quelles sont les parties qui représentent le titre principal, un paragraphe et le titre de l'onglet ?

<html>
  <head><title>Mon premier site</title></head>
  <body>
    <h1>Les volcans du Cameroun</h1>
    <p>Le Mont Cameroun est un volcan actif.</p>
  </body>
</html>

Résolution :
• <title>Mon premier site</title> → titre dans l’onglet du navigateur.
• <h1>Les volcans du Cameroun</h1> → titre principal (niveau 1) visible dans la page.
• <p>Le Mont Cameroun est un volcan actif.</p> → paragraphe de texte.

💬 Astuce : Les balises ouvrantes <h1> et fermantes </h1> fonctionnent toujours par paires (sauf exceptions comme <br>).

📝 Exemple 2 – Créer un formulaire simple (conformément à UE14)

Un site de contact souhaite récupérer le nom et l'email d'un visiteur. Construis un début de formulaire HTML.

<form action="/envoyer.php" method="post">
  <label>Nom : </label>
  <input type="text" name="nom"><br>
  <label>Email : </label>
  <input type="email" name="email"><br>
  <input type="submit" value="Envoyer">
</form>

Explication :
• <form> : conteneur du formulaire. Les attributs action (où envoyer) et method (méthode d’envoi) sont importants.
• <input type="text"> : champ de saisie pour le texte.
• <input type="submit"> : bouton d’envoi.
• La balise <label> améliore l’accessibilité.

💡 À savoir : Dans le programme de Première, on doit définir la balise <form> et connaître sa syntaxe minimale.

🌍 Exemple 3 – Distinguer site web et page web

Le portail "Ministère des Enseignements Secondaires" propose différentes rubriques : Accueil, Actualités, Examens, Contact. Chaque rubrique est un fichier HTML différent (index.html, actualites.html, examens.html, contact.html).

Identification : L’ensemble des fichiers .html et des ressources (images, CSS) forment le site web. Chaque fichier (ex: examens.html) est une page web. Le formulaire de contact se trouve dans contact.html → c’est une page web contenant un formulaire.

📌 À retenir : La barre d’adresse change lorsqu’on navigue entre les pages d’un même site (ex: www.mon-site.cm/contact.html).

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier la balise fermante </html> ou </body>. Le navigateur peut mal interpréter la page.
  • Confusion : <head> et <header> : la première contient des métadonnées (invisible), la seconde (HTML5) est une zone d’en-tête visible dans la page.
  • Attention : Un formulaire sans attribut action envoie les données à la page elle-même, ce qui n’est pas toujours souhaité.
  • Idée reçue : On peut créer une page web uniquement avec un navigateur ? Non, il faut un éditeur de texte pour écrire le code HTML.
  • À ne pas confondre : Site web ≠ moteur de recherche (Google, Bing). Un moteur de recherche indexe des sites web.
