📄 Mise en forme en HTML

Informatique · Première A, ABI, AC, SH · Module 2 – Programmation web

📌 Éléments clés à retenir

🏷️ Balises de structure et de titre

En HTML, la mise en forme commence par des balises qui structurent le contenu. Les titres sont définis avec les balises <h1> à <h6>. <h1> représente le titre principal (niveau 1), <h2> le sous-titre, etc. La balise <br> permet d’insérer un saut de ligne simple.

  • <h1>Titre principal</h1> → Le plus important (un seul par page souvent).
  • <h2>Sous-titre</h2> → Organise les sections.
  • <h3> à <h6> → sous-niveaux hiérarchiques.
  • <br> : saut de ligne immédiat (balise orpheline, pas de balise fermante).
💡 À retenir : Les titres sont automatiquement mis en gras et avec un espacement. On ne doit pas utiliser <h1> pour le style uniquement, mais pour la structure logique du document.

📋 Créer des listes non ordonnées (liste à puces)

Les listes à puces permettent d’organiser des éléments sans ordre numérique. On utilise la balise conteneur <ul> (unordered list) et chaque élément de la liste est placé entre <li> (list item).

  • <ul> : englobe toute la liste.
  • <li> : chaque élément de la liste.
  • Exemple :
    <ul>
      <li>Lait</li>
      <li>Œufs</li>
    </ul>

✍️ Balises de mise en forme de texte

Même si le programme insiste sur titres, sauts de ligne et listes, d’autres balises simples sont fréquentes pour enrichir la présentation : <strong> (gras sémantique), <em> (italique), <p> (paragraphe).

  • <p>...</p> : définit un paragraphe, ajoute automatiquement une marge.
  • <strong>important</strong> : affiche en gras, importance pour les moteurs de recherche.
  • <em>accentué</em> : italique + emphase.

💡 Exemples de compréhension

Exemple 1 : Titres et sauts de ligne

Objectif : créer un mini document présentant un titre principal, un sous-titre, et deux lignes séparées par un saut de ligne.

Code HTML : 
<h1>Bienvenue sur mon site</h1>
<h2>Présentation des services</h2>
<p>Ligne 1 : Nos formations en informatique<br>
Ligne 2 : Sessions pratiques en HTML</p>
Résultat visuel : Le navigateur affiche un grand titre, un sous‑titre, puis un paragraphe avec un saut de ligne entre les deux phrases.
💬 Astuce : la balise <br> est utile pour des adresses ou des poèmes, mais pour structurer des paragraphes, préférez <p>.

Exemple 2 : Liste des prérequis (liste non ordonnée)

Dans une page web sur "Les bases de l'informatique", on souhaite lister les logiciels nécessaires.

Code HTML : 
<h3>Matériel & logiciels recommandés :</h3>
<ul>
    <li>Navigateur Web (Chrome, Firefox)</li>
    <li>Éditeur de texte (VS Code, Notepad++)</li>
    <li>Connexion Internet</li>
</ul>
Rendu :
  • Navigateur Web (Chrome, Firefox)
  • Éditeur de texte (VS Code, Notepad++)
  • Connexion Internet
✨ Les listes non ordonnées sont parfaites pour des caractéristiques, des étapes sans ordre particulier ou des menus.

Exemple 3 : Page personnelle simplifiée

On veut créer un petit extrait de page contenant un titre de niveau 1, un titre de niveau 2, une liste à puces et un saut de ligne avant un message.

Code complet : 
<h1>Mon Portfolio</h1>
<h2>Compétences numériques</h2>
<ul>
    <li>Création de pages web (HTML/CSS)</li>
    <li>Algorithmique et logique</li>
    <li>Sécurité informatique</li>
</ul>
<p>Contactez-moi : <br>email: eleve@lycee.cm</p>
Analyse : Les titres structurent la page, la liste aère l'information et <br> permet de retour à la ligne dans l’adresse mail.

⚠️ Pièges courants & erreurs en HTML

  • Oublier la fermeture des balises : <h1>Titre</h1> → sans </h1>, la mise en forme s’étend à toute la suite.
  • Confondre <br> et <br /> : En HTML5 les deux sont valides, mais le plus simple est <br> (pas de balise fermante).
  • Utiliser <ul> sans <li> : Une liste doit impérativement contenir des éléments <li>, sinon le navigateur peut mal interpréter.
  • Hiérarchie des titres : Ne pas sauter de niveaux (ex: <h1> puis <h3> directement) nuit à l'accessibilité.
  • Sauts de ligne abusifs : Pour espacer des blocs, utilisez CSS ou plusieurs balises <p> plutôt qu’une cascade de <br>.
