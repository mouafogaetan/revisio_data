📄 UE 14 : Insertion d’un formulaire en HTML

Informatique · 1ère A · Unité d’apprentissage 6 : Programmer en HTML

📌 Éléments clés à retenir

🧩 1. La balise <form> : conteneur du formulaire

Toute zone de formulaire (saisie, boutons, menus) doit être encapsulée dans la balise <form>. Elle permet d’envoyer les données vers un serveur (même si en classe on se concentre sur la structure).

  • Syntaxe minimale : <form> ... </form>
  • Attributs fréquents (à connaître) : action (URL de traitement) et method (get/post).
  • Exemple : <form action="traitement.php" method="post">
💡 Astuce : Même sans backend, créez toujours un formulaire avec <form> pour organiser les champs.

✍️ 2. La balise <input> : créer des champs interactifs

<input> est une balise auto-fermante. Elle sert à capturer différentes informations selon l’attribut type.

  • Texte court : <input type="text" name="prenom">
  • Mot de passe : <input type="password" name="mdp"> (caractères masqués)
  • Bouton d’envoi : <input type="submit" value="Envoyer">
  • Email / numérique : type="email" ou type="number"

L’attribut name est essentiel : c’est le nom de la donnée qui sera envoyée.

📝 3. Zones de texte longues & menus déroulants

Pour un commentaire ou un message, on utilise <textarea>. Pour une liste d’options, on utilise <select> + <option>.

  • Textarea : <textarea rows="4" cols="50" name="commentaire"></textarea>
  • Select : 
    <select name="classe">
  <option value="1ereA">Première A</option>
  <option value="1ereB">Première B</option>
</select>

✅ Toujours associer un name pour récupérer la valeur.

🔘 Bouton d’envoi (submit) et bonnes pratiques

Pour valider le formulaire, on utilise soit <input type="submit"> soit la balise <button type="submit">.

  • submit classique : <input type="submit" value="S’inscrire">
  • Pour améliorer l’accessibilité : utiliser <label> associé à un champ. Exemple : <label for="nom">Nom :</label> <input type="text" id="nom" name="nom">

💡 Exemples de compréhension

📝 Exemple 1 : Formulaire d’inscription simple

Énoncé : Créer un formulaire HTML qui demande le nom, l’email et un mot de passe, avec un bouton « Créer mon compte ».
✅ Code HTML à retenir : 
<form action="#" method="post">
  <label for="nom">Nom complet :</label>
  <input type="text" id="nom" name="nom" required><br><br>

  <label for="email">Adresse email :</label>
  <input type="email" id="email" name="email"><br><br>

  <label for="mdp">Mot de passe :</label>
  <input type="password" id="mdp" name="mot_de_passe"><br><br>

  <input type="submit" value="Créer mon compte">
</form>

Explication : L’attribut required empêche l’envoi si le champ est vide. Le type="email" vérifie rapidement le format (@).

💬 Astuce examen : N’oubliez jamais la balise <form> et l’attribut name pour chaque champ !

💬 Exemple 2 : Formulaire de contact (sujet + message)

Contexte : Site web d’un lycée. L’élève doit créer une zone pour choisir le service (info, scolarité) et laisser un message. 
<form method="post">
  <label>Sujet : 
    <select name="service">
      <option>Informatique</option>
      <option>Scolarité</option>
      <option>Vie scolaire</option>
    </select>
  </label><br><br>

  <label>Message :<br>
    <textarea name="msg" rows="4" cols="40" placeholder="Écrivez votre demande ici..."></textarea>
  </label><br><br>

  <button type="submit">Envoyer le message</button>
</form>

Résolution : <select> propose plusieurs choix, <textarea> autorise un long texte. Le bouton <button type="submit"> remplace input submit mais fait la même action.

✨ Rappel : la balise <button> peut contenir du texte enrichi (icônes, gras) alors que input submit est limité.

🔢 Exemple 3 : Utiliser case à cocher et radio

Dans un formulaire d’évaluation, on souhaite savoir le niveau de l’élève (débutant, intermédiaire, avancé) et accepter les conditions. 
<form>
  <p>Niveau :</p>
  <input type="radio" name="niveau" value="debutant" id="deb"> <label for="deb">Débutant</label><br>
  <input type="radio" name="niveau" value="intermediaire" id="inter"> <label for="inter">Intermédiaire</label><br>
  
  <input type="checkbox" name="accepte" id="cgu">
  <label for="cgu">J’accepte les conditions d’utilisation</label><br>
  <input type="submit" value="Valider">
</form>

Remarque : Pour les boutons radio, le name identique permet un seul choix. Le checkbox est indépendant.

⚠️ Pièges courants & erreurs fréquentes

  • ❌ Oublier la balise <form> : Sans elle, les champs existent mais ne forment pas un bloc cohérent. On perd la logique d’envoi.
  • ❌ Confondre name et id : name est utilisé pour transmettre les données, id sert à lier <label> et CSS/JS. Les deux sont utiles mais ne pas omettre name pour les inputs de formulaire.
  • ❌ Ne pas fermer les balises <textarea> : Attention, <textarea> n’est pas auto-fermante ; il faut écrire <textarea></textarea>.
  • ❌ Mauvais type pour un mot de passe : utiliser type="password" au lieu de type="text" pour masquer la saisie.
  • 🎓 Conseil prof : Pensez toujours à la structure : <form> → champs (avec name) → bouton submit.

📎 Synthèse visuelle : balises essentielles

📖 Mini-mémo du développeur web (formulaires HTML)

  • <form> : conteneur principal
  • <input type="text"> : champ texte une ligne
  • <input type="password"> : mot de passe
  • <input type="email"> : adresse email
  • <input type="submit"> : bouton d’envoi
  • <textarea> : zone de texte multiligne
  • <select> + <option> : liste déroulante
  • <label> : améliore l’accessibilité, cliquer sur le texte sélectionne le champ

À retenir pour l’évaluation : Savoir écrire la structure d’un formulaire avec au moins 3 types de champs différents et un bouton d’envoi.
