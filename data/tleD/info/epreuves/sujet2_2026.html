📌 PARTIE I : INTRODUCTION AU LANGAGE C ET STRUCTURES DE CONTRÔLE (7 points)

Exercice 1 : QCM et théorie (3 pts)

1.1 (1 pt) En langage C, quelle est la fonction obligatoire dans tout programme ?

  • A) start()
  • B) principal()
  • C) main()
  • D) execute()

✅ Réponse C : main() — Tout programme C débute par la fonction main(), point d'entrée obligatoire.

1.2 (1 pt) Quel est le bon moyen de déclarer une constante pour la valeur π en C ?

  • A) const float PI = 3.14;
  • B) #define PI 3.14
  • C) int PI = 3.14;
  • D) A et B sont possibles selon le contexte

✅ Réponse D — Avec #define (préprocesseur) ou const (variable qualifiée) sont deux façons valides.

1.3 (1 pt) Vrai ou Faux ? En C, une boucle do...while exécute le bloc d'instructions au moins une fois, même si la condition est fausse.

✅ VRAI — do {...} while (condition); teste la condition après exécution, garantissant au moins une itération.

Exercice 2 : Analyse et écriture de code (4 pts)

2.1 (2 pts) Soit l'extrait de programme C suivant :

int a = 10, b = 3;
if (a > b) {
    printf("Max = %d", a);
} else {
    printf("Max = %d", b);
}

➜ Quelle est la sortie affichée ? Justifiez brièvement.

✅ Sortie : Max = 10. La condition a > b est vraie (10 > 3), donc le bloc if s'exécute.

2.2 (2 pts) Écrire un programme en C qui affiche la table de multiplication de 5 (de 1 à 10) en utilisant une boucle for.

✅ Code possible :

#include <stdio.h>
int main() {
    int i;
    for(i=1; i<=10; i++) {
        printf("5 x %d = %d\n", i, 5*i);
    }
    return 0;
}
📌 PARTIE II : TABLEAUX ET ALGORITHMES DE RECHERCHE (6 points)

Exercice 3 : Manipulation de tableaux (3 pts)

On donne le tableau d'entiers suivant : int notes[5] = {12, 15, 8, 19, 10};

3.1 (1,5 pt) Écrire une boucle en C qui calcule et affiche la somme des éléments du tableau.

int somme = 0;
for(int i=0; i<5; i++) {
    somme += notes[i];
}
printf("Somme = %d", somme); // résultat : 64

3.2 (1,5 pt) En utilisant la recherche séquentielle, écrivez une fonction qui détermine si la valeur 19 se trouve dans le tableau et affiche son indice.

int rechercher(int tab[], int taille, int val) {
    for(int i=0; i<taille; i++)
        if(tab[i] == val) return i;
    return -1;
}
// Appel : rechercher(notes,5,19) → retourne 3

Exercice 4 : Compréhension d’algorithme de tri (3 pts)

On applique l’algorithme du tri par sélection au tableau suivant : [7, 2, 5, 1, 9].

4.1 (1 pt) Décrivez en une phrase le principe du tri par sélection.

✅ Principe : rechercher à chaque étape le plus petit élément du sous-tableau non trié et l'échanger avec l'élément en première position de ce sous-tableau.

4.2 (2 pts) Donnez l’état du tableau après la première itération (premier échange) du tri par sélection.

✅ État après 1ʳᵉ itération : [1, 2, 5, 7, 9] (le 1, plus petit élément, est placé en tête).

📌 PARTIE III : MODÉLISATION CONCEPTUELLE ET LOGIQUE (4 points)

Exercice 5 : MCD et transformation (4 pts)

Un lycée souhaite modéliser la gestion des cours. Chaque PROFESSEUR (id_prof, nom, spécialité) assure plusieurs COURS. Un COURS (id_cours, intitulé, coefficient) est assuré par un seul professeur. Les élèves suivent plusieurs cours ; un cours est suivi par plusieurs élèves. Chaque élève (ELEVE : id_eleve, nom, classe) obtient une NOTE (valeur) pour chaque cours suivi.

5.1 (1,5 pt) Identifier les entités et associations du MCD. Donner les cardinalités de l'association entre PROFESSEUR et COURS.

✅ Entités : PROFESSEUR, COURS, ELEVE, NOTE. Association ENSEIGNER entre PROFESSEUR et COURS : cardinalité (1,1) du côté PROFESSEUR et (0,n) du côté COURS (un prof donne 0 à plusieurs cours, un cours a un seul prof).

5.2 (1,5 pt) Dans le passage au MLD, comment gère-t-on l’association MANY-TO-MANY entre ELEVE et COURS ?

✅ On crée une table d’association SUIVRE contenant les clés étrangères (id_eleve, id_cours) et éventuellement l’attribut NOTE.

5.3 (1 pt) Donner la clé primaire de la table NOTE après transformation MLD.

✅ La table NOTE aura une clé primaire composée de (#id_eleve, #id_cours) car un élève ne peut avoir qu’une seule note par cours.
