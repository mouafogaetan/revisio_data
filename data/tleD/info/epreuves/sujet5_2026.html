📐 PARTIE I : MODÉLISATION CONCEPTUELLE ET LOGIQUE (8 points)

📌 Exercice 1 : Entités, associations et cardinalités (4 pts)

Un cabinet médical souhaite modéliser son système d’information. Voici le cahier des charges :
  • Un Médecin est identifié par son code (idMed). On stocke son nom, sa spécialité.
  • Un Patient est identifié par son numéro de sécurité sociale (idPat). On stocke son nom, prénom et date de naissance.
  • Un médecin peut consulter plusieurs patients. Un patient peut être consulté par plusieurs médecins. Une consultation a lieu à une date précise et donne un diagnostic.
1.1 Identifier les entités et les associations dans ce cahier des charges. Donner leurs propriétés (attributs).
1.2 Déterminer les cardinalités de l’association entre MÉDECIN et PATIENT.
1.3 Proposer un schéma MCD (décrire textuellement ou à l’aide d’un formalisme simple).

✅ Corrigé Entités : MÉDECIN (idMed, nom, spécialité) ; PATIENT (idPat, nom, prénom, dateNaissance). Association : CONSULTER (dateConsult, diagnostic).
Cardinalités : (0,n) du côté Médecin — (0,n) du côté Patient (un médecin consulte 0 à n patients, un patient peut être consulté par 0 à n médecins).
MCD textuel : [MÉDECIN]—0,n CONSULTER 0,n—[PATIENT] avec attributs propres à l'association.

📌 Exercice 2 : Transformation en MLD (4 pts)

À partir du MCD de l'exercice 1, appliquez les règles de transformation pour obtenir le Modèle Logique de Données (schéma relationnel).
2.1 Donner la structure des tables (nom des relations, clés primaires, clés étrangères).
2.2 Expliquer comment on gère l’association "CONSULTER" en MLD.
2.3 Écrire le schéma relationnel complet (notation standard).

✅ Corrigé Tables :
- MEDECIN (idMed, nom, spécialité)
- PATIENT (idPat, nom, prénom, dateNaissance)
- CONSULTATION (idMed, idPat, dateConsult, diagnostic)
Clés étrangères : idMed référence MEDECIN, idPat référence PATIENT. L’association "CONSULTER" devient une table à part (relation many-to-many) avec clé primaire composite.

💾 PARTIE II : LANGAGE SQL (12 points)

📌 Exercice 3 : Définition des données (DDL) – 5 pts

On souhaite implanter la base de données du cabinet médical. Écrire les requêtes SQL suivantes :
3.1 Créer la table MEDECIN avec les types appropriés (INT pour idMed, VARCHAR pour nom et spécialité).
3.2 Créer la table PATIENT avec une contrainte de clé primaire sur idPat (type entier).
3.3 Ajouter une colonne telephone (VARCHAR(15)) dans la table PATIENT après la date de naissance.
3.4 Modifier la colonne specialité de la table MEDECIN pour qu’elle soit de type VARCHAR(100) et non nulle.

✅ Requêtes DDL
CREATE TABLE MEDECIN ( idMed INT PRIMARY KEY, nom VARCHAR(50), specialite VARCHAR(50) );
CREATE TABLE PATIENT ( idPat INT PRIMARY KEY, nom VARCHAR(50), prenom VARCHAR(50), dateNaissance DATE );
ALTER TABLE PATIENT ADD telephone VARCHAR(15) AFTER dateNaissance;
ALTER TABLE MEDECIN MODIFY specialite VARCHAR(100) NOT NULL;

📌 Exercice 4 : Manipulation des données (DML) – 3 pts

À partir des tables créées, écrivez les instructions SQL pour :
4.1 Insérer un médecin (id=101, nom="Mbarga", spécialité="Cardiologie").
4.2 Insérer un patient (idPat=5001, nom="Ngo", prenom="Alice", dateNaissance="2005-06-12").
4.3 Modifier le téléphone du patient idPat=5001 pour qu’il devienne "699887755".
4.4 Supprimer toutes les consultations antérieures au 1er janvier 2025 (table CONSULTATION).

✅ Requêtes DML
INSERT INTO MEDECIN VALUES (101, 'Mbarga', 'Cardiologie');
INSERT INTO PATIENT (idPat, nom, prenom, dateNaissance) VALUES (5001, 'Ngo', 'Alice', '2005-06-12');
UPDATE PATIENT SET telephone = '699887755' WHERE idPat = 5001;
DELETE FROM CONSULTATION WHERE dateConsult < '2025-01-01';

📌 Exercice 5 : Interrogation avancée (SELECT / Jointure) – 4 pts

On souhaite extraire des informations de la base. En se basant sur les tables MEDECIN, PATIENT, CONSULTATION, écrire les requêtes :
5.1 Afficher les noms des médecins et les noms des patients qu’ils ont consultés, avec la date de consultation.
5.2 Lister tous les patients (id, nom) qui ont eu une consultation après le 1er mars 2025.
5.3 Compter le nombre total de consultations par médecin (afficher idMed et nombre).
5.4 Afficher le diagnostic de la consultation du patient "Ngo Alice" avec le docteur Mbarga (idMed=101).

✅ Requêtes SELECT
5.1 SELECT m.nom AS medecin, p.nom AS patient, c.dateConsult FROM CONSULTATION c JOIN MEDECIN m ON c.idMed = m.idMed JOIN PATIENT p ON c.idPat = p.idPat;
5.2 SELECT DISTINCT p.idPat, p.nom FROM PATIENT p JOIN CONSULTATION c ON p.idPat = c.idPat WHERE c.dateConsult > '2025-03-01';
5.3 SELECT idMed, COUNT(*) AS nb_consultations FROM CONSULTATION GROUP BY idMed;
5.4 SELECT c.diagnostic FROM CONSULTATION c JOIN PATIENT p ON c.idPat = p.idPat JOIN MEDECIN m ON c.idMed = m.idMed WHERE p.nom = 'Ngo' AND p.prenom = 'Alice' AND m.nom = 'Mbarga';
