Recherche séquentielle (linéaire)

Algorithmique & Programmation · Terminale C & D · Session Bac 2023

📌 Éléments clés à retenir

🔎 Principe général

La recherche séquentielle (ou linéaire) consiste à parcourir un tableau élément par élément, du premier au dernier, jusqu'à trouver la valeur recherchée ou atteindre la fin du tableau. C'est la méthode la plus simple, utilisable même si le tableau n'est pas trié.

  • Complexité : O(n) dans le pire des cas (élément absent ou en dernière position).
  • Condition d'arrêt : élément trouvé OU fin du tableau atteinte.
  • Sortie : position (indice) de l'élément ou un message d'absence (ex: -1).

🚩 La variable drapeau (booléenne)

On utilise souvent une variable booléenne (vrai/faux) appelée « trouvé » ou « flag » pour indiquer si l'élément a été localisé. Cela permet de sortir de la boucle plus tôt et d'éviter de continuer le parcours inutilement.

  • Initialisation : trouve ← Faux
  • Condition dans la boucle : Tant que (non trouve) et (indice ≤ taille)
  • Si l'élément correspond : trouve ← Vrai et mémoriser l'indice.

📝 Structure algorithmique type

Voici le squelette d'un algorithme de recherche séquentielle dans un tableau T de taille N pour une valeur V recherchée. 

Fonction recherche_sequentielle(T, N, V) : entier
   trouve ← Faux
   i ← 1
   Tant que (i ≤ N) et (non trouve) Faire
       Si T[i] = V Alors
           trouve ← Vrai
       Sinon
           i ← i + 1
       Fin Si
   Fin Tant que
   Si trouve Alors
       Retourner i
   Sinon
       Retourner -1
   Fin Si
Fin Fonction

⚡ Optimisation : On peut aussi sortir directement avec RETOURNER i dès que l'élément est trouvé.

💡 Exemples de compréhension

Exemple 1 : Recherche d’un nombre dans une liste de notes

Soit le tableau T = [12, 15, 8, 20, 17] (indices 1 à 5). On recherche la valeur V = 20.

Déroulement pas à pas :
  • i=1 → T[1]=12 ≠ 20 → i=2
  • i=2 → T[2]=15 ≠ 20 → i=3
  • i=3 → T[3]=8 ≠ 20 → i=4
  • i=4 → T[4]=20 = 20 → trouve = Vrai, on sort.
✅ Résultat : L’élément 20 est à l’indice 4.

💡 Astuce : On commence souvent l'indice à 0 dans certains langages (C, Python). En algorithmique, le choix (0 ou 1) doit être clair. Ici nous utilisons 1 pour respecter les sujets du bac Cameroun.

Exemple 2 : Élément absent du tableau

Tableau T = [2, 5, 7, 9, 11] (indices 1 à 5). Rechercher V = 10.

Parcours complet :
i=1 → 2≠10 → i=2
i=2 → 5≠10 → i=3
i=3 → 7≠10 → i=4
i=4 → 9≠10 → i=5
i=5 → 11≠10 → i=6 (dépasse N=5). Condition i≤N fausse, sortie.
❌ Résultat : Valeur non trouvée → retourne -1.

💬 Une fonction qui retourne -1 est une convention courante pour indiquer l'absence.

Exemple 3 : Écriture en langage C (traduction)

Programme C qui demande à l’utilisateur la valeur à chercher dans un tableau d’entiers. 

#include <stdio.h>

int recherche_sequentielle(int T[], int N, int V) {
    for (int i = 0; i < N; i++) {
        if (T[i] == V)
            return i;   // retourne l'indice (0-based)
    }
    return -1;
}

int main() {
    int notes[] = {12, 15, 8, 20, 17};
    int taille = 5;
    int valeur = 20;
    int position = recherche_sequentielle(notes, taille, valeur);
    if (position != -1)
        printf("Valeur %d trouvée à l'indice %d\n", valeur, position);
    else
        printf("Valeur %d non trouvée\n", valeur);
    return 0;
}

Explication : La boucle for parcourt le tableau de 0 à N-1. Dès que la valeur correspond, on renvoie l'indice. Sinon, après la boucle, on renvoie -1.

⚠️ Pièges courants / À ne pas confondre

  • Confondre indice et valeur : On compare la valeur cherchée avec T[i] et non avec i.
  • Oublier la condition d'arrêt sur l'indice : Sans vérifier que i ≤ N, on risque de sortir du tableau → erreur.
  • Boucle infinie : Si on oublie d'incrémenter i dans le cas où l'élément n'est pas trouvé, la boucle ne progresse pas.
  • Décalage des indices : En C, le premier élément est à l'indice 0, tandis qu'en algorithmique (bac), on débute parfois à 1. Soyez cohérent.
  • Recherche dans tableau non trié : La recherche séquentielle fonctionne dans tous les cas, mais elle est moins efficace qu'une recherche dichotomique pour un tableau trié.

Exercice : Gestion des notes d’une classe

On dispose d’un tableau Notes[] contenant les notes sur 20 de 10 élèves. Écrire un algorithme qui recherche la première occurrence de la note 10 et affiche son indice. Si la note n’existe pas, afficher « Note absente ».

📌 Voir une solution possible 
Variables : Notes[10] : tableau d'entiers
           i, trouve : entier
Début
   trouve ← 0
   i ← 1
   Tant que (i ≤ 10) et (trouve = 0) Faire
        Si Notes[i] = 10 Alors
            trouve ← i
        Sinon
            i ← i + 1
        Fin Si
   Fin Tant que
   Si trouve ≠ 0 Alors
        Afficher("Premier 10 trouvé à l'indice ", trouve)
   Sinon
        Afficher("Note absente")
   Fin Si
Fin
💡 Conseil : Testez mentalement avec des cas particuliers (10 en première position, 10 absent, plusieurs 10).
