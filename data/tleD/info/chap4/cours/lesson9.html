Leçon 9 : Instructions conditionnelles en C

Informatique · Terminale C & D (Scientifiques) · Baccalauréat

📌 Éléments clés à retenir

Structure conditionnelle simple : if

Permet d'exécuter un bloc d'instructions uniquement si une condition (expression) est vraie. La condition s'écrit entre parenthèses et utilise des opérateurs relationnels (==, !=, <, >, <=, >=) et logiques (&&, ||, !). 

if (condition) {
    // instructions exécutées si condition vraie
}
  • Les accolades { } sont obligatoires si le bloc contient plus d'une instruction.
  • Si une seule instruction, les accolades peuvent être omises (mais déconseillé pour la clarté).

Alternative double : if ... else

Permet d'orienter le programme vers deux chemins différents : l'un si la condition est vraie, l'autre si elle est fausse. 

if (condition) {
    // bloc vrai
} else {
    // bloc faux
}
  • Correspond à l'alternative simple/double vue en algorithmique.
  • Le else est optionnel, mais permet de traiter le cas contraire.

Alternatives multiples : else if

Pour tester plusieurs conditions successives. Dès qu'une condition est vraie, le bloc correspondant s'exécute, puis on sort de la structure. 

if (condition1) {
    // bloc 1
} else if (condition2) {
    // bloc 2
} else if (condition3) {
    // bloc 3
} else {
    // bloc par défaut (aucune condition vraie)
}
  • Le else final est facultatif, mais souvent utilisé pour capturer tous les autres cas.
  • Important : l'ordre des tests influence le résultat.

Opérateurs relationnels et logiques courants

  • Égalité : == (attention : = est l'affectation !)
  • Différence : !=
  • Comparaisons : < , > , <= , >=
  • ET logique : && → vrai si les deux conditions sont vraies.
  • OU logique : || → vrai si au moins une condition est vraie.
  • NON logique : ! → inverse la valeur booléenne.

💡 Exemples de compréhension

Exemple 1 : Vérification de la parité

Écrire un programme qui lit un entier saisi par l'utilisateur et affiche "Pair" s'il est pair, "Impair" sinon.

Correction en C : 
#include <stdio.h>
int main() {
    int nombre;
    printf("Entrez un entier : ");
    scanf("%d", &nombre);
    if (nombre % 2 == 0) {
        printf("%d est pair.\n", nombre);
    } else {
        printf("%d est impair.\n", nombre);
    }
    return 0;
}
💡 Astuce : L'opérateur modulo % donne le reste de la division. Si nombre % 2 == 0, alors le nombre est divisible par 2.

Exemple 2 : Mention selon une note

Une note sur 20 : si note < 10 → "Insuffisant", entre 10 et 12 → "Passable", entre 12 et 14 → "Assez bien", entre 14 et 16 → "Bien", au-dessus → "Très bien".

Implémentation avec else if : 
if (note < 10) {
    printf("Insuffisant");
} else if (note < 12) {
    printf("Passable");
} else if (note < 14) {
    printf("Assez bien");
} else if (note < 16) {
    printf("Bien");
} else {
    printf("Très bien");
}
💬 L'ordre croissant des tests évite de répéter les bornes inférieures. On exploite le fait que les conditions s'arrêtent à la première vraie.

Exemple 3 : Condition composée (AND / OR)

Un programme vérifie si une année est bissextile : divisible par 4, mais pas par 100 sauf si divisible par 400.

if ((annee % 4 == 0 && annee % 100 != 0) || (annee % 400 == 0)) {
    printf("%d est bissextile", annee);
} else {
    printf("%d n'est pas bissextile", annee);
}
🔍 Utilisez des parenthèses pour grouper les sous-conditions et éviter toute ambiguïté de priorité.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fatale : Confondre = (affectation) et == (comparaison).
    Exemple : if (a = 5) → affecte 5 à a et la condition est toujours vraie (valeur non nulle).
  • Oubli des accolades : Avec if sans accolades, seule la première instruction est conditionnelle. Les instructions suivantes s'exécutent toujours.
  • Ne pas gérer le else : Parfois, on omet volontairement le else, mais il faut penser au cas non traité.
  • Conditions trop complexes : Évitez les conditions imbriquées profondément ; privilégiez else if ou une logique claire.

#include <stdio.h>
int main() {
    int a, b, c, max;
    printf("Entrez trois entiers : ");
    scanf("%d %d %d", &a, &b, &c);
    max = a;
    if (b > max) max = b;
    if (c > max) max = c;
