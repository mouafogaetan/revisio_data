📋 UE 18: Tableaux & Formulaires en HTML

Informatique · Première C, D, E · Module 2 - Programmation HTML

📌 Éléments clés à retenir

1. Structure d'un tableau HTML

Un tableau permet d'organiser des données en lignes et colonnes. Il est construit avec trois balises principales :

  • <table> : conteneur principal du tableau. L'attribut BORDER définit l'épaisseur de la bordure (ex: BORDER="1").
  • <tr> (Table Row) : définit une ligne du tableau.
  • <td> (Table Data) : définit une cellule (colonne) à l'intérieur d'une ligne.
<table BORDER="1">
    <tr>
        <td>Cellule 1</td>
        <td>Cellule 2</td>
    </tr>
</table>

💡 On peut ajouter <th> (Table Header) pour les cellules d'en-tête (gras et centrées par défaut).

2. Fusion de colonnes avec colspan

L'attribut colspan (column span) permet de fusionner plusieurs colonnes en une seule cellule. Il s'applique à <td> ou <th>.

<td colspan="2">Cellule fusionnée sur 2 colonnes</td>

📌 À retenir : Lorsqu'une cellule fusionne plusieurs colonnes, il faut supprimer les cellules suivantes sur la même ligne pour ne pas décaler l'affichage.

3. Formulaires HTML : interagir avec l'utilisateur

Un formulaire permet de collecter des données saisies par l'utilisateur (texte, choix, clics). Balise principale : <form>. À l'intérieur, on place des champs avec <input> et des boutons.

  • <form> : conteneur du formulaire. Attributs courants : action (destination des données), method (get/post).
  • <input> : champ de saisie. L'attribut type définit le genre : texte, mot de passe, email, radio, checkbox, etc.
  • <button> : bouton cliquable. Le texte entre les balises est affiché sur le bouton.
<form method="post">
    Nom : <input type="text" name="nom"><br>
    <button type="submit">Envoyer</button>
</form>

💡 Exemples de compréhension

Exemple 1 : Tableau des notes d'élèves

Créer un tableau HTML avec 3 colonnes (Nom, Mathématiques, Physique) et 2 lignes d'élèves. Ajouter une ligne d'en-tête.

Résolution :

<table BORDER="1">
  <tr>
    <th>Nom</th>
    <th>Maths</th>
    <th>Physique</th>
  </tr>
  <tr>
    <td>Alice</td>
    <td>15</td>
    <td>14</td>
  </tr>
  <tr>
    <td>Brice</td>
    <td>12</td>
    <td>16</td>
  </tr>
</table>
💬 Astuce : Utilisez <th> pour les titres de colonnes. Cela améliore l'accessibilité et le style.

Exemple 2 : Fusion avec colspan

Créer un tableau où la première ligne affiche "Informations personnelles" sur deux colonnes.

Code HTML :

<table BORDER="1">
  <tr>
    <td colspan="2">Informations personnelles</td>
  </tr>
  <tr>
    <td>Nom</td>
    <td>Age</td>
  </tr>
  <tr>
    <td>Dupont</td>
    <td>17</td>
  </tr>
</table>
💡 Notez que sur la ligne avec colspan="2", il n'y a qu'une seule cellule <td> car elle occupe la place de deux colonnes.

Exemple 3 : Formulaire d'inscription simple

Créer un formulaire avec un champ pour le nom, un champ pour l'email, et un bouton "S'inscrire".

Code :

<form action="/traiter" method="post">
  <label>Nom : <input type="text" name="nom" required></label><br>
  <label>Email : <input type="email" name="email"></label><br>
  <button type="submit">S'inscrire</button>
</form>
🔐 L'attribut type="email" vérifie automatiquement le format (présence de @ et d'un domaine). Utile pour la validation côté client.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fréquente : Oublier de fermer une balise <tr> ou <td> → affichage cassé du tableau.
  • Confusion : <td> (cellule normale) vs <th> (cellule d'en-tête). Pour les titres de colonnes, préférez <th>.
  • Attention : Quand on utilise colspan, le nombre total de cellules dans la ligne doit correspondre au nombre de colonnes définies. Si vous fusionnez 2 colonnes, il y aura une cellule de moins sur cette ligne.
  • Piège formulaire : Un bouton sans type="submit" à l'intérieur d'un <form> ne déclenchera pas l'envoi. Toujours préciser type="submit" pour le bouton principal.
  • Oubli : L'attribut name dans les champs <input> est essentiel pour identifier les données lors de l'envoi du formulaire. Sans name, la valeur n'est pas transmise.
