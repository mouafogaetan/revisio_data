⚙️ Exécution d’un programme C

Informatique · 1ère C · Compilation & exécution pas à pas

📌 Étapes clés de la compilation et exécution

🔧 1. Du code source au fichier exécutable

Un programme écrit en C (fichier .c) passe par plusieurs transformations avant de pouvoir être exécuté par la machine.

  • Prétraitement : Le préprocesseur interprète les directives #include, #define, etc. Il génère un code intermédiaire.
  • Compilation : Traduit le code prétraité en langage assembleur (spécifique au processeur).
  • Assemblage : Convertit le code assembleur en code objet (fichier .o ou .obj).
  • Édition des liens (linkage) : Fusionne les différents fichiers objets et les bibliothèques (ex: stdio.h) pour créer l’exécutable final (.exe sous Windows, aucune extension sous Linux/macOS).

📸 Cycle de transformation d’un programme C

💻 2. Utilisation d’un compilateur C (GCC)

GCC (GNU Compiler Collection) est le compilateur le plus utilisé, disponible sous Linux, macOS et Windows (via MinGW ou WSL).

  • Commande de compilation : gcc mon_programme.c -o mon_programme
  • -o : spécifie le nom du fichier exécutable de sortie.
  • Sans -o, l’exécutable s’appelle a.out (Linux/macOS) ou a.exe (Windows).
  • Option -Wall : affiche tous les avertissements (aide à déboguer).
  • Exemple : gcc -Wall calcul.c -o calcul

🚀 3. Exécution du programme

Une fois l’exécutable généré, on peut le lancer depuis le terminal ou l’invité de commandes.

  • Sur Windows (cmd ou PowerShell) : mon_programme.exe ou simplement mon_programme
  • Sur Linux/macOS : ./mon_programme (nécessite les droits d’exécution, souvent déjà attribués).
  • Le système charge l’exécutable en mémoire, le processeur exécute les instructions (de main() jusqu’à return 0).
  • Les éventuels messages printf s’affichent dans la console.

💡 Exemples concrets de compilation / exécution

🧪 Exemple 1 : Compiler et exécuter "Bonjour le monde"

Fichier hello.c :
#include <stdio.h>
int main() {
    printf("Bonjour, Cameroun !\n");
    return 0;
}

Étapes dans le terminal (Linux) :
1. gcc hello.c -o hello → génère l’exécutable hello.
2. ./hello → affiche Bonjour, Cameroun !.
💡 Sur Windows avec MinGW : gcc hello.c -o hello.exe puis hello.exe.

💬 Astuce : Si vous obtenez une erreur "gcc: command not found", c’est que le compilateur n’est pas installé ou pas dans le PATH.

➕ Exemple 2 : Programme avec saisie utilisateur

somme.c : demande deux entiers, calcule et affiche la somme.

Code :
#include <stdio.h>
int main() {
    int a, b;
    printf("Entrez a : "); scanf("%d", &a);
    printf("Entrez b : "); scanf("%d", &b);
    printf("Somme = %d\n", a+b);
    return 0;
}

Compilation : gcc somme.c -o somme
Exécution : ./somme
L’utilisateur interagit, le programme affiche le résultat.

📌 Pensez à vérifier les erreurs de compilation : la moindre faute de syntaxe empêche la création de l’exécutable.

🐞 Détection d’erreurs avec -Wall

Code volontairement incorrect : printf("Valeur : %d", x); mais variable x non déclarée.

Compilation avec avertissements : gcc -Wall essai.c -o essai
Le compilateur affiche : "‘x’ undeclared (first use in this function)". Cela permet de corriger avant exécution.
Toujours compiler avec -Wall pour améliorer la qualité du code.

⚠️ Pièges courants & bonnes pratiques

  • 🔁 Oublier de recompiler après modification : Si vous changez le code source, vous DEVEZ recompiler avant de relancer l’exécutable, sinon les anciennes instructions persistent.
  • 📁 Mauvais répertoire : Le terminal doit se trouver dans le dossier contenant votre fichier .c, ou indiquer le chemin complet.
  • 🔌 Bibliothèques manquantes : Si vous utilisez math.h, ajoutez -lm lors de la compilation : gcc prog.c -o prog -lm.
  • ❌ Confondre compilation et exécution : La compilation produit l’exécutable ; l’exécution lance celui-ci. Sans compilation réussie, pas d’exécution possible.
  • 🧠 Bonne pratique : Toujours nommer l’exécutable avec -o pour éviter les a.out génériques.

📋 Récapitulatif des commandes essentielles

Terminal / invite de commandes

  • Compiler : gcc source.c -o executable
  • Compiler avec tous les warnings : gcc -Wall source.c -o executable
  • Exécuter (Linux/macOS) : ./executable
  • Exécuter (Windows) : executable.exe ou .\executable.exe
  • Nettoyer les fichiers objets : rm *.o (Linux/macOS) ou del *.obj (Windows)
