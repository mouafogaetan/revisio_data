📌 Structures de contrôle en C

En langage C, les structures de contrôle permettent de diriger le flux d'exécution d'un programme. On distingue trois grandes catégories : les structures conditionnelles (choix), les structures itératives (boucles) et les sauts.

🔀 1. Structures conditionnelles (Alternatives)

Permettent d'exécuter un bloc d'instructions seulement si une condition (expression booléenne) est vraie.

  • if (condition) { ... } : Exécute le bloc si la condition est vraie.
  • if (condition) { ... } else { ... } : Exécute le premier bloc si condition vraie, sinon le second.
  • if (condition1) { ... } else if (condition2) { ... } else { ... } : Enchaînement de tests.
  • switch (variable) { case valeur: ... break; default: ... } : Alternative multiple basée sur la valeur d'une expression entière ou caractère.
// Exemple if-else if (note >= 10) { printf("Admis(e)"); } else { printf("Non admis(e)"); } // Exemple switch switch(option) { case 1: printf("Option Maths"); break; case 2: printf("Option Informatique"); break; default: printf("Option non valide"); }

🔄 2. Boucles (Itérations)

Répètent l'exécution d'un bloc tant qu'une condition est vérifiée ou pour un nombre défini de fois.

  • for (initialisation; condition; incrément) { ... } : Utilisée quand le nombre d'itérations est connu à l'avance.
  • while (condition) { ... } : Teste la condition avant chaque itération (0 à N fois).
  • do { ... } while (condition); : Exécute le bloc au moins une fois, puis teste la condition.
// Boucle for : afficher les entiers de 1 à 5 for (int i = 1; i <= 5; i++) { printf("%d ", i); } // Boucle while : somme des 10 premiers entiers int i=1, somme=0; while (i <= 10) { somme += i; i++; } // Boucle do...while : afficher au moins "Hello" int j = 0; do { printf("Hello "); j++; } while (j < 3);

⤴️ 3. Instructions de saut

Modifient le flux normal d'exécution d'une boucle ou d'une fonction.

  • break; : Sort immédiatement de la boucle ou du switch le plus proche.
  • continue; : Passe à l'itération suivante de la boucle (ignore le reste du corps).
  • return expression; : Termine une fonction et retourne éventuellement une valeur.
  • goto label; : Saut inconditionnel (à éviter car rend le code peu lisible).
// Exemple break et continue for (int i = 1; i <= 10; i++) { if (i == 5) break; // sort quand i=5 if (i % 2 == 0) continue; // saute les pairs printf("%d ", i); // affiche 1,3 }

💡 Exemples de traduction Algorithmique → C

Exemple 1 : Condition SI...ALORS...SINON

Algorithme :
Si (note >= 10) Alors
    Afficher("Admis")
Sinon
    Afficher("Échec")
FinSi

Traduction en C :

if (note >= 10) { printf("Admis\n"); } else { printf("Échec\n"); }
💡 Astuce : En C, toute valeur non nulle est considérée comme "vrai" (true) et 0 comme "faux" (false). Les parenthèses autour de la condition sont obligatoires.

Exemple 2 : Boucle POUR (FOR) - Calcul de somme

Algorithme :
somme ← 0
Pour i allant de 1 à 10 pas=1 faire
    somme ← somme + i
FinPour
Afficher(somme)

Traduction en C :

int somme = 0; for (int i = 1; i <= 10; i++) { somme = somme + i; // ou somme += i } printf("Somme = %d", somme);
💡 L'opérateur i++ incrémente i de 1. On peut aussi utiliser i = i + 1. La boucle for est idéale quand le nombre d'itérations est connu.

Exemple 3 : Boucle TANT QUE (WHILE) - Validation saisie

Algorithme :
age ← 0
TantQue (age < 0 ou age > 120) faire
    Afficher("Âge invalide, ressaisir : ")
    Lire(age)
FinTantQue

Traduction en C :

int age = 0; while (age < 0 || age > 120) { printf("Âge invalide, ressaisir : "); scanf("%d", &age); }
💡 La boucle while vérifie la condition avant d'exécuter le bloc. Si la condition est fausse initialement, le bloc n'est jamais exécuté.

Exemple 4 : Boucle REPETER JUSQU'A (do...while)

Algorithme :
Répéter
    Afficher("Menu : 1-Continuer 2-Quitter")
    Lire(choix)
Jusqu'à (choix == 2)

Traduction en C :

int choix; do { printf("Menu : 1-Continuer 2-Quitter\n"); scanf("%d", &choix); } while (choix != 2);
💡 La boucle do...while garantit l'exécution du bloc au moins une fois, même si la condition est fausse initialement. Attention au point-virgule final !

⚠️ Pièges courants & Erreurs fréquentes

  • Confondre = et == : if (x = 5) est une affectation (toujours vraie car 5 non nul). Pour comparer : if (x == 5).
  • Oublier les accolades { } : Sans accolades, seule la première instruction après if/for/while est conditionnée. Toujours utiliser des accolades même pour une seule instruction (bonne pratique).
  • Point-virgule après while ou for : while(i<10); crée une boucle vide infinie. Le point-virgule termine prématurément la structure.
  • Indices hors limites : Dans les boucles, vérifier que l'indice ne dépasse pas la taille du tableau (ex: for(i=0; i<=N; i++) dépasse si tableau taille N).
  • Ne pas initialiser les variables : Une variable locale non initialisée contient une valeur indéterminée (poubelle).
  • Break dans un switch : Oublier break provoque l'exécution des cas suivants (fall-through).
