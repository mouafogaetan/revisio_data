📌 Éléments clés à retenir

🔁 Définition : structure itérative (boucle)

Une structure itérative (ou boucle) est une structure de contrôle qui permet de répéter une séquence d’instructions un certain nombre de fois ou tant qu’une condition est vraie. Elle évite la duplication de code et rend les programmes plus concis et modulables.

  • Utilité : parcourir des tableaux, calculer des sommes, afficher une série, valider une saisie, etc.
  • En algorithmique, trois familles principales : POUR (itération bornée), TANT QUE et RÉPÉTER JUSQU’À (itérations conditionnelles).

1️⃣ Boucle POUR / FOR (nombre d’itérations connu)

On utilise la boucle POUR quand on connaît à l’avance le nombre de répétitions. Un compteur (souvent noté i) prend successivement des valeurs entre une borne inférieure et supérieure.

Syntaxe algorithmique : 
pour i allant de 1 à N pas 1 faire
    instruction(s)
finpour
Syntaxe en langage C : 
for (i = 1; i <= N; i++) {
    instruction(s);
}
  • Le pas peut être différent de 1 (ex : pas = 2 pour aller de 1 à N par bonds de 2).
  • En C, l’indice commence souvent à 0 (pour les tableaux) : for(i=0; i.
  • L’opérateur ++ incrémente la variable de 1 : i++ équivaut à i = i+1.

2️⃣ Boucle TANT QUE / WHILE (condition pré-test)

La boucle TANT QUE répète un bloc d’instructions tant qu’une condition est vraie. La condition est testée avant chaque itération → si la condition est fausse dès le début, la boucle ne s’exécute pas du tout.

Algorithmique : 
tant que (condition) faire
    instruction(s)
fintantque
Langage C : 
while (condition) {
    instruction(s);
}
  • Exemple : tant que (note < 0 ou note > 20) faire lire(note) ; → validation de saisie.
  • Il faut s’assurer que la condition devienne fausse à un moment (sinon boucle infinie).

3️⃣ Boucle RÉPÉTER JUSQU’À / DO...WHILE (condition post-test)

La boucle RÉPÉTER JUSQU’À exécute le bloc d’instructions au moins une fois puis vérifie la condition d’arrêt. On répète jusqu’à ce que la condition devienne vraie.

Algorithmique : 
répéter
    instruction(s)
jusqu'à (condition)
Langage C : 
do {
    instruction(s);
} while (condition);   // attention : on continue tant que condition vraie en C (différence sémantique)
💡 Piège fréquent : En C, la boucle do...while continue tant que la condition est vraie, alors qu’en algorithmique francophone, "jusqu'à" signifie arrêt quand condition vraie. Bien comprendre cette nuance !

💡 Exemples de compréhension

Exemple 1 : Boucle POUR – Somme des entiers de 1 à N

Calculer la somme S = 1 + 2 + 3 + ... + N, où N est saisi par l’utilisateur.

Résolution algorithmique :

Variables : N, i, S : entiers
Début
    Ecrire("Donner N : ")
    Lire(N)
    S ← 0
    pour i allant de 1 à N pas 1 faire
        S ← S + i
    finpour
    Ecrire("Somme = ", S)
Fin

Traduction en langage C : 

#include <stdio.h>
int main() {
    int N, i, S = 0;
    printf("Donner N : ");
    scanf("%d", &N);
    for(i = 1; i <= N; i++) {
        S = S + i;
    }
    printf("Somme = %d\n", S);
    return 0;
}

💬 Astuce : La variable S (accumulateur) doit être initialisée à 0 avant la boucle. L’opérateur += permet d’écrire S += i; plus simplement.

Exemple 2 : Boucle TANT QUE – Validation de saisie

Demander à l’utilisateur une note comprise entre 0 et 20. Tant que la note n’est pas valide, on redemande.

Algorithmique : 

Variables : note : réel
Début
    Ecrire("Entrez une note (0-20) : ")
    Lire(note)
    tant que (note < 0 OU note > 20) faire
        Ecrire("Erreur, recommencez : ")
        Lire(note)
    fintantque
    Ecrire("Note valide : ", note)
Fin
En langage C : 
float note;
printf("Entrez une note (0-20) : ");
scanf("%f", ¬e);
while(note < 0 || note > 20) {
    printf("Erreur, recommencez : ");
    scanf("%f", ¬e);
}
printf("Note valide : %.2f\n", note);

💡 Cette structure garantit qu’on ne stocke jamais une note invalide. La boucle peut s’exécuter zéro fois si la première note est correcte.

Exemple 3 : Boucle RÉPÉTER – Menu simple (exécution au moins une fois)

Afficher un menu et demander le choix de l’utilisateur jusqu’à ce qu’il tape 0 pour quitter.

Algorithmique (RÉPÉTER JUSQU’À) : 

Variables : choix : entier
Début
    répéter
        Ecrire("1. Jouer  2. Scores  0. Quitter")
        Lire(choix)
        Selon choix faire
            // traitement
        finselon
    jusqu'à (choix = 0)
Fin
Langage C (do...while) : 
int choix;
do {
    printf("1. Jouer  2. Scores  0. Quitter\n");
    scanf("%d", &choix);
    // switch(choix) ...
} while(choix != 0);  // on continue tant que choix différent de 0

⚠️ Remarque importante : la condition d’arrêt en algorithmique (jusqu'à choix = 0) devient en C while(choix != 0) car la boucle continue tant que la condition est vraie (ici choix différent de 0).

📈 Boucles et tableaux (notion clé)

Parcourir un tableau avec une boucle POUR

Soit un tableau tabNotes contenant 10 notes (réelles). Afficher chaque note et calculer la moyenne.

En C : 

float tabNotes[10] = {12.5, 14.0, 8.5, 16.0, 10.0, 13.0, 11.5, 9.0, 15.5, 18.0};
float somme = 0.0;
int i;
for(i = 0; i < 10; i++) {
    printf("Note %d : %.2f\n", i+1, tabNotes[i]);
    somme += tabNotes[i];
}
printf("Moyenne = %.2f\n", somme / 10);
Algorithmique : 
pour i allant de 0 à 9 faire
    Ecrire("Note ", i+1, " : ", tabNotes[i])
    somme ← somme + tabNotes[i]
finpour
moyenne ← somme / 10

💬 À retenir : En algorithmique, les indices commencent souvent à 1 ; en langage C, le premier élément est à l'indice 0. Une boucle for(i=0; i est la convention standard.

⚠️ Pièges courants & erreurs à éviter

  • Boucle infinie : oublier d’incrémenter le compteur ou de modifier la condition (ex: while(i < 10) { ... } sans i++).
  • Confusion entre "TANT QUE" et "RÉPÉTER JUSQU'À" : la première peut ne pas s’exécuter, la seconde toujours au moins une fois.
  • Indice de tableau hors limites : en C, un tableau de taille N se parcourt de 0 à N-1. Un accès à tab[N] provoque une erreur.
  • Ne pas initialiser les accumulateurs : la somme, le produit, etc., doivent être initialisés (ex: S=0 pour une somme, P=1 pour un produit).
  • Erreur dans la condition de la boucle POUR : utiliser <= au lieu de < ou inversement selon la borne souhaitée.
  • Oubli du point-virgule ou des accolades en C qui modifie la portée de la boucle.

🧠 Synthèse mnémotechnique

POUR = “Je sais combien de fois je répète” → compteur précis.
TANT QUE = “Vérifie avant d’entrer, possible zéro fois” → pré-test.
RÉPÉTER JUSQU’À = “Exécute au moins une fois, puis vérifie” → post-test.

✍️ Exercice d’application rapide

À vous de jouer !

Écrire un algorithme (puis sa traduction en C) qui affiche la table de multiplication d’un entier N saisi par l’utilisateur (de 1 à 10).

Indice : Utilisez une boucle POUR. Le résultat doit ressembler à :
5 x 1 = 5
5 x 2 = 10
...
5 x 10 = 50

🔍 Voir une solution possible 
Algorithmique :
Variables : N, i : entiers
Début
    Ecrire("Entrez un nombre : ")
    Lire(N)
    pour i allant de 1 à 10 pas 1 faire
        Ecrire(N, " x ", i, " = ", N*i)
    finpour
Fin

Langage C :
#include <stdio.h>
int main() {
    int N, i;
    printf("Entrez un nombre : ");
    scanf("%d", &N);
    for(i = 1; i <= 10; i++) {
        printf("%d x %d = %d\n", N, i, N*i);
    }
    return 0;
}
