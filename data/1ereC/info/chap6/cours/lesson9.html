📀 Leçon 9 : Généralités sur les bases de données

Informatique · Première C, D, E (Cameroun) · Module 1 - Unité d’apprentissage 6

📌 Éléments clés à retenir

🗄️ Base de données (BD) : définition

Une base de données est un ensemble structuré de données stockées de façon organisée, permettant de conserver, consulter, modifier et gérer efficacement des informations.

  • Elle évite la redondance (données dupliquées) et garantit la cohérence.
  • Exemple : la base de données d’un lycée contient les élèves, les professeurs, les notes, les classes.
  • Les données sont indépendantes des programmes qui les utilisent.

⚙️ SGBD : le moteur de la base

Un Système de Gestion de Base de Données (SGBD) est un logiciel qui permet de créer, manipuler et administrer une base de données. Il sert d’interface entre les utilisateurs (ou applications) et les données stockées.

  • Exemples de SGBD : MySQL, PostgreSQL, Oracle, Microsoft Access, SQLite.
  • Fonctions principales : ajout, modification, suppression, recherche, sécurisation des accès, gestion des sauvegardes.
  • Un SGBD garantit les propriétés ACID (Atomicité, Cohérence, Isolation, Durabilité).
📁 Utilisateur ⇄ SGBD ⇄ Base de données
👨‍💻 Application → 🛢️ SGBD (MySQL) → 💾 Fichiers .ibd / .mdf

📊 Organisation : tables, lignes, colonnes

Dans une base de données relationnelle, les données sont organisées en tables (comme des classeurs Excel). Chaque table contient des lignes (enregistrements) et des colonnes (champs).

  • Champ (colonne) : une catégorie d’information (ex: Nom, Prénom, Date de naissance).
  • Enregistrement (ligne) : un ensemble de valeurs correspondant à un élément unique (ex: les infos d’un élève).
  • Clé primaire (Primary Key) : champ (ou combinaison) qui identifie chaque enregistrement de façon unique. Ne peut pas être nul.
📌 ID_Élève (clé primaire) Nom Prénom Classe

102FoudaKevinPremière D

🔑 Clé primaire : la colonne ID_Élève est unique pour chaque ligne (101, 102...). On ne peut pas avoir deux fois le même ID.

✅ Pourquoi utiliser une base de données ?

  • Centralisation : les données sont regroupées au même endroit.
  • Réduction des redondances : on évite de stocker plusieurs fois la même information.
  • Sécurité : contrôle d’accès (mots de passe, droits utilisateurs).
  • Intégrité des données : respect de contraintes (ex: âge positif, unicité d’un email).
  • Interrogation puissante : avec le langage SQL (SELECT, INSERT, UPDATE, DELETE).
  • Indépendance : les données persistent même si l’application est fermée.

💡 Exemples de compréhension

🏫 Exemple 1 : Base de données d’un collège

Dans un établissement scolaire, on souhaite stocker les informations suivantes : élèves (matricule, nom, prénom, date de naissance, classe) et professeurs (id_prof, nom, matière, téléphone).

🔍 Analyse : On crée deux tables distinctes : Élèves et Professeurs. La table Élèves utilise matricule comme clé primaire (chaque élève a un matricule unique). La table Professeurs utilise id_prof comme clé primaire. Les deux tables sont indépendantes mais peuvent être reliées par une troisième table Cours (non détaillée ici).

💬 Astuce : Toujours chercher l’identifiant unique (naturel ou créé) pour chaque ligne. Un nom ne suffit jamais (plusieurs élèves peuvent s’appeler "Marie").

🛒 Exemple 2 : Base de données d’un site e-commerce

Un site de vente en ligne gère des produits, des clients et des commandes. Pour chaque produit, on a : référence, désignation, prix, stock.

✅ Solution : Table Produits avec la colonne référence comme clé primaire (valeur unique : "P102", "P203"...). Le champ prix doit être un nombre décimal positif ; le stock un entier positif (contrainte d’intégrité).

📦 Produits (référence PK, désignation, prix, stock) ← lié à → 🧾 Commandes (num_cmd PK, date, client_id)
💡 À retenir : Les bases de données évitent la saisie redondante. Dans une commande, on stocke seulement la référence du produit, pas toutes ses caractéristiques.

📝 Exercice-type (examen)

On donne la table suivante nommée Employés :
| ID (PK) | Nom | Salaire | Service |
| 1 | Manga | 250 000 | Informatique |
| 2 | Njikam | 300 000 | Comptabilité |
| 3 | Atangana | 275 000 | Informatique |

Questions :
1. Quel est le champ clé primaire ? ID (car chaque valeur est différente et non nulle).
2. Combien d’enregistrements contient la table ? 3 enregistrements (chaque ligne).
3. Citer deux champs de la table : Salaire, Service.
4. Proposer une contrainte d’intégrité : Salaire > 0 et Service non vide.

⚠️ Pièges courants & erreurs fréquentes

  • Confondre base de données et SGBD : La base de données est le contenu (les données), le SGBD est le logiciel qui la gère (MySQL, Access…).
  • Négliger la clé primaire : Chaque table doit idéalement avoir une clé primaire ; sans elle, on risque des doublons.
  • Stockage d’informations calculées : Dans une bonne base, on évite de stocker l’âge (qui change chaque année) ; on stocke la date de naissance.
  • Redondance incontrôlée : Exemple : stocker le nom du service dans chaque ligne d’employés au lieu d’une table séparée "Services".
  • Ne pas respecter le type de données : Mettre du texte dans une colonne prévue pour des nombres (ex: "dix" au lieu de 10).

📖 Glossaire express (à mémoriser)

  • Base de données (BD) : ensemble structuré de données.
  • SGBD : Système de Gestion de Base de Données (logiciel).
  • Table : structure en lignes et colonnes.
  • Champ (colonne) : propriété ou attribut (ex: "Nom").
  • Enregistrement (ligne, tuple) : instance concrète (ex: "Durand, Paul").
  • Clé primaire (Primary Key) : identifiant unique d’un enregistrement.
  • Contrainte d’intégrité : règle assurant la cohérence des données (ex: "note comprise entre 0 et 20").
