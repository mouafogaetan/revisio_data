📀 Leçon 19 : Concepts fondamentaux des bases de données

Informatique · Terminale A (Littéraire) · Unité 5 : Bases de données

📌 Éléments clés à retenir

🗄️ 1. Qu'est-ce qu'une base de données (BDD) ?

Une base de données est une collection structurée et organisée de données qui permet de les stocker, de les gérer et de les interroger facilement. Elle sert à conserver des informations de manière durable et efficace, comme un classeur numérique géant. On utilise un SGBD (Système de Gestion de Base de Données) pour la créer, la manipuler et la sécuriser.

  • Exemples de SGBD : Microsoft Access, MySQL, LibreOffice Base, Oracle, PostgreSQL.
  • Utilité : Organiser les données (élèves, notes, livres, clients) pour les retrouver rapidement.

📋 2. Table, enregistrement et champ

Une base de données est composée de tables. Une table ressemble à un tableau à double entrée :

  • Table : Ensemble de données sur un même thème (ex: table "Élèves", table "Livres").
  • Enregistrement (ligne) : Correspond à une entrée complète (ex: les infos d'un seul élève).
  • Champ (colonne) : Une catégorie d'information (ex: Nom, Prénom, Âge).
💡 Astuce : Imagine un tableau Excel classique : les lignes = enregistrements, les colonnes = champs. La base de données étend cette logique avec plus de puissance.

🔑 3. Clé primaire (Primary Key)

La clé primaire est un champ (ou une combinaison de champs) qui identifie de manière unique chaque enregistrement dans une table.
Deux enregistrements ne peuvent jamais avoir la même valeur de clé primaire. C’est l’identifiant unique, comme un numéro de carte d’identité.

  • Exemple courant : `id_élève`, `numéro_inscription`, `code_produit`.
  • Règle : Une clé primaire ne peut pas être vide (NULL) et doit toujours être unique.
  • Symbole dans les schémas : souvent une petite clé 🔑 à côté du champ.

💡 Exemples de compréhension

📊 Exemple 1 : Table "Étudiants"

Une petite école souhaite stocker les informations de ses élèves dans une base de données. Voici un extrait de la table `Étudiants` :

id_etudiant (Clé primaire)nomprenomclassemoyenne
101MbargaJulieTerm A15.5
102NgoChristianTerm A12.0
103FoudaAmiraTerm A17.2
104DjibrilKarimTerm A14.0

🔍 Analyse :
• Chaque ligne est un enregistrement (un étudiant différent).
• Les colonnes `id_etudiant`, `nom`, `prenom`, `classe`, `moyenne` sont des champs.
• Le champ `id_etudiant` est la clé primaire car chaque étudiant possède un numéro unique (101, 102…). Aucun doublon n'est possible.

💬 Astuce : Si on voulait identifier un étudiant de manière sûre, on utiliserait son `id_etudiant` plutôt que son nom (car deux élèves peuvent s'appeler "Mbarga Julie").

📚 Exemple 2 : Table "Livres" d'une bibliothèque

Une bibliothèque municipale gère son catalogue avec une base de données. Voici un extrait de la table `Livres` :

ISBN (Clé primaire)titreauteurannée_parutiondisponible
978-2-1234-5678-9Le mystère de l'ordinateurAlice Ngo2021Oui
978-0-345-67890-1Introduction aux bases de donnéesJean Fouda2020Non

📖 Explication : ISBN (numéro international de livre) est un excellent choix de clé primaire car chaque livre a un ISBN différent. Les champs `titre`, `auteur`, `année_parution` et `disponible` décrivent chaque enregistrement.

✨ À retenir : Le choix d’une bonne clé primaire est crucial pour éviter les doublons. Les numéros automatiques (comme `id_etudiant`) sont souvent utilisés.

⚠️ Pièges courants & confusions à éviter

  • Confondre table et base de données : Une base de données contient plusieurs tables. Une table n’est qu’un composant.
  • Oublier l’unicité de la clé primaire : Jamais deux lignes avec la même clé primaire. Exemple : deux élèves ne peuvent pas avoir le même `id_etudiant`.
  • Penser que la clé primaire est obligatoirement un nombre : Elle peut être du texte (comme un code ISBN) tant qu’elle est unique.
  • Confondre SGBD et base de données : Le SGBD est le logiciel (Access, MySQL) qui permet de gérer la base. La base est le contenu (les données).

🎯 Entraînement Baccalauréat

Question type examen (session 2023/2024)

On vous donne la table suivante nommée `Employés` :
| id_employé | nom_complet | service | salaire | |------------|-------------|---------|---------| | E01 | Paul BIA | Compta | 250 000 | | E02 | Rose DIKO | RH | 300 000 | | E03 | Luc OWONO | Info | 275 000 |

1. Quel est le nom de la table ? → Employés
2. Combien d’enregistrements contient cette table ? → 3 enregistrements
3. Citez trois champs de cette table. → id_employé, nom_complet, service, salaire (3 parmi 4)
4. Quel champ constitue la clé primaire ? Justifiez. → id_employé, car chaque valeur est unique (E01, E02, E03).

📌 Conseil : Lisez bien les questions sur la structure. En examen, on peut aussi demander : "Donnez un exemple de SGBD" (Access, MySQL).

📖 Récapitulatif rapide :

Base de données = ensemble de tables organisées
Table = structure en lignes (enregistrements) et colonnes (champs)
Clé primaire = champ unique pour identifier chaque ligne. Ex : id, code, numéro.
