📌 Leçon 24 : La requête DELETE

Informatique · Terminale A (Littéraire) · Unité 5 : Bases de données / SQL

📌 Éléments clés à retenir

🗑️ DELETE : Supprimer des enregistrements

La commande DELETE fait partie du langage SQL (Structured Query Language). Elle permet de supprimer une ou plusieurs lignes (enregistrements) d'une table d'une base de données. C'est une opération irréversible si l'on ne dispose pas de sauvegarde : il faut l'utiliser avec précaution.

  • Syntaxe générale : DELETE FROM nom_table WHERE condition ;
  • WHERE est crucial : sans condition, tous les enregistrements de la table sont supprimés !
  • On peut supprimer une seule ligne, plusieurs lignes répondant à un critère, ou toutes les lignes (DELETE sans WHERE).
  • La commande agit sur le contenu des données, pas sur la structure de la table (celle-ci reste intacte).
💡 À retenir : DELETE ne détruit que les données, pas la table elle-même. Pour supprimer toute la table (structure + données), on utiliserait DROP TABLE.

🎯 L'importance de la clause WHERE

La clause WHERE filtre les enregistrements à supprimer. Si on l'oublie, la requête supprime intégralement toutes les lignes de la table.

  • DELETE FROM Eleves WHERE id_eleve = 12; → Supprime uniquement l'élève dont l'identifiant est 12.
  • DELETE FROM Eleves WHERE classe = 'TermA'; → Supprime tous les élèves de la classe Terminale A.
  • DELETE FROM Eleves; → ⚠️ Vide complètement la table "Eleves".

En environnement professionnel, on effectue souvent un SELECT avec les mêmes conditions avant un DELETE pour vérifier quelles lignes seront impactées.

🔄 DELETE vs TRUNCATE vs DROP

Il est essentiel de ne pas confondre DELETE avec d'autres commandes SQL qui agissent sur les tables.

  • DELETE : supprime des lignes une par une, peut utiliser WHERE, plus lent sur de gros volumes mais transactionnel (peut être annulé dans une transaction).
  • TRUNCATE : vide rapidement toutes les lignes d'une table, ne peut pas utiliser WHERE, réinitialise les compteurs d'auto-incrémentation.
  • DROP TABLE : supprime complètement la table (structure + données).

Dans le cadre du programme de Terminale A, on se concentre sur DELETE avec ou sans condition.

💡 Exemples de compréhension

Exemple 1 – Supprimer un livre spécifique dans une bibliothèque

📚 Table `Livres` :
| id_livre | titre | auteur | année | |----------|--------------------------|-----------------|-------| | 1 | Le Cid | Corneille | 1637 | | 2 | Les Misérables | Victor Hugo | 1862 | | 3 | La Peste | Albert Camus | 1947 |
Question : Écrire la requête SQL qui supprime le livre "Les Misérables".

✅ Requête DELETE : 
DELETE FROM Livres WHERE titre = 'Les Misérables';
Explication : La condition titre = 'Les Misérables' cible uniquement l'enregistrement correspondant. Après exécution, la table ne contiendra plus que les livres "Le Cid" et "La Peste".
💬 Astuce : Pour plus de sécurité, on peut utiliser la clé primaire (id_livre) qui est unique : DELETE FROM Livres WHERE id_livre = 2;.

Exemple 2 – Supprimer tous les élèves d'une classe

🏫 Table `Eleves` : colonnes (id, nom, prenom, classe). On souhaite supprimer tous les élèves de la classe "Terminale A2" qui ont quitté l'établissement.

📝 Requête : 
DELETE FROM Eleves WHERE classe = 'Terminale A2';

Cette requête supprime tous les enregistrements où la colonne `classe` vaut 'Terminale A2'. Si la table contient 15 élèves dans cette classe, ils seront tous supprimés en une seule commande.

🔍 Vérification préalable (bonne pratique) : on peut d'abord lister les lignes concernées avec SELECT * FROM Eleves WHERE classe = 'Terminale A2'; pour confirmer.

⚠️ Exemple d'erreur fréquente – Suppression totale accidentelle

Un stagiaire saisit la requête suivante dans la base de données d'une association : DELETE FROM Adherents ; (sans clause WHERE).

💥 Conséquence : Tous les adhérents sont supprimés de la table. Sans sauvegarde récente, les données sont perdues définitivement.

✅ Règle d'or : Avant d'exécuter un DELETE sur une table réelle, toujours s'assurer que la condition WHERE est correcte. En environnement de travail, on utilise souvent des transactions (BEGIN TRANSACTION; DELETE ...; ROLLBACK; pour tester).
🛡️ Prévention : dans un SGBD comme Access ou MySQL Workbench, on peut désactiver l'exécution automatique des DELETE sans WHERE via les paramètres de sécurité.

⚠️ Pièges courants / À ne pas confondre

  • Erreur fatale : Oublier la clause WHERE → suppression complète de la table. Relire toujours sa requête avant exécution.
  • Confusion DELETE / DROP : DELETE supprime des lignes, DROP TABLE supprime toute la table (structure + données). Une table supprimée avec DROP n'existe plus.
  • Confusion avec TRUNCATE : TRUNCATE ne peut pas utiliser de condition et est plus radical (réinitialise l'auto-incrément). En examen, on attend DELETE avec ou sans condition.
  • Problème d'intégrité référentielle : Si une table est liée par une clé étrangère, on ne peut pas supprimer un enregistrement référencé ailleurs (ex: supprimer un client qui a des commandes). Il faut d'abord supprimer les commandes ou utiliser ON DELETE CASCADE.
  • Oubli des guillemets : Pour les chaînes de caractères dans WHERE, ne pas oublier les quotes simples (ex: WHERE ville = 'Douala').
